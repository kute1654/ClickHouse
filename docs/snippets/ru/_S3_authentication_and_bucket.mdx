<Accordion title="Создайте S3 бакеты и пользователя IAM">
  В этой статье описаны основы настройки пользователя AWS IAM, создания S3 бакета и настройки ClickHouse для использования бакета в качестве S3-диска.
  Рекомендуется согласовать с командой безопасности необходимые разрешения, рассматривая приведённые здесь настройки как отправную точку.

  ### Создание пользователя AWS IAM

  В следующих шагах вы создадите пользователя сервисного аккаунта (не пользователя для авторизации).

  1. Войдите в консоль управления IAM в AWS.

  2. В меню `Users` выберите пункт `Create user`

  <Image size="md" img="/images/_snippets/s3/s3-1.webp" alt="Консоль управления AWS IAM — добавление нового пользователя" border force />

  3. Введите имя пользователя, установите тип учетных данных `Access key - Programmatic access` и выберите `Next: Permissions`

  <Image size="md" img="/images/_snippets/s3/s3-2.webp" alt="Настройка имени пользователя и типа доступа для IAM-пользователя" border force />

  4. Не добавляйте пользователя ни в одну группу; выберите `Next: Tags`

  <Image size="md" img="/images/_snippets/s3/s3-3.webp" alt="Назначение группы для пользователя IAM пропускается" border force />

  5. Если добавлять теги не нужно, выберите `Next: Review`

  <Image size="md" img="/images/_snippets/s3/s3-4.webp" alt="Пропускается присвоение тега пользователю IAM" border force />

  6. Выберите `Create User`

  <Note>
    Предупреждение о том, что у пользователя нет разрешений, можно проигнорировать; в следующем разделе пользователю будут выданы разрешения на бакет
  </Note>

  <Image size="md" img="/images/_snippets/s3/s3-5.webp" alt="Предупреждение при создании пользователя IAM без разрешений" border force />

  7. Пользователь уже создан; нажмите `show` и скопируйте ключ доступа и секретный ключ.

  <Note>
    Сохраните ключи в другом месте; секретный ключ доступа можно увидеть только сейчас.
  </Note>

  <Image size="md" img="/images/_snippets/s3/s3-6.webp" alt="Просмотр и копирование ключей доступа IAM-пользователя" border force />

  8. Нажмите «Закрыть», затем найдите пользователя на странице пользователей.

  <Image size="md" img="/images/_snippets/s3/s3-7.webp" alt="Поиск только что созданного пользователя IAM в списке пользователей" border force />

  9. Скопируйте ARN (Amazon Resource Name) и сохраните его — он понадобится при настройке политики доступа к бакету.

  <Image size="md" img="/images/_snippets/s3/s3-8.webp" alt="Копирование ARN IAM-пользователя" border force />

  ### Создание S3 бакета

  1. В разделе S3 бакета выберите `Create bucket`

  <Image size="md" img="/images/_snippets/s3/s3-9.webp" alt="Начало процесса создания S3 бакета" border force />

  2. Введите имя бакета, остальные параметры оставьте по умолчанию

  <Note>
    Имя бакета должно быть уникальным во всём AWS, а не только в пределах организации, иначе возникнет ошибка.
  </Note>

  3. Оставьте `Block all Public Access` включённым; публичный доступ не требуется.

  <Image size="md" img="/images/_snippets/s3/s3-a.webp" alt="Настройка параметров S3 бакета при отключённом публичном доступе" border force />

  4. Выберите `Create Bucket` внизу страницы

  <Image size="md" img="/images/_snippets/s3/s3-b.webp" alt="Завершение создания S3 бакета" border force />

  5. Выберите ссылку, скопируйте ARN и сохраните его — он понадобится при настройке политики доступа к бакету.

  6. После создания бакета найдите новый S3 бакет в списке S3 бакетов и перейдите по ссылке

  <Image size="md" img="/images/_snippets/s3/s3-c.webp" alt="Поиск только что созданного S3 бакета в списке бакетов" border force />

  7. Выберите `Создать папку`

  <Image size="md" img="/images/_snippets/s3/s3-d.webp" alt="Создание новой папки в S3 бакете" border force />

  8. Введите имя папки, которая будет использоваться как целевая папка для диска S3 в ClickHouse, и выберите `Create folder`

  <Image size="md" img="/images/_snippets/s3/s3-e.webp" alt="Настройка имени папки для использования S3-диска в ClickHouse" border force />

  9. Папка теперь должна отображаться в списке бакетов

  <Image size="md" img="/images/_snippets/s3/s3-f.webp" alt="Просмотр новой папки в S3 бакете" border force />

  10. Установите флажок для новой папки и нажмите `Copy URL`. Сохраните скопированный URL, чтобы использовать его в конфигурации хранилища ClickHouse в следующем разделе.

  <Image size="md" img="/images/_snippets/s3/s3-g.webp" alt="Копирование URL-адреса папки S3 для конфигурации ClickHouse" border force />

  11. Выберите вкладку `Permissions` и нажмите на кнопку `Edit` в разделе `Bucket Policy`

  <Image size="md" img="/images/_snippets/s3/s3-h.webp" alt="Доступ к конфигурации политики S3-бакета" border force />

  12. Добавьте политику бакета, пример ниже:

  ```json
  {
    "Version" : "2012-10-17",
    "Id" : "Policy123456",
    "Statement" : [
      {
        "Sid" : "abc123",
        "Effect" : "Allow",
        "Principal" : {
          "AWS" : "arn:aws:iam::921234567898:user/mars-s3-user"
        },
        "Action" : "s3:*",
        "Resource" : [
          "arn:aws:s3:::mars-doc-test",
          "arn:aws:s3:::mars-doc-test/*"
        ]
      }
    ]
  }
  ```

  ```response
  |Параметр | Описание | Пример значения |
  |----------|-------------|----------------|
  |Version | Версия интерпретатора политики, оставьте без изменений | 2012-10-17 |
  |Sid | Идентификатор политики, задаётся пользователем | abc123 |
  |Effect | Определяет, будут ли запросы пользователя разрешены или запрещены | Allow |
  |Principal | Учётные записи или пользователь, которым будет предоставлен доступ | arn:aws:iam::921234567898:user/mars-s3-user |
  |Action | Какие операции разрешены для бакета| s3:*|
  |Resource | Для каких ресурсов в бакете будут разрешены операции | "arn:aws:s3:::mars-doc-test", "arn:aws:s3:::mars-doc-test/*" |
  ```

  <Note>
    Вам следует вместе с вашей командой по безопасности определить, какие разрешения использовать; приведённые ниже можно рассматривать как отправную точку.
    Дополнительную информацию о политиках и настройках см. в документации AWS:
    https://docs.aws.amazon.com/AmazonS3/latest/userguide/access-policy-language-overview.html
  </Note>

  13. Сохраните конфигурацию политики.
</Accordion>
