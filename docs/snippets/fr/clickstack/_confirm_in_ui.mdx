---
title: Confirmer dans l’UI
---

Ouvrez votre service dans la [ClickHouse Cloud console](https://console.clickhouse.cloud), sélectionnez **ClickStack** dans le menu de gauche, puis **Start Ingestion**.

<Image img="/images/use-cases/observability/clickstack-cloud-first-time.webp" size="lg" alt="Lancer ClickStack" border />

L’étape suivante peut être ignorée, car vous avez déjà configuré votre collector. Cliquez sur **Launch ClickStack** pour continuer.

ClickStack s’ouvrira dans un nouvel onglet et vous devriez être automatiquement redirigé vers la page **Getting Started**. Sinon, sélectionnez **Getting Started** dans le menu de gauche, puis cliquez sur **Start Ingestion**, puis sur **Next**.

<Image img="/images/use-cases/observability/clickstack-start-ingestion.webp" size="lg" alt="ClickStack Start Ingestion" border />

Vous arriverez sur la page **Getting Started**. Si l’on vous invite à démarrer l’ingestion, parcourez les écrans d’onboarding qui vous guident pour lancer un collector Docker. Comme vous avez déjà effectué cette étape, vous pouvez simplement cliquer jusqu’au bout. ClickStack détectera automatiquement vos données de télémétrie et vos tables, après quoi vous pourrez sélectionner **Start Exploring** pour commencer à utiliser la plateforme.

<Image img="/images/use-cases/observability/clickstack-start-exploring.webp" size="lg" alt="ClickStack Start Exploring" border />

Basculez la source sur `Logs` et réglez l’intervalle de temps sur **Last 15 minutes**. Les logs synthétiques de `telemetrygen` devraient apparaître en quelques secondes.

<Image img="/images/use-cases/observability/clickstack-search.webp" size="lg" alt="Vue Search de ClickStack avec des logs qui apparaissent" />

Si rien n’apparaît :

* Vérifiez que la valeur de l’en-tête d’authentification transmise à `telemetrygen` correspond à celle attendue par votre collector.
* Consultez les logs de votre collector et recherchez d’éventuelles erreurs d’export.
* Vérifiez que l’endpoint ClickHouse configuré sur le collector inclut bien le protocole et le port (`https://...:8443`).
