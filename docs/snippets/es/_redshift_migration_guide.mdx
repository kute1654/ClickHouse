<div id="introduction-introduction">
  ## Introducción [#introduction]
</div>

[Amazon Redshift](https://aws.amazon.com/redshift/) es una popular solución de almacenamiento de datos en la nube que forma parte de la oferta de Amazon Web Services. Esta guía presenta distintos enfoques para migrar datos desde una instancia de Redshift a ClickHouse. Cubriremos tres opciones:

<img src="/images/integrations/data-ingestion/redshift/redshift-to-clickhouse.webp" alt="Opciones de migración de Redshift a ClickHouse" />

Desde la perspectiva de la instancia de ClickHouse, puede:

1. **[PUSH](#push-data-from-redshift-to-clickhouse)** enviar datos a ClickHouse mediante una herramienta o servicio ETL/ELT de terceros

2. **[PULL](#pull-data-from-redshift-to-clickhouse)** extraer datos de Redshift mediante ClickHouse JDBC Bridge

3. **[PIVOT](#pivot-data-from-redshift-to-clickhouse-using-s3)** usar almacenamiento de objetos S3 con una lógica de &quot;descargar y luego cargar&quot;

<Note>
  En este tutorial usamos Redshift como fuente de datos. Sin embargo, los enfoques de migración que se presentan aquí no son exclusivos de Redshift, y se pueden seguir pasos similares con cualquier fuente de datos compatible.
</Note>

<div id="push-data-from-redshift-to-clickhouse-push-data-from-redshift-to-clickhouse">
  ## Enviar datos de Redshift a ClickHouse [#push-data-from-redshift-to-clickhouse]
</div>

En este escenario, la idea es aprovechar una herramienta o servicio de terceros (ya sea código personalizado o un [ETL/ELT](https://en.wikipedia.org/wiki/Extract,_transform,_load#ETL_vs._ELT)) para enviar tus datos a tu instancia de ClickHouse. Por ejemplo, puedes usar software como [Airbyte](https://www.airbyte.com/) para mover datos entre tu instancia de Redshift (como origen) y ClickHouse como destino ([consulta nuestra guía de integración de Airbyte](/es/integrations/connectors/data-ingestion/etl-tools/airbyte-and-clickhouse))

<img src="/images/integrations/data-ingestion/redshift/push.webp" alt="Push de Redshift a ClickHouse" />

<div id="pros-pros">
  ### Ventajas [#pros]
</div>

* Puede aprovechar el catálogo existente de conectores del software ETL/ELT.
* Incluye capacidades integradas para mantener los datos sincronizados (lógica de anexado/sobrescritura/incremental).
* Permite escenarios de transformación de datos (por ejemplo, consulta nuestra [guía de integración de dbt](/es/integrations/connectors/data-ingestion/etl-tools/dbt/index)).

<div id="cons-cons">
  ### Desventajas [#cons]
</div>

* Los usuarios deben configurar y mantener una infraestructura de ETL/ELT.
* Introduce un elemento de terceros en la arquitectura, lo que puede convertirse en un posible cuello de botella de escalabilidad.

<div id="pull-data-from-redshift-to-clickhouse-pull-data-from-redshift-to-clickhouse">
  ## Extraer datos de Redshift a ClickHouse [#pull-data-from-redshift-to-clickhouse]
</div>

En este escenario, la idea es usar ClickHouse JDBC Bridge para conectarse directamente a un clúster de Redshift desde una instancia de ClickHouse y ejecutar consultas `INSERT INTO ... SELECT`:

<img src="/images/integrations/data-ingestion/redshift/pull.webp" alt="PULL de Redshift a ClickHouse" />

<div id="pros-pros-1">
  ### Ventajas [#pros-1]
</div>

* Genérico para todas las herramientas compatibles con JDBC
* Solución elegante para poder consultar múltiples fuentes de datos externas desde ClickHouse

<div id="cons-cons-1">
  ### Desventajas [#cons-1]
</div>

* Requiere una instancia de ClickHouse JDBC Bridge, que puede convertirse en un cuello de botella para la escalabilidad

<Note>
  Aunque Redshift se basa en PostgreSQL, no es posible usar la función de tabla ni el motor de tabla de PostgreSQL de ClickHouse, ya que ClickHouse requiere PostgreSQL versión 9 o superior y la API de Redshift se basa en una versión anterior (8.x).
</Note>

<div id="tutorial-tutorial">
  ### Tutorial [#tutorial]
</div>

Para usar esta opción, debe configurar ClickHouse JDBC Bridge. ClickHouse JDBC Bridge es una aplicación Java independiente que gestiona la conectividad JDBC y actúa como proxy entre la instancia de ClickHouse y los orígenes de datos. Para este tutorial, usamos una instancia de Redshift precargada con una [base de datos de ejemplo](https://docs.aws.amazon.com/redshift/latest/dg/c_sampledb.html).

<Steps>
  <Step>
    #### Implementar ClickHouse JDBC Bridge [#deploy-clickhouse-jdbc-bridge]

    Implemente ClickHouse JDBC Bridge. Para obtener más información, consulte nuestra guía de usuario sobre [JDBC para orígenes de datos externos](/es/integrations/connectors/data-ingestion/jdbc-with-clickhouse)

    <Note>
      Si usa ClickHouse Cloud, deberá ejecutar ClickHouse JDBC Bridge en un entorno independiente y conectarse a ClickHouse Cloud mediante la función [remoteSecure](/es/reference/functions/table-functions/remote)
    </Note>
  </Step>

  <Step>
    #### Configure su origen de datos de Redshift [#configure-your-redshift-datasource]

    Configure su origen de datos de Redshift para ClickHouse JDBC Bridge. Por ejemplo, `/etc/clickhouse-jdbc-bridge/config/datasources/redshift.json `

    ```json
    {
     "redshift-server": {
       "aliases": [
         "redshift"
       ],
       "driverUrls": [
       "https://s3.amazonaws.com/redshift-downloads/drivers/jdbc/2.1.0.4/redshift-jdbc42-2.1.0.4.jar"
       ],
       "driverClassName": "com.amazon.redshift.jdbc.Driver",
       "jdbcUrl": "jdbc:redshift://redshift-cluster-1.ckubnplpz1uv.us-east-1.redshift.amazonaws.com:5439/dev",
       "username": "awsuser",
       "password": "<password>",
       "maximumPoolSize": 5
     }
    }
    ```
  </Step>

  <Step>
    #### Consultar su instancia de Redshift desde ClickHouse [#query-your-redshift-instance-from-clickhouse]

    Una vez que ClickHouse JDBC Bridge esté implementado y en ejecución, puede empezar a consultar su instancia de Redshift desde ClickHouse.

    ```sql
    SELECT *
    FROM jdbc('redshift', 'select username, firstname, lastname from users limit 5')
    ```

    ```response
    Query id: 1b7de211-c0f6-4117-86a2-276484f9f4c0

    ┌─username─┬─firstname─┬─lastname─┐
    │ PGL08LJI │ Vladimir  │ Humphrey │
    │ XDZ38RDD │ Barry     │ Roy      │
    │ AEB55QTM │ Reagan    │ Hodge    │
    │ OWY35QYB │ Tamekah   │ Juarez   │
    │ MSD36KVR │ Mufutau   │ Watkins  │
    └──────────┴───────────┴──────────┘

    5 rows in set. Elapsed: 0.438 sec.
    ```

    ```sql
    SELECT *
    FROM jdbc('redshift', 'select count(*) from sales')
    ```

    ```response
    Query id: 2d0f957c-8f4e-43b2-a66a-cc48cc96237b

    ┌──count─┐
    │ 172456 │
    └────────┘

    1 rows in set. Elapsed: 0.304 sec.
    ```
  </Step>

  <Step>
    #### Importar datos de Redshift a ClickHouse [#import-data-from-redshift-to-clickhouse]

    A continuación, mostramos cómo importar datos mediante una sentencia `INSERT INTO ... SELECT`

    ```sql
    CREATE TABLE users_imported
    (
       `username` String,
       `firstname` String,
       `lastname` String
    )
    ENGINE = MergeTree
    ORDER BY firstname
    ```

    ```response
    Query id: c7c4c44b-cdb2-49cf-b319-4e569976ab05

    Ok.

    0 rows in set. Elapsed: 0.233 sec.
    ```

    ```sql
    INSERT INTO users_imported (*) SELECT *
    FROM jdbc('redshift', 'select username, firstname, lastname from users')
    ```

    ```response
    Query id: 9d3a688d-b45a-40f4-a7c7-97d93d7149f1

    Ok.

    0 rows in set. Elapsed: 4.498 sec. Processed 49.99 thousand rows, 2.49 MB (11.11 thousand rows/s., 554.27 KB/s.)
    ```
  </Step>
</Steps>

<div id="pivot-data-from-redshift-to-clickhouse-using-s3-pivot-data-from-redshift-to-clickhouse-using-s3">
  ## Transferir datos de Redshift a ClickHouse mediante S3 [#pivot-data-from-redshift-to-clickhouse-using-s3]
</div>

En este escenario, exportamos los datos a S3 en un formato de pivote intermedio y, en un segundo paso, cargamos los datos desde S3 a ClickHouse.

<img src="/images/integrations/data-ingestion/redshift/pivot.webp" alt="PIVOT desde Redshift mediante S3" />

<div id="pros-pros-2">
  ### Ventajas [#pros-2]
</div>

* Tanto Redshift como ClickHouse cuentan con potentes funciones de integración con S3.
* Aprovecha funciones ya existentes, como el comando `UNLOAD` de Redshift y la función de tabla / el motor de tabla S3 de ClickHouse.
* Escala sin inconvenientes gracias a las lecturas en paralelo y a la alta capacidad de procesamiento desde/hacia S3 en ClickHouse.
* Puede aprovechar formatos sofisticados y comprimidos como Apache Parquet.

<div id="cons-cons-2">
  ### Desventajas [#cons-2]
</div>

* Dos pasos en el proceso (exportar desde Redshift y luego cargar en ClickHouse).

<div id="tutorial-tutorial-1">
  ### Tutorial [#tutorial-1]
</div>

<Steps>
  <Step>
    #### Exporte los datos a un bucket de S3 con UNLOAD [#export-data-into-an-s3-bucket-using-unload]

    Con la funcionalidad [UNLOAD](https://docs.aws.amazon.com/redshift/latest/dg/r_UNLOAD.html) de Redshift, exporte los datos a un bucket privado de S3 ya existente:

    <img src="/images/integrations/data-ingestion/redshift/s3-1.webp" alt="UNLOAD de Redshift a S3" />

    Se generarán archivos parciales que contienen los datos sin procesar en S3

    <img src="/images/integrations/data-ingestion/redshift/s3-2.webp" alt="Datos en S3" />
  </Step>

  <Step>
    #### Cree la tabla en ClickHouse [#create-the-table-in-clickhouse]

    Cree la tabla en ClickHouse:

    ```sql
    CREATE TABLE users
    (
      username String,
      firstname String,
      lastname String
    )
    ENGINE = MergeTree
    ORDER BY username
    ```

    Como alternativa, ClickHouse puede intentar inferir la estructura de la tabla usando `CREATE TABLE ... EMPTY AS SELECT`:

    ```sql
    CREATE TABLE users
    ENGINE = MergeTree ORDER BY username
    EMPTY AS
    SELECT * FROM s3('https://your-bucket.s3.amazonaws.com/unload/users/*', '<aws_access_key>', '<aws_secret_access_key>', 'CSV')
    ```

    Esto funciona especialmente bien cuando los datos están en un formato que contiene información sobre los tipos de datos, como Parquet.
  </Step>

  <Step>
    #### Cargue archivos de S3 en ClickHouse [#load-s3-files-into-clickhouse]

    Cargue los archivos de S3 en ClickHouse mediante una instrucción `INSERT INTO ... SELECT`:

    ```sql
    INSERT INTO users SELECT *
    FROM s3('https://your-bucket.s3.amazonaws.com/unload/users/*', '<aws_access_key>', '<aws_secret_access_key>', 'CSV')
    ```

    ```response
    Query id: 2e7e219a-6124-461c-8d75-e4f5002c8557

    Ok.

    0 rows in set. Elapsed: 0.545 sec. Processed 49.99 thousand rows, 2.34 MB (91.72 thousand rows/s., 4.30 MB/s.)
    ```

    <Note>
      Este ejemplo usa CSV como formato intermedio. Sin embargo, para cargas de trabajo de producción recomendamos Apache Parquet como la mejor opción para migraciones grandes, ya que ofrece compresión y puede reducir los costes de almacenamiento al tiempo que acorta los tiempos de transferencia. (De forma predeterminada, cada grupo de filas se comprime con SNAPPY). ClickHouse también aprovecha la orientación por columnas de Parquet para acelerar la ingestión de datos.
    </Note>
  </Step>
</Steps>
