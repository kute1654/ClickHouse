<div id="introduction-introduction">
  ## المقدمة [#introduction]
</div>

يُعد [Amazon Redshift](https://aws.amazon.com/redshift/) حلاً شائعًا لمستودعات البيانات السحابية، وهو جزء من خدمات Amazon Web Services. يقدّم هذا الدليل أساليب مختلفة لترحيل البيانات من مثيل Redshift إلى ClickHouse. وسنستعرض ثلاثة خيارات:

<img src="/images/integrations/data-ingestion/redshift/redshift-to-clickhouse.webp" alt="خيارات ترحيل Redshift إلى ClickHouse" />

من منظور مثيل ClickHouse، يمكنك:

1. **[PUSH](#push-data-from-redshift-to-clickhouse)** إرسال البيانات إلى ClickHouse باستخدام أداة أو خدمة ETL/ELT من جهة خارجية

2. **[PULL](#pull-data-from-redshift-to-clickhouse)** سحب البيانات من Redshift بالاستفادة من ClickHouse JDBC Bridge

3. **[PIVOT](#pivot-data-from-redshift-to-clickhouse-using-s3)** استخدام تخزين الكائنات S3 وفق آلية &quot;التفريغ ثم التحميل&quot;

<Note>
  استخدمنا Redshift كمصدر بيانات في هذا الدليل. ومع ذلك، فإن أساليب الترحيل المعروضة هنا لا تقتصر على Redshift، ويمكن تطبيق خطوات مماثلة على أي مصدر بيانات متوافق.
</Note>

<div id="push-data-from-redshift-to-clickhouse-push-data-from-redshift-to-clickhouse">
  ## إرسال البيانات من Redshift إلى ClickHouse بأسلوب Push [#push-data-from-redshift-to-clickhouse]
</div>

في سيناريو Push، تتمثل الفكرة في الاستفادة من أداة أو خدمة تابعة لجهة خارجية (سواء كانت شيفرة مخصصة أو أداة [ETL/ELT](https://en.wikipedia.org/wiki/Extract,_transform,_load#ETL_vs._ELT)) لإرسال بياناتك إلى مثيل ClickHouse الخاص بك. على سبيل المثال، يمكنك استخدام أداة مثل [Airbyte](https://www.airbyte.com/) لنقل البيانات بين مثيل Redshift الخاص بك (كمصدر) وClickHouse كوجهة ([راجع دليل التكامل الخاص بنا لـ Airbyte](/ar/integrations/connectors/data-ingestion/etl-tools/airbyte-and-clickhouse))

<img src="/images/integrations/data-ingestion/redshift/push.webp" alt="إرسال البيانات من Redshift إلى ClickHouse بأسلوب Push" />

<div id="pros-pros">
  ### المزايا [#pros]
</div>

* يمكنه الاستفادة من الكتالوج الحالي للموصلات في برمجيات ETL/ELT.
* إمكانات مدمجة للحفاظ على مزامنة البيانات (منطق append/overwrite/increment).
* يتيح سيناريوهات لتحويل البيانات (على سبيل المثال، راجع [دليل التكامل الخاص بنا لـ dbt](/ar/integrations/connectors/data-ingestion/etl-tools/dbt/index)).

<div id="cons-cons">
  ### السلبيات [#cons]
</div>

* يحتاج المستخدمون إلى إعداد بنية تحتية لعمليات ETL/ELT وصيانتها.
* يضيف ذلك عنصرًا تابعًا لجهة خارجية إلى البنية المعمارية، مما قد يتحول إلى عنق زجاجة محتمل يحدّ من قابلية التوسع.

<div id="pull-data-from-redshift-to-clickhouse-pull-data-from-redshift-to-clickhouse">
  ## سحب البيانات من Redshift إلى ClickHouse [#pull-data-from-redshift-to-clickhouse]
</div>

في سيناريو السحب، تتمثل الفكرة في الاستفادة من ClickHouse JDBC Bridge للاتصال مباشرةً بعنقود Redshift انطلاقًا من مثيل ClickHouse، ثم تنفيذ استعلامات `INSERT INTO ... SELECT`:

<img src="/images/integrations/data-ingestion/redshift/pull.webp" alt="السحب من Redshift إلى ClickHouse" />

<div id="pros-pros-1">
  ### المزايا [#pros-1]
</div>

* مناسب لجميع الأدوات المتوافقة مع JDBC
* حل أنيق يتيح الاستعلام من عدة مصادر بيانات خارجية من داخل ClickHouse

<div id="cons-cons-1">
  ### السلبيات [#cons-1]
</div>

* يتطلب مثيلًا من ClickHouse JDBC Bridge، ما قد يصبح عنق زجاجة محتملًا من حيث قابلية التوسع

<Note>
  على الرغم من أن Redshift يستند إلى PostgreSQL، لا يمكن استخدام دالة الجدول أو محرك الجدول الخاصة بـ PostgreSQL في ClickHouse، لأن ClickHouse يتطلب PostgreSQL بالإصدار 9 أو أحدث، بينما تستند واجهة برمجة تطبيقات Redshift إلى إصدار أقدم (8.x).
</Note>

<div id="tutorial-tutorial">
  ### دليل عملي [#tutorial]
</div>

لاستخدام هذا الخيار، تحتاج إلى إعداد ClickHouse JDBC Bridge. ‏ClickHouse JDBC Bridge هو تطبيق Java مستقل يتولى اتصال JDBC ويعمل كوكيل بين مثيل ClickHouse ومصادر البيانات. في هذا الدليل العملي، استخدمنا مثيل Redshift مُجهزًا مسبقًا مع [قاعدة بيانات نموذجية](https://docs.aws.amazon.com/redshift/latest/dg/c_sampledb.html).

<Steps>
  <Step>
    #### نشر ClickHouse JDBC Bridge [#deploy-clickhouse-jdbc-bridge]

    انشر ClickHouse JDBC Bridge. لمزيد من التفاصيل، راجع دليل المستخدم الخاص بنا حول [JDBC لمصادر البيانات الخارجية](/ar/integrations/connectors/data-ingestion/jdbc-with-clickhouse)

    <Note>
      إذا كنت تستخدم ClickHouse Cloud، فستحتاج إلى تشغيل ClickHouse JDBC Bridge في بيئة منفصلة والاتصال بـ ClickHouse Cloud باستخدام الدالة [remoteSecure](/ar/reference/functions/table-functions/remote)
    </Note>
  </Step>

  <Step>
    #### تهيئة مصدر بيانات Redshift [#configure-your-redshift-datasource]

    هيّئ مصدر بيانات Redshift لـ ClickHouse JDBC Bridge. على سبيل المثال: `/etc/clickhouse-jdbc-bridge/config/datasources/redshift.json `

    ```json
    {
     "redshift-server": {
       "aliases": [
         "redshift"
       ],
       "driverUrls": [
       "https://s3.amazonaws.com/redshift-downloads/drivers/jdbc/2.1.0.4/redshift-jdbc42-2.1.0.4.jar"
       ],
       "driverClassName": "com.amazon.redshift.jdbc.Driver",
       "jdbcUrl": "jdbc:redshift://redshift-cluster-1.ckubnplpz1uv.us-east-1.redshift.amazonaws.com:5439/dev",
       "username": "awsuser",
       "password": "<password>",
       "maximumPoolSize": 5
     }
    }
    ```
  </Step>

  <Step>
    #### الاستعلام عن مثيل Redshift من ClickHouse [#query-your-redshift-instance-from-clickhouse]

    بمجرد نشر ClickHouse JDBC Bridge وتشغيله، يمكنك البدء في الاستعلام عن مثيل Redshift من ClickHouse

    ```sql
    SELECT *
    FROM jdbc('redshift', 'select username, firstname, lastname from users limit 5')
    ```

    ```response
    Query id: 1b7de211-c0f6-4117-86a2-276484f9f4c0

    ┌─username─┬─firstname─┬─lastname─┐
    │ PGL08LJI │ Vladimir  │ Humphrey │
    │ XDZ38RDD │ Barry     │ Roy      │
    │ AEB55QTM │ Reagan    │ Hodge    │
    │ OWY35QYB │ Tamekah   │ Juarez   │
    │ MSD36KVR │ Mufutau   │ Watkins  │
    └──────────┴───────────┴──────────┘

    5 rows in set. Elapsed: 0.438 sec.
    ```

    ```sql
    SELECT *
    FROM jdbc('redshift', 'select count(*) from sales')
    ```

    ```response
    Query id: 2d0f957c-8f4e-43b2-a66a-cc48cc96237b

    ┌──count─┐
    │ 172456 │
    └────────┘

    1 rows in set. Elapsed: 0.304 sec.
    ```
  </Step>

  <Step>
    #### استيراد البيانات من Redshift إلى ClickHouse [#import-data-from-redshift-to-clickhouse]

    فيما يلي، نعرض كيفية استيراد البيانات باستخدام عبارة `INSERT INTO ... SELECT`

    ```sql
    CREATE TABLE users_imported
    (
       `username` String,
       `firstname` String,
       `lastname` String
    )
    ENGINE = MergeTree
    ORDER BY firstname
    ```

    ```response
    Query id: c7c4c44b-cdb2-49cf-b319-4e569976ab05

    Ok.

    0 rows in set. Elapsed: 0.233 sec.
    ```

    ```sql
    INSERT INTO users_imported (*) SELECT *
    FROM jdbc('redshift', 'select username, firstname, lastname from users')
    ```

    ```response
    Query id: 9d3a688d-b45a-40f4-a7c7-97d93d7149f1

    Ok.

    0 rows in set. Elapsed: 4.498 sec. Processed 49.99 thousand rows, 2.49 MB (11.11 thousand rows/s., 554.27 KB/s.)
    ```
  </Step>
</Steps>

<div id="pivot-data-from-redshift-to-clickhouse-using-s3-pivot-data-from-redshift-to-clickhouse-using-s3">
  ## نقل البيانات من Redshift إلى ClickHouse باستخدام S3 بأسلوب Pivot [#pivot-data-from-redshift-to-clickhouse-using-s3]
</div>

في هذا السيناريو، نُصدِّر البيانات إلى S3 بتنسيق Pivot وسيط، ثم نحمّلها في خطوة ثانية من S3 إلى ClickHouse.

<img src="/images/integrations/data-ingestion/redshift/pivot.webp" alt="PIVOT من Redshift باستخدام S3" />

<div id="pros-pros-2">
  ### المزايا [#pros-2]
</div>

* يتمتع كل من Redshift وClickHouse بميزات تكامل قوية مع S3.
* يستفيد من الميزات المتاحة حاليًا، مثل الأمر `UNLOAD` في Redshift ودالة الجدول / محرك الجدول S3 في ClickHouse.
* يتوسع بسلاسة بفضل عمليات القراءة المتوازية وإمكانات معدل النقل العالي من S3 وإليه في ClickHouse.
* يمكنه الاستفادة من تنسيقات متقدمة ومضغوطة مثل Apache Parquet.

<div id="cons-cons-2">
  ### السلبيات [#cons-2]
</div>

* خطوتان في العملية (تفريغ البيانات من Redshift ثم تحميلها إلى ClickHouse).

<div id="tutorial-tutorial-1">
  ### دليل عملي [#tutorial-1]
</div>

<Steps>
  <Step>
    #### صدّر البيانات إلى حاوية S3 باستخدام UNLOAD [#export-data-into-an-s3-bucket-using-unload]

    باستخدام ميزة [UNLOAD](https://docs.aws.amazon.com/redshift/latest/dg/r_UNLOAD.html) في Redshift، صدّر البيانات إلى حاوية S3 خاصة موجودة مسبقًا:

    <img src="/images/integrations/data-ingestion/redshift/s3-1.webp" alt="UNLOAD من Redshift إلى S3" />

    سينتج عن ذلك ملفات أجزاء تحتوي على البيانات الخام في S3

    <img src="/images/integrations/data-ingestion/redshift/s3-2.webp" alt="البيانات في S3" />
  </Step>

  <Step>
    #### أنشئ الجدول في ClickHouse [#create-the-table-in-clickhouse]

    أنشئ الجدول في ClickHouse:

    ```sql
    CREATE TABLE users
    (
      username String,
      firstname String,
      lastname String
    )
    ENGINE = MergeTree
    ORDER BY username
    ```

    بدلًا من ذلك، يمكن لـ ClickHouse محاولة استنتاج بنية الجدول باستخدام `CREATE TABLE ... EMPTY AS SELECT`:

    ```sql
    CREATE TABLE users
    ENGINE = MergeTree ORDER BY username
    EMPTY AS
    SELECT * FROM s3('https://your-bucket.s3.amazonaws.com/unload/users/*', '<aws_access_key>', '<aws_secret_access_key>', 'CSV')
    ```

    ويعمل ذلك جيدًا خصوصًا عندما تكون البيانات بتنسيق يتضمّن معلومات عن أنواع البيانات، مثل Parquet.
  </Step>

  <Step>
    #### حمّل ملفات S3 إلى ClickHouse [#load-s3-files-into-clickhouse]

    حمّل ملفات S3 إلى ClickHouse باستخدام تعليمة `INSERT INTO ... SELECT`:

    ```sql
    INSERT INTO users SELECT *
    FROM s3('https://your-bucket.s3.amazonaws.com/unload/users/*', '<aws_access_key>', '<aws_secret_access_key>', 'CSV')
    ```

    ```response
    Query id: 2e7e219a-6124-461c-8d75-e4f5002c8557

    Ok.

    0 rows in set. Elapsed: 0.545 sec. Processed 49.99 thousand rows, 2.34 MB (91.72 thousand rows/s., 4.30 MB/s.)
    ```

    <Note>
      يستخدم هذا المثال CSV بوصفه التنسيق الوسيط. ومع ذلك، نوصي في بيئات الإنتاج باستخدام Apache Parquet باعتباره الخيار الأفضل لعمليات الترحيل الكبيرة، لأنه يوفّر ضغطًا للبيانات ويمكن أن يخفّض بعض تكاليف التخزين ويقلّل زمن النقل. (افتراضيًا، تُضغط كل مجموعة صفوف باستخدام SNAPPY). كما يستفيد ClickHouse من البنية العمودية في Parquet لتسريع إدخال البيانات.
    </Note>
  </Step>
</Steps>
