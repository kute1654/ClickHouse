  # Homebrew를 사용하여 ClickHouse 설치

  <Step>
    ## 커뮤니티 Homebrew formula를 사용해 설치

    [Homebrew](https://brew.sh/)를 사용해 macOS에 ClickHouse를 설치하려면
    ClickHouse 커뮤니티 [homebrew formula](https://formulae.brew.sh/cask/clickhouse)를 사용할 수 있습니다.

    ```bash
    brew install --cask clickhouse
    ```
  </Step>

  <Step>
    ## macOS에서 개발자 확인 오류 수정

    `brew`를 사용해 ClickHouse를 설치하면 macOS에서 오류가 발생할 수 있습니다.
    기본적으로 macOS는 확인할 수 없는 개발자가 만든 애플리케이션이나 도구를 실행하지 않습니다.

    `clickhouse` 명령을 실행하려고 하면 다음과 같은 오류가 표시될 수 있습니다:

    <Image img="/images/knowledgebase/fix-the-developer-verification-error-in-macos/dev-verification-error.webp" size="sm" alt="macOS 개발자 확인 오류 대화 상자" border />

    이 확인 오류를 우회하려면 시스템 설정 창에서 관련 설정을 찾거나, 터미널을 사용하거나, ClickHouse를 다시 설치하여 macOS의 격리 빈에서 앱을 제거해야 합니다.

    ### 시스템 설정 절차

    격리 빈에서 `clickhouse` 실행형 파일을 제거하는 가장 쉬운 방법은 다음과 같습니다:

    1. **시스템 설정**을 엽니다.

    2. **Privacy &amp; Security**로 이동합니다:

       <Image img="/images/knowledgebase/fix-the-developer-verification-error-in-macos/privacy-and-security-default-view.webp" size="md" alt="기본 보기의 macOS Privacy & Security 설정" border />

    3. 창 맨 아래로 스크롤하여 &#95;&quot;clickhouse-macos-aarch64&quot; was blocked from use because it isn&#39;t from an identified developer&quot;&#95;라는 메시지를 찾습니다.

    4. **Allow Anyway**를 클릭합니다.

       <Image img="/images/knowledgebase/fix-the-developer-verification-error-in-macos/privacy-and-security-screen-allow-anyway.webp" size="md" alt="Allow Anyway 버튼이 표시된 macOS Privacy & Security 설정" border />

    5. macOS 사용자 비밀번호를 입력합니다.

    이제 터미널에서 `clickhouse` 명령을 실행할 수 있습니다.

    ### 터미널 절차

    경우에 따라 `Allow Anyway` 버튼을 눌러도 이 문제가 해결되지 않을 수 있습니다. 이런 경우 명령줄에서 이 절차를 수행할 수도 있습니다.
    또는 단순히 명령줄 사용을 선호할 수도 있습니다.

    먼저 Homebrew가 `clickhouse` 실행형 파일을 설치한 위치를 확인합니다:

    ```shell
    which clickhouse
    ```

    그러면 다음과 비슷한 출력이 표시됩니다:

    ```shell
    /opt/homebrew/bin/clickhouse
    ```

    이전 명령에서 확인한 경로를 붙여 `xattr -d com.apple.quarantine`를 실행하면 `clickhouse`를 격리 빈에서 제거할 수 있습니다:

    ```shell
    xattr -d com.apple.quarantine /opt/homebrew/bin/clickhouse
    ```

    이제 `clickhouse` 실행형 파일을 실행할 수 있습니다:

    ```shell
    clickhouse
    ```

    그러면 다음과 비슷한 출력이 표시됩니다:

    ```bash
    Use one of the following commands:
    clickhouse local [args]
    clickhouse client [args]
    clickhouse benchmark [args]
    ```
  </Step>

  <Step>
    ## ClickHouse를 다시 설치하여 문제 해결

    Brew에는 설치된 바이너리가 처음부터 격리되지 않도록 하는 명령줄 옵션이 있습니다.

    먼저 ClickHouse를 제거합니다:

    ```shell
    brew uninstall clickhouse
    ```

    그다음 `--no-quarantine` 옵션으로 ClickHouse를 다시 설치합니다:

    ```shell
    brew install --no-quarantine clickhouse
    ```
  </Step>
</Steps>
