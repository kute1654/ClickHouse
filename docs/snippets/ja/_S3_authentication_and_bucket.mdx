title: S3 の認証とバケット
---

<Accordion title="S3 バケットと IAM ユーザーの作成">
  この記事では、AWS IAM ユーザーの設定、S3 バケットの作成、および ClickHouse でそのバケットを S3 ディスクとして使用するための設定方法の基本について説明します。
  実際に使用するパーミッションはセキュリティチームと相談して決定してください。ここで示す内容はあくまで出発点としてご参照ください。

  ### AWS IAM ユーザーの作成

  以下の手順では、サービスアカウントユーザー (ログインユーザーではなく) を作成します。

  1. AWS IAM Management Console にログインします。

  2. `Users` メニューから `Create user` を選択します

  <Image size="md" img="/images/_snippets/s3/s3-1.webp" alt="AWS IAM Management Console - 新規ユーザーの追加" border force />

  3. ユーザー名を入力し、認証情報の種類を `Access key - Programmatic access` に設定して、`Next: Permissions` をクリックします

  <Image size="md" img="/images/_snippets/s3/s3-2.webp" alt="IAM ユーザーのユーザー名とアクセス種別を設定する" border force />

  4. ユーザーはどのグループにも追加せず、`Next: Tags` を選択してください

  <Image size="md" img="/images/_snippets/s3/s3-3.webp" alt="IAMユーザーへのグループ割り当てをスキップしています" border force />

  5. タグを追加する必要がなければ、`Next: Review` を選択します

  <Image size="md" img="/images/_snippets/s3/s3-4.webp" alt="IAMユーザーへのタグの割り当てをスキップしています" border force />

  6. `Create User` を選択します

  <Note>
    ユーザーに permissions がないという警告メッセージは無視してかまいません。次のセクションで、そのユーザーにバケットへの permissions が付与されます
  </Note>

  <Image size="md" img="/images/_snippets/s3/s3-5.webp" alt="権限を持たないIAMユーザーの作成に関する警告" border force />

  7. ユーザーが作成されたら、`show` をクリックしてアクセスキーとシークレットキーをコピーします。

  <Note>
    キーは必ず別の場所にも保存してください。シークレットアクセスキーを確認できるのはこのときだけです。
  </Note>

  <Image size="md" img="/images/_snippets/s3/s3-6.webp" alt="IAM ユーザーのアクセスキーの表示とコピー" border force />

  8. ［閉じる］をクリックし、次にユーザー画面でそのユーザーを探します。

  <Image size="md" img="/images/_snippets/s3/s3-7.webp" alt="新しく作成した IAM ユーザーをユーザー一覧で確認する" border force />

  9. ARN (Amazon Resource Name) をコピーし、バケットのアクセスポリシーを設定する際に使用できるよう保存しておきます。

  <Image size="md" img="/images/_snippets/s3/s3-8.webp" alt="IAM ユーザーのARNをコピーする" border force />

  ### S3バケットの作成

  1. S3 bucket セクションで、`Create bucket` を選択します。

  <Image size="md" img="/images/_snippets/s3/s3-9.webp" alt="S3 バケットの作成を開始する" border force />

  2. バケット名を入力し、他のオプションはそのままにします

  <Note>
    バケット名は、組織内だけでなくAWS全体で一意である必要があり、そうでない場合はエラーが発生します。
  </Note>

  3. `Block all Public Access` は有効にしたままにしてください。パブリックアクセスは不要です。

  <Image size="md" img="/images/_snippets/s3/s3-a.webp" alt="公開アクセスをブロックした S3 バケットの設定" border force />

  4. ページ下部の `Create Bucket` を選択します

  <Image size="md" img="/images/_snippets/s3/s3-b.webp" alt="S3 バケット作成の完了" border force />

  5. リンクを選択し、ARNをコピーして、バケットのアクセスポリシーを設定するときに使えるよう保存します。

  6. バケットが作成されたら、S3 バケットの一覧で新しい S3 バケットを見つけ、リンクを選択します

  <Image size="md" img="/images/_snippets/s3/s3-c.webp" alt="バケット一覧で新しく作成したS3バケットを見つける" border force />

  7. `Create folder` を選択します

  <Image size="md" img="/images/_snippets/s3/s3-d.webp" alt="S3バケットに新しいフォルダを作成する" border force />

  8. ClickHouse S3 ディスクの保存先となるフォルダ名を入力し、`Create folder` を選択します

  <Image size="md" img="/images/_snippets/s3/s3-e.webp" alt="ClickHouseのS3ディスクで使用するフォルダ名の設定" border force />

  9. フォルダがバケット一覧に表示されているはずです

  <Image size="md" img="/images/_snippets/s3/s3-f.webp" alt="S3バケット内に新しく作成したフォルダを表示する" border force />

  10. 新しいフォルダーのチェックボックスをオンにし、`Copy URL` をクリックします。コピーした URL は、次のセクションで ClickHouse のストレージ構成に使用するため、保存しておいてください。

  <Image size="md" img="/images/_snippets/s3/s3-g.webp" alt="ClickHouse の設定用に S3 フォルダの URL をコピーする" border force />

  11. `Permissions` タブを選択し、`Bucket Policy` セクションで `Edit` ボタンをクリックします

  <Image size="md" img="/images/_snippets/s3/s3-h.webp" alt="S3 バケットポリシー設定にアクセスする" border force />

  12. バケットポリシーを追加します。以下に例を示します。

  ```json
  {
    "Version" : "2012-10-17",
    "Id" : "Policy123456",
    "Statement" : [
      {
        "Sid" : "abc123",
        "Effect" : "Allow",
        "Principal" : {
          "AWS" : "arn:aws:iam::921234567898:user/mars-s3-user"
        },
        "Action" : "s3:*",
        "Resource" : [
          "arn:aws:s3:::mars-doc-test",
          "arn:aws:s3:::mars-doc-test/*"
        ]
      }
    ]
  }
  ```

  ```response
  |パラメータ | 説明 | 値の例 |
  |----------|-------------|----------------|
  |Version | ポリシーインタープリターのバージョン。変更不要 | 2012-10-17 |
  |Sid | ユーザー定義のポリシーID | abc123 |
  |Effect | ユーザーリクエストを許可するか拒否するかを設定する | Allow |
  |Principal | アクセスを許可するアカウントまたはユーザー | arn:aws:iam::921234567898:user/mars-s3-user |
  |Action | バケットに対して許可する操作| s3:*|
  |Resource | 操作を許可するバケット内のリソース | "arn:aws:s3:::mars-doc-test", "arn:aws:s3:::mars-doc-test/*" |
  ```

  <Note>
    使用する権限については、セキュリティチームと連携して決定してください。以下はその出発点としてお考えください。
    ポリシーと設定の詳細については、AWS のドキュメントを参照してください。
    https://docs.aws.amazon.com/AmazonS3/latest/userguide/access-policy-language-overview.html
  </Note>

  13. ポリシー設定を保存します。
</Accordion>
