  <Step>
    ## Instale usando a fórmula comunitária do Homebrew

    Para instalar o ClickHouse no macOS usando [Homebrew](https://brew.sh/), você pode usar
    a [fórmula do Homebrew](https://formulae.brew.sh/cask/clickhouse) da comunidade ClickHouse.

    ```bash
    brew install --cask clickhouse
    ```
  </Step>

  <Step>
    ## Corrija o erro de verificação do desenvolvedor no macOS

    Se você instalar o ClickHouse usando `brew`, poderá encontrar um erro do macOS.
    Por padrão, o macOS não executa aplicativos ou ferramentas criados por um desenvolvedor que não pode ser verificado.

    Ao tentar executar qualquer comando `clickhouse`, você poderá ver este erro:

    <Image img="/images/knowledgebase/fix-the-developer-verification-error-in-macos/dev-verification-error.webp" size="sm" alt="Caixa de diálogo de erro de verificação do desenvolvedor no macOS" border />

    Para contornar esse erro de verificação, você precisa remover o aplicativo da quarentena do macOS, seja encontrando a configuração apropriada na janela de Ajustes do Sistema, usando o terminal ou reinstalando o ClickHouse.

    ### Processo pelos ajustes do sistema

    A maneira mais fácil de remover o executável `clickhouse` da quarentena é:

    1. Abra **Ajustes do Sistema**.

    2. Vá até **Privacidade e Segurança**:

       <Image img="/images/knowledgebase/fix-the-developer-verification-error-in-macos/privacy-and-security-default-view.webp" size="md" alt="Visualização padrão dos ajustes de Privacidade e Segurança no macOS" border />

    3. Role até o fim da janela para encontrar uma mensagem informando que *&quot;clickhouse-macos-aarch64&quot; foi bloqueado para uso porque não é de um desenvolvedor identificado*.

    4. Clique em **Permitir Mesmo Assim**.

       <Image img="/images/knowledgebase/fix-the-developer-verification-error-in-macos/privacy-and-security-screen-allow-anyway.webp" size="md" alt="Ajustes de Privacidade e Segurança no macOS mostrando o botão Permitir Mesmo Assim" border />

    5. Digite a senha do seu usuário no macOS.

    Agora você já deve conseguir executar comandos `clickhouse` no terminal.

    ### Processo pelo terminal

    Às vezes, clicar no botão `Permitir Mesmo Assim` não corrige esse problema; nesse caso, você também pode fazer esse processo pela linha de comando.
    Ou talvez você simplesmente prefira usar a linha de comando!

    Primeiro, descubra onde o Homebrew instalou o executável `clickhouse`:

    ```shell
    which clickhouse
    ```

    Isso deve retornar algo como:

    ```shell
    /opt/homebrew/bin/clickhouse
    ```

    Remova o `clickhouse` da quarentena executando `xattr -d com.apple.quarantine`, seguido do path retornado pelo comando anterior:

    ```shell
    xattr -d com.apple.quarantine /opt/homebrew/bin/clickhouse
    ```

    Agora você já deve conseguir executar o executável `clickhouse`:

    ```shell
    clickhouse
    ```

    Isso deve retornar algo como:

    ```bash
    Use one of the following commands:
    clickhouse local [args]
    clickhouse client [args]
    clickhouse benchmark [args]
    ```
  </Step>

  <Step>
    ## Corrija o problema reinstalando o ClickHouse

    O Brew tem uma opção de linha de comando que evita colocar os binaries instalados em quarentena desde o início.

    Primeiro, desinstale o ClickHouse:

    ```shell
    brew uninstall clickhouse
    ```

    Agora reinstale o ClickHouse com `--no-quarantine`:

    ```shell
    brew install --no-quarantine clickhouse
    ```
  </Step>
