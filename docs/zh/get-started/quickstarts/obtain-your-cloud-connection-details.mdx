你还应完成[创建你的第一个 Cloud 服务](/zh/get-started/quickstarts/create-your-first-service-on-cloud)快速入门，因为你需要一个现有的 ClickHouse Cloud 服务来获取连接信息。

<div id="what-youll-build">
  ## 你将完成的内容
</div>

在本快速入门中，你将学习如何找到 ClickHouse Cloud 服务的连接信息——也就是从任何外部客户端、CLI 工具或应用连接时所需的主机名、端口号、用户名和密码。
完成后，你将知道如何在 Cloud 控制台中找到这些信息，以及如何使用简单的 `curl` 命令验证连通性，从而可以通过 `clickhouse-local`、`clickhouse-client` 或任何 ClickHouse 驱动进行连接。

<Steps titleSize="h3">
  <Step>
    ### 打开 Cloud 控制台

    前往 [ClickHouse Cloud 控制台](https://console.clickhouse.cloud/) 并登录你的账户。

    在主仪表板中，找到你要连接的服务。如果该服务处于休眠状态，点击它并等待其唤醒后再继续——你需要让它保持运行，以便在后续步骤中验证连通性。
  </Step>

  <Step>
    ### 查找连接信息

    点击你的服务以打开详情页，然后点击左侧边栏中的 **Connect** 按钮：

    <img className="block dark:hidden w-2/3 mx-auto" src="/images/get-started/quickstarts/connect-light.webp" alt="ClickHouse Cloud 控制台中的 Connect 按钮（浅色模式）" />

    <img className="hidden dark:block w-2/3 mx-auto" src="/images/get-started/quickstarts/connect-dark.webp" alt="ClickHouse Cloud 控制台中的 Connect 按钮（深色模式）" />

    第一次点击此按钮时，你会看到一个显示凭据的对话框：

    | 详细信息         | 说明                     |
    | ------------ | ---------------------- |
    | **Username** | 你的用户名 (通常为 `default`)  |
    | **Password** | 为你的服务生成的密码             |

    此外，还有一个下拉菜单，提供适用于不同协议和接口的预格式化示例连接字符串 (例如 `clickhouse-client`、HTTPS、JDBC 等) ，你可以直接复制。

    <Warning>
      请下载凭据文件，或复制你的密码并妥善保存。关闭此对话框后，你将无法再次查看该密码。
    </Warning>

    你还可以在服务详情页找到服务的 **主机名** (例如 `abc123.us-east-1.aws.clickhouse.cloud`) 。请将它与凭证一起保存——在后续步骤以及之后的快速入门中都会用到。

    <Note>
      如果你遗失了密码，可以在 Cloud 控制台中该服务的 **Settings** 选项卡内重置。
    </Note>
  </Step>

  <Step>
    ### 了解连接协议

    ClickHouse Cloud 提供两种主要网络协议，每种协议各自使用一个端口：

    * **native protocol (端口 9440)&#x20;**&#x20;- `clickhouse-client`、`clickhouse-local` 以及大多数特定语言驱动使用的二进制协议。这是最快的选项，并支持所有 ClickHouse 功能。在 Cloud 上必须使用 TLS。
    * **HTTPS protocol (端口 8443)&#x20;**&#x20;- 基于 HTTP 的接口，适用于 REST 客户端、`curl`、基于 Web 的工具，以及偏好 HTTP 的驱动。在 Cloud 上同样需要 TLS。

    连接到 ClickHouse Cloud 时，这两种协议都要求使用 TLS 加密——你不能通过明文连接。大多数工具在你指定正确端口时会自动处理这一点，但具体取决于客户端，你可能需要传递 `--secure` 标志或设置 `ssl=true`。

    在本系列的大多数快速入门和 CLI 工作流中，你将使用 **9440 端口上的 native protocol**。
  </Step>

  <Step>
    ### 保存连接信息以便重复使用

    在各个快速入门中，你会频繁用到这些连接信息。为了避免每次都重新输入，你可以在终端会话中将它们导出为环境变量：

    ```bash
    export CLICKHOUSE_HOST=YOUR_HOST.clickhouse.cloud
    export CLICKHOUSE_USER=default
    export CLICKHOUSE_PASSWORD=YOUR_PASSWORD
    ```

    将这些值替换为你的实际主机名、用户名和密码。
    如果你要使用 `clickhouse-client` 或 `clickhouse-local` 连接到 Cloud 服务，请从 “Connect with:” 旁边的下拉菜单中选择 &#39;Native&#39; 以获取主机名。
    复制显示的连接示例中 `--host` 后面的主机名。

    <Warning>
      这些环境变量只会在当前终端会话中保留。不要将密码存储在 shell 配置文件 (`.bashrc`、`.zshrc`) 中，也不要将其提交到版本控制。
    </Warning>
  </Step>
</Steps>
