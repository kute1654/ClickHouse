<Update label="2026年6月3日">
  Open House 2026 周迎来了一波新发布，重点围绕两大主题：在数据所在之处运行 ClickHouse，以及将智能体驱动的 AI 辅助工作流引入平台。

  ### ClickPipes 现已在 GCP 上通过 Private Service Connect 原生运行

  ClickPipes 基础设施现已与您的 ClickHouse Cloud 服务部署在同一 GCP 区域，适用于 5 月 26 日之后创建的所有新 GCP 服务，并支持 Private Service Connect (PSC)。这一变化消除了跨云出口费用，降低了延迟，确保数据留存在本地，同时让您能够通过 GCP 原生私有网络连接到 GCP 数据源，无需再借助 SSH 隧道等变通方案。现有服务可联系我们的团队协助迁移。

  更多详情，请参阅 [ClickPipes 静态 IP 列表](/zh/integrations/clickpipes/home#list-of-static-ips)。

  ### ClickPipes for GCP Pub/Sub (私有预览)

  ClickPipes 现已支持将 GCP Pub/Sub topics 中的数据直接流式传输至 ClickHouse Cloud，为原生 GCP 团队提供了一条直接分析 Cloud Logging、Cloud Monitoring、GKE 等服务数据的路径。可通过 ClickPipes UI、OpenAPI 和 Terraform 使用；私有预览期间需提前[注册](https://clickhouse.com/cloud/clickpipes#pubsub-private-preview)。

  有关设置和配置的详细信息，请参阅 [ClickPipes for GCP Pub/Sub 文档](/zh/integrations/clickpipes/pubsub/overview)。

  ### ClickHouse 智能体：由 Claude 驱动的智能体分析 (public beta)

  ClickHouse 智能体现已在 [ai.clickhouse.cloud](http://ai.clickhouse.cloud) 进入 Public Beta 阶段——这是一个由 Claude 驱动的原生智能体平台，支持对您的 ClickHouse Cloud 数据进行对话式、schema 感知分析。通过无代码 AgentBuilder 构建自定义智能体，接入 MCP servers (Slack、Linear、Notion 等) ，编排子智能体工作流，并在对话中直接生成查询、管道和应用。平台集成了 AWS Bedrock AgentCore，并以托管的 LibreChat 为底层基础。

  详情请参阅 [ClickHouse 智能体文档](/zh/products/cloud/features/ai-ml/agents/index)。

  ### ClickHouse Postgres (Public Beta)

  本地 NVMe 存储上的全托管 Postgres 现已进入 Public Beta 阶段，其 TPS 比 AWS RDS 高出 5 倍以上，并在一个托管技术栈中实现了 OLTP 与 OLAP 的融合。原生 CDC 管道可将数据直接流式传输至 ClickHouse，无需任何中间基础设施；开源的 `pg_clickhouse` 扩展则允许您在标准 Postgres 会话中查询 ClickHouse 表。

  详情请参阅 [Open House 2026 Day 1: Real-time data without lock-in and what teams can build next](https://clickhouse.com/blog/open-house-2026-day-1#postgres-announcements)。

  ### ClickStack Cloud：无服务器可观测性 (私有预览)

  ClickStack Cloud 是由 ClickHouse Cloud 驱动的全托管无服务器可观测性平台。将您的 OpenTelemetry Collector 指向托管的 OTLP 端点，即可立即在 ClickStack UI 中探索日志、指标和链路追踪——摄取、缓冲、扩缩容、存储、查询服务、身份验证和 RBAC 均由平台全权处理。目前在 AWS us-east-1 区域提供有限私有预览。

  更多详情请参阅 [ClickStack Cloud 私有预览](https://clickhouse.com/blog/clickstack-cloud-private-preview) 和 [ClickStack Cloud 候补名单](https://clickhouse.com/cloud/clickstack-cloud-waitlist)。

  ### 托管 ClickStack 的 AI 笔记本 (Beta)

  AI 笔记本为托管 ClickStack 带来了持久、透明的工作区，专为 AI 辅助的事件调查而设计。您可以记录结构化的提示词序列、查询、图表与推理过程，通过分支探索多种假设，并将调查以团队协作制品的形式共享——所有生成的查询与结果均可查看和编辑。可通过 ClickStack UI 左侧导航栏访问。

  更多详情，请参阅：

  * [ClickStack AI Notebooks 文档](/zh/clickstack/notebooks)
  * [AI Notebooks Beta 版 (博客) ](https://clickhouse.com/blog/observability-mcp-server-ai-notebooks#ai-notebooks-in-beta)

  ### 水平自动扩缩容 (私有预览)

  水平自动扩缩容可自动调整 ClickHouse Cloud 集群中的副本数量，在需求激增时快速横向扩展，在负载下降时逐步横向收缩——让工作负载波动较大、高并发场景下的团队无需再手动调整集群规模。私有预览阶段基于并发查询数进行扩缩容；CPU 和内存信号计划在 Public Beta 阶段推出。适用于使用 ClickHouse 26.2+ 的 Enterprise 和 Scale 层级客户。

  注意：在首次 GA 发布时，水平自动扩缩容与垂直自动扩缩容互斥。水平自动扩缩容仅在垂直自动扩缩容禁用时 (即服务设置为固定规格时) 才会生效。同样，垂直自动扩缩容也仅在副本数量固定时才能启用。此行为将在未来的发行版中得到改善，变得更加灵活。

  ### ClickStack MCP Server (开源版)

  全新的开源 ClickStack MCP Server 为外部 AI 智能体提供基于 ClickStack 和 ClickHouse 构建的结构化可观测性调查工具。它针对日志、指标和链路追踪提供语义基本类型，支持日志模式分析、trace 异常调查、关联分析和跨信号调试，并可持久化仪表盘、搜索和告警，同时保留原生 SQL 访问能力以支持深度探索。可接入 Claude Code、Cursor、Codex 等 AI 智能体或自定义框架。

  更多详情请参阅 [ClickStack MCP server 文档](/zh/clickstack/mcp) 和[可观测性 MCP Server + AI Notebooks 博客文章](https://clickhouse.com/blog/observability-mcp-server-ai-notebooks#clickstack-mcp-server)。

  ### 可执行用户自定义函数

  {/* vale on */}

  现在，您可以用 Python 编写函数，将其打包压缩后上传至集群，并像调用任何内置函数一样在 SQL 中调用它。ClickHouse 维护一个长期运行的沙箱进程池，以查询速度将行通过管道传输其中，因此您可以在任何支持 SQL 的场景中调用该函数：即席查询、join 操作，乃至在每次插入时触发的 materialized view。您的模型代码与数据紧密共存，部署只需在 Cloud 控制台中完成一次上传即可。

  为了展示其功能，我们构建了一个 [Demo](https://github.com/ClickHouse/stock-anomaly-udf)。基于 StockHouse 中的实时股票交易数据，我们训练了一个小型 PyTorch 自编码器，在数据摄取时对股票交易 tick 进行异常评分。整个方案只需一个 UDF 和一条 DDL 语句，其余全部为普通 SQL。我们还抢先预览了网络访问 UDFs (私网 Beta) 功能——该功能允许 UDFs 发起出站 HTTPS 调用。我们借助这一特性拉取新闻和 SEC 文件，并让 Claude 对特定交易出现异常的原因进行分类。

  如需了解更多详情，请参阅[公告博客](https://clickhouse.com/blog/executable-udfs-clickhouse-cloud-beta)或[文档](/zh/products/cloud/features/sql-console-features/user-defined-functions)
</Update>

<Update label="2026 年 5 月 18 日">
  ### Python client v1.0.0

  [clickhouse-connect](/zh/integrations/language-clients/python/index) 的重大版本发布，这是 ClickHouse 的 Python client。主要亮点包括：

  * 一流的 SQLAlchemy 方言，更深入地支持 ClickHouse 特有 SQL
  * 基于 aiohttp 构建的原生异步 client，具备完整的功能对等性，包括流式支持
  * 在 日期时间、DateTime64、定宽数值插入、Maps、Decimal 和 BigDecimals 等方面实现了显著的性能提升
  * 通过新的 `tz_mode` 和 `tz_source` parameter 重新设计了时区处理机制
  * 原生支持写入 Variant column
  * 对可选依赖项 (numpy、pandas、pyarrow、polars) 采用延迟导入，使纯导入速度提升 4 倍
  * Experimental 的 Python 3.14 free-threading 支持

  **不兼容变更：** 不再支持 Python &lt; 3.9 和 Pandas 1.x。有关其他与时区相关的不兼容变更，请查阅[发布说明](https://github.com/ClickHouse/clickhouse-connect/releases)。

  ### ClickStack 中基于 SQL 的图表和告警

  [ClickStack](/zh/clickstack/overview) 现在支持使用任意 ClickHouse SQL 查询来构建图表和告警。用户可以直接在仪表盘和告警构建器中编写原始 SQL，完整支持 window functions、CTE、多阶段查询以及 ClickHouse 原生分析函数。SQL 告警支持将整个告警判定逻辑直接编码在查询本身中，从而实现异常检测、滞后平均值和统计尖峰检测。适用于所有 ClickStack 部署形态 (Cloud、私网、BYOC、OSS) 。更多详情请参见 [SQL-based visualizations](/zh/clickstack/features/dashboards/sql-visualizations) 和 [SQL alert result interpretation](/zh/clickstack/features/alerts#sql-result-interpretation)。

  ### 新区域支持

  ### Kafka ClickPipes 的 Protobuf 支持

  [Kafka ClickPipes](/zh/integrations/clickpipes/kafka/index) 现在除 Avro 和 JSON 外，还支持 [**Protobuf**](/zh/integrations/clickpipes/kafka/reference#protobuf) 格式。功能包括通过 Schema Registry 集成自动解析 schema、支持递归引用、well-known types 以及原生 `oneof` 类型。Protobuf 支持适用于所有 Kafka ClickPipes interface (UI、OpenAPI 和 Terraform) 。

  ### Postgres 查询洞察 (私有预览)

  [ClickHouse Cloud Managed Postgres](/zh/products/managed-postgres/overview) (私有预览) 现已新增 **Query insights** 选项卡。它会按影响程度对每种查询模式进行排序，并通过以下三个界面展示每种查询变慢的原因：

  * **Overview** — 一屏总览数据库健康状况
  * **Patterns** — 可按运行时长、CPU、错误和 P95 排序的表格
  * **Per-pattern flyout** — 延迟百分位数、CPU 与 I/O、缓存与磁盘、临时溢写、并行 worker 不足以及 WAL 量

  该功能由开源 Postgres 扩展 [pg&#95;stat&#95;ch](https://github.com/ClickHouse/pg_stat_ch) 提供支持。

<Update label="2026 年 4 月 10 日">
  ### 集中式市场计费

  客户现在可以通过单个 Marketplace 订阅 (AWS、Azure 和 GCP) ，在多个组织之间统一计费。主要更新包括：

  * **Marketplace 订阅共享：** 可将某个组织的计费方式从信用卡计费切换为另一个组织现有的 Marketplace 订阅，无需再额外设置新的 Marketplace 订阅。仅整合 PAYG 用量 (不包括预付费额度) 。

  * **备用付款方式：** 使用市场计费的组织现在可以添加信用卡作为备用付款方式，以便在主 Marketplace 订阅被取消或到期时作为后备。

  * **采用双窗口 recommender 的更智能自动扩缩容：** ClickHouse Cloud 现在在纵向自动扩缩容中采用双回看窗口机制，以 3 小时的“小窗口”和 30 小时的“大窗口”组合，替代此前单一的 30 小时窗口。这样可将缩容延迟从最长 30 小时缩短至最短 3 小时，在不牺牲扩容响应速度的前提下，降低波动性工作负载的基础设施成本。更多详情请阅读[博客文章](https://clickhouse.com/blog/smarter-auto-scaling)。
