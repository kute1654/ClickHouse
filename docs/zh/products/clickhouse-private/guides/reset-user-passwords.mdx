使用私网 API 重置 ClickHouse 用户密码。

<div id="prerequisites">
  ## 前置条件
</div>

* 已安装且可访问 ClickHouse 私网 API (例如，通过端口转发访问 `http://localhost:8080/`) 
* 至少已部署一个 ClickHouse 集群 (本指南使用 `default-xx-01`) 

<Steps>
  <Step>
    ## 1. 生成哈希密码

    使用 SHA-256 对新密码进行哈希计算，并将结果进行 base64 编码：

    ```ini
    PASSWORD='My super secret p@$$w0rd'
    HASHED_PASSWORD=$(echo -n "$PASSWORD" | shasum -a 256 | awk '{printf $1}' | base64)
    ```
  </Step>

  <Step>
    ## 2. 创建请求体

    创建一个包含哈希密码的 JSON 文件：

    ```json
    {
      "user_hashed_password": "<base64-encoded-sha256-hash>",
      "hashing_function": "sha256",
      "username": "default"
    }
    ```

    将其保存为 `password_reset.json`。

    | Field                  | Required | Default   | Description          |
    | ---------------------- | -------- | --------- | -------------------- |
    | `user_hashed_password` | 是        | --        | 新密码经过 Base64 编码后的哈希值 |
    | `hashing_function`     | 否        | `sha256`  | 所使用的哈希算法             |
    | `username`             | 否        | `default` | 要重置密码的用户名            |
  </Step>

  <Step>
    ## 3. 发送请求

    ```bash
    curl -X POST "http://localhost:8080/api/v1/instances/default-xx-01/reset-user-password" \
        -H "Content-Type: application/json" \
        --data-binary "@password_reset.json"
    ```

    成功响应示例如下：

    ```json
    {
      "message": "User password reset successfully"
    }
    ```
  </Step>

  <Step>
    ## 4. 验证变更

    密码重置会在 operator 处理完此次变更后生效。请使用新密码连接到集群进行验证。

    你也可以通过 status 端点监控该操作：

    ```bash
    curl "http://localhost:8080/api/v1/instances/default-xx-01/status"
    ```
  </Step>
</Steps>

<div id="hydra-limitation">
  ## Hydra 限制
</div>

无法在 Hydra 子实例上执行密码重置。如需重置 Hydra 子实例的密码，请改为在父实例上重置。
