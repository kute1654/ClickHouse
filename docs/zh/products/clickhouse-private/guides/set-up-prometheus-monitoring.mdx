使用 kube-prometheus-stack 和 ClickHouse Grafana mixin 配置 Prometheus 抓取 ClickHouse 指标，并通过预置的 Grafana 仪表盘进行可视化。

<div id="prerequisites">
  ## 前置条件
</div>

* 一个正在运行的 ClickHouse 私网部署
* 你的工作站上已安装并可用的 [Helm](https://helm.sh/)
* 拥有目标集群的 `kubectl` 访问权限
* 你的私有镜像仓库中已提供 kube-prometheus-stack Helm 图表及其容器镜像 (请参见下方的 [Airgap 镜像准备](#prepare-images-for-airgap))

<div id="steps">
  ## 步骤
</div>

<Steps>
  <Step>
    ### 1. 安装 kube-prometheus-stack

    将 Prometheus 社区的 Helm 仓库添加到本地 Helm 客户端，并拉取该 Helm chart：

    ```bash
    helm repo add prometheus-community https://prometheus-community.github.io/helm-charts
    helm repo update
    helm pull prometheus-community/kube-prometheus-stack --version <version>
    ```

    然后从你的 Helm 仓库安装它：

    ```bash
    helm install kube-prometheus-stack \
      oci://<your-registry>/helm-charts/kube-prometheus-stack \
      --version <version> \
      --namespace monitoring \
      --create-namespace \
      --values kube-prometheus-stack-values.yaml
    ```

    > **注意：**kube-prometheus-stack 集成了 Prometheus、Alertmanager、Grafana 和 Prometheus Operator。要查看必须镜像到您的私有镜像仓库中的完整镜像列表，请参阅 [Helm Chart 文档](https://github.com/prometheus-community/helm-charts/tree/main/charts/kube-prometheus-stack)。
  </Step>

  <Step>
    ### 2. 在 ClickHouse Operator 中启用 PodMonitor

    ClickHouse Operator 的 Helm 图表包含用于 ClickHouse Server 和 Keeper 的 PodMonitor 定义。请在 operator 的 Helm 配置值中启用它们：

    ```yaml
    metrics:
      enabled: true
      podmonitor:
        enabled: true
        scrape:
          clickhouseServer: true
          clickhouseKeeper: true
    ```

    通过升级 operator 版本来应用这些配置值：

    ```bash
    helm upgrade clickhouse-operator \
      oci://<your-registry>/helm-charts/clickhouse-operator \
      --version <version> \
      --namespace clickhouse-operator-system \
      --values operator-values.yaml
    ```

    这会创建两个 PodMonitor 资源：

    | PodMonitor                  | 目标                                                           | 端口                  | 指标                      |
    | --------------------------- | ------------------------------------------------------------ | ------------------- | ----------------------- |
    | `clickhouse-server-metrics` | 带有 `app.kubernetes.io/name: clickhouse-server` 标签的 Pod (容器组) | `prometheus` (8001) | 服务器指标，以及从容器镜像标签中提取的版本标签 |
    | `clickhouse-keeper-metrics` | 带有 `app.kubernetes.io/name: clickhouse-keeper` 标签的 Pod (容器组) | `prometheus` (8001) | Keeper 指标               |

    这两个 PodMonitor 都带有 `release: kube-prometheus-stack` 标签，与 kube-prometheus-stack 的 Prometheus 实例默认的 `podMonitorSelector` 匹配。

    > **注意：** 如果你的 Prometheus 实例使用的是其他 release 名称，请相应更新 PodMonitor 的标签选择器：可以覆盖 operator chart 模板，或在 kube-prometheus-stack 配置值中配置 `prometheus.prometheusSpec.podMonitorSelector`。
  </Step>

  <Step>
    ### 3. 启用自定义指标处理程序 (可选)

    ClickHouse Server 通过 3123 端口上的专用 HTTP 处理程序暴露额外的 `ClickHouse_CustomMetric_*` 指标。此端点需要身份验证。要启用抓取：

    ```yaml
    metrics:
      enabled: true
      podmonitor:
        enabled: true
        scrape:
          clickhouseServer: true
          clickhouseServerCustomHandler: true
          clickhouseKeeper: true
        basicAuth:
          username: prometheus-internal
          password: "<password>"
    ```

    Operator 会将凭据存储在 Kubernetes Secret 中，并在 PodMonitor 的 `basicAuth` 配置中引用该 Secret。

    有关所有自定义指标及其推荐告警阈值的完整列表，请参阅[指标和告警参考](/zh/products/clickhouse-private/reference/metrics-and-alerts-reference)。
  </Step>

  <Step>
    ### 4. 验证 Prometheus 目标

    部署完成后，确认 Prometheus 已发现并抓取 ClickHouse 目标：

    ```bash
    kubectl port-forward -n monitoring svc/kube-prometheus-stack-prometheus 9090:9090
    ```

    在浏览器中打开 `http://localhost:9090/targets`。查找名为 `podMonitor/clickhouse-operator-system/clickhouse-server-metrics` 和 `podMonitor/clickhouse-operator-system/clickhouse-keeper-metrics` 的目标组。所有目标的 **State** 都应显示为 `UP`。

    运行测试查询，确认指标正在正常采集：

    ```text
    up{job="clickhouse-server-metrics"}
    ```
  </Step>

  <Step>
    ### 5. 导入 ClickHouse Grafana Mixin 仪表盘

    [ClickHouse Grafana mixin](https://grafana.com/docs/grafana-cloud/monitor-infrastructure/integrations/integration-reference/integration-clickhouse/) 为 ClickHouse server 和 Keeper 指标提供了预置仪表盘。在隔离环境中，请手动导入这些仪表盘的 JSON 文件。

    #### 下载仪表盘

    从 [Grafana 集成页面](https://grafana.com/docs/grafana-cloud/monitor-infrastructure/integrations/integration-reference/integration-clickhouse/) 下载 mixin 仪表盘的 JSON 文件，或从现有的 Grafana 实例中导出这些文件。

    #### 通过 Grafana UI 导入

    1. 打开 Grafana (随 kube-prometheus-stack 一起部署) ：

       ```bash
       kubectl port-forward -n monitoring svc/kube-prometheus-stack-grafana 3000:80
       ```

    2. 在 `http://localhost:3000` 登录 (默认凭据：`admin` / `prom-operator`) 。

    3. 前往 **Dashboards &gt; Import**。

    4. 上传每个仪表盘的 JSON 文件，或粘贴其内容。

    5. 在提示时选择你的 Prometheus 数据源。

    #### 通过 ConfigMap 导入

    如需以代码方式管理仪表盘，请在 monitoring 命名空间中创建一个带有 Grafana sidecar 标签的 ConfigMap：

    ```yaml
    apiVersion: v1
    kind: ConfigMap
    metadata:
      name: clickhouse-grafana-dashboards
      namespace: monitoring
      labels:
        grafana_dashboard: "1"
    data:
      clickhouse-overview.json: |
        { ... dashboard JSON ... }
    ```

    Grafana sidecar 会自动发现带有 `grafana_dashboard: "1"` 标签的 ConfigMaps，并加载其中的仪表盘。
  </Step>

  <Step>
    ### 6. 配置告警规则 (可选)

    通过创建 `PrometheusRule` 资源，将推荐的 ClickHouse 告警规则导入到 Prometheus 中。完整的告警定义请参阅[配置告警](/zh/products/clickhouse-private/guides/configure-alerting-for-clickhouse)和[指标与告警参考](/zh/products/clickhouse-private/reference/metrics-and-alerts-reference)。

    示例：

    ```yaml
    apiVersion: monitoring.coreos.com/v1
    kind: PrometheusRule
    metadata:
      name: clickhouse-alerts
      namespace: monitoring
      labels:
        release: kube-prometheus-stack
    spec:
      groups:
        - name: clickhouse-server
          rules:
            - alert: ClickhouseOperatorNotReconciling
              expr: avg(increase(last_cluster_reconcile[90m])) by (app) == 0
              for: 120m
              labels:
                severity: warning
              annotations:
                summary: "Operator has not reconciled {{ $labels.app }} in 2 hours"
            - alert: ClickHouseDataLoss
              expr: ClickHouse_CustomMetric_LostPartCount > 0
              labels:
                severity: critical
              annotations:
                summary: "Lost parts detected on {{ $labels.instance }}"
    ```
  </Step>
</Steps>

<div id="prepare-images-for-airgap">
  ## 为 Airgap 环境准备镜像
</div>

kube-prometheus-stack 需要以下容器镜像。请在安装前将它们同步到你的私有镜像仓库：

| 组件                  | 镜像                                                      |
| ------------------- | ------------------------------------------------------- |
| Prometheus          | `quay.io/prometheus/prometheus`                         |
| Alertmanager        | `quay.io/prometheus/alertmanager`                       |
| Grafana             | `docker.io/grafana/grafana`                             |
| Grafana sidecar     | `quay.io/kiwigrid/k8s-sidecar`                          |
| Prometheus Operator | `quay.io/prometheus-operator/prometheus-operator`       |
| kube-state-metrics  | `registry.k8s.io/kube-state-metrics/kube-state-metrics` |
| node-exporter       | `quay.io/prometheus/node-exporter`                      |

> **注意：**具体的镜像标签取决于你要部署的 kube-prometheus-stack 图表版本。对已拉取的图表运行 `helm template`，即可提取该版本对应的准确镜像引用。
