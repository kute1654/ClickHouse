使用 Private API 调整 ClickHouse 集群的 CPU 和内存资源。

<div id="prerequisites">
  ## 前置条件
</div>

* 已安装并可访问 ClickHouse Private API (例如，通过到 `http://localhost:8080/` 的端口转发) 
* 至少已部署一个 ClickHouse 集群 (本指南使用 `default-xx-01`) 

<Steps>
  <Step>
    ## 1. 创建扩缩容请求

    创建一个包含所需资源分配的 JSON 文件：

    ```json
    {
      "vertical": {
        "resources": {
          "cpu": "4",
          "memory": "16Gi"
        }
      }
    }
    ```

    将其保存为 `scale.json`。
  </Step>

  <Step>
    ## 2. 发送请求

    ```bash
    curl -X POST "http://localhost:8080/api/v1/instances/default-xx-01/scale" \
        -H "Content-Type: application/json" \
        --data-binary "@scale.json"
    ```
  </Step>

  <Step>
    ## 3. CPU:内存比例限制

    API 会强制执行 **1:4 的 CPU 与内存比例** (每 4 GiB 内存对应 1 个 CPU 核心) ，并允许 5% 的误差范围。

    如果你只提供一种资源类型，API 会自动推导另一种。例如，仅指定 `"memory": "16Gi"` 时，CPU 会被设置为 4 核。

    **支持的资源数量格式：**

    | 类型  | 示例                                            |
    | --- | --------------------------------------------- |
    | CPU | `"4"` (核心) 、`"2.5"` (小数) 、`"500m"` (毫核)       |
    | 内存  | `"16Gi"` (gibibytes) 、`"4096Mi"` (mebibytes)  |

    对于内存，建议使用二进制单位 (Ki、Mi、Gi) 。
  </Step>

  <Step>
    ## 4. 监控扩缩容进度

    检查集群状态以跟踪扩缩容操作：

    ```bash
    curl "http://localhost:8080/api/v1/instances/default-xx-01/status"
    ```

    响应包含：- `state`：当前集群状态 - `previousState`：状态变更前的状态 - `message`：有关状态变更的详细信息

    当集群返回运行状态时，扩缩容即完成。
  </Step>
</Steps>

<div id="rolling-restart-behavior">
  ## 滚动重启机制
</div>

API 会更新 `ClickhouseCluster` 自定义资源中的 `ServerPodPolicy`。随后，Operator 会基于新的资源分配对 StatefulSets 执行滚动重启。

> **重要：** 与 ClickHouse Cloud 不同，ClickHouse Private 不使用 [Make Before Break (MBB) scaling](https://clickhouse.com/docs/manage/scaling#how-scaling-works-in-clickhouse-cloud)。垂直扩缩容在滚动重启期间可能会导致短暂的服务中断。
