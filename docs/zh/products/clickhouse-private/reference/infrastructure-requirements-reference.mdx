<div id="operator-node-group-x86">
  #### Operator 节点组 (x86)
</div>

| 属性     | 值                                    |
| ------ | ------------------------------------ |
| AMI    | `AL2023_x86_64`                      |
| 磁盘大小   | 20 GB                                |
| 最小实例规格 | xlarge                               |

自动扩缩器标签 (如果使用 cluster-autoscaler) ：- `k8s.io/cluster-autoscaler/enabled: true` - `k8s.io/cluster-autoscaler/$CLUSTER_NAME: owned`

<div id="iam-service-account-requirements">
  ## IAM / 服务账号要求
</div>

<div id="aws">
  ### AWS
</div>

<div id="ecr-pull-role">
  #### ECR 拉取角色
</div>

* **角色名称：** `ClickHouseAirgapECRPullRole`
* **用途：** 从 ClickHouse 私有 ECR 拉取镜像
* **权限：** `ecr:GetDownloadUrlForLayer`, `ecr:BatchGetImage`, `ecr:BatchCheckLayerAvailability`, `ecr:DescribeImages`, `ecr:ListImages`, `ecr:GetAuthorizationToken`
* **操作：** 将该角色的 ARN 提供给 ClickHouse 团队

<div id="clickhouse-serverkeeper-s3-role">
  #### ClickHouse Server/Keeper S3 角色
</div>

* **每个已预配的 ClickHouse 集群对应一个角色**
* **命名约定：** `CH-S3-$NAME-$REGION-$ORDINAL-Role`
* `$NAME`：集群名称 (例如：`default-xx-01`) 
* `$REGION`：区域缩写 (例如：`us-west-2` 缩写为 `uw2`) 
* `$ORDINAL`：预留字段，设置为 `00`
* 示例：`CH-S3-default-xx-01-uw2-00-Role`
* **权限：** 对 bucket 资源授予 `s3:*` 和 `s3:ListBucket`
* **信任策略：** 通过 IRSA 使用 `sts:AssumeRoleWithWebIdentity`，信任 EKS 集群的 OIDC 提供商

<div id="ebs-csi-driver-role">
  #### EBS CSI 驱动程序角色
</div>

* **示例名称：** `ClickHouse_EksEbsCsiDriverRole`
* **托管策略：** `arn:aws:iam::aws:policy/service-role/AmazonEBSCSIDriverPolicy`
* **信任策略：** 采用 EKS 集群的 OIDC 提供商的 IRSA

<div id="cluster-naming-conventions">
  ## 集群命名规范
</div>

ClickHouse 集群的命名规范在所有平台上均保持一致：

```text
$DESCRIPTOR-$LETTERS-$ORDINAL
```

| 组件            | 描述             | 示例        |
| ------------- | -------------- | --------- |
| `$DESCRIPTOR` | 描述名称 (仅限字母)    | `default` |
| `$LETTERS`    | 保留；任意两个字母      | `xx`      |
| `$ORDINAL`    | 从 `01` 开始递增的序号 | `01`      |

**示例名称：** `default-xx-01`

集群名称必须在**目标 Kubernetes 集群中保持唯一**，并用于： - Kubernetes 命名空间：`ns-$CLUSTER_NAME` - Kubernetes 资源：以 `c-$CLUSTER_NAME` 为前缀 - IAM 角色名称 (AWS) ：`CH-S3-$CLUSTER_NAME-$REGION-$ORDINAL-Role` - S3/GCS 键前缀：`ch-s3-<uuid>` (每个集群唯一)
