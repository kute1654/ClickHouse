<div id="h3-index">
  ## H3 索引
</div>

[H3](https://h3geo.org/) 是一种地理空间索引系统，它将地球表面划分为由大小一致的六边形单元组成的网格。该系统具有层级结构，也就是说，顶层的每个六边形 (“父级”) 都可以细分为七个同样规则但更小的六边形 (“子级”) ，以此类推。

该层级称为 `resolution`，取值范围为 `0` 到 `15`，其中 `0` 是 `base` 级别，对应最大且最粗粒度的单元。

一组纬度和经度坐标可以转换为一个 64 位 H3 index，用于标识某个网格单元。

H3 index 主要用于对位置进行分桶以及其他地理空间操作。

有关 H3 系统的完整说明，请参阅 [Uber Engineering 网站](https://www.uber.com/blog/h3/)。

<div id="h3isvalid">
  ## h3IsValid
</div>

检查该数字是否为有效的 [H3](#h3-index) 索引。

**语法**

```sql
h3IsValid(h3index)
```

**参数**

* `h3index` — 六边形索引编号。[UInt64](/zh/reference/data-types/int-uint)。

**返回值**

* 1 — 该数值为有效的 H3 索引。[UInt8](/zh/reference/data-types/int-uint)。
* 0 — 该数值不是有效的 H3 索引。[UInt8](/zh/reference/data-types/int-uint)。

**示例**

```sql title="Query"
SELECT h3IsValid(630814730351855103) AS h3IsValid;
```

```text title="Response"
┌─h3IsValid─┐
│         1 │
└───────────┘
```

<div id="h3getresolution">
  ## h3GetResolution
</div>

返回给定 [H3](#h3-index) 索引的分辨率。

**语法**

```sql
h3GetResolution(h3index)
```

**参数**

* `h3index` — 六边形索引编号。[UInt64](/zh/reference/data-types/int-uint)。

**返回值**

* 索引分辨率。取值范围：`[0, 15]`。[UInt8](/zh/reference/data-types/int-uint)。
* 如果索引无效，函数会返回一个随机值。请使用 [h3IsValid](#h3isvalid) 验证索引。[UInt8](/zh/reference/data-types/int-uint)。

**示例**

```sql title="Query"
SELECT h3GetResolution(639821929606596015) AS resolution;
```

```text title="Response"
┌─resolution─┐
│         14 │
└────────────┘
```

<div id="geotoh3">
  ## geoToH3
</div>

返回具有指定分辨率的 [H3](#h3-index) 点索引 `(lat, lon)`。

**语法**

```sql
geoToH3(lat, lon, resolution)
```

**参数**

* `lat` — 纬度。[Float64](/zh/reference/data-types/float)。
* `lon` — 经度。[Float64](/zh/reference/data-types/float)。
* `resolution` — 索引分辨率。范围：`[0, 15]`。[UInt8](/zh/reference/data-types/int-uint)。

**返回值**

* 六边形索引编号。[UInt64](/zh/reference/data-types/int-uint)。
* 出错时返回 0。[UInt64](/zh/reference/data-types/int-uint)。

注意：在 ClickHouse v25.4 及更早版本中，`geoToH3()` 接受的参数顺序为 `(lon, lat)`。从 ClickHouse v25.5 起，输入值的顺序为 `(lat, lon)`。可使用设置 `geotoh3_argument_order = 'lon_lat'` 恢复此前的行为。

**示例**

```sql title="Query"
SELECT geoToH3(55.71290588, 37.79506683, 15) AS h3Index;
```

```text title="Response"
┌────────────h3Index─┐
│ 644325524701193974 │
└────────────────────┘
```

<div id="h3polygontocellswithcontainment">
  ## h3PolygonToCellsWithContainment
</div>

使用 H3 Experimental 包含模式，返回覆盖给定几何体 (Ring 或 (多) Polygon) 且具有指定分辨率的六边形单元。

此函数将 `flags` 映射为 H3 包含模式：

* `0` — `CONTAINMENT_CENTER`
* `1` — `CONTAINMENT_FULL`
* `2` — `CONTAINMENT_OVERLAPPING`
* `3` — `CONTAINMENT_OVERLAPPING_BBOX`

**语法**

```sql
h3PolygonToCellsWithContainment(geometry, resolution, flags)
```

**参数**

* `geometry` 可以是以下 [Geo 数据类型](/zh/reference/data-types/geo) 之一，或其底层基本类型：
  * [Ring](/zh/reference/data-types/geo#ring)
  * [Polygon](/zh/reference/data-types/geo#polygon)
  * [MultiPolygon](/zh/reference/data-types/geo#multipolygon)
* `resolution` — 索引分辨率。范围：`[0, 15]`。[UInt8](/zh/reference/data-types/int-uint)。
* `flags` — 包含模式。范围：`[0, 3]`。使用整数字面量，例如 `0`、`1`、`2` 和 `3`，或 [UInt32](/zh/reference/data-types/int-uint) 值 (例如 `toUInt32(2)`) 。这些值会作为 `UInt32` 传递给 H3 API。其他原生整数类型会通过精确类型转换进行转换。

**返回值**

* 根据所选包含模式返回的 H3 索引数组。[Array](/zh/reference/data-types/array)([UInt64](/zh/reference/data-types/int-uint))。

**示例**

```sql title="Query"
SELECT
    h3PolygonToCellsWithContainment([(-122.4089866999972145, 37.813318999983238), (-122.3544736999993603, 37.7198061999978478), (-122.4798767000009008, 37.8151571999998453)], 7, 0) AS center_cells,
```
