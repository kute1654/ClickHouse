<div id="description">
  ## 描述
</div>

此表会输出有关 ZooKeeper 的汇总内部信息，节点取自配置文件。

<div id="columns">
  ## 列
</div>

{/*AUTOGENERATED_START*/}

* `zookeeper_cluster_name` ([String](/zh/reference/data-types/string)) — ZooKeeper 集群名称。
* `host` ([String](/zh/reference/data-types/string)) — ClickHouse 所连接的 ZooKeeper 节点的主机名/IP。
* `port` ([UInt16](/zh/reference/data-types/int-uint)) — ClickHouse 所连接的 ZooKeeper 节点端口。
* `index` ([Nullable(UInt8)](/zh/reference/data-types/int-uint)) — ClickHouse 连接到的 ZooKeeper 节点的索引。该索引来自 ZooKeeper 配置。如果未连接，此列为 NULL。
* `is_connected` ([Nullable(UInt8)](/zh/reference/data-types/int-uint)) — zookeeper 是否处于连接状态。
* `is_readonly` ([UInt8](/zh/reference/data-types/int-uint)) — 是否为只读状态。
* `version` ([String](/zh/reference/data-types/string)) — ZooKeeper 版本。
* `avg_latency` ([UInt64](/zh/reference/data-types/int-uint)) — 平均延迟。
* `max_latency` ([UInt64](/zh/reference/data-types/int-uint)) — 最大延迟时间。
* `min_latency` ([UInt64](/zh/reference/data-types/int-uint)) — 最低延迟。
* `packets_received` ([UInt64](/zh/reference/data-types/int-uint)) — 接收的数据包数量。
* `packets_sent` ([UInt64](/zh/reference/data-types/int-uint)) — 发送的数据包数量。
* `outstanding_requests` ([UInt64](/zh/reference/data-types/int-uint)) — 未完成的请求数。
* `server_state` ([String](/zh/reference/data-types/string)) — 服务器状态。
* `is_leader` ([UInt8](/zh/reference/data-types/int-uint)) — 是否为 zookeeper leader。
* `znode_count` ([UInt64](/zh/reference/data-types/int-uint)) — znode 数量。
* `watch_count` ([UInt64](/zh/reference/data-types/int-uint)) — watch 的数量。
* `ephemerals_count` ([UInt64](/zh/reference/data-types/int-uint)) — 临时列的数量。
* `approximate_data_size` ([UInt64](/zh/reference/data-types/int-uint)) — 估算数据大小。
* `followers` ([UInt64](/zh/reference/data-types/int-uint)) — leader 的跟随者。此字段仅由 leader 公开。
* `synced_followers` ([UInt64](/zh/reference/data-types/int-uint)) — leader 的已同步跟随者。该字段仅由 leader 提供。
* `pending_syncs` ([UInt64](/zh/reference/data-types/int-uint)) — leader 的待处理同步数。此字段仅在 leader 上可见。
* `open_file_descriptor_count` ([UInt64](/zh/reference/data-types/int-uint)) — 打开的文件描述符数量。仅适用于 Unix 平台。
* `max_file_descriptor_count` ([UInt64](/zh/reference/data-types/int-uint)) — 最大文件描述符数。仅在 Unix 平台上可用。
* `connections` ([UInt64](/zh/reference/data-types/int-uint)) — 与 ZooKeeper 的连接。
* `outstanding` ([UInt64](/zh/reference/data-types/int-uint)) — ZooKeeper 的 outstanding 值。
* `zxid` ([UInt64](/zh/reference/data-types/int-uint)) — ZooKeeper 的 zxid。
* `node_count` ([UInt64](/zh/reference/data-types/int-uint)) — ZooKeeper 节点数。
* `snapshot_dir_size` ([UInt64](/zh/reference/data-types/int-uint)) — ZooKeeper 快照目录的大小。
* `log_dir_size` ([UInt64](/zh/reference/data-types/int-uint)) — ZooKeeper 日志目录的大小。
* `first_log_idx` ([UInt64](/zh/reference/data-types/int-uint)) — ZooKeeper 的首个日志索引。
* `first_log_term` ([UInt64](/zh/reference/data-types/int-uint)) — ZooKeeper 的首个日志任期。
* `last_log_idx` ([UInt64](/zh/reference/data-types/int-uint)) — ZooKeeper 的最后日志索引。
* `last_log_term` ([UInt64](/zh/reference/data-types/int-uint)) — ZooKeeper 的最新日志任期。
* `last_committed_idx` ([UInt64](/zh/reference/data-types/int-uint)) — ZooKeeper 的最近一次已提交索引。
* `leader_committed_log_idx` ([UInt64](/zh/reference/data-types/int-uint)) — ZooKeeper leader 已提交的日志索引。
* `target_committed_log_idx` ([UInt64](/zh/reference/data-types/int-uint)) — ZooKeeper 的目标已提交日志索引。
* `last_snapshot_idx` ([UInt64](/zh/reference/data-types/int-uint)) — ZooKeeper 的最新快照索引。
