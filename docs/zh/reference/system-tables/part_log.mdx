<div id="description">
  ## 描述
</div>

仅当指定了 [part&#95;log](/zh/reference/settings/server-settings/settings#part_log) 服务器设置时，才会创建 `system.part_log` 表。

此表包含 [MergeTree](/zh/reference/engines/table-engines/mergetree-family/mergetree) 家族表中与 [数据分区片段](/zh/reference/engines/table-engines/mergetree-family/custom-partitioning-key) 相关的事件信息，例如添加数据或合并数据。

`system.part_log` 表包含以下列：

<div id="columns">
  ## 列
</div>

{/*AUTOGENERATED_START*/}

* `hostname` ([LowCardinality(String)](/zh/reference/data-types/lowcardinality)) — 执行该查询的服务器主机名。
* `query_id` ([String](/zh/reference/data-types/string)) — 创建此数据分区片段的 INSERT 查询标识符。
* `event_type` ([Enum8(&#39;NewPart&#39; = 1, &#39;MergeParts&#39; = 2, &#39;DownloadPart&#39; = 3, &#39;RemovePart&#39; = 4, &#39;MutatePart&#39; = 5, &#39;MovePart&#39; = 6, &#39;MergePartsStart&#39; = 7, &#39;MutatePartStart&#39; = 8)](/zh/reference/data-types/enum)) — 发生在数据分区片段上的事件类型。可能具有以下值之一：NewPart — 插入新的数据分区片段，MergePartsStart — 数据分区片段的合并已开始，MergeParts — 数据分区片段的合并已完成，DownloadPart — 下载数据分区片段，RemovePart — 使用 [DETACH PARTITION](/zh/reference/statements/alter/partition#detach-partitionpart) 删除或分离数据分区片段。MutatePartStart — 数据分区片段的变更已开始，MutatePart — 数据分区片段的变更已完成，MovePart — 将数据分区片段从一个磁盘移动到另一个磁盘。
* `merge_reason` ([Enum8(&#39;NotAMerge&#39; = 1, &#39;RegularMerge&#39; = 2, &#39;TTLDeleteMerge&#39; = 3, &#39;TTLRecompressMerge&#39; = 4, &#39;TTLDropMerge&#39; = 5)](/zh/reference/data-types/enum)) — `MERGE_PARTS` 类型事件发生的原因。可能具有以下值之一：NotAMerge — 当前事件的类型不是 `MERGE_PARTS`；RegularMerge — 常规合并；TTLDeleteMerge、TTLDropMerge — 清理过期数据；TTLRecompressMerge — 重新压缩数据分区片段。
* `merge_algorithm` ([Enum8(&#39;Undecided&#39; = 0, &#39;Vertical&#39; = 1, &#39;Horizontal&#39; = 2)](/zh/reference/data-types/enum)) — 类型为 MERGE&#95;PARTS 的事件所使用的合并算法。可取以下值之一：Undecided、Horizontal、Vertical
* `event_date` ([Date](/zh/reference/data-types/date)) — 事件日期。
* `event_time` ([DateTime](/zh/reference/data-types/datetime)) — 事件发生时间。
* `event_time_microseconds` ([DateTime64(6)](/zh/reference/data-types/datetime64)) — 精确到微秒的事件时间。
* `duration_ms` ([UInt64](/zh/reference/data-types/int-uint)) — 此操作的耗时。
* `database` ([String](/zh/reference/data-types/string)) — 该数据分区片段所在数据库的名称。
* `table` ([String](/zh/reference/data-types/string)) — 数据分区片段所在表的名称。
* `table_uuid` ([UUID](/zh/reference/data-types/uuid)) — 该数据分区片段所属的表的 UUID。
* `part_name` ([String](/zh/reference/data-types/string)) — 数据分区片段名称。
* `partition_id` ([String](/zh/reference/data-types/string)) — 数据分区片段被插入到的分区 ID。若按 `tuple()` 进行分区，则该列的值为 `all`。
* `partition` ([String](/zh/reference/data-types/string)) — 分区名称。
* `part_type` ([String](/zh/reference/data-types/string)) — 分片类型。可选值：Wide 和 Compact。
* `part_storage_type` ([String](/zh/reference/data-types/string)) — DataPartStorage 的类型。可能的值：Packed - 所有文件都存储在同一个 blob 中，Full - 每个文件对应一个 blob。
* `disk_name` ([String](/zh/reference/data-types/string)) — 数据分区片段所在磁盘的名称。
* `path_on_disk` ([String](/zh/reference/data-types/string)) — 存放数据分区片段文件的文件夹的绝对路径。
* `rows` ([UInt64](/zh/reference/data-types/int-uint)) — 数据分区片段中的行数。
* `size_in_bytes` ([UInt64](/zh/reference/data-types/int-uint)) — 数据分区片段在磁盘上占用的大小 (以字节为单位) 。
* `merged_from` ([Array(String)](/zh/reference/data-types/array)) — 包含组成当前 part 的源 parts 名称的数组。
* `bytes_uncompressed` ([UInt64](/zh/reference/data-types/int-uint)) — 结果分片的未压缩大小 (字节) 。
* `read_rows` ([UInt64](/zh/reference/data-types/int-uint)) — 合并过程中读取的行数。
* `read_bytes` ([UInt64](/zh/reference/data-types/int-uint)) — 合并期间读取的字节数。
* `peak_memory_usage` ([UInt64](/zh/reference/data-types/int-uint)) — 合并期间的最大 RAM 使用量
* `deduplication_block_ids` ([Array(String)](/zh/reference/data-types/array)) — 在插入此分片时用于去重的块 ID 数组。
* `error` ([UInt16](/zh/reference/data-types/int-uint)) — 已发生的异常的错误代码。
* `exception` ([String](/zh/reference/data-types/string)) — 已发生错误的文本消息。
* `mutation_ids` ([Array(String)](/zh/reference/data-types/array)) — 对于类型为 MUTATE&#95;PART&#95;START 和 MUTATE&#95;PART 的事件，这是应用到源分片 (merged&#95;from) 上的变更 ID 数组。
* `ProfileEvents` ([Map(LowCardinality(String), UInt64)](/zh/reference/data-types/map)) — 此操作期间记录的所有 profile events。
* `projections_duration_ms` ([Map(LowCardinality(String), UInt64)](/zh/reference/data-types/map)) — 各投影合并/重建的耗时 (毫秒) 。

**别名：**

* `ProfileEvents.Names` — `mapKeys(ProfileEvents)` 的别名。
* `ProfileEvents.Values` — `mapValues(ProfileEvents)` 的别名。
* `name` — `part_name` 的别名。

{/*AUTOGENERATED_END*/}

`system.part_log` 表会在首次向 `MergeTree` 表中插入数据后创建。
