<div id="description">
  ## 描述
</div>

包含已执行 drop table 操作但尚未完成数据清理的表的信息。

<div id="columns">
  ## 列
</div>

{/*AUTOGENERATED_START*/}

* `index` ([UInt32](/zh/reference/data-types/index)) — `marked_dropped_tables` 队列中的索引。
* `database` ([String](/zh/reference/data-types/index)) — 数据库名称。
* `table` ([String](/zh/reference/data-types/index)) — 表名。
* `uuid` ([UUID](/zh/reference/data-types/index)) — 表 UUID。
* `engine` ([String](/zh/reference/data-types/index)) — 表引擎名称。
* `metadata_dropped_path` ([String](/zh/reference/data-types/index)) — 表的元数据文件在 `metadata_dropped` 目录中的路径。
* `table_dropped_time` ([DateTime](/zh/reference/data-types/index)) — 计划下一次尝试删除表数据的时间。通常为删除该表的时间加上 `database_atomic_delay_before_drop_table_sec`。

{/*AUTOGENERATED_END*/}

<div id="example">
  ## 示例
</div>

以下示例说明如何获取 `dropped_tables` 的相关信息。

```sql
SELECT *
FROM system.dropped_tables\G
```
