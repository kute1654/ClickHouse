<div id="description">
  ## 描述
</div>

该系统表包含 ClickHouse 中现有 Iceberg 表的按文件划分的元数据；每个表当前快照引用的每个数据文件或删除文件各对应一行。如果 ClickHouse 中没有任何 Iceberg 表，该表将为空。

<div id="columns">
  ## 列
</div>

{/*AUTOGENERATED_START*/}

* `database` ([String](/zh/reference/data-types/index)) — 数据库名称。
* `table` ([String](/zh/reference/data-types/index)) — 表名称。
* `snapshot_id` ([Int64](/zh/reference/data-types/index)) — 添加该文件时对应的快照 ID。
* `content` ([Enum8(&#39;DATA&#39; = 0, &#39;POSITION&#95;DELETE&#39; = 1, &#39;EQUALITY&#95;DELETE&#39; = 2)](/zh/reference/data-types/index)) — 文件内容类型。
* `file_path` ([String](/zh/reference/data-types/index)) — 文件解析得到的存储路径。
* `file_format` ([String](/zh/reference/data-types/index)) — 文件格式，例如 &#39;PARQUET&#39;。
* `record_count` ([Int64](/zh/reference/data-types/index)) — 文件中的记录数。
* `file_size_in_bytes` ([Int64](/zh/reference/data-types/index)) — 文件大小 (字节) 。
* `partition` ([String](/zh/reference/data-types/index)) — 分区 Tuple 的文本表示。
* `schema_id` ([Int32](/zh/reference/data-types/index)) — 为此清单条目解析得到的 schema ID。
* `sequence_number` ([Int64](/zh/reference/data-types/index)) — 清单条目解析得到的序列号 (格式 v1 始终为 0) 。
* `sort_order_id` ([Nullable(Int32)](/zh/reference/data-types/index)) — 文件的排序顺序 ID (如果已指定) 。
* `null_value_counts` ([Map(Int32, Int64)](/zh/reference/data-types/index)) — 各列的 NULL 值计数 (列 id -&gt; 计数) 。
* `column_sizes` ([Map(Int32, Int64)](/zh/reference/data-types/index)) — 各列在磁盘上的大小 (字节)  (列 id -&gt; 字节数) 。
* `value_counts` ([Map(Int32, Int64)](/zh/reference/data-types/index)) — 各列的总值计数 (列 id -&gt; 计数) 。
* `equality_ids` ([Array(Int32)](/zh/reference/data-types/index)) — 等值删除文件的等值字段 ID (非等值删除文件为空) 。

{/*AUTOGENERATED_END*/}
