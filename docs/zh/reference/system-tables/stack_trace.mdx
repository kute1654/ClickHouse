<div id="description">
  ## 描述
</div>

包含服务器所有线程的堆栈跟踪，供开发者查看服务器内部状态。

要分析堆栈帧，请使用 `addressToLine`、`addressToLineWithInlines`、`addressToSymbol` 和 `demangle` [内部信息函数](/zh/reference/functions/regular-functions/introspection)。

<div id="columns">
  ## 列
</div>

{/*AUTOGENERATED_START*/}

* `thread_name` ([String](/zh/reference/data-types/index)) — 线程名称。
* `thread_id` ([UInt64](/zh/reference/data-types/index)) — 线程标识符
* `query_id` ([String](/zh/reference/data-types/index)) — 此线程所属查询的 ID。
* `trace` ([Array(UInt64)](/zh/reference/data-types/index)) — 此线程的 堆栈跟踪，本质上就是一个地址数组。
* `untracked_memory` ([Int64](/zh/reference/data-types/index)) — 线程级的非原子内存分配计数器，尚未传播到上级 MemoryTracker。若自上次 flush 以来释放的内存多于分配的内存，其值可能为负。

{/*AUTOGENERATED_END*/}

<div id="example">
  ## 示例
</div>

启用内部信息函数：

```sql
SET allow_introspection_functions = 1;
```

从 ClickHouse 目标文件中获取符号：

```sql
WITH arrayMap(x -> demangle(addressToSymbol(x)), trace) AS all SELECT thread_name, thread_id, query_id, arrayStringConcat(all, '\n') AS res FROM system.stack_trace LIMIT 1;
```

获取 ClickHouse 源代码中的文件名和行号：

```sql
WITH arrayMap(x -> addressToLine(x), trace) AS all, arrayFilter(x -> x LIKE '%/dbms/%', all) AS dbms SELECT thread_name, thread_id, query_id, arrayStringConcat(notEmpty(dbms) ? dbms : all, '\n') AS res FROM system.stack_trace LIMIT 1;
```
