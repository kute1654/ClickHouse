<div id="description">
  ## 描述
</div>

包含驻留在本地服务器上的[调度节点](/zh/concepts/features/configuration/server-config/workload-scheduling#hierarchy)的相关信息和状态。
此表可用于监控。表中每个调度节点对应一行。

<div id="columns">
  ## 列
</div>

{/*AUTOGENERATED_START*/}

* `resource` ([String](/zh/reference/data-types/index)) — 资源名称
* `path` ([String](/zh/reference/data-types/index)) — 该资源调度层级内调度节点的路径
* `type` ([String](/zh/reference/data-types/index)) — 调度节点的类型。
* `weight` ([Float64](/zh/reference/data-types/index)) — 节点的权重，由 `fair` 类型的父节点使用。
* `priority` ([Int64](/zh/reference/data-types/index)) — 节点的优先级，由 &#39;priority&#39; 类型的父节点使用 (值越小表示优先级越高) 。
* `is_active` ([UInt8](/zh/reference/data-types/index)) — 此节点当前是否处于活跃状态——有等待出队的资源请求，且约束条件已满足。
* `active_children` ([UInt64](/zh/reference/data-types/index)) — 处于活跃状态的子节点数量。
* `dequeued_requests` ([UInt64](/zh/reference/data-types/index)) — 从该节点出队的资源请求总数。
* `canceled_requests` ([UInt64](/zh/reference/data-types/index)) — 从该节点取消的资源请求总数。
* `dequeued_cost` ([Int64](/zh/reference/data-types/index)) — 从该节点出队的所有请求的成本总和 (例如以字节为单位的大小) 。
* `throughput` ([Float64](/zh/reference/data-types/index)) — 当前平均吞吐量 (每秒出队成本) 。
* `canceled_cost` ([Int64](/zh/reference/data-types/index)) — 从该节点取消的所有请求的成本总和 (例如以字节为单位的大小) 。
* `busy_periods` ([UInt64](/zh/reference/data-types/index)) — 该节点停用的总次数。
* `vruntime` ([Nullable(Float64)](/zh/reference/data-types/index)) — 仅适用于 `fair` 节点的子节点。节点的虚拟运行时间，由 SFQ 算法用于选择下一个要处理的子节点，并以最大-最小公平的方式进行调度。
* `system_vruntime` ([Nullable(Float64)](/zh/reference/data-types/index)) — 仅适用于 `fair` 节点。表示上一次处理的资源请求的 `vruntime`。在激活子节点时，用作 `vruntime` 的新值。
* `queue_length` ([Nullable(UInt64)](/zh/reference/data-types/index)) — 仅适用于 `fifo` 节点。当前驻留在队列中的资源请求数量。
* `queue_cost` ([Nullable(Int64)](/zh/reference/data-types/index)) — 仅适用于 `fifo` 节点。当前驻留在队列中的所有请求的成本总和 (例如以字节为单位的大小) 。
* `budget` ([Nullable(Int64)](/zh/reference/data-types/index)) — 仅适用于 `fifo` 节点。新资源请求可用的 &#39;成本单位&#39; 数量。在资源请求的估算成本与实际成本不一致时可能会出现 (例如在读/写失败后) 。
* `is_satisfied` ([Nullable(UInt8)](/zh/reference/data-types/index)) — 仅适用于约束节点 (例如 `inflight_limit`) 。如果该节点的所有约束都已满足，则等于 `1`。
* `inflight_requests` ([Nullable(Int64)](/zh/reference/data-types/index)) — 仅适用于 `inflight_limit` 节点。已从该节点出队且当前处于消费状态的资源请求数量。
* `inflight_cost` ([Nullable(Int64)](/zh/reference/data-types/index)) — 仅适用于 `inflight_limit` 节点。已从该节点出队且当前处于消费状态的所有资源请求的成本总和 (例如字节数) 。
* `max_requests` ([Nullable(Int64)](/zh/reference/data-types/index)) — 仅适用于 `inflight_limit` 节点。会导致违反约束的 inflight&#95;requests 上限。
* `max_cost` ([Nullable(Int64)](/zh/reference/data-types/index)) — 仅适用于 `inflight_limit` 节点。会导致违反约束的 inflight&#95;cost 上限。
* `max_speed` ([Nullable(Float64)](/zh/reference/data-types/index)) — 仅适用于 `bandwidth_limit` 节点。带宽上限，以每秒标记数表示。
* `max_burst` ([Nullable(Float64)](/zh/reference/data-types/index)) — 仅适用于 `bandwidth_limit` 节点。令牌桶限流器中可用标记数的上限。
* `throttling_us` ([Nullable(Int64)](/zh/reference/data-types/index)) — 仅适用于 `bandwidth_limit` 节点。该节点处于限流状态的总微秒数。
* `tokens` ([Nullable(Float64)](/zh/reference/data-types/index)) — 仅适用于 `bandwidth_limit` 节点。令牌桶限流器中当前可用的标记数。

{/*AUTOGENERATED_END*/}

```sql
SELECT *
FROM system.scheduler
WHERE resource = 'network_read' AND path = '/prio/fair/prod'
FORMAT Vertical
```
