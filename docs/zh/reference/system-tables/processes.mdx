<div id="description">
  ## 描述
</div>

该系统表用于实现 `SHOW PROCESSLIST` 查询。

<div id="columns">
  ## 列
</div>

{/*AUTOGENERATED_START*/}

* `is_initial_query` ([UInt8](/zh/reference/data-types/index)) — 此查询是否直接来自用户，或由 ClickHouse server 在分布式查询执行过程中发起。
* `user` ([String](/zh/reference/data-types/index)) — 发起该查询的用户。请注意，在分布式处理场景下，查询会以 `default` 用户身份发送到远程服务器。该字段包含的是当前这个具体查询的用户名，而不是由该查询发起的其他查询的用户名。
* `query_id` ([String](/zh/reference/data-types/index)) — 查询 ID (如果已定义) 。
* `address` ([IPv6](/zh/reference/data-types/index)) — 发起该查询的 IP 地址。在分布式处理场景下也是如此。要追踪分布式查询最初从哪里发起，请查看查询请求发起方服务器上的 system.processes。
* `port` ([UInt16](/zh/reference/data-types/index)) — 发起该查询的客户端端口。
* `initial_user` ([String](/zh/reference/data-types/index)) — 运行初始查询的用户名 (用于分布式查询执行) 。
* `initial_query_id` ([String](/zh/reference/data-types/index)) — 初始查询的 ID (用于分布式查询执行) 。
* `initial_address` ([IPv6](/zh/reference/data-types/index)) — 发起父查询的 IP 地址。
* `initial_port` ([UInt16](/zh/reference/data-types/index)) — 发起父查询时使用的客户端端口。
* `interface` ([UInt8](/zh/reference/data-types/index)) — 用于发送查询的接口。TCP = 1，HTTP = 2，GRPC = 3，MYSQL = 4，POSTGRESQL = 5，LOCAL = 6，TCP&#95;INTERSERVER = 7。
* `os_user` ([String](/zh/reference/data-types/index)) — 运行 clickhouse-client 的操作系统用户名。
* `client_hostname` ([String](/zh/reference/data-types/index)) — 运行 clickhouse-client 或其他 TCP 客户端的客户端机器主机名。
* `client_name` ([String](/zh/reference/data-types/index)) — clickhouse-client 或其他 TCP 客户端的名称。
* `client_agent` ([LowCardinality(String)](/zh/reference/data-types/index)) — 调用客户端的 AI 编码 agent (例如 `claude-code`、`cursor`) ，从环境变量中检测得到。如果未检测到 agent，则为空。
* `client_revision` ([UInt64](/zh/reference/data-types/index)) — clickhouse-client 或其他 TCP 客户端的修订版本。
* `client_version_major` ([UInt64](/zh/reference/data-types/index)) — clickhouse-client 或其他 TCP 客户端的主版本号。
* `client_version_minor` ([UInt64](/zh/reference/data-types/index)) — clickhouse-client 或其他 TCP 客户端的次版本号。
* `client_version_patch` ([UInt64](/zh/reference/data-types/index)) — clickhouse-client 或其他 TCP 客户端版本的补丁号。
* `http_method` ([UInt8](/zh/reference/data-types/index)) — 发起该查询的 HTTP 方法。可能的值：0 — 查询是从 TCP 接口发起的。1 — 使用了 GET 方法。2 — 使用了 POST 方法。
* `http_user_agent` ([String](/zh/reference/data-types/index)) — HTTP 查询中传递的 HTTP 请求头 UserAgent。
* `http_referer` ([String](/zh/reference/data-types/index)) — HTTP 查询中传递的 HTTP 请求头 Referer (包含发起查询页面的完整地址或部分地址) 。
* `forwarded_for` ([String](/zh/reference/data-types/index)) — HTTP 查询中传递的 HTTP 请求头 X-Forwarded-For。
* `quota_key` ([String](/zh/reference/data-types/index)) — 在 quotas 设置中指定的配额键 (参见 keyed) 。
* `distributed_depth` ([UInt64](/zh/reference/data-types/index)) — 查询在 server 节点之间内部转发的次数。
* `elapsed` ([Float64](/zh/reference/data-types/index)) — 自请求开始执行以来经过的时间 (秒) 。
* `is_cancelled` ([UInt8](/zh/reference/data-types/index)) — 查询是否已取消。
* `is_all_data_sent` ([UInt8](/zh/reference/data-types/index)) — 是否已将所有数据发送给客户端 (换句话说，查询是否已在服务器端完成) 。
* `read_rows` ([UInt64](/zh/reference/data-types/index)) — 从表中读取的行数。对于分布式处理，在请求发起服务器上，该值为所有远程服务器的总和。
* `read_bytes` ([UInt64](/zh/reference/data-types/index)) — 从表中读取的未压缩字节数。对于分布式处理，在请求发起服务器上，该值为所有远程服务器的总和。
* `total_rows_approx` ([UInt64](/zh/reference/data-types/index)) — 预计需要读取的总行数。对于分布式处理，在请求发起服务器上，该值为所有远程服务器的总和。在请求处理过程中，如果发现了新的待处理数据源，该值可能会更新。
* `written_rows` ([UInt64](/zh/reference/data-types/index)) — 写入存储的行数。
* `written_bytes` ([UInt64](/zh/reference/data-types/index)) — 写入存储的字节数。
* `memory_usage` ([Int64](/zh/reference/data-types/index)) — 查询使用的 RAM 量。它可能不包含某些类型的专用内存。
* `peak_memory_usage` ([Int64](/zh/reference/data-types/index)) — 当前内存使用量的峰值。
* `query` ([String](/zh/reference/data-types/index)) — 查询文本。对于 INSERT，不包含要插入的数据。
* `normalized_query_hash` ([UInt64](/zh/reference/data-types/index)) — 数值型哈希值；例如，对于仅字面量值不同的查询，该值是相同的。
* `query_kind` ([String](/zh/reference/data-types/index)) — 查询类型，例如 SELECT、INSERT 等。
* `thread_ids` ([Array(UInt64)](/zh/reference/data-types/index)) — 参与此查询的所有线程标识符列表。
* `peak_threads_usage` ([UInt64](/zh/reference/data-types/index)) — 同时执行该查询的线程最大数量。
* `ProfileEvents` ([Map(LowCardinality(String), UInt64)](/zh/reference/data-types/index)) — 为此查询计算的 ProfileEvents。
* `Settings` ([Map(LowCardinality(String), LowCardinality(String))](/zh/reference/data-types/index)) — 已修改的用户级设置列表。
* `current_database` ([String](/zh/reference/data-types/index)) — 当前数据库的名称。
* `is_internal` ([UInt8](/zh/reference/data-types/index)) — 表示该查询是否为内部执行的辅助查询。

{/*AUTOGENERATED_END*/}

SELECT * FROM system.processes LIMIT 10 FORMAT Vertical;
