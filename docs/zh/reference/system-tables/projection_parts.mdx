<div id="description">
  ## 描述
</div>

此表包含 MergeTree 家族中各表的 projection parts 信息。

<div id="columns">
  ## 列
</div>

{/*AUTOGENERATED_START*/}

* `partition` ([String](/zh/reference/data-types/index)) — 分区名称。
* `name` ([String](/zh/reference/data-types/index)) — 数据分区片段名称。
* `part_type` ([String](/zh/reference/data-types/index)) — 数据分区片段的存储格式。可能的值：Wide (每列一个文件) 和 Compact (所有列共用一个文件) 。
* `parent_name` ([String](/zh/reference/data-types/index)) — 源 (父) 数据分区片段的名称。
* `parent_uuid` ([UUID](/zh/reference/data-types/index)) — 源 (父) 数据分区片段的 UUID。
* `parent_part_type` ([String](/zh/reference/data-types/index)) — 源 (父) 数据分区片段的存储格式。
* `active` ([UInt8](/zh/reference/data-types/index)) — 指示数据分区片段是否处于活动状态的标志。如果数据分区片段处于活动状态，则会在表中使用；否则即将被删除。非活动数据分区片段会在合并和变更操作后出现。
* `marks` ([UInt64](/zh/reference/data-types/index)) — 标记数量。若要获取数据分区片段中行数的近似值，可将标记数乘以 index granularity (通常为 8192)  (此提示不适用于 adaptive granularity) 。
* `rows` ([UInt64](/zh/reference/data-types/index)) — 行数。
* `bytes_on_disk` ([UInt64](/zh/reference/data-types/index)) — 磁盘上所有数据分区片段文件的总大小 (以字节为单位) 。
* `data_compressed_bytes` ([UInt64](/zh/reference/data-types/index)) — 数据分区片段中压缩数据的总大小。不包括所有辅助文件 (例如标记文件) 。
* `data_uncompressed_bytes` ([UInt64](/zh/reference/data-types/index)) — 数据分区片段中未压缩数据的总大小。不包括所有辅助文件 (例如标记文件) 。
* `marks_bytes` ([UInt64](/zh/reference/data-types/index)) — 标记文件的大小。
* `parent_marks` ([UInt64](/zh/reference/data-types/index)) — 源 (父) 数据分区片段中的标记数量。
* `parent_rows` ([UInt64](/zh/reference/data-types/index)) — 源 (父) 数据分区片段中的行数。
* `parent_bytes_on_disk` ([UInt64](/zh/reference/data-types/index)) — 所有源 (父) 数据分区片段文件的总大小 (以字节为单位) 。
* `parent_data_compressed_bytes` ([UInt64](/zh/reference/data-types/index)) — 源 (父) 数据分区片段中压缩数据的总大小。
* `parent_data_uncompressed_bytes` ([UInt64](/zh/reference/data-types/index)) — 源 (父) 数据分区片段中未压缩数据的总大小。
* `parent_marks_bytes` ([UInt64](/zh/reference/data-types/index)) — 源 (父) 数据分区片段中标记文件的大小。
* `modification_time` ([DateTime](/zh/reference/data-types/index)) — 包含该数据分区片段的目录被修改的时间。这通常对应于数据分区片段的创建时间。
* `remove_time` ([DateTime](/zh/reference/data-types/index)) — 数据分区片段变为非活动状态的时间。
* `refcount` ([UInt32](/zh/reference/data-types/index)) — 数据分区片段被使用的位置数。值大于 2 表示该数据分区片段被用于查询或合并操作。
* `min_date` ([Date](/zh/reference/data-types/index)) — 数据分区片段中日期键的最小值。
* `max_date` ([Date](/zh/reference/data-types/index)) — 数据分区片段中日期键的最大值。
* `min_time` ([DateTime](/zh/reference/data-types/index)) — 数据分区片段中日期时间键的最小值。
* `max_time` ([DateTime](/zh/reference/data-types/index)) — 数据分区片段中日期和时间键的最大值。
* `partition_id` ([String](/zh/reference/data-types/index)) — 分区 ID。
* `min_block_number` ([Int64](/zh/reference/data-types/index)) — 合并后构成当前数据分区片段的最小数据分区片段编号。
* `max_block_number` ([Int64](/zh/reference/data-types/index)) — 合并后构成当前数据分区片段的最大数据分区片段编号。
* `level` ([UInt32](/zh/reference/data-types/index)) — 合并树的深度。0 表示当前数据分区片段是通过插入创建的，而不是通过合并其他数据分区片段创建的。
* `data_version` ([UInt64](/zh/reference/data-types/index)) — 用于确定应将哪些变更应用到该数据分区片段的编号 (即版本高于 data&#95;version 的变更) 。
* `primary_key_bytes_in_memory` ([UInt64](/zh/reference/data-types/index)) — 主键值占用的内存量 (以字节为单位) 。
* `primary_key_bytes_in_memory_allocated` ([UInt64](/zh/reference/data-types/index)) — 为主键值预留的内存量 (以字节为单位) 。
* `is_frozen` ([UInt8](/zh/reference/data-types/index)) — 表示是否存在分区数据备份的标志。1 表示备份存在，0 表示备份不存在。
* `database` ([String](/zh/reference/data-types/index)) — 数据库名称。
* `table` ([String](/zh/reference/data-types/index)) — 表名称。
* `engine` ([String](/zh/reference/data-types/index)) — 不带参数的表引擎名称。
* `disk_name` ([String](/zh/reference/data-types/index)) — 存储该数据分区片段的磁盘名称。
* `path` ([String](/zh/reference/data-types/index)) — 包含数据分区片段文件的文件夹绝对路径。
* `hash_of_all_files` ([String](/zh/reference/data-types/index)) — 压缩文件的 sipHash128。
* `hash_of_uncompressed_files` ([String](/zh/reference/data-types/index)) — 未压缩文件的 sipHash128 (例如带有标记、索引文件等的文件) 。
* `uncompressed_hash_of_compressed_files` ([String](/zh/reference/data-types/index)) — 将压缩文件中的数据视为未压缩时计算得到的 sipHash128。
* `delete_ttl_info_min` ([DateTime](/zh/reference/data-types/index)) — TTL DELETE 规则的日期和时间键最小值。
* `delete_ttl_info_max` ([DateTime](/zh/reference/data-types/index)) — TTL DELETE 规则的日期和时间键最大值。
* `move_ttl_info.expression` ([Array(String)](/zh/reference/data-types/index)) — 表达式数组。每个表达式定义一条 TTL MOVE 规则。
* `move_ttl_info.min` ([Array(DateTime)](/zh/reference/data-types/index)) — 日期和时间值数组。每个元素描述一条 TTL MOVE 规则的最小键值。
* `move_ttl_info.max` ([Array(DateTime)](/zh/reference/data-types/index)) — 日期和时间值数组。每个元素描述一条 TTL MOVE 规则的最大键值。
* `default_compression_codec` ([String](/zh/reference/data-types/index)) — 用于压缩此数据分区片段的 codec 名称 (当列未显式指定 codec 时) 。
* `recompression_ttl_info.expression` ([Array(String)](/zh/reference/data-types/index)) — TTL 表达式。
* `recompression_ttl_info.min` ([Array(DateTime)](/zh/reference/data-types/index)) — 此数据分区片段内计算出的 TTL 表达式最小值。用于判断是否至少有一行的 TTL 已过期。
* `recompression_ttl_info.max` ([Array(DateTime)](/zh/reference/data-types/index)) — 该分片中计算出的 TTL 表达式的最大值。用于判断是否所有行的 TTL 都已过期。
* `group_by_ttl_info.expression` ([Array(String)](/zh/reference/data-types/index)) — TTL 表达式。
* `group_by_ttl_info.min` ([Array(DateTime)](/zh/reference/data-types/index)) — 该分片中计算出的 TTL 表达式的最小值。用于判断是否至少有一行的 TTL 已过期。
* `group_by_ttl_info.max` ([Array(DateTime)](/zh/reference/data-types/index)) — 该分片中计算出的 TTL 表达式的最大值。用于判断是否所有行的 TTL 都已过期。
* `rows_where_ttl_info.expression` ([Array(String)](/zh/reference/data-types/index)) — TTL 表达式。
* `rows_where_ttl_info.min` ([Array(DateTime)](/zh/reference/data-types/index)) — 该分片中计算出的 TTL 表达式的最小值。用于判断是否至少有一行的 TTL 已过期。
* `rows_where_ttl_info.max` ([Array(DateTime)](/zh/reference/data-types/index)) — 该分片中计算出的 TTL 表达式的最大值。用于判断是否所有行的 TTL 都已过期。
* `is_broken` ([UInt8](/zh/reference/data-types/index)) — 投影分片是否已损坏
* `exception_code` ([Int32](/zh/reference/data-types/index)) — 用于说明投影分片损坏状态的异常消息
* `exception` ([String](/zh/reference/data-types/index)) — 用于说明投影分片损坏状态的异常代码
