<div id="description">
  ## 描述
</div>

包含在后台定期计算的指标。例如，已使用的 RAM 量。

<div id="columns">
  ## 列
</div>

{/*AUTOGENERATED_START*/}

* `metric` ([String](/zh/reference/data-types/string)) — 指标名称。
* `value` ([Float64](/zh/reference/data-types/float)) — 指标值。
* `description` ([String](/zh/reference/data-types/string) - 指标说明)

{/*AUTOGENERATED_END*/}

<div id="metric-descriptions">
  ## 指标说明
</div>

以下说明由 `utils/generate-async-metrics-docs` 根据 C++ 源代码生成。唯一的权威来源是 `src/Common/AsynchronousMetrics.cpp`、`src/Interpreters/ServerAsynchronousMetrics.cpp` 和 `src/Coordination/KeeperAsynchronousMetrics.cpp` 中每项指标注册语句旁边的字符串字面量。包含可变后缀 (按磁盘、按 CPU、按接口等) 的指标名称会以 `*name*` 占位符显示；运行中的服务器在上报时会将其替换为实际的后缀。

<div id="blockactivetime_name">
  ### BlockActiveTime_*name*
</div>

块设备的 IO 请求排队等待的时间 (以秒为单位) 。这是一个系统级指标，包含主机上的所有进程，而不仅仅是 clickhouse-server。来源：`/sys/block`。另请参见 https://www.kernel.org/doc/Documentation/block/stat.txt

<div id="blockactivetimeperop_name">
  ### BlockActiveTimePerOp_*name*
</div>

与 `BlockActiveTime` 指标类似，但其值会除以 IO 操作次数，以计算每次操作的耗时。

<div id="blockdiscardbytes_name">
  ### BlockDiscardBytes_*name*
</div>

块设备上被丢弃的字节数。这些操作与 SSD 相关。ClickHouse 不使用丢弃操作，但系统上的其他进程可能会使用。这是一个系统级指标，包含主机上的所有进程，而不只是 clickhouse-server。来源：`/sys/block`。参见 https://www.kernel.org/doc/Documentation/block/stat.txt

<div id="blockdiscardmerges_name">
  ### BlockDiscardMerges_*name*
</div>

由操作系统 IO 调度器向块设备发起并合并的 discard 操作次数。这些操作与 SSD 有关。ClickHouse 不使用 discard 操作，但系统上的其他进程可能会使用。这是一个系统级指标，包含主机上的所有进程，而不只是 clickhouse-server。来源：`/sys/block`。参见 https://www.kernel.org/doc/Documentation/block/stat.txt

<div id="blockdiscardops_name">
  ### BlockDiscardOps_*name*
</div>

块设备发出的 discard 操作请求次数。这些操作主要与 SSD 相关。ClickHouse 不使用 discard 操作，但系统上的其他进程可能会使用。这是一个系统级指标，包含主机上的所有进程，而不只是 clickhouse-server。来源：`/sys/block`。参见 https://www.kernel.org/doc/Documentation/block/stat.txt

<div id="blockdiscardtime_name">
  ### BlockDiscardTime_*name*
</div>

块设备请求执行的丢弃操作所耗费的时间 (以秒为单位) ，为所有此类操作耗时的总和。这些操作与 SSD 有关。ClickHouse 不使用丢弃操作，但系统上的其他进程可能会使用。这是一个系统级指标，包含主机上所有进程的数据，而不只是 clickhouse-server。来源：`/sys/block`。参见 https://www.kernel.org/doc/Documentation/block/stat.txt

<div id="blockinflightops_name">
  ### BlockInFlightOps_*name*
</div>

该值统计已提交给设备驱动程序但尚未完成的 I/O 请求数量。不包括仍在队列中、尚未提交给设备驱动程序的 I/O 请求。这是一个系统级指标，涵盖主机上的所有进程，而不只是 clickhouse-server。来源：`/sys/block`。参见 https://www.kernel.org/doc/Documentation/block/stat.txt

<div id="blockqueuetime_name">
  ### BlockQueueTime_*name*
</div>

该值表示此块设备上的 IO 请求累计等待了多少毫秒。如果有多个 IO 请求同时在等待，该值会按“等待毫秒数 × 等待请求数”的乘积增加。这是一个系统级指标，涵盖主机上的所有进程，而不仅仅是 clickhouse-server。来源：`/sys/block`。另请参见 https://www.kernel.org/doc/Documentation/block/stat.txt

<div id="blockqueuetimeperop_name">
  ### BlockQueueTimePerOp_*name*
</div>

与 `BlockQueueTime` 指标类似，但其值会除以 IO 操作次数，以计算每次操作的耗时。

<div id="blockreadbytes_name">
  ### BlockReadBytes_*name*
</div>

从块设备读取的字节数。由于使用了可减少 IO 的操作系统页缓存，该值可能会低于从 虚拟文件系统 读取的字节数。这是一个系统级指标，包含主机上的所有进程，而不仅仅是 clickhouse-server。来源：`/sys/block`。参见 https://www.kernel.org/doc/Documentation/block/stat.txt

<div id="blockreadmerges_name">
  ### BlockReadMerges_*name*
</div>

由操作系统 IO 调度器合并的块设备读取请求次数。这是一个系统级指标，包含主机上的所有进程，而不仅仅是 clickhouse-server。来源：`/sys/block`。参见 https://www.kernel.org/doc/Documentation/block/stat.txt

<div id="blockreadops_name">
  ### BlockReadOps_*name*
</div>

从块设备发起的读操作次数。这是一个系统级指标，包含主机上的所有进程，而不只是 clickhouse-server。来源：`/sys/block`。参见 https://www.kernel.org/doc/Documentation/block/stat.txt

<div id="blockreadtime_name">
  ### BlockReadTime_*name*
</div>

从块设备发起的读操作所耗费的时间 (以秒为单位) ，为所有此类操作的总和。这是一个系统级指标，包含主机上的所有进程，而不只是 clickhouse-server。来源：`/sys/block`。参见 https://www.kernel.org/doc/Documentation/block/stat.txt

<div id="blockwritebytes_name">
  ### BlockWriteBytes_*name*
</div>

写入块设备的字节数。由于使用了可减少 IO 的操作系统 page cache，其值可能低于写入 虚拟文件系统 的字节数。由于直写缓存，块设备上的写入可能会晚于 虚拟文件系统 中对应的写入发生。这是一个系统级指标，包含主机上的所有进程，而不仅仅是 clickhouse-server。来源：`/sys/block`。另请参阅 https://www.kernel.org/doc/Documentation/block/stat.txt

<div id="blockwritemerges_name">
  ### BlockWriteMerges_*name*
</div>

由操作系统 IO 调度器从块设备发起并合并的写操作次数。这是一个系统级指标，包含主机上的所有进程，而不只是 clickhouse-server。来源：`/sys/block`。参见 https://www.kernel.org/doc/Documentation/block/stat.txt

<div id="blockwriteops_name">
  ### BlockWriteOps_*name*
</div>

向块设备请求的写操作次数。这是一个系统级指标，包含主机上的所有进程，而不只是 clickhouse-server。来源：`/sys/block`。参见 https://www.kernel.org/doc/Documentation/block/stat.txt

<div id="blockwritetime_name">
  ### BlockWriteTime_*name*
</div>

从块设备发起的写操作所耗费的时间 (以秒为单位) ，为所有操作的总和。这是一个系统级指标，涵盖主机上的所有进程，而不仅仅是 clickhouse-server。来源：`/sys/block`。参见 https://www.kernel.org/doc/Documentation/block/stat.txt

<div id="cgroupmaxcpu">
  ### CGroupMaxCPU
</div>

根据 CGroups 计算的最大 CPU 核心数。

<div id="cgroupmemorytotal">
  ### CGroupMemoryTotal
</div>

cgroup 中的内存总量，单位为字节。如果该值为 0，则限制与 OSMemoryTotal 相同。

<div id="cgroupmemoryused">
  ### CGroupMemoryUsed
</div>

cgroup 中已使用的内存量，单位为字节。在 cgroup v2 中，它表示 anon + sock + 不可回收的内核内存；在 cgroup v1 中，它表示 RSS。两种情况下都不包含内核的 OS page cache (基于文件的缓存) 。

<div id="cgroupmemoryusedwithoutpagecache">
  ### CGroupMemoryUsedWithoutPageCache
</div>

cgroup 中已使用的内存量，单位为字节，不包括 ClickHouse 用户态页缓存。其值等于 CGroupMemoryUsed 减去用户态页缓存的大小。禁用用户态页缓存时，该值等于 CGroupMemoryUsed。

<div id="cgroupsystemtime">
  ### CGroupSystemTime
</div>

CPU 核心运行 操作系统内核 (system) 代码的时间占比。

<div id="cgroupsystemtimenormalized">
  ### CGroupSystemTimeNormalized
</div>

该值与 `CGroupSystemTime` 类似，但会除以可用的 CPU 核心数，因此无论核心数量多少，结果都落在 [0..1] 区间内。这样一来，即使 集群 中各 server 的核心数不一致，你也可以对多个 server 上的该指标值求平均，并仍然得到平均资源利用率指标。如果已指定，则可以使用 Cgroup CPU 配额除以其周期的结果来代替实际的 CPU 核心数；在这种情况下，该指标的值在某些时刻可能会超过 1。

<div id="cgroupusertime">
  ### CGroupUserTime
</div>

CPU 核心执行用户态代码的时间占比。它还包括由于 CPU 内部原因导致 CPU 未得到充分利用的时间 (例如内存加载、流水线停顿、分支预测错误、运行另一个 SMT 核心时的时间) 。

<div id="cgroupusertimenormalized">
  ### CGroupUserTimeNormalized
</div>

该值与 `CGroupUserTime` 类似，但会除以可用 CPU 核心数，因此无论核心数多少，结果都会落在 [0..1] 区间内。这样一来，即使集群中各服务器的核心数并不一致，你仍然可以对多台服务器上的这一指标求平均，并得到平均资源利用率指标。如果指定了，也可以用 Cgroup CPU 配额除以其 period 的结果来代替实际 CPU 核心数；在这种情况下，该指标的值在某些时刻可能会超过 1。
