<div id="description">
  ## 描述
</div>

此表包含 ZooKeeper 操作的聚合统计信息 (例如操作次数、平均延迟、错误数) ，按 `(session_id, parent_path, operation, component, is_subrequest)` 分组，并定期写入磁盘。

与会记录每个单独请求和响应的 [system.zookeeper&#95;log](/zh/reference/system-tables/zookeeper_log) 不同，此表会将操作聚合到不同分组中，因此更加轻量，也更适合生产环境中的工作负载。

属于 `Multi` 或 `MultiRead` 批次的操作会通过 `is_subrequest` 列单独跟踪。子请求的延迟为零，因为总延迟归属于外层的 `Multi`/`MultiRead` 操作。

<div id="columns">
  ## 列
</div>

{/*AUTOGENERATED_START*/}

* `hostname` ([LowCardinality(String)](/zh/reference/data-types/lowcardinality)) — 服务器的主机名。
* `event_date` ([Date](/zh/reference/data-types/date)) — 该组被写入磁盘时的日期。
* `event_time` ([DateTime](/zh/reference/data-types/datetime)) — 该组被写入磁盘时的时间。
* `session_id` ([Int64](/zh/reference/data-types/int-uint)) — 会话 ID。
* `parent_path` ([String](/zh/reference/data-types/string)) — 路径前缀。
* `operation` ([Enum16(&#39;Close&#39; = -11, &#39;Error&#39; = -1, &#39;Watch&#39; = 0, &#39;Create&#39; = 1, &#39;Remove&#39; = 2, &#39;Exists&#39; = 3, &#39;Get&#39; = 4, &#39;Set&#39; = 5, &#39;GetACL&#39; = 6, &#39;SetACL&#39; = 7, &#39;SimpleList&#39; = 8, &#39;Sync&#39; = 9, &#39;Heartbeat&#39; = 11, &#39;List&#39; = 12, &#39;Check&#39; = 13, &#39;Multi&#39; = 14, &#39;Create2&#39; = 15, &#39;Reconfig&#39; = 16, &#39;CheckWatch&#39; = 17, &#39;RemoveWatch&#39; = 18, &#39;MultiRead&#39; = 22, &#39;Auth&#39; = 100, &#39;SetWatch&#39; = 101, &#39;SetWatch2&#39; = 105, &#39;AddWatch&#39; = 106, &#39;FilteredList&#39; = 500, &#39;CheckNotExists&#39; = 501, &#39;CreateIfNotExists&#39; = 502, &#39;RemoveRecursive&#39; = 503, &#39;CheckStat&#39; = 504, &#39;TryRemove&#39; = 505, &#39;FilteredListWithStatsAndData&#39; = 506, &#39;ListRecursive&#39; = 507, &#39;SessionID&#39; = 997)](/zh/reference/data-types/enum)) — ZooKeeper 操作类型。
* `is_subrequest` ([UInt8](/zh/reference/data-types/int-uint)) — 此操作是否是 Multi 或 MultiRead 操作中的子请求。
* `count` ([UInt32](/zh/reference/data-types/int-uint)) — (session&#95;id, parent&#95;path, operation, component, is&#95;subrequest) 组中的操作数量。
* `errors` ([Map(Enum8(&#39;ZNOWATCHER&#39; = -121, &#39;ZNOTREADONLY&#39; = -119, &#39;ZSESSIONMOVED&#39; = -118, &#39;ZNOTHING&#39; = -117, &#39;ZCLOSING&#39; = -116, &#39;ZAUTHFAILED&#39; = -115, &#39;ZINVALIDACL&#39; = -114, &#39;ZINVALIDCALLBACK&#39; = -113, &#39;ZSESSIONEXPIRED&#39; = -112, &#39;ZNOTEMPTY&#39; = -111, &#39;ZNODEEXISTS&#39; = -110, &#39;ZNOCHILDRENFOREPHEMERALS&#39; = -108, &#39;ZBADVERSION&#39; = -103, &#39;ZNOAUTH&#39; = -102, &#39;ZNONODE&#39; = -101, &#39;ZAPIERROR&#39; = -100, &#39;ZOUTOFMEMORY&#39; = -10, &#39;ZINVALIDSTATE&#39; = -9, &#39;ZBADARGUMENTS&#39; = -8, &#39;ZOPERATIONTIMEOUT&#39; = -7, &#39;ZUNIMPLEMENTED&#39; = -6, &#39;ZMARSHALLINGERROR&#39; = -5, &#39;ZCONNECTIONLOSS&#39; = -4, &#39;ZDATAINCONSISTENCY&#39; = -3, &#39;ZRUNTIMEINCONSISTENCY&#39; = -2, &#39;ZSYSTEMERROR&#39; = -1, &#39;ZOK&#39; = 0), UInt32)](/zh/reference/data-types/map)) — (session&#95;id, parent&#95;path, operation, component, is&#95;subrequest) 组中的错误。
* `average_latency` ([Float64](/zh/reference/data-types/float)) — (session&#95;id, parent&#95;path, operation, component, is&#95;subrequest) 组中所有操作的平均延迟，单位为微秒。子请求的延迟为零，因为延迟归属于外层的 Multi 或 MultiRead 操作。
* `component` ([LowCardinality(String)](/zh/reference/data-types/lowcardinality)) — 触发该事件的组件。
