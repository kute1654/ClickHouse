<div id="description">
  ## 描述
</div>

包含所有 `BACKUP` 或 `RESTORE` 操作及其当前状态和其他属性的列表。请注意，该表不是持久化的，仅显示自上次服务器重启后执行的操作。

<div id="columns">
  ## 列
</div>

{/*AUTOGENERATED_START*/}

* `id` ([String](/zh/reference/data-types/index)) — 操作 ID，可通过 SETTINGS id=... 传入，也可以是随机生成的 UUID。
* `name` ([String](/zh/reference/data-types/index)) — 操作名称，即形如 `Disk('backups', 'my_backup')` 的字符串
* `base_backup_name` ([String](/zh/reference/data-types/index)) — 基础备份操作名称，即形如 `Disk('backups', 'my_base_backup')` 的字符串
* `query_id` ([String](/zh/reference/data-types/index)) — 发起备份的查询的 Query ID。
* `status` ([Enum8(&#39;CREATING&#95;BACKUP&#39; = 0, &#39;BACKUP&#95;CREATED&#39; = 1, &#39;BACKUP&#95;FAILED&#39; = 2, &#39;RESTORING&#39; = 3, &#39;RESTORED&#39; = 4, &#39;RESTORE&#95;FAILED&#39; = 5, &#39;BACKUP&#95;CANCELLED&#39; = 6, &#39;RESTORE&#95;CANCELLED&#39; = 7)](/zh/reference/data-types/index)) — 备份或恢复操作的状态。
* `error` ([String](/zh/reference/data-types/index)) — 错误消息 (如有) 。
* `start_time` ([DateTime64(6)](/zh/reference/data-types/index)) — 操作开始时间。
* `end_time` ([DateTime64(6)](/zh/reference/data-types/index)) — 操作完成时间。
* `num_files` ([UInt64](/zh/reference/data-types/index)) — 备份中存储的文件数量。
* `total_size` ([UInt64](/zh/reference/data-types/index)) — 备份中存储的文件总大小。
* `num_entries` ([UInt64](/zh/reference/data-types/index)) — 备份中的条目数，即如果备份以文件夹形式存储，则表示该文件夹中的文件数量。
* `uncompressed_size` ([UInt64](/zh/reference/data-types/index)) — 备份的未压缩大小。
* `compressed_size` ([UInt64](/zh/reference/data-types/index)) — 备份的压缩后大小。
* `files_read` ([UInt64](/zh/reference/data-types/index)) — 返回从该备份执行 RESTORE 期间读取的文件数量。
* `bytes_read` ([UInt64](/zh/reference/data-types/index)) — 返回从该备份执行 RESTORE 期间读取的文件总大小。
* `ProfileEvents` ([Map(LowCardinality(String), UInt64)](/zh/reference/data-types/index)) — 此操作期间捕获的所有 profile events。

{/*AUTOGENERATED_END*/}
