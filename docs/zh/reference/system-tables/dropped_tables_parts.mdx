<div id="description">
  ## 描述
</div>

包含来自 [system.dropped&#95;tables](/zh/reference/system-tables/dropped_tables) 的已删除 [MergeTree](/zh/reference/engines/table-engines/mergetree-family/mergetree) 表的 parts 信息

此表的 schema 与 [system.parts](/zh/reference/system-tables/parts) 相同

<div id="columns">
  ## 列
</div>

{/*自动生成开始*/}

* `partition` ([String](/zh/reference/data-types/index)) — 分区标识符 (由表分区键表达式派生的字符串表示形式) 。
* `name` ([String](/zh/reference/data-types/index)) —
  数据分区片段的名称。数据分区片段的命名结构可用于判断数据、摄取和合并模式的许多特征。数据分区片段的命名格式如下：

```text
<partition_id>_<minimum_block_number>_<maximum_block_number>_<level>_<data_version>
```

* 定义：
  * `partition_id` - 标识分区键
  * `minimum_block_number` - 标识该 parts 中的最小块编号。ClickHouse 总是合并连续的块
  * `maximum_block_number` - 标识该 parts 中的最大块编号
  * `level` - 该 parts 每额外合并一次，值就加一。level 为 0 表示这是一个尚未合并的新 parts。需要注意的是，ClickHouse 中的所有 parts 始终都是不可变的
  * `data_version` - 可选值，在 parts 发生变更时递增 (同样，变更后的数据也只会写入新的 parts，因为 parts 是不可变的)

* `uuid` ([UUID](/zh/reference/data-types/index)) — 数据分区片段的 UUID。

* `part_type` ([String](/zh/reference/data-types/index)) — 数据分区片段的存储格式。可能的值：`Wide` — 每一列存储在单独的文件中；`Compact` — 所有列都存储在一个文件中。数据存储格式由 MergeTree 表的 `min_bytes_for_wide_part` 和 `min_rows_for_wide_part` 设置控制。

* `active` ([UInt8](/zh/reference/data-types/index)) — 指示数据分区片段是否处于活动状态的标志。如果数据分区片段处于活动状态，则会在表中使用；否则，即将被删除。非活动的数据分区片段会在合并和变更操作后出现。

* `marks` ([UInt64](/zh/reference/data-types/index)) — 标记数。要获取数据分区片段中行数的近似值，可将标记数乘以索引粒度 (通常为 8192)  (此方法不适用于自适应粒度) 。

* `rows` ([UInt64](/zh/reference/data-types/index)) — 行数。

* `files` ([UInt64](/zh/reference/data-types/index)) — 数据分区片段中的文件数量。

* `bytes_on_disk` ([UInt64](/zh/reference/data-types/index)) — 数据分区片段中所有文件的总大小 (以字节为单位) 。

* `data_compressed_bytes` ([UInt64](/zh/reference/data-types/index)) — 数据分区片段中压缩数据的总大小。不包括所有辅助文件 (例如标记文件) 。

* `data_uncompressed_bytes` ([UInt64](/zh/reference/data-types/index)) — 数据分区片段中未压缩数据的总大小。不包括所有辅助文件 (例如标记文件) 。

* `primary_key_size` ([UInt64](/zh/reference/data-types/index)) — 磁盘上 primary.idx/cidx 文件中主键值占用的内存量 (以字节为单位) 。

* `marks_bytes` ([UInt64](/zh/reference/data-types/index)) — 标记文件的大小。

* `secondary_indices_compressed_bytes` ([UInt64](/zh/reference/data-types/index)) — 数据分区片段中次级索引压缩数据的总大小。不包括所有辅助文件 (例如标记文件) 。

* `secondary_indices_uncompressed_bytes` ([UInt64](/zh/reference/data-types/index)) — 数据分区片段中次级索引未压缩数据的总大小。不包括所有辅助文件 (例如标记文件) 。

* `secondary_indices_marks_bytes` ([UInt64](/zh/reference/data-types/index)) — 次级索引标记文件的大小。

* `modification_time` ([日期时间](/zh/reference/data-types/index)) — 包含该数据分区片段的目录被修改的时间。通常对应于数据分区片段的创建时间。

* `remove_time` ([日期时间](/zh/reference/data-types/index)) — 数据分区片段变为非活动状态的时间。

* `refcount` ([UInt32](/zh/reference/data-types/index)) — 数据分区片段被使用的位置数。值大于 2 表示该数据分区片段正用于查询或合并。

* `min_date` ([Date](/zh/reference/data-types/index)) — 数据分区片段中日期键的最小值。

* `max_date` ([Date](/zh/reference/data-types/index)) — 数据分区片段中日期键的最大值。

* `min_time` ([日期时间](/zh/reference/data-types/index)) — 数据分区片段中日期时间键的最小值。

* `max_time` ([日期时间](/zh/reference/data-types/index)) — 数据分区片段中日期时间键的最大值。

* `partition_id` ([String](/zh/reference/data-types/index)) — 分区 ID。

* `min_block_number` ([Int64](/zh/reference/data-types/index)) — 合并后构成当前分片的数据分区片段的最小编号。

* `max_block_number` ([Int64](/zh/reference/data-types/index)) — 合并后构成当前分片的数据分区片段的最大编号。

* `level` ([UInt32](/zh/reference/data-types/index)) — 合并树的深度。零表示当前分片是通过插入创建的，而不是通过合并其他 parts 创建的。

* `data_version` ([UInt64](/zh/reference/data-types/index)) — 用于确定应对该数据分区片段应用哪些变更的数值 (版本高于 data&#95;version 的变更) 。

* `primary_key_bytes_in_memory` ([UInt64](/zh/reference/data-types/index)) — 主键值占用的内存量 (字节) 。启用 `primary_key_lazy_load` 且主键未加载时，该值为 0。非零时，这些字节驻留在该分区片段本身中，并计入 `jemalloc.mergetree_arena.active_bytes`。这些字节绝不会计入 `PrimaryIndexCacheBytes` —— 对于每个分区片段，两者是互斥的：索引要么位于分区片段中 (此指标) ，要么位于共享的 `PrimaryIndexCache` 中 (另一个指标) ，具体取决于 `primary_key_lazy_load` 和 `use_primary_key_cache`。

* `primary_key_bytes_in_memory_allocated` ([UInt64](/zh/reference/data-types/index)) — 为主键值预留的内存量 (字节) 。启用 `primary_key_lazy_load` 且主键未加载时，该值为 0。非零时，计入 `jemalloc.mergetree_arena.active_bytes`。有关它与 `PrimaryIndexCacheBytes` 之间的关系，请参见 `primary_key_bytes_in_memory` 的说明。

* `index_granularity_bytes_in_memory` ([UInt64](/zh/reference/data-types/index)) — 索引粒度值占用的内存量 (字节)  (在 primary&#95;key&#95;lazy&#95;load=1 且 use&#95;primary&#95;key&#95;cache=1 的情况下为 0) 。非零时，这些字节归该分区片段所有，并计入 `jemalloc.mergetree_arena.active_bytes`。

* `index_granularity_bytes_in_memory_allocated` ([UInt64](/zh/reference/data-types/index)) — 为索引粒度值预留的内存量 (字节)  (在 primary&#95;key&#95;lazy&#95;load=1 且 use&#95;primary&#95;key&#95;cache=1 的情况下为 0) 。非零时，计入 `jemalloc.mergetree_arena.active_bytes`。

* `is_frozen` ([UInt8](/zh/reference/data-types/index)) — 表示是否存在分区数据备份的标志。1 表示备份存在。0 表示备份不存在。更多详情，请参见 FREEZE PARTITION。

* `database` ([String](/zh/reference/data-types/index)) — 数据库名称。

* `table` ([String](/zh/reference/data-types/index)) — 表名称。

* `engine` ([String](/zh/reference/data-types/index)) — 不带参数的表引擎名称。

* `disk_name` ([String](/zh/reference/data-types/index)) — 存储该数据分区片段的磁盘名称。

* `path` ([String](/zh/reference/data-types/index)) — 包含数据分区片段文件的文件夹绝对路径。

* `hash_of_all_files` ([String](/zh/reference/data-types/index)) — 已压缩文件的 sipHash128。

* `hash_of_uncompressed_files` ([String](/zh/reference/data-types/index)) — 未压缩文件的 sipHash128 (例如标记文件、索引文件等) 。

* `uncompressed_hash_of_compressed_files` ([String](/zh/reference/data-types/index)) — 将压缩文件中的数据视为未压缩时计算得到的 sipHash128。

* `delete_ttl_info_min` ([DateTime](/zh/reference/data-types/index)) — TTL DELETE 规则的日期时间键最小值。

* `delete_ttl_info_max` ([DateTime](/zh/reference/data-types/index)) — TTL DELETE 规则的日期时间键最大值。

* `move_ttl_info.expression` ([Array(String)](/zh/reference/data-types/index)) — 表达式数组。每个表达式定义一条 TTL MOVE 规则。

* `move_ttl_info.min` ([Array(DateTime)](/zh/reference/data-types/index)) — 日期时间值数组。每个元素描述一条 TTL MOVE 规则的最小键值。

* `move_ttl_info.max` ([Array(DateTime)](/zh/reference/data-types/index)) — 日期时间值数组。每个元素描述一条 TTL MOVE 规则的最大键值。

* `default_compression_codec` ([String](/zh/reference/data-types/index)) — 用于压缩该数据分区片段的 codec 名称 (在列未显式指定 codec 的情况下) 。

* `recompression_ttl_info.expression` ([Array(String)](/zh/reference/data-types/index)) — TTL 表达式。

* `recompression_ttl_info.min` ([Array(DateTime)](/zh/reference/data-types/index)) — 该分区片段内计算出的 TTL 表达式最小值。用于判断是否至少有一行数据的 TTL 已过期。

* `recompression_ttl_info.max` ([Array(DateTime)](/zh/reference/data-types/index)) — 该 part 中计算出的生存时间 (TTL) 表达式的最大值。用于判断是否所有行的生存时间 (TTL) 都已过期。

* `group_by_ttl_info.expression` ([Array(String)](/zh/reference/data-types/index)) — 生存时间 (TTL) 表达式。

* `group_by_ttl_info.min` ([Array(DateTime)](/zh/reference/data-types/index)) — 该 part 中计算出的生存时间 (TTL) 表达式的最小值。用于判断是否至少有一行的生存时间 (TTL) 已过期。

* `group_by_ttl_info.max` ([Array(DateTime)](/zh/reference/data-types/index)) — 该 part 中计算出的生存时间 (TTL) 表达式的最大值。用于判断是否所有行的生存时间 (TTL) 都已过期。

* `rows_where_ttl_info.expression` ([Array(String)](/zh/reference/data-types/index)) — 生存时间 (TTL) 表达式。

* `rows_where_ttl_info.min` ([Array(DateTime)](/zh/reference/data-types/index)) — 该 part 中计算出的生存时间 (TTL) 表达式的最小值。用于判断是否至少有一行的生存时间 (TTL) 已过期。

* `rows_where_ttl_info.max` ([Array(DateTime)](/zh/reference/data-types/index)) — 该 part 中计算出的生存时间 (TTL) 表达式的最大值。用于判断是否所有行的生存时间 (TTL) 都已过期。

* `projections` ([Array(String)](/zh/reference/data-types/index)) — 为该 part 计算出的投影名称列表。

* `visible` ([UInt8](/zh/reference/data-types/index)) — 指示该 part 对 SELECT 查询是否可见的标志。

* `creation_tid` ([Tuple(UInt64, UInt64, UUID)](/zh/reference/data-types/index)) — 已创建或正在尝试创建此对象的事务 ID。

* `removal_tid_lock` ([UInt64](/zh/reference/data-types/index)) — `removal_tid` 的哈希值，用于锁定要移除的对象。

* `removal_tid` ([Tuple(UInt64, UInt64, UUID)](/zh/reference/data-types/index)) — 已移除或正在尝试移除此对象的事务 ID

* `creation_csn` ([UInt64](/zh/reference/data-types/index)) — 创建此对象的事务的 CSN

* `removal_csn` ([UInt64](/zh/reference/data-types/index)) — 移除此对象的事务的 CSN

* `has_lightweight_delete` ([UInt8](/zh/reference/data-types/index)) — 指示该 part 是否带有轻量级删除掩码的标志。

* `last_removal_attempt_time` ([DateTime](/zh/reference/data-types/index)) — 服务器上次尝试删除该 part 的时间。
