<div id="description">
  ## 说明
</div>

包含运行查询时所执行的依赖视图的信息，例如视图类型或执行时间。

要开始记录日志：

1. 在 [query&#95;views&#95;log](/zh/reference/settings/server-settings/settings#query_views_log) 部分中配置参数。
2. 将 [log&#95;query&#95;views](/zh/reference/settings/session-settings#log_query_views) 设置为 1。

数据的刷新周期通过 [query&#95;views&#95;log](/zh/reference/settings/server-settings/settings#query_views_log) 服务器设置部分中的 `flush_interval_milliseconds` 参数来设置。要强制刷新，请使用 [SYSTEM FLUSH LOGS](/zh/reference/statements/system#flush-logs) 查询。

ClickHouse 不会自动从表中删除数据。更多信息，请参见[简介](/zh/reference/system-tables/overview#system-tables-introduction)。

你可以使用 [log&#95;queries&#95;probability](/zh/reference/settings/session-settings#log_queries_probability)) 设置，减少记录到 `query_views_log` 表中的查询数量。

<div id="columns">
  ## 列
</div>

{/*AUTOGENERATED_START*/}

* `hostname` ([LowCardinality(String)](/zh/reference/data-types/lowcardinality)) — 执行该查询的 server 的主机名。
* `event_date` ([Date](/zh/reference/data-types/date)) — 该视图最后一次事件发生的日期。
* `event_time` ([DateTime](/zh/reference/data-types/datetime)) — 该视图执行完成的日期和时间。
* `event_time_microseconds` ([DateTime64(6)](/zh/reference/data-types/datetime64)) — 该视图执行完成的日期和时间，精确到微秒。
* `view_duration_ms` ([UInt64](/zh/reference/data-types/int-uint)) — 视图执行耗时 (其各阶段耗时总和) ，单位为毫秒。
* `initial_query_id` ([String](/zh/reference/data-types/string)) — 初始查询的 ID (用于 Distributed 查询执行) 。
* `view_name` ([String](/zh/reference/data-types/string)) — 视图名称。
* `view_uuid` ([UUID](/zh/reference/data-types/uuid)) — 视图的 UUID。
* `view_type` ([Enum8(&#39;Default&#39; = 1, &#39;Materialized&#39; = 2, &#39;Live&#39; = 3, &#39;Window&#39; = 4)](/zh/reference/data-types/enum)) — 视图类型。取值：&#39;Default&#39; = 1 — 默认视图，不应出现在此日志中；&#39;Materialized&#39; = 2 — Materialized 视图；&#39;Live&#39; = 3 — Live 视图。
* `view_query` ([String](/zh/reference/data-types/string)) — 视图执行的查询语句。
* `view_target` ([String](/zh/reference/data-types/string)) — 视图目标表的名称。
* `read_rows` ([UInt64](/zh/reference/data-types/int-uint)) — 读取的行数。
* `read_bytes` ([UInt64](/zh/reference/data-types/int-uint)) — 读取的字节数。
* `written_rows` ([UInt64](/zh/reference/data-types/int-uint)) — 写入的行数。
* `written_bytes` ([UInt64](/zh/reference/data-types/int-uint)) — 写入的字节数。
* `peak_memory_usage` ([Int64](/zh/reference/data-types/int-uint)) — 此视图上下文中已分配内存与已释放内存之差的最大值。
* `ProfileEvents` ([Map(LowCardinality(String), UInt64)](/zh/reference/data-types/map)) — 用于衡量不同指标的 ProfileEvents。其说明可在 system.events 表中找到。
* `status` ([Enum8(&#39;QueryStart&#39; = 1, &#39;QueryFinish&#39; = 2, &#39;ExceptionBeforeStart&#39; = 3, &#39;ExceptionWhileProcessing&#39; = 4)](/zh/reference/data-types/enum)) — 视图状态。取值：&#39;QueryStart&#39; = 1 — 视图执行成功启动，不应出现；&#39;QueryFinish&#39; = 2 — 视图执行成功结束；&#39;ExceptionBeforeStart&#39; = 3 — 视图执行开始前发生异常；&#39;ExceptionWhileProcessing&#39; = 4 — 视图执行过程中发生异常。
* `exception_code` ([Int32](/zh/reference/data-types/int-uint)) — 异常代码。
* `exception` ([String](/zh/reference/data-types/string)) — 异常消息。
* `stack_trace` ([String](/zh/reference/data-types/string)) — 堆栈跟踪。如果查询成功完成，则为空字符串。

**别名：**

* `ProfileEvents.Names` — `mapKeys(ProfileEvents)` 的别名。
* `ProfileEvents.Values` — `mapValues(ProfileEvents)` 的别名。

{/*AUTOGENERATED_END*/}
