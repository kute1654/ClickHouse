<div id="description">
  ## 描述
</div>

包含致命错误的堆栈跟踪信息。默认情况下，数据库中不存在此表，只有在发生致命错误时才会创建。

<div id="columns">
  ## 列
</div>

{/*AUTOGENERATED_START*/}

* `hostname` ([LowCardinality(String)](/zh/reference/data-types/lowcardinality)) — 发生崩溃的主机名。
* `event_date` ([Date](/zh/reference/data-types/date)) — 崩溃发生的日期。
* `event_time` ([DateTime](/zh/reference/data-types/datetime)) — 崩溃发生的时间。
* `timestamp_ns` ([UInt64](/zh/reference/data-types/int-uint)) — 事件的纳秒级时间戳。
* `signal` ([Int32](/zh/reference/data-types/int-uint)) — 信号编号。
* `signal_code` ([Int32](/zh/reference/data-types/int-uint)) — 信号代码，提供有关信号成因的额外上下文。
* `thread_id` ([UInt64](/zh/reference/data-types/int-uint)) — 线程 ID。
* `query_id` ([String](/zh/reference/data-types/string)) — 查询 ID。
* `query` ([String](/zh/reference/data-types/string)) — 崩溃发生时正在执行的查询文本。
* `trace` ([Array(UInt64)](/zh/reference/data-types/array)) — 崩溃发生瞬间的堆栈跟踪。每个元素都是 ClickHouse server 进程中的一个虚拟内存地址。
* `trace_full` ([Array(String)](/zh/reference/data-types/array)) — 崩溃发生瞬间的堆栈跟踪。每个元素都包含 ClickHouse server 进程中的一个被调用方法。
* `fault_address` ([Nullable(UInt64)](/zh/reference/data-types/nullable)) — 导致故障的内存地址。
* `fault_access_type` ([String](/zh/reference/data-types/string)) — 导致故障的内存访问类型 (例如 &#39;read&#39;、&#39;write&#39;) 。
* `signal_description` ([String](/zh/reference/data-types/string)) — 基于 signal&#95;code 的可读描述 (例如 &#39;Address not mapped to object&#39;) 。
* `current_exception_trace_full` ([Array(String)](/zh/reference/data-types/array)) — 当前异常的完整堆栈跟踪，栈帧中的符号已解析 (仅适用于 std::terminate 崩溃) 。
* `version` ([String](/zh/reference/data-types/string)) — ClickHouse server 版本。
* `revision` ([UInt32](/zh/reference/data-types/int-uint)) — ClickHouse server 修订号。
* `build_id` ([String](/zh/reference/data-types/string)) — 由编译器生成的 BuildID。
* `git_hash` ([String](/zh/reference/data-types/string)) — ClickHouse 源代码的 Git 提交哈希。
* `architecture` ([String](/zh/reference/data-types/string)) — CPU 架构 (例如 x86&#95;64、aarch64) 。

{/*AUTOGENERATED_END*/}
