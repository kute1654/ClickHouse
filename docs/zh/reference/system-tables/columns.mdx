<div id="description">
  ## 描述
</div>

包含所有表中各列的信息。

你可以使用此表获取与 [DESCRIBE TABLE](/zh/reference/statements/describe-table) 查询类似的信息，但可以一次查看多个表。

来自[临时表](/zh/reference/statements/create/table#temporary-tables)的列，仅会在创建它们的那些 `session` 中显示在 `system.columns` 里。它们的 `database` 字段为空。

<div id="columns">
  ## 列
</div>

{/*AUTOGENERATED_START*/}

* `database` ([String](/zh/reference/data-types/index)) — 数据库名称。
* `table` ([String](/zh/reference/data-types/index)) — 表名。
* `name` ([String](/zh/reference/data-types/index)) — 列名。
* `type` ([String](/zh/reference/data-types/index)) — 列类型。
* `position` ([UInt64](/zh/reference/data-types/index)) — 列在表中的序号位置，从 1 开始。
* `default_kind` ([String](/zh/reference/data-types/index)) — 默认值的表达式类型 (DEFAULT、MATERIALIZED、ALIAS) ，如果未定义则为空字符串。
* `default_expression` ([String](/zh/reference/data-types/index)) — 默认值表达式，如果未定义则为空字符串。
* `data_compressed_bytes` ([UInt64](/zh/reference/data-types/index)) — 压缩后数据的大小，以字节为单位。
* `data_uncompressed_bytes` ([UInt64](/zh/reference/data-types/index)) — 未压缩数据的大小，以字节为单位。
* `marks_bytes` ([UInt64](/zh/reference/data-types/index)) — 标记的大小，以字节为单位。
* `comment` ([String](/zh/reference/data-types/index)) — 列注释，如果未定义则为空字符串。
* `is_in_partition_key` ([UInt8](/zh/reference/data-types/index)) — 指示该列是否在分区表达式中的标志。
* `is_in_sorting_key` ([UInt8](/zh/reference/data-types/index)) — 指示该列是否在排序键表达式中的标志。
* `is_in_primary_key` ([UInt8](/zh/reference/data-types/index)) — 指示该列是否在主键表达式中的标志。
* `is_in_sampling_key` ([UInt8](/zh/reference/data-types/index)) — 指示该列是否在采样键表达式中的标志。
* `compression_codec` ([String](/zh/reference/data-types/index)) — 压缩编解码器名称。
* `character_octet_length` ([Nullable(UInt64)](/zh/reference/data-types/index)) — 二进制数据、字符数据、文本数据和图像的最大字节长度。在 ClickHouse 中，这仅对 FixedString 数据类型有意义。否则返回 NULL 值。
* `numeric_precision` ([Nullable(UInt64)](/zh/reference/data-types/index)) — 近似数值数据、精确数值数据、整数数据或货币数据的精度。在 ClickHouse 中，对于整数类型表示比特宽度，对于 Decimal 类型表示十进制精度。否则返回 NULL 值。
* `numeric_precision_radix` ([Nullable(UInt64)](/zh/reference/data-types/index)) — 数制的基数，即近似数值数据、精确数值数据、整数数据或货币数据精度所基于的进制。在 ClickHouse 中，对于整数类型为 2，对于 Decimal 类型为 10。否则返回 NULL 值。
* `numeric_scale` ([Nullable(UInt64)](/zh/reference/data-types/index)) — 近似数值数据、精确数值数据、整数数据或货币数据的标度。在 ClickHouse 中，这仅对 Decimal 类型有意义。否则返回 NULL 值。
* `datetime_precision` ([Nullable(UInt64)](/zh/reference/data-types/index)) — DateTime64 数据类型的十进制精度。对于其他数据类型，返回 NULL 值。
* `serialization_hint` ([Nullable(String)](/zh/reference/data-types/index)) — 提示列在插入时根据统计信息选择序列化方式。
* `statistics` ([String](/zh/reference/data-types/index)) — 为该列创建的统计信息类型。

{/*AUTOGENERATED_END*/}
