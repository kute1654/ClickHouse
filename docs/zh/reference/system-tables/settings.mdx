<div id="description">
  ## 描述
</div>

包含当前用户的会话设置信息。

<div id="columns">
  ## 列
</div>

{/*AUTOGENERATED_START*/}

* `name` ([String](/zh/reference/data-types/index)) — 设置名称。
* `value` ([String](/zh/reference/data-types/index)) — 设置值。
* `changed` ([UInt8](/zh/reference/data-types/index)) — 显示该设置是否已在配置中显式定义或被显式修改。
* `description` ([String](/zh/reference/data-types/index)) — 设置的简要说明。
* `min` ([Nullable(String)](/zh/reference/data-types/index)) — 该设置的最小值 (如果已通过约束设置) 。如果该设置没有最小值，则为 NULL。
* `max` ([Nullable(String)](/zh/reference/data-types/index)) — 该设置的最大值 (如果已通过约束设置) 。如果该设置没有最大值，则为 NULL。
* `disallowed_values` ([Array(String)](/zh/reference/data-types/index)) — 不允许的值列表
* `readonly` ([UInt8](/zh/reference/data-types/index)) — 显示当前用户是否可以更改该设置：0 — 当前用户可以更改该设置，1 — 当前用户不能更改该设置。
* `type` ([String](/zh/reference/data-types/index)) — 可赋给此设置的值类型。
* `default` ([String](/zh/reference/data-types/index)) — 设置的默认值。
* `alias_for` ([String](/zh/reference/data-types/index)) — 如果该设置是另一个设置的别名，则为原始设置的名称。
* `is_obsolete` ([UInt8](/zh/reference/data-types/index)) — 显示该设置是否已废弃。
* `tier` ([Enum8(&#39;Production&#39; = 0, &#39;Obsolete&#39; = 4, &#39;Experimental&#39; = 8, &#39;Beta&#39; = 12)](/zh/reference/data-types/index)) —
  此特性的支持级别。ClickHouse 特性按层级组织，具体取决于其当前
  开发状态以及使用时可预期的行为：

- PRODUCTION：该特性属于稳定版本，可安全使用，且与其他 PRODUCTION 特性交互时不会出现问题。
- BETA：该特性稳定且安全。与其他特性一同使用时的结果未知，且不保证正确性。欢迎测试并提交反馈。
- EXPERIMENTAL：该特性仍在开发中。仅适用于开发者和 ClickHouse 爱好者。该特性可能可用，也可能不可用，并且可能随时被移除。
- OBSOLETE：不再受支持。它要么已被移除，要么将在未来版本中被移除。

{/*AUTOGENERATED_END*/}

<div id="example">
  ## 示例
</div>

以下示例展示了如何获取名称中包含 `min_i` 的设置信息。

```sql
SELECT *
FROM system.settings
WHERE name LIKE '%min_insert_block_size_%'
FORMAT Vertical
```

使用 `WHERE changed` 很有用，例如，当你想检查以下内容时：

* 配置文件中的设置是否已正确加载并生效。
* 当前会话中发生变更的设置。

```sql
SELECT * FROM system.settings WHERE changed AND name='load_balancing'
```
