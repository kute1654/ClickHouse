<div id="description">
  ## 说明
</div>

包含通过 jemalloc 分配器在不同大小类 (bins) 中进行的内存分配信息，这些信息聚合自所有 Arena。
由于 jemalloc 中的线程本地缓存，这些统计信息可能无法做到绝对准确。

<div id="columns">
  ## 列
</div>

{/*AUTOGENERATED_START*/}

* `index` ([UInt16](/zh/reference/data-types/index)) — 按大小排序的 bin 索引。
* `large` ([UInt8](/zh/reference/data-types/index)) — 大型分配时为 True，小型分配时为 False。
* `size` ([UInt64](/zh/reference/data-types/index)) — 此 bin 中分配的大小。
* `allocations` ([Int64](/zh/reference/data-types/index)) — 分配次数。
* `deallocations` ([Int64](/zh/reference/data-types/index)) — 释放次数。
* `nregs` ([Int64](/zh/reference/data-types/index)) — 每个 slab 的 region 数量。
* `curslabs` ([Int64](/zh/reference/data-types/index)) — 当前 slab 数量。
* `curregs` ([Int64](/zh/reference/data-types/index)) — 此大小类当前的 region 数量。

{/*AUTOGENERATED_END*/}

<div id="example">
  ## 示例
</div>

找出对当前总体内存使用量贡献最大的各项内存分配大小。

```sql
SELECT
    *,
    allocations - deallocations AS active_allocations,
    size * active_allocations AS allocated_bytes
FROM system.jemalloc_bins
WHERE allocated_bytes > 0
ORDER BY allocated_bytes DESC
LIMIT 10
```
