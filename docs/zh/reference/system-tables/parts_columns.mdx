<div id="description">
  ## 描述
</div>

包含有关 [MergeTree](/zh/reference/engines/table-engines/mergetree-family/mergetree) 表的 parts 和列的信息。
每一行对应一个数据分区片段。

<div id="columns">
  ## 列
</div>

{/*AUTOGENERATED_START*/}

* `partition` ([String](/zh/reference/data-types/index)) — 分区名称。
* `name` ([String](/zh/reference/data-types/index)) — 数据分区片段的名称。
* `uuid` ([UUID](/zh/reference/data-types/index)) — 数据分区片段的 UUID。
* `part_type` ([String](/zh/reference/data-types/index)) — 数据分区片段的存储格式。可能的值：Wide — 每列都存储在文件系统中的单独文件中，Compact — 所有列都存储在文件系统中的同一个文件中。
* `active` ([UInt8](/zh/reference/data-types/index)) — 用于指示数据分区片段是否处于活动状态的标志。如果数据分区片段处于活动状态，则会被表使用。否则，它会被删除。非活动的数据分区片段在合并后仍会保留。
* `marks` ([UInt64](/zh/reference/data-types/index)) — 标记数。要获取数据分区片段中行数的近似值，可将 marks 乘以 index granularity (通常为 8192)  (此提示不适用于 adaptive granularity) 。
* `rows` ([UInt64](/zh/reference/data-types/index)) — 行数。
* `bytes_on_disk` ([UInt64](/zh/reference/data-types/index)) — 所有数据分区片段文件的总大小 (以字节为单位) 。
* `data_compressed_bytes` ([UInt64](/zh/reference/data-types/index)) — 数据分区片段中压缩数据的总大小。不包括所有辅助文件 (例如标记文件) 。
* `data_uncompressed_bytes` ([UInt64](/zh/reference/data-types/index)) — 数据分区片段中未压缩数据的总大小。不包括所有辅助文件 (例如标记文件) 。
* `marks_bytes` ([UInt64](/zh/reference/data-types/index)) — 标记文件的大小。
* `modification_time` ([日期时间](/zh/reference/data-types/index)) — 存放数据分区片段的目录被修改的时间。这通常对应于数据分区片段的创建时间。
* `remove_time` ([日期时间](/zh/reference/data-types/index)) — 数据分区片段变为非活动状态的时间。
* `refcount` ([UInt32](/zh/reference/data-types/index)) — 引用该数据分区片段的位置数量。值大于 2 表示该数据分区片段正用于查询或合并。
* `min_date` ([Date](/zh/reference/data-types/index)) — 如果 Date 列包含在分区键中，则为该列的最小值。
* `max_date` ([Date](/zh/reference/data-types/index)) — 如果 Date 列包含在分区键中，则为该列的最大值。
* `min_time` ([日期时间](/zh/reference/data-types/index)) — 如果 日期时间 列包含在分区键中，则为该列的最小值。
* `max_time` ([日期时间](/zh/reference/data-types/index)) — 如果 日期时间 列包含在分区键中，则为该列的最大值。
* `partition_id` ([String](/zh/reference/data-types/index)) — 分区 ID。
* `min_block_number` ([Int64](/zh/reference/data-types/index)) — 合并后构成当前数据分区片段的最小数据分区片段编号。
* `max_block_number` ([Int64](/zh/reference/data-types/index)) — 合并后构成当前数据分区片段的最大数据分区片段编号。
* `level` ([UInt32](/zh/reference/data-types/index)) — 合并树的层级。零表示当前数据分区片段是通过插入创建的，而不是通过合并其他 parts 创建的。
* `data_version` ([UInt64](/zh/reference/data-types/index)) — 用于确定哪些变更应应用于该数据分区片段的编号 (版本高于 data&#95;version 的变更) 。
* `primary_key_bytes_in_memory` ([UInt64](/zh/reference/data-types/index)) — 主键值使用的内存量 (以字节为单位) 。
* `primary_key_bytes_in_memory_allocated` ([UInt64](/zh/reference/data-types/index)) — 为主键值预留的内存量 (以字节为单位) 。
* `database` ([String](/zh/reference/data-types/index)) — 数据库名称。
* `table` ([String](/zh/reference/data-types/index)) — 表名称。
* `engine` ([String](/zh/reference/data-types/index)) — 不带参数的表引擎名称。
* `disk_name` ([String](/zh/reference/data-types/index)) — 存储数据分区片段的磁盘名称。
* `path` ([String](/zh/reference/data-types/index)) — 包含数据分区片段文件的文件夹绝对路径。
* `column` ([String](/zh/reference/data-types/index)) — 列名称。
* `type` ([String](/zh/reference/data-types/index)) — 列类型。
* `column_position` ([UInt64](/zh/reference/data-types/index)) — 列在表中的序号，从 1 开始。
* `default_kind` ([String](/zh/reference/data-types/index)) — 默认值的表达式类型 (DEFAULT、MATERIALIZED、ALIAS) ；如果未定义，则为空字符串。
* `default_expression` ([String](/zh/reference/data-types/index)) — 默认值表达式；如果未定义，则为空字符串。
* `column_bytes_on_disk` ([UInt64](/zh/reference/data-types/index)) — 列在磁盘上的总大小 (以字节为单位) 。
* `column_data_compressed_bytes` ([UInt64](/zh/reference/data-types/index)) — 列中压缩数据的总大小 (以字节为单位) 。
* `column_data_uncompressed_bytes` ([UInt64](/zh/reference/data-types/index)) — 列中未压缩数据的总大小 (以字节为单位) 。
* `column_marks_bytes` ([UInt64](/zh/reference/data-types/index)) — 列的标记大小 (以字节为单位) 。
* `column_modification_time` ([Nullable(日期时间)](/zh/reference/data-types/index)) — 列的最后修改时间。
* `column_ttl_min` ([Nullable(日期时间)](/zh/reference/data-types/index)) — 列计算得出的生存时间 (TTL) 表达式最小值。
* `column_ttl_max` ([Nullable(日期时间)](/zh/reference/data-types/index)) — 列计算得出的生存时间 (TTL) 表达式最大值。
* `statistics` ([Array(String)](/zh/reference/data-types/index)) — 列的统计信息。
* `estimates.min` ([Nullable(String)](/zh/reference/data-types/index)) — 列的估计最小值。
* `estimates.max` ([Nullable(String)](/zh/reference/data-types/index)) — 列的估计最大值。
* `estimates.cardinality` ([Nullable(UInt64)](/zh/reference/data-types/index)) — 列的估计基数。
* `estimates.null_count` ([Nullable(UInt64)](/zh/reference/data-types/index)) — 列中 NULL 值的估计数量。
* `serialization_kind` ([String](/zh/reference/data-types/index)) — 列的序列化类型
* `substreams` ([Array(String)](/zh/reference/data-types/index)) — 列被序列化到的子流名称
* `filenames` ([Array(String)](/zh/reference/data-types/index)) — 列的每个子流对应的文件名
* `subcolumns.names` ([Array(String)](/zh/reference/data-types/index)) — 列的各个子列名称
* `subcolumns.types` ([Array(String)](/zh/reference/data-types/index)) — 列的各个子列类型
* `subcolumns.serializations` ([Array(String)](/zh/reference/data-types/index)) — 列的各个子列的序列化类型
* `subcolumns.bytes_on_disk` ([Array(UInt64)](/zh/reference/data-types/index)) — 每个子列在磁盘上的大小 (以字节为单位)
* `subcolumns.data_compressed_bytes` ([Array(UInt64)](/zh/reference/data-types/index)) — 每个子列的压缩后数据大小 (以字节为单位)
* `subcolumns.data_uncompressed_bytes` ([Array(UInt64)](/zh/reference/data-types/index)) — 每个子列的未压缩数据大小 (以字节为单位)
* `subcolumns.marks_bytes` ([Array(UInt64)](/zh/reference/data-types/index)) — 列的每个子列的标记大小 (以字节为单位)

{/*AUTOGENERATED_END*/}
