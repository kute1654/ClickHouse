<div id="description">
  ## 描述
</div>

包含有关 [配额](/zh/reference/system-tables/quotas) 的信息。

<div id="columns">
  ## 列
</div>

{/*AUTOGENERATED_START*/}

* `name` ([String](/zh/reference/data-types/index)) — 配额名称。
* `id` ([UUID](/zh/reference/data-types/index)) — 配额 ID。
* `storage` ([String](/zh/reference/data-types/index)) — 配额的存储位置。可能的值：如果配额在 users.xml 文件中配置，则为 &#39;users.xml&#39;；如果配额通过 SQL 查询配置，则为 &#39;disk&#39;。
* `keys` ([Array(Enum8(&#39;user&#95;name&#39; = 1, &#39;ip&#95;address&#39; = 2, &#39;forwarded&#95;ip&#95;address&#39; = 3, &#39;client&#95;key&#39; = 4, &#39;normalized&#95;query&#95;hash&#39; = 7))](/zh/reference/data-types/index)) — 键用于指定配额的共享方式。如果两个连接使用相同的配额和键，它们会共享同一组资源配额。取值：[] — 所有用户共享同一个配额，[&#39;user&#95;name&#39;] — 用户名相同的连接共享同一个配额，[&#39;ip&#95;address&#39;] — 来自同一 IP 的连接共享同一个配额，[&#39;client&#95;key&#39;] — 键相同的连接共享同一个配额。该键必须由客户端显式提供。使用 clickhouse-client 时，可通过 --quota&#95;key 参数传递键值，或在客户端配置文件中使用 quota&#95;key 参数。使用 HTTP interface 时，使用 X-ClickHouse-Quota 请求头，[&#39;user&#95;name&#39;, &#39;client&#95;key&#39;] — client&#95;key 相同的连接共享同一个配额。如果客户端未提供键，则按 `user_name` 跟踪配额，[&#39;client&#95;key&#39;, &#39;ip&#95;address&#39;] — client&#95;key 相同的连接共享同一个配额。如果客户端未提供键，则按 ip&#95;address 跟踪配额。
* `durations` ([Array(UInt32)](/zh/reference/data-types/index)) — 以秒为单位的时间间隔长度。
* `apply_to_all` ([UInt8](/zh/reference/data-types/index)) — 逻辑值，表示配额应用于哪些用户。值：0 — 配额适用于 apply&#95;to&#95;list 中指定的用户。1 — 配额适用于除 apply&#95;to&#95;except 中列出的用户之外的所有用户。
* `apply_to_list` ([Array(String)](/zh/reference/data-types/index)) — 应应用此配额的用户名/角色列表。
* `apply_to_except` ([Array(String)](/zh/reference/data-types/index)) — 不应应用此配额的用户名/角色列表。
* `ipv4_prefix_bits` ([Nullable(UInt8)](/zh/reference/data-types/index)) — 用于 IPv4 地址掩码的前缀位数。来自同一子网的连接共享同一个配额。仅在键为 `ip_address` 或 `forwarded_ip_address` 时使用。
* `ipv6_prefix_bits` ([Nullable(UInt8)](/zh/reference/data-types/index)) — 用于 IPv6 地址掩码的前缀位数。来自同一子网的连接共享同一个配额。仅在键为 `ip_address` 或 `forwarded_ip_address` 时使用。

{/*AUTOGENERATED_END*/}
