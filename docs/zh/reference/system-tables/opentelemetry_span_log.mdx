<div id="description">
  ## 描述
</div>

包含有关已执行查询的 [trace spans](https://opentracing.io/docs/overview/spans/) 信息。

<div id="columns">
  ## 列
</div>

{/*AUTOGENERATED_START*/}

* `hostname` ([LowCardinality(String)](/zh/reference/data-types/lowcardinality)) — 捕获此 span 的主机名。
* `trace_id` ([UUID](/zh/reference/data-types/uuid)) — 已执行查询的 trace ID。
* `span_id` ([UInt64](/zh/reference/data-types/int-uint)) — trace span 的 ID。
* `parent_span_id` ([UInt64](/zh/reference/data-types/int-uint)) — 父 trace span 的 ID。
* `operation_name` ([LowCardinality(String)](/zh/reference/data-types/lowcardinality)) — 操作名称。
* `kind` ([Enum8(&#39;INTERNAL&#39; = 0, &#39;SERVER&#39; = 1, &#39;CLIENT&#39; = 2, &#39;PRODUCER&#39; = 3, &#39;CONSUMER&#39; = 4)](/zh/reference/data-types/enum)) — 该 span 的 SpanKind。INTERNAL — 表示该 span 代表应用程序内部的操作。SERVER — 表示该 span 表示服务端对同步 RPC 或其他远程请求的处理。CLIENT — 表示该 span 描述对某个远程服务的请求。PRODUCER — 表示该 span 描述异步请求的发起方。这个父 span 通常会在对应的子 CONSUMER span 结束之前结束，甚至可能早于子 span 开始。CONSUMER - 表示该 span 描述异步 PRODUCER 请求的子级。
* `start_time_us` ([UInt64](/zh/reference/data-types/int-uint)) — trace span 的开始时间 (以微秒为单位) 。
* `finish_time_us` ([UInt64](/zh/reference/data-types/int-uint)) — trace span 的结束时间 (以微秒为单位) 。
* `finish_date` ([Date](/zh/reference/data-types/date)) — trace span 的结束日期。
* `status_code` ([Enum8(&#39;UNSET&#39; = 0, &#39;OK&#39; = 1, &#39;ERROR&#39; = 2)](/zh/reference/data-types/enum)) — span 的状态码。
* `status_message` ([LowCardinality(String)](/zh/reference/data-types/lowcardinality)) — 错误消息。
* `attribute` ([Map(LowCardinality(String), String)](/zh/reference/data-types/map)) — trace span 相关的 attribute。它们根据 OpenTelemetry 标准中的建议进行填充。

**别名：**

* `attribute.names` — `mapKeys(attribute)` 的别名。
* `attribute.values` — `mapValues(attribute)` 的别名。

{/*AUTOGENERATED_END*/}
