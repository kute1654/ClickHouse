`INFORMATION_SCHEMA` (或 `information_schema`) 是一个系统数据库，提供一种近乎标准化的、[与 DBMS 无关的视图](https://en.wikipedia.org/wiki/Information_schema)，用于查看数据库对象的元数据。`INFORMATION_SCHEMA` 中的视图通常不如常规系统表完善，但工具可以借助它们以跨 DBMS 的方式获取基础信息。`INFORMATION_SCHEMA` 中各视图的结构和内容应以向后兼容的方式演进，也就是说，只会添加新功能，而不会更改或移除现有功能。从内部实现来看，`INFORMATION_SCHEMA` 中的视图通常映射到常规系统表，例如 [system.columns](/zh/reference/system-tables/columns)、[system.databases](/zh/reference/system-tables/databases) 和 [system.tables](/zh/reference/system-tables/tables)。

```sql
SHOW TABLES FROM INFORMATION_SCHEMA;

-- or:
SHOW TABLES FROM information_schema;
```

```text
┌─name────────────────────┐
│ COLUMNS                 │
│ KEY_COLUMN_USAGE        │
│ REFERENTIAL_CONSTRAINTS │
│ SCHEMATA                │
| STATISTICS              |
│ TABLES                  │
│ VIEWS                   │
│ columns                 │
│ key_column_usage        │
│ referential_constraints │
│ schemata                │
| statistics              |
│ tables                  │
│ views                   │
└─────────────────────────┘
```

`INFORMATION_SCHEMA` 包含以下视图：

* [COLUMNS](#columns)
* [KEY&#95;COLUMN&#95;USAGE](#key_column_usage)
* [REFERENTIAL&#95;CONSTRAINTS](#referential_constraints)
* [SCHEMATA](#schemata)
* [STATISTICS](#statistics)
* [TABLES](#tables)
* [VIEWS](#views)

出于兼容其他数据库的考虑，还提供了不区分大小写的对应视图，例如 `INFORMATION_SCHEMA.columns`。这些视图中的所有列也是如此——同时提供了小写 (例如 `table_name`) 和大写 (`TABLE_NAME`) 两种形式。

<div id="schemata">
  ## SCHEMATA
</div>

包含从 [system.databases](/zh/reference/system-tables/databases) 系统表读取的列，以及 ClickHouse 中不受支持或没有实际意义 (始终为 `NULL`) 但标准要求必须存在的列。

列：

* `catalog_name` ([String](/zh/reference/data-types/string)) — 数据库名称。
* `schema_name` ([String](/zh/reference/data-types/string)) — 数据库名称。
* `schema_owner` ([String](/zh/reference/data-types/string)) — schema 所有者名称，始终为 `'default'`。
* `default_character_set_catalog` ([Nullable](/zh/reference/data-types/nullable)([String](/zh/reference/data-types/string))) — `NULL`，不受支持。
* `default_character_set_schema` ([Nullable](/zh/reference/data-types/nullable)([String](/zh/reference/data-types/string))) — `NULL`，不受支持。
* `default_character_set_name` ([Nullable](/zh/reference/data-types/nullable)([String](/zh/reference/data-types/string))) — `NULL`，不受支持。
* `sql_path` ([Nullable](/zh/reference/data-types/nullable)([String](/zh/reference/data-types/string))) — `NULL`，不受支持。

**示例**

```sql title="Query"
SELECT catalog_name,
       schema_name,
       schema_owner,
       default_character_set_catalog,
       default_character_set_schema,
       default_character_set_name,
       sql_path
FROM information_schema.schemata
WHERE schema_name ILIKE 'information_schema' 
LIMIT 1 
FORMAT Vertical;
```

<div id="tables">
  ## TABLES
</div>

包含从 [system.tables](/zh/reference/system-tables/tables) 系统表中读取的各列。

列：

* `table_catalog` ([String](/zh/reference/data-types/string)) — 表所在数据库的名称。
* `table_schema` ([String](/zh/reference/data-types/string)) — 表所在数据库的名称。
* `table_name` ([String](/zh/reference/data-types/string)) — 表名。
* `table_type` ([String](/zh/reference/data-types/string)) — 表类型。可能的值：
  * `BASE TABLE`
  * `VIEW`
  * `FOREIGN TABLE`
  * `LOCAL TEMPORARY`
  * `SYSTEM VIEW`
* `table_rows` ([Nullable](/zh/reference/data-types/nullable)([UInt64](/zh/reference/data-types/int-uint))) — 总行数。若无法确定，则为 NULL。
* `data_length` ([Nullable](/zh/reference/data-types/nullable)([UInt64](/zh/reference/data-types/int-uint))) — 磁盘上数据的大小。若无法确定，则为 NULL。
* `index_length` ([Nullable](/zh/reference/data-types/nullable)([UInt64](/zh/reference/data-types/int-uint))) — 主键、次级索引以及所有标记的总大小。
* `table_collation` ([Nullable](/zh/reference/data-types/nullable)([String](/zh/reference/data-types/string))) — 表的默认排序规则。始终为 `utf8mb4_0900_ai_ci`。
* `table_comment` ([Nullable](/zh/reference/data-types/nullable)([String](/zh/reference/data-types/string))) — 创建表时使用的注释。

**示例**

```sql title="Query"
SELECT table_catalog, 
       table_schema, 
       table_name, 
       table_type, 
       table_collation, 
       table_comment
FROM INFORMATION_SCHEMA.TABLES
WHERE (table_schema = currentDatabase() OR table_schema = '')
  AND table_name NOT LIKE '%inner%'
LIMIT 1 
FORMAT Vertical;
```

```text title="Response"
Row 1:
──────
table_catalog:   default
table_schema:    default
table_name:      describe_example
table_type:      BASE TABLE
table_collation: utf8mb4_0900_ai_ci
table_comment:   
```

<div id="views">
  ## VIEWS
</div>

包含在使用表引擎 [View](/zh/reference/engines/table-engines/special/view) 时，从 [system.tables](/zh/reference/system-tables/tables) 系统表中读取的列。

列：

{/*AUTOGENERATED_START*/}

* `table_catalog` ([String](/zh/reference/data-types/string)) — 表所在数据库的名称。
* `table_schema` ([String](/zh/reference/data-types/string)) — 表所在数据库的名称。
* `table_name` ([String](/zh/reference/data-types/string)) — 表名。
* `view_definition` ([String](/zh/reference/data-types/string)) — 视图的 `SELECT` 查询。
* `check_option` ([String](/zh/reference/data-types/string)) — `NONE`，不进行检查。
* `is_updatable` ([Enum8](/zh/reference/data-types/enum)) — `NO`，视图不可更新。
* `is_insertable_into` ([Enum8](/zh/reference/data-types/enum)) — 显示所创建的视图是否为 [materialized](/zh/reference/statements/create/view#materialized-view)。可能的值：
  * `NO` — 所创建的视图不是 materialized。
  * `YES` — 所创建的视图是 materialized。
* `is_trigger_updatable` ([Enum8](/zh/reference/data-types/enum)) — `NO`，触发器不可更新。
* `is_trigger_deletable` ([Enum8](/zh/reference/data-types/enum)) — `NO`，触发器不可删除。
* `is_trigger_insertable_into` ([Enum8](/zh/reference/data-types/enum)) — `NO`，不会向触发器中插入数据。

{/*AUTOGENERATED_END*/}

**示例**

```sql title="Query"
CREATE VIEW v (n Nullable(Int32), f Float64) AS SELECT n, f FROM t;
CREATE MATERIALIZED VIEW mv ENGINE = Null AS SELECT * FROM system.one;
SELECT table_catalog,
       table_schema,
       table_name,
       view_definition,
       check_option,
       is_updatable,
       is_insertable_into,
       is_trigger_updatable,
       is_trigger_deletable,
       is_trigger_insertable_into
FROM information_schema.views
WHERE table_schema = currentDatabase() 
LIMIT 1
FORMAT Vertical;
```

```text title="Response"
Row 1:
──────
table_catalog:              default
table_schema:               default
table_name:                 mv
view_definition:            SELECT * FROM system.one
check_option:               NONE
is_updatable:               NO
is_insertable_into:         YES
is_trigger_updatable:       NO
is_trigger_deletable:       NO
is_trigger_insertable_into: NO
```

<div id="key_column_usage">
  ## KEY_COLUMN_USAGE
</div>

包含 [system.tables](/zh/reference/system-tables/tables) 系统表中受约束限制的列。

列：

* `constraint_catalog` ([String](/zh/reference/data-types/string)) — 当前未使用。始终为 `def`。
* `constraint_schema` ([String](/zh/reference/data-types/string)) — 该约束所属 schema (数据库) 的名称。
* `constraint_name` ([Nullable](/zh/reference/data-types/nullable)([String](/zh/reference/data-types/string))) — 约束名称。
* `table_catalog` ([String](/zh/reference/data-types/string)) — 当前未使用。始终为 `def`。
* `table_schema` ([String](/zh/reference/data-types/string)) — 该表所属 schema (数据库) 的名称。
* `table_name` ([String](/zh/reference/data-types/string)) — 具有该约束的表名称。
* `column_name` ([Nullable](/zh/reference/data-types/nullable)([String](/zh/reference/data-types/string))) — 具有该约束的列名称。
* `ordinal_position` ([UInt32](/zh/reference/data-types/int-uint)) — 当前未使用。始终为 `1`。
* `position_in_unique_constraint` ([Nullable](/zh/reference/data-types/nullable)([UInt32](/zh/reference/data-types/int-uint))) — 当前未使用。始终为 `NULL`。
* `referenced_table_schema` ([Nullable](/zh/reference/data-types/nullable)([String](/zh/reference/data-types/string))) — 当前未使用。始终为 `NULL`。
* `referenced_table_name` ([Nullable](/zh/reference/data-types/nullable)([String](/zh/reference/data-types/string))) — 当前未使用。始终为 `NULL`。
* `referenced_column_name` ([Nullable](/zh/reference/data-types/nullable)([String](/zh/reference/data-types/string))) — 当前未使用。始终为 `NULL`。

**示例**

```sql title="Query"
CREATE TABLE test (i UInt32, s String) ENGINE MergeTree ORDER BY i;
SELECT constraint_catalog,
       constraint_schema,
       constraint_name,
       table_catalog,
       table_schema,
       table_name,
       column_name,
       ordinal_position,
       position_in_unique_constraint,
       referenced_table_schema,
       referenced_table_name,
       referenced_column_name
FROM information_schema.key_column_usage 
WHERE table_name = 'test' 
FORMAT Vertical;
```
