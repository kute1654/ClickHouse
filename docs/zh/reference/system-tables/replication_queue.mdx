<div id="description">
  ## 描述
</div>

包含 ReplicatedMergeTree 家族中的表在 ClickHouse Keeper 或 ZooKeeper 中存储的复制队列任务信息。

<div id="columns">
  ## 列
</div>

{/*AUTOGENERATED_START*/}

* `database` ([String](/zh/reference/data-types/index)) — 数据库名称。
* `table` ([String](/zh/reference/data-types/index)) — 表名称。
* `replica_name` ([String](/zh/reference/data-types/index)) — ClickHouse Keeper 中的副本名称。同一张表的不同副本具有不同名称。
* `position` ([UInt32](/zh/reference/data-types/index)) — 任务在队列中的位置。
* `node_name` ([String](/zh/reference/data-types/index)) — ClickHouse Keeper 中的节点名称。
* `type` ([String](/zh/reference/data-types/index)) — 队列中任务的类型，可以是以下之一：• GET&#95;PART — 从另一个副本获取 part，• ATTACH&#95;PART — 附加 part，也可能来自本地副本 (如果在 detached 文件夹中找到) 。可将其视为经过一些优化的 GET&#95;PART，因为两者几乎相同，• MERGE&#95;PARTS — 合并 parts，• DROP&#95;RANGE — 删除指定分区中指定编号范围内的 parts。• CLEAR&#95;COLUMN — 注意：已弃用。从指定分区中删除特定列。• CLEAR&#95;INDEX — 注意：已弃用。从指定分区中删除特定索引。• REPLACE&#95;RANGE — 删除某个范围内的 parts，并用新的 parts 替换它们。• MUTATE&#95;PART — 对该 part 应用一个或多个变更。• ALTER&#95;METADATA — 根据全局 /metadata 和 /columns 路径应用 alter 变更。
* `create_time` ([DateTime](/zh/reference/data-types/index)) — 任务提交执行的日期和时间。
* `required_quorum` ([UInt32](/zh/reference/data-types/index)) — 等待确认该任务已完成的副本数量。此列仅与 GET&#95;PARTS 任务相关。
* `source_replica` ([String](/zh/reference/data-types/index)) — 源副本名称。
* `new_part_name` ([String](/zh/reference/data-types/index)) — 新 part 的名称。
* `parts_to_merge` ([Array(String)](/zh/reference/data-types/index)) — 要合并或更新的 part 名称。
* `is_detach` ([UInt8](/zh/reference/data-types/index)) — 该标志表示队列中是否存在 DETACH&#95;PARTS 任务。
* `is_currently_executing` ([UInt8](/zh/reference/data-types/index)) — 该标志表示某个特定任务当前是否正在执行。
* `num_tries` ([UInt32](/zh/reference/data-types/index)) — 完成该任务的失败尝试次数。
* `last_exception` ([String](/zh/reference/data-types/index)) — 最后一次发生的错误消息文本 (如果有) 。
* `last_exception_time` ([DateTime](/zh/reference/data-types/index)) — 最后一次发生错误的日期和时间。
* `last_attempt_time` ([DateTime](/zh/reference/data-types/index)) — 最后一次尝试执行该任务的日期和时间。
* `num_postponed` ([UInt32](/zh/reference/data-types/index)) — 该操作被推迟的次数。
* `postpone_reason` ([String](/zh/reference/data-types/index)) — 任务被推迟的原因。
* `last_postpone_time` ([DateTime](/zh/reference/data-types/index)) — 任务最后一次被推迟的日期和时间。
* `merge_type` ([String](/zh/reference/data-types/index)) — 当前合并的类型。如果是变更，则为空。

{/*AUTOGENERATED_END*/}
