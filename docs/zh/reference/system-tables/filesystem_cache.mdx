<div id="description">
  ## 描述
</div>

包含文件系统缓存中所有远程对象条目的相关信息。

<div id="columns">
  ## 列
</div>

{/*AUTOGENERATED_START*/}

* `cache_name` ([String](/zh/reference/data-types/index)) — 缓存对象的名称。可用于 `SYSTEM DESCRIBE FILESYSTEM CACHE <name>` 和 `SYSTEM DROP FILESYSTEM CACHE <name>` 命令
* `cache_base_path` ([String](/zh/reference/data-types/index)) — 用于存储所有缓存文件 (由 `cache_name` 标识的缓存) 的基础目录路径。
* `cache_path` ([String](/zh/reference/data-types/index)) — 特定缓存文件的路径，对应源文件中的一个 File 段
* `key` ([String](/zh/reference/data-types/index)) — File 段的缓存键
* `file_segment_range_begin` ([UInt64](/zh/reference/data-types/index)) — 对应 File 段范围起始位置的偏移量
* `file_segment_range_end` ([UInt64](/zh/reference/data-types/index)) — 对应 File 段范围结束位置 (含该位置) 的偏移量
* `size` ([UInt64](/zh/reference/data-types/index)) — File 段的大小
* `state` ([String](/zh/reference/data-types/index)) — File 段状态 (DOWNLOADED、DOWNLOADING、PARTIALLY&#95;DOWNLOADED，...)
* `finished_download_time` ([DateTime](/zh/reference/data-types/index)) — File 段完成下载的时间。
* `cache_hits` ([UInt64](/zh/reference/data-types/index)) — 对应 File 段的缓存命中次数
* `references` ([UInt64](/zh/reference/data-types/index)) — 对应 File 段的引用数量。值为 1 表示当前没有对象在使用它 (唯一存在的引用仅来自缓存存储本身)
* `downloaded_size` ([UInt64](/zh/reference/data-types/index)) — File 段的已下载大小
* `kind` ([String](/zh/reference/data-types/index)) — File 段类型 (用于区分作为 &#39;缓存中的临时数据&#39; 一部分添加的 File 段)
* `unbound` ([UInt8](/zh/reference/data-types/index)) — 内部实现标志
* `user_id` ([String](/zh/reference/data-types/index)) — 创建该 File 段的用户 ID
* `segment_type` ([String](/zh/reference/data-types/index)) — 分段类型。用于区分数据文件 (`.json`、`.txt` 等) 与数据文件 (`.bin`、标记文件) 。
* `file_size` ([Nullable(UInt64)](/zh/reference/data-types/index)) — 当前 File 段所属文件的大小

{/*AUTOGENERATED_END*/}
