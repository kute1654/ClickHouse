<div id="description">
  ## 说明
</div>

如果此节点未配置为运行进程内 ClickHouse Keeper，则此表不存在。该表为 Raft 集群中的每个成员各包含一行，将静态集群拓扑 (来自 Raft 配置) 与本地节点自身的日志位置结合在一起。

每个节点只会填写一个 `last_log_index` 值——也就是与其自身 `server_id` 对应的那一行。其他节点的日志位置不会在此处显示，因为这些位置仅在 leader 上跟踪，而这个视图在整个集群中并不是对称的。

<div id="columns">
  ## 列
</div>

{/*AUTOGENERATED_START*/}

* `server_id` ([Int32](/zh/reference/data-types/int-uint)) — 此集群成员的 Raft server ID。
* `host` ([String](/zh/reference/data-types/string)) — 从端点中解析出的主机 (最后一个冒号之前的前缀) 。
* `endpoint` ([String](/zh/reference/data-types/string)) — 配置中的原始 Raft 端点 (`host:port`) 。
* `is_observer` ([Bool](/zh/reference/data-types/boolean)) — 如果此成员是通过 `<can_become_leader>false</can_become_leader>` 配置的无投票权 observer，则为 `true`。
* `priority` ([Int32](/zh/reference/data-types/int-uint)) — 此成员的 Raft 优先级；在 leader election 期间，值越高越优先。
* `is_leader` ([Bool](/zh/reference/data-types/boolean)) — 如果此成员是当前的 Raft leader，则为 `true`。
* `is_self` ([Bool](/zh/reference/data-types/boolean)) — 如果此行描述的是本地 Keeper 节点，则为 `true`。
* `last_log_index` ([Nullable(UInt64)](/zh/reference/data-types/int-uint)) — 此节点本地 Raft 日志存储中的最高日志索引。仅在与当前节点对应的行 (`is_self = true`) 中填充；其他行为 `NULL`。

{/*AUTOGENERATED_END*/}

示例：

```sql
SELECT * FROM system.keeper_cluster ORDER BY server_id;
```
