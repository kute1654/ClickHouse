本地文件源会从本地文件系统中的文件加载字典数据。这适用于以 TSV、CSV 或任何其他[支持的格式](/zh/reference/formats/index)存储为平面文件的小型静态查找表。

设置示例：

<Tabs>
  <Tab title="DDL">
    ```sql
    SOURCE(FILE(path './user_files/os.tsv' format 'TabSeparated'))
    ```
  </Tab>

  <Tab title="配置文件">
    ```xml
    <source>
      <file>
        <path>/opt/dictionaries/os.tsv</path>
        <format>TabSeparated</format>
      </file>
    </source>
    ```
  </Tab>
</Tabs>

<br />

设置字段：

| Setting  | Description                                           |
| -------- | ----------------------------------------------------- |
| `path`   | 文件的绝对路径。                                              |
| `format` | 文件格式。支持 [Formats](/zh/reference/formats/index) 中描述的所有格式。 |

当通过 DDL 命令 (`CREATE DICTIONARY ...`) 创建源为 `FILE` 的字典时，源文件必须位于 `user_files` 目录中，以防止数据库用户访问 ClickHouse 节点上的任意文件。
