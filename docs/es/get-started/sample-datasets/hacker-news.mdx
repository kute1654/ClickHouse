  <Step>
    ### Definir un esquema

    Una optimizaciГіn inmediata y evidente es definir un tipo para cada campo.
    AdemГЎs de declarar el campo de tiempo como de tipo `DateTime`, definimos un tipo adecuado para cada uno de los campos siguientes despuГ©s de eliminar el dataset existente.
    En ClickHouse, el id de la clave primaria de los datos se define mediante la clГЎusula `ORDER BY`.

    Seleccionar los tipos adecuados y elegir quГ© columnas incluir en la clГЎusula `ORDER BY`
    ayudarГЎ a mejorar la velocidad de las consultas y la compresiГіn.

    Ejecute la consulta siguiente para eliminar el esquema anterior y crear el esquema mejorado:

    ```sql title="Query"
    DROP TABLE IF EXISTS hackernews;

    CREATE TABLE hackernews
    (
        `id` UInt32,
        `deleted` UInt8,
        `type` Enum('story' = 1, 'comment' = 2, 'poll' = 3, 'pollopt' = 4, 'job' = 5),
        `by` LowCardinality(String),
        `time` DateTime,
        `text` String,
        `dead` UInt8,
        `parent` UInt32,
        `poll` UInt32,
        `kids` Array(UInt32),
        `url` String,
        `score` Int32,
        `title` String,
        `parts` Array(UInt32),
        `descendants` Int32
    )
        ENGINE = MergeTree
    ORDER BY id
    ```

    Con un esquema optimizado, ahora puedes insertar los datos desde el sistema de archivos local.
    Una vez mГЎs, con `clickhouse-client`, inserta el archivo mediante la clГЎusula `INFILE` con un `INSERT INTO` explГӯcito.

    ```sql title="Query"
    INSERT INTO hackernews FROM INFILE '/data/hacknernews.csv.gz' FORMAT CSVWithNames
    ```
  </Step>

<div id="parquet">
  ## Parquet
</div>

Uno de los puntos fuertes de ClickHouse es su capacidad para manejar todo tipo de [formatos](/es/reference/formats/index).
CSV representa un caso de uso bastante ideal, pero no es el mГЎs eficiente para el intercambio de datos.

A continuaciГіn, cargarГЎs los datos desde un archivo Parquet, que es un formato eficiente orientado a columna.

Parquet tiene un conjunto mГӯnimo de tipos, que ClickHouse debe respetar, y esta informaciГіn de tipos estГЎ codificada en el propio formato.
La inferencia de tipos a partir de un archivo Parquet conducirГЎ invariablemente a un esquema ligeramente distinto del archivo CSV.
