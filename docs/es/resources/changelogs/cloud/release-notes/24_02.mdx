<div id="backward-incompatible-change">
  #### Cambio incompatible con versiones anteriores
</div>

* Validar tipos sospechosos o experimentales en tipos anidados. Anteriormente no validábamos esos tipos (salvo JSON) en tipos anidados como Array/Tuple/Map. [#59385](https://github.com/ClickHouse/ClickHouse/pull/59385) ([Kruglov Pavel](https://github.com/Avogar)).
* La cláusula de ordenación `ORDER BY ALL` (introducida en la versión 23.12) se sustituye por `ORDER BY *`. La sintaxis anterior era demasiado propensa a errores en tablas con una columna `all`. [#59450](https://github.com/ClickHouse/ClickHouse/pull/59450) ([Robert Schulze](https://github.com/rschu1ze)).
* Se añade una comprobación básica del número de hilos y de los tamaños de bloque. [#60138](https://github.com/ClickHouse/ClickHouse/pull/60138) ([Raúl Marín](https://github.com/Algunenano)).
* Rechazar las consultas INSERT entrantes cuando las configuraciones a nivel de consulta `async_insert` y `deduplicate_blocks_in_dependent_materialized_views` estén habilitadas simultáneamente. Este comportamiento se controla mediante la configuración `throw_if_deduplication_in_dependent_materialized_views_enabled_with_async_insert` y está habilitado de forma predeterminada. Esto da continuidad a https://github.com/ClickHouse/ClickHouse/pull/59699 y es necesario para desbloquear https://github.com/ClickHouse/ClickHouse/pull/59915. [#60888](https://github.com/ClickHouse/ClickHouse/pull/60888) ([Nikita Mikhaylov](https://github.com/nikitamikhaylov)).
* La utilidad `clickhouse-copier` se ha trasladado a un repositorio independiente en GitHub: https://github.com/ClickHouse/copier. Ya no se incluye en el paquete, pero sigue disponible como descarga independiente. * Para mejorar la compatibilidad con MySQL, la función `locate` ahora acepta de forma predeterminada los argumentos `(needle, haystack[, start_pos])`. El comportamiento anterior `(haystack, needle, [, start_pos])` se puede restaurar estableciendo `function_locate_has_mysql_compatible_argument_order = 0`. [#61092](https://github.com/ClickHouse/ClickHouse/pull/61092) ([Robert Schulze](https://github.com/rschu1ze)).
* Las partes de datos en memoria obsoletas quedaron en desuso desde la versión 23.5 y dejaron de ser compatibles a partir de la versión 23.10. Ahora se elimina el código restante. Continuación de [#55186](https://github.com/ClickHouse/ClickHouse/issues/55186) y [#45409](https://github.com/ClickHouse/ClickHouse/issues/45409). Es poco probable que haya usado partes de datos en memoria, porque solo estaban disponibles antes de la versión 23.5 y únicamente cuando se habilitaban manualmente especificando las SETTINGS correspondientes para una tabla MergeTree. Para comprobar si tiene partes de datos en memoria, ejecute la siguiente consulta: `SELECT part_type, count() FROM system.parts GROUP BY part_type ORDER BY part_type`. Para deshabilitar el uso de partes de datos en memoria, ejecute `ALTER TABLE ... MODIFY SETTING min_bytes_for_compact_part = DEFAULT, min_rows_for_compact_part = DEFAULT`. Antes de actualizar desde versiones antiguas de ClickHouse, compruebe primero que no tenga partes de datos en memoria. Si hay partes de datos en memoria, deshabilítelas primero; después, espere hasta que ya no haya partes de datos en memoria y continúe con la actualización. [#61127](https://github.com/ClickHouse/ClickHouse/pull/61127) ([Alexey Milovidov](https://github.com/alexey-milovidov)).
* Prohibir por defecto `SimpleAggregateFunction` en el `ORDER BY` de las tablas `MergeTree` (igual que `AggregateFunction`, se prohíben porque no son comparables) (use `allow_suspicious_primary_key` para permitirlas). [#61399](https://github.com/ClickHouse/ClickHouse/pull/61399) ([Azat Khuzhin](https://github.com/azat)).
* ClickHouse permite datos binarios arbitrarios en el tipo de datos String, que normalmente es UTF-8. Las cadenas de Parquet/ORC/Arrow solo admiten UTF-8. Por eso, puede elegir qué tipo de datos de Arrow usar para el tipo de datos String de ClickHouse: String o Binary. Esto se controla mediante las opciones de configuración `output_format_parquet_string_as_string`, `output_format_orc_string_as_string` y `output_format_arrow_string_as_string`. Aunque Binary sería más correcto y compatible, usar String de forma predeterminada se ajusta a las expectativas de los usuarios en la mayoría de los casos. Parquet/ORC/Arrow admite muchos métodos de compresión, incluidos lz4 y zstd. ClickHouse admite todos y cada uno de los métodos de compresión. Algunas herramientas menos completas no admiten el método de compresión `lz4`, que es más rápido, por eso establecemos `zstd` como valor predeterminado. Esto se controla mediante las opciones de configuración `output_format_parquet_compression_method`, `output_format_orc_compression_method` y `output_format_arrow_compression_method`. Cambiamos el valor predeterminado a `zstd` para Parquet y ORC, pero no para Arrow (ya que se usa principalmente en casos de bajo nivel). [#61817](https://github.com/ClickHouse/ClickHouse/pull/61817) ([Alexey Milovidov](https://github.com/alexey-milovidov)).
* Corrección del problema de seguridad de la vista materializada, que permitía a un usuario insertar en una tabla sin los privilegios necesarios. La corrección valida que el usuario tenga permiso para insertar no solo en una vista materializada, sino también en todas las tablas subyacentes. Esto significa que algunas consultas que antes funcionaban ahora pueden fallar con Not enough privileges. Para solucionar este problema, la versión introduce una nueva funcionalidad de SQL security para las vistas [https://clickhouse.com/docs/sql-reference/statements/create/view#sql&#95;security](/es/reference/statements/create/view#sql_security). [#54901](https://github.com/ClickHouse/ClickHouse/pull/54901) ([pufit](https://github.com/pufit))

<div id="performance-improvement">
  #### Mejora del rendimiento
</div>

* Elimina las funciones de agregación min/max/any/anyLast sobre las claves de GROUP BY en la sección SELECT. [#52230](https://github.com/ClickHouse/ClickHouse/pull/52230) ([JackyWoo](https://github.com/JackyWoo)).
* Mejora el rendimiento del método de agregación serializado cuando hay varias columnas [nullable]. Esta es una versión general de [#51399](https://github.com/ClickHouse/ClickHouse/issues/51399) que no compromete la integridad de la abstracción. [#55809](https://github.com/ClickHouse/ClickHouse/pull/55809) ([Amos Bird](https://github.com/amosbird)).
* Salida diferida de la fase build del join para mejorar el rendimiento del join ALL. [#58278](https://github.com/ClickHouse/ClickHouse/pull/58278) ([LiuNeng](https://github.com/liuneng1994)).
* Mejoras en las funciones de agregación ArgMin / ArgMax / any / anyLast / anyHeavy, así como en las consultas `ORDER BY {u8/u16/u32/u64/i8/i16/u32/i64) LIMIT 1`. [#58640](https://github.com/ClickHouse/ClickHouse/pull/58640) ([Raúl Marín](https://github.com/Algunenano)).
* Optimiza el rendimiento condicional de `sum`/`avg` para `bigint` y tipos `decimal` grandes al reducir los fallos de predicción de saltos. [#59504](https://github.com/ClickHouse/ClickHouse/pull/59504) ([李扬](https://github.com/taiyang-li)).
* Mejora el rendimiento de las consultas `SELECT` con mutaciones activas. [#59531](https://github.com/ClickHouse/ClickHouse/pull/59531) ([Azat Khuzhin](https://github.com/azat)).
* Optimización trivial del filtro de columnas. Se evitan las columnas de filtro cuyo tipo de dato subyacente no es numérico al filtrarlas con `result_size_hint = -1`. La memoria pico puede reducirse al 44 % de la original en algunos casos. [#59698](https://github.com/ClickHouse/ClickHouse/pull/59698) ([李扬](https://github.com/taiyang-li)).
* La clave primaria usará menos memoria. [#60049](https://github.com/ClickHouse/ClickHouse/pull/60049) ([Alexey Milovidov](https://github.com/alexey-milovidov)).
* Se reduce el uso de memoria de la clave primaria y de algunas otras operaciones. [#60050](https://github.com/ClickHouse/ClickHouse/pull/60050) ([Alexey Milovidov](https://github.com/alexey-milovidov)).
* Las claves primarias de las tablas se cargarán en memoria de forma diferida en el primer acceso. Esto está controlado por el nuevo ajuste de MergeTree `primary_key_lazy_load`, que está activado de forma predeterminada. Esto ofrece varias ventajas: - no se cargará para las tablas que no se utilicen; - si no hay suficiente memoria, se lanzará una excepción en el primer uso en lugar de al iniciar el servidor. Esto también presenta varias desventajas: - el coste en latencia de cargar la clave primaria se asumirá en la primera consulta en lugar de antes de aceptar conexiones; en teoría, esto podría introducir un problema de tipo thundering herd. Esto cierra [#11188](https://github.com/ClickHouse/ClickHouse/issues/11188). [#60093](https://github.com/ClickHouse/ClickHouse/pull/60093) ([Alexey Milovidov](https://github.com/alexey-milovidov)).
* Función vectorizada `dotProduct`, útil para la búsqueda vectorial. [#60202](https://github.com/ClickHouse/ClickHouse/pull/60202) ([Robert Schulze](https://github.com/rschu1ze)).
* Si la clave primaria de la tabla contiene sobre todo columnas poco útiles, no las mantengas en memoria. Esto se controla con una nueva configuración `primary_key_ratio_of_unique_prefix_values_to_skip_suffix_columns`, cuyo valor predeterminado es `0.9`, lo que significa lo siguiente: en una clave primaria compuesta, si una columna cambia de valor al menos en 0.9 de todas las ocasiones, no se cargarán las columnas posteriores. [#60255](https://github.com/ClickHouse/ClickHouse/pull/60255) ([Alexey Milovidov](https://github.com/alexey-milovidov)).
* Ejecutar la función multiIf de manera columnar cuando el tipo subyacente de result&#95;type sea numérico. [#60384](https://github.com/ClickHouse/ClickHouse/pull/60384) ([李扬](https://github.com/taiyang-li)).
* Como se muestra en la Fig. 1, sustituir &quot;&amp;&amp;&quot; por &quot;&amp;&quot; podría generar código SIMD. ![imagen](https://github.com/ClickHouse/ClickHouse/assets/26588299/a5a72ac4-6dc6-4d52-835a-4f512e55f0b9) Fig. 1. Código compilado a partir de &#39;&amp;&amp;&#39; (izquierda) y &#39;&amp;&#39; (derecha). [#60498](https://github.com/ClickHouse/ClickHouse/pull/60498) ([Zhiguo Zhou](https://github.com/ZhiguoZh)).
* Mutexes más rápidos (casi el doble) (antes eran más lentos por ThreadFuzzer). [#60823](https://github.com/ClickHouse/ClickHouse/pull/60823) ([Azat Khuzhin](https://github.com/azat)).
* Mover el drenaje de conexiones de prepare a work y drenar varias conexiones en paralelo. [#60845](https://github.com/ClickHouse/ClickHouse/pull/60845) ([lizhuoyu5](https://github.com/lzydmxy)).
* Optimiza insertManyFrom para números Nullable o cadenas Nullable. [#60846](https://github.com/ClickHouse/ClickHouse/pull/60846) ([李扬](https://github.com/taiyang-li)).
* Se optimizó la función `dotProduct` para evitar copias de memoria innecesarias y costosas. [#60928](https://github.com/ClickHouse/ClickHouse/pull/60928) ([Robert Schulze](https://github.com/rschu1ze)).
* Las operaciones con la caché del sistema de archivos se verán menos afectadas por la contención de bloqueos. [#61066](https://github.com/ClickHouse/ClickHouse/pull/61066) ([Alexey Milovidov](https://github.com/alexey-milovidov)).
* Optimiza ColumnString::replicate y evita que memcpySmallAllowReadWriteOverflow15Impl se optimice como la `memcpy` integrada. Cierra [#61074](https://github.com/ClickHouse/ClickHouse/issues/61074). ColumnString::replicate es 2,46 veces más rápido en x86-64. [#61075](https://github.com/ClickHouse/ClickHouse/pull/61075) ([李扬](https://github.com/taiyang-li)).
* Impresión 30 veces más rápida para enteros de 256 bits. [#61100](https://github.com/ClickHouse/ClickHouse/pull/61100) ([Raúl Marín](https://github.com/Algunenano)).
* Si una consulta con un error de sintaxis contenía un matcher COLUMNS con una expresión regular, la expresión regular se compilaba cada vez durante el backtracking del parser, en lugar de compilarse una sola vez. Esto era un error fundamental. La expresión regular compilada se almacenaba en el AST. Pero la letra A de AST significa &quot;abstracto&quot;, lo que implica que no debería contener objetos pesados. Algunas partes del AST pueden crearse y descartarse durante el parsing, incluso durante un gran número de retrocesos. Esto provoca lentitud en la fase de análisis y, en consecuencia, permite un DoS por parte de un usuario readonly. Pero el problema principal es que impide avanzar en los fuzzers. [#61543](https://github.com/ClickHouse/ClickHouse/pull/61543) ([Alexey Milovidov](https://github.com/alexey-milovidov)).

* Permitir declarar `enum` en la estructura de una tabla externa. [#57857](https://github.com/ClickHouse/ClickHouse/pull/57857) ([Duc Canh Le](https://github.com/canhld94)).
* Al ejecutar `ALTER COLUMN MATERIALIZE` en una columna con una expresión `DEFAULT` o `MATERIALIZED`, ahora se escriben los valores correctos: el valor predeterminado para las partes existentes con valor predeterminado, o el valor no predeterminado para las partes existentes con valor no predeterminado. Anteriormente, se escribía el valor predeterminado en todas las partes existentes. [#58023](https://github.com/ClickHouse/ClickHouse/pull/58023) ([Duc Canh Le](https://github.com/canhld94)).
* Se habilitó una lógica de backoff (p. ej., exponencial). Permitirá reducir el uso de CPU, el uso de memoria y el tamaño de los archivos de registro. [#58036](https://github.com/ClickHouse/ClickHouse/pull/58036) ([MikhailBurdukov](https://github.com/MikhailBurdukov)).
* Tener en cuenta las filas eliminadas con eliminación ligera al seleccionar las partes que se van a fusionar. [#58223](https://github.com/ClickHouse/ClickHouse/pull/58223) ([Zhuo Qiu](https://github.com/jewelzqiu)).
* Permite definir `volume_priority` en `storage_configuration`. [#58533](https://github.com/ClickHouse/ClickHouse/pull/58533) ([Andrey Zvonov](https://github.com/zvonand)).
* Se añadió soporte para el tipo Date32 en el códec T64. [#58738](https://github.com/ClickHouse/ClickHouse/pull/58738) ([Hongbin Ma](https://github.com/binmahone)).
