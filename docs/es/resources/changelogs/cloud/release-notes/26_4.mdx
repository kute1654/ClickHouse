<div id="backward-incompatible-changes">
  ## Cambios incompatibles con versiones anteriores
</div>

<div id="query-and-syntax-changes">
  ### Cambios en consultas y sintaxis
</div>

* El operador `IN` ahora usa una semántica de valores exactos para el tipo `Bool`: solo `0` y `1` del conjunto coinciden con valores `Bool`. Antes, los valores numéricos mayores que `255` en el conjunto de `IN` se convertían incorrectamente a true, por lo que `SELECT CAST(1, 'Bool') IN (256)` devolvía `1`. Ahora devuelve correctamente `0`. [#93115](https://github.com/ClickHouse/ClickHouse/pull/93115) ([Ashrith Bandla](https://github.com/ashrithb)).
* Se corrigió la precedencia del operador `NOT` para ajustarla al estándar SQL: `NOT` ahora tiene menor precedencia que `IS NULL`, `BETWEEN`, `LIKE` y los operadores aritméticos. Por ejemplo, `NOT (x) IS NULL` ahora se interpreta como `NOT (x IS NULL)` en lugar de `(NOT x) IS NULL`. Esto puede cambiar el resultado de las consultas que dependían del comportamiento anterior (no estándar). [#97680](https://github.com/ClickHouse/ClickHouse/pull/97680) ([Alexey Milovidov](https://github.com/alexey-milovidov)).
* `SELECT` ya no se permite como identificador sin comillas en un elemento de la lista de expresiones `WITH`. [#101059](https://github.com/ClickHouse/ClickHouse/pull/101059) ([Aruj Bansal](https://github.com/arujbansal)).
* El operador `IN` ahora rechaza conversiones de `Decimal` con pérdida dentro de tipos compuestos (`Tuple`, `Array`, `Map`), lo que hace que su comportamiento sea coherente con las comparaciones escalares de nivel superior. Antes, las comprobaciones de precisión solo se aplicaban a valores escalares de nivel superior: por ejemplo, `CAST('33.3', 'Decimal64(1)') IN (33.33)` devolvía correctamente `0`, pero `CAST(['33.3'], 'Array(Decimal64(1))') IN ([33.33])` devolvía incorrectamente `1`. Ahora ambos casos devuelven correctamente `0`. [#101812](https://github.com/ClickHouse/ClickHouse/pull/101812) ([Nihal Z. Miaji](https://github.com/nihalzp)).

<div id="data-type-changes">
  ### Cambios en los tipos de datos
</div>

* Las versiones de serialización de los tipos de datos ahora también se propagan a los tipos de datos anidados. Por ejemplo, la versión de serialización `with_size_stream` de String antes solo se aplicaba a las columnas String de nivel superior y a los elementos de Tuple; ahora se aplica a cualquier tipo String dentro de cualquier tipo anidado, como Array, Map, Variant o JSON. Esto se controla mediante el setting de MergeTree `propagate_types_serialization_versions_to_nested_types`, que ahora está habilitado de forma predeterminada. Después de este cambio, las partes de datos recién creadas no pueden leerse con versiones anteriores, pero las partes antiguas sí pueden leerse en la versión nueva. Las actualizaciones son seguras, pero no las reversiones a versiones anteriores. [#94859](https://github.com/ClickHouse/ClickHouse/pull/94859) ([Pavel Kruglov](https://github.com/Avogar)).
* Se corrigió el orden de bytes de la serialización binaria de `QBit` para usar el formato little-endian. [#101378](https://github.com/ClickHouse/ClickHouse/pull/101378) ([Raufs Dunamalijevs](https://github.com/rienath)).

<div id="storage-and-index-changes">
  ### Cambios en el almacenamiento y los índices
</div>

* Se corrigieron los metadatos de las proyecciones normales para que las proyecciones con claves de ordenación de varias columnas se reconozcan correctamente. [#91352](https://github.com/ClickHouse/ClickHouse/pull/91352) ([Amos Bird](https://github.com/amosbird)).
* Se corrigió un problema por el que los archivos de los índices de omisión no respetaban la configuración `replace_long_file_name_to_hash`, lo que provocaba errores de &quot;File name too long&quot; y fallos al leer índices con nombres largos. Ahora, cuando los nombres de archivo de los índices de omisión superan `max_file_name_length`, se reemplazan por un hash, igual que ocurre con los archivos de columna. Esto es retrocompatible (los servidores nuevos leen las partes antiguas), pero una reversión (o el uso de servidores antiguos durante una actualización gradual) puede hacer que se ignoren los índices con nombres largos. [#97128](https://github.com/ClickHouse/ClickHouse/pull/97128) ([Raúl Marín](https://github.com/Algunenano)).

<div id="removed-features">
  ### Características eliminadas
</div>

* Se eliminó el tipo de índice de omisión `hypothesis`. Era una característica experimental poco conocida y de utilidad práctica limitada. Crear tablas con `INDEX ... TYPE hypothesis` ahora genera un error. [#96874](https://github.com/ClickHouse/ClickHouse/pull/96874) ([Alexey Milovidov](https://github.com/alexey-milovidov)).
* Se eliminó la función experimental `detectProgrammingLanguage`. [#99567](https://github.com/ClickHouse/ClickHouse/pull/99567) ([Alexey Milovidov](https://github.com/alexey-milovidov)).

<div id="settings-and-configuration-changes">
  ### Cambios en los ajustes y la configuración
</div>

* Se cambió el valor predeterminado de `mysql_datatypes_support_level` de vacío a `decimal,datetime64,date2Date32`, lo que permite, de forma predeterminada, la correspondencia correcta de MySQL `DATE` con `Date32`, `DECIMAL`/`NUMERIC` con `Decimal` y `DATETIME`/`TIMESTAMP` con precisión con `DateTime64`. Anteriormente, las columnas MySQL `DATE` se correspondían con `Date`, que no puede representar fechas anteriores a 1970-01-01, lo que provocaba corrupción de datos. [#97716](https://github.com/ClickHouse/ClickHouse/pull/97716) ([Alexey Milovidov](https://github.com/alexey-milovidov)).

<div id="insert-and-deduplication-changes">
  ### Cambios en las inserciones y la deduplicación
</div>

* La inserción asíncrona ahora está habilitada de forma predeterminada: ClickHouse agrupa por defecto todas las inserciones pequeñas en lotes. Esto se rige por la configuración `compatibility`: establecer `compatibility` en una versión inferior a 26.2 restablece el valor predeterminado anterior de `false`. Las inserciones asíncronas pueden activarse o desactivarse en varios niveles: en los perfiles de usuario de la configuración, para la sesión, para la consulta o para la tabla MergeTree. [#97590](https://github.com/ClickHouse/ClickHouse/pull/97590) ([Sema Checherinda](https://github.com/CheSema)).

<div id="system-table-changes">
  ### Cambios en las tablas del sistema
</div>

* Se revirtió el cambio en la columna `skip_indexes` de `system.query_log`. [#101018](https://github.com/ClickHouse/ClickHouse/pull/101018) ([Alexey Milovidov](https://github.com/alexey-milovidov)).

<div id="other-breaking-changes">
  ### Otros cambios incompatibles
</div>

* `mergeTreeAnalyzeIndexes{,UUID}` ahora acepta un Array de nombres de partes en lugar de una expresión regular, ya que las expresiones regulares son lentas (funcionalidad experimental). [#98474](https://github.com/ClickHouse/ClickHouse/pull/98474) ([Azat Khuzhin](https://github.com/azat)).
* La biblioteca H3 se ha actualizado a la versión 4, lo que mejora la precisión de los cálculos de longitud, área y otras métricas. Este cambio no es retrocompatible porque los nuevos resultados difieren de los anteriores. [#100348](https://github.com/ClickHouse/ClickHouse/pull/100348) ([Alexey Milovidov](https://github.com/alexey-milovidov)).
* Se ha cambiado la forma en que la tabla `Merge` maneja las columnas virtuales. Si la tabla subyacente contiene `_table` o `_database`, estas columnas se leen desde el almacenamiento; de lo contrario, se rellenan después del paso de lectura mediante el paso de expresión. [#101742](https://github.com/ClickHouse/ClickHouse/pull/101742) ([Mikhail Artemenko](https://github.com/Michicosun)).

<div id="new-features">
  ## Nuevas características
</div>

<div id="functions">
  ### Funciones
</div>

* Se añadió la función `naturalSortKey(s)` para claves de ordenación natural. [#90322](https://github.com/ClickHouse/ClickHouse/pull/90322) ([Nazarii Piontko](https://github.com/nazarii-piontko)).
* Se añadió la función `arrayAutocorrelation(arr [, max_lag])`, que calcula la autocorrelación normalizada de un array numérico para cada desfase. Admite arrays de enteros, coma flotante y decimales. [#94776](https://github.com/ClickHouse/ClickHouse/pull/94776) ([Wenyu Chen](https://github.com/wenyuchen96)).
* La función `has()` ahora es compatible con el tipo JSON para comprobar la existencia de rutas, de forma similar a `Map`. [#96927](https://github.com/ClickHouse/ClickHouse/pull/96927) ([DQ](https://github.com/il9ue)).
* Se añadió la función de tabla `mergeTreeTextIndex(database, table, index)`, que lee datos directamente desde un índice de texto. Puede usarse para introspección o para realizar agregaciones sobre datos del índice de texto. [#97003](https://github.com/ClickHouse/ClickHouse/pull/97003) ([Anton Popov](https://github.com/CurtizJ)).
* Se añadió la función SQL `obfuscateQuery`. [#98305](https://github.com/ClickHouse/ClickHouse/pull/98305) ([Xuewei Wang](https://github.com/Sallery-X)).
* Se añadieron las funciones `caseFoldUTF8` y `removeDiacriticsUTF8` para el plegado de mayúsculas y minúsculas en Unicode y la eliminación de marcas diacríticas. [#98973](https://github.com/ClickHouse/ClickHouse/pull/98973) ([George Larionov](https://github.com/george-larionov)).
* La función `printf` ahora admite cadenas de formato no constantes, lo que permite usar un patrón de formato distinto por fila según los valores de las columnas. [#98991](https://github.com/ClickHouse/ClickHouse/pull/98991) ([Yash](https://github.com/Onyx2406)).
* Se añadió la función `highlight()`, que envuelve las apariciones de términos de búsqueda en una cadena de texto con etiquetas HTML (por defecto `<em>`/`</em>`). Admite coincidencias ASCII sin distinción entre mayúsculas y minúsculas, la combinación automática de coincidencias superpuestas y etiquetas de apertura/cierre personalizadas. [#99131](https://github.com/ClickHouse/ClickHouse/pull/99131) ([Peng](https://github.com/fastio)).
* Se añadió la función de cadena `normalizeUTF8NFKCCasefold` para la normalización Unicode NFKC&#95;Casefold, que combina la normalización NFKC con el plegado de mayúsculas y minúsculas. [#99276](https://github.com/ClickHouse/ClickHouse/pull/99276) ([George Larionov](https://github.com/george-larionov)).
* Se añadió el tokenizador `unicode_word` para índices de texto completo y la función `tokens`. Divide el texto usando reglas de límite de palabra de Unicode: las palabras ASCII se forman con caracteres conectores (guion bajo, dos puntos, punto y comilla simple), mientras que los caracteres Unicode no ASCII se convierten en tokens de un solo carácter. Las stop words configurables usan por defecto signos de puntuación CJK comunes. [#99357](https://github.com/ClickHouse/ClickHouse/pull/99357) ([Amos Bird](https://github.com/amosbird)).
* Se añadió la función `arrayTranspose`, que toma una matriz bidimensional (matriz) y la transpone. [#101214](https://github.com/ClickHouse/ClickHouse/pull/101214) ([Vitaly Baranov](https://github.com/vitlibar)).
* Se añadió la función `hasPhrase` (alias `matchPhrase`) para la búsqueda de frases (secuencias continuas de tokens). La búsqueda es de fuerza bruta y aún no es compatible con el índice de texto. [#101997](https://github.com/ClickHouse/ClickHouse/pull/101997) ([Elmi Ahmadov](https://github.com/ahmadov)).
* Se añadió `dotProduct` como función de distancia compatible para índices de similitud vectorial. La dirección de ordenación ahora se valida para todas las funciones de distancia a fin de evitar resultados incorrectos sin avisar. [#102254](https://github.com/ClickHouse/ClickHouse/pull/102254) ([Renzo](https://github.com/RenzoMXD)).
* La función `stem` ahora puede aplicar stemming a todas las palabras/tokens de columnas `String`, `FixedString`, `Array([Fixed]String)`, `Nullable`, `LowCardinality` y `Const`. [#99137](https://github.com/ClickHouse/ClickHouse/pull/99137) ([Jimmy Aguilar Mena](https://github.com/Ergus)).
* La función `stem` ya no es experimental (anteriormente, era necesario habilitar `allow_experimental_nlp_functions`). [#102399](https://github.com/ClickHouse/ClickHouse/pull/102399) ([Jimmy Aguilar Mena](https://github.com/Ergus)).
* Se añadió la función `JSONAllValues`, que devuelve todos los valores de una columna `JSON` como `Array(String)`, con los valores serializados en representación textual y ordenados por sus nombres de path. También se añadió compatibilidad con índices de texto para la expresión `JSONAllValues` en columnas `JSON`: cuando se crea un índice de texto en `JSONAllValues(json_column)`, se usa automáticamente para filtrar consultas sobre subcolumnas de `JSON` (p. ej., `json_column.key1 = 'value'`). [#100730](https://github.com/ClickHouse/ClickHouse/pull/100730) ([Anton Popov](https://github.com/CurtizJ)).
* Se añadieron unidades compatibles con PostgreSQL al operador `EXTRACT`: `EPOCH`, `DOW`, `DOY`, `ISODOW`, `ISOYEAR`, `WEEK`, `CENTURY`, `DECADE` y `MILLENNIUM`. También se corrigió `EXTRACT(WEEK FROM date)`, que anteriormente generaba un error. [#100274](https://github.com/ClickHouse/ClickHouse/pull/100274) ([Alexey Milovidov](https://github.com/alexey-milovidov)).
* Se añadió compatibilidad con la función `toDaysInMonth()` mediante azúcar sintáctico del parser para la sintaxis de la función estándar SQL `OVERLAY`. La función `overlay` ya existe; esto añade compatibilidad con la forma basada en palabras clave que usa `PLACING`, `FROM` y `FOR` como separadores. [#101681](https://github.com/ClickHouse/ClickHouse/pull/101681) ([Desel72](https://github.com/Desel72)).
* Los valores de `Date` y `Date32` ahora pueden sumarse a los valores de `Time` y `Time64` mediante el operador `+`, lo que produce un resultado `DateTime` o `DateTime64`. Por ejemplo, `SELECT toDate('2024-01-15') + toTime('14:30:25')` devuelve `2024-01-15 14:30:25`. El resultado se calcula en la zona horaria de la sesión, y los resultados fuera de rango se gestionan de acuerdo con la configuración `date_time_overflow_behavior`. [#102421](https://github.com/ClickHouse/ClickHouse/pull/102421) ([Nihal Z. Miaji](https://github.com/nihalzp)).

<div id="sql-and-query-features">
  ### Características de SQL y de las consultas
</div>

* Cuando un parámetro tiene tipo `Nullable` y no se especifica, ahora se asume que su valor es `NULL`. [#93869](https://github.com/ClickHouse/ClickHouse/pull/93869) ([Vikash Kumar](https://github.com/vikashkumar2020)).
* ClickHouse ahora puede omitir partes de datos completas en consultas `SELECT` basándose en estadísticas mín./máx. [#94140](https://github.com/ClickHouse/ClickHouse/pull/94140) ([zoomxi](https://github.com/zoomxi)).
* Se añadió compatibilidad con las CTE materializadas, que evalúan las CTE una sola vez durante la ejecución de la consulta y almacenan sus resultados en tablas temporales. [#94849](https://github.com/ClickHouse/ClickHouse/pull/94849) ([Dmitry Novik](https://github.com/novikd)).
* Ahora se pueden usar ciertas funciones sin paréntesis en SQL. [#95949](https://github.com/ClickHouse/ClickHouse/pull/95949) ([Aly Kafoury](https://github.com/AlyHKafoury)).
* Se añadió la sintaxis de subcolumna combinada `json.@path` para el tipo `JSON`. Devuelve el valor literal como `Dynamic` si la ruta contiene un escalar, el subobjeto como `Dynamic` si la ruta contiene un objeto anidado, o `NULL` si la ruta no existe. [#98788](https://github.com/ClickHouse/ClickHouse/pull/98788) ([Pavel Kruglov](https://github.com/Avogar)).
* Se añadió la palabra clave `SOME` para expresiones de subconsulta. Se comporta de forma idéntica a `ANY`. [#99842](https://github.com/ClickHouse/ClickHouse/pull/99842) ([Artem Kytkin](https://github.com/Vinceent)).
* Se añadió compatibilidad con `SET TIME ZONE 'tz'` como alias de `SET session_timezone`. [#99883](https://github.com/ClickHouse/ClickHouse/pull/99883) ([phulv94](https://github.com/phulv94)).
* Se añadió compatibilidad con la cláusula `VALUES` estándar de SQL como expresión de tabla en `FROM`, por ejemplo, `SELECT * FROM (VALUES (1, 'a'), (2, 'b')) AS t(id, val)`. [#100143](https://github.com/ClickHouse/ClickHouse/pull/100143) ([Desel72](https://github.com/Desel72)).
* Se añadió compatibilidad con literales compuestos de intervalos del estándar SQL con calificadores de rango `TO`, por ejemplo, `INTERVAL '1:30' HOUR TO MINUTE`, que se descomponen internamente en sumas de intervalos. [#100453](https://github.com/ClickHouse/ClickHouse/pull/100453) ([Desel72](https://github.com/Desel72)).
* Se añadió el comando `SYSTEM FLUSH OBJECT STORAGE QUEUE db.table PATH 'x'` para los modos ordered y unordered. [#100709](https://github.com/ClickHouse/ClickHouse/pull/100709) ([Bharat Nallan](https://github.com/bharatnc)).

<div id="table-engines-and-storage">
  ### Motores de tabla y almacenamiento
</div>

* Se añadió compatibilidad con la lectura incremental para el motor de tabla `Paimon`, con seguimiento del progreso de instantáneas respaldado por Keeper, incluidas lecturas delta de instantáneas concretos mediante `paimon_target_snapshot_id`. [#93655](https://github.com/ClickHouse/ClickHouse/pull/93655) ([XiaoBinMu](https://github.com/Binnn-MX)).
* Se añadió compatibilidad con ZooKeeper auxiliar en `DatabaseReplicated`. [#95590](https://github.com/ClickHouse/ClickHouse/pull/95590) ([RinChanNOW](https://github.com/RinChanNOWWW)).
* Se añadió compatibilidad para leer tipos timestamp de nanosegundos de Iceberg v3 (`timestamp_ns` y `timestamptz_ns`), asignados a `DateTime64(9)` y `DateTime64(9, 'UTC')`. Esto permite que ClickHouse lea datos de Iceberg con estos tipos en el esquema; no constituye compatibilidad completa con Iceberg v3 ni permite escribir Iceberg v3 con estos tipos. [#97132](https://github.com/ClickHouse/ClickHouse/pull/97132) ([Dmitry Kovalev](https://github.com/dk-github)).
* Se añadió `ALTER TABLE ... EXECUTE expire_snapshots('<timestamp>')` para tablas Iceberg. [#97904](https://github.com/ClickHouse/ClickHouse/pull/97904) ([murphy-4o](https://github.com/murphy-4o)).
* Se añadió una nueva caché SLRU para metadatos de Parquet para mejorar el rendimiento de lectura al eliminar la necesidad de volver a descargar archivos solo para leer sus metadatos. [#98140](https://github.com/ClickHouse/ClickHouse/pull/98140) ([Grant Holly](https://github.com/grantholly-clickhouse)).
* Se añadió compatibilidad con índices de omisión de MergeTree para columnas JSON usando `JSONAllPaths` con los tipos de índice `bloom_filter`, `tokenbf_v1`, `ngrambf_v1` y `text` (invertido), lo que permite omitir gránulos según el conjunto de rutas JSON presentes en cada gránulo. [#98886](https://github.com/ClickHouse/ClickHouse/pull/98886) ([Pavel Kruglov](https://github.com/Avogar)).
* Se añadió el índice de proyección `commit_order`, que reorganiza los datos en orden de inserción. [#99004](https://github.com/ClickHouse/ClickHouse/pull/99004) ([Mikhail Artemenko](https://github.com/Michicosun)).
* Se añadió serialización en buckets para columnas `Map` en MergeTree (`map_serialization_version = 'with_buckets'`). Las claves se dividen en buckets basados en hash, de modo que al leer una sola clave (`m['key']`) solo se lee un bucket en lugar de toda la columna, lo que proporciona una aceleración de entre 2 y 49 veces en búsquedas de una sola clave, según el tamaño del mapa. El número de buckets y la estrategia de bucketización pueden controlarse mediante los nuevos ajustes de MergeTree `map_serialization_version`, `max_buckets_in_map`, `map_buckets_strategy`, `map_buckets_coefficient` y `map_buckets_min_avg_size`. El ajuste `map_serialization_version_for_zero_level_parts` mantiene la serialización básica para las inserciones a fin de evitar la sobrecarga de escritura, mientras que las partes fusionadas usan buckets. Los índices de omisión sobre `mapKeys`/`mapValues` son compatibles con la serialización en buckets, y `optimize_functions_to_subcolumns` reescribe `m['key']` como una lectura de subcolumna por clave cuando hay buckets disponibles. [#99200](https://github.com/ClickHouse/ClickHouse/pull/99200) ([Pavel Kruglov](https://github.com/Avogar)).

<div id="insert-and-deduplication">
  ### Inserción y deduplicación
</div>

* Se implementó un nuevo comportamiento de `max_insert_block_size_rows`, `max_insert_block_size_bytes`, `min_insert_block_size_rows` y `min_insert_block_size_bytes` en el proceso de compactación, controlado por la configuración de compatibilidad `use_strict_insert_block_limits`. [#94207](https://github.com/ClickHouse/ClickHouse/pull/94207) ([Kirill Kopnev](https://github.com/Fgrtue)).

<div id="settings-and-configuration">
  ### Ajustes y configuración
</div>

* Se añadió el ajuste de MergeTree `table_readonly` para marcar tablas como de solo lectura y evitar inserciones y modificaciones. [#97652](https://github.com/ClickHouse/ClickHouse/pull/97652) ([Alexey Milovidov](https://github.com/alexey-milovidov)).
* Se añadió el ajuste `use_partition_pruning` (alias `use_partition_key`). Establécelo en `false` para desactivar la poda de particiones basada en la clave de partición. [#97888](https://github.com/ClickHouse/ClickHouse/pull/97888) ([Nihal Z. Miaji](https://github.com/nihalzp)).
* Cada entrada `type=http` en `<protocols>` ahora puede especificar una clave `<handlers>` personalizada que apunte a una sección de configuración `<http_handlers_*>` independiente, lo que permite aplicar distintas reglas de enrutamiento HTTP por puerto. [#98414](https://github.com/ClickHouse/ClickHouse/pull/98414) ([Amos Bird](https://github.com/amosbird)).
* Se añadieron dos ajustes de MergeTree — `replicated_fetches_min_part_level` y `replicated_fetches_min_part_level_timeout_seconds` — que permiten a las réplicas omitir la recuperación de partes recién insertadas (sin fusionar) desde otras réplicas, reduciendo la sobrecarga de replicación durante periodos de ingestión intensa. [#98625](https://github.com/ClickHouse/ClickHouse/pull/98625) ([tanner-bruce](https://github.com/tanner-bruce)).
* Se añadió el ajuste de servidor `restore_access_entities_with_current_grants`. Cuando está habilitado, los usuarios/roles restaurados ven sus privilegios limitados a lo que el usuario que realiza la restauración puede conceder (con la misma semántica que `GRANT CURRENT GRANTS`), en lugar de fallar con `ACCESS_DENIED`. [#98795](https://github.com/ClickHouse/ClickHouse/pull/98795) ([pufit](https://github.com/pufit)).
* Se añadieron los ajustes `max_skip_unavailable_shards_num` y `max_skip_unavailable_shards_ratio` para limitar cuántos segmentos pueden omitirse sin aviso cuando `skip_unavailable_shards` está habilitado. Si el número o la proporción de segmentos no disponibles supera el umbral configurado, se lanza una excepción en lugar de devolver resultados incompletos sin aviso. [#99369](https://github.com/ClickHouse/ClickHouse/pull/99369) ([Alexey Milovidov](https://github.com/alexey-milovidov)).
* Se implementaron cuotas por hash de consulta normalizada para proteger los servicios públicos de ClickHouse frente a abusos. `NORMALIZED_QUERY_HASH` se admite como tipo de clave de cuota — una cuota independiente por cada consulta normalizada única, de modo que `CREATE QUOTA q KEYED BY normalized_query_hash` controla cada consulta distinta por separado. `QUERIES_PER_NORMALIZED_HASH` se admite como tipo de recurso de cuota — limita el número máximo de ejecuciones de una misma consulta normalizada dentro de un intervalo, por lo que `MAX queries_per_normalized_hash = 100` evita que un mismo patrón de consulta se ejecute más de 100 veces. [#99586](https://github.com/ClickHouse/ClickHouse/pull/99586) ([Alexey Milovidov](https://github.com/alexey-milovidov)).
* El ajuste de MergeTree `auto_statistics_types` ahora usa `minmax, uniq` de forma predeterminada, por lo que las estadísticas minmax y uniq se crean automáticamente para todas las columnas adecuadas de las tablas nuevas. `materialize_statistics_on_insert` ahora usa `false` de forma predeterminada, por lo que las estadísticas se generan durante las fusiones en lugar de en el momento de la inserción, reduciendo la sobrecarga de inserción. Establece `materialize_statistics_on_insert = 1` para restaurar el comportamiento anterior. [#101275](https://github.com/ClickHouse/ClickHouse/pull/101275) ([Han Fei](https://github.com/hanfei1991)).
* Se añadió el ajuste de actualización `prefer_dependency_replica` para las cadenas de dependencias de vistas materializadas, con el fin de reducir la pérdida de datos causada por el retraso de replicación entre réplicas. [#101591](https://github.com/ClickHouse/ClickHouse/pull/101591) ([Seva Potapov](https://github.com/seva-potapov)).

Cuando está habilitado, convertir `NULL` desde tipos que pueden ser `Nullable` devuelve `NULL`; de lo contrario, `NULL` lanza el error `CANNOT_INSERT_NULL_IN_ORDINARY_COLUMN`. Esto puede mejorar la calidad del plan para esquemas en estrella y en copo de nieve. La funcionalidad se controla mediante la nueva configuración `enable_join_transitive_predicates` (desactivada de forma predeterminada). [#98513](https://github.com/ClickHouse/ClickHouse/pull/98513) ([Yarik Briukhovetskyi](https://github.com/yariks5s)).

* Se limitó el número de nodos usados en la heurística automática de réplicas paralelas al número real de nodos del clúster (en lugar de solo la configuración `max_parallel_replicas`). [#98668](https://github.com/ClickHouse/ClickHouse/pull/98668) ([Nikita Taranov](https://github.com/nickitat)).
