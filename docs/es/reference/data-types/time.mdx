El tipo de dato `Time` representa una hora con componentes de horas, minutos y segundos.
Es independiente de cualquier fecha del calendario y es adecuado para valores que no requieren componentes de dГӯa, mes ni aГұo.

Sintaxis:

```sql
Time
```

Rango de representaciГіn de texto: [-999:59:59, 999:59:59].

ResoluciГіn: 1 segundo.

<div id="implementation-details">
  ## Detalles de implementaciГіn
</div>

**RepresentaciГіn y rendimiento**.
El tipo de dato `Time` almacena internamente un entero con signo de 32 bits que codifica los segundos.
Los valores de tipo `Time` y `DateTime` tienen el mismo tamaГұo en bytes y, por lo tanto, un rendimiento comparable.

**NormalizaciГіn**.
Al interpretar cadenas como `Time`, los componentes horarios se normalizan y no se validan.
Por ejemplo, `25:70:70` se interpreta como `26:11:10`.

**Valores negativos**.
Se admiten los signos menos iniciales y se conservan.
Los valores negativos suelen surgir de operaciones aritmГ©ticas sobre valores `Time`.
Para el tipo `Time`, las entradas negativas se conservan tanto en entradas de texto (p. ej., `'-01:02:03'`) como en entradas numГ©ricas (p. ej., `-3723`).

**SaturaciГіn**.
El componente de hora del dГӯa se limita al intervalo [-999:59:59, 999:59:59].
Los valores con horas superiores a 999 (o inferiores a -999) se representan en texto y se vuelven a leer como `999:59:59` (o `-999:59:59`).

**Zonas horarias**.
`Time` no admite zonas horarias; es decir, los valores `Time` se interpretan sin contexto regional.
Especificar una zona horaria para `Time` como parГЎmetro de tipo o durante la creaciГіn del valor genera un error.
Del mismo modo, no se admite aplicar ni cambiar la zona horaria en columnas `Time`, y esto produce un error.
Los valores `Time` no se reinterpretan silenciosamente segГәn distintas zonas horarias.
