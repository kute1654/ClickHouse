Permite conectarse a bases de datos en un servidor [PostgreSQL](https://www.postgresql.org) remoto. Admite operaciones de lectura y escritura (consultas `SELECT` e `INSERT`) para intercambiar datos entre ClickHouse y PostgreSQL.

Proporciona acceso en tiempo real a la lista y a la estructura de las tablas del PostgreSQL remoto mediante las consultas `SHOW TABLES` y `DESCRIBE TABLE`.

Admite modificaciones en la estructura de la tabla (`ALTER TABLE ... ADD|DROP COLUMN`). Si el parГЎmetro `use_table_cache` (consulte ParГЎmetros del motor mГЎs abajo) estГЎ configurado en `1`, la estructura de la tabla se almacena en cachГ© y no se comprueba si se ha modificado, pero puede actualizarse con las consultas `DETACH` y `ATTACH`.

<div id="creating-a-database">
  ## Crear una base de datos
</div>

```sql
CREATE DATABASE test_database
ENGINE = PostgreSQL('host:port', 'database', 'user', 'password'[, `schema`, `use_table_cache`]);
```

**ParГЎmetros del motor**

* `host:port` вҖ” DirecciГіn del servidor PostgreSQL.
* `database` вҖ” Nombre de la base de datos remota.
* `user` вҖ” Usuario de PostgreSQL.
* `password` вҖ” ContraseГұa del usuario.
* `schema` вҖ” Esquema de PostgreSQL.
* `use_table_cache` вҖ”  Indica si la estructura de la tabla de la base de datos se almacena en cachГ©. Opcional. Valor predeterminado: `0`.

  ## Compatibilidad con tipos de datos
</div>

| PostgreSQL       | ClickHouse                                                            |
| ---------------- | --------------------------------------------------------------------- |
| DATE             | [Date](/es/reference/data-types/date)                                    |
| TIMESTAMP        | [DateTime](/es/reference/data-types/datetime)                            |
| REAL             | [Float32](/es/reference/data-types/float)                                |
| DOUBLE           | [Float64](/es/reference/data-types/float)                                |
| DECIMAL, NUMERIC | [Decimal](/es/reference/data-types/decimal) (ver la nota a continuaciГіn) |
| SMALLINT         | [Int16](/es/reference/data-types/int-uint)                               |
| INTEGER          | [Int32](/es/reference/data-types/int-uint)                               |
| BIGINT           | [Int64](/es/reference/data-types/int-uint)                               |
| SERIAL           | [UInt32](/es/reference/data-types/int-uint)                              |
| BIGSERIAL        | [UInt64](/es/reference/data-types/int-uint)                              |
| TEXT, CHAR       | [String](/es/reference/data-types/string)                                |
| INTEGER          | Nullable([Int32](/es/reference/data-types/int-uint))                     |
| ARRAY            | [Array](/es/reference/data-types/array)                                  |

<Note>
  PostgreSQL `numeric(p, 0)` con una precisiГіn `p` superior a 76 (el mГЎximo admitido por `Decimal256`) вҖ”por ejemplo, `numeric(78, 0)`, usado habitualmente para almacenar enteros de 256 bitsвҖ” se asigna a [`Int256`](/es/reference/data-types/int-uint) en lugar de `Decimal`. Los valores que no caben en el rango de `Int256` se rechazan con un error.
</Note>
