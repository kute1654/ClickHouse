Este motor ofrece integraciГіn con el ecosistema de [Azure Blob Storage](https://azure.microsoft.com/en-us/products/storage/blobs).

<div id="create-table">
  ## CREATE TABLE
</div>

```sql
CREATE TABLE azure_blob_storage_table (name String, value UInt32)
    ENGINE = AzureBlobStorage(connection_string|storage_account_url, container_name, blobpath, [account_name, account_key, format, compression, partition_strategy, partition_columns_in_data_file, extra_credentials(client_id=, tenant_id=)])
    [PARTITION BY expr]
    [SETTINGS ...]
```

<div id="engine-parameters">
  ### ParГЎmetros del motor
</div>

* `endpoint` вҖ” URL del endpoint de AzureBlobStorage con contenedor y prefijo. Opcionalmente, puede incluir account&#95;name si el mГ©todo de autenticaciГіn utilizado lo requiere. (`http://azurite1:{port}/[account_name]{container_name}/{data_prefix}`) o estos parГЎmetros pueden proporcionarse por separado mediante storage&#95;account&#95;url, account&#95;name y container. Para especificar el prefijo, se debe usar endpoint.
* `endpoint_contains_account_name` - Esta marca se usa para indicar si endpoint incluye account&#95;name, ya que solo es necesario para ciertos mГ©todos de autenticaciГіn. (Valor predeterminado: true)
* `connection_string|storage_account_url` вҖ” connection&#95;string incluye el nombre y la clave de la cuenta ([Crear una connection string](https://learn.microsoft.com/en-us/azure/storage/common/storage-configure-connection-string?toc=%2Fazure%2Fstorage%2Fblobs%2Ftoc.json\&bc=%2Fazure%2Fstorage%2Fblobs%2Fbreadcrumb%2Ftoc.json#configure-a-connection-string-for-an-azure-storage-account)) o tambiГ©n puede proporcionar aquГӯ la URL de la cuenta de almacenamiento, asГӯ como el nombre y la clave de la cuenta como parГЎmetros independientes (consulte los parГЎmetros account&#95;name y account&#95;key)
* `container_name` - Nombre del contenedor
* `blobpath` - ruta del archivo. Admite los siguientes comodines en modo readonly: `*`, `**`, `?`, `{abc,def}` y `{N..M}`, donde `N`, `M` вҖ” nГәmeros, `'abc'`, `'def'` вҖ” cadenas.
* `account_name` - si se usa storage&#95;account&#95;url, aquГӯ puede especificarse el nombre de la cuenta
* `account_key` - si se usa storage&#95;account&#95;url, aquГӯ puede especificarse la clave de la cuenta
* `format` вҖ” El [formato](/es/reference/formats/index) del archivo.
* `compression` вҖ” Valores admitidos: `none`, `gzip/gz`, `brotli/br`, `xz/LZMA`, `zstd/zst`. De forma predeterminada, detectarГЎ automГЎticamente la compresiГіn segГәn la extensiГіn del archivo. (equivale a establecer `auto`).
* `partition_strategy` вҖ“ Opciones: `wildcard` o `hive`. `wildcard` requiere `{_partition_id}` en la ruta, que se sustituye por la clave de particiГіn. `hive` no permite comodines, asume que la ruta es la raГӯz de la tabla y genera directorios particionados con estilo Hive, con ID de Snowflake como nombres de archivo y el formato de archivo como extensiГіn. El valor predeterminado es la configuraciГіn `file_like_engine_default_partition_strategy` (`wildcard` en configuraciones de `compatibility` anteriores a `26.6`, `hive` en caso contrario).
* `partition_columns_in_data_file` - Solo se usa con la estrategia de particiГіn `hive`. Indica a ClickHouse si debe esperar que las columnas de particiГіn se escriban en el archivo de datos. El valor predeterminado es `false`.
* `extra_credentials` - Use `client_id` y `tenant_id` para la autenticaciГіn. Si se proporcionan extra&#95;credentials, tienen prioridad sobre `account_name` y `account_key`.

**Ejemplo**

Los usuarios pueden usar el emulador Azurite para el desarrollo local de Azure Storage. MГЎs detalles [aquГӯ](https://learn.microsoft.com/en-us/azure/storage/common/storage-use-azurite?tabs=docker-hub%2Cblob-storage). Si usa una instancia local de Azurite, puede que deba sustituir `http://localhost:10000` por `http://azurite1:10000` en los comandos siguientes, donde asumimos que Azurite estГЎ disponible en el host `azurite1`.

<div id="virtual-columns">
  ## Columnas virtuales
</div>

* `_path` вҖ” Ruta del archivo. Tipo: `LowCardinality(String)`.
* `_file` вҖ” Nombre del archivo. Tipo: `LowCardinality(String)`.
* `_size` вҖ” TamaГұo del archivo en bytes. Tipo: `Nullable(UInt64)`. Si se desconoce el tamaГұo, el valor es `NULL`.
* `_time` вҖ” Hora de la Гәltima modificaciГіn del archivo. Tipo: `Nullable(DateTime)`. Si se desconoce la hora, el valor es `NULL`.

<div id="authentication">
  ## AutenticaciГіn
</div>

Actualmente, hay 3 formas de autenticarse:

* `Managed Identity` - Se puede usar proporcionando un `endpoint`, `connection_string` o `storage_account_url`.
* `SAS Token` - Se puede usar proporcionando un `endpoint`, `connection_string` o `storage_account_url`. Se identifica por la presencia de `?` en la URL. Consulta [azureBlobStorage](/es/reference/functions/table-functions/azureBlobStorage#using-shared-access-signatures-sas-sas-tokens) para ver ejemplos.
* `Workload Identity` - Se puede usar proporcionando un `endpoint` o `storage_account_url`. Si el parГЎmetro `use_workload_identity` estГЎ establecido en la configuraciГіn, se usa [workload identity](https://github.com/Azure/azure-sdk-for-cpp/tree/main/sdk/identity/azure-identity#authenticate-azure-hosted-applications) para la autenticaciГіn.

<div id="data-cache">
  ### CachГ© de datos
</div>

El motor de tabla `Azure` admite el almacenamiento en cachГ© de datos en disco local.
Consulte las opciones de configuraciГіn y el uso de la cachГ© del sistema de archivos en esta [secciГіn](/es/concepts/features/configuration/server-config/storing-data#using-local-cache).
La cachГ© se basa en la ruta y el ETag del objeto de almacenamiento, por lo que ClickHouse no leerГЎ una versiГіn obsoleta de la cachГ©.

Para habilitar la cachГ©, use la configuraciГіn `filesystem_cache_name = '<name>'` y `enable_filesystem_cache = 1`.

```sql
SELECT *
FROM azureBlobStorage('DefaultEndpointsProtocol=http;AccountName=devstoreaccount1;AccountKey=Eby8vdM02xNOcqFlqUwJPLlmEtlCDXJ1OUzFT50uSRZ6IFsuFq2UVErCz4I6tq/K1SZFPTOtr/KBHBeksoGMGw==;BlobEndpoint=http://azurite1:10000/devstoreaccount1/;', 'testcontainer', 'test_table', 'CSV')
SETTINGS filesystem_cache_name = 'cache_for_azure', enable_filesystem_cache = 1;
```

1. aГұade la siguiente secciГіn al archivo de configuraciГіn de ClickHouse:

```xml
<clickhouse>
    <filesystem_caches>
        <cache_for_azure>
            <path>path to cache directory</path>
            <max_size>10Gi</max_size>
        </cache_for_azure>
    </filesystem_caches>
</clickhouse>
```

2. reutilizar la configuraciГіn de la cachГ© (y, por lo tanto, el almacenamiento en cachГ©) de la secciГіn `storage_configuration` de ClickHouse, [descrita aquГӯ](/es/concepts/features/configuration/server-config/storing-data#using-local-cache)

<div id="partition-by">
  ### PARTITION BY
</div>

`PARTITION BY` вҖ” Opcional. En la mayorГӯa de los casos no necesita una clave de particiГіn y, si la necesita, por lo general no hace falta que sea mГЎs granular que por mes. El particionado no acelera las consultas (a diferencia de la expresiГіn `ORDER BY`). Nunca debe usar un particionado demasiado granular. No particione sus datos por identificadores o nombres de clientes (en su lugar, haga que el identificador o nombre del cliente sea la primera columna de la expresiГіn `ORDER BY`).

Para particionar por mes, use la expresiГіn `toYYYYMM(date_column)`, donde `date_column` es una columna de tipo [Date](/es/reference/data-types/date). Los nombres de las particiones aquГӯ tienen el formato `"YYYYMM"`.

<div id="partition-strategy">
  #### Estrategia de particiГіn
</div>

`wildcard`: sustituye el comodГӯn `{_partition_id}` en la ruta del archivo por la clave de particiГіn real. No admite lectura. Se selecciona por defecto solo con configuraciones de `compatibility` anteriores a `26.6`; de lo contrario, el valor predeterminado es `hive` (consulte la configuraciГіn `file_like_engine_default_partition_strategy`).

`hive` implementa el particionado de estilo Hive para lectura y escritura. La lectura se implementa mediante un patrГіn glob recursivo. La escritura genera archivos con el siguiente formato: `<prefix>/<key1=val1/key2=val2...>/<snowflakeid>.<toLower(file_format)>`.

Nota: al usar la estrategia de particiГіn `hive`, la configuraciГіn `use_hive_partitioning` no tiene efecto.
