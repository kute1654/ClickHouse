El motor de tabla File guarda los datos en un archivo en uno de los [formatos de archivo](/es/reference/formats/index#formats-overview) compatibles (`TabSeparated`, `Native`, etc.).

Casos de uso:

* ExportaciГіn de datos de ClickHouse a un archivo.
* ConversiГіn de datos de un formato a otro.
* ActualizaciГіn de datos en ClickHouse mediante la ediciГіn de un archivo en un disco.

<Note>
  Este motor no estГЎ disponible actualmente en ClickHouse Cloud; [utilice en su lugar la funciГіn de tabla S3](/es/reference/functions/table-functions/s3).
</Note>

<div id="usage-in-clickhouse-server">
  ## Uso en ClickHouse Server
</div>

```sql

```

El parГЎmetro `Format` especifica uno de los formatos de archivo disponibles. Para ejecutar
consultas `SELECT`, el formato debe ser compatible con la entrada y, para ejecutar
consultas `INSERT`, con la salida. Los formatos disponibles se enumeran en la
secciГіn [Formatos](/es/reference/formats/index#formats-overview).

ClickHouse no permite especificar una ruta del sistema de archivos para `File`. UsarГЎ la carpeta definida por la opciГіn [path](/es/reference/settings/server-settings/settings) en la configuraciГіn del servidor.

Al crear una tabla con `File(Format)`, se crea un subdirectorio vacГӯo en esa carpeta. Cuando se escriben datos en esa tabla, se guardan en el archivo `data.Format` de ese subdirectorio.

Puede crear manualmente este subdirectorio y archivo en el sistema de archivos del servidor y luego [ATTACH](/es/reference/statements/attach) a la definiciГіn de la tabla con el nombre correspondiente, para poder consultar los datos de ese archivo.

<Note>
  Tenga cuidado con esta funcionalidad, porque ClickHouse no realiza un seguimiento de los cambios externos en este tipo de archivos. El resultado de escrituras simultГЎneas a travГ©s de ClickHouse y fuera de ClickHouse no estГЎ definido.
</Note>

<div id="usage-in-clickhouse-local">
  ## Uso en ClickHouse-local
</div>

En [clickhouse-local](/es/concepts/features/tools-and-utilities/clickhouse-local), el motor File acepta una ruta de archivo ademГЎs de `Format`. Los flujos predeterminados de entrada/salida pueden especificarse mediante nombres numГ©ricos o legibles para humanos, como `0` o `stdin`, `1` o `stdout`. TambiГ©n es posible leer y escribir archivos comprimidos segГәn un parГЎmetro adicional del motor o la extensiГіn del archivo (`gz`, `br` o `xz`).

**Ejemplo:**

```bash
$ echo -e "1,2\n3,4" | clickhouse-local -q "CREATE TABLE table (a Int64, b Int64) ENGINE = File(CSV, stdin); SELECT a, b FROM table; DROP TABLE table"
```

<div id="details-of-implementation">
  ## Detalles de la implementaciГіn
</div>

* Se pueden ejecutar varias consultas `SELECT` de forma concurrente, pero las consultas `INSERT` deben esperar unas a otras.
* Se admite la creaciГіn de un archivo nuevo mediante una consulta `INSERT`.
* Si el archivo existe, `INSERT` le aГұadirГЎ nuevos valores.
* No se admite:
  * `ALTER`
  * `SELECT ... SAMPLE`
  * ГҚndices
  * ReplicaciГіn

<div id="partition-by">
  ## PARTITION BY
</div>

`PARTITION BY` вҖ” Opcional. Es posible crear archivos independientes particionando los datos segГәn una clave de particiГіn. En la mayorГӯa de los casos, no necesita una clave de particiГіn y, si la necesita, por lo general no deberГӯa ser mГЎs granular que por mes. El particionado no acelera las consultas (a diferencia de la expresiГіn `ORDER BY`). Nunca debe usar un particionado demasiado granular. No particione sus datos por identificadores ni nombres de clientes (en su lugar, haga que el identificador o el nombre del cliente sea la primera columna de la expresiГіn `ORDER BY`).

Para particionar por mes, use la expresiГіn `toYYYYMM(date_column)`, donde `date_column` es una columna de tipo [Date](/es/reference/data-types/date). Los nombres de las particiones aquГӯ tienen el formato `"YYYYMM"`.

<div id="virtual-columns">
  ## Columnas virtuales
</div>

* `_path` вҖ” Ruta del archivo. Tipo: `LowCardinality(String)`.
* `_file` вҖ” Nombre del archivo. Tipo: `LowCardinality(String)`.
* `_size` вҖ” TamaГұo del archivo en bytes. Tipo: `Nullable(UInt64)`. Si se desconoce el tamaГұo, el valor es `NULL`.
* `_time` вҖ” Fecha y hora de la Гәltima modificaciГіn del archivo. Tipo: `Nullable(DateTime)`. Si se desconoce la hora, el valor es `NULL`.
