<div id="description">
  ## Descripción
</div>

Recopila en una sola tabla la documentación integrada de los componentes uniformes del sistema. Cada fila corresponde a una entidad (una función, un motor de tabla, un tipo de dato, etc.) y contiene la documentación de referencia de esa entidad en formato Markdown: el mismo contenido que se publica en el sitio web y que exponen las tablas `system.*` específicas de cada tipo.

La `description` se compone a partir de las partes estructuradas de la documentación integrada (`description`, `syntax`, argumentos, ejemplos, etc.), de modo que una sola columna contiene la documentación completa de una entidad. Los alias se muestran como una referencia breve a la entidad canónica; por ejemplo, ``Alias of `trunc`.``

Esta tabla, en cierto modo, recopila la información disponible en las tablas de documentación específicas de cada tipo ([`system.functions`](/es/reference/system-tables/functions), [`system.table_engines`](/es/reference/system-tables/table_engines), [`system.data_type_families`](/es/reference/system-tables/data_type_families) y otras). Está pensada, en particular, como base para un comando `help` interactivo en el client, pero también es útil por sí sola.

Se recopilan los siguientes tipos de entidades (el valor de la columna `type` se muestra entre paréntesis):

* Funciones (`Function`)
* Funciones de agregación (`Aggregate Function`)
* Funciones de tabla (`Table Function`)
* Motores de tabla (`Table Engine`)
* Motores de base de datos (`Database Engine`)
* Tipos de datos (`Data Type`)
* Diseños de diccionario (`Dictionary Layout`)
* Fuentes de diccionario (`Dictionary Source`)
* Combinadores de funciones de agregación (`Aggregate Function Combinator`)
* Tipos de índices de omisión de datos (`Data Skipping Index`)
* Tipos de disco (`Disk Type`)
* Ajustes (`Setting`)
* Ajustes de MergeTree (`MergeTree Setting`)
* Ajustes del servidor (`Server Setting`)
* Formatos (`Format`)

En el caso de los ajustes, la documentación es la descripción del ajuste; los ajustes obsoletos no se exponen.

<div id="columns">
  ## Columnas
</div>

{/*AUTOGENERATED_START*/}

* `name` ([String](/es/reference/data-types/index)) — El nombre de la entidad, p. ej. `domainWithoutWWW` o `MergeTree`.
* `type` ([Enum8(&#39;Function&#39; = 1, &#39;Aggregate Function&#39; = 2, &#39;Table Function&#39; = 3, &#39;Table Engine&#39; = 4, &#39;Database Engine&#39; = 5, &#39;Data Type&#39; = 6, &#39;Dictionary Layout&#39; = 7, &#39;Dictionary Source&#39; = 8, &#39;Aggregate Function Combinator&#39; = 9, &#39;Data Skipping Index&#39; = 10, &#39;Disk Type&#39; = 11, &#39;Setting&#39; = 12, &#39;MergeTree Setting&#39; = 13, &#39;Server Setting&#39; = 14, &#39;Format&#39; = 15)](/es/reference/data-types/index)) — El tipo de la entidad, p. ej. `Function` o `Table Engine`.
* `description` ([String](/es/reference/data-types/index)) — La documentación de referencia de la entidad en formato Markdown, generada a partir de la documentación integrada (el mismo contenido que se publica en el sitio web), incluida la sintaxis, los ejemplos y otras partes estructuradas, si las hay.

<div id="example">
  ## Ejemplo
</div>

Consulta la documentación de una entidad concreta:

```sql title="Query"
SELECT description
FROM system.documentation
WHERE type = 'Table Engine' AND name = 'MergeTree'
FORMAT TSVRaw;
```

El mismo nombre puede referirse a varios tipos de entidades (por ejemplo, existe tanto la función de tabla `file` como el origen del diccionario `file`), por lo que conviene buscar ese nombre en todos los tipos:

```sql title="Query"
SELECT type, name
FROM system.documentation
WHERE name = 'file'
ORDER BY type;
```

Cuenta las entidades documentadas de cada tipo:

```sql title="Query"
SELECT type, count()
FROM system.documentation
GROUP BY type
ORDER BY count() DESC;
```
