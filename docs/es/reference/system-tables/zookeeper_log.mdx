<div id="description">
  ## Descripción
</div>

Esta tabla contiene información sobre los parámetros de la solicitud al servidor ZooKeeper y sobre la respuesta que devuelve.

En las solicitudes, solo se rellenan las columnas con los parámetros de la solicitud, y las demás columnas se rellenan con valores predeterminados (`0` o `NULL`). Cuando llega la respuesta, los datos de esta se añaden a las demás columnas.

<div id="columns">
  ## Columnas
</div>

{/*AUTOGENERATED_START*/}

* `hostname` ([LowCardinality(String)](/es/reference/data-types/lowcardinality)) — Nombre de host del servidor que ejecuta la consulta.
* `type` ([Enum8(&#39;Request&#39; = 1, &#39;Response&#39; = 2, &#39;Finalize&#39; = 3)](/es/reference/data-types/enum)) — Tipo de evento en el cliente de ZooKeeper. Puede tener uno de los siguientes valores: Request — La solicitud se ha enviado, Response — Se ha recibido la respuesta, Finalize — Se ha perdido la conexión y no se ha recibido ninguna respuesta.
* `event_date` ([Date](/es/reference/data-types/date)) — La fecha en que ocurrió el evento.
* `event_time` ([DateTime64(6)](/es/reference/data-types/datetime64)) — La fecha y hora en la que ocurrió el evento.
* `thread_id` ([UInt64](/es/reference/data-types/int-uint)) — El ID del hilo que ejecutó esta petición.
* `query_id` ([String](/es/reference/data-types/string)) — El identificador de la consulta en cuyo ámbito se ejecutó esta solicitud.
* `address` ([IPv6](/es/reference/data-types/ipv6)) — Dirección IP del servidor de ZooKeeper utilizado para realizar la petición.
* `port` ([UInt16](/es/reference/data-types/int-uint)) — El puerto del servidor de ZooKeeper utilizado para realizar la solicitud.
* `session_id` ([Int64](/es/reference/data-types/int-uint)) — El ID de sesión que establece el servidor de ZooKeeper para cada conexión.
* `duration_microseconds` ([UInt64](/es/reference/data-types/int-uint)) — El tiempo que ZooKeeper tarda en ejecutar la solicitud.
* `xid` ([Int64](/es/reference/data-types/int-uint)) — El ID de la solicitud dentro de la sesión. Suele ser un número de solicitud secuencial. Es el mismo tanto para la fila de solicitud como para la fila correspondiente de respuesta/finalize.
* `has_watch` ([UInt8](/es/reference/data-types/int-uint)) — Indica si se ha establecido el watch.
* `op_num` ([Enum16(&#39;Close&#39; = -11, &#39;Error&#39; = -1, &#39;Watch&#39; = 0, &#39;Create&#39; = 1, &#39;Remove&#39; = 2, &#39;Exists&#39; = 3, &#39;Get&#39; = 4, &#39;Set&#39; = 5, &#39;GetACL&#39; = 6, &#39;SetACL&#39; = 7, &#39;SimpleList&#39; = 8, &#39;Sync&#39; = 9, &#39;Heartbeat&#39; = 11, &#39;List&#39; = 12, &#39;Check&#39; = 13, &#39;Multi&#39; = 14, &#39;Create2&#39; = 15, &#39;Reconfig&#39; = 16, &#39;CheckWatch&#39; = 17, &#39;RemoveWatch&#39; = 18, &#39;MultiRead&#39; = 22, &#39;Auth&#39; = 100, &#39;SetWatch&#39; = 101, &#39;SetWatch2&#39; = 105, &#39;AddWatch&#39; = 106, &#39;FilteredList&#39; = 500, &#39;CheckNotExists&#39; = 501, &#39;CreateIfNotExists&#39; = 502, &#39;RemoveRecursive&#39; = 503, &#39;CheckStat&#39; = 504, &#39;TryRemove&#39; = 505, &#39;FilteredListWithStatsAndData&#39; = 506, &#39;ListRecursive&#39; = 507, &#39;SessionID&#39; = 997)](/es/reference/data-types/enum)) — Tipo de solicitud o respuesta.
* `path` ([String](/es/reference/data-types/string)) — La ruta del nodo de ZooKeeper especificado en la solicitud, o una cadena vacía si la solicitud no requiere especificar una ruta.
* `data` ([String](/es/reference/data-types/string)) — Los datos escritos en el nodo de ZooKeeper (para las solicitudes SET y CREATE, lo que la solicitud quería escribir; para la respuesta a la solicitud GET, lo que se leyó) o una cadena vacía.
* `is_ephemeral` ([UInt8](/es/reference/data-types/int-uint)) — Indica si el nodo de ZooKeeper se está creando como efímero.
* `is_sequential` ([UInt8](/es/reference/data-types/int-uint)) — Indica si el nodo de ZooKeeper se crea como sequential.
* `version` ([Nullable(Int32)](/es/reference/data-types/nullable)) — La versión del nodo de ZooKeeper que la solicitud espera al ejecutarse. Esto es compatible con las solicitudes CHECK, SET y REMOVE (corresponde a -1 si la solicitud no comprueba la versión, o a NULL en otras solicitudes que no admiten la comprobación de versión).
* `requests_size` ([UInt32](/es/reference/data-types/int-uint)) — El número de solicitudes incluidas en la solicitud múltiple (se trata de una solicitud especial que consta de varias solicitudes ordinarias consecutivas y las ejecuta de forma atómica). Todas las solicitudes incluidas en la solicitud múltiple tendrán el mismo xid.
* `request_idx` ([UInt32](/es/reference/data-types/int-uint)) — El número de la solicitud incluida en una solicitud múltiple (para una solicitud múltiple: 0 y luego, en orden, a partir de 1).
* `zxid` ([Int64](/es/reference/data-types/int-uint)) — ID de transacción de ZooKeeper. El número de serie emitido por el servidor ZooKeeper en respuesta a una solicitud ejecutada correctamente (0 si la solicitud no se ejecutó/devolvió un error/el cliente no sabe si la solicitud se ejecutó).
* `error` ([Nullable(Enum8(&#39;ZNOWATCHER&#39; = -121, &#39;ZNOTREADONLY&#39; = -119, &#39;ZSESSIONMOVED&#39; = -118, &#39;ZNOTHING&#39; = -117, &#39;ZCLOSING&#39; = -116, &#39;ZAUTHFAILED&#39; = -115, &#39;ZINVALIDACL&#39; = -114, &#39;ZINVALIDCALLBACK&#39; = -113, &#39;ZSESSIONEXPIRED&#39; = -112, &#39;ZNOTEMPTY&#39; = -111, &#39;ZNODEEXISTS&#39; = -110, &#39;ZNOCHILDRENFOREPHEMERALS&#39; = -108, &#39;ZBADVERSION&#39; = -103, &#39;ZNOAUTH&#39; = -102, &#39;ZNONODE&#39; = -101, &#39;ZAPIERROR&#39; = -100, &#39;ZOUTOFMEMORY&#39; = -10, &#39;ZINVALIDSTATE&#39; = -9, &#39;ZBADARGUMENTS&#39; = -8, &#39;ZOPERATIONTIMEOUT&#39; = -7, &#39;ZUNIMPLEMENTED&#39; = -6, &#39;ZMARSHALLINGERROR&#39; = -5, &#39;ZCONNECTIONLOSS&#39; = -4, &#39;ZDATAINCONSISTENCY&#39; = -3, &#39;ZRUNTIMEINCONSISTENCY&#39; = -2, &#39;ZSYSTEMERROR&#39; = -1, &#39;ZOK&#39; = 0))](/es/reference/data-types/nullable)) — Código de error. Puede tener muchos valores; estos son solo algunos: ZOK — La solicitud se ejecutó correctamente, ZCONNECTIONLOSS — Se perdió la conexión, ZOPERATIONTIMEOUT — Ha expirado el tiempo de espera de ejecución de la solicitud, ZSESSIONEXPIRED — La sesión ha expirado, NULL — La solicitud se ha completado.
* `watch_type` ([Nullable(Enum8(&#39;NOTWATCHING&#39; = -2, &#39;SESSION&#39; = -1, &#39;CREATED&#39; = 1, &#39;DELETED&#39; = 2, &#39;CHANGED&#39; = 3, &#39;CHILD&#39; = 4))](/es/reference/data-types/nullable)) — El tipo de evento watch (para las respuestas con op&#95;num = Watch); para el resto de las respuestas: NULL.
* `watch_state` ([Nullable(Enum16(&#39;AUTH&#95;FAILED&#39; = -113, &#39;EXPIRED&#95;SESSION&#39; = -112, &#39;CONNECTING&#39; = 1, &#39;ASSOCIATING&#39; = 2, &#39;CONNECTED&#39; = 3, &#39;READONLY&#39; = 5, &#39;NOTCONNECTED&#39; = 999))](/es/reference/data-types/nullable)) — El estado del evento de watch (para las respuestas con op&#95;num = Watch), para el resto de las respuestas: NULL.
* `path_created` ([String](/es/reference/data-types/string)) — La ruta del nodo de ZooKeeper creado (para las respuestas a la solicitud CREATE); puede diferir de la ruta si el nodo se crea en modo secuencial.
* `stat_czxid` ([Int64](/es/reference/data-types/int-uint)) — El zxid del cambio que provocó la creación de este nodo de ZooKeeper.
* `stat_mzxid` ([Int64](/es/reference/data-types/int-uint)) — El zxid del último cambio que modificó este nodo de ZooKeeper.
* `stat_pzxid` ([Int64](/es/reference/data-types/int-uint)) — El ID de la transacción del cambio que modificó por última vez los nodos secundarios de este nodo de ZooKeeper.
* `stat_version` ([Int32](/es/reference/data-types/int-uint)) — El número de cambios en los datos de este nodo de ZooKeeper.
* `stat_cversion` ([Int32](/es/reference/data-types/int-uint)) — El número de cambios en los nodos hijo de este nodo de ZooKeeper.
* `stat_dataLength` ([Int32](/es/reference/data-types/int-uint)) — La longitud del campo de datos de este nodo de ZooKeeper.
* `stat_numChildren` ([Int32](/es/reference/data-types/int-uint)) — El número de hijos de este nodo de ZooKeeper.
* `children` ([Array(String)](/es/reference/data-types/array)) — La lista de nodos hijos de ZooKeeper (en las respuestas a la solicitud LIST).

{/*AUTOGENERATED_END*/}
