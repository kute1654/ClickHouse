<div id="description">
  ## Descripción
</div>

Contiene información sobre la configuración global del servidor, especificada en `config.xml`.
Actualmente, la tabla solo muestra la configuración del primer nivel de `config.xml` y no admite configuraciones anidadas (p. ej., [logger](/es/reference/settings/server-settings/settings#logger)).

<div id="columns">
  ## Columnas
</div>

{/*AUTOGENERATED_START*/}

* `name` ([String](/es/reference/data-types/index)) — Nombre del ajuste del servidor.
* `value` ([String](/es/reference/data-types/index)) — Valor del ajuste del servidor.
* `default` ([String](/es/reference/data-types/index)) — Valor predeterminado del ajuste del servidor.
* `changed` ([UInt8](/es/reference/data-types/index)) — Indica si un ajuste se especificó en config.xml
* `description` ([String](/es/reference/data-types/index)) — Descripción breve del ajuste del servidor.
* `type` ([String](/es/reference/data-types/index)) — Tipo de valor del ajuste del servidor.
* `changeable_without_restart` ([Enum8(&#39;No&#39; = 0, &#39;IncreaseOnly&#39; = 1, &#39;DecreaseOnly&#39; = 2, &#39;Yes&#39; = 3)](/es/reference/data-types/index)) — Indica si un ajuste puede cambiarse en tiempo de ejecución.
* `is_obsolete` ([UInt8](/es/reference/data-types/index)) — Indica si un ajuste es obsoleto.

{/*AUTOGENERATED_END*/}

<div id="example">
  ## Ejemplo
</div>

El siguiente ejemplo muestra cómo obtener información sobre los ajustes del servidor cuyos nombres contienen `thread_pool`.

```sql
SELECT *
FROM system.server_settings
WHERE name LIKE '%thread_pool%'
```

El uso de `WHERE changed` puede ser útil, por ejemplo, cuando quiera comprobar
si los ajustes de los archivos de configuración se han cargado correctamente y se están usando.

{/* */}

```sql
SELECT * FROM system.server_settings WHERE changed AND name='max_thread_pool_size'
```
