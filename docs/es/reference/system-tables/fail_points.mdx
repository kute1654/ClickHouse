<div id="description">
  ## Descripción
</div>

Contiene una lista de todos los failpoints disponibles registrados en el servidor, junto con su tipo y si están habilitados actualmente.

Los failpoints se pueden habilitar y deshabilitar en tiempo de ejecución mediante las sentencias `SYSTEM ENABLE FAILPOINT` y `SYSTEM DISABLE FAILPOINT`.

<div id="columns">
  ## Columnas
</div>

{/*AUTOGENERATED_START*/}

* `name` ([String](/es/reference/data-types/index)) — Nombre del failpoint.
* `type` ([Enum8(&#39;once&#39; = 0, &#39;regular&#39; = 1, &#39;pauseable&#95;once&#39; = 2, &#39;pauseable&#39; = 3)](/es/reference/data-types/index)) — Tipo de failpoint: &#39;once&#39; se activa una sola vez y luego se deshabilita automáticamente; &#39;regular&#39; se activa cada vez; &#39;pauseable&#95;once&#39; bloquea la ejecución una vez; &#39;pauseable&#39; bloquea la ejecución cada vez hasta que se reanuda.
* `enabled` ([UInt8](/es/reference/data-types/index)) — Indica si el failpoint está habilitado (1) o deshabilitado (0).

{/*AUTOGENERATED_END*/}

<div id="example">
  ## Ejemplo
</div>

```sql title="Query"
SYSTEM ENABLE FAILPOINT replicated_merge_tree_insert_retry_pause;
SELECT * FROM system.fail_points WHERE enabled = 1
```
