<div id="description">
  ## Descripción
</div>

Esta tabla no existe si este nodo no está configurado para ejecutar ClickHouse Keeper en el mismo proceso. Contiene una fila por cada archivo de instantánea de Raft gestionado por la máquina de estados de Keeper que se ejecuta en el mismo proceso, incluidas las instantáneas que se están recibiendo actualmente del líder.

<div id="columns">
  ## Columnas
</div>

{/*AUTOGENERATED_START*/}

* `last_log_index` ([UInt64](/es/reference/data-types/int-uint)) — Último índice de log cubierto por la instantánea.
* `path` ([String](/es/reference/data-types/string)) — Ruta del archivo de instantánea en el disco.
* `disk_name` ([String](/es/reference/data-types/string)) — Nombre del disco que almacena la instantánea.
* `size_bytes` ([UInt64](/es/reference/data-types/int-uint)) — Tamaño del archivo de instantánea en disco.
* `last_modified_at` ([DateTime](/es/reference/data-types/datetime)) — Fecha y hora de la última modificación del archivo de instantánea.
* `is_received` ([Bool](/es/reference/data-types/boolean)) — `true` si la instantánea se está recibiendo actualmente desde el líder. Para esas filas, `size_bytes` y `last_modified_at` reflejan el archivo parcial tal como se ha escrito hasta el momento y pueden indicar un valor inferior al real.
* `exists_on_disk` ([Bool](/es/reference/data-types/boolean)) — Indica si el archivo de instantánea está presente actualmente en disco. Siempre es `true` para las instantáneas finalizadas (`is_received` = `false`), a menos que el archivo se haya eliminado fuera de banda o esté corrupto; puede ser `false` para las instantáneas que se están recibiendo (`is_received` = `true`) antes de que se haya escrito ningún byte.

{/*AUTOGENERATED_END*/}

Ejemplo:

```sql
SELECT * FROM system.keeper_snapshots ORDER BY last_log_index;
```
