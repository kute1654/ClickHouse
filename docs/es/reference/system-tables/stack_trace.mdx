<div id="description">
  ## Descripción
</div>

Contiene las trazas de pila de todos los hilos del servidor. Permite a los desarrolladores inspeccionar el estado del servidor.

Para analizar los marcos de pila, use `addressToLine`, `addressToLineWithInlines`, `addressToSymbol` y `demangle` [funciones de introspección](/es/reference/functions/regular-functions/introspection).

<div id="columns">
  ## Columnas
</div>

{/*AUTOGENERATED_START*/}

* `thread_name` ([String](/es/reference/data-types/index)) — El nombre del hilo.
* `thread_id` ([UInt64](/es/reference/data-types/index)) — El identificador del hilo
* `query_id` ([String](/es/reference/data-types/index)) — El ID de la consulta a la que pertenece este hilo.
* `trace` ([Array(UInt64)](/es/reference/data-types/index)) — La traza de pila de este hilo. Básicamente, es solo un array de direcciones.
* `untracked_memory` ([Int64](/es/reference/data-types/index)) — Contador por hilo, no atómico, de asignaciones de memoria que aún no se han propagado al MemoryTracker padre. Puede ser negativo si se ha liberado más memoria de la que se ha asignado desde el último flush.

{/*AUTOGENERATED_END*/}

<div id="example">
  ## Ejemplo
</div>

Habilitar las funciones de introspección:

```sql
SET allow_introspection_functions = 1;
```

Obtención de símbolos de los archivos de objetos de ClickHouse:

```sql
WITH arrayMap(x -> demangle(addressToSymbol(x)), trace) AS all SELECT thread_name, thread_id, query_id, arrayStringConcat(all, '\n') AS res FROM system.stack_trace LIMIT 1;
```

Cómo obtener nombres de archivo y números de línea en el código fuente de ClickHouse:

```sql
WITH arrayMap(x -> addressToLine(x), trace) AS all, arrayFilter(x -> x LIKE '%/dbms/%', all) AS dbms SELECT thread_name, thread_id, query_id, arrayStringConcat(notEmpty(dbms) ? dbms : all, '\n') AS res FROM system.stack_trace LIMIT 1;
```
