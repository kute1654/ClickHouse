<div id="description">
  ## Descripción
</div>

Contiene información sobre las inserciones asíncronas. Cada entrada representa una consulta de inserción puesta en búfer como parte de una consulta de inserción asíncrona.

Para activar el registro, configure los parámetros en la sección [asynchronous&#95;insert&#95;log](/es/reference/settings/server-settings/settings#asynchronous_insert_log).

El intervalo de vaciado de los datos se establece en el parámetro `flush_interval_milliseconds` de la sección de configuración del servidor [asynchronous&#95;insert&#95;log](/es/reference/settings/server-settings/settings#asynchronous_insert_log). Para forzar el vaciado, use la consulta [SYSTEM FLUSH LOGS](/es/reference/statements/system#flush-logs).

ClickHouse no elimina los datos de la tabla automáticamente. Consulte [Introducción](/es/reference/system-tables/overview#system-tables-introduction) para obtener más detalles.

<div id="columns">
  ## Columnas
</div>

{/*AUTOGENERATED_START*/}

* `hostname` ([LowCardinality(String)](/es/reference/data-types/lowcardinality)) — Hostname del servidor que ejecuta la consulta.
* `event_date` ([Date](/es/reference/data-types/date)) — La fecha en la que se produjo la inserción asíncrona.
* `event_time` ([DateTime](/es/reference/data-types/datetime)) — La fecha y hora en que la inserción asíncrona terminó de ejecutarse.
* `event_time_microseconds` ([DateTime64(6)](/es/reference/data-types/datetime64)) — La fecha y hora en que la inserción asíncrona terminó de ejecutarse con precisión de microsegundos.
* `query` ([String](/es/reference/data-types/string)) — Cadena de la consulta.
* `database` ([LowCardinality(String)](/es/reference/data-types/lowcardinality)) — El nombre de la base de datos en la que se encuentra la tabla.
* `table` ([LowCardinality(String)](/es/reference/data-types/lowcardinality)) — Nombre de la tabla.
* `format` ([LowCardinality(String)](/es/reference/data-types/lowcardinality)) — Nombre del formato.
* `query_id` ([String](/es/reference/data-types/string)) — ID de la consulta inicial.
* `bytes` ([UInt64](/es/reference/data-types/int-uint)) — Número de bytes insertados.
* `rows` ([UInt64](/es/reference/data-types/int-uint)) — Número de filas insertadas.
* `exception` ([String](/es/reference/data-types/string)) — Mensaje de la excepción.
* `status` ([Enum8(&#39;Ok&#39; = 0, &#39;ParsingError&#39; = 1, &#39;FlushError&#39; = 2)](/es/reference/data-types/enum)) — Estado del insert. Valores: &#39;Ok&#39; = 0 — Insert correcto, &#39;ParsingError&#39; = 1 — Excepción al analizar los datos, &#39;FlushError&#39; = 2 — Excepción al hacer flush de los datos.
* `data_kind` ([Enum8(&#39;Parsed&#39; = 0, &#39;Preprocessed&#39; = 1)](/es/reference/data-types/enum)) — El estado de los datos. Valores: &#39;Parsed&#39; y &#39;Preprocessed&#39;.
* `flush_time` ([DateTime](/es/reference/data-types/datetime)) — La fecha y hora en que se produjo el flush.
* `flush_time_microseconds` ([DateTime64(6)](/es/reference/data-types/datetime64)) — La fecha y hora en que se produjo el flush con precisión de microsegundos.
* `flush_query_id` ([String](/es/reference/data-types/string)) — ID de la consulta de flush.
* `timeout_milliseconds` ([UInt64](/es/reference/data-types/int-uint)) — El tiempo de espera adaptativo calculado para esta entrada.

{/*AUTOGENERATED_END*/}
