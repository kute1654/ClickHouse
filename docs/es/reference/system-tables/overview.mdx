<div id="system-tables-introduction">
  ## DescripciГіn general de las tablas del sistema
</div>

Las tablas del sistema proporcionan informaciГіn sobre:

* Estados, procesos y entorno del servidor.
* Los procesos internos del servidor.
* Las opciones utilizadas al compilar el binario de ClickHouse.

Las tablas del sistema:

* EstГЎn ubicadas en la base de datos `system`.
* Solo estГЎn disponibles para lectura.
* No se pueden eliminar ni modificar, pero sГӯ se pueden desvincular.

La mayorГӯa de las tablas del sistema almacenan sus datos en RAM. Un servidor ClickHouse crea estas tablas del sistema al iniciarse.

A diferencia de otras tablas del sistema, las tablas de registro del sistema [metric&#95;log](/es/reference/system-tables/metric_log), [query&#95;log](/es/reference/system-tables/query_log), [query&#95;thread&#95;log](/es/reference/system-tables/query_thread_log), [trace&#95;log](/es/reference/system-tables/trace_log), [part&#95;log](/es/reference/system-tables/part_log), [crash&#95;log](/es/reference/system-tables/crash_log), [text&#95;log](/es/reference/system-tables/text_log) y [backup&#95;log](/es/reference/system-tables/backup_log) usan el motor de tabla [MergeTree](/es/reference/engines/table-engines/mergetree-family/mergetree) y almacenan sus datos en un sistema de archivos de forma predeterminada. Si elimina una tabla del sistema de archivos, el servidor ClickHouse volverГЎ a crear una vacГӯa la prГіxima vez que se escriban datos. Si el esquema de una tabla del sistema cambia en una nueva versiГіn, ClickHouse cambia el nombre de la tabla actual y crea una nueva.

Las tablas de registro del sistema se pueden personalizar creando un archivo de configuraciГіn con el mismo nombre que la tabla en `/etc/clickhouse-server/config.d/`, o configurando los elementos correspondientes en `/etc/clickhouse-server/config.xml`. Los elementos que se pueden personalizar son:

* `database`: base de datos a la que pertenece la tabla de registro del sistema. Esta opciГіn ahora estГЎ obsoleta. Todas las tablas de registro del sistema estГЎn en la base de datos `system`.
* `table`: tabla en la que insertar datos.
* `partition_by`: especifica la expresiГіn [PARTITION BY](/es/reference/engines/table-engines/mergetree-family/custom-partitioning-key).
* `ttl`: especifica la expresiГіn [TTL](/es/reference/statements/alter/ttl) de la tabla.
* `flush_interval_milliseconds`: intervalo de volcado de datos a disco.
* `engine`: proporciona la expresiГіn completa del motor (empezando por `ENGINE =` ) con parГЎmetros. Esta opciГіn entra en conflicto con `partition_by` y `ttl`. Si se configuran juntas, el servidor generarГЎ una excepciГіn y finalizarГЎ.

Un ejemplo:

```xml
<clickhouse>
    <query_log>
        <database>system</database>
        <table>query_log</table>
        <partition_by>toYYYYMM(event_date)</partition_by>
        <ttl>event_date + INTERVAL 30 DAY DELETE</ttl>
        <!--
        <engine>ENGINE = MergeTree PARTITION BY toYYYYMM(event_date) ORDER BY (event_date, event_time) SETTINGS index_granularity = 1024</engine>
        -->
        <flush_interval_milliseconds>7500</flush_interval_milliseconds>
        <max_size_rows>1048576</max_size_rows>
        <reserved_size_rows>8192</reserved_size_rows>
        <buffer_size_rows_flush_threshold>524288</buffer_size_rows_flush_threshold>
        <flush_on_crash>false</flush_on_crash>
    </query_log>
</clickhouse>
```

De forma predeterminada, el crecimiento de la tabla es ilimitado. Para controlar el tamaГұo de una tabla, puede usar la configuraciГіn de [TTL](/es/reference/statements/alter/ttl) para eliminar registros de log antiguos. TambiГ©n puede usar la funciГіn de particionado de las tablas con motor `MergeTree`.

<div id="system-tables-sources-of-system-metrics">
  ## Fuentes de mГ©tricas del sistema
</div>

Para recopilar mГ©tricas del sistema, el servidor de ClickHouse utiliza:

* la capability `CAP_NET_ADMIN`.
* [procfs](https://en.wikipedia.org/wiki/Procfs) (solo en Linux).

**procfs**

Si el servidor de ClickHouse no tiene la capability `CAP_NET_ADMIN`, intenta usar `ProcfsMetricsProvider` como alternativa. `ProcfsMetricsProvider` permite recopilar mГ©tricas del sistema por consulta (de CPU y E/S).

Si procfs es compatible y estГЎ habilitado en el sistema, el servidor de ClickHouse recopila estas mГ©tricas:

* `OSCPUVirtualTimeMicroseconds`
* `OSCPUWaitMicroseconds`
* `OSIOWaitMicroseconds`
* `OSReadChars`
* `OSWriteChars`
* `OSReadBytes`
* `OSWriteBytes`

<Note>
  `OSIOWaitMicroseconds` estГЎ deshabilitada de forma predeterminada en los kernels de Linux a partir de la versiГіn 5.14.x.
  Puede habilitarla con `sudo sysctl kernel.task_delayacct=1` o creando un archivo `.conf` en `/etc/sysctl.d/` con `kernel.task_delayacct = 1`
</Note>
