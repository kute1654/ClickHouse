<div id="description">
  ## Descripción
</div>

Contiene información sobre los ajustes de sesión del usuario actual.

<div id="columns">
  ## Columnas
</div>

{/*AUTOGENERATED_START*/}

* `name` ([String](/es/reference/data-types/index)) — Nombre del ajuste.
* `value` ([String](/es/reference/data-types/index)) — Valor del ajuste.
* `changed` ([UInt8](/es/reference/data-types/index)) — Muestra si el ajuste se definió explícitamente en la configuración o se cambió explícitamente.
* `description` ([String](/es/reference/data-types/index)) — Breve descripción del ajuste.
* `min` ([Nullable(String)](/es/reference/data-types/index)) — Valor mínimo del ajuste, si se ha establecido alguno mediante restricciones. Si el ajuste no tiene valor mínimo, contiene NULL.
* `max` ([Nullable(String)](/es/reference/data-types/index)) — Valor máximo del ajuste, si se ha establecido alguno mediante restricciones. Si el ajuste no tiene valor máximo, contiene NULL.
* `disallowed_values` ([Array(String)](/es/reference/data-types/index)) — Lista de valores no permitidos
* `readonly` ([UInt8](/es/reference/data-types/index)) — Muestra si el usuario actual puede cambiar el ajuste: 0 — El usuario actual puede cambiar el ajuste, 1 — El usuario actual no puede cambiar el ajuste.
* `type` ([String](/es/reference/data-types/index)) — El tipo de valor que se puede asignar a este ajuste.
* `default` ([String](/es/reference/data-types/index)) — Valor predeterminado del ajuste.
* `alias_for` ([String](/es/reference/data-types/index)) — Nombre del ajuste original si este ajuste es un alias de otro ajuste.
* `is_obsolete` ([UInt8](/es/reference/data-types/index)) — Muestra si un ajuste está obsoleto.
* `tier` ([Enum8(&#39;Production&#39; = 0, &#39;Obsolete&#39; = 4, &#39;Experimental&#39; = 8, &#39;Beta&#39; = 12)](/es/reference/data-types/index)) —
  Nivel de soporte de esta característica. Las características de ClickHouse se organizan en niveles, que varían según el estado actual de su
  desarrollo y las expectativas que se pueden tener al utilizarlas:

- PRODUCTION: La característica es estable, segura de usar y no presenta problemas al interactuar con otras características PRODUCTION.
- BETA: La característica es estable y segura. Se desconoce el resultado de usarla junto con otras características y no se garantiza su correcto funcionamiento. Se agradecen las pruebas y los informes.
- EXPERIMENTAL: La característica está en desarrollo. Está destinada solo a desarrolladores y entusiastas de ClickHouse. Puede funcionar o no, y podría eliminarse en cualquier momento.
- OBSOLETE: Ya no es compatible. O bien ya se ha eliminado o se eliminará en futuras versiones.

{/*AUTOGENERATED_END*/}

<div id="example">
  ## Ejemplo
</div>

El siguiente ejemplo muestra cómo obtener información sobre los ajustes cuyo nombre contiene `min_i`.

```sql
SELECT *
FROM system.settings
WHERE name LIKE '%min_insert_block_size_%'
FORMAT Vertical
```

El uso de `WHERE changed` puede ser útil, por ejemplo, cuando quieras comprobar:

* Si los ajustes definidos en los archivos de configuración se han cargado correctamente y se están utilizando.
* Qué ajustes han cambiado en la sesión actual.

```sql
SELECT * FROM system.settings WHERE changed AND name='load_balancing'
```
