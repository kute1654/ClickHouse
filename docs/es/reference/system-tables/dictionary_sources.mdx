<div id="description">
  ## DescripciГіn
</div>

Contiene la lista de fuentes de diccionario admitidas por el servidor, junto con la documentaciГіn integrada para cada fuente. Una fuente de diccionario determina desde dГіnde se cargan los datos del diccionario; se especifica en la clГЎusula `SOURCE` de una consulta `CREATE DICTIONARY`.

<div id="columns">
  ## Columnas
</div>

{/*AUTOGENERATED_START*/}

* `name` ([String](/es/reference/data-types/index)) вҖ” El nombre de la fuente de diccionario, tal como se especifica en la clГЎusula SOURCE.
* `description` ([String](/es/reference/data-types/index)) вҖ” Una descripciГіn general de lo que hace la fuente de diccionario.
* `syntax` ([String](/es/reference/data-types/index)) вҖ” La estructura de la clГЎusula SOURCE utilizada para especificar el origen. Tenga en cuenta que algunos orГӯgenes estГЎn sujetos a control de acceso cuando un diccionario se crea a partir de una consulta DDL (en lugar de un archivo de configuraciГіn del servidor); consulte la `description` de cada origen para obtener mГЎs detalles.
* `examples` ([String](/es/reference/data-types/index)) вҖ” Ejemplos de uso.
* `introduced_in` ([String](/es/reference/data-types/index)) вҖ” La versiГіn de ClickHouse en la que se introdujo por primera vez el origen, con el formato major.minor.
* `related` ([Array(String)](/es/reference/data-types/index)) вҖ” Los nombres de fuentes de diccionario relacionados.

{/*AUTOGENERATED_END*/}

<div id="example">

```sql title="Query"

WHERE name IN ('clickhouse', 'file')
```
