<div id="description">
  ## Descripción
</div>

Esta tabla no existe si este nodo no está configurado para ejecutar un ClickHouse Keeper en el mismo proceso. Contiene una fila por cada archivo de changelog de Raft (`changelog_<from>_<to>.bin[.zstd]`) gestionado por el almacén de logs de Keeper en el mismo proceso, incluido el archivo activo al que actualmente se siguen añadiendo entradas.

<div id="columns">
  ## Columnas
</div>

{/*AUTOGENERATED_START*/}

* `from_log_index` ([UInt64](/es/reference/data-types/int-uint)) — Primer índice del log de Raft en el archivo (incluido).
* `to_log_index` ([UInt64](/es/reference/data-types/int-uint)) — Último índice del log de Raft cubierto por el nombre del archivo (incluido). En el archivo activo, este es el objetivo de rotación y puede estar por delante de `last_entry_index`.
* `last_entry_index` ([Nullable(UInt64)](/es/reference/data-types/int-uint)) — Índice de log más alto realmente añadido a este archivo. `NULL` si el archivo activo aún no ha recibido ninguna entrada o si el archivo está dañado.
* `entries` ([UInt64](/es/reference/data-types/int-uint)) — Número de entradas añadidas a este archivo. `ALIAS` de `ifNull(last_entry_index - from_log_index + 1, 0)`.
* `path` ([String](/es/reference/data-types/string)) — Ruta del archivo en el disco.
* `disk_name` ([String](/es/reference/data-types/string)) — Nombre del disco que contiene el archivo.
* `size_bytes` ([UInt64](/es/reference/data-types/int-uint)) — Tamaño del archivo en disco.
* `modification_time` ([DateTime](/es/reference/data-types/datetime)) — Fecha y hora de la última modificación del archivo.
* `is_compressed` ([Bool](/es/reference/data-types/boolean)) — La carga útil del archivo está comprimida con zstd.
* `active` ([Bool](/es/reference/data-types/boolean)) — Actualmente se están añadiendo datos a este archivo.
* `is_broken` ([Bool](/es/reference/data-types/boolean)) — Se encontró un registro final corrupto durante el inicio.

{/*AUTOGENERATED_END*/}

Ejemplo:

```sql
SELECT from_log_index, to_log_index, entries, path, active FROM system.keeper_changelogs ORDER BY from_log_index;
```

```text
┌─from_log_index─┬─to_log_index─┬─entries─┬─path───────────────────────────┬─active─┐
│              1 │         1000 │    1000 │ changelog_1_1000.bin.zstd      │ false  │
│           1001 │         2000 │     537 │ changelog_1001_2000.bin.zstd   │ true   │
└────────────────┴──────────────┴─────────┴────────────────────────────────┴────────┘
```
