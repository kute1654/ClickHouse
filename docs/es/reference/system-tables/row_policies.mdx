<div id="description">
  ## Descripción
</div>

Contiene filtros para una tabla concreta, así como una lista de roles y/o usuarios que deben utilizar esta ROW POLICY.

<div id="columns">
  ## Columnas
</div>

{/*AUTOGENERATED_START*/}

* `name` ([String](/es/reference/data-types/index)) — Nombre de una ROW POLICY.
* `short_name` ([String](/es/reference/data-types/index)) — Nombre corto de una ROW POLICY. Los nombres de las ROW POLICY son compuestos; por ejemplo: myfilter ON mydb.mytable. Aquí, &#39;myfilter ON mydb.mytable&#39; es el nombre de la ROW POLICY y &#39;myfilter&#39; es su nombre corto.
* `database` ([String](/es/reference/data-types/index)) — Nombre de la base de datos.
* `table` ([String](/es/reference/data-types/index)) — Nombre de la tabla. Vacío si la política es para la base de datos.
* `id` ([UUID](/es/reference/data-types/index)) — ID de la ROW POLICY.
* `storage` ([String](/es/reference/data-types/index)) — Nombre del directorio donde se almacena la ROW POLICY.
* `select_filter` ([Nullable(String)](/es/reference/data-types/index)) — Expresión que se usa para filtrar en consultas SELECT.
* `is_restrictive` ([UInt8](/es/reference/data-types/index)) — Indica si la ROW POLICY restringe el acceso a las filas. Valor: • 0 — La ROW POLICY se define con la cláusula `AS PERMISSIVE`, • 1 — La ROW POLICY se define con la cláusula AS RESTRICTIVE.
* `apply_to_all` ([UInt8](/es/reference/data-types/index)) — Indica que las ROW POLICY se establecen para todos los roles y/o usuarios.
* `apply_to_list` ([Array(String)](/es/reference/data-types/index)) — Lista de roles y/o usuarios a los que se aplican las ROW POLICY.
* `apply_to_except` ([Array(String)](/es/reference/data-types/index)) — Las ROW POLICY se aplican a todos los roles y/o usuarios excepto a los enumerados.

{/*AUTOGENERATED_END*/}
