<div id="description">
  ## Descripción
</div>

Contiene una lista de todos los formatos, junto con indicadores de si un formato es adecuado para la entrada/salida o si admite paralelización.

<div id="columns">
  ## Columnas
</div>

{/*AUTOGENERATED_START*/}

* `name` ([String](/es/reference/data-types/index)) — Nombre del formato.
* `is_input` ([UInt8](/es/reference/data-types/index)) — Indicador que indica si el formato es apto para la entrada de datos.
* `is_output` ([UInt8](/es/reference/data-types/index)) — Indicador que indica si el formato es apto para la salida de datos.
* `supports_parallel_parsing` ([UInt8](/es/reference/data-types/index)) — Indicador que indica si el formato admite el análisis en paralelo.
* `supports_parallel_formatting` ([UInt8](/es/reference/data-types/index)) — Indicador que indica si el formato admite el formateo en paralelo.
* `is_tty_friendly` ([UInt8](/es/reference/data-types/index)) — Indicador que indica si el formato suele mostrarse correctamente en la terminal. En otros formatos, la CLI pedirá confirmación antes de mostrar la salida.
* `content_type` ([String](/es/reference/data-types/index)) — HTTP Content-Type correspondiente al formato de salida. Puede depender de la configuración actual del formato.
* `supports_random_access` ([UInt8](/es/reference/data-types/index)) — Indicador que indica si el formato admite acceso aleatorio en la entrada.
* `has_schema_inference` ([UInt8](/es/reference/data-types/index)) — El formato puede inferir dinámicamente el esquema a partir de los datos (ya sea del encabezado o metadata integrados, o del propio fragmento de datos).
* `has_external_schema` ([UInt8](/es/reference/data-types/index)) — El formato tiene un esquema fijo o acepta un esquema predefinido en su propio formato.
* `prefers_large_blocks` ([UInt8](/es/reference/data-types/index)) — El formato escribirá bloques más grandes en la salida y generará bloques más grandes en la entrada.
* `supports_append` ([UInt8](/es/reference/data-types/index)) — Con este formato, es posible añadir datos a un único archivo.
* `supports_subsets_of_columns` ([UInt8](/es/reference/data-types/index)) — El formato de entrada puede reconocer cuándo se omiten determinadas columnas.
* `description` ([String](/es/reference/data-types/index)) — Descripción general del formato.
* `examples` ([String](/es/reference/data-types/index)) — Ejemplos de uso.
* `introduced_in` ([String](/es/reference/data-types/index)) — La versión de ClickHouse en la que se introdujo por primera vez el formato, con la forma major.minor.
* `related` ([Array(String)](/es/reference/data-types/index)) — Nombres de formatos relacionados.

{/*AUTOGENERATED_END*/}
