<div id="description">
  ## Descripción
</div>

Contiene información sobre los hilos que ejecutan consultas, por ejemplo, el nombre del hilo, la hora de inicio y la duración del procesamiento de la consulta.

Para habilitar el registro:

1. Configure los parámetros en la sección [query&#95;thread&#95;log](/es/reference/settings/server-settings/settings#query_thread_log).
2. Establezca [log&#95;query&#95;threads](/es/reference/settings/session-settings#log_query_threads) en 1.

El intervalo de vaciado de los datos se establece en el parámetro `flush_interval_milliseconds` de la sección de configuración del servidor [query&#95;thread&#95;log](/es/reference/settings/server-settings/settings#query_thread_log). Para forzar el vaciado, use la consulta [SYSTEM FLUSH LOGS](/es/reference/statements/system#flush-logs).

ClickHouse no elimina automáticamente los datos de la tabla. Consulte [Introducción](/es/reference/system-tables/overview#system-tables-introduction) para obtener más información.

Puede usar la configuración [log&#95;queries&#95;probability](/es/reference/settings/session-settings#log_queries_probability)) para reducir la cantidad de consultas registradas en la tabla `query_thread_log`.

<div id="columns">
  ## Columnas
</div>

{/*AUTOGENERATED_START*/}

* `hostname` ([LowCardinality(String)](/es/reference/data-types/lowcardinality)) — Nombre de host del servidor que ejecuta la consulta.
* `event_date` ([Date](/es/reference/data-types/date)) — La fecha en la que el hilo terminó de ejecutar la consulta.
* `event_time` ([DateTime](/es/reference/data-types/datetime)) — La fecha y hora en que el hilo terminó de ejecutar la consulta.
* `event_time_microseconds` ([DateTime64(6)](/es/reference/data-types/datetime64)) — La fecha y hora en que el hilo terminó de ejecutar la consulta, con precisión de microsegundos.
* `query_start_time` ([DateTime](/es/reference/data-types/datetime)) — Hora de inicio de la ejecución de la consulta.
* `query_start_time_microseconds` ([DateTime64(6)](/es/reference/data-types/datetime64)) — Momento de inicio de la ejecución de la consulta con precisión de microsegundos.
* `query_duration_ms` ([UInt64](/es/reference/data-types/int-uint)) — Duración de ejecución de la consulta.
* `read_rows` ([UInt64](/es/reference/data-types/int-uint)) — Número de filas leídas.
* `read_bytes` ([UInt64](/es/reference/data-types/int-uint)) — Número de bytes leídos.
* `written_rows` ([UInt64](/es/reference/data-types/int-uint)) — Para las consultas INSERT, el número de filas escritas. Para el resto de las consultas, el valor de la columna es 0.
* `written_bytes` ([UInt64](/es/reference/data-types/int-uint)) — En las consultas INSERT, la cantidad de bytes escritos. En otras consultas, el valor de la columna es 0.
* `memory_usage` ([Int64](/es/reference/data-types/int-uint)) — La diferencia entre la memoria asignada y la liberada en el contexto de este hilo.
* `peak_memory_usage` ([Int64](/es/reference/data-types/int-uint)) — La diferencia máxima entre la cantidad de memoria asignada y la liberada en el contexto de este hilo.
* `thread_name` ([LowCardinality(String)](/es/reference/data-types/lowcardinality)) — Nombre del thread.
* `thread_id` ([UInt64](/es/reference/data-types/int-uint)) — ID interno del thread.
* `master_thread_id` ([UInt64](/es/reference/data-types/int-uint)) — ID inicial en el SO del hilo inicial.
* `current_database` ([LowCardinality(String)](/es/reference/data-types/lowcardinality)) — Nombre de la base de datos actual.
* `query` ([String](/es/reference/data-types/string)) — Texto de la consulta.
* `normalized_query_hash` ([UInt64](/es/reference/data-types/int-uint)) — El hash de la consulta normalizada, con las constantes eliminadas, etc.
* `is_initial_query` ([UInt8](/es/reference/data-types/int-uint)) — Tipo de consulta. Valores posibles: 1 — La consulta la inició el cliente, 0 — La inició otra consulta para la ejecución distribuida de consultas.
* `connection_address` ([IPv6](/es/reference/data-types/ipv6)) — La dirección IP del cliente desde la que se realizó la conexión. Si la conexión se realiza a través de un proxy, esta será la dirección del proxy.
* `connection_port` ([UInt16](/es/reference/data-types/int-uint)) — El puerto del cliente desde el que se estableció la conexión. Cuando la conexión se realiza a través de un proxy, este será el puerto del proxy.
* `user` ([LowCardinality(String)](/es/reference/data-types/lowcardinality)) — Nombre del usuario que inició la consulta actual.
* `query_id` ([String](/es/reference/data-types/string)) — identificador de la consulta.
* `address` ([IPv6](/es/reference/data-types/ipv6)) — Dirección IP utilizada para realizar la consulta. Si la conexión se realiza a través de un proxy y `auth_use_forwarded_address` está configurado, esta será la dirección del cliente en lugar de la del proxy.
* `port` ([UInt16](/es/reference/data-types/int-uint)) — El puerto del cliente que se utilizó para realizar la consulta. Cuando la conexión se realiza a través de un proxy y `auth_use_forwarded_address` está configurado, este será el puerto del cliente en lugar del del proxy.
* `initial_user` ([LowCardinality(String)](/es/reference/data-types/lowcardinality)) — Nombre del usuario que ejecutó la consulta inicial (para la ejecución distribuida de consultas).
* `initial_query_id` ([String](/es/reference/data-types/string)) — ID de la consulta inicial (para la ejecución distribuida de consultas).
* `initial_address` ([IPv6](/es/reference/data-types/ipv6)) — dirección IP desde la que se lanzó la consulta principal.
* `initial_port` ([UInt16](/es/reference/data-types/int-uint)) — El puerto del cliente utilizado para realizar la consulta principal.
* `initial_query_start_time` ([DateTime](/es/reference/data-types/datetime)) — Hora de inicio de la ejecución de la consulta inicial.
* `initial_query_start_time_microseconds` ([DateTime64(6)](/es/reference/data-types/datetime64)) — Hora de inicio de la ejecución de la consulta inicial
* `authenticated_user` ([LowCardinality(String)](/es/reference/data-types/lowcardinality)) — Nombre del usuario autenticado en la sesión.
* `interface` ([UInt8](/es/reference/data-types/int-uint)) — Interfaz desde la cual se inició la consulta. Valores posibles: 1 — TCP, 2 — HTTP.
* `is_secure` ([UInt8](/es/reference/data-types/int-uint)) — Indica si la conexión era segura.
* `os_user` ([LowCardinality(String)](/es/reference/data-types/lowcardinality)) — nombre de usuario del sistema operativo que ejecuta `clickhouse-client`.
* `client_hostname` ([LowCardinality(String)](/es/reference/data-types/lowcardinality)) — Nombre de host del equipo cliente en el que se ejecuta `clickhouse-client` u otro cliente TCP.
* `client_name` ([LowCardinality(String)](/es/reference/data-types/lowcardinality)) — El nombre de `clickhouse-client` o de otro cliente TCP.
* `client_agent` ([LowCardinality(String)](/es/reference/data-types/lowcardinality)) — El agente de IA para programación que invocó al cliente (por ejemplo, `claude-code` o `cursor`), detectado a partir de variables de entorno. Vacío si no se detectó ningún agente.
* `client_revision` ([UInt32](/es/reference/data-types/int-uint)) — Revisión de `clickhouse-client` o de otro cliente TCP.
* `client_version_major` ([UInt32](/es/reference/data-types/int-uint)) — Versión principal de clickhouse-client o de cualquier otro cliente TCP.
* `client_version_minor` ([UInt32](/es/reference/data-types/int-uint)) — Versión menor de clickhouse-client o de otro cliente TCP.
* `client_version_patch` ([UInt32](/es/reference/data-types/int-uint)) — Componente de parche de la versión de clickhouse-client o de cualquier otro cliente TCP.
* `script_query_number` ([UInt32](/es/reference/data-types/int-uint)) — Un número secuencial de consulta en un script con múltiples consultas.
* `script_line_number` ([UInt32](/es/reference/data-types/int-uint)) — Número de línea en un script con varias consultas donde comienza la consulta actual.
* `http_method` ([UInt8](/es/reference/data-types/int-uint)) — Método HTTP que inició la consulta. Valores posibles: 0 — La consulta se inició desde la interfaz TCP, 1 — se utilizó el método GET., 2 — se utilizó el método POST.
* `http_user_agent` ([LowCardinality(String)](/es/reference/data-types/lowcardinality)) — El encabezado UserAgent que se pasa en la solicitud HTTP.
* `http_referer` ([String](/es/reference/data-types/string)) — encabezado HTTP `Referer` enviado en la consulta HTTP (contiene una dirección absoluta o parcial de la página que realiza la consulta).
* `forwarded_for` ([String](/es/reference/data-types/string)) — encabezado HTTP `X-Forwarded-For` incluido en la consulta HTTP.
* `quota_key` ([String](/es/reference/data-types/string)) — La &#39;clave de cuota&#39; especificada en la configuración de cuotas.
* `distributed_depth` ([UInt64](/es/reference/data-types/int-uint)) — Cuántas veces se reenvió una consulta entre servidores.
* `revision` ([UInt32](/es/reference/data-types/int-uint)) — revisión de ClickHouse.
* `ProfileEvents` ([Map(LowCardinality(String), UInt64)](/es/reference/data-types/map)) — `ProfileEvents` que miden distintas métricas de este hilo. Su descripción puede encontrarse en la tabla system.events.

**Alias:**

* `ProfileEvents.Names` — Alias de `mapKeys(ProfileEvents)`.
* `ProfileEvents.Values` — Alias de `mapValues(ProfileEvents)`.
