<div id="description">
  ## Descripción
</div>

Contiene información sobre las [QUOTAS](/es/reference/system-tables/quotas).

<div id="columns">
  ## Columnas
</div>

{/*AUTOGENERATED_START*/}

* `name` ([String](/es/reference/data-types/index)) — Nombre de la QUOTA.
* `id` ([UUID](/es/reference/data-types/index)) — ID de la QUOTA.
* `storage` ([String](/es/reference/data-types/index)) — Almacenamiento de las QUOTAS. Valor posible: &#39;users.xml&#39; si la QUOTA está configurada en el archivo users.xml, &#39;disk&#39; si la QUOTA está configurada mediante una consulta SQL.
* `keys` ([Array(Enum8(&#39;user&#95;name&#39; = 1, &#39;ip&#95;address&#39; = 2, &#39;forwarded&#95;ip&#95;address&#39; = 3, &#39;client&#95;key&#39; = 4, &#39;normalized&#95;query&#95;hash&#39; = 7))](/es/reference/data-types/index)) — La clave especifica cómo debe compartirse la QUOTA. Si dos conexiones usan la misma QUOTA y la misma clave, comparten las mismas cantidades de recursos. Valores: [] — Todos los usuarios comparten la misma QUOTA, [&#39;user&#95;name&#39;] — Las conexiones con el mismo nombre de usuario comparten la misma QUOTA, [&#39;ip&#95;address&#39;] — Las conexiones desde la misma dirección IP comparten la misma QUOTA. [&#39;client&#95;key&#39;] — Las conexiones con la misma clave comparten la misma QUOTA. El cliente debe proporcionar explícitamente una clave. Al usar clickhouse-client, pase un valor de clave en el parámetro --quota&#95;key, o use el parámetro quota&#95;key en el archivo de configuración del cliente. Al usar la interfaz HTTP, use la cabecera X-ClickHouse-Quota, [&#39;user&#95;name&#39;, &#39;client&#95;key&#39;] — Las conexiones con la misma client&#95;key comparten la misma QUOTA. Si el cliente no proporciona una clave, la QUOTA se registra para `user_name`, [&#39;client&#95;key&#39;, &#39;ip&#95;address&#39;] — Las conexiones con la misma client&#95;key comparten la misma QUOTA. Si el cliente no proporciona una clave, la QUOTA se registra para ip&#95;address.
* `durations` ([Array(UInt32)](/es/reference/data-types/index)) — Duración de los intervalos de tiempo en segundos.
* `apply_to_all` ([UInt8](/es/reference/data-types/index)) — Valor lógico. Muestra a qué usuarios se aplica la QUOTA. Valores: 0 — La QUOTA se aplica a los usuarios especificados en apply&#95;to&#95;list. 1 — La QUOTA se aplica a todos los usuarios excepto a los enumerados en apply&#95;to&#95;except.
* `apply_to_list` ([Array(String)](/es/reference/data-types/index)) — Lista de nombres de usuario/roles a los que debe aplicarse la QUOTA.
* `apply_to_except` ([Array(String)](/es/reference/data-types/index)) — Lista de nombres de usuario/roles a los que no debe aplicarse la QUOTA.
* `ipv4_prefix_bits` ([Nullable(UInt8)](/es/reference/data-types/index)) — Número de bits de prefijo para el enmascaramiento de direcciones IPv4. Las conexiones desde la misma subred comparten la misma QUOTA. Solo se usa cuando la clave es `ip_address` o `forwarded_ip_address`.
* `ipv6_prefix_bits` ([Nullable(UInt8)](/es/reference/data-types/index)) — Número de bits de prefijo para el enmascaramiento de direcciones IPv6. Las conexiones desde la misma subred comparten la misma QUOTA. Solo se usa cuando la clave es `ip_address` o `forwarded_ip_address`.
