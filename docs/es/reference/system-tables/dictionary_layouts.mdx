<div id="description">
  ## Descripción
</div>

Contiene la lista de las estructuras de diccionario admitidas por el servidor, junto con la documentación integrada de cada una. Una estructura de diccionario determina cómo se almacena un diccionario en memoria (o en disco) y cómo se realizan las búsquedas en él; se especifica en la cláusula `LAYOUT` de una consulta `CREATE DICTIONARY`.

<div id="columns">
  ## Columnas
</div>

{/*AUTOGENERATED_START*/}

* `name` ([String](/es/reference/data-types/index)) — El nombre de la estructura de diccionario, tal como se especifica en la cláusula LAYOUT.
* `is_complex` ([UInt8](/es/reference/data-types/index)) — Si la estructura de diccionario requiere una clave compleja (una clave que consta de varios atributos o de un tipo no entero).
* `description` ([String](/es/reference/data-types/index)) — Una descripción general de lo que hace la estructura de diccionario.
* `syntax` ([String](/es/reference/data-types/index)) — Cómo se especifica la estructura de diccionario en la cláusula LAYOUT de una consulta CREATE DICTIONARY.
* `examples` ([String](/es/reference/data-types/index)) — Ejemplos de uso.
* `introduced_in` ([String](/es/reference/data-types/index)) — La versión de ClickHouse en la que se introdujo por primera vez la estructura de diccionario, en el formato major.minor.
* `related` ([Array(String)](/es/reference/data-types/index)) — Los nombres de las estructura de diccionario relacionadas.

{/*AUTOGENERATED_END*/}

<div id="example">
  ## Ejemplo
</div>

```sql title="Query"
SELECT name, is_complex, syntax
FROM system.dictionary_layouts
WHERE name IN ('flat', 'hashed', 'complex_key_hashed')
ORDER BY name
```

```text title="Response"
┌─name───────────────┬─is_complex─┬─syntax───────────────────────────────────────────────────┐
│ complex_key_hashed │          1 │ LAYOUT(COMPLEX_KEY_HASHED())                             │
│ flat               │          0 │ LAYOUT(FLAT([INITIAL_ARRAY_SIZE n] [MAX_ARRAY_SIZE n]))  │
│ hashed             │          0 │ LAYOUT(HASHED())                                         │
└────────────────────┴────────────┴──────────────────────────────────────────────────────────┘
```
