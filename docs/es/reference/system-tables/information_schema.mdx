`INFORMATION_SCHEMA` (o `information_schema`) es una base de datos del sistema que proporciona una [vista independiente del SGBD](https://en.wikipedia.org/wiki/Information_schema) de los metadatos de los objetos de la base de datos, de forma casi estandarizada. Por lo general, las vistas de `INFORMATION_SCHEMA` son menos completas que las tablas del sistema normales, pero las herramientas pueden usarlas para obtener información básica de forma transversal entre distintos SGBD. Se espera que la estructura y el contenido de las vistas de `INFORMATION_SCHEMA` evolucionen de forma retrocompatible; es decir, solo se añade funcionalidad nueva, pero la funcionalidad existente no se modifica ni se elimina. En cuanto a la implementación interna, las vistas de `INFORMATION_SCHEMA` suelen corresponder a tablas del sistema normales como [system.columns](/es/reference/system-tables/columns), [system.databases](/es/reference/system-tables/databases) y [system.tables](/es/reference/system-tables/tables).

```sql
SHOW TABLES FROM INFORMATION_SCHEMA;

-- or:
SHOW TABLES FROM information_schema;
```

│ KEY_COLUMN_USAGE        │

`INFORMATION_SCHEMA` contiene las siguientes vistas:

* [COLUMNS](#columns)
* [KEY&#95;COLUMN&#95;USAGE](#key_column_usage)
* [REFERENTIAL&#95;CONSTRAINTS](#referential_constraints)
* [SCHEMATA](#schemata)
* [STATISTICS](#statistics)
* [TABLES](#tables)
* [VIEWS](#views)

Se ofrecen vistas equivalentes que no distinguen entre mayúsculas y minúsculas, por ejemplo, `INFORMATION_SCHEMA.columns`, por motivos de compatibilidad con otras bases de datos. Lo mismo se aplica a todas las columnas de estas vistas: se ofrecen variantes tanto en minúsculas (por ejemplo, `table_name`) como en mayúsculas (`TABLE_NAME`).

<div id="schemata">
  ## SCHEMATA
</div>

Contiene columnas leídas de la tabla del sistema [system.databases](/es/reference/system-tables/databases) y columnas que no son compatibles con ClickHouse o que no tienen sentido (siempre `NULL`), pero que el estándar requiere.

Columnas:

* `catalog_name` ([String](/es/reference/data-types/string)) — El nombre de la base de datos.
* `schema_name` ([String](/es/reference/data-types/string)) — El nombre de la base de datos.
* `schema_owner` ([String](/es/reference/data-types/string)) — Nombre del propietario del esquema, siempre `'default'`.
* `default_character_set_catalog` ([Nullable](/es/reference/data-types/nullable)([String](/es/reference/data-types/string))) — `NULL`, no se admite.
* `default_character_set_schema` ([Nullable](/es/reference/data-types/nullable)([String](/es/reference/data-types/string))) — `NULL`, no se admite.
* `default_character_set_name` ([Nullable](/es/reference/data-types/nullable)([String](/es/reference/data-types/string))) — `NULL`, no se admite.
* `sql_path` ([Nullable](/es/reference/data-types/nullable)([String](/es/reference/data-types/string))) — `NULL`, no se admite.

**Ejemplo**

```sql title="Query"
SELECT catalog_name,
       schema_name,
       schema_owner,
       default_character_set_catalog,
       default_character_set_schema,
       default_character_set_name,
       sql_path
FROM information_schema.schemata
WHERE schema_name ILIKE 'information_schema' 
LIMIT 1 
FORMAT Vertical;
```

<div id="tables">
  ## TABLES
</div>

Contiene las columnas leídas de la tabla del sistema [system.tables](/es/reference/system-tables/tables).

Columnas:

* `table_catalog` ([String](/es/reference/data-types/string)) — El nombre de la base de datos en la que se encuentra la tabla.
* `table_schema` ([String](/es/reference/data-types/string)) — El nombre de la base de datos en la que se encuentra la tabla.
* `table_name` ([String](/es/reference/data-types/string)) — Nombre de la tabla.
* `table_type` ([String](/es/reference/data-types/string)) — Tipo de tabla. Valores posibles:
  * `BASE TABLE`
  * `VIEW`
  * `FOREIGN TABLE`
  * `LOCAL TEMPORARY`
  * `SYSTEM VIEW`
* `table_rows` ([Nullable](/es/reference/data-types/nullable)([UInt64](/es/reference/data-types/int-uint))) — El número total de filas. NULL si no se pudo determinar.
* `data_length` ([Nullable](/es/reference/data-types/nullable)([UInt64](/es/reference/data-types/int-uint))) — El tamaño de los datos en disco. NULL si no se pudo determinar.
* `index_length` ([Nullable](/es/reference/data-types/nullable)([UInt64](/es/reference/data-types/int-uint))) — El tamaño total de la clave primaria, los índices secundarios y todas las marcas.
* `table_collation` ([Nullable](/es/reference/data-types/nullable)([String](/es/reference/data-types/string))) — La intercalación predeterminada de la tabla. Siempre `utf8mb4_0900_ai_ci`.
* `table_comment` ([Nullable](/es/reference/data-types/nullable)([String](/es/reference/data-types/string))) — El comentario utilizado al crear la tabla.

**Ejemplo**

```sql title="Query"
SELECT table_catalog, 
       table_schema, 
       table_name, 
       table_type, 
       table_collation, 
       table_comment
FROM INFORMATION_SCHEMA.TABLES
WHERE (table_schema = currentDatabase() OR table_schema = '')
  AND table_name NOT LIKE '%inner%'
LIMIT 1 
FORMAT Vertical;
```

```text title="Response"
Row 1:
──────
table_catalog:   default
table_schema:    default
table_name:      describe_example
table_type:      BASE TABLE
table_collation: utf8mb4_0900_ai_ci
table_comment:   
```

<div id="views">
  ## VIEWS
</div>

Contiene las columnas leídas de la tabla del sistema [system.tables](/es/reference/system-tables/tables) cuando se utiliza el motor de tabla [View](/es/reference/engines/table-engines/special/view).

Columnas:

{/*AUTOGENERATED_START*/}

* `table_catalog` ([String](/es/reference/data-types/string)) — El nombre de la base de datos en la que se encuentra la tabla.
* `table_schema` ([String](/es/reference/data-types/string)) — El nombre de la base de datos en la que se encuentra la tabla.
* `table_name` ([String](/es/reference/data-types/string)) — Nombre de la tabla.
* `view_definition` ([String](/es/reference/data-types/string)) — consulta `SELECT` de la vista.
* `check_option` ([String](/es/reference/data-types/string)) — `NONE`, sin comprobación.
* `is_updatable` ([Enum8](/es/reference/data-types/enum)) — `NO`, la vista no es actualizable.
* `is_insertable_into` ([Enum8](/es/reference/data-types/enum)) — Indica si la vista creada está [materializada](/es/reference/statements/create/view#materialized-view). Valores posibles:
  * `NO` — La vista creada no está materializada.
  * `YES` — La vista creada está materializada.
* `is_trigger_updatable` ([Enum8](/es/reference/data-types/enum)) — `NO`, el trigger no es actualizable.
* `is_trigger_deletable` ([Enum8](/es/reference/data-types/enum)) — `NO`, el trigger no se elimina.
* `is_trigger_insertable_into` ([Enum8](/es/reference/data-types/enum)) — `NO`, no se insertan datos en el trigger.

{/*AUTOGENERATED_END*/}

**Ejemplo**

```sql title="Query"
CREATE VIEW v (n Nullable(Int32), f Float64) AS SELECT n, f FROM t;
CREATE MATERIALIZED VIEW mv ENGINE = Null AS SELECT * FROM system.one;
SELECT table_catalog,
       table_schema,
       table_name,
       view_definition,
       check_option,
       is_updatable,
       is_insertable_into,
       is_trigger_updatable,
       is_trigger_deletable,
       is_trigger_insertable_into
FROM information_schema.views
WHERE table_schema = currentDatabase() 
LIMIT 1
FORMAT Vertical;
```

```text title="Response"
Row 1:
──────
table_catalog:              default
table_schema:               default
table_name:                 mv
view_definition:            SELECT * FROM system.one
check_option:               NONE
is_updatable:               NO
is_insertable_into:         YES
is_trigger_updatable:       NO
is_trigger_deletable:       NO
is_trigger_insertable_into: NO
```

<div id="key_column_usage">
  ## KEY_COLUMN_USAGE
</div>

Contiene columnas de la tabla del sistema [system.tables](/es/reference/system-tables/tables) que están restringidas por restricciones.

Columnas:

* `constraint_catalog` ([String](/es/reference/data-types/string)) — Actualmente no se usa. Siempre es `def`.
* `constraint_schema` ([String](/es/reference/data-types/string)) — El nombre del esquema (base de datos) al que pertenece la restricción.
* `constraint_name` ([Nullable](/es/reference/data-types/nullable)([String](/es/reference/data-types/string))) — El nombre de la restricción.
* `table_catalog` ([String](/es/reference/data-types/string)) — Actualmente no se usa. Siempre es `def`.
* `table_schema` ([String](/es/reference/data-types/string)) — El nombre del esquema (base de datos) al que pertenece la tabla.
* `table_name` ([String](/es/reference/data-types/string)) — El nombre de la tabla que tiene la restricción.
* `column_name` ([Nullable](/es/reference/data-types/nullable)([String](/es/reference/data-types/string))) — El nombre de la columna que tiene la restricción.
* `ordinal_position` ([UInt32](/es/reference/data-types/int-uint)) — Actualmente no se usa. Siempre es `1`.
* `position_in_unique_constraint` ([Nullable](/es/reference/data-types/nullable)([UInt32](/es/reference/data-types/int-uint))) — Actualmente no se usa. Siempre es `NULL`.
* `referenced_table_schema` ([Nullable](/es/reference/data-types/nullable)([String](/es/reference/data-types/string))) — Actualmente no se usa. Siempre es NULL.
* `referenced_table_name` ([Nullable](/es/reference/data-types/nullable)([String](/es/reference/data-types/string))) — Actualmente no se usa. Siempre es NULL.
* `referenced_column_name` ([Nullable](/es/reference/data-types/nullable)([String](/es/reference/data-types/string))) — Actualmente no se usa. Siempre es NULL.

**Ejemplo**

```sql title="Query"
CREATE TABLE test (i UInt32, s String) ENGINE MergeTree ORDER BY i;
SELECT constraint_catalog,
       constraint_schema,
       constraint_name,
       table_catalog,
       table_schema,
       table_name,
       column_name,
       ordinal_position,
       position_in_unique_constraint,
       referenced_table_schema,
       referenced_table_name,
       referenced_column_name
FROM information_schema.key_column_usage 
WHERE table_name = 'test' 
FORMAT Vertical;
```
