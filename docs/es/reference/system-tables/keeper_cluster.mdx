<div id="description">
  ## Descripción
</div>

Esta tabla no existe si este nodo no está configurado para ejecutar ClickHouse Keeper en el mismo proceso. Contiene una fila por cada miembro del clúster de Raft y combina la topología estática del clúster (de la configuración de Raft) con la posición en el log del propio nodo local.

Cada nodo completa exactamente un valor de `last_log_index`: la fila que coincide con su propio `server_id`. Las posiciones en el log de los demás nodos no se muestran aquí porque solo se registran en el líder y esa vista no es simétrica en todo el clúster.

<div id="columns">
  ## Columnas
</div>

{/*AUTOGENERATED_START*/}

* `server_id` ([Int32](/es/reference/data-types/int-uint)) — id del servidor Raft de este miembro del clúster de Raft.
* `host` ([String](/es/reference/data-types/string)) — Host extraído del endpoint (prefijo antes del último signo de dos puntos).
* `endpoint` ([String](/es/reference/data-types/string)) — Endpoint de Raft en bruto, tal como está configurado (`host:port`).
* `is_observer` ([Bool](/es/reference/data-types/boolean)) — `true` si este miembro es un observador sin derecho a voto, configurado mediante `<can_become_leader>false</can_become_leader>`.
* `priority` ([Int32](/es/reference/data-types/int-uint)) — Prioridad de Raft de este miembro; se prefieren los valores más altos durante la elección de líder.
* `is_leader` ([Bool](/es/reference/data-types/boolean)) — `true` si este miembro es el líder actual de Raft.
* `is_self` ([Bool](/es/reference/data-types/boolean)) — `true` si esta fila describe el nodo local de Keeper.
* `last_log_index` ([Nullable(UInt64)](/es/reference/data-types/int-uint)) — Índice de log más alto en el almacenamiento local del log de Raft de este nodo. Se completa solo en la fila que coincide con el nodo actual (`is_self = true`); `NULL` en las demás filas.

{/*AUTOGENERATED_END*/}

Ejemplo:

```sql
SELECT * FROM system.keeper_cluster ORDER BY server_id;
```
