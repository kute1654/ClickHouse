<div id="description">
  ## Descripción
</div>

Contiene información sobre los [recursos](/es/concepts/features/configuration/server-config/workload-scheduling#workload_entity_storage) ubicados en el servidor local. La tabla contiene una fila por cada recurso.

<div id="columns">
  ## Columnas
</div>

{/*AUTOGENERATED_START*/}

* `name` ([String](/es/reference/data-types/index)) — El nombre del recurso.
* `read_disks` ([Array(String)](/es/reference/data-types/index)) — La lista de nombres de discos que usan este recurso para operaciones de lectura.
* `write_disks` ([Array(String)](/es/reference/data-types/index)) — La lista de nombres de discos que usan este recurso para operaciones de escritura.
* `unit` ([String](/es/reference/data-types/index)) — Unidad del recurso utilizada para medir el costo.
* `create_query` ([String](/es/reference/data-types/index)) — La consulta CREATE del recurso.

{/*AUTOGENERATED_END*/}

Ejemplo:

```sql
SELECT *
FROM system.resources
FORMAT Vertical
```

```text
Row 1:
──────
name:         io_read
read_disks:   ['s3']
write_disks:  []
create_query: CREATE RESOURCE io_read (READ DISK s3)

Row 2:
──────
name:         io_write
read_disks:   []
write_disks:  ['s3']
create_query: CREATE RESOURCE io_write (WRITE DISK s3)
```
