<div id="description">
  ## Descripción
</div>

Esta tabla del sistema se utiliza para implementar la consulta `SHOW PROCESSLIST`.

<div id="columns">
  ## Columnas
</div>

{/*AUTOGENERATED_START*/}

* `is_initial_query` ([UInt8](/es/reference/data-types/index)) — Si esta consulta proviene directamente del usuario o fue emitida por ClickHouse server en el contexto de la ejecución distribuida de consultas.
* `user` ([String](/es/reference/data-types/index)) — El usuario que realizó la consulta. Tenga en cuenta que, en el procesamiento distribuido, las consultas se envían a servidores remotos con el usuario `default`. El campo contiene el nombre de usuario de una consulta específica, no de la consulta que esta inició.
* `query_id` ([String](/es/reference/data-types/index)) — ID de la consulta, si está definido.
* `address` ([IPv6](/es/reference/data-types/index)) — La dirección IP desde la que se realizó la consulta. Lo mismo aplica al procesamiento distribuido. Para rastrear desde dónde se realizó originalmente una consulta distribuida, consulte system.processes en el servidor que solicitó la consulta.
* `port` ([UInt16](/es/reference/data-types/index)) — El puerto del client desde el que se realizó la consulta.
* `initial_user` ([String](/es/reference/data-types/index)) — Nombre del usuario que ejecutó la consulta inicial (para la ejecución distribuida de consultas).
* `initial_query_id` ([String](/es/reference/data-types/index)) — ID de la consulta inicial (para la ejecución distribuida de consultas).
* `initial_address` ([IPv6](/es/reference/data-types/index)) — Dirección IP desde la que se lanzó la consulta principal.
* `initial_port` ([UInt16](/es/reference/data-types/index)) — El puerto del client que se utilizó para realizar la consulta principal.
* `interface` ([UInt8](/es/reference/data-types/index)) — La interfaz utilizada para enviar la consulta. TCP = 1, HTTP = 2, GRPC = 3, MYSQL = 4, POSTGRESQL = 5, LOCAL = 6, TCP&#95;INTERSERVER = 7.
* `os_user` ([String](/es/reference/data-types/index)) — Nombre de usuario del sistema operativo que ejecuta clickhouse-client.
* `client_hostname` ([String](/es/reference/data-types/index)) — Nombre de host de la máquina client en la que se ejecuta clickhouse-client u otro client TCP.
* `client_name` ([String](/es/reference/data-types/index)) — El nombre de clickhouse-client u otro client TCP.
* `client_agent` ([LowCardinality(String)](/es/reference/data-types/index)) — El agente de IA para programación que invocó al client (por ejemplo, `claude-code`, `cursor`), detectado a partir de variables de entorno. Vacío si no se detectó ningún agente.
* `client_revision` ([UInt64](/es/reference/data-types/index)) — Revisión de clickhouse-client u otro client TCP.
* `client_version_major` ([UInt64](/es/reference/data-types/index)) — Versión principal de clickhouse-client u otro client TCP.
* `client_version_minor` ([UInt64](/es/reference/data-types/index)) — Versión secundaria de clickhouse-client u otro client TCP.
* `client_version_patch` ([UInt64](/es/reference/data-types/index)) — Componente de parche de la versión de clickhouse-client u otro client TCP.
* `http_method` ([UInt8](/es/reference/data-types/index)) — Método HTTP que inició la consulta. Valores posibles: 0 — La consulta se lanzó desde la interfaz TCP. 1 — Se utilizó el método GET. 2 — Se utilizó el método POST.
* `http_user_agent` ([String](/es/reference/data-types/index)) — Encabezado HTTP UserAgent enviado en la consulta HTTP.
* `http_referer` ([String](/es/reference/data-types/index)) — Encabezado HTTP Referer enviado en la consulta HTTP (contiene una dirección absoluta o parcial de la página que realiza la consulta).
* `forwarded_for` ([String](/es/reference/data-types/index)) — Encabezado HTTP X-Forwarded-For enviado en la consulta HTTP.
* `quota_key` ([String](/es/reference/data-types/index)) — La clave de cuota especificada en la configuración de cuotas (consulte keyed).
* `distributed_depth` ([UInt64](/es/reference/data-types/index)) — El número de veces que la consulta se retransmitió internamente entre nodos del servidor.
* `elapsed` ([Float64](/es/reference/data-types/index)) — El tiempo en segundos transcurrido desde que comenzó la ejecución de la solicitud.
* `is_cancelled` ([UInt8](/es/reference/data-types/index)) — Indica si la consulta fue cancelada.
* `is_all_data_sent` ([UInt8](/es/reference/data-types/index)) — Indica si se enviaron todos los datos al client (en otras palabras, si la consulta había finalizado en el server).
* `read_rows` ([UInt64](/es/reference/data-types/index)) — El número de filas leídas de la tabla. Para el procesamiento distribuido, en el server solicitante, este es el total de todos los servers remotos.
* `read_bytes` ([UInt64](/es/reference/data-types/index)) — El número de bytes sin comprimir leídos de la tabla. Para el procesamiento distribuido, en el server solicitante, este es el total de todos los servers remotos.
* `total_rows_approx` ([UInt64](/es/reference/data-types/index)) — La estimación del número total de filas que deben leerse. Para el procesamiento distribuido, en el server solicitante, este es el total de todos los servers remotos. Puede actualizarse durante el procesamiento de la solicitud, cuando se descubren nuevas fuentes que procesar.
* `written_rows` ([UInt64](/es/reference/data-types/index)) — La cantidad de filas escritas en el almacenamiento.
* `written_bytes` ([UInt64](/es/reference/data-types/index)) — La cantidad de bytes escritos en el almacenamiento.
* `memory_usage` ([Int64](/es/reference/data-types/index)) — Cantidad de RAM que usa la consulta. Puede que no incluya algunos tipos de memoria dedicada.
* `peak_memory_usage` ([Int64](/es/reference/data-types/index)) — El pico actual de uso de memoria.
* `query` ([String](/es/reference/data-types/index)) — El texto de la consulta. Para INSERT, no incluye los datos que se van a insertar.
* `normalized_query_hash` ([UInt64](/es/reference/data-types/index)) — Un valor hash numérico, idéntico para consultas que difieren solo en los valores de los literales.
* `query_kind` ([String](/es/reference/data-types/index)) — El tipo de consulta: SELECT, INSERT, etc.
* `thread_ids` ([Array(UInt64)](/es/reference/data-types/index)) — La lista de identificadores de todos los hilos que participaron en esta consulta.
* `peak_threads_usage` ([UInt64](/es/reference/data-types/index)) — Número máximo de hilos simultáneos que ejecutan la consulta.
* `ProfileEvents` ([Map(LowCardinality(String), UInt64)](/es/reference/data-types/index)) — `ProfileEvents` calculados para esta consulta.
* `Settings` ([Map(LowCardinality(String), LowCardinality(String))](/es/reference/data-types/index)) — La lista de ajustes de nivel de usuario modificados.
* `current_database` ([String](/es/reference/data-types/index)) — El nombre de la base de datos actual.
* `is_internal` ([UInt8](/es/reference/data-types/index)) — Indica si es una consulta auxiliar ejecutada internamente.

{/*AUTOGENERATED_END*/}
