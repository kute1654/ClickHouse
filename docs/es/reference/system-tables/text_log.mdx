<div id="description">
  ## Descripción
</div>

Contiene registros de registro. El nivel de registro que se envía a esta tabla puede limitarse mediante la configuración del servidor `text_log.level`.

<div id="columns">
  ## Columnas
</div>

{/*AUTOGENERATED_START*/}

* `hostname` ([LowCardinality(String)](/es/reference/data-types/lowcardinality)) — Nombre de host del servidor que ejecuta la consulta.
* `event_date` ([Date](/es/reference/data-types/date)) — Fecha de la entrada.
* `event_time` ([DateTime](/es/reference/data-types/datetime)) — Hora de la entrada.
* `event_time_microseconds` ([DateTime64(6)](/es/reference/data-types/datetime64)) — Hora de la entrada con precisión de microsegundos.
* `thread_name` ([LowCardinality(String)](/es/reference/data-types/lowcardinality)) — Nombre del hilo desde el que se realizó el registro.
* `thread_id` ([UInt64](/es/reference/data-types/int-uint)) — ID del hilo del sistema operativo.
* `level` ([Enum8(&#39;Fatal&#39; = 1, &#39;Critical&#39; = 2, &#39;Error&#39; = 3, &#39;Warning&#39; = 4, &#39;Notice&#39; = 5, &#39;Information&#39; = 6, &#39;Debug&#39; = 7, &#39;Trace&#39; = 8, &#39;Test&#39; = 9)](/es/reference/data-types/enum)) — Nivel de la entrada. Valores posibles: 1 o &#39;Fatal&#39;, 2 o &#39;Critical&#39;, 3 o &#39;Error&#39;, 4 o &#39;Warning&#39;, 5 o &#39;Notice&#39;, 6 o &#39;Information&#39;, 7 o &#39;Debug&#39;, 8 o &#39;Trace&#39;.
* `query_id` ([String](/es/reference/data-types/string)) — ID de la consulta.
* `logger_name` ([LowCardinality(String)](/es/reference/data-types/lowcardinality)) — Nombre del logger (p. ej., DDLWorker).
* `message` ([String](/es/reference/data-types/string)) — El mensaje en sí.
* `revision` ([UInt32](/es/reference/data-types/int-uint)) — Revisión de ClickHouse.
* `source_file` ([LowCardinality(String)](/es/reference/data-types/lowcardinality)) — Archivo fuente desde el que se realizó el registro.
* `source_line` ([UInt64](/es/reference/data-types/int-uint)) — Línea del archivo fuente desde la que se realizó el registro.
* `message_format_string` ([LowCardinality(String)](/es/reference/data-types/lowcardinality)) — Cadena de formato utilizada para dar formato al mensaje.
* `value1` ([String](/es/reference/data-types/string)) — Argumento 1 utilizado para dar formato al mensaje.
* `value2` ([String](/es/reference/data-types/string)) — Argumento 2 utilizado para dar formato al mensaje.
* `value3` ([String](/es/reference/data-types/string)) — Argumento 3 utilizado para dar formato al mensaje.
* `value4` ([String](/es/reference/data-types/string)) — Argumento 4 utilizado para dar formato al mensaje.
* `value5` ([String](/es/reference/data-types/string)) — Argumento 5 utilizado para dar formato al mensaje.
* `value6` ([String](/es/reference/data-types/string)) — Argumento 6 utilizado para dar formato al mensaje.
* `value7` ([String](/es/reference/data-types/string)) — Argumento 7 utilizado para dar formato al mensaje.
* `value8` ([String](/es/reference/data-types/string)) — Argumento 8 utilizado para dar formato al mensaje.
* `value9` ([String](/es/reference/data-types/string)) — Argumento 9 utilizado para dar formato al mensaje.
* `value10` ([String](/es/reference/data-types/string)) — Argumento 10 utilizado para dar formato al mensaje.
