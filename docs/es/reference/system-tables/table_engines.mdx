<div id="description">
  ## Descripción
</div>

Contiene una descripción de los motores de tabla compatibles con el servidor e información sobre las funcionalidades que admiten.

Esta tabla contiene las siguientes columnas (el tipo de cada columna se muestra entre corchetes):

{/*AUTOGENERATED_START*/}

* `name` ([String](/es/reference/data-types/index)) — El nombre del motor de tabla.
* `supports_settings` ([UInt8](/es/reference/data-types/index)) — Indicador que señala si el motor de tabla admite la cláusula SETTINGS.
* `supports_skipping_indices` ([UInt8](/es/reference/data-types/index)) — Indicador que señala si el motor de tabla admite índices de omisión.
* `supports_projections` ([UInt8](/es/reference/data-types/index)) — Indicador que señala si el motor de tabla admite proyecciones.
* `supports_sort_order` ([UInt8](/es/reference/data-types/index)) — Indicador que señala si el motor de tabla admite las cláusulas PARTITION&#95;BY, PRIMARY&#95;KEY, ORDER&#95;BY y SAMPLE&#95;BY.
* `supports_ttl` ([UInt8](/es/reference/data-types/index)) — Indicador que señala si el motor de tabla admite TTL.
* `supports_replication` ([UInt8](/es/reference/data-types/index)) — Indicador que señala si el motor de tabla admite la replicación de datos.
* `supports_deduplication` ([UInt8](/es/reference/data-types/index)) — Indicador que señala si el motor de tabla admite la deduplicación de datos.
* `supports_parallel_insert` ([UInt8](/es/reference/data-types/index)) — Indicador que señala si el motor de tabla admite inserciones en paralelo (consulte la configuración max&#95;insert&#95;threads).
* `description` ([String](/es/reference/data-types/index)) — Una descripción de lo que hace el motor de tabla. En los motores que tienen una página de documentación dedicada, esto contiene el contenido completo en Markdown de esa página; para los demás motores, es un resumen conciso.
* `syntax` ([String](/es/reference/data-types/index)) — Cómo se usa el motor de tabla al crear una tabla. Para la mayoría de los motores, esta es la cláusula ENGINE de una consulta CREATE TABLE, pero algunos motores (como los distintos tipos de vistas o el motor Loop) se usan mediante otras formas, como CREATE VIEW o un SELECT desde una función de tabla.
* `examples` ([String](/es/reference/data-types/index)) — Ejemplos de uso.
* `introduced_in` ([String](/es/reference/data-types/index)) — La versión de ClickHouse en la que se introdujo por primera vez el motor de tabla, en el formato major.minor.
* `related` ([Array(String)](/es/reference/data-types/index)) — Los nombres de los motores de tabla relacionados.

{/*AUTOGENERATED_END*/}

<div id="example">
  ## Ejemplo
</div>

```sql title="Query"
SELECT
    name,
    supports_settings,
    supports_skipping_indices,
    supports_sort_order,
    supports_ttl,
    supports_replication,
    supports_deduplication,
    supports_parallel_insert
FROM system.table_engines
WHERE name IN ('Kafka', 'MergeTree', 'ReplicatedCollapsingMergeTree')
```
