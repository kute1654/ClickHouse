<div id="description">
  ## Descripción
</div>

Contiene información sobre todas las entradas de la caché del sistema de archivos para objetos remotos.

<div id="columns">
  ## Columnas
</div>

{/*AUTOGENERATED_START*/}

* `cache_name` ([String](/es/reference/data-types/index)) — Nombre del objeto de caché. Se puede usar en los comandos `SYSTEM DESCRIBE FILESYSTEM CACHE <name>` y `SYSTEM DROP FILESYSTEM CACHE <name>`
* `cache_base_path` ([String](/es/reference/data-types/index)) — Ruta al directorio base donde se almacenan todos los archivos de caché (de una caché identificada por `cache_name`).
* `cache_path` ([String](/es/reference/data-types/index)) — Ruta a un archivo de caché concreto, correspondiente a un segmento de archivo en un archivo de origen
* `key` ([String](/es/reference/data-types/index)) — Clave de caché del segmento de archivo
* `file_segment_range_begin` ([UInt64](/es/reference/data-types/index)) — Desplazamiento correspondiente al inicio del rango del segmento de archivo
* `file_segment_range_end` ([UInt64](/es/reference/data-types/index)) — Desplazamiento correspondiente al final (incluido) del rango del segmento de archivo
* `size` ([UInt64](/es/reference/data-types/index)) — Tamaño del segmento de archivo
* `state` ([String](/es/reference/data-types/index)) — Estado del segmento de archivo (DOWNLOADED, DOWNLOADING, PARTIALLY&#95;DOWNLOADED, ...)
* `finished_download_time` ([DateTime](/es/reference/data-types/index)) — Hora a la que terminó de descargarse el segmento de archivo.
* `cache_hits` ([UInt64](/es/reference/data-types/index)) — Número de aciertos de caché del segmento de archivo correspondiente
* `references` ([UInt64](/es/reference/data-types/index)) — Número de referencias al segmento de archivo correspondiente. El valor 1 significa que nadie lo está usando en este momento (la única referencia existente está en el propio almacenamiento de caché)
* `downloaded_size` ([UInt64](/es/reference/data-types/index)) — Tamaño descargado del segmento de archivo
* `kind` ([String](/es/reference/data-types/index)) — Tipo del segmento de archivo (se usa para distinguir entre segmentos de archivo añadidos como parte de &#39;Datos temporales en caché&#39;)
* `unbound` ([UInt8](/es/reference/data-types/index)) — indicador interno de implementación
* `user_id` ([String](/es/reference/data-types/index)) — Id. del usuario que creó el segmento de archivo
* `segment_type` ([String](/es/reference/data-types/index)) — Tipo del segmento. Se usa para separar los archivos de datos(`.json`, `.txt`, etc.) de los archivos de datos(`.bin`, archivos de marcas).
* `file_size` ([Nullable(UInt64)](/es/reference/data-types/index)) — Tamaño del archivo al que pertenece el segmento de archivo actual

{/*AUTOGENERATED_END*/}
