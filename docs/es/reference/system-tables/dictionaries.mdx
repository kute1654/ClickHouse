<div id="description">
  ## Descripción
</div>

Contiene información sobre [diccionarios](/es/reference/statements/create/dictionary).

<div id="columns">
  ## Columnas
</div>

{/*AUTOGENERATED_START*/}

* `database` ([String](/es/reference/data-types/index)) — Nombre de la base de datos que contiene el diccionario creado por una consulta DDL. Cadena vacía para otros diccionarios.
* `name` ([String](/es/reference/data-types/index)) — Nombre del diccionario.
* `uuid` ([UUID](/es/reference/data-types/index)) — UUID del diccionario.
* `status` ([Enum8(&#39;NOT&#95;LOADED&#39; = 0, &#39;LOADED&#39; = 1, &#39;FAILED&#39; = 2, &#39;LOADING&#39; = 3, &#39;FAILED&#95;AND&#95;RELOADING&#39; = 4, &#39;LOADED&#95;AND&#95;RELOADING&#39; = 5, &#39;NOT&#95;EXIST&#39; = 6)](/es/reference/data-types/index)) — Estado del diccionario. Posibles valores:
  * **NOT&#95;LOADED** — El diccionario no se cargó porque no se utilizó
  * **LOADED** — El diccionario se cargó correctamente
  * **FAILED** — No se pudo cargar el diccionario debido a un error
  * **LOADING** — El diccionario se está cargando en este momento
  * **LOADED&#95;AND&#95;RELOADING** — El diccionario se cargó correctamente
  * **and is being reloaded right now (frequent reasons: SYSTEM RELOAD DICTIONARY query** —
  * **timeout** —
  * **dictionary config has changed)** —
  * **FAILED&#95;AND&#95;RELOADING** — No se pudo cargar el diccionario debido a un error y se está cargando en este momento.
* `origin` ([String](/es/reference/data-types/index)) — Ruta al archivo de configuración que describe el diccionario.
* `type` ([String](/es/reference/data-types/index)) — Tipo de asignación de un diccionario. Almacenamiento de diccionarios en memoria.
* `key.names` ([Array(String)](/es/reference/data-types/index)) — Array de nombres de clave proporcionado por el diccionario.
* `key.types` ([Array(String)](/es/reference/data-types/index)) — Array correspondiente de tipos de clave proporcionado por el diccionario.
* `attribute.names` ([Array(String)](/es/reference/data-types/index)) — Array de nombres de atributos proporcionado por el diccionario.
* `attribute.types` ([Array(String)](/es/reference/data-types/index)) — Array correspondiente de tipos de atributos proporcionado por el diccionario.
* `bytes_allocated` ([UInt64](/es/reference/data-types/index)) — Cantidad de RAM asignada al diccionario.
* `hierarchical_index_bytes_allocated` ([UInt64](/es/reference/data-types/index)) — Cantidad de RAM asignada al índice jerárquico.
* `query_count` ([UInt64](/es/reference/data-types/index)) — Número de consultas desde que se cargó el diccionario o desde el último reinicio correcto.
* `hit_rate` ([Float64](/es/reference/data-types/index)) — En los diccionarios de caché, el porcentaje de usos en los que el valor estaba en la caché.
* `found_rate` ([Float64](/es/reference/data-types/index)) — El porcentaje de usos en los que se encontró el valor.
* `element_count` ([UInt64](/es/reference/data-types/index)) — Número de elementos almacenados en el diccionario.
* `load_factor` ([Float64](/es/reference/data-types/index)) — Porcentaje de ocupación del diccionario (en un diccionario hash, el porcentaje de ocupación de la tabla hash).
* `source` ([String](/es/reference/data-types/index)) — Texto que describe la fuente de datos del diccionario.
* `lifetime_min` ([UInt64](/es/reference/data-types/index)) — Tiempo de vida mínimo del diccionario en memoria, tras el cual ClickHouse intenta recargarlo (si `invalidate&#95;query` está configurado, solo si ha cambiado). Se establece en segundos.
* `lifetime_max` ([UInt64](/es/reference/data-types/index)) — Tiempo de vida máximo del diccionario en memoria, tras el cual ClickHouse intenta recargarlo (si `invalidate&#95;query` está configurado, solo si ha cambiado). Se establece en segundos.
* `loading_start_time` ([DateTime](/es/reference/data-types/index)) — Hora de inicio de la carga del diccionario.
* `last_successful_update_time` ([DateTime](/es/reference/data-types/index)) — Hora de finalización de la carga o actualización del diccionario. Ayuda a supervisar problemas en los orígenes de los diccionarios e investigar sus causas.
* `error_count` ([UInt64](/es/reference/data-types/index)) — Número de errores desde la última carga correcta. Ayuda a supervisar problemas en los orígenes de los diccionarios e investigar sus causas.
* `loading_duration` ([Float32](/es/reference/data-types/index)) — Duración de la carga de un diccionario.
* `last_exception` ([String](/es/reference/data-types/index)) — Texto del error que se produce al crear o recargar el diccionario si no pudo crearse.
* `comment` ([String](/es/reference/data-types/index)) — Texto del comentario del diccionario.

{/*AUTOGENERATED_END*/}

<div id="example">
  ## Ejemplo
</div>

Configure el diccionario:

```sql
CREATE DICTIONARY dictionary_with_comment
(
    id UInt64,
    value String
)
PRIMARY KEY id
SOURCE(CLICKHOUSE(HOST 'localhost' PORT tcpPort() TABLE 'source_table'))
LAYOUT(FLAT())
LIFETIME(MIN 0 MAX 1000)
COMMENT 'The temporary dictionary';
```

Asegúrese de que el diccionario esté cargado.

```sql
SELECT * FROM system.dictionaries LIMIT 1 FORMAT Vertical;
```
