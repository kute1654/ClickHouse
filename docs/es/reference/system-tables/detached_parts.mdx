<div id="purpose">
  ## Descripción
</div>

Contiene información sobre las partes separadas de las tablas [MergeTree](/es/reference/engines/table-engines/mergetree-family/mergetree). La columna `reason` especifica por qué se separó la parte.

En el caso de las partes separadas por el usuario, el motivo está vacío. Estas partes pueden adjuntarse con el comando [ALTER TABLE ATTACH PARTITION|PART](/es/reference/statements/alter/partition#attach-partitionpart).

Para ver la descripción de las demás columnas, consulte [system.parts](/es/reference/system-tables/parts).

Si el nombre de la parte no es válido, los valores de algunas columnas pueden ser `NULL`. Estas partes pueden eliminarse con [ALTER TABLE DROP DETACHED PART](/es/reference/statements/alter/partition#drop-detached-partitionpart).

<div id="columns">
  ## Columnas
</div>

{/*AUTOGENERATED_START*/}

* `database` ([String](/es/reference/data-types/index)) — El nombre de la base de datos a la que pertenece esta parte.
* `table` ([String](/es/reference/data-types/index)) — El nombre de la tabla a la que pertenece esta parte.
* `partition_id` ([Nullable(String)](/es/reference/data-types/index)) — El identificador de la partición a la que pertenece esta parte.
* `name` ([String](/es/reference/data-types/index)) — El nombre de la parte.
* `bytes_on_disk` ([UInt64](/es/reference/data-types/index)) — El tamaño total en bytes de todos los archivos de la parte de datos.
* `modification_time` ([DateTime](/es/reference/data-types/index)) — La hora a la que se modificó el directorio que contiene la parte de datos. Esto suele corresponder al momento en que se ejecutó `detach`.
* `disk` ([String](/es/reference/data-types/index)) — El nombre del disco que almacena esta parte de datos.
* `path` ([String](/es/reference/data-types/index)) — La ruta en el disco al archivo de esta parte de datos.
* `reason` ([Nullable(String)](/es/reference/data-types/index)) — La explicación de por qué esta parte se separó.
* `min_block_number` ([Nullable(Int64)](/es/reference/data-types/index)) — El número mínimo de partes de datos que componen la parte actual después de la fusión.
* `max_block_number` ([Nullable(Int64)](/es/reference/data-types/index)) — El número máximo de partes de datos que componen la parte actual después de la fusión.
* `level` ([Nullable(UInt32)](/es/reference/data-types/index)) — Profundidad del árbol de fusiones. Cero significa que la parte actual se creó mediante `insert` y no por la fusión de otras partes.

{/*AUTOGENERATED_END*/}
