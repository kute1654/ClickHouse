Proporciona una interfaz similar a una tabla para las tablas de [Delta Lake](https://github.com/delta-io/delta) en Amazon S3, Azure Blob Storage o un sistema de archivos montado localmente, con soporte tanto para operaciones de lectura como de escritura (desde la v25.10)

<div id="syntax">
  ## Sintaxis
</div>

`deltaLake` es un alias de `deltaLakeS3`, admitido por motivos de compatibilidad.

```sql
deltaLake(url [,aws_access_key_id, aws_secret_access_key] [,format] [,structure] [,compression] [,extra_credentials])

deltaLakeS3(url [,aws_access_key_id, aws_secret_access_key] [,format] [,structure] [,compression] [,extra_credentials])

deltaLakeAzure(connection_string|storage_account_url, container_name, blobpath, [,account_name], [,account_key] [,format] [,compression_method])

deltaLakeLocal(path, [,format])
```

<div id="arguments">
  ## Argumentos
</div>

Los argumentos de esta funciГіn de tabla son los mismos que los de las funciones de tabla `s3`, `azureBlobStorage`, `HDFS` y `file`, respectivamente.
El argumento `format` se refiere al formato de los archivos de datos de la tabla Delta Lake.

Se puede usar un parГЎmetro opcional `extra_credentials` para pasar un `role_arn` para el acceso basado en roles en ClickHouse Cloud. Consulta [Secure S3](/es/products/cloud/guides/data-sources/accessing-s3-data-securely) para ver los pasos de configuraciГіn.

<div id="returned_value">
  ## Valor devuelto
</div>

Devuelve una tabla con la estructura especificada para leer o escribir datos en la tabla Delta Lake especificada.

  ## Ejemplos
</div>

<div id="reading-data">
  ### Lectura de datos
</div>

Considere una tabla en S3 ubicada en `https://clickhouse-public-datasets.s3.amazonaws.com/delta_lake/hits/`.
Para leer los datos de la tabla en ClickHouse, ejecute:

```sql title="Query"
SELECT
    URL,
    UserAgent
FROM deltaLake('https://clickhouse-public-datasets.s3.amazonaws.com/delta_lake/hits/')
WHERE URL IS NOT NULL
LIMIT 2
```

  ### InserciГіn de datos



<div id="virtual-columns">
  ## Columnas virtuales
</div>

* `_path` вҖ” Ruta del archivo. Tipo: `LowCardinality(String)`.
* `_file` вҖ” Nombre del archivo. Tipo: `LowCardinality(String)`.
* `_size` вҖ” TamaГұo del archivo en bytes. Tipo: `Nullable(UInt64)`. Si se desconoce el tamaГұo del archivo, el valor es `NULL`.
* `_time` вҖ” Hora de la Гәltima modificaciГіn del archivo. Tipo: `Nullable(DateTime)`. Si se desconoce la hora, el valor es `NULL`.
* `_etag` вҖ” El etag del archivo. Tipo: `LowCardinality(String)`. Si se desconoce el etag, el valor es `NULL`.
