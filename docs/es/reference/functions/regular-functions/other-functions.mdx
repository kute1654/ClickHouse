<div id="bar">
  ## bar
</div>

Introducido en: v1.1.0

Crea un grГЎfico de barras.
Dibuja una banda con un ancho proporcional a (x - min) y de width caracteres cuando x = max.
La banda se dibuja con una precisiГіn de un octavo de carГЎcter.

**Sintaxis**


* `width` вҖ” Opcional. El ancho de la barra en caracteres. El valor predeterminado es `80`. [`const (U)Int*`](/es/reference/data-types/int-uint) o [`const Float*`](/es/reference/data-types/float) o [`const Decimal`](/es/reference/data-types/decimal)

**Valor devuelto**

Devuelve una cadena que representa una barra de arte Unicode. [`String`](/es/reference/data-types/string)

**Ejemplo de uso**

**Requisitos previos**

Antes de evaluar modelos catboost, la biblioteca `libcatboostmodel.<so|dylib>` debe estar disponible. Consulta la [documentaciГіn de CatBoost](https://catboost.ai/docs/concepts/c-plus-plus-api_dynamic-c-pluplus-wrapper.html) para ver cГіmo compilarla.

```xml

...

...
```

<div id="flipCoordinates">
  ## flipCoordinates
</div>

Invierte las coordenadas x e y de los objetos geomГ©tricos. Esta operaciГіn intercambia la latitud y la longitud, lo que resulta Гәtil para convertir entre distintos sistemas de coordenadas o corregir el orden de las coordenadas.

Tipos admitidos: Point (Tuple(Float64, Float64)), Ring (Array(Point)), Polygon (Array(Ring)), MultiPolygon (Array(Polygon)), LineString (Array(Point)), MultiLineString (Array(LineString)) o Geometry (una variante que contiene cualquiera de estos tipos).

**Valor devuelto**

La geometrГӯa con las coordenadas invertidas. El tipo devuelto coincide con el tipo de entrada. [`Point`](/es/reference/data-types/geo#point) o [`Ring`](/es/reference/data-types/geo#ring) o [`Polygon`](/es/reference/data-types/geo#polygon) o [`MultiPolygon`](/es/reference/data-types/geo#multipolygon) o [`LineString`](/es/reference/data-types/geo#linestring) o [`MultiLineString`](/es/reference/data-types/geo#multilinestring) o [`Geometry`](/es/reference/data-types/geo)

**Sintaxis**

Valores aceptables: `nanoseconds`, `microseconds`, `milliseconds`, `seconds`, `minutes`, `hours`, `days`, `months`, `years`. Si el valor especificado explГӯcitamente es mayor que `maximum_unit`, se lanzarГЎ una excepciГіn. Valor predeterminado: `seconds` si `maximum_unit` es `seconds` o una unidad mayor; `nanoseconds` en caso contrario. [`const String`](/es/reference/data-types/string)

<div id="indexHint">
  ## indexHint
</div>

Introducido en: v1.1.0

Esta funciГіn estГЎ pensada para depuraciГіn e introspecciГіn.
Ignora su argumento y siempre devuelve 1.
Los argumentos no se evalГәan.

Durante el anГЎlisis del Гӯndice, se asume que el argumento de esta funciГіn no estГЎ envuelto en `indexHint`.
Esto le permite seleccionar datos en rangos del Гӯndice mediante la condiciГіn correspondiente, pero sin aplicar despuГ©s un filtrado por esa condiciГіn.
El Гӯndice de ClickHouse es disperso, y usar `indexHint` devolverГЎ mГЎs datos que especificar directamente la misma condiciГіn.

<Accordion title="ExplicaciГіn">
  Cuando ejecuta:

  ```sql
  SELECT * FROM test WHERE key = 123;
  ```

  ClickHouse hace dos cosas:

  1. Usa el Гӯndice para encontrar quГ© grГЎnulos (bloques de ~8192 filas) podrГӯan contener `key = 123`
  2. Lee esos grГЎnulos y los filtra fila por fila para devolver solo las filas donde `key = 123`

  AsГӯ que, aunque lea 8192 filas del disco, solo devuelve la Гәnica fila que realmente coincide.

  Con `indexHint`, cuando ejecuta:

  ```sql
  SELECT * FROM test WHERE indexHint(key = 123);
  ```

  ClickHouse hace solo una cosa:

  1. Usa el Гӯndice para encontrar quГ© grГЎnulos podrГӯan contener `key = 123` y devuelve todas las filas de esos grГЎnulos **sin** filtrarlas.

  Devuelve las 8192 filas, incluidas las filas donde `key = 456`, `key = 789`, etc. (Todo lo que haya quedado almacenado en el mismo grГЎnulo).
  `indexHint()` no sirve para mejorar el rendimiento. Sirve para depurar y entender cГіmo funciona el Гӯndice de ClickHouse:

  * ВҝQuГ© grГЎnulos selecciona mi condiciГіn?
  * ВҝCuГЎntas filas hay en esos grГЎnulos?
  * ВҝSe estГЎ usando mi Гӯndice de forma eficaz?
</Accordion>

Nota: No es posible optimizar una consulta con la funciГіn `indexHint`. La funciГіn `indexHint` no optimiza la consulta, ya que no proporciona ninguna informaciГіn adicional para el anГЎlisis de la consulta. Tener una expresiГіn dentro de la funciГіn `indexHint` no es en modo alguno mejor que no usar la funciГіn `indexHint`. La funciГіn `indexHint` solo puede usarse con fines de introspecciГіn y depuraciГіn, y no mejora el rendimiento. Si ve que alguien que no sea colaborador de ClickHouse usa `indexHint`, probablemente sea un error y deberГӯa eliminarlo.

**Sintaxis**

```sql

```

**Argumentos**

* `expression` вҖ” Cualquier expresiГіn para la selecciГіn del rango del Гӯndice. [`Expression`](/es/reference/data-types/special-data-types/expression)

**Valor devuelto**

Devuelve `1` en todos los casos. [`UInt8`](/es/reference/data-types/int-uint)

**Ejemplos**

**Ejemplo de uso con filtro por fecha**

<div id="obfuscateQuery">

<div id="sleep">

</div>

Introducido en: v1.1.0

Pausa la ejecuciГіn de una consulta durante el nГәmero de segundos especificado.
La funciГіn se utiliza principalmente para pruebas y depuraciГіn.

Por lo general, la funciГіn `sleep()` no debe usarse en entornos de producciГіn, ya que puede afectar negativamente al rendimiento de las consultas y a la capacidad de respuesta del sistema.
Sin embargo, puede resultar Гәtil en los siguientes casos:

1. **Pruebas**: Al probar o ejecutar benchmarks de ClickHouse, es posible que desee simular retrasos o introducir pausas para observar cГіmo se comporta el sistema en determinadas condiciones.

3. **SimulaciГіn**: En algunos casos, es posible que desee simular situaciones reales en las que se producen retrasos o pausas, como la latencia de red o las dependencias de sistemas externos.

<Warning>

</Warning>

**Sintaxis**

**Argumentos**

<div id="transform">
  ## transform
</div>

Introducido en: v1.1.0

Transforma un valor segГәn una asignaciГіn definida explГӯcitamente de unos elementos a otros.

Hay dos variantes de esta funciГіn:

* `transform(x, array_from, array_to, default)` - transforma `x` mediante arrays de asignaciГіn, con un valor predeterminado para los elementos que no coincidan
* `transform(x, array_from, array_to)` - realiza la misma transformaciГіn, pero devuelve el `x` original si no se encuentra ninguna coincidencia

La funciГіn busca `x` en `array_from` y devuelve el elemento correspondiente de `array_to` en el mismo Гӯndice.
Si `x` no se encuentra en `array_from`, devuelve el valor `default` (versiГіn de 4 parГЎmetros) o el `x` original (versiГіn de 3 parГЎmetros).
Si hay varios elementos coincidentes en `array_from`, devuelve el elemento correspondiente a la primera coincidencia.

Requisitos:

* `array_from` y `array_to` deben tener el mismo nГәmero de elementos
* Para la versiГіn de 4 parГЎmetros: `transform(T, Array(T), Array(U), U) -> U` donde `T` y `U` pueden ser tipos compatibles distintos
* Para la versiГіn de 3 parГЎmetros: `transform(T, Array(T), Array(T)) -> T` donde todos los tipos deben ser iguales

**Sintaxis**

```sql
transform(x, array_from, array_to[, default])
```

**Argumentos**

* `x` вҖ” Valor que se va a transformar. [`(U)Int*`](/es/reference/data-types/int-uint) o [`Decimal`](/es/reference/data-types/decimal) o [`Float*`](/es/reference/data-types/float) o [`String`](/es/reference/data-types/string) o [`Date`](/es/reference/data-types/date) o [`DateTime`](/es/reference/data-types/datetime)
* `array_from` вҖ” Array constante de valores en el que buscar coincidencias. [`Array((U)Int*)`](/es/reference/data-types/array) o [`Array(Decimal)`](/es/reference/data-types/array) o [`Array(Float*)`](/es/reference/data-types/array) o [`Array(String)`](/es/reference/data-types/array) o [`Array(Date)`](/es/reference/data-types/array) o [`Array(DateTime)`](/es/reference/data-types/array)
* `array_to` вҖ” Array constante de valores que se devolverГЎn para las coincidencias correspondientes de `array_from`. [`Array((U)Int*)`](/es/reference/data-types/array) o [`Array(Decimal)`](/es/reference/data-types/array) o [`Array(Float*)`](/es/reference/data-types/array) o [`Array(String)`](/es/reference/data-types/array) o [`Array(Date)`](/es/reference/data-types/array) o [`Array(DateTime)`](/es/reference/data-types/array)
* `default` вҖ” Opcional. Valor que se devolverГЎ si `x` no se encuentra en `array_from`. Si se omite, devuelve x sin cambios. [`(U)Int*`](/es/reference/data-types/int-uint) o [`Decimal`](/es/reference/data-types/decimal) o [`Float*`](/es/reference/data-types/float) o [`String`](/es/reference/data-types/string) o [`Date`](/es/reference/data-types/date) o [`DateTime`](/es/reference/data-types/datetime)

**Valor devuelto**

Devuelve el valor correspondiente de `array_to` si x coincide con un elemento de `array_from`; de lo contrario, devuelve default (si se proporciona) o x (si no se proporciona default). [`Any`](/es/reference/data-types/index)

**Ejemplos**

```sql title=Query
FROM test.hits
WHERE SearchEngineID != 0
```
