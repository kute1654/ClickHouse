<div id="h3-index">
  ## índice H3
</div>

[H3](https://h3geo.org/) es un sistema de indexación geográfica en el que la superficie de la Tierra se divide en una cuadrícula de celdas hexagonales uniformes. Este sistema es jerárquico; es decir, cada hexágono del nivel superior (&quot;padre&quot;) puede dividirse en siete hexágonos uniformes, pero más pequeños (&quot;hijos&quot;), y así sucesivamente.

El nivel de la jerarquía se denomina `resolution` y puede tomar un valor de `0` a `15`, donde `0` es el nivel `base`, con las celdas más grandes y menos detalladas.

Un par de latitud y longitud puede transformarse en un índice H3 de 64 bits, que identifica una celda de la cuadrícula.

El índice H3 se utiliza principalmente para agrupar ubicaciones y realizar otras manipulaciones geoespaciales.

La descripción completa del sistema H3 está disponible en [el sitio de Uber Engineering](https://www.uber.com/blog/h3/).

<div id="h3isvalid">
  ## h3IsValid
</div>

Comprueba si el número es un índice [H3](#h3-index) válido.

**Sintaxis**

```sql
h3IsValid(h3index)
```

**Parámetro**

* `h3index` — Número de índice hexagonal. [UInt64](/es/reference/data-types/int-uint).

**Valores devueltos**

* 1 — El número es un índice H3 válido. [UInt8](/es/reference/data-types/int-uint).
* 0 — El número no es un índice H3 válido. [UInt8](/es/reference/data-types/int-uint).

**Ejemplo**

```sql title="Query"
SELECT h3IsValid(630814730351855103) AS h3IsValid;
```

```text title="Response"
┌─h3IsValid─┐
│         1 │
└───────────┘
```

<div id="h3getresolution">
  ## h3GetResolution
</div>

Define la resolución del índice [H3](#h3-index) dado.

**Sintaxis**

```sql
h3GetResolution(h3index)
```

**Parámetro**

* `h3index` — Número de índice hexagonal. [UInt64](/es/reference/data-types/int-uint).

**Valores devueltos**

* Resolución del índice. Rango: `[0, 15]`. [UInt8](/es/reference/data-types/int-uint).
* Si el índice no es válido, la función devuelve un valor aleatorio. Use [h3IsValid](#h3isvalid) para verificar el índice. [UInt8](/es/reference/data-types/int-uint).

**Ejemplo**

```sql title="Query"
SELECT h3GetResolution(639821929606596015) AS resolution;
```

```text title="Response"
┌─resolution─┐
│         14 │
└────────────┘
```

<div id="geotoh3">
  ## geoToH3
</div>

Devuelve el índice [H3](#h3-index) del punto `(lat, lon)` con la resolución especificada.

**Sintaxis**

```sql
geoToH3(lat, lon, resolution)
```

**Argumentos**

* `lat` — Latitud. [Float64](/es/reference/data-types/float).
* `lon` — Longitud. [Float64](/es/reference/data-types/float).
* `resolution` — Resolución del índice. Rango: `[0, 15]`. [UInt8](/es/reference/data-types/int-uint).

**Valores devueltos**

* Número de índice hexagonal. [UInt64](/es/reference/data-types/int-uint).
* 0 en caso de error. [UInt64](/es/reference/data-types/int-uint).

Nota: En ClickHouse v25.4 o anteriores, `geoToH3()` toma los valores en el orden `(lon, lat)`. A partir de ClickHouse v25.5, los valores de entrada están en el orden `(lat, lon)`. El comportamiento anterior puede restaurarse mediante la configuración `geotoh3_argument_order = 'lon_lat'`.

**Ejemplo**

```sql title="Query"
SELECT geoToH3(55.71290588, 37.79506683, 15) AS h3Index;
```

```text title="Response"
┌────────────h3Index─┐
│ 644325524701193974 │
└────────────────────┘
```

<div id="h3polygontocellswithcontainment">
  ## h3PolygonToCellsWithContainment
</div>

Devuelve los hexágonos (con la resolución especificada) que cubren la geometría proporcionada, ya sea un anillo, un polígono o un multipolígono, utilizando modos experimentales de contención de H3.

Esta función asigna los valores de `flags` a los modos de contención de H3:

* `0` — `CONTAINMENT_CENTER`
* `1` — `CONTAINMENT_FULL`
* `2` — `CONTAINMENT_OVERLAPPING`
* `3` — `CONTAINMENT_OVERLAPPING_BBOX`

**Sintaxis**

```sql
h3PolygonToCellsWithContainment(geometry, resolution, flags)
```

**Argumentos**

* `geometry` puede ser uno de los siguientes [Geo Data Types](/es/reference/data-types/geo) o de sus tipos primitivos subyacentes:
  * [Ring](/es/reference/data-types/geo#ring)
  * [Polygon](/es/reference/data-types/geo#polygon)
  * [MultiPolygon](/es/reference/data-types/geo#multipolygon)
* `resolution` — Resolución del índice. Rango: `[0, 15]`. [UInt8](/es/reference/data-types/int-uint).
* `flags` — Modo de contención. Rango: `[0, 3]`. Use literales enteros como `0`, `1`, `2` y `3`, o valores [UInt32](/es/reference/data-types/int-uint) (por ejemplo, `toUInt32(2)`). Los valores se pasan a la API de H3 como `UInt32`. Los demás tipos enteros nativos se convierten mediante una conversión exacta.

**Valores devueltos**

* Array de índices H3 según el modo de contención seleccionado. [Array](/es/reference/data-types/array)([UInt64](/es/reference/data-types/int-uint)).

**Ejemplo**

```sql title="Query"
SELECT
    h3PolygonToCellsWithContainment([(-122.4089866999972145, 37.813318999983238), (-122.3544736999993603, 37.7198061999978478), (-122.4798767000009008, 37.8151571999998453)], 7, 0) AS center_cells,
```
