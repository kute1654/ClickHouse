<div id="overview">
  ## DescripciГіn general
</div>

<Note>
  Las funciones mencionadas en esta secciГіn estГЎn optimizadas para ofrecer el mГЎximo rendimiento y, en su mayor parte, no siguen el estГЎndar RFC-3986.
  Las funciones que implementan RFC-3986 llevan `RFC` al final del nombre y, por lo general, son mГЎs lentas.
</Note>

Por lo general, puedes usar las variantes de funciГіn sin `RFC` al trabajar con dominios registrados pГәblicamente que no contienen ni cadenas de informaciГіn de usuario ni sГӯmbolos `@`.
La tabla siguiente detalla quГ© sГӯmbolos de una URL pueden (`вң”`) o no pueden (`вң—`) analizarse con las respectivas variantes `RFC` y sin `RFC`:

| SГӯmbolo     | non-`RFC` | `RFC` |   |
| ----------- | --------- | ----- | - |
| &#39; &#39; | вң—         | вң—     |   |
| \t          | вң—         | вң—     |   |
| &lt;        | вң—         | вң—     |   |
| &gt;        | вң—         | вң—     |   |
| %           | вң—         | вң”*    |   |
| &#123;      | вң—         | вң—     |   |
| &#125;      | вң—         | вң—     |   |
|             |           | вң—     | вң— |
| \\  | вң—         | вң—     |   |
| ^           | вң—         | вң—     |   |
| ~           | вң—         | вң”*    |   |
| [           | вң—         | вң—     |   |
| ]           | вң—         | вң”     |   |
| ;           | вң—         | вң”*    |   |
| =           | вң—         | вң”*    |   |
| &amp;       | вң—         | вң”*    |   |

Los sГӯmbolos marcados con `*` son subdelimitadores en RFC 3986 y estГЎn permitidos en la informaciГіn de usuario despuГ©s del sГӯmbolo `@`.

Hay dos tipos de funciones de URL:

* Funciones que extraen partes de una URL. Si la parte correspondiente no estГЎ presente en una URL, se devuelve una cadena vacГӯa.
* Funciones que eliminan una parte de una URL. Si la URL no tiene nada parecido, la URL permanece sin cambios.

</Note>

<div id="URLHierarchy">
  ## URLHierarchy
</div>

Introducido en: v1.1.0

Devuelve un array que contiene la URL truncada al final en los sГӯmbolos `/`, `?` y `#` de la ruta y la cadena de consulta. Los caracteres separadores consecutivos se consideran uno solo. El resultado incluye el protocolo y el host como primer elemento, seguido de rutas cada vez mГЎs largas que forman una jerarquГӯa.

**Sintaxis**

```sql
URLHierarchy(url)
```

**Argumentos**

* `url` вҖ” La URL que se debe procesar. [`String`](/es/reference/data-types/string)

**Valor devuelto**

Devuelve un array de URL cada vez mГЎs largas que forman una jerarquГӯa. [`Array(String)`](/es/reference/data-types/array)

**Ejemplos**

**Uso bГЎsico**

```sql title=Query
SELECT URLHierarchy('https://example.com/a/b?c=1')
```

```response title=Response
['https://example.com/','https://example.com/a/','https://example.com/a/b','https://example.com/a/b?c=1']
```

<div id="URLPathHierarchy">
  ## URLPathHierarchy
</div>

Introducido en: v1.1.0

Devuelve un array que contiene el componente de ruta de la URL, truncado al final por los sГӯmbolos `/`, `?` y `#`. A diferencia de `URLHierarchy`, el resultado no incluye el protocolo ni el host; empieza en la ruta. Los caracteres separadores consecutivos se cuentan como uno.

**Sintaxis**

```sql

```

**Argumentos**

* `url` вҖ” La URL que se va a procesar. [`String`](/es/reference/data-types/string)

**Valor devuelto**

Devuelve un array de componentes de la ruta de la URL cada vez mГЎs largos que forman una jerarquГӯa. [`Array(String)`](/es/reference/data-types/array)

**Ejemplos**

**Uso bГЎsico**

```sql title=Query

```

