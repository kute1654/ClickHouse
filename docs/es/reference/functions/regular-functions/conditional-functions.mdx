<div id="case-statement">
  ### sentencia CASE
</div>

La expresiГіn CASE en ClickHouse ofrece una lГіgica condicional similar al operador CASE de SQL. EvalГәa las condiciones y devuelve valores en funciГіn de la primera condiciГіn que se cumple.

ClickHouse admite dos formas de CASE:

1. `CASE WHEN ... THEN ... ELSE ... END`
   <br />
   Esta forma ofrece total flexibilidad y se implementa internamente mediante la funciГіn [multiIf](/es/reference/functions/regular-functions/conditional-functions#multiIf). Cada condiciГіn se evalГәa de forma independiente, y las expresiones pueden incluir valores no constantes.

```sql

    number,
    CASE
        WHEN number % 2 = 0 THEN number + 1
        WHEN number % 2 = 1 THEN number * 10
        ELSE number
    END AS result
FROM system.numbers
WHERE number < 5;

-- se traduce a

    number,
    multiIf((number % 2) = 0, number + 1, (number % 2) = 1, number * 10, number) AS result
```

<div id="caveats">
  #### Consideraciones
</div>

ClickHouse determina el tipo de resultado de una expresiГіn CASE (o su equivalente interno, como `multiIf`) antes de evaluar cualquier condiciГіn. Esto es importante cuando las expresiones de retorno difieren en el tipo, por ejemplo, con distintas zonas horarias o tipos numГ©ricos.

* El tipo de resultado se elige en funciГіn del tipo compatible mГЎs amplio entre todas las ramas.
* Una vez elegido este tipo, todas las demГЎs ramas se convierten implГӯcitamente a Г©l, aunque su lГіgica nunca llegue a ejecutarse en tiempo de ejecuciГіn.
* En tipos como DateTime64, donde la zona horaria forma parte de la firma del tipo, esto puede dar lugar a comportamientos inesperados: la primera zona horaria encontrada puede usarse para todas las ramas, incluso cuando otras ramas especifican zonas horarias diferentes.

Por ejemplo, a continuaciГіn todas las filas devuelven la marca temporal en la zona horaria de la primera rama coincidente, es decir, `Asia/Kolkata`

```sql

    CASE

FROM system.numbers
WHERE number < 3;

-- se traduce a

SELECT
    number,
    multiIf(number = 0, fromUnixTimestamp64Milli(0, 'Asia/Kolkata'), number = 1, fromUnixTimestamp64Milli(0, 'America/Los_Angeles'), fromUnixTimestamp64Milli(0, 'UTC')) AS tz
FROM system.numbers
WHERE number < 3

в”Ңв”Җnumberв”Җв”¬в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җtzв”Җв”җ

в”Ӯ      1 в”Ӯ 1970-01-01 05:30:00.000 в”Ӯ
в”Ӯ      2 в”Ӯ 1970-01-01 05:30:00.000 в”Ӯ
в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҙв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ
```

<div id="clamp">
  ## clamp
</div>

Introducido en: v24.5.0

Limita un valor para que se mantenga dentro de los lГӯmites mГӯnimo y mГЎximo especificados.

<div id="greatest">
  ## greatest
</div>

Introducido en: v1.1.0

Devuelve el valor mГЎximo entre los argumentos.
Los argumentos `NULL` se ignoran.

* Para arrays, devuelve el array lexicogrГЎficamente mayor.
* Para los tipos `DateTime`, el tipo del resultado se amplГӯa al tipo mГЎs grande (p. ej., `DateTime64` si se mezcla con `DateTime32`).

<Info>
  **Use la configuraciГіn `least_greatest_legacy_null_behavior` para cambiar el comportamiento de `NULL`**

  La versiГіn [24.12](/es/resources/changelogs/oss/2024#a-id2412a-clickhouse-release-2412-2024-12-19) introdujo un cambio incompatible con versiones anteriores por el que los valores `NULL` se ignoran, mientras que antes devolvГӯa `NULL` si uno de los argumentos era `NULL`.

**Valor devuelto**

<div id="if">
  ## if
</div>

Introducido en: v1.1.0

Realiza una bifurcaciГіn condicional.

* Si la condiciГіn `cond` se evalГәa como un valor distinto de cero, la funciГіn devuelve el resultado de la expresiГіn `then`.
* Si `cond` se evalГәa como cero o NULL, se devuelve el resultado de la expresiГіn `else`.

La configuraciГіn [`short_circuit_function_evaluation`](/es/reference/settings/session-settings#short_circuit_function_evaluation) controla si se utiliza la evaluaciГіn de cortocircuito.

Si estГЎ habilitada, la expresiГіn `then` se evalГәa solo en las filas en las que `cond` es verdadero, y la expresiГіn `else`, en las que `cond` es falso.

Por ejemplo, con la evaluaciГіn de cortocircuito, no se produce ninguna excepciГіn por divisiГіn entre cero al ejecutar la siguiente consulta:

```sql
SELECT if(number = 0, 0, intDiv(42, number)) FROM numbers(10)
```

`then` y `else` deben ser de tipo similar.

**Sintaxis**

```sql
if(cond, then, else)
```

**Argumentos**

* `cond` вҖ” La condiciГіn evaluada. [`UInt8`](/es/reference/data-types/int-uint) o [`Nullable(UInt8)`](/es/reference/data-types/nullable) o [`NULL`](/es/reference/syntax#null)
* `then` вҖ” La expresiГіn que se devuelve si `cond` es verdadera. - `else` вҖ” La expresiГіn que se devuelve si `cond` es falsa o `NULL`.

**Valor devuelto**

El resultado de las expresiones `then` o `else`, segГәn la condiciГіn `cond`.

**Ejemplos**

**Ejemplo de uso**

```sql title=Query
SELECT if(1, 2 + 2, 2 + 6) AS res;
```

```response title=Response
в”Ӯ   4 в”Ӯ
```

<div id="multiIf">
  ## multiIf
</div>

Introducido en: v1.1.0

Permite escribir el operador [`CASE`](/es/reference/operators/index#conditional-expression) de forma mГЎs compacta en la consulta.
EvalГәa cada condiciГіn en orden. Para la primera condiciГіn que sea verdadera (distinta de cero y no `NULL`), devuelve el valor de la rama correspondiente.
Si ninguna de las condiciones es verdadera, devuelve el valor de `else`.

El ajuste [`short_circuit_function_evaluation`](/es/reference/settings/session-settings#short_circuit_function_evaluation) controla
si se utiliza la evaluaciГіn de cortocircuito. Si estГЎ habilitada, la expresiГіn `then_i` se evalГәa solo en las filas donde
`((NOT cond_1) AND ... AND (NOT cond_{i-1}) AND cond_i)` es verdadera.

Por ejemplo, con la evaluaciГіn de cortocircuito, no se lanza ninguna excepciГіn por divisiГіn entre cero al ejecutar la siguiente consulta:

```sql
SELECT multiIf(number = 2, intDiv(1, number), number = 5) FROM numbers(10)
```

Todas las expresiones de las ramas y de else deben tener un supertipo comГәn. Las condiciones `NULL` se tratan como falsas.

**Sintaxis**

```sql
multiIf(cond_1, then_1, cond_2, then_2, ..., else)
```

**Alias**: `caseWithoutExpression`, `caseWithoutExpr`

**Argumentos**

* `cond_N` вҖ” La enГ©sima condiciГіn evaluada que determina si se devuelve `then_N`. [`UInt8`](/es/reference/data-types/int-uint) o [`Nullable(UInt8)`](/es/reference/data-types/nullable) o [`NULL`](/es/reference/syntax#null)
* `then_N` вҖ” El resultado de la funciГіn cuando `cond_N` es verdadero. - `else` вҖ” El resultado de la funciГіn si ninguna condiciГіn es verdadera.

**Valor devuelto**

Devuelve el resultado de `then_N` para el `cond_N` que se cumpla; en caso contrario, devuelve el valor de `else`.

**Ejemplos**

**Ejemplo de uso**

```sql title=Query
CREATE TABLE LEFT_RIGHT (left Nullable(UInt8), right Nullable(UInt8)) ENGINE = Memory;
INSERT INTO LEFT_RIGHT VALUES (NULL, 4), (1, 3), (2, 2), (3, 1), (4, NULL);

SELECT
    left,
    right,
    multiIf(left < right, 'left is smaller', left > right, 'left is greater', left = right, 'Both equal', 'Null value') AS result
FROM LEFT_RIGHT;
```
