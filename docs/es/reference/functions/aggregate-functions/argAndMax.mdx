<div id="argAndMax">
  ## argAndMax
</div>

Introducido en: v1.1.0

Calcula los valores `arg` y `val` correspondientes al valor mГЎximo de `val`.
Si hay varias filas con el mismo `val` mГЎximo, no es determinista cuГЎl de los `arg` y `val` asociados se devuelve.
Tanto `arg` como `max` se comportan como [funciones de agregado](/es/reference/functions/aggregate-functions/index): ambos [omiten `Null`](/es/reference/functions/aggregate-functions/index#null-processing) durante el procesamiento y devuelven valores distintos de `Null` si hay valores distintos de `Null` disponibles.

<Note>
  La Гәnica diferencia con `argMax` es que `argAndMax` devuelve tanto el argumento como el valor.
</Note>

**Sintaxis**

```sql
argAndMax(arg, val)
```

**Argumentos**

* `arg` вҖ” Argumento del que se debe encontrar el valor mГЎximo. [`const String`](/es/reference/data-types/string)
* `val` вҖ” El valor mГЎximo. [`(U)Int8/16/32/64`](/es/reference/data-types/int-uint) o [`Float*`](/es/reference/data-types/float) o [`Date`](/es/reference/data-types/date) o [`DateTime`](/es/reference/data-types/datetime) o [`Tuple`](/es/reference/data-types/tuple)

**Valor devuelto**

Devuelve una tupla que contiene el valor de `arg` correspondiente al valor mГЎximo de `val` y el valor mГЎximo de `val`. [`Tuple`](/es/reference/data-types/tuple)
