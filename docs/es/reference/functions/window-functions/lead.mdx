Devuelve un valor evaluado en la fila situada `offset` filas despuГ©s de la fila actual dentro del marco ordenado.
Esta funciГіn es similar a [`leadInFrame`](/es/reference/functions/window-functions/leadInFrame), pero siempre utiliza el marco `ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING`.

**Sintaxis**

```sql
lead(x[, offset[, default]])
  OVER ([[PARTITION BY grouping_column] [ORDER BY sorting_column]] | [window_name])
FROM table_name
WINDOW window_name as ([[PARTITION BY grouping_column] [ORDER BY sorting_column])
```

Para obtener mГЎs informaciГіn sobre la sintaxis de las funciones de ventana, consulte: [Funciones de ventana - Sintaxis](/es/reference/functions/window-functions/index#syntax).

**ParГЎmetros**

* `x` вҖ” Nombre de la columna.
* `offset` вҖ” Desplazamiento que se aplica. [(U)Int*](/es/reference/data-types/int-uint). (Opcional; `1` de forma predeterminada).
* `default` вҖ” Valor que se devolverГЎ si la fila calculada supera los lГӯmites del marco de ventana. (Opcional: el valor predeterminado del tipo de columna si se omite).

**Valor devuelto**

* valor evaluado en la fila situada `offset` filas despuГ©s de la fila actual dentro del marco ordenado.

**Ejemplo**

Este ejemplo analiza [datos histГіricos](https://www.kaggle.com/datasets/sazidthe1/nobel-prize-data) de los ganadores del Premio Nobel y usa la funciГіn `lead` para devolver una lista de ganadores sucesivos en la categorГӯa de fГӯsica.

```sql title="Query"
CREATE OR REPLACE VIEW nobel_prize_laureates
AS SELECT *
FROM file('nobel_laureates_data.csv');
```

```sql title="Query"
SELECT
    fullName,
    lead(year, 1, year) OVER (PARTITION BY category ORDER BY year ASC
      ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING
    ) AS year,
    category,
    motivation
FROM nobel_prize_laureates
WHERE category = 'physics'
ORDER BY year DESC
LIMIT 9
```
