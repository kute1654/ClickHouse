Devuelve informaciГіn sobre las columnas de la tabla.

**Sintaxis**

```sql
DESC|DESCRIBE TABLE [db.]table [INTO OUTFILE filename] [FORMAT format]
```

La sentencia `DESCRIBE` devuelve una fila por cada columna de la tabla con los siguientes valores de tipo [String](/es/reference/data-types/string):

* `name` вҖ” El nombre de la columna.
* `type` вҖ” El tipo de la columna.
* `default_type` вҖ” Una clГЎusula utilizada en la [expresiГіn predeterminada](/es/reference/statements/create/table) de la columna: `DEFAULT`, `MATERIALIZED` o `ALIAS`. Si no hay ninguna expresiГіn predeterminada, se devuelve una cadena vacГӯa.
* `default_expression` вҖ” Una expresiГіn especificada despuГ©s de la clГЎusula `DEFAULT`.
* `comment` вҖ” Un [comentario de columna](/es/reference/statements/alter/column#comment-column).
* `codec_expression` вҖ” Un [codec](/es/reference/statements/create/table#column_compression_codec) aplicado a la columna.
* `ttl_expression` вҖ” Una expresiГіn [TTL](/es/reference/engines/table-engines/mergetree-family/mergetree#table_engine-mergetree-ttl).
* `is_subcolumn` вҖ” Un indicador cuyo valor es `1` para las subcolumnas internas. Solo se incluye en el resultado si la descripciГіn de subcolumnas estГЎ habilitada mediante la configuraciГіn [describe&#95;include&#95;subcolumns](/es/reference/settings/session-settings#describe_include_subcolumns).

Todas las columnas de las estructuras de datos [Nested](/es/reference/data-types/nested-data-structures/index) se describen por separado. El nombre de cada columna lleva como prefijo el nombre de la columna padre y un punto.

Para mostrar las subcolumnas internas de otros tipos de datos, use la configuraciГіn [describe&#95;include&#95;subcolumns](/es/reference/settings/session-settings#describe_include_subcolumns).

**Ejemplo**

```sql title="Query"
CREATE TABLE describe_example (
    id UInt64, text String DEFAULT 'unknown' CODEC(ZSTD),
    user Tuple (name String, age UInt8)
) ENGINE = MergeTree() ORDER BY id;

DESCRIBE TABLE describe_example;
DESCRIBE TABLE describe_example SETTINGS describe_include_subcolumns=1;
```
