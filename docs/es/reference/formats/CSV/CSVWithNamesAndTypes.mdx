<div id="description">
  ## Descripción
</div>

También imprime dos filas de encabezado con nombres de columnas y tipos, de forma similar a [TabSeparatedWithNamesAndTypes](/es/reference/formats/TabSeparated/TabSeparatedWithNamesAndTypes).

<div id="example-usage">
  ## Ejemplo de uso
</div>

<div id="inserting-data">
  ### Inserción de datos
</div>

<Tip>
  A partir de la [versión](https://github.com/ClickHouse/ClickHouse/releases) 23.1, ClickHouse detecta automáticamente los encabezados en los archivos CSV al usar el formato `CSV`, por lo que no es necesario usar `CSVWithNames` ni `CSVWithNamesAndTypes`.
</Tip>

Usando el siguiente archivo CSV, llamado `football_types.csv`:


Crear una tabla:

```sql
CREATE TABLE football
(
    `date` Date,
    `season` Int16,
    `home_team` LowCardinality(String),
    `away_team` LowCardinality(String),
    `home_team_goals` Int8,
    `away_team_goals` Int8
)
ENGINE = MergeTree
ORDER BY (date, home_team);
```

Inserte datos usando el formato `CSVWithNamesAndTypes`:

```sql
INSERT INTO football FROM INFILE 'football_types.csv' FORMAT CSVWithNamesAndTypes;
```

<div id="reading-data">
  ### Lectura de datos
</div>

Lea los datos con el formato `CSVWithNamesAndTypes`:

```sql
SELECT *
FROM football
FORMAT CSVWithNamesAndTypes
```

<div id="format-settings">
  ## Configuración de formato
</div>

<Note>
  Si la configuración [input&#95;format&#95;with&#95;names&#95;use&#95;header](/es/reference/settings/formats#input_format_with_names_use_header) está establecida en `1`,
  las columnas de los datos de entrada se asignarán a las columnas de la tabla según sus nombres; las columnas con nombres desconocidos se omitirán si la configuración [input&#95;format&#95;skip&#95;unknown&#95;fields](/es/reference/settings/formats#input_format_skip_unknown_fields) está establecida en `1`.
  De lo contrario, se omitirá la primera fila.
</Note>

<Note>
  Si la configuración [input&#95;format&#95;with&#95;types&#95;use&#95;header](/es/reference/settings/formats#input_format_with_types_use_header) está establecida en `1`,
  los tipos de los datos de entrada se compararán con los tipos de las columnas correspondientes de la tabla. De lo contrario, se omitirá la segunda fila.
</Note>
