<div id="description">
  ## DescripciГіn
</div>

En este formato, todos los datos se representan como un Гәnico objeto JSON, y cada fila se representa como un campo independiente de ese objeto, de forma similar al formato [`JSONEachRow`](/es/reference/formats/JSON/JSONEachRow).

<div id="example-usage">
  ## Ejemplo de uso
</div>

<div id="basic-example">
  ### Ejemplo bГЎsico
</div>

Dado el siguiente JSON:

```json
{
  "row_1": {"num": 42, "str": "hello", "arr":  [0,1]},
  "row_2": {"num": 43, "str": "hello", "arr":  [0,1,2]},
  "row_3": {"num": 44, "str": "hello", "arr":  [0,1,2,3]}
}
```

Para usar el nombre de un objeto como valor de una columna, puede utilizar la SETTING especial [`format_json_object_each_row_column_for_object_name`](/es/reference/settings/formats#format_json_object_each_row_column_for_object_name).
El valor de esta SETTING se establece en el nombre de una columna, que se utiliza como clave JSON para una fila en el objeto resultante.

<div id="json-inserting-data">
  ### Insertar datos
</div>

```sql title="Query"
INSERT INTO UserActivity FORMAT JSONEachRow {"PageViews":5, "UserID":"4324182021466249494", "Duration":146,"Sign":-1} {"UserID":"4324182021466249494","PageViews":6,"Duration":185,"Sign":1}
```

ClickHouse permite:

* Cualquier orden de los pares clave-valor dentro del objeto.
* Omitir algunos valores.

ClickHouse ignora los espacios entre los elementos y las comas despuГ©s de los objetos. Puede pasar todos los objetos en una sola lГӯnea. No es necesario separarlos con saltos de lГӯnea.

<div id="omitted-values-processing">
  #### Procesamiento de los valores omitidos
</div>

ClickHouse sustituye los valores omitidos por los valores predeterminados de los [tipos de datos](/es/reference/data-types/index) correspondientes.

Si se especifica `DEFAULT expr`, ClickHouse aplica reglas de sustituciГіn diferentes segГәn la SETTING [input&#95;format&#95;defaults&#95;for&#95;omitted&#95;fields](/es/reference/settings/formats#input_format_defaults_for_omitted_fields).

Considere la siguiente tabla:

```sql title="Query"
CREATE TABLE IF NOT EXISTS example_table
(
    x UInt32,
    a DEFAULT x * 2
) ENGINE = Memory;
```

* Si `input_format_defaults_for_omitted_fields = 0`, el valor predeterminado de `x` y `a` es `0` (el valor predeterminado del tipo de dato `UInt32`).
* Si `input_format_defaults_for_omitted_fields = 1`, el valor predeterminado de `x` es `0`, pero el valor predeterminado de `a` es `x * 2`.

<Note>
  Al insertar datos con `input_format_defaults_for_omitted_fields = 1`, ClickHouse consume mГЎs recursos de cГіmputo que con la inserciГіn usando `input_format_defaults_for_omitted_fields = 0`.
</Note>

<div id="jsoneachrow-nested">
  ### Uso de estructuras anidadas
</div>

Si tiene una tabla con columnas del tipo de dato [`Nested`](/es/reference/data-types/nested-data-structures/index), puede insertar datos JSON con la misma estructura. Habilite esta funciГіn con la SETTING [input&#95;format&#95;import&#95;nested&#95;json](/es/reference/settings/formats#input_format_import_nested_json).

Por ejemplo, considere la siguiente tabla:

```sql title="Query"
CREATE TABLE json_each_row_nested (n Nested (s String, i Int32) ) ENGINE = Memory
```

Como puede ver en la descripciГіn del tipo de dato `Nested`, ClickHouse trata cada componente de la estructura anidada como una columna independiente (`n.s` y `n.i` para nuestra tabla). Puede insertar datos de la siguiente manera:

```sql title="Query"
INSERT INTO json_each_row_nested FORMAT JSONEachRow {"n.s": ["abc", "def"], "n.i": [1, 23]}
```

Para insertar datos como un objeto JSON jerГЎrquico, establezca [`input_format_import_nested_json=1`](/es/reference/settings/formats#input_format_import_nested_json).

```json
{
    "n": {
        "s": ["abc", "def"],
        "i": [1, 23]
    }
}
```

Sin esta SETTING, ClickHouse genera una excepciГіn.

```sql title="Query"
SELECT name, value FROM system.settings WHERE name = 'input_format_import_nested_json'
```
