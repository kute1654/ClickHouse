Este tutorial le guía paso a paso para desplegar ClickHouse en nodos bare-metal usando Terraform y Ansible en instancias de AWS EC2 con RKE2 como distribución de Kubernetes. Al final, tendrá un clúster de RKE2 en ejecución con ClickHouse desplegado mediante el operator.

Para consultar las especificaciones detalladas de infraestructura, vea [reference/infrastructure-requirements.md](/es/products/clickhouse-private/reference/infrastructure-requirements-reference).

***

* credentials de AWS IAM con permisos de EC2, VPC y S3

<Steps>
  <Step>
    ## Fase 1: Instalar Terraform

    ```text
    dnf install -y unzip git
    ```

    ```bash
    curl -O https://releases.hashicorp.com/terraform/1.13.2/terraform_1.13.2_linux_amd64.zip
    ```

    ```text
    unzip terraform_1.13.2_linux_amd64.zip
    ```

    ```bash
    sudo cp terraform /usr/local/bin
    ```

    Verifica la instalación:

    ```text
    terraform version
    ```

    Resultado esperado:

    ```text
    Terraform v1.13.2
    on linux_amd64
    ```

    ***
  </Step>

  <Step>
    ## Fase 2: Descargar scripts de ejemplo

    Descarga y extrae los scripts de ejemplo proporcionados por ClickHouse:

    ```text
    unzip example-scripts.zip
    cd ./bare-metal
    ```

    El archivo incluye módulos de Terraform, playbooks de Ansible, plantillas de inventario y archivos de valores de Helm.

    ***
  </Step>

  <Step>
    ## Fase 3: Configurar las credenciales de AWS

    Cree un archivo de credenciales con la clave de acceso y la clave secreta de AWS IAM:

    ```bash
    cat > aws-creds.sh <<EOF
    export AWS_ACCESS_KEY_ID=<Your AWS ACCESS KEY ID>
    export AWS_SECRET_ACCESS_KEY=<Your AWS SECRET ACCESS KEY>
    export AWS_REGION=us-east-2
    EOF
    ```

    Reemplace `<Your AWS ACCESS KEY ID>` y `<Your AWS SECRET ACCESS KEY>` por los valores de la consola de AWS IAM.

    Cargue las credenciales:

    ```bash
    source ./aws-creds.sh
    ```

    ***
  </Step>

  <Step>
    ## Fase 4: Inicializar y aplicar Terraform

    Inicializa el módulo de Terraform para RKE2. Añade `-migrate-state` si mueves el estado del entorno local a S3.

    ```text
    terraform -chdir=modules/rke2 init \
    -backend-config="encrypt=true" \
    -backend-config="region=us-east-2" \
    -backend-config="bucket=chb-tfstate-359774249275" \
    -backend-config="use_lockfile=true" \
    -backend-config="key=chb/rke2/terraform.tfstate"
    ```

    Aplica la configuración de Terraform para aprovisionar la infraestructura:

    ```text
    terraform -chdir=modules/rke2 apply \
    -var-file=../../vars/common.tfvars \
    -var-file=../../vars/instances.tfvars \
    -var-file=../../vars/rke2.tfvars
    ```

    Esto crea las instancias de EC2, la configuración de red y los grupos de seguridad necesarios para el clúster de RKE2.

    ***
  </Step>

  <Step>
    ## Fase 5: Instalar las dependencias de Ansible

    Vaya al directorio de Ansible e instale los roles y las colecciones de Galaxy necesarios:

    ```bash
    cd ~/bare-metal/ansible
    ```

    ```text
    ansible-galaxy role install lablabs.rke2
    ```

    ```text
    ansible-galaxy collection install kubernetes.core
    ```

    ***
  </Step>

  <Step>
    ## Fase 6: Aprovisionar RKE2

    ### Verificar que los nodos estén listos

    ```text
    ansible all -i inventory.yml -a 'cloud-init status'
    ```

    ### Configurar NVMe

    ```text
    ansible-playbook -i inventory.yml disks_nvme.yml
    ```

    ### Ejecutar el playbook de RKE2

    ```text
    ansible-playbook -i inventory.yml rke2_playbook.yml
    ```

    Este playbook instala y configura RKE2 en todos los nodos. Aprovisiona el primer nodo servidor, espera a que esté listo y, a continuación, incorpora al clúster los nodos servidor y agente restantes.

    La salida del playbook mostrará tareas en ejecución en todos los nodos (por ejemplo, de `chb-rke2s101` a `chb-rke2s302`). Cuando termine, deberías ver un resumen como este:


  </Step>

  <Step>
    ## Fase 7: Configurar cert-manager y los certificados

    Instale cert-manager, trust-manager y genere los certificados para ClickHouse con el playbook de Ansible:

    ```bash
    cd ~/bare-metal/ansible
    ansible-playbook -i 127.0.0.1, --connection=local manage_helm_applications.yml
    ```

    Esta guía realiza lo siguiente:

    1. Instala el gráfico de Helm de cert-manager
    2. Instala el gráfico de Helm de trust-manager
    3. Crea un emisor autofirmado y un certificado de CA
    4. Crea un emisor para el certificado de CA
    5. Configura Rancher (opcional)
    6. Crea el espacio de nombres del clúster de ClickHouse
    7. Genera los certificados del servidor de ClickHouse y de Keeper
    8. Copia los certificados a los secretos de Kubernetes correspondientes

    Resultado esperado:

    ```text
    PLAY RECAP *****************************************************************
    127.0.0.1                  : ok=18   changed=13   unreachable=0    failed=0    skipped=0    rescued=0    ignored=0
    ```

    ***
  </Step>

  <Step>
    ## Fase 8: Instalar almacenamiento local

    ### Instalar local-path-provisioner

    ```bash
    kubectl apply -f https://raw.githubusercontent.com/rancher/local-path-provisioner/v0.0.32/deploy/local-path-storage.yaml
    ```

    ### Crear una StorageClass

    ```bash
    cd ~/bare-metal/ch-cluster-deployment
    kubectl apply -f local-nvme-sc.yaml
    ```

    El YAML de StorageClass debería verse así (actualiza los valores de topology para que coincidan con los nombres de tus nodos):

    ```yaml
    apiVersion: storage.k8s.io/v1
    kind: StorageClass
    metadata:
      name: local-nvme-sc
    allowVolumeExpansion: true
    parameters:
      path: /nvme/disk
    provisioner: rancher.io/local-path
    reclaimPolicy: Delete
    volumeBindingMode: WaitForFirstConsumer
    allowedTopologies:
    - matchLabelExpressions:
        - key: topology.kubernetes.io/zone
          values:
            - <your_cluster_name>-keeper
            - <your_host_1>
            - <your_host_2>
            - <your_host_3>
            - <your_host_4>
            - <your_host_5>
            - <your_host_6>
        - key: directpv.min.io/region
          values:
            - default
    ```

    ***
  </Step>

  <Step>
    ## Fase 9: Etiquetar los nodos

    Etiquete los nodos con las etiquetas de grupo de ClickHouse y de zona topológica requeridas por el operator:

    ```bash
    kubectl get nodes |awk '{ print $1}'|grep '01'|xargs -I {} -n 1 kubectl label nodes '{}' clickhouseGroup=keeper-amd64  topology.kubernetes.io/zone=c-baremetal-ch-01-keeper --overwrite
    kubectl get nodes |awk '{ print $1}'|grep '02'|xargs -I {} -n 1 kubectl label nodes '{}' clickhouseGroup=server-amd64  --overwrite
    kubectl label nodes chb-rke2s102 topology.kubernetes.io/zone=chb-rke2s102 --overwrite
    kubectl label nodes chb-rke2s202 topology.kubernetes.io/zone=chb-rke2s202 --overwrite
    kubectl label nodes chb-rke2s302 topology.kubernetes.io/zone=chb-rke2s302 --overwrite
    ```

    Ajusta los nombres y las etiquetas de los nodos para que se adapten a tu entorno.

    ***
  </Step>

  <Step>
    ## Fase 10: Instalar ClickHouse Operator y el clúster

    ### Configurar variables de entorno

    Cree y ejecute el script de variables de Helm. Actualice todos los valores de marcadores de posición:

    ```bash
    #!/bin/bash
    # AWS/S3 creds
    export AWS_ACCESS_KEY_ID=<your_aws_key>
    export AWS_SECRET_ACCESS_KEY=<your_aws_secret>
    export AWS_REGION=<your_aws_region>
    export S3_ACCESS_KEY_ID=<s3_or_minio_key>
    export S3_SECRET_ACCESS_KEY=<s3_or_minio_secret>
    export S3_REGION=us-east-1
    export S3_BUCKET_NAME=<your_s3_bucket>
    export S3_KEY_PREFIX=<ch-s3-unique-uuid>
    export S3_ENDPOINT=https://s3.us-east-1.amazonaws.com
    export ECR_HOST=<ecr_host>
    # Set AZs as determined by VPC subnets. If not relevant then use your k8s host names.
    export AZ_LIST='["<your_host_1>", "<your_host_2>", "<your_host_3>","<your_host_4>", "<your_host_5>", "<your_host_6>"]'

    PASSWORD='My super secret p@$$w0rd'
    export HASHED_PASSWORD=$(echo -n "$PASSWORD" | sha256sum | awk '{printf $1}' | base64 -w 0)
    export CLUSTER_NAME="baremetal-ch-01"
    export CHART_VERSION="1.1.311"

    export SERVER_NG_GROUP="server-amd64"
    export KEEPER_NG_GROUP="keeper-amd64"
    export KEEPER_CPU="4"
    export KEEPER_MEMORY="16Gi"
    export SERVER_CPU="64"
    export SERVER_MEMORY="128Gi"
    export CACHE_DISK_SIZE_GI=379Gi
    ```

    Cargue el script con source:

    ```bash
    source ./set_helm_vars.sh
    ```

    ### Instalar el Operator

    Inicie sesión en el registro de Helm e instale el operador:

    ```text
    # Log into the Helm registry
    aws ecr get-login-password | helm registry login --username AWS --password-stdin $ECR_HOST

    # Create image pull secret
    kubectl create secret docker-registry all-icr-io \
      --docker-server=$ECR_HOST \
      --docker-username=AWS \
      --docker-password=$(aws ecr get-login-password) \
      -n clickhouse-operator-system

    OPERATOR_VERSION=1.18587.1

    helm upgrade clickhouse-operator \
      oci://$ECR_HOST/helm/clickhouse-operator-helm \
      --install --reuse-values \
      --version="$OPERATOR_VERSION" \
      --create-namespace \
      -n clickhouse-operator-system \
      --set-json="image.repository=\"$ECR_HOST/clickhouse-operator\"" \
      --set-json='cilium.enabled=false' \
      --set-json='idleScalerEnabled=false' \
      --set-json='webhooks.enabled=false' \
      --set-json='operator.debug=true' \
      --set-json='operator.metricsScraper.enabled=false' \
      --set-json='operator.featureFlags.backupOnASeparatePod=true' \
      --set-json='operator.featureFlags.serverCMEKEnabled=true' \
      --set-json='imagePullSecrets=[{"name":"all-icr-io"}]' \
      --set-json="operator.availabilityZones=$AZ_LIST"
    ```

    ### Desplegar el clúster de ClickHouse

    Cree el archivo de valores de Helm (`clickhouse-values.yaml`) con la configuración de su clúster:

    ```yaml expandable
    imagePullSecrets:
    - name: all-icr-io

    account:
      hashedPassword: "$HASHED_PASSWORD"
    server:
      ssdCacheConfiguration:
        cacheDiskSize: "$CACHE_DISK_SIZE_GI"
      openSSL:
        enabled: true
        required: true
      replicaCount: 3
      arm64: false
      podPolicy:
        nodeSelector:
          clickhouseGroup: "$SERVER_NG_GROUP"
        resources:
          limits:
            cpu: "$SERVER_CPU"
            memory: "$SERVER_MEMORY"
          requests:
            cpu: "$SERVER_CPU"
            memory: "$SERVER_MEMORY"
      tolerations:
        - effect: NoSchedule
          key: "clickhouse.com/do-not-schedule"
          operator: Exists
      storage:
        storageClassName: local-nvme-sc
        resources:
          requests: "300Gi"
        s3:
          bucketName: "$S3_BUCKET_NAME"
          keyPrefix: "$S3_KEY_PREFIX"
          endpoint: "$S3_ENDPOINT"
          region: "$S3_REGION"
      image:
        repository: "$ECR_HOST/clickhouse-server"
        tag: "25.12.1.1497"
      additionalEnvVars:
        - name: AWS_ACCESS_KEY_ID
          value: "${S3_ACCESS_KEY_ID}"
        - name: AWS_SECRET_ACCESS_KEY
          value: "${S3_SECRET_ACCESS_KEY}"

    keeper:
      openSSL:
        enabled: true
        required: true
      image:
        repository: "$ECR_HOST/clickhouse-keeper"
        tag: "26.2.1.258"
      arm64: false
      podPolicy:
        nodeSelector:
          clickhouseGroup: "$KEEPER_NG_GROUP"
        resources:
          limits:
            cpu: "$KEEPER_CPU"
            memory: "$KEEPER_MEMORY"
          requests:
            cpu: "$KEEPER_CPU"
            memory: "$KEEPER_MEMORY"
      tolerations:
        - effect: NoSchedule
          key: "clickhouse.com/do-not-schedule"
          operator: Exists
      storage:
        storageClassName: local-nvme-sc
        resources:
          requests: 30Gi

    serviceAccount:
      create: false
      name: default

    storageClass:
      create: false
      name: local-nvme-sc
      provisioner: rancher.io/local-path
    ```

    Instale el clúster con el archivo de valores:

    ```text
    # Create namespace
    kubectl create namespace ns-$CLUSTER_NAME

    # Create image pull secret in the cluster namespace
    kubectl create secret docker-registry all-icr-io \
      --docker-server=$ECR_HOST \
      --docker-username=AWS \
      --docker-password=$(aws ecr get-login-password) \
      -n ns-$CLUSTER_NAME

    # Expand variables and install
    TEMP_VALUES_FILE=$(mktemp)
    envsubst < clickhouse-values.yaml > "$TEMP_VALUES_FILE"

    helm upgrade "$CLUSTER_NAME" \
      oci://$ECR_HOST/helm/onprem-clickhouse-cluster \
      --install --reuse-values \
      --version="$CHART_VERSION" \
      -n ns-$CLUSTER_NAME \
      --create-namespace \
      --values "$TEMP_VALUES_FILE" \
      --timeout 10m0s

    rm "$TEMP_VALUES_FILE"
    ```

    ### Correcciones posteriores a la instalación

    Tras desplegar el clúster, es posible que necesite aplicar un parche al StatefulSet de Keeper para añadir secretos de extracción de imágenes y credenciales de S3:

    ```text
    # Copy pull secret to cluster namespace
    kubectl get secret all-icr-io -n default -o yaml | \
      sed "s/namespace: default/namespace: ns-$CLUSTER_NAME/" | \
      kubectl apply -n ns-$CLUSTER_NAME --force -f -

    # Patch keeper StatefulSet (use sts_keeper_patch_template.yaml from example scripts)
    kubectl patch sts c-$CLUSTER_NAME-keeper -n ns-$CLUSTER_NAME \
      --patch "$(envsubst < sts_keeper_patch_template.yaml)"

    # Restart keeper pods
    kubectl delete pod c-$CLUSTER_NAME-keeper-{0,1,2} -n ns-$CLUSTER_NAME
    ```

    Espere a que los pods de Keeper se vuelvan a desplegar y, si es necesario, corrija los PVs del servidor:

    ```text
    ./sts_server_patch.sh "ns-$CLUSTER_NAME" all-icr-io
    kubectl delete pod -l kind=clickhouse-server
    ```

    ***
  </Step>
</Steps>

<div id="verify-installation">
  ## Verificar la instalación
</div>

Una vez que todos los pods estén en ejecución, realiza el reenvío de puertos y compruébalo. Como esta implementación usa TLS (con OpenSSL habilitado), utiliza el puerto 9440:

```bash
kubectl port-forward svc/c-$CLUSTER_NAME-server-any 9440:9440 -n ns-$CLUSTER_NAME
```

Conéctese mediante el cliente de ClickHouse:

```bash
clickhouse client --host localhost --port 9440 --password "$PASSWORD" --secure --accept-invalid-certificate
```

Ejecute una consulta de prueba:

```sql
SELECT 1;
```

Resultado esperado:

```text
   ┌─1─┐
1. │ 1 │
   └───┘
1 row in set. Elapsed: 0.001 sec.
```
