Restablece las contraseñas de usuario de ClickHouse mediante la API privada.

<div id="prerequisites">
  ## Requisitos previos
</div>

* ClickHouse API privada instalada y accesible (p. ej., mediante reenvío de puertos a `http://localhost:8080/`)
* Al menos un clúster de ClickHouse desplegado (esta guía usa `default-xx-01`)

<Steps>
  <Step>
    ## 1. Generar el hash de una contraseña

    Calcula el hash de la nueva contraseña con SHA-256 y codifica el resultado en base64:

    ```ini
    PASSWORD='My super secret p@$$w0rd'
    HASHED_PASSWORD=$(echo -n "$PASSWORD" | shasum -a 256 | awk '{printf $1}' | base64)
    ```
  </Step>

  <Step>
    ## 2. Crear el cuerpo de la solicitud

    Crea un archivo JSON con el hash de la contraseña:

    ```json
    {
      "user_hashed_password": "<base64-encoded-sha256-hash>",
      "hashing_function": "sha256",
      "username": "default"
    }
    ```

    Guárdalo como `password_reset.json`.

    | Campo                  | Obligatorio | Predeterminado | Descripción                                       |
    | ---------------------- | ----------- | -------------- | ------------------------------------------------- |
    | `user_hashed_password` | Sí          | --             | Hash codificado en base64 de la nueva contraseña  |
    | `hashing_function`     | No          | `sha256`       | Algoritmo de hash utilizado                       |
    | `username`             | No          | `default`      | Nombre de usuario cuya contraseña se restablecerá |
  </Step>

  <Step>
    ## 3. Enviar la solicitud

    ```bash
    curl -X POST "http://localhost:8080/api/v1/instances/default-xx-01/reset-user-password" \
        -H "Content-Type: application/json" \
        --data-binary "@password_reset.json"
    ```

    Una respuesta exitosa:

    ```json
    {
      "message": "User password reset successfully"
    }
    ```
  </Step>

  <Step>
    ## 4. Verificar el cambio

    El restablecimiento de la contraseña surte efecto después de que el operador procese el cambio. Verifícalo conectándote al clúster con la nueva contraseña.

    También puedes supervisar la operación mediante el endpoint `status`:

    ```bash
    curl "http://localhost:8080/api/v1/instances/default-xx-01/status"
    ```
  </Step>
</Steps>

<div id="hydra-limitation">
  ## Limitación de Hydra
</div>

No se puede restablecer la contraseña en las instancias secundarias de Hydra. Para restablecer la contraseña de una instancia secundaria de Hydra, hágalo en la instancia principal.
