ClickHouse permite importar datos desde CSV y exportarlos a CSV. Como los archivos CSV pueden presentar distintas variantes de formato, incluidas filas de encabezado, delimitadores personalizados y sГӯmbolos de escape, ClickHouse ofrece formatos y ajustes para gestionar cada caso de forma eficiente.

<div id="importing-data-from-a-csv-file">
  ## Importar datos desde un archivo CSV
</div>

Antes de importar los datos, vamos a crear una tabla con una estructura adecuada:

```sql
CREATE TABLE sometable
(
    `path` String,
    `month` Date,
    `hits` UInt32
)
ENGINE = MergeTree
ORDER BY tuple(month, path)
```

Para importar datos del [archivo CSV](https://clickhouse-docs-assets.s3.us-east-1.amazonaws.com/data_small.csv) a la tabla `sometable`, podemos pasar el archivo directamente a `clickhouse-client` mediante una tuberГӯa:

```bash
clickhouse-client -q "INSERT INTO sometable FORMAT CSV" < data_small.csv
```

Tenga en cuenta que usamos [FORMAT CSV](/es/reference/formats/CSV/CSV) para indicarle a ClickHouse que estamos cargando datos en formato CSV. Como alternativa, podemos cargar los datos desde un archivo local mediante la clГЎusula [FROM INFILE](/es/reference/statements/insert-into#inserting-data-from-a-file):

```sql
INSERT INTO sometable
FROM INFILE 'data_small.csv'
FORMAT CSV
```

AquГӯ usamos la clГЎusula `FORMAT CSV` para que ClickHouse reconozca el formato del archivo. TambiГ©n podemos cargar datos directamente desde URL con la funciГіn [url()](/es/reference/functions/table-functions/url) o desde archivos en S3 con la funciГіn [s3()](/es/reference/functions/table-functions/s3).

<Tip>
  Podemos omitir la configuraciГіn explГӯcita del formato para `file()` e `INFILE`/`OUTFILE`.
  En ese caso, ClickHouse detectarГЎ automГЎticamente el formato en funciГіn de la extensiГіn del archivo.
</Tip>

<div id="csv-files-with-headers">
  ### Archivos CSV con encabezados
</div>

Supongamos que nuestro [archivo CSV incluye encabezados](https://clickhouse-docs-assets.s3.us-east-1.amazonaws.com/data_small_headers.csv):

```bash
head data-small-headers.csv
```

```response
"path","month","hits"
"Akiba_Hebrew_Academy","2017-08-01",241
"Aegithina_tiphia","2018-02-01",34
```

Para importar los datos de este archivo, podemos usar el formato [CSVWithNames](/es/reference/formats/CSV/CSVWithNames):

```bash
clickhouse-client -q "INSERT INTO sometable FORMAT CSVWithNames" < data_small_headers.csv
```

En este caso, ClickHouse omite la primera fila al importar los datos del archivo.

<Tip>
  A partir de la [versiГіn](https://github.com/ClickHouse/ClickHouse/releases) 23.1, ClickHouse detecta automГЎticamente los encabezados en los archivos CSV al usar el formato `CSV`, por lo que no es necesario utilizar `CSVWithNames` ni `CSVWithNamesAndTypes`.
</Tip>

<div id="csv-files-with-custom-delimiters">
  ### Archivos CSV con delimitadores personalizados
</div>

Si el archivo CSV usa un delimitador distinto de la coma, podemos usar la opciГіn [format&#95;csv&#95;delimiter](/es/reference/settings/formats#format_csv_delimiter) para establecer el sГӯmbolo correspondiente:

```sql
SET format_csv_delimiter = ';'
```

Ahora, al importar desde un archivo CSV, se utilizarГЎ el sГӯmbolo `;` como delimitador en lugar de una coma.

<div id="skipping-lines-in-a-csv-file">
  ### Omitir lГӯneas en un archivo CSV
</div>

A veces, es necesario omitir un cierto nГәmero de lГӯneas al importar datos de un archivo CSV. Esto puede hacerse con la opciГіn [input&#95;format&#95;csv&#95;skip&#95;first&#95;lines](/es/reference/settings/formats#input_format_csv_skip_first_lines):

```sql
SET input_format_csv_skip_first_lines = 10
```

En este caso, vamos a omitir las diez primeras lГӯneas del archivo CSV:

```sql
SELECT count(*) FROM file('data-small.csv', CSV)
```

```response
в”Ңв”Җcount()в”Җв”җ
в”Ӯ     990 в”Ӯ
в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ
```

El [archivo](https://clickhouse-docs-assets.s3.us-east-1.amazonaws.com/data_small.csv) tiene 1k filas, pero ClickHouse solo cargГі 990, ya que le indicamos que omitiera las primeras 10.

<Tip>
  Al usar la funciГіn `file()`, en ClickHouse Cloud tendrГЎs que ejecutar los comandos en `clickhouse client` en la mГЎquina donde se encuentra el archivo. Otra opciГіn es usar [`clickhouse-local`](/es/concepts/features/tools-and-utilities/clickhouse-local) para explorar archivos de forma local.
</Tip>

<div id="treating-null-values-in-csv-files">
  ### Tratamiento de los valores NULL en archivos CSV
</div>

Los valores NULL pueden codificarse de distintas maneras segГәn la aplicaciГіn que haya generado el archivo. De forma predeterminada, ClickHouse usa `\N` como valor NULL en CSV. Pero podemos cambiarlo con la opciГіn [format&#95;csv&#95;null&#95;representation](/es/reference/settings/formats#format_tsv_null_representation).

Supongamos que tenemos el siguiente archivo CSV:

```bash
> cat nulls.csv
Donald,90
Joe,Nothing
Nothing,70
```

Si cargamos datos de este archivo, ClickHouse considerarГЎ `Nothing` como un String (lo cual es correcto):

```sql
SELECT * FROM file('nulls.csv')
```

```response
в”Ңв”Җc1в”Җв”Җв”Җв”Җв”Җв”Җв”¬в”Җc2в”Җв”Җв”Җв”Җв”Җв”Җв”җ
в”Ӯ Donald  в”Ӯ 90      в”Ӯ
в”Ӯ Joe     в”Ӯ Nothing в”Ӯ
в”Ӯ Nothing в”Ӯ 70      в”Ӯ
в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҙв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ
```

Si queremos que ClickHouse trate `Nothing` como `NULL`, podemos indicarlo con la siguiente opciГіn:

```sql
SET format_csv_null_representation = 'Nothing'
```

Ahora tenemos `NULL` donde deberГӯa estar:

```sql
SELECT * FROM file('nulls.csv')
```

```response
в”Ңв”Җc1в”Җв”Җв”Җв”Җв”Җв”¬в”Җc2в”Җв”Җв”Җв”җ
в”Ӯ Donald в”Ӯ 90   в”Ӯ
в”Ӯ Joe    в”Ӯ бҙәбөҒбҙёбҙё в”Ӯ
в”Ӯ бҙәбөҒбҙёбҙё   в”Ӯ 70   в”Ӯ
в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҙв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ
```

<div id="tsv-tab-separated-files">
  ## Archivos TSV (separados por tabuladores)
</div>

El formato de datos separados por tabuladores se utiliza ampliamente como formato de intercambio de datos. Para cargar datos desde un [archivo TSV](https://clickhouse-docs-assets.s3.us-east-1.amazonaws.com/data_small.tsv) en ClickHouse, se utiliza el formato [TabSeparated](/es/reference/formats/TabSeparated/TabSeparated):

```bash
clickhouse-client -q "INSERT INTO sometable FORMAT TabSeparated" < data_small.tsv
```

TambiГ©n existe el formato [TabSeparatedWithNames](/es/reference/formats/TabSeparated/TabSeparatedWithNames), que permite trabajar con archivos TSV que tienen encabezados. Y, al igual que con CSV, podemos omitir las primeras X lГӯneas con la opciГіn [input&#95;format&#95;tsv&#95;skip&#95;first&#95;lines](/es/reference/settings/formats#input_format_tsv_skip_first_lines).

<div id="raw-tsv">
  ### TSV sin procesar
</div>

A veces, los archivos TSV se guardan sin aplicar escape a los tabuladores ni a los saltos de lГӯnea. Debemos usar [TabSeparatedRaw](/es/reference/formats/TabSeparated/TabSeparatedRaw) para procesar esos archivos.

<div id="exporting-to-csv">
  ## Exportar a CSV
</div>

Cualquiera de los formatos de los ejemplos anteriores tambiГ©n puede usarse para exportar datos. Para exportar datos desde una tabla (o una consulta) en formato CSV, usamos la misma clГЎusula `FORMAT`:

```sql

FROM sometable
LIMIT 5
FORMAT CSV
```

```response

```

Para aГұadir un encabezado al archivo CSV, utilizamos el formato [CSVWithNames](/es/reference/formats/CSV/CSVWithNames):

```sql
SELECT *
FROM sometable
LIMIT 5
FORMAT CSVWithNames
```

```response

"Aegithina_tiphia","2018-02-01",34
```

<div id="saving-exported-data-to-a-csv-file">
  ### Guardar los datos exportados en un archivo CSV
</div>

Para guardar los datos exportados en un archivo, podemos usar la clГЎusula [INTO...OUTFILE](/es/reference/statements/select/into-outfile):

```sql

INTO OUTFILE 'out.csv'
FORMAT CSVWithNames
```

```response

```

Observe que ClickHouse tardГі solo **~1** segundo en guardar 36 millones de filas en un archivo CSV.

Si queremos usar delimitadores distintos de la coma, podemos utilizar para ello la opciГіn de configuraciГіn [format&#95;csv&#95;delimiter](/es/reference/settings/formats#format_csv_delimiter):

Ahora ClickHouse usarГЎ `|` como delimitador del formato CSV:

<div id="exporting-csv-for-windows">
  ### ExportaciГіn de CSV para Windows
</div>

Si queremos que un archivo CSV funcione correctamente en un entorno Windows, conviene considerar habilitar la opciГіn [output&#95;format&#95;csv&#95;crlf&#95;end&#95;of&#95;line](/es/reference/settings/formats#output_format_csv_crlf_end_of_line).
```sql
SET output_format_csv_crlf_end_of_line = 1;
```

<div id="custom-delimiters-separators-and-escaping-rules">
  ## Delimitadores, separadores y reglas de escape personalizadas
</div>

En casos complejos, los datos de texto pueden tener un formato muy personalizado y aun asГӯ conservar una estructura. ClickHouse tiene un formato especial [CustomSeparated](/es/reference/formats/CustomSeparated/CustomSeparated) para estos casos, que permite definir reglas de escape personalizadas, delimitadores, separadores de lГӯnea y sГӯmbolos de inicio y fin.

Supongamos que tenemos los siguientes datos en el archivo:

```text
row('Akiba_Hebrew_Academy';'2017-08-01';241),row('Aegithina_tiphia';'2018-02-01';34),...
```

Podemos ver que las filas individuales estГЎn encerradas en `row()`, las lГӯneas se separan con `,` y los valores individuales se delimitan con `;`. En este caso, podemos usar la siguiente configuraciГіn para leer datos de este archivo:

```sql
SET format_custom_row_before_delimiter = 'row(';
SET format_custom_row_after_delimiter = ')';
SET format_custom_field_delimiter = ';';
SET format_custom_row_between_delimiter = ',';
SET format_custom_escaping_rule = 'Quoted';
```

Ahora podemos cargar datos desde nuestro [archivo con formato personalizado](https://clickhouse-docs-assets.s3.us-east-1.amazonaws.com/data_small_custom.txt):

```sql
SELECT *
FROM file('data_small_custom.txt', CustomSeparated)
LIMIT 3
```

```response
в”Ңв”Җc1в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”¬в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җc2в”Җв”¬в”Җв”Җc3в”Җв”җ
в”Ӯ Akiba_Hebrew_Academy      в”Ӯ 2017-08-01 в”Ӯ 241 в”Ӯ
в”Ӯ Aegithina_tiphia          в”Ӯ 2018-02-01 в”Ӯ  34 в”Ӯ
в”Ӯ 1971-72_Utah_Stars_season в”Ӯ 2016-10-01 в”Ӯ   1 в”Ӯ
в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҙв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҙв”Җв”Җв”Җв”Җв”Җв”ҳ
```

TambiГ©n podemos usar [CustomSeparatedWithNames](/es/reference/formats/CustomSeparated/CustomSeparatedWithNames) para exportar e importar correctamente los encabezados. Explora los formatos [regex y Template](/es/guides/clickhouse/data-formats/templates-regex) para abordar casos aГәn mГЎs complejos.

<div id="working-with-large-csv-files">
  ## Trabajar con archivos CSV grandes
</div>

Los archivos CSV pueden ser grandes, y ClickHouse funciona de forma eficiente con archivos de cualquier tamaГұo. Los archivos grandes suelen venir comprimidos, y ClickHouse los maneja sin necesidad de descomprimirlos antes de procesarlos. Podemos usar una clГЎusula `COMPRESSION` durante un insert:

```sql
INSERT INTO sometable
FROM INFILE 'data_csv.csv.gz'
COMPRESSION 'gzip' FORMAT CSV
```

Si se omite una clГЎusula `COMPRESSION`, ClickHouse seguirГЎ intentando determinar la compresiГіn del archivo en funciГіn de su extensiГіn. El mismo enfoque puede utilizarse para exportar archivos directamente en formatos comprimidos:

```sql
SELECT *
FROM for_csv
INTO OUTFILE 'data_csv.csv.gz'
COMPRESSION 'gzip' FORMAT CSV
```

Esto crearГЎ el archivo comprimido `data_csv.csv.gz`.
