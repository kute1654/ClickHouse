A menudo tenemos que trabajar con datos en formatos de texto personalizados. Puede tratarse de un formato no estГЎndar, JSON no vГЎlido o un CSV daГұado. <div id="importing-based-on-a-template">
  ## ImportaciГіn basada en Template
</div>

Supongamos que queremos importar datos del siguiente [archivo de log](https://clickhouse-docs-assets.s3.us-east-1.amazonaws.com/error.log):



Podemos usar el formato [Template](/es/reference/formats/Template/Template) para importar estos datos. Tenemos que definir una cadena de plantilla con marcadores de posiciГіn para los valores de cada fila de los datos de entrada:

```response

```

Vamos a crear una tabla para importar nuestros datos en ella:

```sql
CREATE TABLE error_log
(
    `time` DateTime,
    `ip` String,
    `host` String,
    `request` String
)
ENGINE = MergeTree
ORDER BY (host, request, time)
```

Para importar datos con una plantilla determinada, tenemos que guardar la cadena de la plantilla en un archivo ([row.template](https://clickhouse-docs-assets.s3.us-east-1.amazonaws.com/row.template) en nuestro caso):

```response
${time:Escaped} [error]  client: ${ip:CSV}, server: ${host:CSV} ${request:JSON}
```

Definimos el nombre de una columna y la regla de escape con el formato `${name:escaping}`. AquГӯ hay varias opciones disponibles, como CSV, JSON, Escaped o Quoted, que implementan las [reglas de escape correspondientes](/es/reference/formats/Template/Template).

Ahora podemos usar el archivo indicado como argumento de la opciГіn de configuraciГіn `format_template_row` al importar datos (*ten en cuenta que los archivos de plantilla y de datos **no deben tener** un sГӯmbolo `\n` adicional al final del archivo*):

```sql
INSERT INTO error_log FROM INFILE 'error.log'
SETTINGS format_template_row = 'row.template'
FORMAT Template
```

Y podemos comprobar que los datos se cargaron en la tabla:

```sql
SELECT
    request,
    count(*)
FROM error_log
GROUP BY request
```

<div id="skipping-whitespaces">
  ### OmisiГіn de espacios en blanco
</div>

Considere usar [TemplateIgnoreSpaces](/es/reference/formats/Template/TemplateIgnoreSpaces), que permite omitir los espacios en blanco entre delimitadores en Template:

```text
Template:               -->  "p1: ${p1:CSV}, p2: ${p2:CSV}"
TemplateIgnoreSpaces    -->  "p1:${p1:CSV}, p2:${p2:CSV}"
```

<div id="exporting-data-using-templates">
  ## ExportaciГіn de datos con Template
</div>

TambiГ©n podemos exportar datos a cualquier formato de texto con Template. En este caso, tenemos que crear dos archivos:

[Plantilla del conjunto de resultados](https://clickhouse-docs-assets.s3.us-east-1.amazonaws.com/output.results), que define la estructura del conjunto de resultados completo:

```response
== Top 10 IPs ==
${data}
--- ${rows_read:XML} rows read in ${time:XML} ---
```

AquГӯ, `rows_read` y `time` son mГ©tricas del sistema disponibles para cada peticiГіn. Mientras que `data` representa las filas generadas (`${data}` siempre debe aparecer como el primer marcador de posiciГіn en este archivo), a partir de una plantilla definida en un [**archivo de plantilla de filas**](https://clickhouse-docs-assets.s3.us-east-1.amazonaws.com/output.rows):

```response
${ip:Escaped} generated ${total:Escaped} requests
```

Ahora, usemos estas Template para exportar la siguiente consulta:

```sql
SELECT
    ip,
    count() AS total
FROM error_log GROUP BY ip ORDER BY total DESC LIMIT 10
FORMAT Template SETTINGS format_template_resultset = 'output.results',
                         format_template_row = 'output.rows';
```

<div id="exporting-to-html-files">
  ### ExportaciГіn a archivos HTML
</div>

Los resultados basados en plantillas tambiГ©n se pueden exportar a archivos mediante la clГЎusula [`INTO OUTFILE`](/es/reference/statements/select/into-outfile). Generemos archivos HTML a partir de los formatos [conjunto de resultados](https://clickhouse-docs-assets.s3.us-east-1.amazonaws.com/html.results) y [fila](https://clickhouse-docs-assets.s3.us-east-1.amazonaws.com/html.row) proporcionados:

```sql
SELECT
    ip,
    count() AS total
FROM error_log GROUP BY ip ORDER BY total DESC LIMIT 10
INTO OUTFILE 'out.html'
FORMAT Template
SETTINGS format_template_resultset = 'html.results',
         format_template_row = 'html.row'
```

<div id="importing-data-based-on-regular-expressions">
  ## ImportaciГіn de datos basada en expresiones regulares
</div>

El formato [Regexp](/es/reference/formats/Regexp) cubre casos mГЎs complejos en los que los datos de entrada deben procesarse de una forma mГЎs sofisticada. Analicemos nuestro archivo de ejemplo [error.log](https://clickhouse-docs-assets.s3.us-east-1.amazonaws.com/error.log), pero esta vez capturemos el nombre del archivo y el protocolo para guardarlos en columnas independientes. Primero, preparemos una tabla nueva para ello:

```sql
CREATE TABLE error_log
(
    `time` DateTime,
    `ip` String,
    `host` String,
    `file` String,
    `protocol` String
)
ENGINE = MergeTree
ORDER BY (host, file, time)
```

Ahora podemos importar datos a partir de una expresiГіn regular:

```sql
INSERT INTO error_log FROM INFILE 'error.log'
SETTINGS
  format_regexp = '(.+?) \\[error\\]  client: (.+), server: (.+?) "GET .+?([^/]+\\.[^ ]+) (.+?)"'
FORMAT Regexp
```

