<div id="description">
  ## DescripciГіn
</div>

El combinador [`SimpleState`](/es/reference/functions/aggregate-functions/combinators#-simplestate) puede aplicarse a la funciГіn [`min`](/es/reference/functions/aggregate-functions/min)
para devolver el valor mГӯnimo de entre todos los valores de entrada. Devuelve el
resultado con el tipo [`SimpleAggregateFunction`](/es/reference/data-types/simpleaggregatefunction).

<div id="example-usage">
  ## Ejemplo de uso
</div>

Veamos un ejemplo prГЎctico con una tabla que registra lecturas diarias de
temperatura. Para cada ubicaciГіn, queremos mantener la temperatura mГЎs baja
registrada. Al usar el tipo `SimpleAggregateFunction` con `min`, el valor
almacenado se actualiza automГЎticamente cuando se registra una temperatura inferior.

Cree la tabla de origen para las lecturas de temperatura en bruto:

```sql

(
    location_id UInt32,
    location_name String,
    temperature Int32,
    recorded_at DateTime DEFAULT now()
)
    ENGINE = MergeTree()
ORDER BY (location_id, recorded_at);
```

```sql

(
    location_id UInt32,
    location_name String,
    min_temp SimpleAggregateFunction(min, Int32),  -- Almacena la temperatura mГӯnima
    max_temp SimpleAggregateFunction(max, Int32)   -- Almacena la temperatura mГЎxima
)
ENGINE = AggregatingMergeTree()
ORDER BY location_id;
```

Cree una vista materializada incremental que actuarГЎ como un disparador de inserciГіn

```sql

TO temperature_extremes
AS SELECT
    location_id,
    location_name,
    minSimpleState(temperature) AS min_temp,     -- Uso del combinador SimpleState
    maxSimpleState(temperature) AS max_temp      -- Uso del combinador SimpleState
FROM raw_temperature_readings
GROUP BY location_id, location_name;
```
