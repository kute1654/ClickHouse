<div id="december-18-2023">
  ## 2023年12月18日
</div>

このリリースでは、GCP の新しいリージョン (us-east1) 、セキュアなエンドポイント接続をセルフサービスで設定できる機能、DBT 1.7 を含む追加のインテグレーションのサポート、さらに多数のバグ修正とセキュリティ強化が追加されました。

<div id="general-changes-1">
  ### 一般的な変更
</div>

* ClickHouse Cloud が GCP の us-east1 (サウスカロライナ) リージョンで利用可能になりました
* OpenAPI 経由で AWS プライベートリンクおよび GCP Private Service Connect を設定できるようになりました

<div id="console-changes-7">
  ### コンソールの変更点
</div>

* Developerロールを持つユーザーが、SQL Consoleにシームレスにログインできるようになりました
* オンボーディング中にアイドル制御を設定するワークフローを簡素化しました

<div id="integrations-changes-7">
  ### インテグレーションの変更点
</div>

* DBT コネクタ: DBT v1.7 までをサポート
* Metabase: Metabase v0.48 をサポート
* PowerBI Connector: PowerBI Cloud で実行可能に
* ClickPipes の内部ユーザーの権限を設定可能に
* Kafka Connect
  * 重複排除ロジックと Nullable 型のインジェストを改善
  * テキストベースのフォーマット (CSV、TSV) をサポート
* Apache Beam: Boolean 型および LowCardinality 型をサポート
* Nodejs client: Parquet フォーマットをサポート

<div id="security-announcements">
  ### セキュリティに関するお知らせ
</div>

* 3 件のセキュリティ脆弱性を修正しました。詳細は [security changelog](/ja/resources/changelogs/security-changelog) を参照してください。
  * CVE 2023-47118 (CVSS 7.0) - デフォルトでポート 9000/tcp で動作するネイティブインターフェイスに影響するヒープバッファオーバーフロー脆弱性
  * CVE-2023-48704 (CVSS 7.0) - デフォルトでポート 9000/tcp で動作するネイティブインターフェイスに影響するヒープバッファオーバーフロー脆弱性
  * CVE 2023-48298 (CVSS 5.9) - FPC 圧縮コーデックにおける整数アンダーフロー脆弱性

<div id="november-22-2023">
  ## 2023年11月22日
</div>

このリリースでは、コアデータベースのバージョンをアップグレードし、ログインおよび認証フローを改善するとともに、Kafka Connect Sink のプロキシ対応を追加しました。

<div id="clickhouse-version-upgrade-2">
  ### ClickHouse バージョンのアップグレード
</div>

* Parquet ファイルの読み取りパフォーマンスが大幅に向上しました。詳細は [23.8 release blog](https://clickhouse.com/blog/clickhouse-release-23-08) を参照してください。
* JSON の型推論サポートが追加されました。詳細は [23.9 release blog](https://clickhouse.com/blog/clickhouse-release-23-09) を参照してください。
* `ArrayFold` など、アナリスト向けの強力な関数が導入されました。詳細は [23.10 release blog](https://clickhouse.com/blog/clickhouse-release-23-10) を参照してください。
* **ユーザー向けの後方互換性のない変更**: JSON フォーマットで文字列から数値を推論しないようにするため、設定 `input_format_json_try_infer_numbers_from_strings` はデフォルトで無効になりました。これを有効にすると、サンプルデータに数値のように見える文字列が含まれている場合、パースエラーが発生する可能性があります。
* 数十件に及ぶ新機能、パフォーマンス改善、バグ修正が含まれています。詳細は [コアデータベースの変更履歴](/ja/resources/changelogs/oss/2026) を参照してください。

<div id="console-changes-7">
  ### コンソールの変更点
</div>

* ログインおよび認証フローを改善しました。
* 大規模なスキーマへの対応を強化するため、AI ベースのクエリ候補を改善しました。

<div id="integrations-changes-7">
  ### インテグレーションの変更点
</div>

* Kafka Connect Sink: プロキシ対応、`topic-tablename` マッピング、Keeper の *exactly-once* 配信プロパティの設定機能を追加しました。
* Node.js クライアント: Parquet フォーマットのサポートを追加しました。
* Metabase: `datetimeDiff` 関数に対応しました。
* Python クライアント: カラム名での特殊文字をサポートしました。タイムゾーンパラメータのバインドを修正しました。

<div id="november-2-2023">
  ## 2023年11月2日
</div>

このリリースでは、アジアにおける開発向けサービスのリージョン対応を拡充し、顧客管理の暗号化キーにキーローテーション機能を追加するとともに、請求コンソールの税設定をより細かく行えるよう改善し、サポート対象の各種言語クライアント全体で多数のバグ修正を実施しました。

<div id="general-updates-1">
  ### 全般的な更新
</div>

* AWS の `ap-south-1` (ムンバイ) および `ap-southeast-1` (シンガポール) で Development サービスが利用可能になりました
* 顧客管理暗号鍵 (CMEK) のキーローテーションをサポートしました

<div id="console-changes-7">
  ### コンソールの変更点
</div>

* クレジットカード追加時に、税金に関する詳細な設定を行えるようになりました

<div id="integrations-changes-7">
  ### インテグレーションの変更点
</div>

* MySQL
  * MySQL 経由での Tableau Online と QuickSight のサポートを改善
* Kafka Connector
  * テキストベースのフォーマット (CSV、TSV) をサポートする新しい StringConverter を導入
  * Bytes および Decimal データ型のサポートを追加
  * Retryable Exceptions を調整し、errors.tolerance=all の場合でも常に再試行されるように変更
* Node.js client
  * 大規模なデータセットのストリーミング時に、破損した結果が返される問題を修正
* Python クライアント
  * 大量 insert 時のタイムアウトを修正
  * NumPy/Pandas の Date32 に関する問題を修正
* Golang client
  * JSON カラムへの空の map の挿入、圧縮 buffer のクリーンアップ、クエリのエスケープ、IPv4 および IPv6 で zero/nil の場合に panic が発生する問題を修正
  * キャンセルされた insert にウォッチドッグを追加
* DBT
  * テストを追加し、分散テーブルのサポートを改善

<div id="october-19-2023">
  ## 2023年10月19日
</div>

このリリースでは、SQLコンソールの使いやすさとパフォーマンスが改善されたほか、Metabase コネクタでの IP データ型の処理が向上し、Java クライアントおよび Node.js クライアントに新機能が追加されました。

<div id="console-changes-7">
  ### コンソールの変更点
</div>

* SQL Consoleの使いやすさを改善 (例: クエリ実行後もカラム幅を保持)
* SQL Consoleのパフォーマンスを改善

<div id="integrations-changes-7">
  ### インテグレーションの変更点
</div>

* Java クライアント:
  * パフォーマンス向上と既存の接続の再利用のため、デフォルトのネットワークライブラリを変更しました
  * プロキシのサポートを追加しました
  * Trust Store を使用したセキュアな接続のサポートを追加しました
* Node.js クライアント: insert クエリでの keep-alive の動作を修正しました
* Metabase: IPv4/IPv6 カラムのシリアライゼーションを修正しました

<div id="september-28-2023">
  ## 2023年9月28日
</div>

このリリースでは、Kafka 向け ClickPipes、Confluent Cloud、Amazon MSK、Kafka Connect ClickHouse Sink の一般提供に加え、IAM ロールを介して Amazon S3 へのアクセスを保護するためのセルフサービスのワークフローと、AI を活用したクエリ候補 (プライベートプレビュー) が提供されます。

<div id="console-changes-7">
  ### コンソールの変更点
</div>

* [IAMロールを使用したAmazon S3へのアクセス](/ja/products/cloud/guides/data-sources/accessing-s3-data-securely)を保護するためのセルフサービス ワークフローを追加しました
* AI支援によるクエリ候補をプライベートプレビューとして導入しました (お試しになりたい場合は、[ClickHouse Cloud Supportにお問い合わせください](https://console.clickhouse.cloud/support)。)

<div id="integrations-changes-7">
  ### インテグレーションの変更点
</div>

* Kafka、Confluent Cloud、Amazon MSK 向けに、ターンキー型のデータインジェストサービスである ClickPipes の一般提供を発表 ([リリースブログ](https://clickhouse.com/blog/clickpipes-is-generally-available)を参照)
* Kafka Connect ClickHouse Sink が一般提供に到達
  * `clickhouse.settings` プロパティを使用した、カスタム ClickHouse settings のサポートを拡張
  * 動的フィールドを考慮するよう、重複排除の動作を改善
  * ClickHouse からテーブルの変更を再取得するための `tableRefreshInterval` のサポートを追加
* [PowerBI](/ja/integrations/connectors/data-visualization/powerbi-and-clickhouse) と ClickHouse のデータ型間の型マッピング、および SSL 接続の問題を修正

<div id="september-7-2023">
  ## 2023年9月7日
</div>

このリリースには、PowerBI Desktop公式コネクタのベータリリース、インド向けのクレジットカード決済処理の改善、ならびにサポート対象の各言語クライアントに対する複数の改善が含まれます。

<div id="console-changes-7">
  ### コンソールの変更点
</div>

* 未使用クレジットと支払いの再試行を追加し、インドからの請求に対応

<div id="integrations-changes-7">
  ### インテグレーションの変更点
</div>

* Kafka Connector: ClickHouse 設定の構成をサポートし、error.tolerance 設定オプションを追加
* PowerBI Desktop: 公式コネクタのベータ版をリリース
* Grafana: Point Geo 型のサポートを追加し、Data Analyst ダッシュボードのパネルを修正し、timeInterval マクロを修正
* Python クライアント: Pandas 2.1.0 に対応し、Python 3.7 のサポートを終了し、Nullable JSON 型のサポートを追加
* Node.js クライアント: default&#95;format 設定のサポートを追加
* Golang クライアント: bool 型の処理を修正し、文字列の制限を削除

<div id="aug-24-2023">
  ## 2023年8月24日
</div>

このリリースでは、ClickHouse データベースで MySQL インターフェイスがサポートされ、PowerBI 向けの新しい公式コネクタが導入されたほか、Cloud Console に新しい「Running Queries」ビューが追加され、ClickHouse のバージョンが 23.7 に更新されました。

<div id="general-updates-1">
  ### 全般的な更新
</div>

* [MySQL wire protocol](/ja/concepts/features/interfaces/mysql) のサポートを追加しました。これにより、既存の多くの BI ツールとの互換性が実現するなど、さまざまな用途に対応できます。この機能を組織で有効にするには、サポートにお問い合わせください。
* 新しい公式 PowerBI コネクタを追加しました

<div id="console-changes-7">
  ### コンソールの変更点
</div>

* SQL Console の「Running Queries」ビューに対応

<div id="clickhouse-237-version-upgrade">
  ### ClickHouse 23.7 へのバージョンアップグレード
</div>

* Azure Table function のサポートを追加し、Geo データ型を本番利用可能とし、join のパフォーマンスを改善しました。詳細は 23.5 リリースの [ブログ](https://clickhouse.com/blog/clickhouse-release-23-05) を参照してください
* MongoDB インテグレーションのサポートをバージョン 6.0 まで拡張しました。詳細は 23.6 リリースの [ブログ](https://clickhouse.com/blog/clickhouse-release-23-06) を参照してください
* Parquet フォーマットへの書き込みパフォーマンスを 6 倍に向上させ、PRQL クエリ言語のサポートを追加し、SQL 互換性を改善しました。詳細は 23.7 リリースの [プレゼンテーション](https://presentations.clickhouse.com/release_23.7/) を参照してください
* 数十件に及ぶ新機能、パフォーマンス改善、バグ修正が含まれます。23.5、23.6、23.7 の詳細は [変更履歴](/ja/resources/changelogs/oss/2026) を参照してください

<div id="integrations-changes-7">
  ### インテグレーションの変更点
</div>

* Kafka Connector: Avro の Date 型および Time 型のサポートを追加
* JavaScript client: Web 環境向けの安定版をリリース
* Grafana: フィルタロジックとデータベース名の処理を改善し、秒未満の精度を持つ TimeInteval のサポートを追加
* Golang Client: バッチ処理と非同期データ読み込みに関する複数の問題を修正
* Metabase: v0.47 をサポートし、接続の権限借用を追加、データ型のマッピングを修正

<div id="july-27-2023">
  ## 2023年7月27日
</div>

このリリースでは、Kafka 向け ClickPipes のプライベートプレビュー、新しいデータ読み込み機能、Cloud Console を使って URL からファイルを読み込める機能が追加されました。

<div id="integrations-changes-7">
  ### インテグレーションの変更点
</div>

* Kafka 向け [ClickPipes](https://clickhouse.com/cloud/clickpipes) のプライベートプレビューを開始しました。これは、Kafka および Confluent Cloud からの大量のデータ取り込みを数クリックで行える、クラウドネイティブなインテグレーションエンジンです。ウェイトリストへの登録は[こちら](https://clickhouse.com/cloud/clickpipes#joinwaitlist)からお願いします。
* JavaScript クライアント: Web 環境 (ブラウザ、Cloudflare Workers) のサポートをリリースしました。コミュニティがカスタム環境向けのコネクタを作成できるよう、コードをリファクタリングしました。
* Kafka Connector: Timestamp および Time の Kafka 型に対応したインラインスキーマのサポートを追加しました
* Python クライアント: insert の圧縮および LowCardinality の読み取りに関する問題を修正しました

<div id="console-changes-7">
  ### コンソールの変更点
</div>

* テーブル作成時の設定オプションを拡充した新しいデータ読み込み機能を追加しました
* Cloud Console を使用して、URL からファイルを読み込めるようになりました
* 招待フローを改善し、別の組織に参加するための追加オプションや、保留中の招待をすべて確認する機能を追加しました

<div id="july-14-2023">
  ## 2023年7月14日
</div>

このリリースでは、Dedicated Services を起動できるようになったほか、オーストラリアの新しい AWS リージョンが追加され、ディスク上のデータ暗号化に独自の秘密鍵を使用できるようになりました。

<div id="general-updates-3">
  ### 全般的なアップデート
</div>

* 新しい AWS オーストラリアリージョン: シドニー (ap-southeast-2)
* 高負荷かつ低レイテンシが求められるワークロード向けの専用 tier サービス (設定するには [support](https://console.clickhouse.cloud/support) までお問い合わせください)
* ディスク上のデータを暗号化するための顧客管理の秘密鍵 (BYOK)  (設定するには [support](https://console.clickhouse.cloud/support) までお問い合わせください)

<div id="console-changes-7">
  ### コンソールの変更点
</div>

* 非同期挿入向けのオブザーバビリティ用メトリクスダッシュボードを改善
* サポート連携時のチャットボットの動作を改善

<div id="integrations-changes-7">
  ### インテグレーションの変更点
</div>

* NodeJS client: ソケットのタイムアウトによる接続失敗の不具合を修正
* Python クライアント: insert クエリに QuerySummary を追加し、データベース名で特殊文字を使用できるように対応
* Metabase: JDBC ドライバーのバージョンを更新し、DateTime64 のサポートを追加、パフォーマンスを改善。

<div id="core-database-changes">
  ### コアデータベースの変更
</div>

* ClickHouse Cloud では [クエリキャッシュ](/ja/concepts/features/performance/caches/query-cache) を有効にできます。有効にすると、成功したクエリの結果はデフォルトで 1 分間キャッシュされ、以降のクエリではそのキャッシュ済みの結果が使用されます。

<div id="june-20-2023">
  ## 2023年6月20日
</div>

このリリースでは、ClickHouse Cloud on GCP が一般提供となり、Cloud API 用の Terraform プロバイダーが追加され、ClickHouse のバージョンは 23.4 に更新されました。

* Cloud API 向けの [Terraform プロバイダー](https://registry.terraform.io/providers/ClickHouse/clickhouse/latest/docs) が利用可能になりました

<div id="console-changes-7">
  ### コンソールの変更点
</div>

* サービス用の新しい一元化された設定ページを追加
* ストレージとコンピュートのメータリング精度を調整

<div id="integrations-changes-7">
  ### インテグレーションの変更点
</div>

* Python クライアント: insert のパフォーマンスを改善し、マルチプロセシングをサポートするため内部依存関係をリファクタリング
* Kafka Connector: Confluent Cloud へのアップロードとインストールが可能になり、一時的な接続問題に対する再試行を追加し、誤ったコネクタ状態を自動的にリセット

<div id="clickhouse-234-version-upgrade">
  ### ClickHouse 23.4 へのバージョンアップ
</div>

* 並列レプリカで JOIN をサポート (設定するには [サポート](https://console.clickhouse.cloud/support) までご連絡ください)
* 論理削除のパフォーマンスを改善
* 大規模な挿入処理時のキャッシュを改善

<div id="administration-changes-1">
  ### 管理関連の変更
</div>

* 「default」以外のユーザーでもローカル Dictionary を作成できるように拡張

<div id="may-30-2023">
  ## 2023年5月30日
</div>

このリリースでは、Control Plane 操作用の ClickHouse Cloud Programmatic API の一般提供開始 (詳細は[ブログ](https://clickhouse.com/blog/using-the-new-clickhouse-cloud-api-to-automate-deployments)を参照) 、IAMロールを使用した S3 へのアクセス、追加のスケーリングオプションが導入されました。

<div id="general-changes-2">
  ### 全般的な変更
</div>

* ClickHouse Cloud の API をサポート。新しい Cloud API により、既存の CI/CD パイプラインにサービス管理をシームレスに統合し、サービスをプログラムから管理できるようになりました
* IAMロールを使用した S3 アクセス。IAMロールを活用して、プライベートな Amazon Simple Storage Service (S3) バケットに安全にアクセスできるようになりました (設定するにはサポートにお問い合わせください)

<div id="scaling-changes">
  ### スケーリングの変更
</div>

* [水平スケーリング](/ja/products/cloud/features/autoscaling/horizontal#manual-horizontal-scaling)。より高い並列処理が必要なワークロードでは、最大 10 個のレプリカまで構成できるようになりました (設定するにはサポートにお問い合わせください)
* [CPU ベースのオートスケーリング](/ja/products/cloud/features/autoscaling/overview)。CPU 負荷がボトルネックとなるワークロードで、オートスケーリング ポリシーの追加トリガーを利用できるようになりました

<div id="console-changes-7">
  ### コンソールの変更点
</div>

* Dev service を Production service に移行 (有効化するにはサポートにお問い合わせください)
* インスタンス作成フローに、スケーリング構成を制御する機能を追加
