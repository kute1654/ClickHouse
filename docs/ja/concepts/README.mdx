<div id="concepts">
  # 概念
</div>

**ここに含める内容:** ClickHouse の仕組みや、どのような機能があるかを説明するセクションです。中核となる仕組み (パーツ、マージ、パーティション、プライマリインデックス) 、ベストプラクティスの
ガイダンス、機能ごとの詳細解説 (materialized view、辞書、プロジェクション、
パフォーマンス、運用、設定、セキュリティ、バックアップ/リストア、
ツールとユーティリティ) を扱います。

**対象読者:** 機能が*何*であり、*なぜ*そのように
動作するのかを理解したいエンジニア。タスク指向の内容ではありません — 「X をどうやるか」は
`/guides` を使ってください。

**サブディレクトリ:**

* `core-concepts/` — 基本的な仕組み (パーツ、マージ、索引) 。
* `best-practices/` — ClickHouse を使ううえで押さえておくべき、実践的なガイダンス。
* `features/` — ClickHouse DB の中核機能。
* `advanced-guides/` — 他に当てはまらない、より長文の資料。

何かが何であり、どのように動作するかを説明するページを追加する場合は、ここに配置します。
実際の機能の使い方を手順付きで説明するページを追加する場合は、`/guides` に配置します。
