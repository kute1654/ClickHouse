<div id="argMin">
  ## argMin
</div>

導入バージョン: v1.1.0

最小の `val` 値に対応する `arg` の値を計算します。最小の `val` を持つ行が複数ある場合、対応する `arg` のうちどれが返されるかは決定論的ではありません。
`arg` と `min` はどちらも [aggregate functions](/ja/reference/functions/aggregate-functions/index) として振る舞い、処理中はともに [skip `Null`](/ja/reference/functions/aggregate-functions/index#null-processing) し、`Null` ではない値があれば `Null` ではない値を返します。

**関連項目**

* [Tuple](/ja/reference/data-types/tuple)

**構文**

```sql
argMin(arg, val)
```

**別名**: `min_by`

**引数**

* `arg` — 最大値を求める対象の引数。 [`const String`](/ja/reference/data-types/string)
* `val` — 最小値。 [`(U)Int8/16/32/64`](/ja/reference/data-types/int-uint) または [`Float*`](/ja/reference/data-types/float) または [`Date`](/ja/reference/data-types/date) または [`DateTime`](/ja/reference/data-types/datetime) または [`Tuple`](/ja/reference/data-types/tuple)

**戻り値**

最小の `val` 値に対応する `arg` の値を返します。戻り値の型は `arg` と同じです。

**例**

**基本的な使い方**

```sql title=Query
SELECT argMin(user, salary) FROM salary;
```

```response title=Response
┌─argMin(user, salary)─┐
│ worker               │
└──────────────────────┘
```

**NULL を扱う拡張例**

```sql title=Query
CREATE TABLE test
(
    a Nullable(String),
    b Nullable(Int64)
)
ENGINE = Memory AS
SELECT *
FROM VALUES((NULL, 0), ('a', 1), ('b', 2), ('c', 2), (NULL, NULL), ('d', NULL));

SELECT argMin(a, b), min(b) FROM test;
```

```response title=Response
┌─argMin(a, b)─┬─min(b)─┐
│ a            │      0 │
└──────────────┴────────┘
```

**引数でTupleを使用する**

```sql title=Query
SELECT argMin(a, (b, a)), min(tuple(b, a)) FROM test;
```

```response title=Response
┌─argMin(a, tuple(b, a))─┬─min(tuple(b, a))─┐
│ d                      │ (NULL,NULL)      │
└────────────────────────┴──────────────────┘
```
