<div id="argAndMin">
  ## argAndMin
</div>

導入バージョン: v1.1.0

最小の `val` 値に対応する `arg` と `val` の値を計算します。
最小値に等しい `val` を持つ行が複数ある場合、対応する `arg` と `val` のうちどれが返されるかは決定論的ではありません。
`arg` と `min` はどちらも [集約関数](/ja/reference/functions/aggregate-functions/index) として動作し、処理中はともに [`Null` をスキップ](/ja/reference/functions/aggregate-functions/index#null-processing) し、`Null` でない値が存在する場合は `Null` でない値を返します。

<Note>
  `argMin` との唯一の違いは、`argAndMin` が引数と値の両方を返すことです。
</Note>

**関連項目**

* [argMin](/ja/reference/functions/aggregate-functions/argMin)
* [Tuple](/ja/reference/data-types/tuple)

**構文**

```sql
argAndMin(arg, val)
```

**引数**

* `arg` — 最小値を求める対象の引数です。[`const String`](/ja/reference/data-types/string)
* `val` — 最小値です。[`(U)Int8/16/32/64`](/ja/reference/data-types/int-uint) または [`Float*`](/ja/reference/data-types/float) または [`Date`](/ja/reference/data-types/date) または [`DateTime`](/ja/reference/data-types/datetime) または [`Tuple`](/ja/reference/data-types/tuple)

**戻り値**

最小の `val` 値に対応する `arg` 値と、その最小の `val` 値を含むタプルを返します。[`Tuple`](/ja/reference/data-types/tuple)

**例**

**基本的な使い方**

```sql title=Query
SELECT argAndMin(user, salary) FROM salary;
```

```response title=Response
┌─argAndMin(user, salary)─┐
│ ('worker',1000)         │
└─────────────────────────┘
```

**NULL処理を含む拡張例**

```sql title=Query
CREATE TABLE test
(
    a Nullable(String),
    b Nullable(Int64)
)
ENGINE = Memory AS
SELECT *
FROM VALUES((NULL, 0), ('a', 1), ('b', 2), ('c', 2), (NULL, NULL), ('d', NULL));

SELECT argMin(a,b), argAndMin(a, b), min(b) FROM test;
```

```response title=Response
┌─argMin(a, b)─┬─argAndMin(a, b)─┬─min(b)─┐
│ a            │ ('a',1)         │      0 │
└──────────────┴─────────────────┴────────┘
```

**引数でのTupleの使用**

```sql title=Query
SELECT argAndMin(a, (b, a)), min(tuple(b, a)) FROM test;
```

```response title=Response
┌─argAndMin(a, (b, a))─┬─min((b, a))─┐
│ ('a',(1,'a'))        │ (0,NULL)    │
└──────────────────────┴─────────────┘
```
