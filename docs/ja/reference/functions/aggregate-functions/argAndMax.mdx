<div id="argAndMax">
  ## argAndMax
</div>

導入バージョン: v1.1.0

最大の `val` に対応する `arg` と `val` を計算します。
最大値である同じ `val` を持つ行が複数ある場合、対応する `arg` と `val` のうちどれが返されるかは決定論的ではありません。
`arg` 部分と `max` 部分はどちらも [集約関数](/ja/reference/functions/aggregate-functions/index) として動作し、処理中はともに [`Null` をスキップ](/ja/reference/functions/aggregate-functions/index#null-processing) し、`Null` 以外の値が存在する場合は `Null` 以外の値を返します。

<Note>
  `argMax` との唯一の違いは、`argAndMax` が引数と値の両方を返すことです。
</Note>

**関連項目**

* [argMax](/ja/reference/functions/aggregate-functions/argMax)
* [Tuple](/ja/reference/data-types/tuple)

**構文**

```sql
argAndMax(arg, val)
```

**引数**

* `arg` — 最大値を求める対象の引数。[`const String`](/ja/reference/data-types/string)
* `val` — 最大値。[`(U)Int8/16/32/64`](/ja/reference/data-types/int-uint) または [`Float*`](/ja/reference/data-types/float) または [`Date`](/ja/reference/data-types/date) または [`DateTime`](/ja/reference/data-types/datetime) または [`Tuple`](/ja/reference/data-types/tuple)

**戻り値**

最大の `val` 値に対応する `arg` の値と、その最大の `val` 値を含むタプルを返します。[`Tuple`](/ja/reference/data-types/tuple)

**例**

**基本的な使い方**

```sql title=Query
SELECT argAndMax(user, salary) FROM salary;
```

```response title=Response
┌─argAndMax(user, salary)─┐
│ ('director',5000)       │
└─────────────────────────┘
```

**NULL処理を含む拡張例**

```sql title=Query
CREATE TABLE test
(
    a Nullable(String),
    b Nullable(Int64)
)
ENGINE = Memory AS
SELECT *
FROM VALUES(('a', 1), ('b', 2), ('c', 2), (NULL, 3), (NULL, NULL), ('d', NULL));

SELECT argMax(a, b), argAndMax(a, b), max(b) FROM test;
```

```response title=Response
┌─argMax(a, b)─┬─argAndMax(a, b)─┬─max(b)─┐
│ b            │ ('b',2)         │      3 │
└──────────────┴─────────────────┴────────┘
```

**引数でTupleを使用する**

```sql title=Query
SELECT argAndMax(a, (b,a)) FROM test;
```

```response title=Response
┌─argAndMax(a, (b, a))─┐
│ ('c',(2,'c'))        │
└──────────────────────┘
```
