MergeTreeテーブルの索引ファイルおよびmarkファイルの内容を表します。イントロスペクションに使用できます。

<div id="syntax">
  ## 構文
</div>

```sql
mergeTreeIndex(database, table [, with_marks = true] [, with_minmax = true])
```

<div id="arguments">
  ## 引数
</div>

| 引数            | 説明                      |
| ------------- | ----------------------- |
| `database`    | 索引とマークを読み取る対象のデータベース名。  |
| `table`       | 索引とマークを読み取る対象のテーブル名。    |
| `with_marks`  | 結果にマーク付きのカラムを含めるかどうか。   |
| `with_minmax` | 結果に min-max 索引を含めるかどうか。 |

<div id="returned_value">
  ## 戻り値
</div>

ログソーステーブルのプライマリインデックスおよび min-max 索引 (有効な場合) の値を持つカラム、ログソーステーブルのデータパーツ内の取り得るすべてのファイルについて マーク (有効な場合) の値を持つカラム、ならびに仮想カラムを含むテーブルオブジェクトです。

* `part_name` - データパーツの名前。
* `mark_number` - データパーツ内の現在の mark の番号。
* `rows_in_granule` - 現在の granule 内の行数。

カラムがデータパーツ内に存在しない場合、またはそのサブストリームのいずれかの マーク が書き込まれていない場合 (たとえば compact パーツ) には、マーク カラムに `(NULL, NULL)` が含まれることがあります。

<div id="usage-example">
  ## 使用例
</div>

```sql
CREATE TABLE test_table
(
    `id` UInt64,
    `n` UInt64,
    `arr` Array(UInt64)
)
ENGINE = MergeTree
ORDER BY id
SETTINGS index_granularity = 3, min_bytes_for_wide_part = 0, min_rows_for_wide_part = 8;

INSERT INTO test_table SELECT number, number, range(number % 5) FROM numbers(5);

INSERT INTO test_table SELECT number, number, range(number % 5) FROM numbers(10, 10);
```

```sql
SELECT * FROM mergeTreeIndex(currentDatabase(), test_table, with_marks = true);
```

```sql
DESCRIBE mergeTreeIndex(currentDatabase(), test_table, with_marks = true) SETTINGS describe_compact_output = 1;
```
