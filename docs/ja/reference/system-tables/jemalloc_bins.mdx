<div id="description">
  ## 説明
</div>

すべてのアリーナから集計した、jemalloc アロケータによる各サイズクラス (ビン) のメモリ割り当てに関する情報が含まれています。
jemalloc のスレッドローカルキャッシュの影響により、これらの統計は必ずしも完全に正確とは限りません。

<div id="columns">
  ## カラム
</div>

{/*AUTOGENERATED_START*/}

* `index` ([UInt16](/ja/reference/data-types/index)) — サイズ順に並んだビンの索引。
* `large` ([UInt8](/ja/reference/data-types/index)) — 大きな割り当ての場合はTrue、小さな割り当ての場合はFalseです。
* `size` ([UInt64](/ja/reference/data-types/index)) — このビンにおける割り当てサイズ。
* `allocations` ([Int64](/ja/reference/data-types/index)) — 割り当て数。
* `deallocations` ([Int64](/ja/reference/data-types/index)) — 解放数。
* `nregs` ([Int64](/ja/reference/data-types/index)) — スラブごとのリージョン数。
* `curslabs` ([Int64](/ja/reference/data-types/index)) — 現在のスラブ数。
* `curregs` ([Int64](/ja/reference/data-types/index)) — このサイズクラスにおける現在のリージョン数。

{/*AUTOGENERATED_END*/}

<div id="example">
  ## 例
</div>

現在のメモリ使用量全体に最も大きく寄与している割り当てのサイズを確認します。

```sql
SELECT
    *,
    allocations - deallocations AS active_allocations,
    size * active_allocations AS allocated_bytes
FROM system.jemalloc_bins
WHERE allocated_bytes > 0
ORDER BY allocated_bytes DESC
LIMIT 10
```
