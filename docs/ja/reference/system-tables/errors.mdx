<div id="description">
  ## 説明
</div>

発生回数とともにエラーコードが含まれます。

発生していないものも含め、考えられるすべてのエラーコードを表示するには、設定 [system&#95;events&#95;show&#95;zero&#95;values](/ja/reference/settings/session-settings#system_events_show_zero_values) を 1 に設定します。

<div id="columns">
  ## カラム
</div>

{/*AUTOGENERATED_START*/}

* `name` ([String](/ja/reference/data-types/index)) — エラー名 (errorCodeToName) 。
* `code` ([Int32](/ja/reference/data-types/index)) — エラーコード番号。
* `value` ([UInt64](/ja/reference/data-types/index)) — このエラーが発生した回数。
* `last_error_time` ([DateTime](/ja/reference/data-types/index)) — 最後にエラーが発生した時刻。
* `last_error_message` ([String](/ja/reference/data-types/index)) — 最後のエラーメッセージ。
* `last_error_format_string` ([String](/ja/reference/data-types/index)) — 最後のエラーのフォーマット文字列。
* `last_error_trace` ([Array(UInt64)](/ja/reference/data-types/index)) — 呼び出されたメソッドが保存されている物理アドレスの一覧を表すスタックトレース。
* `remote` ([UInt8](/ja/reference/data-types/index)) — リモート例外 (つまり、いずれかの分散クエリ中に受信したもの) 。
* `query_id` ([String](/ja/reference/data-types/index)) — エラーの原因となったクエリの ID (利用可能な場合) 。

{/*AUTOGENERATED_END*/}

<Note>
  一部のエラーでは、クエリが正常に実行されていてもカウンターが増加することがあります。対応するエラーが誤検知ではないと確信できる場合を除き、このテーブルをサーバーの監視用途に使用することは推奨されません。
</Note>

<div id="example">
  ## 例
</div>

```sql title="Query"
SELECT name, code, value
FROM system.errors
WHERE value > 0
ORDER BY code ASC
LIMIT 1

┌─name─────────────┬─code─┬─value─┐
│ CANNOT_OPEN_FILE │   76 │     1 │
└──────────────────┴──────┴───────┘
```

```sql title="Response"
WITH arrayMap(x -> demangle(addressToSymbol(x)), last_error_trace) AS all
SELECT name, arrayStringConcat(all, '\n') AS res
FROM system.errors
LIMIT 1
SETTINGS allow_introspection_functions=1\G
```
