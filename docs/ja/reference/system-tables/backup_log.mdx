<div id="description">
  ## 説明
</div>

`BACKUP` および `RESTORE` 操作に関する情報を含むログエントリです。

<div id="columns">
  ## カラム
</div>

{/*AUTOGENERATED_START*/}

* `hostname` ([LowCardinality(String)](/ja/reference/data-types/lowcardinality)) — クエリを実行しているサーバーのホスト名。
* `event_date` ([Date](/ja/reference/data-types/date)) — エントリの日付。
* `event_time` ([DateTime](/ja/reference/data-types/datetime)) — エントリの時刻。
* `event_time_microseconds` ([DateTime64(6)](/ja/reference/data-types/datetime64)) — マイクロ秒精度のエントリ時刻。
* `id` ([String](/ja/reference/data-types/string)) — バックアップまたは復元操作の識別子。
* `name` ([String](/ja/reference/data-types/string)) — バックアップストレージの名前 (FROM 句または TO 句の内容) 。
* `base_backup_name` ([String](/ja/reference/data-types/string)) — インクリメンタルバックアップの場合のベースバックアップ名。
* `query_id` ([String](/ja/reference/data-types/string)) — バックアップ操作に関連付けられたクエリの ID。
* `status` ([Enum8(&#39;CREATING&#95;BACKUP&#39; = 0, &#39;BACKUP&#95;CREATED&#39; = 1, &#39;BACKUP&#95;FAILED&#39; = 2, &#39;RESTORING&#39; = 3, &#39;RESTORED&#39; = 4, &#39;RESTORE&#95;FAILED&#39; = 5, &#39;BACKUP&#95;CANCELLED&#39; = 6, &#39;RESTORE&#95;CANCELLED&#39; = 7)](/ja/reference/data-types/enum)) — 操作の状態。
* `error` ([String](/ja/reference/data-types/string)) — 失敗した操作のエラーメッセージ (成功した操作では空文字列) 。
* `start_time` ([DateTime64(6)](/ja/reference/data-types/datetime64)) — 操作の開始時刻。
* `end_time` ([DateTime64(6)](/ja/reference/data-types/datetime64)) — 操作の終了時刻。
* `num_files` ([UInt64](/ja/reference/data-types/int-uint)) — バックアップに格納されているファイル数。
* `total_size` ([UInt64](/ja/reference/data-types/int-uint)) — バックアップに格納されているファイルの合計サイズ。
* `num_entries` ([UInt64](/ja/reference/data-types/int-uint)) — バックアップ内のエントリ数。つまり、バックアップがフォルダーとして保存されている場合はそのフォルダー内のファイル数、アーカイブとして保存されている場合はそのアーカイブ内のファイル数です。インクリメンタルバックアップである場合や、空ファイルまたは重複ファイルを含む場合は、`num_files` と一致しません。常に `num_entries ≤ num_files` が成り立ちます。
* `uncompressed_size` ([UInt64](/ja/reference/data-types/int-uint)) — バックアップの非圧縮サイズ。
* `compressed_size` ([UInt64](/ja/reference/data-types/int-uint)) — バックアップの圧縮サイズ。バックアップがアーカイブとして保存されていない場合は、uncompressed&#95;size と等しくなります。
* `files_read` ([UInt64](/ja/reference/data-types/int-uint)) — 復元操作中に読み取られたファイル数。
* `bytes_read` ([UInt64](/ja/reference/data-types/int-uint)) — 復元操作中に読み取られたファイルの合計サイズ。

{/*AUTOGENERATED_END*/}
