<div id="description">
  ## 説明
</div>

[MergeTree](/ja/reference/engines/table-engines/mergetree-family/mergetree) テーブルのパーツに関する情報を含みます。

各行は 1 つのデータパーツを表します。

<div id="columns">
  ## カラム
</div>

{/*AUTOGENERATED_START*/}

* `partition` ([String](/ja/reference/data-types/index)) — パーティション識別子 (テーブルのパーティションキー式から導出された文字列表現) 。
* `name` ([String](/ja/reference/data-types/index)) —
  データパーツの名前。パーツの命名構造を使うと、データ、取り込み、マージのパターンに関するさまざまな特性を判断できます。パーツの命名フォーマットは次のとおりです。

```text
<partition_id>_<minimum_block_number>_<maximum_block_number>_<level>_<data_version>
```

* 定義:
  * `partition_id` - パーティションキーを識別します
  * `minimum_block_number` - パーツ内の最小ブロック番号を示します。ClickHouse は常に連続するブロックをマージします
  * `maximum_block_number` - パーツ内の最大ブロック番号を示します
  * `level` - そのパーツに対してマージが 1 回追加されるごとに 1 ずつ増加します。レベルが 0 の場合、これはまだマージされていない新しいパーツであることを示します。ClickHouse のすべてのパーツは常に不変であることを覚えておくことが重要です
  * `data_version` - オプションの値で、パーツが変更されると増加します (繰り返しになりますが、パーツは不変であるため、変更されたデータも常に新しいパーツにのみ書き込まれます)

* `uuid` ([UUID](/ja/reference/data-types/index)) — データパーツの UUID。

* `part_type` ([String](/ja/reference/data-types/index)) — データパーツの保存フォーマット。設定可能な値: `Wide` — 各カラムが個別のファイルに保存されます。`Compact` — すべてのカラムが 1 つのファイルに保存されます。データの保存フォーマットは、MergeTree テーブルの `min_bytes_for_wide_part` および `min_rows_for_wide_part` 設定によって制御されます。

* `active` ([UInt8](/ja/reference/data-types/index)) — データパーツがアクティブかどうかを示すフラグ。データパーツがアクティブな場合、そのテーブルで使用されます。そうでない場合は、削除される予定です。非アクティブなデータパーツは、マージ操作や mutation 操作の後に現れます。

* `marks` ([UInt64](/ja/reference/data-types/index)) — マーク数。データパーツ内のおおよその行数を求めるには、マーク数に index granularity (通常は 8192) を掛けます (この方法は adaptive granularity では機能しません) 。

* `rows` ([UInt64](/ja/reference/data-types/index)) — 行数。

* `files` ([UInt64](/ja/reference/data-types/index)) — データパーツ内のファイル数。

* `bytes_on_disk` ([UInt64](/ja/reference/data-types/index)) — すべてのデータパーツファイルの合計サイズ (バイト単位) 。

* `data_compressed_bytes` ([UInt64](/ja/reference/data-types/index)) — データパーツ内の圧縮データの合計サイズ。補助ファイル (たとえば、マークを含むファイル) は含まれません。

* `data_uncompressed_bytes` ([UInt64](/ja/reference/data-types/index)) — データパーツ内の非圧縮データの合計サイズ。補助ファイル (たとえば、マークを含むファイル) は含まれません。

* `primary_key_size` ([UInt64](/ja/reference/data-types/index)) — ディスク上の primary.idx/cidx ファイル内で主キーの値が使用するメモリ量 (バイト単位) 。

* `marks_bytes` ([UInt64](/ja/reference/data-types/index)) — マークを含むファイルのサイズ。

* `secondary_indices_compressed_bytes` ([UInt64](/ja/reference/data-types/index)) — データパーツ内のセカンダリ索引用圧縮データの合計サイズ。補助ファイル (たとえば、マークを含むファイル) は含まれません。

* `secondary_indices_uncompressed_bytes` ([UInt64](/ja/reference/data-types/index)) — データパーツ内のセカンダリ索引用非圧縮データの合計サイズ。補助ファイル (たとえば、マークを含むファイル) は含まれません。

* `secondary_indices_marks_bytes` ([UInt64](/ja/reference/data-types/index)) — セカンダリ索引のマークを含むファイルのサイズ。

* `modification_time` ([DateTime](/ja/reference/data-types/index)) — データパーツを含むディレクトリが変更された時刻。通常、これはデータパーツの作成時刻に対応します。

* `remove_time` ([DateTime](/ja/reference/data-types/index)) — データパーツが非アクティブになった時刻。

* `refcount` ([UInt32](/ja/reference/data-types/index)) — データパーツが使用されている箇所の数。値が 2 より大きい場合、そのデータパーツはクエリまたはマージで使用されています。

* `min_date` ([Date](/ja/reference/data-types/index)) — データパーツ内の日付キーの最小値。

* `max_date` ([Date](/ja/reference/data-types/index)) — データパーツ内の日付キーの最大値。

* `min_time` ([DateTime](/ja/reference/data-types/index)) — データパーツ内の日付と時刻キーの最小値。

* `max_time` ([DateTime](/ja/reference/data-types/index)) — データパーツ内の日付と時刻キーの最大値。

* `partition_id` ([String](/ja/reference/data-types/index)) — パーティションの ID。

* `min_block_number` ([Int64](/ja/reference/data-types/index)) — マージ後に現在のパーツを構成するデータパーツの最小番号。

* `max_block_number` ([Int64](/ja/reference/data-types/index)) — マージ後に現在のパーツを構成するデータパーツの最大番号。

* `level` ([UInt32](/ja/reference/data-types/index)) — マージツリーの深さ。0 は、現在のパーツが他のパーツのマージではなく挿入によって作成されたことを意味します。

* `data_version` ([UInt64](/ja/reference/data-types/index)) — どの mutation をデータパートに適用するかを判定するための数値 (`data_version` より大きいバージョンの mutation) 。

* `primary_key_bytes_in_memory` ([UInt64](/ja/reference/data-types/index)) — 主キーの値に使用されるメモリ量 (バイト単位) 。`primary_key_lazy_load` が有効で、かつキーが読み込まれていない場合は 0 になります。0 以外の場合、これらのバイトは part 自体に保持され、`jemalloc.mergetree_arena.active_bytes` に計上されます。これらが `PrimaryIndexCacheBytes` に計上されることは決してありません。part ごとに両者は相互排他的です。つまり、`primary_key_lazy_load` と `use_primary_key_cache` に応じて、索引は part 内 (このメトリクス) または共有の `PrimaryIndexCache` (もう一方) に存在します。

* `primary_key_bytes_in_memory_allocated` ([UInt64](/ja/reference/data-types/index)) — 主キーの値のために予約されるメモリ量 (バイト単位) 。`primary_key_lazy_load` が有効で、かつキーが読み込まれていない場合は 0 になります。0 以外の場合、`jemalloc.mergetree_arena.active_bytes` に含まれます。`PrimaryIndexCacheBytes` との関係については、`primary_key_bytes_in_memory` の注記を参照してください。

* `index_granularity_bytes_in_memory` ([UInt64](/ja/reference/data-types/index)) — 索引粒度の値に使用されるメモリ量 (バイト単位)  (primary&#95;key&#95;lazy&#95;load=1 かつ use&#95;primary&#95;key&#95;cache=1 の場合は 0 になります) 。0 以外の場合、これらのバイトは part に保持され、`jemalloc.mergetree_arena.active_bytes` に計上されます。

* `index_granularity_bytes_in_memory_allocated` ([UInt64](/ja/reference/data-types/index)) — 索引粒度の値のために予約されるメモリ量 (バイト単位)  (primary&#95;key&#95;lazy&#95;load=1 かつ use&#95;primary&#95;key&#95;cache=1 の場合は 0 になります) 。0 以外の場合、`jemalloc.mergetree_arena.active_bytes` に含まれます。

* `is_frozen` ([UInt8](/ja/reference/data-types/index)) — パーティションのデータバックアップが存在することを示すフラグ。1 はバックアップが存在することを示し、0 はバックアップが存在しないことを示します。詳細については、FREEZE PARTITION を参照してください。

* `database` ([String](/ja/reference/data-types/index)) — データベース名。

* `table` ([String](/ja/reference/data-types/index)) — テーブル名。

* `engine` ([String](/ja/reference/data-types/index)) — パラメーターを含まないテーブルエンジン名。

* `disk_name` ([String](/ja/reference/data-types/index)) — データパートを格納するディスクの名前。

* `path` ([String](/ja/reference/data-types/index)) — データパートのファイルを含むフォルダーへの絶対パス。

* `hash_of_all_files` ([String](/ja/reference/data-types/index)) — 圧縮ファイルの sipHash128。

* `hash_of_uncompressed_files` ([String](/ja/reference/data-types/index)) — 非圧縮ファイル (マーク、索引ファイルなど) の sipHash128。

* `uncompressed_hash_of_compressed_files` ([String](/ja/reference/data-types/index)) — 圧縮ファイル内のデータを、あたかも非圧縮であるかのように扱った場合の sipHash128。

* `delete_ttl_info_min` ([DateTime](/ja/reference/data-types/index)) — TTL DELETE ルールに対する日付と時刻キーの最小値。

* `delete_ttl_info_max` ([DateTime](/ja/reference/data-types/index)) — TTL DELETE ルールに対する日付と時刻キーの最大値。

* `move_ttl_info.expression` ([Array(String)](/ja/reference/data-types/index)) — 式の Array。各式は TTL MOVE ルールを定義します。

* `move_ttl_info.min` ([Array(DateTime)](/ja/reference/data-types/index)) — 日付と時刻の値の Array。各要素は TTL MOVE ルールの最小キー値を表します。

* `move_ttl_info.max` ([Array(DateTime)](/ja/reference/data-types/index)) — 日付と時刻の値の Array。各要素は TTL MOVE ルールの最大キー値を表します。

* `default_compression_codec` ([String](/ja/reference/data-types/index)) — このデータパートの圧縮に使用される codec の名前 (カラムに明示的な codec がない場合) 。

* `recompression_ttl_info.expression` ([Array(String)](/ja/reference/data-types/index)) — TTL 式。

* `recompression_ttl_info.min` ([Array(DateTime)](/ja/reference/data-types/index)) — この part 内で計算された TTL 式の最小値。少なくとも 1 行に有効期限切れの TTL があるかどうかを把握するために使用されます。

* `recompression_ttl_info.max` ([Array(DateTime)](/ja/reference/data-types/index)) — このパート内で計算された有効期限 (TTL) 式の最大値。TTL が期限切れの行をこのパートがすべて含んでいるかどうかを把握するために使用されます。

* `group_by_ttl_info.expression` ([Array(String)](/ja/reference/data-types/index)) — 有効期限 (TTL) 式。

* `group_by_ttl_info.min` ([Array(DateTime)](/ja/reference/data-types/index)) — このパート内で計算された有効期限 (TTL) 式の最小値。TTL が期限切れの行をこのパートが少なくとも 1 行含んでいるかどうかを把握するために使用されます。

* `group_by_ttl_info.max` ([Array(DateTime)](/ja/reference/data-types/index)) — このパート内で計算された有効期限 (TTL) 式の最大値。TTL が期限切れの行をこのパートがすべて含んでいるかどうかを把握するために使用されます。

* `rows_where_ttl_info.expression` ([Array(String)](/ja/reference/data-types/index)) — 有効期限 (TTL) 式。

* `rows_where_ttl_info.min` ([Array(DateTime)](/ja/reference/data-types/index)) — このパート内で計算された有効期限 (TTL) 式の最小値。TTL が期限切れの行をこのパートが少なくとも 1 行含んでいるかどうかを把握するために使用されます。

* `rows_where_ttl_info.max` ([Array(DateTime)](/ja/reference/data-types/index)) — このパート内で計算された有効期限 (TTL) 式の最大値。TTL が期限切れの行をこのパートがすべて含んでいるかどうかを把握するために使用されます。

* `projections` ([Array(String)](/ja/reference/data-types/index)) — このパートに対して計算されたプロジェクション名の一覧。

* `visible` ([UInt8](/ja/reference/data-types/index)) — このパートが SELECT クエリで可視かどうかを示すフラグ。

* `creation_tid` ([Tuple(UInt64, UInt64, UUID)](/ja/reference/data-types/index)) — このオブジェクトを作成した、または作成しようとしているトランザクションの ID。

* `removal_tid_lock` ([UInt64](/ja/reference/data-types/index)) — removal&#95;tid のハッシュ。削除のためにオブジェクトをロックするのに使用されます。

* `removal_tid` ([Tuple(UInt64, UInt64, UUID)](/ja/reference/data-types/index)) — このオブジェクトを削除した、または削除しようとしているトランザクションの ID。

* `creation_csn` ([UInt64](/ja/reference/data-types/index)) — このオブジェクトを作成したトランザクションの CSN。

* `removal_csn` ([UInt64](/ja/reference/data-types/index)) — このオブジェクトを削除したトランザクションの CSN。

* `has_lightweight_delete` ([UInt8](/ja/reference/data-types/index)) — このパートに論理削除マスクがあるかどうかを示すフラグ。

* `last_removal_attempt_time` ([DateTime](/ja/reference/data-types/index)) — サーバーがこのパートの削除を最後に試みた時刻。

* `removal_state` ([String](/ja/reference/data-types/index)) — パート削除プロセスの現在の状態。
