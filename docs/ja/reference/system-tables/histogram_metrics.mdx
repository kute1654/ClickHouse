<div id="description">
  ## 説明
</div>

このテーブルには、即時に計算でき、Prometheus フォーマットでエクスポート可能なヒストグラムメトリクスが含まれています。内容は常に最新です。非推奨の `system.latency_log` を置き換えるものです。

<div id="columns">
  ## カラム
</div>

{/*AUTOGENERATED_START*/}

* `metric` ([String](/ja/reference/data-types/index)) — メトリック名。
* `value` ([Float64](/ja/reference/data-types/index)) — メトリック値。
* `description` ([String](/ja/reference/data-types/index)) — メトリックの説明。
* `labels` ([Map(String, String)](/ja/reference/data-types/index)) — メトリックのラベル。

{/*AUTOGENERATED_END*/}

<div id="example">
  ## 例
</div>

次のようなクエリを使用すると、すべてのヒストグラムメトリクスをPrometheusフォーマットでエクスポートできます。

```sql
SELECT
  metric AS name,
  toFloat64(value) AS value,
  description AS help,
  labels,
  'histogram' AS type
FROM system.histogram_metrics
FORMAT Prometheus
```

<div id="http_pool_tcp_buf_bytes">
  ### http_pool_tcp_buf_bytes
</div>

HTTP 接続プールソケットのカーネル TCP バッファメモリ (`sk_rmem_alloc` / `sk_wmem_alloc`) 。非同期メトリクスがスクレイプされるたびに、各稼働中のソケットについて 1 回観測されます。ラベル: `group` は `disk`、`storage`、`http` のいずれか、`direction` は `rcv` または `snd` です。
