<div id="description">
  ## 説明
</div>

ローカルサーバー上に存在する[リソース](/ja/concepts/features/configuration/server-config/workload-scheduling#workload_entity_storage)に関する情報が含まれています。このテーブルには、各リソースに対応する行が1つずつ含まれます。

<div id="columns">
  ## カラム
</div>

{/*AUTOGENERATED_START*/}

* `name` ([String](/ja/reference/data-types/index)) — リソースの名前。
* `read_disks` ([Array(String)](/ja/reference/data-types/index)) — 読み取り操作でこのリソースを使用するディスク名の一覧。
* `write_disks` ([Array(String)](/ja/reference/data-types/index)) — 書き込み操作でこのリソースを使用するディスク名の一覧。
* `unit` ([String](/ja/reference/data-types/index)) — コスト計測に使用されるリソースの単位。
* `create_query` ([String](/ja/reference/data-types/index)) — リソースの CREATE クエリ。

{/*AUTOGENERATED_END*/}

例:

```sql
SELECT *
FROM system.resources
FORMAT Vertical
```

```text
Row 1:
──────
name:         io_read
read_disks:   ['s3']
write_disks:  []
create_query: CREATE RESOURCE io_read (READ DISK s3)

Row 2:
──────
name:         io_write
read_disks:   []
write_disks:  ['s3']
create_query: CREATE RESOURCE io_write (WRITE DISK s3)
```
