<div id="description">
  ## 説明
</div>

クエリ実行時に実行される依存ビューに関する情報 (たとえば、ビューの種類や実行時間など) が含まれます。

ログの記録を開始するには、次の手順を実行します。

1. [query&#95;views&#95;log](/ja/reference/settings/server-settings/settings#query_views_log) セクションでパラメータを設定します。
2. [log&#95;query&#95;views](/ja/reference/settings/session-settings#log_query_views) を 1 に設定します。

データのフラッシュ間隔は、[query&#95;views&#95;log](/ja/reference/settings/server-settings/settings#query_views_log) サーバー設定セクションの `flush_interval_milliseconds` パラメータで設定します。フラッシュを強制するには、[SYSTEM FLUSH LOGS](/ja/reference/statements/system#flush-logs) クエリを使用します。

ClickHouse はテーブルからデータを自動では削除しません。詳細については、[Introduction](/ja/reference/system-tables/overview#system-tables-introduction) を参照してください。

`query_views_log` テーブルに記録されるクエリ数を減らすには、[log&#95;queries&#95;probability](/ja/reference/settings/session-settings#log_queries_probability)) 設定を使用できます。

<div id="columns">
  ## カラム
</div>

{/*AUTOGENERATED_START*/}

* `hostname` ([LowCardinality(String)](/ja/reference/data-types/lowcardinality)) — クエリを実行するサーバーのホスト名。
* `event_date` ([Date](/ja/reference/data-types/date)) — ビューで最後のイベントが発生した日付。
* `event_time` ([DateTime](/ja/reference/data-types/datetime)) — ビューの実行が完了した日時。
* `event_time_microseconds` ([DateTime64(6)](/ja/reference/data-types/datetime64)) — ビューの実行が完了した日時 (マイクロ秒精度) 。
* `view_duration_ms` ([UInt64](/ja/reference/data-types/int-uint)) — ビューの実行時間 (各ステージの合計、ミリ秒単位) 。
* `initial_query_id` ([String](/ja/reference/data-types/string)) — 初期クエリの ID (分散クエリ実行時) 。
* `view_name` ([String](/ja/reference/data-types/string)) — ビュー名。
* `view_uuid` ([UUID](/ja/reference/data-types/uuid)) — ビューの UUID。
* `view_type` ([Enum8(&#39;Default&#39; = 1, &#39;Materialized&#39; = 2, &#39;Live&#39; = 3, &#39;Window&#39; = 4)](/ja/reference/data-types/enum)) — ビューの種類。値: &#39;Default&#39; = 1 — デフォルトビュー。このログには現れないはずです。&#39;Materialized&#39; = 2 — materialized view、&#39;Live&#39; = 3 — Live view。
* `view_query` ([String](/ja/reference/data-types/string)) — ビューが実行したクエリ。
* `view_target` ([String](/ja/reference/data-types/string)) — ビューのターゲットテーブル名。
* `read_rows` ([UInt64](/ja/reference/data-types/int-uint)) — 読み取った行数。
* `read_bytes` ([UInt64](/ja/reference/data-types/int-uint)) — 読み取ったバイト数。
* `written_rows` ([UInt64](/ja/reference/data-types/int-uint)) — 書き込んだ行数。
* `written_bytes` ([UInt64](/ja/reference/data-types/int-uint)) — 書き込んだバイト数。
* `peak_memory_usage` ([Int64](/ja/reference/data-types/int-uint)) — このビューのコンテキストにおける、割り当て済みメモリ量と解放済みメモリ量の差の最大値。
* `ProfileEvents` ([Map(LowCardinality(String), UInt64)](/ja/reference/data-types/map)) — 各種メトリクスを測定する ProfileEvents。各項目の説明は system.events テーブルにあります。
* `status` ([Enum8(&#39;QueryStart&#39; = 1, &#39;QueryFinish&#39; = 2, &#39;ExceptionBeforeStart&#39; = 3, &#39;ExceptionWhileProcessing&#39; = 4)](/ja/reference/data-types/enum)) — ビューのステータス。値: &#39;QueryStart&#39; = 1 — ビュー実行の開始に成功。現れないはずです。&#39;QueryFinish&#39; = 2 — ビュー実行の正常終了、&#39;ExceptionBeforeStart&#39; = 3 — ビュー実行開始前の例外、&#39;ExceptionWhileProcessing&#39; = 4 — ビュー実行中の例外。
* `exception_code` ([Int32](/ja/reference/data-types/int-uint)) — 例外コード。
* `exception` ([String](/ja/reference/data-types/string)) — 例外メッセージ。
* `stack_trace` ([String](/ja/reference/data-types/string)) — スタックトレース。クエリが正常に完了した場合は空文字列です。

**別名:**

* `ProfileEvents.Names` — `mapKeys(ProfileEvents)` の別名。
* `ProfileEvents.Values` — `mapValues(ProfileEvents)` の別名。

{/*AUTOGENERATED_END*/}
