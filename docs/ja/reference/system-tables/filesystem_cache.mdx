<div id="description">
  ## 説明
</div>

リモートオブジェクトに対するファイルシステムキャッシュ内のすべてのエントリに関する情報が含まれます。

<div id="columns">
  ## カラム
</div>

{/*AUTOGENERATED_START*/}

* `cache_name` ([String](/ja/reference/data-types/index)) — キャッシュオブジェクトの名前。`SYSTEM DESCRIBE FILESYSTEM CACHE <name>`、`SYSTEM DROP FILESYSTEM CACHE <name>` コマンドで使用できます
* `cache_base_path` ([String](/ja/reference/data-types/index)) — すべてのキャッシュファイル (`cache_name` で識別されるキャッシュ) の保存先となるベースディレクトリのパス。
* `cache_path` ([String](/ja/reference/data-types/index)) — ソースファイル内のファイルセグメントに対応する、特定のキャッシュファイルのパス
* `key` ([String](/ja/reference/data-types/index)) — ファイルセグメントのキャッシュキー
* `file_segment_range_begin` ([UInt64](/ja/reference/data-types/index)) — ファイルセグメント範囲の先頭に対応するオフセット
* `file_segment_range_end` ([UInt64](/ja/reference/data-types/index)) — ファイルセグメント範囲の末尾 (この位置を含む) に対応するオフセット
* `size` ([UInt64](/ja/reference/data-types/index)) — ファイルセグメントのサイズ
* `state` ([String](/ja/reference/data-types/index)) — ファイルセグメントの状態 (DOWNLOADED、DOWNLOADING、PARTIALLY&#95;DOWNLOADED、...)
* `finished_download_time` ([DateTime](/ja/reference/data-types/index)) — ファイルセグメントのダウンロード完了時刻。
* `cache_hits` ([UInt64](/ja/reference/data-types/index)) — 対応するファイルセグメントのキャッシュヒット数
* `references` ([UInt64](/ja/reference/data-types/index)) — 対応するファイルセグメントへの参照数。値が 1 の場合、現時点では誰もこれを使用していないことを意味します (存在する唯一の参照はキャッシュストレージ自体にあります)
* `downloaded_size` ([UInt64](/ja/reference/data-types/index)) — ファイルセグメントのダウンロード済みサイズ
* `kind` ([String](/ja/reference/data-types/index)) — ファイルセグメントの種類 (&#39;Temporary data in cache&#39; の一部として追加されたファイルセグメントを区別するために使用)
* `unbound` ([UInt8](/ja/reference/data-types/index)) — 内部実装用フラグ
* `user_id` ([String](/ja/reference/data-types/index)) — ファイルセグメントを作成したユーザーのユーザー ID
* `segment_type` ([String](/ja/reference/data-types/index)) — セグメントのタイプ。データファイル (`.json`、`.txt` など) とデータファイル (`.bin`、マークファイル) を区別するために使用されます。
* `file_size` ([Nullable(UInt64)](/ja/reference/data-types/index)) — 現在のファイルセグメントが属するファイルのサイズ

{/*AUTOGENERATED_END*/}
