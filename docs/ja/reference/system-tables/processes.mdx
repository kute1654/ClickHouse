<div id="description">
  ## 説明
</div>

このシステムテーブルは、`SHOW PROCESSLIST` クエリの実装に使用されます。

<div id="columns">
  ## カラム
</div>

{/*AUTOGENERATED_START*/}

* `is_initial_query` ([UInt8](/ja/reference/data-types/index)) — このクエリがユーザーから直接送信されたものか、または distributed query execution の一環として ClickHouse server によって発行されたものかを示します。
* `user` ([String](/ja/reference/data-types/index)) — クエリを実行したユーザー。distributed processing では、クエリは default user でリモートサーバーに送信される点に注意してください。このフィールドには、このクエリが起点となって実行されたクエリではなく、そのクエリ自体の username が含まれます。
* `query_id` ([String](/ja/reference/data-types/index)) — 定義されている場合の Query ID。
* `address` ([IPv6](/ja/reference/data-types/index)) — クエリの送信元 IP アドレス。distributed processing でも同様です。分散クエリが最初にどこから送信されたかを追跡するには、クエリ要求元サーバーの system.processes を参照してください。
* `port` ([UInt16](/ja/reference/data-types/index)) — クエリの送信元 client のポート。
* `initial_user` ([String](/ja/reference/data-types/index)) — 初期クエリを実行したユーザー名 (distributed query execution の場合) 。
* `initial_query_id` ([String](/ja/reference/data-types/index)) — 初期クエリの ID (distributed query execution の場合) 。
* `initial_address` ([IPv6](/ja/reference/data-types/index)) — 親クエリの送信元 IP アドレス。
* `initial_port` ([UInt16](/ja/reference/data-types/index)) — 親クエリの送信に使用された client のポート。
* `interface` ([UInt8](/ja/reference/data-types/index)) — クエリの送信に使用されたインターフェイス。TCP = 1, HTTP = 2, GRPC = 3, MYSQL = 4, POSTGRESQL = 5, LOCAL = 6, TCP&#95;INTERSERVER = 7。
* `os_user` ([String](/ja/reference/data-types/index)) — clickhouse-client を実行しているオペレーティングシステムの username。
* `client_hostname` ([String](/ja/reference/data-types/index)) — clickhouse-client または別の TCP client を実行している client マシンの hostname。
* `client_name` ([String](/ja/reference/data-types/index)) — clickhouse-client または別の TCP client の名前。
* `client_agent` ([LowCardinality(String)](/ja/reference/data-types/index)) — client を呼び出した AI コーディング agent (例: `claude-code`、`cursor`) 。環境変数から検出されます。agent が検出されなかった場合は空です。
* `client_revision` ([UInt64](/ja/reference/data-types/index)) — clickhouse-client または別の TCP client のリビジョン。
* `client_version_major` ([UInt64](/ja/reference/data-types/index)) — clickhouse-client または別の TCP client のメジャーバージョン。
* `client_version_minor` ([UInt64](/ja/reference/data-types/index)) — clickhouse-client または別の TCP client のマイナーバージョン。
* `client_version_patch` ([UInt64](/ja/reference/data-types/index)) — clickhouse-client または別の TCP client のバージョンのパッチ部分。
* `http_method` ([UInt8](/ja/reference/data-types/index)) — クエリを開始した HTTP メソッド。設定可能な値: 0 — クエリは TCP インターフェイスから開始されました。1 — GET メソッドが使用されました。2 — POST メソッドが使用されました。
* `http_user_agent` ([String](/ja/reference/data-types/index)) — HTTP クエリで渡された UserAgent HTTP header。
* `http_referer` ([String](/ja/reference/data-types/index)) — HTTP クエリで渡された Referer HTTP header (クエリを実行したページの絶対または部分アドレスを含みます) 。
* `forwarded_for` ([String](/ja/reference/data-types/index)) — HTTP クエリで渡された X-Forwarded-For HTTP header。
* `quota_key` ([String](/ja/reference/data-types/index)) — quotas 設定で指定された quota key (keyed を参照) 。
* `distributed_depth` ([UInt64](/ja/reference/data-types/index)) — クエリが内部的に server ノード間で再送信された回数。
* `elapsed` ([Float64](/ja/reference/data-types/index)) — リクエスト実行開始からの経過時間 (秒) 。
* `is_cancelled` ([UInt8](/ja/reference/data-types/index)) — クエリがキャンセルされたかどうか。
* `is_all_data_sent` ([UInt8](/ja/reference/data-types/index)) — すべてのデータがクライアントに送信されたかどうか (つまり、クエリがサーバー側で完了したかどうか) 。
* `read_rows` ([UInt64](/ja/reference/data-types/index)) — テーブルから読み取られた行数。分散処理では、要求元サーバー上では、これはすべてのリモートサーバーの合計です。
* `read_bytes` ([UInt64](/ja/reference/data-types/index)) — テーブルから読み取られた非圧縮バイト数。分散処理では、要求元サーバー上では、これはすべてのリモートサーバーの合計です。
* `total_rows_approx` ([UInt64](/ja/reference/data-types/index)) — 読み取る必要がある総行数のおおよその値。分散処理では、要求元サーバー上では、これはすべてのリモートサーバーの合計です。これはリクエスト処理中に、処理対象の新しいログソースが判明した時点で更新されることがあります。
* `written_rows` ([UInt64](/ja/reference/data-types/index)) — ストレージに書き込まれた行数。
* `written_bytes` ([UInt64](/ja/reference/data-types/index)) — ストレージに書き込まれたバイト数。
* `memory_usage` ([Int64](/ja/reference/data-types/index)) — クエリが使用する RAM 量。一部の専用メモリの種類は含まれない場合があります。
* `peak_memory_usage` ([Int64](/ja/reference/data-types/index)) — メモリ使用量の現在のピーク値。
* `query` ([String](/ja/reference/data-types/index)) — クエリテキスト。INSERT の場合、挿入するデータは含まれません。
* `normalized_query_hash` ([UInt64](/ja/reference/data-types/index)) — 数値の hash value。たとえば、リテラルの値だけが異なるクエリでは同一になります。
* `query_kind` ([String](/ja/reference/data-types/index)) — クエリの種類。SELECT、INSERT など。
* `thread_ids` ([Array(UInt64)](/ja/reference/data-types/index)) — このクエリに関与したすべてのスレッドの識別子の一覧。
* `peak_threads_usage` ([UInt64](/ja/reference/data-types/index)) — クエリを実行していた同時スレッド数の最大値。
* `ProfileEvents` ([Map(LowCardinality(String), UInt64)](/ja/reference/data-types/index)) — このクエリに対して計算された ProfileEvents。
* `Settings` ([Map(LowCardinality(String), LowCardinality(String))](/ja/reference/data-types/index)) — 変更されたユーザーレベル設定の一覧。
* `current_database` ([String](/ja/reference/data-types/index)) — 現在のデータベース名。
* `is_internal` ([UInt8](/ja/reference/data-types/index)) — 内部的に実行された補助クエリかどうかを示します。

{/*AUTOGENERATED_END*/}
