<div id="description">
  ## 説明
</div>

現在の状態やその他の属性を含む、すべての `BACKUP` または `RESTORE` 操作の一覧です。なお、このテーブルは永続的ではなく、最後にサーバーを再起動して以降に実行された操作のみが表示されます。

<div id="columns">
  ## カラム
</div>

{/*AUTOGENERATED_START*/}

* `id` ([String](/ja/reference/data-types/index)) — 操作 ID。`SETTINGS id=...` で指定することも、ランダムに生成された UUID にすることもできます。
* `name` ([String](/ja/reference/data-types/index)) — 操作名。`Disk('backups', 'my_backup')` のような文字列です。
* `base_backup_name` ([String](/ja/reference/data-types/index)) — ベースバックアップの操作名。`Disk('backups', 'my_base_backup')` のような文字列です。
* `query_id` ([String](/ja/reference/data-types/index)) — バックアップを開始したクエリの Query ID。
* `status` ([Enum8(&#39;CREATING&#95;BACKUP&#39; = 0, &#39;BACKUP&#95;CREATED&#39; = 1, &#39;BACKUP&#95;FAILED&#39; = 2, &#39;RESTORING&#39; = 3, &#39;RESTORED&#39; = 4, &#39;RESTORE&#95;FAILED&#39; = 5, &#39;BACKUP&#95;CANCELLED&#39; = 6, &#39;RESTORE&#95;CANCELLED&#39; = 7)](/ja/reference/data-types/index)) — バックアップまたは復元操作のステータス。
* `error` ([String](/ja/reference/data-types/index)) — エラーが発生した場合のエラーメッセージ。
* `start_time` ([DateTime64(6)](/ja/reference/data-types/index)) — 操作が開始された時刻。
* `end_time` ([DateTime64(6)](/ja/reference/data-types/index)) — 操作が完了した時刻。
* `num_files` ([UInt64](/ja/reference/data-types/index)) — バックアップに保存されているファイル数。
* `total_size` ([UInt64](/ja/reference/data-types/index)) — バックアップに保存されているファイルの合計サイズ。
* `num_entries` ([UInt64](/ja/reference/data-types/index)) — バックアップ内のエントリ数。つまり、バックアップがフォルダとして保存されている場合は、そのフォルダ内のファイル数です。
* `uncompressed_size` ([UInt64](/ja/reference/data-types/index)) — バックアップの非圧縮サイズ。
* `compressed_size` ([UInt64](/ja/reference/data-types/index)) — バックアップの圧縮サイズ。
* `files_read` ([UInt64](/ja/reference/data-types/index)) — このバックアップから RESTORE 中に読み取られたファイル数を返します。
* `bytes_read` ([UInt64](/ja/reference/data-types/index)) — このバックアップから RESTORE 中に読み取られたファイルの合計サイズを返します。
* `ProfileEvents` ([Map(LowCardinality(String), UInt64)](/ja/reference/data-types/index)) — この操作中に収集されたすべての profile events。

{/*AUTOGENERATED_END*/}
