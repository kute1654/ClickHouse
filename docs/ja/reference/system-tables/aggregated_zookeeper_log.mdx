<div id="description">
  ## 説明
</div>

このテーブルには、`(session_id, parent_path, operation, component, is_subrequest)` ごとにグループ化された ZooKeeper 操作の集計統計 (例: 操作数、平均レイテンシ、エラー) が格納されており、定期的にディスクへフラッシュされます。

個々のリクエストとレスポンスをすべて記録する [system.zookeeper&#95;log](/ja/reference/system-tables/zookeeper_log) とは異なり、このテーブルでは操作をグループ単位で集計するため、はるかに軽量で、本番環境のワークロードにより適しています。

`Multi` または `MultiRead` バッチの一部である操作は、`is_subrequest` カラムによって個別に追跡されます。サブリクエストのレイテンシは 0 です。これは、合計レイテンシがそれを含む `Multi`/`MultiRead` 操作に割り当てられるためです。
