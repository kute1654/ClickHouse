<div id="description">
  ## 説明
</div>

設定ファイルで利用可能なクラスターと、それらに含まれるサーバーに関する情報が含まれます。

<div id="Columns">
  ## カラム
</div>

{/*AUTOGENERATED_START*/}

* `cluster` ([String](/ja/reference/data-types/index)) — クラスター名。
* `shard_num` ([UInt32](/ja/reference/data-types/index)) — クラスター内の分片番号。1 から始まります。
* `shard_name` ([String](/ja/reference/data-types/index)) — クラスター内の分片名。
* `shard_weight` ([UInt32](/ja/reference/data-types/index)) — データ書き込み時の分片の相対的な重み。
* `internal_replication` ([UInt8](/ja/reference/data-types/index)) — このホストが、自身でデータをレプリケートできる構成の一部であるかどうかを示すフラグ。
* `replica_num` ([UInt32](/ja/reference/data-types/index)) — 分片内のレプリカ番号。1 から始まります。
* `host_name` ([String](/ja/reference/data-types/index)) — config で指定されたホスト名。
* `host_address` ([String](/ja/reference/data-types/index)) — DNS から取得されたホストの IP アドレス。
* `port` ([UInt16](/ja/reference/data-types/index)) — サーバーへの接続に使用するポート。
* `is_local` ([UInt8](/ja/reference/data-types/index)) — ホストがローカルかどうかを示すフラグ。
* `user` ([String](/ja/reference/data-types/index)) — サーバーへの接続に使用するユーザー名。
* `default_database` ([String](/ja/reference/data-types/index)) — デフォルトのデータベース名。
* `errors_count` ([UInt32](/ja/reference/data-types/index)) — このホストからレプリカへの到達に失敗した回数。
* `slowdowns_count` ([UInt32](/ja/reference/data-types/index)) — hedged requests で connection を確立する際に、レプリカの切り替えにつながった遅延の回数。
* `estimated_recovery_time` ([UInt32](/ja/reference/data-types/index)) — レプリカの error 回数が 0 にリセットされ、正常な状態に戻ったと見なされるまでの残り秒数。
* `database_shard_name` ([String](/ja/reference/data-types/index)) — `Replicated` データベースの分片名 (`Replicated` データベースに属するクラスターの場合) 。
* `database_replica_name` ([String](/ja/reference/data-types/index)) — `Replicated` データベースのレプリカ名 (`Replicated` データベースに属するクラスターの場合) 。
* `is_shared_catalog_cluster` ([UInt8](/ja/reference/data-types/index)) — クラスターが Shared Catalog に属しているかどうかを示す Bool。
* `is_active` ([Nullable(UInt8)](/ja/reference/data-types/index)) — `Replicated` データベースのレプリカの状態 (`Replicated` データベースに属するクラスターの場合) : 1 は &#39;レプリカはオンライン&#39;、0 は &#39;レプリカはオフライン&#39;、NULL は &#39;不明&#39; を意味します。
* `unsynced_after_recovery` ([Nullable(UInt8)](/ja/reference/data-types/index)) — レプリカの作成または復旧後に、`Replicated` データベースのレプリカのレプリケーションラグが max&#95;replication&#95;lag&#95;to&#95;enqueue を超えているかどうかを示します。
* `replication_lag` ([Nullable(UInt32)](/ja/reference/data-types/index)) — `Replicated` データベースのレプリカのレプリケーションラグ (`Replicated` データベースに属するクラスターの場合) 。
* `recovery_time` ([Nullable(UInt64)](/ja/reference/data-types/index)) — `Replicated` データベースのレプリカの復旧時間 (`Replicated` データベースに属するクラスターの場合) 。単位はミリ秒です。

{/*AUTOGENERATED_END*/}
