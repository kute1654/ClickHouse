<div id="description">
  ## 説明
</div>

[system.dropped&#95;tables](/ja/reference/system-tables/dropped_tables) から削除された [MergeTree](/ja/reference/engines/table-engines/mergetree-family/mergetree) テーブルのパーツに関する情報が含まれます

このテーブルのスキーマは [system.parts](/ja/reference/system-tables/parts) と同じです

<div id="columns">
  ## カラム
</div>

{/*AUTOGENERATED_START*/}

* `partition` ([String](/ja/reference/data-types/index)) — パーティション識別子 (テーブルのパーティションキー式に基づく文字列表現) 。
* `name` ([String](/ja/reference/data-types/index)) —
  データパーツの名前。パーツの命名構造から、データ、取り込み、マージのさまざまなパターンを判断できます。パーツの命名フォーマットは次のとおりです:

```text
<partition_id>_<minimum_block_number>_<maximum_block_number>_<level>_<data_version>
```

* 定義:
  * `partition_id` - パーティションキーを識別します
  * `minimum_block_number` - パーツ内の最小ブロック番号を示します。ClickHouse は常に連続したブロックをマージします
  * `maximum_block_number` - パーツ内の最大ブロック番号を示します
  * `level` - パーツに対してマージが 1 回行われるごとに 1 ずつ増加します。レベルが 0 の場合、これはまだマージされていない新しいパーツであることを示します。ClickHouse のすべてのパーツは常に不変であることに注意してください
  * `data_version` - オプションの値で、パーツが変更されると増加します (繰り返しになりますが、パーツは不変であるため、変更されたデータも常に新しいパーツにのみ書き込まれます)

* `uuid` ([UUID](/ja/reference/data-types/index)) — データパーツの UUID。

* `part_type` ([String](/ja/reference/data-types/index)) — データパーツの保存フォーマット。設定可能な値: `Wide` — 各カラムが個別のファイルに保存される、`Compact` — すべてのカラムが 1 つのファイルに保存される。データの保存フォーマットは、MergeTree テーブルの `min_bytes_for_wide_part` および `min_rows_for_wide_part` 設定によって制御されます。

* `active` ([UInt8](/ja/reference/data-types/index)) — データパーツがアクティブかどうかを示すフラグ。データパーツがアクティブな場合、そのパーツはテーブルで使用されます。そうでない場合は、削除される予定です。非アクティブなデータパーツは、マージ操作およびミューテーション操作の後に現れます。

* `marks` ([UInt64](/ja/reference/data-types/index)) — マークの数。データパーツ内のおおよその行数を取得するには、marks にインデックス粒度 (通常は 8192) を掛けます (この目安は adaptive granularity では機能しません) 。

* `rows` ([UInt64](/ja/reference/data-types/index)) — 行数。

* `files` ([UInt64](/ja/reference/data-types/index)) — データパーツ内のファイル数。

* `bytes_on_disk` ([UInt64](/ja/reference/data-types/index)) — すべてのデータパーツファイルの合計サイズ (バイト単位) 。

* `data_compressed_bytes` ([UInt64](/ja/reference/data-types/index)) — データパーツ内の compressed データの合計サイズ。補助ファイル (たとえば、marks を含むファイル) は含まれません。

* `data_uncompressed_bytes` ([UInt64](/ja/reference/data-types/index)) — データパーツ内の非圧縮データの合計サイズ。補助ファイル (たとえば、marks を含むファイル) は含まれません。

* `primary_key_size` ([UInt64](/ja/reference/data-types/index)) — ディスク上の primary.idx/cidx ファイル内で主キー値が使用するメモリ量 (バイト単位) 。

* `marks_bytes` ([UInt64](/ja/reference/data-types/index)) — marks を含むファイルのサイズ。

* `secondary_indices_compressed_bytes` ([UInt64](/ja/reference/data-types/index)) — データパーツ内のセカンダリ索引用 compressed データの合計サイズ。補助ファイル (たとえば、marks を含むファイル) は含まれません。

* `secondary_indices_uncompressed_bytes` ([UInt64](/ja/reference/data-types/index)) — データパーツ内のセカンダリ索引用非圧縮データの合計サイズ。補助ファイル (たとえば、marks を含むファイル) は含まれません。

* `secondary_indices_marks_bytes` ([UInt64](/ja/reference/data-types/index)) — セカンダリ索引用の marks を含むファイルのサイズ。

* `modification_time` ([DateTime](/ja/reference/data-types/index)) — データパーツを含むディレクトリが変更された時刻。通常、これはデータパーツの作成時刻に対応します。

* `remove_time` ([DateTime](/ja/reference/data-types/index)) — データパーツが非アクティブになった時刻。

* `refcount` ([UInt32](/ja/reference/data-types/index)) — データパーツが使用されている箇所の数。2 より大きい値は、そのデータパーツがクエリまたはマージで使用されていることを示します。

* `min_date` ([Date](/ja/reference/data-types/index)) — データパーツ内の日付キーの最小値。

* `max_date` ([Date](/ja/reference/data-types/index)) — データパーツ内の日付キーの最大値。

* `min_time` ([DateTime](/ja/reference/data-types/index)) — データパーツ内の日付と時刻キーの最小値。

* `max_time` ([DateTime](/ja/reference/data-types/index)) — データパーツ内の日付と時刻キーの最大値。

* `partition_id` ([String](/ja/reference/data-types/index)) — パーティションの ID。

* `min_block_number` ([Int64](/ja/reference/data-types/index)) — マージ後に現在のパーツを構成するデータパーツの最小番号。

* `max_block_number` ([Int64](/ja/reference/data-types/index)) — マージ後に現在のパーツを構成するデータパーツの最大番号。

* `level` ([UInt32](/ja/reference/data-types/index)) — マージツリーの深さ。0 は、現在のパーツが他のパーツのマージではなく insert によって作成されたことを意味します。

* `data_version` ([UInt64](/ja/reference/data-types/index)) — データパートにどのミューテーションを適用すべきかを判定するための数値 (`data_version`より大きいバージョンのミューテーション) 。

* `primary_key_bytes_in_memory` ([UInt64](/ja/reference/data-types/index)) — 主キーの値が使用するメモリ量 (バイト単位) 。`primary_key_lazy_load`が有効で、かつキーが読み込まれていない場合は 0 になります。0 以外の場合、これらのバイトはパート自体に存在し、`jemalloc.mergetree_arena.active_bytes` に計上されます。これらが `PrimaryIndexCacheBytes` に計上されることはありません。パートごとに両者は相互排他的です。つまり、索引は `primary_key_lazy_load` と `use_primary_key_cache` の設定に応じて、パート内 (このメトリクス) または共有の `PrimaryIndexCache` (もう一方) に存在します。

* `primary_key_bytes_in_memory_allocated` ([UInt64](/ja/reference/data-types/index)) — 主キーの値のために確保されているメモリ量 (バイト単位) 。`primary_key_lazy_load`が有効で、かつキーが読み込まれていない場合は 0 になります。0 以外の場合は、`jemalloc.mergetree_arena.active_bytes` に含まれます。`PrimaryIndexCacheBytes` との関係については、`primary_key_bytes_in_memory` の注記を参照してください。

* `index_granularity_bytes_in_memory` ([UInt64](/ja/reference/data-types/index)) — 索引粒度の値が使用するメモリ量 (バイト単位)  (primary&#95;key&#95;lazy&#95;load=1 かつ use&#95;primary&#95;key&#95;cache=1 の場合は 0 になります) 。0 以外の場合、これらのバイトはパートに属し、`jemalloc.mergetree_arena.active_bytes` に計上されます。

* `index_granularity_bytes_in_memory_allocated` ([UInt64](/ja/reference/data-types/index)) — 索引粒度の値のために確保されているメモリ量 (バイト単位)  (primary&#95;key&#95;lazy&#95;load=1 かつ use&#95;primary&#95;key&#95;cache=1 の場合は 0 になります) 。0 以外の場合は、`jemalloc.mergetree_arena.active_bytes` に含まれます。

* `is_frozen` ([UInt8](/ja/reference/data-types/index)) — パーティションのデータバックアップが存在することを示すフラグ。1 はバックアップが存在すること、0 はバックアップが存在しないことを示します。詳細は FREEZE PARTITION を参照してください。

* `database` ([String](/ja/reference/data-types/index)) — データベース名。

* `table` ([String](/ja/reference/data-types/index)) — テーブル名。

* `engine` ([String](/ja/reference/data-types/index)) — パラメータを含まないテーブルエンジン名。

* `disk_name` ([String](/ja/reference/data-types/index)) — データパートを格納しているディスク名。

* `path` ([String](/ja/reference/data-types/index)) — データパートのファイルを含むフォルダーへの絶対パス。

* `hash_of_all_files` ([String](/ja/reference/data-types/index)) — 圧縮ファイルの sipHash128。

* `hash_of_uncompressed_files` ([String](/ja/reference/data-types/index)) — 非圧縮ファイル (マーク、索引ファイルなど) の sipHash128。

* `uncompressed_hash_of_compressed_files` ([String](/ja/reference/data-types/index)) — 圧縮ファイル内のデータを、非圧縮であるものとして扱った場合の sipHash128。

* `delete_ttl_info_min` ([DateTime](/ja/reference/data-types/index)) — TTL DELETE ルールに対する日付と時刻キーの最小値。

* `delete_ttl_info_max` ([DateTime](/ja/reference/data-types/index)) — TTL DELETE ルールに対する日付と時刻キーの最大値。

* `move_ttl_info.expression` ([Array(String)](/ja/reference/data-types/index)) — 式の Array。各式は TTL MOVE ルールを定義します。

* `move_ttl_info.min` ([Array(DateTime)](/ja/reference/data-types/index)) — 日付と時刻の値の Array。各要素は TTL MOVE ルールの最小キー値を表します。

* `move_ttl_info.max` ([Array(DateTime)](/ja/reference/data-types/index)) — 日付と時刻の値の Array。各要素は TTL MOVE ルールの最大キー値を表します。

* `default_compression_codec` ([String](/ja/reference/data-types/index)) — このデータパートの圧縮に使用されるコーデック名 (カラムに明示的なコーデックがない場合) 。

* `recompression_ttl_info.expression` ([Array(String)](/ja/reference/data-types/index)) — TTL 式。

* `recompression_ttl_info.min` ([Array(DateTime)](/ja/reference/data-types/index)) — このパート内で計算された TTL 式の最小値。TTL が期限切れの行が少なくとも 1 行存在するかどうかを把握するために使用されます。

* `recompression_ttl_info.max` ([Array(DateTime)](/ja/reference/data-types/index)) — このパート内で計算された TTL 式の最大値。すべての行の有効期限 (TTL) が切れているかどうかを把握するために使用されます。

* `group_by_ttl_info.expression` ([Array(String)](/ja/reference/data-types/index)) — TTL 式。

* `group_by_ttl_info.min` ([Array(DateTime)](/ja/reference/data-types/index)) — このパート内で計算された TTL 式の最小値。少なくとも 1 行の有効期限 (TTL) が切れているかどうかを把握するために使用されます。

* `group_by_ttl_info.max` ([Array(DateTime)](/ja/reference/data-types/index)) — このパート内で計算された TTL 式の最大値。すべての行の有効期限 (TTL) が切れているかどうかを把握するために使用されます。

* `rows_where_ttl_info.expression` ([Array(String)](/ja/reference/data-types/index)) — TTL 式。

* `rows_where_ttl_info.min` ([Array(DateTime)](/ja/reference/data-types/index)) — このパート内で計算された TTL 式の最小値。少なくとも 1 行の有効期限 (TTL) が切れているかどうかを把握するために使用されます。

* `rows_where_ttl_info.max` ([Array(DateTime)](/ja/reference/data-types/index)) — このパート内で計算された TTL 式の最大値。すべての行の有効期限 (TTL) が切れているかどうかを把握するために使用されます。

* `projections` ([Array(String)](/ja/reference/data-types/index)) — このパートに対して計算されたプロジェクション名の一覧。

* `visible` ([UInt8](/ja/reference/data-types/index)) — このパートが SELECT クエリから参照可能かどうかを示すフラグ。

* `creation_tid` ([Tuple(UInt64, UInt64, UUID)](/ja/reference/data-types/index)) — このオブジェクトを作成した、または作成しようとしているトランザクションの ID。

* `removal_tid_lock` ([UInt64](/ja/reference/data-types/index)) — `removal_tid` のハッシュ。削除のためにオブジェクトをロックする際に使用されます。

* `removal_tid` ([Tuple(UInt64, UInt64, UUID)](/ja/reference/data-types/index)) — このオブジェクトを削除した、または削除しようとしているトランザクションの ID

* `creation_csn` ([UInt64](/ja/reference/data-types/index)) — このオブジェクトを作成したトランザクションの CSN

* `removal_csn` ([UInt64](/ja/reference/data-types/index)) — このオブジェクトを削除したトランザクションの CSN

* `has_lightweight_delete` ([UInt8](/ja/reference/data-types/index)) — このパートに論理削除マスクがあるかどうかを示すフラグ。

* `last_removal_attempt_time` ([DateTime](/ja/reference/data-types/index)) — サーバーがこのパートの削除を最後に試行した時刻。

* `removal_state` ([String](/ja/reference/data-types/index)) — パートの削除処理の現在の状態。
