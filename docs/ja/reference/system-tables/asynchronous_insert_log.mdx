<div id="description">
  ## 説明
</div>

非同期 INSERT に関する情報が含まれます。各エントリは、非同期 INSERT 用にバッファリングされた INSERT クエリを表します。

ログ記録を開始するには、[asynchronous&#95;insert&#95;log](/ja/reference/settings/server-settings/settings#asynchronous_insert_log) セクションでパラメータを設定します。

データのフラッシュ間隔は、[asynchronous&#95;insert&#95;log](/ja/reference/settings/server-settings/settings#asynchronous_insert_log) サーバー設定セクションの `flush_interval_milliseconds` パラメータで設定します。フラッシュを強制するには、[SYSTEM FLUSH LOGS](/ja/reference/statements/system#flush-logs) クエリを使用します。

ClickHouse はテーブルからデータを自動的に削除しません。詳しくは、[Introduction](/ja/reference/system-tables/overview#system-tables-introduction) を参照してください。

<div id="columns">
  ## カラム
</div>

{/*AUTOGENERATED_START*/}

* `hostname` ([LowCardinality(String)](/ja/reference/data-types/lowcardinality)) — クエリを実行したサーバーのホスト名。
* `event_date` ([Date](/ja/reference/data-types/date)) — 非同期 INSERT が発生した日付。
* `event_time` ([DateTime](/ja/reference/data-types/datetime)) — 非同期 INSERT の実行が完了した日時。
* `event_time_microseconds` ([DateTime64(6)](/ja/reference/data-types/datetime64)) — 非同期 INSERT の実行が完了した日時 (マイクロ秒精度) 。
* `query` ([String](/ja/reference/data-types/string)) — クエリ文字列。
* `database` ([LowCardinality(String)](/ja/reference/data-types/lowcardinality)) — テーブルが属するデータベース名。
* `table` ([LowCardinality(String)](/ja/reference/data-types/lowcardinality)) — テーブル名。
* `format` ([LowCardinality(String)](/ja/reference/data-types/lowcardinality)) — フォーマット名。
* `query_id` ([String](/ja/reference/data-types/string)) — 元のクエリの ID。
* `bytes` ([UInt64](/ja/reference/data-types/int-uint)) — 挿入されたバイト数。
* `rows` ([UInt64](/ja/reference/data-types/int-uint)) — 挿入された行数。
* `exception` ([String](/ja/reference/data-types/string)) — 例外メッセージ。
* `status` ([Enum8(&#39;Ok&#39; = 0, &#39;ParsingError&#39; = 1, &#39;FlushError&#39; = 2)](/ja/reference/data-types/enum)) — INSERT のステータス。値: &#39;Ok&#39; = 0 — 挿入成功、&#39;ParsingError&#39; = 1 — データのパース中に発生した例外、&#39;FlushError&#39; = 2 — データのフラッシュ中に発生した例外。
* `data_kind` ([Enum8(&#39;Parsed&#39; = 0, &#39;Preprocessed&#39; = 1)](/ja/reference/data-types/enum)) — データの状態。値: &#39;Parsed&#39; および &#39;Preprocessed&#39;。
* `flush_time` ([DateTime](/ja/reference/data-types/datetime)) — フラッシュが発生した日時。
* `flush_time_microseconds` ([DateTime64(6)](/ja/reference/data-types/datetime64)) — フラッシュが発生した日時 (マイクロ秒精度) 。
* `flush_query_id` ([String](/ja/reference/data-types/string)) — フラッシュ クエリの ID。
* `timeout_milliseconds` ([UInt64](/ja/reference/data-types/int-uint)) — このエントリに対して計算された適応タイムアウト。

{/*AUTOGENERATED_END*/}
