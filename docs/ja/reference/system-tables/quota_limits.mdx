<div id="description">
  ## 説明
</div>

すべてのQUOTAのすべてのインターバルに対する上限情報が含まれます。1 つのQUOTAに対応する行数は、0 行を含め任意です。

<div id="columns">
  ## カラム
</div>

{/*AUTOGENERATED_START*/}

* `quota_name` ([String](/ja/reference/data-types/index)) — QUOTA名。
* `duration` ([UInt32](/ja/reference/data-types/index)) — リソース消費量の計算に用いる時間間隔の長さ (秒) 。
* `is_randomized_interval` ([UInt8](/ja/reference/data-types/index)) — ブール値。このインターバルがランダム化されているかどうかを示します。ランダム化されていない場合、インターバルは常に同じ時刻に開始されます。たとえば、1 分のインターバルは常に分単位のちょうどの時刻に開始されます (つまり、11:20:00 に開始することはあっても、11:20:01 に開始することはありません) 。また、1 日のインターバルは常に UTC の午前 0 時に開始されます。インターバルがランダム化されている場合、最初のインターバルはランダムな時刻に開始され、以降のインターバルは順次開始されます。値: 0 — インターバルはランダム化されていません、1 — インターバルはランダム化されています。
* `max_queries` ([Nullable(UInt64)](/ja/reference/data-types/index)) — 実行可能な、すべての種類のクエリの最大数。
* `max_query_selects` ([Nullable(UInt64)](/ja/reference/data-types/index)) — 実行可能な SELECT クエリの最大数。
* `max_query_inserts` ([Nullable(UInt64)](/ja/reference/data-types/index)) — 実行可能な INSERT クエリの最大数。
* `max_errors` ([Nullable(UInt64)](/ja/reference/data-types/index)) — 指定された期間内に許可される、エラーになったクエリの最大数。
* `max_result_rows` ([Nullable(UInt64)](/ja/reference/data-types/index)) — 指定された期間内に許可される、すべてのクエリの結果セットに含まれる行の総数の最大値。
* `max_result_bytes` ([Nullable(UInt64)](/ja/reference/data-types/index)) — 指定された期間内に許可される、すべてのクエリの結果セットに含まれる総バイト数の最大値。
* `max_read_rows` ([Nullable(UInt64)](/ja/reference/data-types/index)) — 指定された期間内に許可される、すべてのクエリの実行中に読み取る行数の最大値。
* `max_read_bytes` ([Nullable(UInt64)](/ja/reference/data-types/index)) — 指定された期間内に許可される、すべてのクエリの実行中に読み取るバイト数の最大値。
* `max_execution_time` ([Nullable(Float64)](/ja/reference/data-types/index)) — 指定された期間内に、すべてのクエリの実行に許可される最大時間 (ナノ秒)
* `max_written_bytes` ([Nullable(UInt64)](/ja/reference/data-types/index)) — 指定された期間内に許可される、すべてのクエリの実行中に書き込まれるバイト数の最大値。
* `max_failed_sequential_authentications` ([Nullable(UInt64)](/ja/reference/data-types/index)) — 指定された期間内に許可される、連続する認証失敗の最大回数。
* `max_queries_per_normalized_hash` ([Nullable(UInt64)](/ja/reference/data-types/index)) — 指定された期間内に許可される、任意の単一の正規化クエリの最大実行回数。
