<div id="description">
  ## 説明
</div>

[クォータ](/ja/reference/system-tables/quotas) に関する情報が含まれます。

<div id="columns">
  ## カラム
</div>

{/*AUTOGENERATED_START*/}

* `name` ([String](/ja/reference/data-types/index)) — クォータ名。
* `id` ([UUID](/ja/reference/data-types/index)) — クォータ ID。
* `storage` ([String](/ja/reference/data-types/index)) — クォータの保存先。指定可能な値: users.xml ファイルでクォータが設定されている場合は &#39;users.xml&#39;、SQL クエリでクォータが設定されている場合は &#39;disk&#39;。
* `keys` ([Array(Enum8(&#39;user&#95;name&#39; = 1, &#39;ip&#95;address&#39; = 2, &#39;forwarded&#95;ip&#95;address&#39; = 3, &#39;client&#95;key&#39; = 4, &#39;normalized&#95;query&#95;hash&#39; = 7))](/ja/reference/data-types/index)) — Key は、クォータをどのように共有するかを指定します。2 つの connection が同じクォータと Key を使用する場合、同じ量のリソースを共有します。値: [] — すべてのユーザーで同じクォータを共有します、[&#39;user&#95;name&#39;] — 同じユーザー名の connection で同じクォータを共有します、[&#39;ip&#95;address&#39;] — 同じ IP アドレスからの connection で同じクォータを共有します。[&#39;client&#95;key&#39;] — 同じ Key を持つ connection で同じクォータを共有します。Key はクライアントが明示的に指定する必要があります。clickhouse-client を使用する場合は、`--quota_key` パラメータに Key の値を渡すか、クライアントの設定ファイルで `quota_key` パラメータを使用します。HTTP インターフェイスを使用する場合は、X-ClickHouse-Quota header を使用します。[&#39;user&#95;name&#39;, &#39;client&#95;key&#39;] — 同じ client&#95;key を持つ connection で同じクォータを共有します。クライアントから Key が指定されない場合、クォータは `user_name` ごとに追跡されます。[&#39;client&#95;key&#39;, &#39;ip&#95;address&#39;] — 同じ client&#95;key を持つ connection で同じクォータを共有します。クライアントから Key が指定されない場合、クォータは ip&#95;address ごとに追跡されます。
* `durations` ([Array(UInt32)](/ja/reference/data-types/index)) — 秒単位の時間間隔の長さ。
* `apply_to_all` ([UInt8](/ja/reference/data-types/index)) — 論理値。クォータがどのユーザーに適用されるかを示します。値: 0 — クォータは apply&#95;to&#95;list で指定したユーザーに適用されます。1 — クォータは apply&#95;to&#95;except に列挙されたユーザーを除くすべてのユーザーに適用されます。
* `apply_to_list` ([Array(String)](/ja/reference/data-types/index)) — クォータを適用するユーザー名/ロールの一覧。
* `apply_to_except` ([Array(String)](/ja/reference/data-types/index)) — クォータを適用しないユーザー名/ロールの一覧。
* `ipv4_prefix_bits` ([Nullable(UInt8)](/ja/reference/data-types/index)) — IPv4 アドレスのマスキングに使用するプレフィックスビット数。同じサブネットからの connection は同じクォータを共有します。key が `ip_address` または `forwarded_ip_address` の場合にのみ使用されます。
* `ipv6_prefix_bits` ([Nullable(UInt8)](/ja/reference/data-types/index)) — IPv6 アドレスのマスキングに使用するプレフィックスビット数。同じサブネットからの connection は同じクォータを共有します。key が `ip_address` または `forwarded_ip_address` の場合にのみ使用されます。
