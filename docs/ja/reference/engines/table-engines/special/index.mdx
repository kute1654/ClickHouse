テーブルエンジンは、主に次の3つのカテゴリに分かれます。

* 主な本番用途に使われる [MergeTree エンジンファミリー](/ja/reference/engines/table-engines/mergetree-family/index)
* 小規模な一時データ向けの [Log エンジンファミリー](/ja/reference/engines/table-engines/log-family/index)
* [インテグレーション向けテーブルエンジン](/ja/reference/engines/table-engines/integrations/index)

そのほかのエンジンは、それぞれ用途が固有で、現時点ではファミリーとして分類されていないため、この &quot;special&quot; カテゴリに含められています。

| [Join テーブルエンジン](/ja/reference/engines/table-engines/special/join)                              | JOIN 演算で使用するための、任意の事前構築済みデータ構造です。                                                                                                      |

| [Buffer テーブルエンジン](/ja/reference/engines/table-engines/special/buffer)                          | 書き込むデータを RAM にバッファリングし、定期的に別のテーブルへフラッシュします。読み取り時には、バッファともう一方のテーブルの両方から同時にデータが読み取られます。                                                  |

| [GenerateRandom テーブルエンジン](/ja/reference/engines/table-engines/special/generate)                | GenerateRandom テーブルエンジンは、指定されたテーブルスキーマに基づいてランダムデータを生成します。                                                                              |
| [KeeperMap テーブルエンジン](/ja/reference/engines/table-engines/special/keepermap)                    | このエンジンを使用すると、Keeper/ZooKeeper クラスターを、線形化可能な書き込みと逐次一貫性のある読み取りを備えた一貫性のあるキー・バリューストアとして利用できます。                                             |
| [FileLog テーブルエンジン](/ja/reference/engines/table-engines/special/filelog)                        | このエンジンでは、アプリケーションのログファイルをレコードのストリームとして処理できます。                                                                                          |
