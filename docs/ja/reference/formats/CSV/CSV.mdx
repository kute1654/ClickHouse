<div id="description">
  ## 説明
</div>

カンマ区切り値フォーマット ([RFC](https://tools.ietf.org/html/rfc4180)) 。
フォーマット時、行は二重引用符で囲まれます。文字列内の二重引用符は、二重引用符 2 つを連続して出力します。
文字をエスケープするための、それ以外のルールはありません。

* Date 型および date-time は二重引用符で囲まれます。
* 数値は引用符なしで出力されます。
* 値は区切り文字で区切られ、デフォルトは `,` です。区切り文字は設定 [format&#95;csv&#95;delimiter](/ja/reference/settings/formats#format_csv_delimiter) で定義されます。
* 行は Unix のラインフィード (LF) で区切られます。
* Array は CSV で次のようにシリアライズされます。
  * まず、Array は TabSeparated フォーマットと同様に文字列にシリアライズされます
  * 生成された文字列は、二重引用符で囲んで CSV に出力されます。
* CSV フォーマットの Tuple は個別のカラムとしてシリアライズされます (つまり、Tuple 内のネストは失われます) 。

```bash
$ clickhouse-client --format_csv_delimiter="|" --query="INSERT INTO test.csv FORMAT CSV" < data.csv
```

<Note>
  デフォルトの区切り文字は `,` です
  詳しくは、設定 [format&#95;csv&#95;delimiter](/ja/reference/settings/formats#format_csv_delimiter) を参照してください。
</Note>

パース時には、すべての値を引用符ありでも引用符なしでもパースできます。二重引用符と単一引用符の両方をサポートしています。

行は引用符なしで記述することもできます。この場合、区切り文字または改行文字 (CR または LF) までをパースします。
ただし、RFC には準拠していませんが、引用符なしの行をパースする場合、先頭および末尾のスペースとタブは無視されます。
改行文字としては、Unix (LF) 、Windows (CR LF) 、および Mac OS Classic (CR LF) の各タイプをサポートしています。

`NULL` は、設定 [format&#95;csv&#95;null&#95;representation](/ja/reference/settings/formats#format_csv_null_representation) に従ってフォーマットされます (デフォルト値は `\N` です) 。

入力データでは、`ENUM` の値は名前または id として表現できます。
まず、入力値を `ENUM` 名に一致させようとします。
一致しなかった場合、入力値が数値であれば、その数値を `ENUM` id に一致させようとします。
入力データに `ENUM` id しか含まれていない場合は、`ENUM` のパースを最適化するために、設定 [input&#95;format&#95;csv&#95;enum&#95;as&#95;number](/ja/reference/settings/formats#input_format_csv_enum_as_number) を有効にすることを推奨します。
