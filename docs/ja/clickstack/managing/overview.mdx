<div id="core-concepts">
  ## ClickHouse の中核概念
</div>

ClickStack の管理タスクの多くでは、基盤となる ClickHouse データベースに関する理解が必要です。管理作業やパフォーマンス関連の操作を行う前に、以下に示す ClickHouse の中核概念を確認することをお勧めします。

| 概念                 | 説明                                                                                                                                                            |
| ------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **テーブル**           | ClickStack のデータソースが基盤となる ClickHouse テーブルにどのように対応するか。ClickHouse テーブルでは主に [MergeTree](/ja/reference/engines/table-engines/mergetree-family/mergetree) エンジンが使用されます。 |
| **パーツ**            | データが不変のパーツとしてどのように書き込まれ、時間の経過とともにマージされるか。                                                                                                                     |
| **パーティション**        | データ管理、クエリ、最適化を容易にする、テーブルパーツの論理的なグループ化。                                                                                                                        |
| **マージ**            | クエリ対象となるパーツ数を減らし、パフォーマンスを維持するために、パーツを結合するバックグラウンドプロセス。                                                                                                        |
| **グラニュール**         | クエリ実行中に読み取られ、プルーニングされるデータの最小単位。                                                                                                                               |
| **プライマリ (ソート) キー** | `ORDER BY` キーが、ディスク上のデータレイアウト、圧縮、およびクエリのプルーニング動作をどのように定義するか。                                                                                                  |

これらの概念は ClickHouse のパフォーマンスを理解するうえで基本となるものであり、ClickStack の管理時に適切な運用判断を行う助けになります。
