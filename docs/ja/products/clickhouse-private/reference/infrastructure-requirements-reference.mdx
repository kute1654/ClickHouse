<div id="iam-service-account-requirements">
  ## IAM / サービスアカウントの要件
</div>

<div id="aws">
  ### AWS
</div>

<div id="ecr-pull-role">
  #### ECR Pull ロール
</div>

* **ロール名:** `ClickHouseAirgapECRPullRole`
* **目的:** ClickHouse のプライベート ECR からイメージを Pull する
* **権限:** `ecr:GetDownloadUrlForLayer`, `ecr:BatchGetImage`, `ecr:BatchCheckLayerAvailability`, `ecr:DescribeImages`, `ecr:ListImages`, `ecr:GetAuthorizationToken`
* **アクション:** ロールの ARN を ClickHouse チームに提供する

<div id="clickhouse-serverkeeper-s3-role">
  #### ClickHouse Server/Keeper S3 ロール
</div>

* **プロビジョニングされた ClickHouse クラスターごとに 1 つのロール**
* **命名規則:** `CH-S3-$NAME-$REGION-$ORDINAL-Role`
* `$NAME`: クラスター名 (例: `default-xx-01`) 
* `$REGION`: 短縮したリージョン名 (例: `us-west-2` の場合は `uw2`) 
* `$ORDINAL`: 予約済み。`00` を設定
* 例: `CH-S3-default-xx-01-uw2-00-Role`
* **権限:** バケットリソースに対する `s3:*` および `s3:ListBucket`
* **信頼ポリシー:** EKS クラスターの OIDC プロバイダーを使用する `sts:AssumeRoleWithWebIdentity` 経由の IRSA

<div id="ebs-csi-driver-role">
  #### EBS CSI ドライバーのロール
</div>

* **名前の例:** `ClickHouse_EksEbsCsiDriverRole`
* **マネージドポリシー:** `arn:aws:iam::aws:policy/service-role/AmazonEBSCSIDriverPolicy`
* **信頼ポリシー:** EKS クラスターの OIDC プロバイダーを使用する IRSA

<div id="cluster-naming-conventions">
  ## クラスターの命名規則
</div>

ClickHouseクラスターの命名規則は、すべてのプラットフォームで統一されています。

```text
$DESCRIPTOR-$LETTERS-$ORDINAL
```

| コンポーネント       | 説明             | 例         |
| ------------- | -------------- | --------- |
| `$DESCRIPTOR` | 識別用の名前 (英字のみ)  | `default` |
| `$LETTERS`    | 予約済み。任意の英字2文字  | `xx`      |
| `$ORDINAL`    | `01` から始まる連番   | `01`      |

**名前の例:** `default-xx-01`

クラスター名は**対象の Kubernetes クラスター内で一意**である必要があり、以下で使用されます: - Kubernetes ネームスペース: `ns-$CLUSTER_NAME` - Kubernetes リソース: `c-$CLUSTER_NAME` をプレフィックスとして付与 - IAM ロール名 (AWS) : `CH-S3-$CLUSTER_NAME-$REGION-$ORDINAL-Role` - S3/GCS の秘密鍵プレフィックス: `ch-s3-<uuid>` (クラスターごとに一意)
