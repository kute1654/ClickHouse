Private API を使用して ClickHouse ユーザーのパスワードをリセットします。

<div id="prerequisites">
  ## 前提条件
</div>

* ClickHouse Private API がインストールされており、アクセス可能であること (例: `http://localhost:8080/` へのポートフォワード経由) 
* 少なくとも 1 つの ClickHouse クラスターがデプロイされていること (このガイドでは `default-xx-01` を使用します) 

<Steps>
  <Step>
    ## 1. ハッシュ化されたパスワードを生成する

    新しいパスワードを SHA-256 でハッシュ化し、その結果を base64 エンコードします。

    ```ini
    PASSWORD='My super secret p@$$w0rd'
    HASHED_PASSWORD=$(echo -n "$PASSWORD" | shasum -a 256 | awk '{printf $1}' | base64)
    ```
  </Step>

  <Step>
    ## 2. リクエストボディを作成する

    ハッシュ化されたパスワードを含む JSONファイルを作成します。

    ```json
    {
      "user_hashed_password": "<base64-encoded-sha256-hash>",
      "hashing_function": "sha256",
      "username": "default"
    }
    ```

    これを `password_reset.json` として保存します。

    | フィールド                  | 必須  | デフォルト     | 説明                           |
    | ---------------------- | --- | --------- | ---------------------------- |
    | `user_hashed_password` | はい  | --        | 新しいパスワードの Base64 エンコード済みハッシュ |
    | `hashing_function`     | いいえ | `sha256`  | 使用するハッシュアルゴリズム               |
    | `username`             | いいえ | `default` | パスワードをリセットする対象のユーザー名         |
  </Step>

  <Step>
    ## 3. リクエストを送信する

    ```bash
    curl -X POST "http://localhost:8080/api/v1/instances/default-xx-01/reset-user-password" \
        -H "Content-Type: application/json" \
        --data-binary "@password_reset.json"
    ```

    成功時のレスポンス:

    ```json
    {
      "message": "User password reset successfully"
    }
    ```
  </Step>

  <Step>
    ## 4. 変更を確認する

    パスワードのリセットは、オペレーターが変更を処理した後に有効になります。新しいパスワードでクラスターに接続して確認してください。

    status エンドポイントで処理状況を確認することもできます。

    ```bash
    curl "http://localhost:8080/api/v1/instances/default-xx-01/status"
    ```
  </Step>
</Steps>

<div id="hydra-limitation">
  ## Hydra の制限
</div>

Hydra の子インスタンスでは、パスワードを**リセットできません**。Hydra の子インスタンスのパスワードをリセットする場合は、代わりに親インスタンスでリセットしてください。
