Console での監視に加えて、ClickPipes はスクレイピング用に [Prometheus 互換エンドポイント](/ja/products/cloud/features/monitoring/prometheus) へメトリクスを公開しています。これらのメトリクスは、他の ClickHouse Cloud サービスのメトリクスとあわせて公開されるため、既存のオブザーバビリティスタック (例: [Grafana](/ja/products/cloud/features/monitoring/prometheus#integrating-with-grafana)、[Datadog](/ja/products/cloud/features/monitoring/prometheus#integrating-with-datadog)) に ClickPipes の監視を統合できます。

<div id="metric-labels">
  ## メトリクスラベル
</div>

ClickPipes のメトリクスでは、[標準のサービスレベルラベル](/ja/products/cloud/features/monitoring/prometheus#metric-labels) (`clickhouse_org`、`clickhouse_service`、`clickhouse_service_name`) に加えて、以下の ClickPipes 固有のラベルを使用します。

| Label              | Description                                            |
| ------------------ | ------------------------------------------------------ |
| `clickpipe_id`     | ClickPipe の一意な識別子                                      |
| `clickpipe_name`   | ClickPipe の名前                                          |
| `clickpipe_source` | ソースの種類 (例: `kafka`、`postgres`、`s3`)                    |
| `clickpipe_state`  | ClickPipe の状態 (例: `Stopped`、`Provisioning`、`Running`)  |

<div id="available-metrics">
  ## 利用可能なメトリクス
</div>

<div id="metrics-data-transfer">
  ### データ転送
</div>

| メトリクス                                     | Type  | Description                                                              |
| ------------------------------------------ | ----- | ------------------------------------------------------------------------ |
| `ClickPipes_FetchedBytes_Total`            | カウンター | ソースから取得した非圧縮バイト数の合計。                                                     |
| `ClickPipes_FetchedBytesCompressed_Total`  | カウンター | ソースから取得した圧縮バイト数の合計。ソースデータが非圧縮の場合は、`ClickPipes_FetchedBytes_Total` と同じです。 |
| `ClickPipes_FetchedBytesInitialLoad_Total` | カウンター | **CDC ClickPipes** 初期ロードフェーズ中に取得した非圧縮バイト数の合計。                            |
| `ClickPipes_FetchedBytesResync_Total`      | カウンター | **CDC ClickPipes** 再同期処理中に取得した非圧縮バイト数の合計。                                |
| `ClickPipes_SentBytes_Total`               | カウンター | ClickHouse に送信した非圧縮バイト数の合計。                                              |
| `ClickPipes_SentBytesCompressed_Total`     | カウンター | ClickHouse に送信した圧縮バイト数の合計。                                               |

<div id="metrics-events">
  ### イベントとレコード
</div>

| メトリクス                               | タイプ   | 説明                                                                                                |
| ----------------------------------- | ----- | ------------------------------------------------------------------------------------------------- |
| `ClickPipes_FetchedEvents_Total`    | カウンター | ソースから取得したレコードの総数。                                                                                 |
| `ClickPipes_SentEvents_Total`       | カウンター | ClickHouse に送信されたレコードの総数。                                                                         |
| `ClickPipes_FetchedEvents_PerTable` | Gauge | **CDC ClickPipes** 宛先テーブルおよびイベントタイプごとに取得したレコード数。追加の `destination_table` と `event_type` ラベルを使用します。 |

<div id="metrics-errors">
  ### エラー
</div>

| メトリクス                     | 型     | 説明                                                                   |
| ------------------------- | ----- | -------------------------------------------------------------------- |
| `ClickPipes_Errors_Total` | カウンター | インジェスト中に発生した[エラー](/ja/integrations/clickpipes/home#error-reporting)の総数。 |

<div id="metrics-latency">
  ### レイテンシ
</div>

| メトリクス                                     | 型             | 説明                                                                                            |
| ----------------------------------------- | ------------- | --------------------------------------------------------------------------------------------- |
| `ClickPipes_Latency`                      | Gauge (*avg*) | レコードがソースで生成されてから ClickHouse に書き込まれるまでの時間 (ミリ秒) 。                                              |
| `ClickPipes_SourceReplicationLatency_MiB` | Gauge (*avg*) | **CDC ClickPipes** におけるソース側のレプリケーションラグ (MiB) 。Postgres ClickPipes では、これはレプリケーションスロットのラグを表します。 |

<div id="metrics-resources">
  ### リソース使用量
</div>

| メトリクス                            | 型              | 説明                                  |
| -------------------------------- | -------------- | ----------------------------------- |
| `ClickPipes_Replica_CPUUsage`    | Gauge (*avg*)  | インジェスト用レプリカの平均 CPU 使用量 (ミリコア単位) 。   |
| `ClickPipes_Replica_CPULimit`    | Gauge (*last*) | インジェスト用レプリカに設定された CPU 制限 (ミリコア単位) 。 |
| `ClickPipes_Replica_MemoryUsage` | Gauge (*avg*)  | インジェスト用レプリカの平均メモリ使用量 (バイト単位) 。      |
| `ClickPipes_Replica_MemoryLimit` | Gauge (*last*) | インジェスト用レプリカに設定されたメモリ制限 (バイト単位) 。    |

<div id="metrics-state">
  ### 状態と進捗
</div>

| メトリクス                           | 型     | 説明                                                              |
| ------------------------------- | ----- | --------------------------------------------------------------- |
| `ClickPipes_Info`               | Gauge | 常に `1` です。`clickpipe_state` ラベルを使用して、パイプの状態に基づいて絞り込みやアラートを行います。 |
| `ClickPipes_LastFetchedBatchId` | Gauge | ソースから最後に取得されたバッチの ID。                                           |
| `ClickPipes_LastSentBatchId`    | Gauge | ClickHouse に最後に送信されたバッチの ID。                                    |
