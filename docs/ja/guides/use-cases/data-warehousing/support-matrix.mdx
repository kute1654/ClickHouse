このページでは、ClickHouse のデータレイクインテグレーションに関する包括的なサポートマトリクスを紹介します。各オープンテーブルフォーマットで利用できる機能、ClickHouse が接続可能なカタログ、および各カタログでサポートされている機能をまとめています。

<div id="format-support">
  ## オープンテーブルフォーマットのサポート
</div>

ClickHouse は、[Apache Iceberg](/ja/reference/engines/table-engines/integrations/iceberg)、[Delta Lake](/ja/reference/engines/table-engines/integrations/deltalake)、[Apache Hudi](/ja/reference/engines/table-engines/integrations/hudi)、[Apache Paimon](/ja/reference/functions/table-functions/paimon) の 4 つのオープンテーブルフォーマットに対応しています。サポートマトリクスを表示するには、以下からフォーマットを選択してください。

**凡例:** ✅ 対応 | ⚠️ 一部対応 / 実験的 | ❌ 未対応

  <Tab title="Delta Lake">
    バージョン 25.6 以降、ClickHouse は Delta Lake テーブルの読み取りに Delta Lake Rust kernel を使用し、より広範な機能をサポートしています。ただし、Azure Blob Storage 上のデータにアクセスする際には既知の問題があります。そのため、Azure Blob Storage 上のデータを読み取る場合、この Kernel は無効になります。以下に、この kernel が必要な機能を示します。

    | Feature                   | Status | Notes                                                                                                                                                                   |
    | ------------------------- | :----: | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
    | **Storage backends**      |        |                                                                                                                                                                         |
    | AWS S3                    |    ✅   | [`deltaLake()`](/ja/reference/functions/table-functions/deltalake) または `deltaLakeS3()` を使用                                                                                 |
    | GCS                       |    ✅   | [`deltaLake()`](/ja/reference/functions/table-functions/deltalake) または `deltaLakeS3()` を使用                                                                                 |
    | Azure Blob Storage        |    ✅   | [`deltaLakeAzure()`](/ja/reference/functions/table-functions/deltalake) を使用                                                                                                |
    | HDFS                      |    ❌   | サポートされていません                                                                                                                                                             |
    | Local filesystem          |    ✅   | [`deltaLakeLocal()`](/ja/reference/functions/table-functions/deltalake) を使用                                                                                                |
    | **Access methods**        |        |                                                                                                                                                                         |
    | Table function            |    ✅   | バックエンドごとのバリアントを持つ [`deltaLake()`](/ja/reference/functions/table-functions/deltalake)                                                                                       |
    | Table engine              |    ✅   | [`DeltaLake`](/ja/reference/engines/table-engines/integrations/deltalake)                                                                                                  |
    | Cluster-distributed reads |    ✅   | [`deltaLakeCluster`](/ja/reference/functions/table-functions/deltalakeCluster), [`deltaLakeAzureCluster`](/ja/reference/functions/table-functions/deltalakeCluster)           |

    | Read support              |    ✅   | すべての ClickHouse SQL 関数を含む完全な SELECT サポート                                                                                                                                |
    | Partition pruning         |    ✅   | Delta Kernel が必要です。                                                                                                                                                     |
    | Schema evolution          |    ✅   | Delta Kernel が必要です。                                                                                                                                                     |
    | Time travel               |    ✅   | Delta Kernel が必要です。                                                                                                                                                     |
    | Deletion vectors          |    ✅   |                                                                                                                                                                         |
    | Column mapping            |    ✅   |                                                                                                                                                                         |
    | Change data feed          |    ✅   | Delta Kernel が必要です。                                                                                                                                                     |
    | Virtual columns           |    ✅   | `_path`, `_file`, `_size`, `_time`, `_etag`。詳細は [Virtual columns](/ja/reference/functions/table-functions/deltalake#virtual-columns) を参照してください。                            |
    | **Write features**        |        |                                                                                                                                                                         |
    | INSERT                    |    ✅   | Experimental。`allow_experimental_delta_lake_writes = 1` が必要です。[DeltaLake engine](/ja/reference/engines/table-engines/integrations/deltalake) を参照してください。Delta Kernel が必要です。 |
    | DELETE / UPDATE / MERGE   |    ❌   | サポートされていません                                                                                                                                                             |
    | CREATE empty table        |    ❌   | 新しい空の Delta Lake テーブルの作成はサポートされていません。`CREATE TABLE` 操作では、object storage 上に既存の Delta Lake が存在することを前提としています。                                                              |
    | **Caching**               |        |                                                                                                                                                                         |
    | Data caching              |    ✅   | S3/Azure/HDFS ストレージエンジンと同じ仕組みです。詳細は [Data cache](/ja/reference/engines/table-engines/integrations/deltalake#data-cache) を参照してください。                                         |
  </Tab>

<div id="catalog-support">
  ## カタログのサポート
</div>

ClickHouse は、[`DataLakeCatalog`](/ja/reference/engines/database-engines/datalake) データベースエンジンを使用して外部データカタログに接続できます。このエンジンにより、カタログが ClickHouse データベースとして公開されます。カタログに登録されたテーブルは自動的に表示され、標準 SQL でクエリできます。

現在サポートされているカタログは次のとおりです。セットアップ手順の詳細については、各カタログのリファレンスガイドを参照してください。

| カタログ                                                                         | フォーマット         |  読み取り | テーブルの作成 | INSERT | 参照ガイド                                                                       |
| ---------------------------------------------------------------------------- | -------------- | :---: | :-----: | :----: | --------------------------------------------------------------------------- |
| [AWS Glue Catalog](/ja/guides/use-cases/data-warehousing/glue-catalog)          | Iceberg        | ✅ ベータ |    ❌    |    ❌   | [Glue カタログガイド](/ja/guides/use-cases/data-warehousing/glue-catalog)             |
| [BigLake Metastore](/ja/guides/use-cases/data-warehousing/biglake-catalog)      | Iceberg        | ✅ ベータ |    ❌    |    ❌   | [BigLake Metastore ガイド](/ja/guides/use-cases/data-warehousing/biglake-catalog) |
| [Databricks Unity Catalog](/ja/guides/use-cases/data-warehousing/unity-catalog) | Delta, Iceberg | ✅ ベータ |  ✅ ベータ  |  ✅ ベータ | [Unity Catalog ガイド](/ja/guides/use-cases/data-warehousing/unity-catalog)       |
| [Iceberg REST](/ja/guides/use-cases/data-warehousing/rest-catalog)              | Iceberg        | ✅ ベータ |    ❌    |    ❌   | [REST カタログガイド](/ja/guides/use-cases/data-warehousing/rest-catalog)             |
| [Lakekeeper](/ja/guides/use-cases/data-warehousing/lakekeeper-catalog)          | Iceberg        | ✅ ベータ |    ❌    |    ❌   | [Lakekeeper カタログガイド](/ja/guides/use-cases/data-warehousing/lakekeeper-catalog) |

現在、すべてのカタログインテグレーションで実験的またはベータの設定を有効にする必要があります。Microsoft OneLake と Databricks Unity Catalog を除き、すべてのカタログは **読み取り専用** アクセスです。つまり、テーブルはクエリできますが、カタログ接続を通じて作成したり書き込んだりすることはできません。カタログから ClickHouse にデータを読み込んで分析を高速化するには、[分析高速化ガイド](/ja/guides/use-cases/data-warehousing/getting-started/accelerating-analytics) で説明されている `INSERT INTO SELECT` を使用してください。オープンテーブルフォーマットにデータを書き戻すには、[データ書き込みガイド](/ja/guides/use-cases/data-warehousing/getting-started/writing-data) で説明されているように、スタンドアロンの Iceberg テーブルを作成してください。
