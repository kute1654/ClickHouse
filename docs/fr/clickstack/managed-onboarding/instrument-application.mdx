Ce guide explique comment instrumenter une petite application Node.js avec OpenTelemetry et envoyer ses logs, métriques et traces vers Managed ClickStack. Le backend est instrumenté sans modifier le code source de l’application.

Le [HackerNews Analyzer](https://github.com/ClickHouse/hn-news-analyzer) est une application Node.js qui interroge le jeu de données HackerNews hébergé dans la démo publique de ClickHouse. Chaque graphique, tableau et champ de recherche repose sur une véritable requête ClickHouse, de sorte que chaque interaction produit une trace dont le span principal correspond à l’appel HTTPS émis par le backend vers ClickHouse.

Ce guide suppose que vous avez terminé [Configuration de votre OpenTelemetry Collector](/fr/clickstack/managed-onboarding/setting-up-your-opentelemetry-collector) et qu’un collecteur ClickStack est en cours d’exécution et accessible depuis la machine sur laquelle vous exécutez cette application. **Assurez-vous d’avoir noté son endpoint OTLP** ainsi que le `OTLP_AUTH_TOKEN` que vous avez défini lors de son déploiement.

<div id="prerequisites">
  ## Prérequis
</div>

* Un collecteur ClickStack accessible depuis cette machine. Si vous n&#39;en avez pas encore déployé un, suivez d&#39;abord [Configurer votre OpenTelemetry Collector](/fr/clickstack/managed-onboarding/setting-up-your-opentelemetry-collector).
* L&#39;endpoint OTLP de ce collecteur et le `OTLP_AUTH_TOKEN` que vous y avez configuré.
* Node 18+ et npm.
