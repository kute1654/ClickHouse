<div id="private-regions">
  ## Régions privées
</div>

<EnterprisePlanFeatureBadge feature="Fonctionnalité Régions privées" />

Nous proposons des régions privées pour nos services du niveau Enterprise. Veuillez [nous contacter](https://clickhouse.com/company/contact) pour toute demande de région privée.

Points importants à prendre en compte pour les régions privées :

* Les services ne bénéficient pas de la mise à l’échelle automatique ; en revanche, la mise à l’échelle verticale et horizontale manuelle est prise en charge.
* Les services ne peuvent pas être mis en veille.
* La page d’état n’est pas disponible pour les régions privées.

Des exigences supplémentaires peuvent s’appliquer pour la conformité HIPAA (y compris la signature d’un BAA). Notez que HIPAA est actuellement disponible uniquement pour les services du niveau Enterprise

<div id="hipaa-compliant-regions">
  ## Régions conformes à HIPAA
</div>

<EnterprisePlanFeatureBadge feature="HIPAA" support="true" />

Les clients doivent signer un Business Associate Agreement (BAA) et demander l’onboarding auprès des équipes commerciales ou du support afin de mettre en place des services dans des régions conformes à HIPAA. La conformité HIPAA est prise en charge dans les régions suivantes :

* AWS af-south-1 (Afrique du Sud) **Région privée**
* AWS ca-central-1 (Canada) **Région privée**
* AWS eu-central-1 (Francfort)
* AWS eu-north-1 (Stockholm) **Région privée**
* AWS eu-west-1 (Irlande)
* AWS eu-west-2 (Londres)
* AWS sa-east-1 (Amérique du Sud) **Région privée**
* AWS us-east-1 (N. Virginia)
* AWS us-east-2 (Ohio)
* AWS us-west-2 (Oregon)
* GCP europe-west4 (Pays-Bas)
* GCP us-central1 (Iowa)
* GCP us-east1 (Caroline du Sud)
