Balises de l’autoscaler (si vous utilisez cluster-autoscaler) : - `k8s.io/cluster-autoscaler/enabled: true` - `k8s.io/cluster-autoscaler/$CLUSTER_NAME: owned`

  ## Exigences IAM / compte de service
</div>

<div id="aws">
  ### AWS
</div>

<div id="ecr-pull-role">
  #### Rôle ECR pour la récupération
</div>

* **Nom du rôle :** `ClickHouseAirgapECRPullRole`
* **Objectif :** Récupérer des images depuis l&#39;ECR privé de ClickHouse
* **Autorisations :** `ecr:GetDownloadUrlForLayer`, `ecr:BatchGetImage`, `ecr:BatchCheckLayerAvailability`, `ecr:DescribeImages`, `ecr:ListImages`, `ecr:GetAuthorizationToken`
* **Action :** Fournir l&#39;ARN du rôle à l&#39;équipe ClickHouse

<div id="clickhouse-serverkeeper-s3-role">
  #### Rôle S3 de ClickHouse Server/Keeper
</div>

* **Un rôle par cluster ClickHouse provisionné**
* **Convention de nommage :** `CH-S3-$NAME-$REGION-$ORDINAL-Role`
* `$NAME` : nom du cluster (p. ex., `default-xx-01`)
* `$REGION` : région abrégée (p. ex., `uw2` pour `us-west-2`)
* `$ORDINAL` : réservé, défini à `00`
* Exemple : `CH-S3-default-xx-01-uw2-00-Role`
* **Autorisations :** `s3:*`, `s3:ListBucket` sur la ressource du bucket
* **Politique de confiance :** IRSA via `sts:AssumeRoleWithWebIdentity` avec le fournisseur OIDC du cluster EKS

<div id="ebs-csi-driver-role">
  #### Rôle du driver EBS CSI
</div>

* **Nom de l’exemple :** `ClickHouse_EksEbsCsiDriverRole`
* **Stratégie gérée :** `arn:aws:iam::aws:policy/service-role/AmazonEBSCSIDriverPolicy`
* **Stratégie d’approbation :** IRSA avec le fournisseur OIDC du cluster EKS

<div id="cluster-naming-conventions">
  ## Conventions de nommage des clusters
</div>

La convention de nommage des clusters ClickHouse est la même sur toutes les plateformes :

```text
$DESCRIPTOR-$LETTERS-$ORDINAL
```

| Composant     | Description                                   | Exemple   |
| ------------- | --------------------------------------------- | --------- |
| `$DESCRIPTOR` | Nom descriptif (lettres uniquement)           | `default` |
| `$LETTERS`    | Réservé ; deux lettres quelconques            | `xx`      |
| `$ORDINAL`    | Numéro d’ordre incrémentiel commençant à `01` | `01`      |

**Nom d’exemple :** `default-xx-01`

Le nom du cluster doit être **unique au sein du cluster Kubernetes cible** et est utilisé dans : - l’espace de noms Kubernetes : `ns-$CLUSTER_NAME` - les ressources Kubernetes : préfixées par `c-$CLUSTER_NAME` - les noms de rôles IAM (AWS) : `CH-S3-$CLUSTER_NAME-$REGION-$ORDINAL-Role` - les préfixes de clés S3/GCS : `ch-s3-<uuid>` (unique par cluster)
