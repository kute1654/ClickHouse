Réinitialiser les mots de passe des utilisateurs ClickHouse à l’aide de la Private API.

<div id="prerequisites">
  ## Prérequis
</div>

* ClickHouse Private API installée et accessible (par ex. via une redirection de port vers `http://localhost:8080/`)
* Au moins un cluster ClickHouse déployé (ce guide utilise `default-xx-01`)

<Steps>
  <Step>
    ## 1. Générer un mot de passe haché

    Hachez le nouveau mot de passe avec SHA-256, puis encodez le résultat en base64 :

    ```ini
    PASSWORD='My super secret p@$$w0rd'
    HASHED_PASSWORD=$(echo -n "$PASSWORD" | shasum -a 256 | awk '{printf $1}' | base64)
    ```
  </Step>

  <Step>
    ## 2. Créer le corps de la requête

    Créez un fichier JSON contenant le mot de passe haché :

    ```json
    {
      "user_hashed_password": "<base64-encoded-sha256-hash>",
      "hashing_function": "sha256",
      "username": "default"
    }
    ```

    Enregistrez-le sous `password_reset.json`.

    | Champ                  | Obligatoire | Par défaut | Description                                                   |
    | ---------------------- | ----------- | ---------- | ------------------------------------------------------------- |
    | `user_hashed_password` | Oui         | --         | Hachage du nouveau mot de passe, encodé en base64             |
    | `hashing_function`     | Non         | `sha256`   | Algorithme de hachage utilisé                                 |
    | `username`             | Non         | `default`  | Nom d’utilisateur dont le mot de passe doit être réinitialisé |
  </Step>

  <Step>
    ## 3. Envoyer la requête

    ```bash
    curl -X POST "http://localhost:8080/api/v1/instances/default-xx-01/reset-user-password" \
        -H "Content-Type: application/json" \
        --data-binary "@password_reset.json"
    ```

    Réponse en cas de succès :

    ```json
    {
      "message": "User password reset successfully"
    }
    ```
  </Step>

  <Step>
    ## 4. Vérifier la modification

    La réinitialisation du mot de passe prend effet une fois que l’opérateur a traité la modification. Vérifiez en vous connectant au cluster avec le nouveau mot de passe.

    Vous pouvez également suivre l’opération via l’endpoint de statut :

    ```bash
    curl "http://localhost:8080/api/v1/instances/default-xx-01/status"
    ```
  </Step>
</Steps>

<div id="hydra-limitation">
  ## Limitation de Hydra
</div>

La réinitialisation des mots de passe **n’est pas possible** sur les instances enfants Hydra. Pour réinitialiser le mot de passe d’une instance enfant Hydra, effectuez plutôt la réinitialisation sur l’instance parente.
