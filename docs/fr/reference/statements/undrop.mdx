Annule la suppression de la table.

À partir de la version 23.3 de ClickHouse, il est possible d’exécuter UNDROP sur une table dans une base de données Atomic
pendant le délai `database_atomic_delay_before_drop_table_sec` (8 minutes par défaut) après l’exécution de l’instruction DROP TABLE. Les tables supprimées sont répertoriées dans
une table système appelée `system.dropped_tables`.

Si vous avez une vue matérialisée sans clause `TO` associée à la table supprimée, vous devrez également exécuter UNDROP sur la table interne de cette vue.

<Tip>
  Voir aussi [DROP TABLE](/fr/reference/statements/drop)
</Tip>

Syntaxe :

```sql
UNDROP TABLE [db.]name [UUID '<uuid>'] [ON CLUSTER cluster]
```

**Exemple**

```sql
CREATE TABLE tab
(
    `id` UInt8
)
ENGINE = MergeTree
ORDER BY id;

DROP TABLE tab;

SELECT *
FROM system.dropped_tables
FORMAT Vertical;
```

```response

──────

table_dropped_time:    2023-04-05 14:12:12

1 row in set. Elapsed: 0.001 sec. 
```

````sql
UNDROP TABLE tab;

SELECT *
FROM system.dropped_tables
FORMAT Vertical;

```response
Ok.

0 rows in set. Elapsed: 0.001 sec. 
````
