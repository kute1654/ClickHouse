<div id="ssd_cache">
  ## ssd_cache
</div>

Semblable à `cache`, mais stocke les données sur SSD et l’index en RAM. Tous les paramètres des dictionnaires de cache liés à la file d’attente des mises à jour peuvent également s’appliquer aux dictionnaires de cache SSD.

La clé du dictionnaire est de type [UInt64](/fr/reference/data-types/int-uint).

<Tabs>
  <Tab title="DDL">
    ```sql
    LAYOUT(SSD_CACHE(BLOCK_SIZE 4096 FILE_SIZE 16777216 READ_BUFFER_SIZE 1048576
        PATH '/var/lib/clickhouse/user_files/test_dict'))
    ```
  </Tab>

  <Tab title="Fichier de configuration">
    ```xml
    <layout>
        <ssd_cache>
            <!-- Taille du bloc de lecture élémentaire en octets. Il est recommandé qu’elle soit égale à la taille de page du SSD. -->
            <block_size>4096</block_size>
            <!-- Taille maximale du fichier de cache en octets. -->
            <file_size>16777216</file_size>
            <!-- Taille du tampon en RAM, en octets, pour la lecture des éléments depuis le SSD. -->
            <read_buffer_size>131072</read_buffer_size>
            <!-- Taille du tampon en RAM, en octets, pour l’agrégation des éléments avant leur écriture sur le SSD. -->
            <write_buffer_size>1048576</write_buffer_size>
            <!-- Chemin où sera stocké le fichier de cache. -->
            <path>/var/lib/clickhouse/user_files/test_dict</path>
        </ssd_cache>
    </layout>
    ```
  </Tab>
</Tabs>

<br />

<div id="complex_key_ssd_cache">
  ## complex_key_ssd_cache
</div>

Ce type de stockage est utilisé avec des [clés](/fr/reference/statements/create/dictionary/attributes#composite-key) composites. Semblable à `ssd_cache`.
