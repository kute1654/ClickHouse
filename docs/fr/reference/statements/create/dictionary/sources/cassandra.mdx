Exemple de paramètres :

<Tabs>
  <Tab title="DDL">
    ```sql
    SOURCE(CASSANDRA(
        host 'localhost'
        port 9042
        user 'username'
        password 'qwerty123'
        keyspace 'database_name'
        column_family 'table_name'
        allow_filtering 1
        partition_key_prefix 1
        consistency 'One'
        where '"SomeColumn" = 42'
        max_threads 8
        query 'SELECT id, value_1, value_2 FROM database_name.table_name'
    ))
    ```
  </Tab>

  <Tab title="Fichier de configuration">
    ```xml
    <source>
        <cassandra>
            <host>localhost</host>
            <port>9042</port>
            <user>username</user>
            <password>qwerty123</password>
            <keyspase>database_name</keyspase>
            <column_family>table_name</column_family>
            <allow_filtering>1</allow_filtering>
            <partition_key_prefix>1</partition_key_prefix>
            <consistency>One</consistency>
            <where>"SomeColumn" = 42</where>
            <max_threads>8</max_threads>
            <query>SELECT id, value_1, value_2 FROM database_name.table_name</query>
        </cassandra>
    </source>
    ```
  </Tab>
</Tabs>

Champs des paramètres :

| Paramètre              | Description                                                                                                                                                                                                                                                                                                                                                                              |
| ---------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `host`                 | L’hôte Cassandra ou une liste d’hôtes séparés par des virgules.                                                                                                                                                                                                                                                                                                                          |
| `port`                 | Le port des serveurs Cassandra. S’il n’est pas spécifié, le port par défaut `9042` est utilisé.                                                                                                                                                                                                                                                                                          |
| `user`                 | Nom de l’utilisateur Cassandra.                                                                                                                                                                                                                                                                                                                                                          |
| `password`             | Mot de passe de l’utilisateur Cassandra.                                                                                                                                                                                                                                                                                                                                                 |
| `keyspace`             | Nom du keyspace (base de données).                                                                                                                                                                                                                                                                                                                                                       |
| `column_family`        | Nom de la famille de colonnes (table).                                                                                                                                                                                                                                                                                                                                                   |
| `allow_filtering`      | Indicateur permettant d’autoriser, ou non, des conditions potentiellement coûteuses sur les colonnes de clé de clustering. La valeur par défaut est `1`.                                                                                                                                                                                                                                 |
| `partition_key_prefix` | Nombre de colonnes de clé de partition dans la clé primaire de la table Cassandra. Nécessaire pour les dictionnaires à clé composite. L’ordre des colonnes de clé dans la définition du dictionnaire doit être le même que dans Cassandra. La valeur par défaut est `1` (la première colonne de clé est une clé de partition et les autres colonnes de clé sont des clés de clustering). |
| `consistency`          | Niveau de cohérence. Valeurs possibles : `One`, `Two`, `Three`, `All`, `EachQuorum`, `Quorum`, `LocalQuorum`, `LocalOne`, `Serial`, `LocalSerial`. La valeur par défaut est `One`.                                                                                                                                                                                                       |
| `where`                | Critères de sélection facultatifs.                                                                                                                                                                                                                                                                                                                                                       |
| `max_threads`          | Nombre maximal de threads à utiliser pour charger les données depuis plusieurs partitions dans les dictionnaires à clé composite.                                                                                                                                                                                                                                                        |
| `query`                | Requête personnalisée. Facultatif.                                                                                                                                                                                                                                                                                                                                                       |

<Note>
  Les champs `column_family` et `where` ne peuvent pas être utilisés avec le champ `query`. De plus, l’un des champs `column_family` ou `query` doit être défini.
</Note>
