Renvoyer une valeur évaluée sur la ligne située à un décalage de offset lignes après la ligne actuelle dans la fenêtre ordonnée.
Cette fonction est similaire à [`leadInFrame`](/fr/reference/functions/window-functions/leadInFrame), mais utilise toujours la fenêtre `ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING`.

**Syntaxe**

```sql
lead(x[, offset[, default]])
  OVER ([[PARTITION BY grouping_column] [ORDER BY sorting_column]] | [window_name])
FROM table_name
WINDOW window_name as ([[PARTITION BY grouping_column] [ORDER BY sorting_column])
```

Pour plus de détails sur la syntaxe des fonctions de fenêtre, voir : [Window Functions - Syntax](/fr/reference/functions/window-functions/index#syntax).

**Paramètres**

* `x` — Nom de la colonne.
* `offset` — Décalage à appliquer. [(U)Int*](/fr/reference/data-types/int-uint). (Facultatif - `1` par défaut).
* `default` — Valeur à renvoyer si la ligne calculée dépasse les limites du cadre de la fenêtre. (Facultatif - valeur par défaut du type de colonne si omise).

**Valeur renvoyée**

* valeur évaluée sur la ligne située `offset` lignes après la ligne actuelle dans le cadre ordonné.

**Exemple**

Cet exemple examine des [données historiques](https://www.kaggle.com/datasets/sazidthe1/nobel-prize-data) sur les lauréats du prix Nobel et utilise la fonction `lead` pour renvoyer une liste de lauréats successifs dans la catégorie de physique.

```sql title="Query"
CREATE OR REPLACE VIEW nobel_prize_laureates
AS SELECT *
FROM file('nobel_laureates_data.csv');
```

```sql title="Query"
SELECT
    fullName,
    lead(year, 1, year) OVER (PARTITION BY category ORDER BY year ASC
      ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING
    ) AS year,
    category,
    motivation
FROM nobel_prize_laureates
WHERE category = 'physics'
ORDER BY year DESC
LIMIT 9
```
