<div id="cramersVBiasCorrected">
  ## cramersVBiasCorrected
</div>

Introduit dans : v22.1.0

Le V de Cramer est une mesure d’association entre deux colonnes d’une table.
Le résultat de la fonction [`cramersV`](/fr/reference/functions/aggregate-functions/cramersV) va de 0 (ce qui correspond à l’absence d’association entre les variables) à 1, et ne peut atteindre 1 que lorsque chaque valeur est entièrement déterminée par l’autre.
Cette fonction peut être fortement biaisée ; cette version du V de Cramer utilise donc la [correction de biais](https://en.wikipedia.org/wiki/Cram%C3%A9r%27s_V#Bias_correction).

**Syntaxe**

```sql
cramersVBiasCorrected(column1, column2)
```

**Arguments**

* `column1` — Première colonne à comparer. [`Any`](/fr/reference/data-types/index)
* `column2` — Deuxième colonne à comparer. [`Any`](/fr/reference/data-types/index)

**Valeur renvoyée**

Renvoie une valeur comprise entre 0 (correspondant à l’absence d’association entre les valeurs des colonnes) et 1 (association complète). [`Float64`](/fr/reference/data-types/float)

**Exemples**

**Comparaison avec cramersV classique**

```sql title=Query
SELECT
    cramersV(a, b),
    cramersVBiasCorrected(a, b)
FROM
    (
        SELECT
            number % 10 AS a,
            number % 4 AS b
        FROM
            numbers(150)
    );
```

```response title=Response
┌─────cramersV(a, b)─┬─cramersVBiasCorrected(a, b)─┐
│ 0.5798088336225178 │          0.5305112825189074 │
└────────────────────┴─────────────────────────────┘
```
