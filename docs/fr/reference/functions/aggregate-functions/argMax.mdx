<div id="argMax">
  ## argMax
</div>

Introduit dans : v1.1.0

Calcule la valeur `arg` correspondant à la valeur `val` maximale. S&#39;il existe plusieurs lignes dont la valeur `val` est égale au maximum, la valeur `arg` associée renvoyée n&#39;est pas déterministe.
`arg` et `max` se comportent tous deux comme des [fonctions d&#39;agrégation](/fr/reference/functions/aggregate-functions/index) : ils [ignorent `Null`](/fr/reference/functions/aggregate-functions/index#null-processing) pendant le traitement et renvoient des valeurs non `Null` si de telles valeurs sont disponibles.

**Voir aussi**

* [Tuple](/fr/reference/data-types/tuple)

**Syntaxe**

```sql
argMax(arg, val)
```

**Alias** : `max_by`

**Arguments**

* `arg` — Argument dont on cherche la valeur correspondant au maximum. [`const String`](/fr/reference/data-types/string)
* `val` — Valeur à maximiser. [`(U)Int8/16/32/64`](/fr/reference/data-types/int-uint) ou [`Float*`](/fr/reference/data-types/float) ou [`Date`](/fr/reference/data-types/date) ou [`DateTime`](/fr/reference/data-types/datetime) ou [`Tuple`](/fr/reference/data-types/tuple)

**Valeur renvoyée**

Renvoie la valeur de `arg` correspondant à la valeur maximale de `val`. Le type renvoyé correspond à celui de `arg`.

**Exemples**

**Utilisation de base**

```sql title=Query
SELECT argMax(user, salary) FROM salary;
```

```response title=Response
┌─argMax(user, salary)─┐
│ director             │
└──────────────────────┘
```

**Exemple détaillé avec gestion des valeurs NULL**

```sql title=Query
CREATE TABLE test
(
    a Nullable(String),
    b Nullable(Int64)
)
ENGINE = Memory AS
SELECT *
FROM VALUES(('a', 1), ('b', 2), ('c', 2), (NULL, 3), (NULL, NULL), ('d', NULL));

SELECT argMax(a, b), max(b) FROM test;
```

```response title=Response
┌─argMax(a, b)─┬─max(b)─┐
│ b            │      3 │
└──────────────┴────────┘
```

**Utilisation de Tuple dans les arguments**

```sql title=Query
SELECT argMax(a, (b,a)) FROM test;
```

```response title=Response
┌─argMax(a, tuple(b, a))─┐
│ c                      │
└────────────────────────┘
```
