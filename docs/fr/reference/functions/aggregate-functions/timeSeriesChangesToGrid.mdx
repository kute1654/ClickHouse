<div id="timeSeriesChangesToGrid">
  ## timeSeriesChangesToGrid
</div>

Introduit dans : v25.6.0

Fonction d’agrégation qui prend des séries temporelles sous forme de paires d’horodatages et de valeurs, et calcule des [changements de type PromQL](https://prometheus.io/docs/prometheus/latest/querying/functions/#changes) à partir de ces données sur une grille temporelle régulière définie par un horodatage de début, un horodatage de fin et un pas. Pour chaque point de la grille, les échantillons utilisés pour calculer `changes` sont pris en compte dans la fenêtre temporelle spécifiée.

<Note>
  Cette fonction est expérimentale. Pour l’activer, définissez `allow_experimental_ts_to_grid_aggregate_function=true`.
</Note>

**Syntaxe**

```sql
timeSeriesChangesToGrid(start_timestamp, end_timestamp, grid_step, staleness)(timestamp, value)
```

**Paramètres**

* `start_timestamp` — Spécifie le début de la grille. - `end_timestamp` — Spécifie la fin de la grille. - `grid_step` — Spécifie le pas de la grille en secondes. - `staleness` — Spécifie l’« obsolescence » maximale, en secondes, des échantillons pris en compte.

**Arguments**

* `timestamp` — Horodatage de l’échantillon. Peut s’agir de valeurs individuelles ou de tableaux. - `value` — Valeur de la série temporelle correspondant à l’horodatage. Peut s’agir de valeurs individuelles ou de tableaux.

**Valeur renvoyée**

Les valeurs `changes` sur la grille spécifiée, sous la forme d’un `Array(Nullable(Float64))`. Le tableau renvoyé contient une valeur pour chaque point de la grille temporelle. La valeur est NULL s’il n’existe aucun échantillon dans la fenêtre permettant de calculer la valeur `changes` pour un point donné de la grille.

**Exemples**

**Calculer les valeurs `changes` sur la grille [90, 105, 120, 135, 150, 165, 180, 195, 210, 225]**

```sql title=Query
WITH
    -- NOTE: the gap between 130 and 190 is to show how values are filled for ts = 180 according to window parameter
    [110, 120, 130, 190, 200, 210, 220, 230]::Array(DateTime) AS timestamps,
    [1, 1, 3, 5, 5, 8, 12, 13]::Array(Float32) AS values, -- array of values corresponding to timestamps above
    90 AS start_ts,       -- start of timestamp grid
    90 + 135 AS end_ts,   -- end of timestamp grid
    15 AS step_seconds,   -- step of timestamp grid
    45 AS window_seconds  -- "staleness" window
SELECT timeSeriesChangesToGrid(start_ts, end_ts, step_seconds, window_seconds)(timestamp, value)
FROM
(
    -- This subquery converts arrays of timestamps and values into rows of `timestamp`, `value`
    SELECT
        arrayJoin(arrayZip(timestamps, values)) AS ts_and_val,
        ts_and_val.1 AS timestamp,
        ts_and_val.2 AS value
);
```

```response title=Response
┌─timeSeriesChangesToGrid(start_ts, end_ts, step_seconds, window_seconds)(timestamp, value)─┐
│ [NULL,NULL,0,1,1,1,NULL,0,1,2]                                                            │
└───────────────────────────────────────────────────────────────────────────────────────────┘
```

**Même requête avec des arguments de type Array**

```sql title=Query
WITH
    [110, 120, 130, 190, 200, 210, 220, 230]::Array(DateTime) AS timestamps,
    [1, 1, 3, 5, 5, 8, 12, 13]::Array(Float32) AS values,
    90 AS start_ts,
    90 + 135 AS end_ts,
    15 AS step_seconds,
    45 AS window_seconds
SELECT timeSeriesChangesToGrid(start_ts, end_ts, step_seconds, window_seconds)(timestamps, values);
```

```response title=Response
┌─timeSeriesChangesToGrid(start_ts, end_ts, step_seconds, window_seconds)(timestamp, value)─┐
│ [NULL,NULL,0,1,1,1,NULL,0,1,2]                                                            │
└───────────────────────────────────────────────────────────────────────────────────────────┘
```
