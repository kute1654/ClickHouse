<div id="contingency">
  ## contingency
</div>

Introduit dans : v22.1.0

La fonction `contingency` calcule le [coefficient de contingence](https://en.wikipedia.org/wiki/Contingency_table#Cram%C3%A9r's_V_and_the_contingency_coefficient_C), une valeur qui mesure l’association entre deux colonnes d’une table.
Le calcul est similaire à celui de la fonction [`cramersV`](/fr/reference/functions/aggregate-functions/cramersV), mais avec un dénominateur différent sous la racine carrée.

**Syntaxe**

```sql
contingency(column1, column2)
```

**Arguments**

* `column1` — Première colonne à comparer. [`Any`](/fr/reference/data-types/index)
* `column2` — Deuxième colonne à comparer. [`Any`](/fr/reference/data-types/index)

**Valeur renvoyée**

Renvoie une valeur comprise entre 0 et 1. Plus le résultat est élevé, plus l’association entre les deux colonnes est forte. [`Float64`](/fr/reference/data-types/float)

**Exemples**

**Comparaison avec cramersV**

```sql title=Query
SELECT
    cramersV(a, b),
    contingency(a, b)
FROM
(
    SELECT
        number % 10 AS a,
        number % 4 AS b
    FROM
        numbers(150)
);
```

```response title=Response
┌─────cramersV(a, b)─┬──contingency(a, b)─┐
│ 0.5798088336225178 │ 0.708607540104077  │
└────────────────────┴────────────────────┘
```
