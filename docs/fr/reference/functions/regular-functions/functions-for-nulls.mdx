<div id="assumeNotNull">
  ## assumeNotNull
</div>

Introduit dans : v1.1.0

Renvoie la valeur correspondante non `Nullable` pour une valeur de type [`Nullable`](/fr/reference/data-types/nullable).
Si la valeur d’origine est `NULL`, un résultat arbitraire peut être renvoyé.

Voir aussi : les fonctions [`ifNull`](#ifNull) et [`coalesce`](#coalesce).

**Syntaxe**

```sql
assumeNotNull(x)
```

**Arguments**

* `x` — La valeur d&#39;origine de n&#39;importe quel type `Nullable`. [`Nullable(T)`](/fr/reference/data-types/nullable)

**Valeur renvoyée**

Renvoie une valeur non nullable si la valeur d&#39;origine n&#39;était pas `NULL` ; sinon, si la valeur d&#39;entrée est `NULL`, une valeur arbitraire. [`Any`](/fr/reference/data-types/index)

**Exemples**

**Exemple d&#39;utilisation**

```sql title=Query
CREATE TABLE t_null (x Int8, y Nullable(Int8))
ENGINE=MergeTree()
ORDER BY x;

INSERT INTO t_null VALUES (1, NULL), (2, 3);

SELECT assumeNotNull(y) FROM table;
SELECT toTypeName(assumeNotNull(y)) FROM t_null;
```

```response title=Response
┌─assumeNotNull(y)─┐
│                0 │
│                3 │
└──────────────────┘
┌─toTypeName(assumeNotNull(y))─┐
│ Int8                         │
│ Int8                         │
└──────────────────────────────┘
```

<div id="coalesce">
  ## coalesce
</div>

Introduite dans : v1.1.0

Renvoie le premier argument non-`NULL`.

**Syntaxe**

```sql
coalesce(x[, y, ...])
```

**Arguments**

* `x[, y, ...]` — Un nombre quelconque de paramètres de type non composite. Tous les paramètres doivent être de types de données compatibles entre eux. [`Any`](/fr/reference/data-types/index)

**Valeur renvoyée**

Renvoie le premier argument non `NULL`, ou `NULL` si tous les arguments sont `NULL`. [`Any`](/fr/reference/data-types/index) ou [`NULL`](/fr/reference/syntax#null)

**Exemples**

**Exemple d&#39;utilisation**

```sql title=Query
-- Consider a list of contacts that may specify multiple ways to contact a customer.

CREATE TABLE aBook
(
    name String,
    mail Nullable(String),
    phone Nullable(String),
    telegram Nullable(UInt32)
)
ENGINE = MergeTree
ORDER BY tuple();

INSERT INTO aBook VALUES ('client 1', NULL, '123-45-67', 123), ('client 2', NULL, NULL, NULL);

-- The mail and phone fields are of type String, but the telegram field is UInt32 so it needs to be converted to String.

-- Get the first available contact method for the customer from the contact list

SELECT name, coalesce(mail, phone, CAST(telegram,'Nullable(String)')) FROM aBook;
```

```response title=Response
┌─name─────┬─coalesce(mail, phone, CAST(telegram, 'Nullable(String)'))─┐
│ client 1 │ 123-45-67                                                 │
│ client 2 │ ᴺᵁᴸᴸ                                                      │
└──────────┴───────────────────────────────────────────────────────────┘
```

<div id="firstNonDefault">
  ## firstNonDefault
</div>

Introduit dans : v25.9.0

Renvoie la première valeur non par défaut parmi un ensemble d&#39;arguments

**Syntaxe**

```sql
firstNonDefault(arg1[, arg2[ ...]])
```

**Arguments**

* `arg1` — Premier argument à vérifier - `arg2` — Deuxième argument à vérifier - `...` — Arguments supplémentaires à vérifier

**Valeur renvoyée**

Le type de résultat est le supertype de tous les arguments

**Exemples**

**entiers**

```sql title=Query
SELECT firstNonDefault(0, 1, 2)
```

```response title=Response
1
```

**chaînes de caractères**

```sql title=Query
SELECT firstNonDefault('', 'hello', 'world')
```

```response title=Response
'hello'
```

**valeurs nulles**

```sql title=Query
SELECT firstNonDefault(NULL, 0 :: UInt8, 1 :: UInt8)
```

```response title=Response
1
```

**zéro de type Nullable**

```sql title=Query
SELECT firstNonDefault(NULL, 0 :: Nullable(UInt8), 1 :: Nullable(UInt8))
```

```response title=Response
0
```

<div id="ifNull">
  ## ifNull
</div>

Introduit dans : v1.1.0

Renvoie une autre valeur si le premier argument est `NULL`.

**Syntaxe**

```sql
ifNull(x, alt)
```

**Arguments**

* `x` — La valeur à vérifier pour déterminer si elle est `NULL`. [`Any`](/fr/reference/data-types/index)
* `alt` — La valeur que la fonction renvoie si `x` est `NULL`. [`Any`](/fr/reference/data-types/index)

**Valeur renvoyée**

Renvoie la valeur de `x` si elle n&#39;est pas `NULL`, sinon `alt`. [`Any`](/fr/reference/data-types/index)

**Exemples**

**Exemple d&#39;utilisation**

```sql title=Query
SELECT ifNull('a', 'b'), ifNull(NULL, 'b');
```

```response title=Response
┌─ifNull('a', 'b')─┬─ifNull(NULL, 'b')─┐
│ a                │ b                 │
└──────────────────┴───────────────────┘
```

<div id="isNotNull">
  ## isNotNull
</div>

Introduit dans : v1.1.0

Vérifie si l’argument n’est pas `NULL`.

Voir aussi : l’opérateur [`IS NOT NULL`](/fr/reference/operators/index#is_not_null).

**Syntaxe**

```sql
isNotNull(x)
```

**Arguments**

* `x` — Une valeur de type de données non composite. [`Any`](/fr/reference/data-types/index)

**Valeur renvoyée**

Renvoie `1` si `x` n&#39;est pas `NULL`, sinon `0`. [`UInt8`](/fr/reference/data-types/int-uint)

**Exemples**

**Exemple d&#39;utilisation**

```sql title=Query
CREATE TABLE t_null
(
  x Int32,
  y Nullable(Int32)
)
ENGINE = MergeTree
ORDER BY tuple();

INSERT INTO t_null VALUES (1, NULL), (2, 3);

SELECT x FROM t_null WHERE isNotNull(y);
```

```response title=Response
┌─x─┐
│ 2 │
└───┘
```

<div id="isNullable">
  ## isNullable
</div>

Introduit dans : v22.7.0

Vérifie si le type de données de l’argument est `Nullable` (c’est-à-dire s’il accepte les valeurs `NULL`).

**Syntaxe**

```sql
isNullable(x)
```

**Arguments**

* `x` — Une valeur de n’importe quel type de données. [`Any`](/fr/reference/data-types/index)

**Valeur renvoyée**

Renvoie `1` si `x` est du type de données `Nullable`, sinon `0`. [`UInt8`](/fr/reference/data-types/int-uint)

**Exemples**

**Exemple d’utilisation**

```sql title=Query
CREATE TABLE tab (
    ordinary_col UInt32,
    nullable_col Nullable(UInt32)
)
ENGINE = MergeTree
ORDER BY tuple();
INSERT INTO tab (ordinary_col, nullable_col) VALUES (1,1), (2, 2), (3,3);
SELECT isNullable(ordinary_col), isNullable(nullable_col) FROM tab;
```

```response title=Response
┌───isNullable(ordinary_col)──┬───isNullable(nullable_col)──┐
│                           0 │                           1 │
│                           0 │                           1 │
│                           0 │                           1 │
└─────────────────────────────┴─────────────────────────────┘
```

<div id="isZeroOrNull">
  ## isZeroOrNull
</div>

Introduit dans : v20.3.0

Vérifie si l’argument vaut soit zéro (`0`), soit `NULL`.

**Syntaxe**

```sql
isZeroOrNull(x)
```

**Arguments**

* `x` — Une valeur numérique. [`UInt`](/fr/reference/data-types/int-uint)

**Valeur renvoyée**

Renvoie `1` si `x` est `NULL` ou égal à zéro, sinon `0`. [`UInt8/16/32/64`](/fr/reference/data-types/int-uint) ou [`Float32/Float64`](/fr/reference/data-types/float)

**Exemples**

**Exemple d&#39;utilisation**

```sql title=Query
CREATE TABLE t_null
(
  x Int32,
  y Nullable(Int32)
)
ENGINE = MergeTree
ORDER BY tuple();

INSERT INTO t_null VALUES (1, NULL), (2, 0), (3, 3);

SELECT x FROM t_null WHERE isZeroOrNull(y);
```

```response title=Response
┌─x─┐
│ 1 │
│ 2 │
└───┘
```

<div id="nullIf">
  ## nullIf
</div>

Introduite dans : v1.1.0

Renvoie `NULL` si les deux arguments sont égaux.

**Syntaxe**

```sql
nullIf(x, y)
```

**Arguments**

* `x` — La première valeur. [`Any`](/fr/reference/data-types/index)
* `y` — La deuxième valeur. [`Any`](/fr/reference/data-types/index)

**Valeur renvoyée**

Renvoie `NULL` si les deux arguments sont égaux ; sinon, renvoie le premier argument. [`NULL`](/fr/reference/syntax#null) ou [`Nullable(x)`](/fr/reference/data-types/nullable)

**Exemples**

**Exemple d&#39;utilisation**

```sql title=Query
SELECT nullIf(1, 1), nullIf(1, 2);
```

```response title=Response
┌─nullIf(1, 1)─┬─nullIf(1, 2)─┐
│         ᴺᵁᴸᴸ │            1 │
└──────────────┴──────────────┘
```

<div id="toNullable">
  ## toNullable
</div>

Introduit dans : v1.1.0


**Arguments**

* `x` — Une valeur de n&#39;importe quel type non composite. [`Any`](/fr/reference/data-types/index)

**Valeur renvoyée**

Renvoie la valeur d&#39;entrée, mais avec le type `Nullable`. [`Nullable(Any)`](/fr/reference/data-types/nullable)

**Exemples**

**Exemple d&#39;utilisation**

```sql title=Query
SELECT toTypeName(10), toTypeName(toNullable(10));
```

```response title=Response
┌─toTypeName(10)─┬─toTypeName(toNullable(10))─┐
│ UInt8          │ Nullable(UInt8)            │
└────────────────┴────────────────────────────┘
```
