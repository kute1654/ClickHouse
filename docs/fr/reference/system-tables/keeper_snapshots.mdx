<div id="description">
  ## Description
</div>

Cette table n’existe pas si ce nœud n’est pas configuré pour exécuter un ClickHouse Keeper in-process. Elle contient une ligne par fichier de snapshot Raft suivi par la machine à états du Keeper in-process, y compris les snapshots en cours de réception provenant du leader.

<div id="columns">
  ## Colonnes
</div>

{/*AUTOGENERATED_START*/}

* `last_log_index` ([UInt64](/fr/reference/data-types/int-uint)) — Dernier index de journal couvert par le snapshot.
* `path` ([String](/fr/reference/data-types/string)) — Chemin du fichier de snapshot sur le disque.
* `disk_name` ([String](/fr/reference/data-types/string)) — Nom du disque qui stocke le snapshot.
* `size_bytes` ([UInt64](/fr/reference/data-types/int-uint)) — Taille du fichier de snapshot sur le disque.
* `last_modified_at` ([DateTime](/fr/reference/data-types/datetime)) — Date et heure de dernière modification du fichier de snapshot.
* `is_received` ([Bool](/fr/reference/data-types/boolean)) — `true` si le snapshot est en cours de réception depuis le leader. Pour ces lignes, `size_bytes` et `last_modified_at` reflètent le fichier partiel tel qu’écrit jusqu’à présent et peuvent donc être sous-estimés.
* `exists_on_disk` ([Bool](/fr/reference/data-types/boolean)) — Indique si le fichier de snapshot est actuellement présent sur le disque. Toujours `true` pour les snapshots finalisés (`is_received` = `false`), sauf si le fichier a été supprimé par un processus externe ou s’il est corrompu ; peut être `false` pour les snapshots en cours de réception (`is_received` = `true`) avant l’écriture du moindre octet.

{/*AUTOGENERATED_END*/}

Exemple :

```sql
SELECT * FROM system.keeper_snapshots ORDER BY last_log_index;
```

└────────────────┴───────────────────────────────┴───────────┴────────────┴─────────────────────┴─────────────┴────────────────┘
