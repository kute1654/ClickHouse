<div id="description">
  ## Description
</div>

Contient des filtres pour une table donnée, ainsi qu&#39;une liste de rôles et/ou d&#39;utilisateurs auxquels cette politique de ligne doit s&#39;appliquer.

<div id="columns">
  ## Colonnes
</div>

{/*AUTOGENERATED_START*/}

* `name` ([String](/fr/reference/data-types/index)) — Nom d’une politique de ligne.
* `short_name` ([String](/fr/reference/data-types/index)) — Nom court d’une politique de ligne. Les noms des politiques de ligne sont composés, par exemple : myfilter ON mydb.mytable. Ici, &#39;myfilter ON mydb.mytable&#39; est le nom de la politique de ligne, et &#39;myfilter&#39; en est le nom court.
* `database` ([String](/fr/reference/data-types/index)) — Nom de la base de données.
* `table` ([String](/fr/reference/data-types/index)) — Nom de la table. Vide si la politique concerne la base de données.
* `id` ([UUID](/fr/reference/data-types/index)) — ID de la politique de ligne.
* `storage` ([String](/fr/reference/data-types/index)) — Nom du répertoire dans lequel la politique de ligne est stockée.
* `select_filter` ([Nullable(String)](/fr/reference/data-types/index)) — Expression utilisée pour le filtrage dans les requêtes SELECT.
* `is_restrictive` ([UInt8](/fr/reference/data-types/index)) — Indique si la politique de ligne restreint l’accès aux lignes. Valeur : • 0 — La politique de ligne est définie avec la clause `AS PERMISSIVE`, • 1 — La politique de ligne est définie avec la clause `AS RESTRICTIVE`.
* `apply_to_all` ([UInt8](/fr/reference/data-types/index)) — Indique que la politique de ligne est définie pour tous les rôles et/ou utilisateurs.
* `apply_to_list` ([Array(String)](/fr/reference/data-types/index)) — Liste des rôles et/ou utilisateurs auxquels la politique de ligne s’applique.
* `apply_to_except` ([Array(String)](/fr/reference/data-types/index)) — La politique de ligne s’applique à tous les rôles et/ou utilisateurs, à l’exception de ceux qui sont listés.

{/*AUTOGENERATED_END*/}
