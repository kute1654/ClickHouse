<div id="description">
  ## Description
</div>

Contient des informations et l’état des tables répliquées présentes sur le serveur local.
Cette table peut être utilisée à des fins de supervision. Elle contient une ligne pour chaque table Replicated*.

<div id="columns">
  ## Colonnes
</div>

{/*AUTOGENERATED_START*/}

* `database` ([String](/fr/reference/data-types/index)) — Nom de la base de données.
* `table` ([String](/fr/reference/data-types/index)) — Nom de la table.
* `uuid` ([UUID](/fr/reference/data-types/index)) — UUID de la table.
* `engine` ([String](/fr/reference/data-types/index)) — Nom du moteur de table.
* `is_leader` ([UInt8](/fr/reference/data-types/index)) — Indique si la réplique est leader. Plusieurs répliques peuvent être leaders en même temps. Il est possible d’empêcher une réplique de devenir leader à l’aide du paramètre merge&#95;tree replicated&#95;can&#95;become&#95;leader. Les leaders sont responsables de la planification des fusions en arrière-plan. Notez que des écritures peuvent être effectuées sur n’importe quelle réplique disponible ayant une session dans ZK, qu’elle soit leader ou non.
* `can_become_leader` ([UInt8](/fr/reference/data-types/index)) — Indique si la réplique peut devenir leader.
* `is_readonly` ([UInt8](/fr/reference/data-types/index)) — Indique si la réplique est en mode lecture seule. Ce mode est activé si la configuration ne contient pas de sections avec ClickHouse Keeper, si une erreur inconnue s’est produite lors de la réinitialisation des sessions dans ClickHouse Keeper, ainsi que pendant la réinitialisation de session dans ClickHouse Keeper.
* `readonly_start_time` ([Nullable(DateTime)](/fr/reference/data-types/index)) — Horodatage du passage de la réplique en mode lecture seule. NULL si la réplique n’est pas en mode lecture seule.
* `is_session_expired` ([UInt8](/fr/reference/data-types/index)) — Indique si la session avec ClickHouse Keeper a expiré. C’est essentiellement la même chose que `is_readonly`.
* `future_parts` ([UInt32](/fr/reference/data-types/index)) — Le nombre de parties de données qui apparaîtront à la suite d’INSERT ou de fusions qui n’ont pas encore été effectués.
* `parts_to_check` ([UInt32](/fr/reference/data-types/index)) — Le nombre de parties de données dans la file d’attente de vérification. Une partie est placée dans la file d’attente de vérification s’il y a des soupçons qu’elle soit endommagée.
* `zookeeper_name` ([String](/fr/reference/data-types/index)) — Le nom du cluster [Zoo]Keeper (éventuellement auxiliaire) où sont stockées les métadonnées de la table
* `zookeeper_path` ([String](/fr/reference/data-types/index)) — Chemin vers les données de la table dans ClickHouse Keeper.
* `replica_name` ([String](/fr/reference/data-types/index)) — Nom de la réplique dans ClickHouse Keeper. Les différentes répliques d’une même table portent des noms différents.
* `replica_path` ([String](/fr/reference/data-types/index)) — Chemin vers les données de la réplique dans ClickHouse Keeper. Équivaut à la concaténation de &#39;zookeeper&#95;path/replicas/replica&#95;path&#39;.
* `columns_version` ([Int32](/fr/reference/data-types/index)) — Numéro de version de la structure de la table. Indique combien de fois ALTER a été exécuté. Si les répliques ont des versions différentes, cela signifie que certaines répliques n’ont pas encore exécuté tous les ALTER.
* `queue_size` ([UInt32](/fr/reference/data-types/index)) — Taille de la file d’attente des opérations en attente d’exécution. Ces opérations incluent l’insertion de blocs de données, les fusions et certaines autres actions. Cela correspond généralement à future&#95;parts.
* `inserts_in_queue` ([UInt32](/fr/reference/data-types/index)) — Nombre d’insertions de blocs de données à effectuer. Les insertions sont généralement répliquées assez rapidement. Si ce nombre est élevé, cela signifie que quelque chose ne va pas.
* `merges_in_queue` ([UInt32](/fr/reference/data-types/index)) — Le nombre de fusions en attente d’exécution. Les fusions sont parfois longues, cette valeur peut donc rester supérieure à zéro pendant longtemps.
* `part_mutations_in_queue` ([UInt32](/fr/reference/data-types/index)) — Le nombre de mutations en attente d’exécution.
* `queue_oldest_time` ([DateTime](/fr/reference/data-types/index)) — Si `queue_size` est supérieur à 0, indique quand l’opération la plus ancienne a été ajoutée à la file d’attente.
* `inserts_oldest_time` ([DateTime](/fr/reference/data-types/index)) — Voir `queue_oldest_time`.
* `merges_oldest_time` ([DateTime](/fr/reference/data-types/index)) — Voir `queue_oldest_time`.
* `part_mutations_oldest_time` ([DateTime](/fr/reference/data-types/index)) — Voir `queue_oldest_time`.
* `oldest_part_to_get` ([String](/fr/reference/data-types/index)) — Le nom de la partie à récupérer depuis d’autres répliques, obtenu à partir de l’entrée GET&#95;PARTS la plus ancienne dans la file d’attente de réplication.
* `oldest_part_to_merge_to` ([String](/fr/reference/data-types/index)) — Le nom de la part résultante dans laquelle fusionner, obtenu à partir de l’entrée MERGE&#95;PARTS la plus ancienne de la file d’attente de réplication.
* `oldest_part_to_mutate_to` ([String](/fr/reference/data-types/index)) — Le nom de la part résultante vers laquelle effectuer la mutation, obtenu à partir de l’entrée MUTATE&#95;PARTS la plus ancienne de la file d’attente de réplication.
* `log_max_index` ([UInt64](/fr/reference/data-types/index)) — Le numéro d’entrée maximal dans le journal d’activité général. Cette colonne et les trois suivantes (`log_pointer`, `total_replicas`, `active_replicas`) n’ont une valeur non nulle que lorsqu’il existe une session active avec ClickHouse Keeper.
* `log_pointer` ([UInt64](/fr/reference/data-types/index)) — Le numéro d’entrée maximal du journal d’activité général que la réplique a copié dans sa file d’exécution, plus un. Si log&#95;pointer est nettement inférieur à log&#95;max&#95;index, cela indique un problème.
* `last_queue_update` ([DateTime](/fr/reference/data-types/index)) — Date de la dernière mise à jour de la file d’attente.
* `absolute_delay` ([UInt64](/fr/reference/data-types/index)) — Le retard actuel de la réplique, en secondes.
* `total_replicas` ([UInt32](/fr/reference/data-types/index)) — Le nombre total de répliques connues pour cette table.
* `active_replicas` ([UInt32](/fr/reference/data-types/index)) — Le nombre de répliques de cette table qui ont une session dans ClickHouse Keeper (c’est-à-dire le nombre de répliques fonctionnelles).
* `lost_part_count` ([UInt64](/fr/reference/data-types/index)) — Le nombre total de parts de données perdues dans la table par l’ensemble des répliques depuis la création de la table. La valeur est stockée de manière persistante dans ClickHouse Keeper et ne peut qu’augmenter.
* `last_queue_update_exception` ([String](/fr/reference/data-types/index)) — Indique que la file d’attente contient des entrées corrompues. C’est particulièrement important lorsque ClickHouse rompt la rétrocompatibilité entre versions et que les entrées de journal écrites par des versions plus récentes ne peuvent pas être analysées par les anciennes versions.
* `zookeeper_exception` ([String](/fr/reference/data-types/index)) — Le dernier message d’exception obtenu si l’erreur s’est produite lors de la récupération des informations depuis ClickHouse Keeper.
* `replica_is_active` ([Map(String, UInt8)](/fr/reference/data-types/index)) — Correspondance entre le nom de la réplique et son état d’activité.

{/*AUTOGENERATED_END*/}
