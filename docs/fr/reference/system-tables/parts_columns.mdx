<div id="description">
  ## Description
</div>

Contient des informations sur les parties de données et les colonnes des tables [MergeTree](/fr/reference/engines/table-engines/mergetree-family/mergetree).
Chaque ligne décrit une partie de données.

<div id="columns">
  ## Colonnes
</div>

{/*AUTOGENERATED_START*/}

* `partition` ([String](/fr/reference/data-types/index)) — Le nom de la partition.
* `name` ([String](/fr/reference/data-types/index)) — Nom de la partie de données.
* `uuid` ([UUID](/fr/reference/data-types/index)) — L’UUID de la partie.
* `part_type` ([String](/fr/reference/data-types/index)) — Le format de stockage de la partie de données. Valeurs possibles : Wide — chaque colonne est stockée dans un fichier distinct dans un système de fichiers, Compact — toutes les colonnes sont stockées dans un seul fichier dans un système de fichiers.
* `active` ([UInt8](/fr/reference/data-types/index)) — Indicateur signalant si la partie de données est active. Si une partie de données est active, elle est utilisée dans une table. Sinon, elle est supprimée. Les parties de données inactives restent présentes après une fusion.
* `marks` ([UInt64](/fr/reference/data-types/index)) — Le nombre de marks. Pour obtenir le nombre approximatif de lignes dans une partie de données, multipliez le nombre de marks par la granularité d’index (généralement 8192) (cette indication ne fonctionne pas avec la granularité adaptative).
* `rows` ([UInt64](/fr/reference/data-types/index)) — Le nombre de lignes.
* `bytes_on_disk` ([UInt64](/fr/reference/data-types/index)) — Taille totale de tous les fichiers de la partie de données, en octets.
* `data_compressed_bytes` ([UInt64](/fr/reference/data-types/index)) — Taille totale des données compressées dans la partie de données. Les fichiers auxiliaires (par exemple, les fichiers de marks) ne sont pas inclus.
* `data_uncompressed_bytes` ([UInt64](/fr/reference/data-types/index)) — Taille totale des données non compressées dans la partie de données. Les fichiers auxiliaires (par exemple, les fichiers de marks) ne sont pas inclus.
* `marks_bytes` ([UInt64](/fr/reference/data-types/index)) — La taille du fichier de marks.
* `modification_time` ([DateTime](/fr/reference/data-types/index)) — L’heure à laquelle le répertoire contenant la partie de données a été modifié. Cela correspond généralement à l’heure de création de la partie de données.
* `remove_time` ([DateTime](/fr/reference/data-types/index)) — L’heure à laquelle la partie de données est devenue inactive.
* `refcount` ([UInt32](/fr/reference/data-types/index)) — Le nombre d’endroits où la partie de données est utilisée. Une valeur supérieure à 2 indique que la partie de données est utilisée dans des requêtes ou des fusions.
* `min_date` ([Date](/fr/reference/data-types/index)) — La valeur minimale de la colonne Date si celle-ci est incluse dans la clé de partitionnement.
* `max_date` ([Date](/fr/reference/data-types/index)) — La valeur maximale de la colonne Date si celle-ci est incluse dans la clé de partitionnement.
* `min_time` ([DateTime](/fr/reference/data-types/index)) — La valeur minimale de la colonne DateTime si celle-ci est incluse dans la clé de partitionnement.
* `max_time` ([DateTime](/fr/reference/data-types/index)) — La valeur maximale de la colonne DateTime si celle-ci est incluse dans la clé de partitionnement.
* `partition_id` ([String](/fr/reference/data-types/index)) — ID de la partition.
* `min_block_number` ([Int64](/fr/reference/data-types/index)) — Le nombre minimal de parties de données qui composent la partie actuelle après fusion.
* `max_block_number` ([Int64](/fr/reference/data-types/index)) — Le nombre maximal de parties de données qui composent la partie actuelle après fusion.
* `level` ([UInt32](/fr/reference/data-types/index)) — Profondeur de l’arbre de fusion. Zéro signifie que la partie actuelle a été créée par insertion plutôt que par fusion d’autres parties.
* `data_version` ([UInt64](/fr/reference/data-types/index)) — Nombre utilisé pour déterminer quelles mutations doivent être appliquées à la partie de données (mutations avec une version supérieure à data&#95;version).
* `primary_key_bytes_in_memory` ([UInt64](/fr/reference/data-types/index)) — La quantité de mémoire (en octets) utilisée par les valeurs de clé primaire.
* `primary_key_bytes_in_memory_allocated` ([UInt64](/fr/reference/data-types/index)) — La quantité de mémoire (en octets) réservée pour les valeurs de clé primaire.
* `database` ([String](/fr/reference/data-types/index)) — Nom de la base de données.
* `table` ([String](/fr/reference/data-types/index)) — Nom de la table.
* `engine` ([String](/fr/reference/data-types/index)) — Nom du moteur de table sans paramètres.
* `disk_name` ([String](/fr/reference/data-types/index)) — Nom du disque qui stocke la part de données.
* `path` ([String](/fr/reference/data-types/index)) — Chemin absolu vers le dossier contenant les fichiers de la part de données.
* `column` ([String](/fr/reference/data-types/index)) — Nom de la colonne.
* `type` ([String](/fr/reference/data-types/index)) — Type de la colonne.
* `column_position` ([UInt64](/fr/reference/data-types/index)) — Position ordinale d’une colonne dans une table, à partir de 1.
* `default_kind` ([String](/fr/reference/data-types/index)) — Type d’expression (DEFAULT, MATERIALIZED, ALIAS) pour la valeur par défaut, ou une chaîne vide s’il n’est pas défini.
* `default_expression` ([String](/fr/reference/data-types/index)) — Expression de la valeur par défaut, ou une chaîne vide si elle n’est pas définie.
* `column_bytes_on_disk` ([UInt64](/fr/reference/data-types/index)) — Taille totale de la colonne en octets.
* `column_data_compressed_bytes` ([UInt64](/fr/reference/data-types/index)) — Taille totale des données compressées de la colonne, en octets.
* `column_data_uncompressed_bytes` ([UInt64](/fr/reference/data-types/index)) — Taille totale des données décompressées de la colonne, en octets.
* `column_marks_bytes` ([UInt64](/fr/reference/data-types/index)) — Taille des marques de la colonne, en octets.
* `column_modification_time` ([Nullable(DateTime)](/fr/reference/data-types/index)) — Date et heure de la dernière modification de la colonne.
* `column_ttl_min` ([Nullable(DateTime)](/fr/reference/data-types/index)) — Valeur minimale de l’expression TTL calculée pour la colonne.
* `column_ttl_max` ([Nullable(DateTime)](/fr/reference/data-types/index)) — Valeur maximale de l’expression TTL calculée pour la colonne.
* `statistics` ([Array(String)](/fr/reference/data-types/index)) — Statistiques de la colonne.
* `estimates.min` ([Nullable(String)](/fr/reference/data-types/index)) — Valeur minimale estimée de la colonne.
* `estimates.max` ([Nullable(String)](/fr/reference/data-types/index)) — Valeur maximale estimée de la colonne.
* `estimates.cardinality` ([Nullable(UInt64)](/fr/reference/data-types/index)) — Cardinalité estimée de la colonne.
* `estimates.null_count` ([Nullable(UInt64)](/fr/reference/data-types/index)) — Nombre estimé de valeurs NULL dans la colonne.
* `serialization_kind` ([String](/fr/reference/data-types/index)) — Type de sérialisation d’une colonne
* `substreams` ([Array(String)](/fr/reference/data-types/index)) — Noms des sous-flux dans lesquels la colonne est sérialisée
* `filenames` ([Array(String)](/fr/reference/data-types/index)) — Noms des fichiers correspondant à chaque sous-flux d’une colonne
* `subcolumns.names` ([Array(String)](/fr/reference/data-types/index)) — Noms des sous-colonnes d’une colonne
* `subcolumns.types` ([Array(String)](/fr/reference/data-types/index)) — Types des sous-colonnes d’une colonne
* `subcolumns.serializations` ([Array(String)](/fr/reference/data-types/index)) — Modes de sérialisation des sous-colonnes d’une colonne
* `subcolumns.bytes_on_disk` ([Array(UInt64)](/fr/reference/data-types/index)) — Taille sur disque, en octets, de chaque sous-colonne
* `subcolumns.data_compressed_bytes` ([Array(UInt64)](/fr/reference/data-types/index)) — Taille des données compressées de chaque sous-colonne, en octets
* `subcolumns.data_uncompressed_bytes` ([Array(UInt64)](/fr/reference/data-types/index)) — Taille des données non compressées de chaque sous-colonne, en octets
* `subcolumns.marks_bytes` ([Array(UInt64)](/fr/reference/data-types/index)) — Taille des marques de chaque sous-colonne d’une colonne, en octets

{/*AUTOGENERATED_END*/}
