<div id="description">
  ## Description
</div>

Contient des informations sur les tâches dans les pools de planification en arrière-plan. Les pools de planification en arrière-plan sont utilisés pour exécuter des tâches périodiques telles que les envois distribués, les vidages des tampons et les opérations du broker de messages.

<div id="columns">
  ## Colonnes
</div>

{/*AUTOGENERATED_START*/}

* `pool` ([String](/fr/reference/data-types/index)) — Nom du pool. Valeurs possibles : `schedule` — pool de planification à usage général, `buffer_flush` — pool pour le flush des données de la table Buffer, `distributed` — pool pour les opérations sur les tables distribuées, `message_broker` — pool pour les opérations du broker de messages.
* `database` ([String](/fr/reference/data-types/index)) — Nom de la base de données.
* `table` ([String](/fr/reference/data-types/index)) — Nom de la table.
* `table_uuid` ([UUID](/fr/reference/data-types/index)) — UUID de la table.
* `query_id` ([String](/fr/reference/data-types/index)) — ID de requête (si en cours d’exécution). Remarque : il ne s’agit pas d’une véritable requête, mais d’un ID généré aléatoirement pour faire correspondre les logs dans `system.text_log`.
* `elapsed_ms` ([UInt64](/fr/reference/data-types/index)) — Temps d’exécution de la tâche (si en cours d’exécution).
* `log_name` ([String](/fr/reference/data-types/index)) — Nom du log de la tâche.
* `deactivated` ([UInt8](/fr/reference/data-types/index)) — Indique si la tâche est désactivée (toujours faux, car les tâches désactivées sont retirées du pool).
* `scheduled` ([UInt8](/fr/reference/data-types/index)) — Indique si la tâche est planifiée pour être exécutée.
* `delayed` ([UInt8](/fr/reference/data-types/index)) — Indique si la tâche est planifiée avec un délai.
* `executing` ([UInt8](/fr/reference/data-types/index)) — Indique si la tâche est actuellement en cours d’exécution.

{/*AUTOGENERATED_END*/}
