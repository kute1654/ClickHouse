<div id="description">
  ## Description
</div>

Contient des métriques pouvant être calculées instantanément ou ayant une valeur courante. Par exemple, le nombre de requêtes traitées simultanément ou le retard actuel de la réplique. Cette table est toujours à jour.

<div id="columns">
  ## Colonnes
</div>

{/*AUTOGENERATED_START*/}

* `metric` ([String](/fr/reference/data-types/index)) — Nom de la métrique.
* `value` ([Int64](/fr/reference/data-types/index)) — Valeur de la métrique.
* `description` ([String](/fr/reference/data-types/index)) — Description de la métrique.

{/*AUTOGENERATED_END*/}

Vous trouverez toutes les métriques prises en charge dans le fichier source [src/Common/CurrentMetrics.cpp](https://github.com/ClickHouse/ClickHouse/blob/master/src/Common/CurrentMetrics.cpp).

<div id="example">
  ## Exemple
</div>

```sql
SELECT * FROM system.metrics LIMIT 10
```

```text
┌─metric───────────────────────────────┬─value─┬─description────────────────────────────────────────────────────────────┐
│ Query                                │     1 │ Number of executing queries                                            │
│ Merge                                │     0 │ Number of executing background merges                                  │
│ PartMutation                         │     0 │ Number of mutations (ALTER DELETE/UPDATE)                              │
│ ReplicatedFetch                      │     0 │ Number of data parts being fetched from replicas                       │
│ ReplicatedSend                       │     0 │ Number of data parts being sent to replicas                            │
│ ReplicatedChecks                     │     0 │ Number of data parts checking for consistency                          │
│ BackgroundMergesAndMutationsPoolTask │     0 │ Number of active merges and mutations in an associated background pool │
│ BackgroundFetchesPoolTask            │     0 │ Number of active fetches in an associated background pool              │
│ BackgroundCommonPoolTask             │     0 │ Number of active tasks in an associated background pool                │
│ BackgroundMovePoolTask               │     0 │ Number of active tasks in BackgroundProcessingPool for moves           │
└──────────────────────────────────────┴───────┴────────────────────────────────────────────────────────────────────────┘
```

<div id="compiledexpressioncachebytes">
  ### CompiledExpressionCacheBytes
</div>

Capacité réservée en blocs de pages (arrondie à des pages entières avec un facteur de surprovisionnement de 2x) par `JITModuleMemoryManager` pour les sections exécutables et de données des fonctions compilées par JIT mises en cache. PAS le nombre réel d’octets de code machine utilisés (il est plus faible). L’allocation se fait via `posix_memalign`, redirigé vers jemalloc ; cette capacité est donc comptabilisée dans l’arène JIT dédiée et constitue un sous-ensemble de `jemalloc.jit_arena.active_bytes`.
