<div id="description">
  ## Description
</div>

Contient des informations sur les vues dépendantes exécutées lors de l’exécution d’une requête, par exemple le type de vue ou le temps d’exécution.

Pour activer la journalisation :

1. Configurez les paramètres dans la section [query&#95;views&#95;log](/fr/reference/settings/server-settings/settings#query_views_log).
2. Définissez [log&#95;query&#95;views](/fr/reference/settings/session-settings#log_query_views) sur 1.

L’intervalle de vidage des données est défini dans le paramètre `flush_interval_milliseconds` de la section des paramètres du serveur [query&#95;views&#95;log](/fr/reference/settings/server-settings/settings#query_views_log). Pour forcer le vidage, utilisez la requête [SYSTEM FLUSH LOGS](/fr/reference/statements/system#flush-logs).

ClickHouse ne supprime pas automatiquement les données de la table. Voir [Introduction](/fr/reference/system-tables/overview#system-tables-introduction) pour plus de détails.

Vous pouvez utiliser le paramètre [log&#95;queries&#95;probability](/fr/reference/settings/session-settings#log_queries_probability)) pour réduire le nombre de requêtes enregistrées dans la table `query_views_log`.

<div id="columns">
  ## Colonnes
</div>

{/*AUTOGENERATED_START*/}

* `hostname` ([LowCardinality(String)](/fr/reference/data-types/lowcardinality)) — Nom d’hôte du serveur qui exécute la requête.
* `event_date` ([Date](/fr/reference/data-types/date)) — Date à laquelle le dernier événement de la vue s’est produit.
* `event_time` ([DateTime](/fr/reference/data-types/datetime)) — Date et heure auxquelles la vue a terminé son exécution.
* `event_time_microseconds` ([DateTime64(6)](/fr/reference/data-types/datetime64)) — Date et heure auxquelles la vue a terminé son exécution avec une précision à la microseconde.
* `view_duration_ms` ([UInt64](/fr/reference/data-types/int-uint)) — Durée d’exécution de la vue (somme de ses étapes), en millisecondes.
* `initial_query_id` ([String](/fr/reference/data-types/string)) — ID de la requête initiale (pour l’exécution distribuée des requêtes).
* `view_name` ([String](/fr/reference/data-types/string)) — Nom de la vue.
* `view_uuid` ([UUID](/fr/reference/data-types/uuid)) — UUID de la vue.
* `view_type` ([Enum8(&#39;Default&#39; = 1, &#39;Materialized&#39; = 2, &#39;Live&#39; = 3, &#39;Window&#39; = 4)](/fr/reference/data-types/enum)) — Type de la vue. Valeurs : &#39;Default&#39; = 1 — Vues par défaut. Ne devrait pas apparaître dans ce journal, &#39;Materialized&#39; = 2 — Vues matérialisées, &#39;Live&#39; = 3 — Vues Live.
* `view_query` ([String](/fr/reference/data-types/string)) — Requête exécutée par la vue.
* `view_target` ([String](/fr/reference/data-types/string)) — Nom de la table cible de la vue.
* `read_rows` ([UInt64](/fr/reference/data-types/int-uint)) — Nombre de lignes lues.
* `read_bytes` ([UInt64](/fr/reference/data-types/int-uint)) — Nombre d’octets lus.
* `written_rows` ([UInt64](/fr/reference/data-types/int-uint)) — Nombre de lignes écrites.
* `written_bytes` ([UInt64](/fr/reference/data-types/int-uint)) — Nombre d’octets écrits.
* `peak_memory_usage` ([Int64](/fr/reference/data-types/int-uint)) — Différence maximale entre la quantité de mémoire allouée et celle libérée dans le contexte de cette vue.
* `ProfileEvents` ([Map(LowCardinality(String), UInt64)](/fr/reference/data-types/map)) — `ProfileEvents` mesure différentes métriques. Leur description figure dans la table system.events.
* `status` ([Enum8(&#39;QueryStart&#39; = 1, &#39;QueryFinish&#39; = 2, &#39;ExceptionBeforeStart&#39; = 3, &#39;ExceptionWhileProcessing&#39; = 4)](/fr/reference/data-types/enum)) — Statut de la vue. Valeurs : &#39;QueryStart&#39; = 1 — Démarrage réussi de l’exécution de la vue. Ne devrait pas apparaître, &#39;QueryFinish&#39; = 2 — Fin réussie de l’exécution de la vue, &#39;ExceptionBeforeStart&#39; = 3 — Exception avant le début de l’exécution de la vue, &#39;ExceptionWhileProcessing&#39; = 4 — Exception pendant l’exécution de la vue.
* `exception_code` ([Int32](/fr/reference/data-types/int-uint)) — Code d’une exception.
* `exception` ([String](/fr/reference/data-types/string)) — Message d’exception.
* `stack_trace` ([String](/fr/reference/data-types/string)) — Stack trace. Chaîne vide si la requête s’est terminée avec succès.

**Alias :**

* `ProfileEvents.Names` — alias de `mapKeys(ProfileEvents)`.
* `ProfileEvents.Values` — alias de `mapValues(ProfileEvents)`.

{/*AUTOGENERATED_END*/}
