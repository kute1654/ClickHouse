<div id="description">
  ## Description
</div>

Contient des informations sur toutes les entrées du cache du système de fichiers associées aux objets distants.

<div id="columns">
  ## Colonnes
</div>

{/*AUTOGENERATED_START*/}

* `cache_name` ([String](/fr/reference/data-types/index)) — Nom de l’objet cache. Peut être utilisé dans les commandes `SYSTEM DESCRIBE FILESYSTEM CACHE <name>` et `SYSTEM DROP FILESYSTEM CACHE <name>`
* `cache_base_path` ([String](/fr/reference/data-types/index)) — Chemin vers le répertoire racine où sont stockés tous les fichiers du cache (d’un cache identifié par `cache_name`).
* `cache_path` ([String](/fr/reference/data-types/index)) — Chemin vers un fichier de cache particulier, correspondant à un segment de fichier dans un fichier source
* `key` ([String](/fr/reference/data-types/index)) — Clé de cache du segment de fichier
* `file_segment_range_begin` ([UInt64](/fr/reference/data-types/index)) — Décalage correspondant au début de la plage du segment de fichier
* `file_segment_range_end` ([UInt64](/fr/reference/data-types/index)) — Décalage correspondant à la fin (incluse) de la plage du segment de fichier
* `size` ([UInt64](/fr/reference/data-types/index)) — Taille du segment de fichier
* `state` ([String](/fr/reference/data-types/index)) — État du segment de fichier (DOWNLOADED, DOWNLOADING, PARTIALLY&#95;DOWNLOADED, ...)
* `finished_download_time` ([DateTime](/fr/reference/data-types/index)) — Heure à laquelle le téléchargement du segment de fichier s’est terminé.
* `cache_hits` ([UInt64](/fr/reference/data-types/index)) — Nombre d’accès au cache pour le segment de fichier correspondant
* `references` ([UInt64](/fr/reference/data-types/index)) — Nombre de références au segment de fichier correspondant. La valeur 1 signifie que personne ne l’utilise actuellement (la seule référence existante se trouve dans le stockage du cache lui-même)
* `downloaded_size` ([UInt64](/fr/reference/data-types/index)) — Taille téléchargée du segment de fichier
* `kind` ([String](/fr/reference/data-types/index)) — Nature du segment de fichier (utilisée pour distinguer les segments de fichier ajoutés dans le cadre de &#39;Temporary data in cache&#39;)
* `unbound` ([UInt8](/fr/reference/data-types/index)) — Indicateur interne d’implémentation
* `user_id` ([String](/fr/reference/data-types/index)) — ID de l’utilisateur ayant créé le segment de fichier
* `segment_type` ([String](/fr/reference/data-types/index)) — Type du segment. Utilisé pour distinguer les fichiers de données (`.json`, `.txt`, etc.) des fichiers de données (`.bin`, fichiers de marques).
* `file_size` ([Nullable(UInt64)](/fr/reference/data-types/index)) — Taille du fichier auquel appartient le segment de fichier actuel

{/*AUTOGENERATED_END*/}
