<div id="description">
  ## Description
</div>

Cette table contient des informations sur les paramètres de la requête envoyée au serveur ZooKeeper et sur la réponse qu’il renvoie.

Pour les requêtes, seules les colonnes contenant les paramètres de la requête sont renseignées, tandis que les autres contiennent leurs valeurs par défaut (`0` ou `NULL`). Lorsque la réponse arrive, ses données sont ajoutées aux autres colonnes.

<div id="columns">
  ## Colonnes
</div>

{/*AUTOGENERATED_START*/}

* `hostname` ([LowCardinality(String)](/fr/reference/data-types/lowcardinality)) — Nom d’hôte du serveur qui exécute la requête.
* `type` ([Enum8(&#39;Request&#39; = 1, &#39;Response&#39; = 2, &#39;Finalize&#39; = 3)](/fr/reference/data-types/enum)) — Type d’événement dans le client ZooKeeper. Peut prendre l’une des valeurs suivantes : Request — la requête a été envoyée, Response — la réponse a été reçue, Finalize — la connexion est perdue, aucune réponse n’a été reçue.
* `event_date` ([Date](/fr/reference/data-types/date)) — La date à laquelle l’événement s’est produit.
* `event_time` ([DateTime64(6)](/fr/reference/data-types/datetime64)) — La date et l’heure auxquelles l’événement s’est produit.
* `thread_id` ([UInt64](/fr/reference/data-types/int-uint)) — L’ID du thread qui a exécuté cette requête.
* `query_id` ([String](/fr/reference/data-types/string)) — L’ID d’une query dans le cadre de laquelle cette requête a été exécutée.
* `address` ([IPv6](/fr/reference/data-types/ipv6)) — Adresse IP du serveur ZooKeeper utilisé pour effectuer la requête.
* `port` ([UInt16](/fr/reference/data-types/int-uint)) — Le port du serveur ZooKeeper utilisé pour effectuer la requête.
* `session_id` ([Int64](/fr/reference/data-types/int-uint)) — L’ID de session que le serveur ZooKeeper définit pour chaque connexion.
* `duration_microseconds` ([UInt64](/fr/reference/data-types/int-uint)) — Le temps mis par ZooKeeper pour exécuter la requête.
* `xid` ([Int64](/fr/reference/data-types/int-uint)) — L’ID de la requête au sein de la session. Il s’agit généralement d’un numéro de requête séquentiel. Il est identique pour la ligne de requête et la ligne de réponse/finalisation correspondante.
* `has_watch` ([UInt8](/fr/reference/data-types/int-uint)) — Indique si un watch a été défini pour la requête.
* `op_num` ([Enum16(&#39;Close&#39; = -11, &#39;Error&#39; = -1, &#39;Watch&#39; = 0, &#39;Create&#39; = 1, &#39;Remove&#39; = 2, &#39;Exists&#39; = 3, &#39;Get&#39; = 4, &#39;Set&#39; = 5, &#39;GetACL&#39; = 6, &#39;SetACL&#39; = 7, &#39;SimpleList&#39; = 8, &#39;Sync&#39; = 9, &#39;Heartbeat&#39; = 11, &#39;List&#39; = 12, &#39;Check&#39; = 13, &#39;Multi&#39; = 14, &#39;Create2&#39; = 15, &#39;Reconfig&#39; = 16, &#39;CheckWatch&#39; = 17, &#39;RemoveWatch&#39; = 18, &#39;MultiRead&#39; = 22, &#39;Auth&#39; = 100, &#39;SetWatch&#39; = 101, &#39;SetWatch2&#39; = 105, &#39;AddWatch&#39; = 106, &#39;FilteredList&#39; = 500, &#39;CheckNotExists&#39; = 501, &#39;CreateIfNotExists&#39; = 502, &#39;RemoveRecursive&#39; = 503, &#39;CheckStat&#39; = 504, &#39;TryRemove&#39; = 505, &#39;FilteredListWithStatsAndData&#39; = 506, &#39;ListRecursive&#39; = 507, &#39;SessionID&#39; = 997)](/fr/reference/data-types/enum)) — Le type de requête ou de réponse.
* `path` ([String](/fr/reference/data-types/string)) — Le chemin du nœud ZooKeeper spécifié dans la requête, ou une chaîne vide si la requête ne nécessite pas de chemin.
* `data` ([String](/fr/reference/data-types/string)) — Les données écrites dans le nœud ZooKeeper (pour les requêtes SET et CREATE — ce que la requête voulait écrire ; pour la réponse à la requête GET — ce qui a été lu) ou une chaîne vide.
* `is_ephemeral` ([UInt8](/fr/reference/data-types/int-uint)) — Indique si le nœud ZooKeeper est créé en mode éphémère.
* `is_sequential` ([UInt8](/fr/reference/data-types/int-uint)) — Indique si le nœud ZooKeeper est créé en mode séquentiel.
* `version` ([Nullable(Int32)](/fr/reference/data-types/nullable)) — La version du nœud ZooKeeper attendue par la requête au moment de son exécution. Pris en charge pour les requêtes CHECK, SET et REMOVE (la valeur est -1 si la requête ne vérifie pas la version, ou NULL pour les autres requêtes qui ne prennent pas en charge cette vérification).
* `requests_size` ([UInt32](/fr/reference/data-types/int-uint)) — Le nombre de requêtes incluses dans la requête multiple (il s’agit d’une requête spéciale composée de plusieurs requêtes ordinaires consécutives et exécutée de manière atomique). Toutes les requêtes incluses dans la requête multiple auront le même xid.
* `request_idx` ([UInt32](/fr/reference/data-types/int-uint)) — Le numéro de la requête incluse dans la requête multiple (pour la requête multiple — 0, puis dans l’ordre à partir de 1).
* `zxid` ([Int64](/fr/reference/data-types/int-uint)) — ID de transaction ZooKeeper. Numéro de série émis par le serveur ZooKeeper en réponse à une requête exécutée avec succès (0 si la requête n’a pas été exécutée/a renvoyé une erreur/le client ne sait pas si la requête a été exécutée).
* `error` ([Nullable(Enum8(&#39;ZNOWATCHER&#39; = -121, &#39;ZNOTREADONLY&#39; = -119, &#39;ZSESSIONMOVED&#39; = -118, &#39;ZNOTHING&#39; = -117, &#39;ZCLOSING&#39; = -116, &#39;ZAUTHFAILED&#39; = -115, &#39;ZINVALIDACL&#39; = -114, &#39;ZINVALIDCALLBACK&#39; = -113, &#39;ZSESSIONEXPIRED&#39; = -112, &#39;ZNOTEMPTY&#39; = -111, &#39;ZNODEEXISTS&#39; = -110, &#39;ZNOCHILDRENFOREPHEMERALS&#39; = -108, &#39;ZBADVERSION&#39; = -103, &#39;ZNOAUTH&#39; = -102, &#39;ZNONODE&#39; = -101, &#39;ZAPIERROR&#39; = -100, &#39;ZOUTOFMEMORY&#39; = -10, &#39;ZINVALIDSTATE&#39; = -9, &#39;ZBADARGUMENTS&#39; = -8, &#39;ZOPERATIONTIMEOUT&#39; = -7, &#39;ZUNIMPLEMENTED&#39; = -6, &#39;ZMARSHALLINGERROR&#39; = -5, &#39;ZCONNECTIONLOSS&#39; = -4, &#39;ZDATAINCONSISTENCY&#39; = -3, &#39;ZRUNTIMEINCONSISTENCY&#39; = -2, &#39;ZSYSTEMERROR&#39; = -1, &#39;ZOK&#39; = 0))](/fr/reference/data-types/nullable)) — Code d’erreur. Peut prendre de nombreuses valeurs, dont notamment : ZOK — la requête a été exécutée avec succès, ZCONNECTIONLOSS — la connexion a été perdue, ZOPERATIONTIMEOUT — le délai d’exécution de la requête a expiré, ZSESSIONEXPIRED — la session a expiré, NULL — la requête est terminée.
* `watch_type` ([Nullable(Enum8(&#39;NOTWATCHING&#39; = -2, &#39;SESSION&#39; = -1, &#39;CREATED&#39; = 1, &#39;DELETED&#39; = 2, &#39;CHANGED&#39; = 3, &#39;CHILD&#39; = 4))](/fr/reference/data-types/nullable)) — Le type de l’événement watch (pour les réponses avec op&#95;num = Watch) ; pour les autres réponses : NULL.
* `watch_state` ([Nullable(Enum16(&#39;AUTH&#95;FAILED&#39; = -113, &#39;EXPIRED&#95;SESSION&#39; = -112, &#39;CONNECTING&#39; = 1, &#39;ASSOCIATING&#39; = 2, &#39;CONNECTED&#39; = 3, &#39;READONLY&#39; = 5, &#39;NOTCONNECTED&#39; = 999))](/fr/reference/data-types/nullable)) — L’état de l’événement watch (pour les réponses avec op&#95;num = Watch) ; pour les autres réponses : NULL.
* `path_created` ([String](/fr/reference/data-types/string)) — Le chemin du nœud ZooKeeper créé (pour les réponses à la requête CREATE) ; il peut différer du chemin si le nœud est créé en mode sequential.
* `stat_czxid` ([Int64](/fr/reference/data-types/int-uint)) — Le zxid de la modification à l’origine de la création de ce nœud ZooKeeper.
* `stat_mzxid` ([Int64](/fr/reference/data-types/int-uint)) — Le zxid de la modification qui a modifié ce nœud ZooKeeper pour la dernière fois.
* `stat_pzxid` ([Int64](/fr/reference/data-types/int-uint)) — L’ID de transaction de la modification qui a modifié les nœuds enfants de ce nœud ZooKeeper pour la dernière fois.
* `stat_version` ([Int32](/fr/reference/data-types/int-uint)) — Le nombre de modifications apportées aux données de ce nœud ZooKeeper.
* `stat_cversion` ([Int32](/fr/reference/data-types/int-uint)) — Le nombre de modifications apportées aux nœuds enfants de ce nœud ZooKeeper.
* `stat_dataLength` ([Int32](/fr/reference/data-types/int-uint)) — La longueur du champ de données de ce nœud ZooKeeper.
* `stat_numChildren` ([Int32](/fr/reference/data-types/int-uint)) — Le nombre de nœuds enfants de ce nœud ZooKeeper.
* `children` ([Array(String)](/fr/reference/data-types/array)) — La liste des nœuds enfants ZooKeeper (pour les réponses à la requête LIST).

{/*AUTOGENERATED_END*/}
