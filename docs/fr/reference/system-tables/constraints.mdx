<div id="description">
  ## Description
</div>

Contient des informations sur les contraintes existantes de toutes les tables.

Les contraintes définies sur les [tables temporaires](/fr/reference/statements/create/table#temporary-tables) ne sont visibles dans `system.constraints` que dans la session où elles ont été créées. Elles s’affichent avec un champ `database` vide.

<div id="columns">
  ## Colonnes
</div>

{/*AUTOGENERATED_START*/}

* `database` ([String](/fr/reference/data-types/index)) — Nom de la base de données.
* `table` ([String](/fr/reference/data-types/index)) — Nom de la table.
* `name` ([String](/fr/reference/data-types/index)) — Nom de la contrainte.
* `type` ([Enum8(&#39;CHECK&#39; = 0, &#39;ASSUME&#39; = 1)](/fr/reference/data-types/index)) — Type de contrainte.
* `expression` ([String](/fr/reference/data-types/index)) — Expression de la contrainte.

{/*AUTOGENERATED_END*/}

<div id="example">
  ## Exemple
</div>

```sql
SELECT * FROM system.constraints LIMIT 2 FORMAT Vertical;
```

```text
Row 1:
──────
database:    default
table:       hits
name:        check_hits
type:        CHECK
expression:  CounterID > 0

Row 2:
──────
database:    default
table:       hits
name:        assume_positive
type:        ASSUME
expression:  WatchID > 0
```
