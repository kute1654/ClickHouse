<div id="purpose">
  ## Description
</div>

Contient des informations sur les parties détachées des tables [MergeTree](/fr/reference/engines/table-engines/mergetree-family/mergetree). La colonne `reason` précise pourquoi la partie a été détachée.

Pour les parties détachées par l&#39;utilisateur, la raison est vide. Ces parties peuvent être rattachées à l&#39;aide de la commande [ALTER TABLE ATTACH PARTITION|PART](/fr/reference/statements/alter/partition#attach-partitionpart).

Pour la description des autres colonnes, voir [system.parts](/fr/reference/system-tables/parts).

Si le nom de la partie n&#39;est pas valide, les valeurs de certaines colonnes peuvent être `NULL`. Ces parties peuvent être supprimées à l&#39;aide de [ALTER TABLE DROP DETACHED PART](/fr/reference/statements/alter/partition#drop-detached-partitionpart).

<div id="columns">
  ## Colonnes
</div>

{/*AUTOGENERATED_START*/}

* `database` ([String](/fr/reference/data-types/index)) — Le nom de la base de données à laquelle cette partie appartient.
* `table` ([String](/fr/reference/data-types/index)) — Le nom de la table à laquelle cette partie appartient.
* `partition_id` ([Nullable(String)](/fr/reference/data-types/index)) — L’identifiant de la partition à laquelle cette partie appartient.
* `name` ([String](/fr/reference/data-types/index)) — Le nom de la partie.
* `bytes_on_disk` ([UInt64](/fr/reference/data-types/index)) — La taille totale, en octets, de tous les fichiers de cette partie de données.
* `modification_time` ([DateTime](/fr/reference/data-types/index)) — La date et l’heure de modification du répertoire contenant cette partie de données. Cela correspond généralement au moment où le détachement a eu lieu.
* `disk` ([String](/fr/reference/data-types/index)) — Le nom du disque sur lequel cette partie de données est stockée.
* `path` ([String](/fr/reference/data-types/index)) — Le chemin, sur le disque, vers le fichier de cette partie de données.
* `reason` ([Nullable(String)](/fr/reference/data-types/index)) — L’explication de la raison pour laquelle cette partie a été détachée.
* `min_block_number` ([Nullable(Int64)](/fr/reference/data-types/index)) — Le numéro minimal des parties de données qui composent la partie actuelle après fusion.
* `max_block_number` ([Nullable(Int64)](/fr/reference/data-types/index)) — Le numéro maximal des parties de données qui composent la partie actuelle après fusion.
* `level` ([Nullable(UInt32)](/fr/reference/data-types/index)) — Profondeur de l’arbre de fusion. Zéro signifie que la partie actuelle a été créée par insert plutôt que par fusion d’autres parties.

{/*AUTOGENERATED_END*/}
