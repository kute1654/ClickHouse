<div id="description">
  ## Description
</div>

Contient des informations sur les [quotas](/fr/reference/system-tables/quotas).

<div id="columns">
  ## Colonnes
</div>

{/*AUTOGENERATED_START*/}

* `name` ([String](/fr/reference/data-types/index)) — Nom du quota.
* `id` ([UUID](/fr/reference/data-types/index)) — ID du quota.
* `storage` ([String](/fr/reference/data-types/index)) — Stockage des quotas. Valeur possible : &#39;users.xml&#39; si un quota est configuré dans le fichier users.xml, &#39;disk&#39; si un quota est configuré par une requête SQL.
* `keys` ([Array(Enum8(&#39;user&#95;name&#39; = 1, &#39;ip&#95;address&#39; = 2, &#39;forwarded&#95;ip&#95;address&#39; = 3, &#39;client&#95;key&#39; = 4, &#39;normalized&#95;query&#95;hash&#39; = 7))](/fr/reference/data-types/index)) — La clé indique comment le quota doit être partagé. Si deux connexions utilisent le même quota et la même clé, elles partagent les mêmes quantités de ressources. Valeurs : [] — Tous les utilisateurs partagent le même quota, [&#39;user&#95;name&#39;] — Les connexions avec le même nom d’utilisateur partagent le même quota, [&#39;ip&#95;address&#39;] — Les connexions provenant de la même adresse IP partagent le même quota. [&#39;client&#95;key&#39;] — Les connexions avec la même clé partagent le même quota. Une clé doit être explicitement fournie par un client. Lorsque vous utilisez clickhouse-client, transmettez une valeur de clé via le paramètre --quota&#95;key, ou utilisez le paramètre quota&#95;key dans le fichier de configuration du client. Lorsque vous utilisez l’interface HTTP, utilisez l’en-tête X-ClickHouse-Quota, [&#39;user&#95;name&#39;, &#39;client&#95;key&#39;] — Les connexions avec le même client&#95;key partagent le même quota. Si aucune clé n’est fournie par un client, le quota est suivi pour `user_name`, [&#39;client&#95;key&#39;, &#39;ip&#95;address&#39;] — Les connexions avec le même client&#95;key partagent le même quota. Si aucune clé n’est fournie par un client, le quota est suivi pour ip&#95;address.
* `durations` ([Array(UInt32)](/fr/reference/data-types/index)) — Durées des intervalles de temps, en secondes.
* `apply_to_all` ([UInt8](/fr/reference/data-types/index)) — Valeur logique. Elle indique à quels utilisateurs le quota s’applique. Valeurs : 0 — Le quota s’applique aux utilisateurs spécifiés dans apply&#95;to&#95;list. 1 — Le quota s’applique à tous les utilisateurs sauf à ceux répertoriés dans apply&#95;to&#95;except.
* `apply_to_list` ([Array(String)](/fr/reference/data-types/index)) — Liste des noms d’utilisateurs/rôles auxquels le quota doit s’appliquer.
* `apply_to_except` ([Array(String)](/fr/reference/data-types/index)) — Liste des noms d’utilisateurs/rôles auxquels le quota ne doit pas s’appliquer.
* `ipv4_prefix_bits` ([Nullable(UInt8)](/fr/reference/data-types/index)) — Nombre de bits de préfixe pour le masquage des adresses IPv4. Les connexions provenant du même sous-réseau partagent le même quota. Utilisé uniquement lorsque la clé est `ip_address` ou `forwarded_ip_address`.
* `ipv6_prefix_bits` ([Nullable(UInt8)](/fr/reference/data-types/index)) — Nombre de bits de préfixe pour le masquage des adresses IPv6. Les connexions provenant du même sous-réseau partagent le même quota. Utilisé uniquement lorsque la clé est `ip_address` ou `forwarded_ip_address`.
