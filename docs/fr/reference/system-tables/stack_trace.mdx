<div id="description">
  ## Description
</div>

Contient les traces de pile de tous les threads du serveur. Permet aux développeurs d&#39;inspecter l&#39;état du serveur.

Pour analyser les stack frames, utilisez `addressToLine`, `addressToLineWithInlines`, `addressToSymbol` et `demangle` [fonctions d&#39;introspection](/fr/reference/functions/regular-functions/introspection).

<div id="columns">
  ## Colonnes
</div>

{/*AUTOGENERATED_START*/}

* `thread_name` ([String](/fr/reference/data-types/index)) — Le nom du thread.
* `thread_id` ([UInt64](/fr/reference/data-types/index)) — L’identifiant du thread
* `query_id` ([String](/fr/reference/data-types/index)) — L’identifiant de la requête à laquelle ce thread appartient.
* `trace` ([Array(UInt64)](/fr/reference/data-types/index)) — La pile d’appels de ce thread. Il s’agit essentiellement d’un simple tableau d’adresses.
* `untracked_memory` ([Int64](/fr/reference/data-types/index)) — Compteur non atomique, par thread, des allocations mémoire qui n’ont pas encore été propagées au MemoryTracker parent. Peut être négatif si davantage de mémoire a été libérée qu’allouée depuis le dernier flush.

{/*AUTOGENERATED_END*/}

<div id="example">
  ## Exemple
</div>

Activation des fonctions d’introspection :

```sql
SET allow_introspection_functions = 1;
```

Récupération des symboles à partir des fichiers objets de ClickHouse :

```sql
WITH arrayMap(x -> demangle(addressToSymbol(x)), trace) AS all SELECT thread_name, thread_id, query_id, arrayStringConcat(all, '\n') AS res FROM system.stack_trace LIMIT 1;
```

Récupérer les noms de fichiers et les numéros de ligne dans le code source de ClickHouse :

```sql
WITH arrayMap(x -> addressToLine(x), trace) AS all, arrayFilter(x -> x LIKE '%/dbms/%', all) AS dbms SELECT thread_name, thread_id, query_id, arrayStringConcat(notEmpty(dbms) ? dbms : all, '\n') AS res FROM system.stack_trace LIMIT 1;
```
