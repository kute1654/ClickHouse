<div id="description">
  ## Description
</div>

Contient la liste des layouts de dictionnaire pris en charge par le serveur, ainsi que la documentation intégrée pour chacun d&#39;eux. Le layout de dictionnaire détermine comment celui-ci est stocké en mémoire (ou sur disque) et consulté ; il est spécifié dans la clause `LAYOUT` d&#39;une requête `CREATE DICTIONARY`.

<div id="columns">
  ## Colonnes
</div>

{/*AUTOGENERATED_START*/}

* `name` ([String](/fr/reference/data-types/index)) — Le nom du layout de dictionnaire, tel qu’il est spécifié dans la clause LAYOUT.
* `is_complex` ([UInt8](/fr/reference/data-types/index)) — Indique si le layout nécessite une clé complexe (une clé composée de plusieurs attributs ou d’un type non entier).
* `description` ([String](/fr/reference/data-types/index)) — Une description générale du rôle du layout de dictionnaire.
* `syntax` ([String](/fr/reference/data-types/index)) — Indique comment le layout est spécifié dans la clause LAYOUT d’une requête CREATE DICTIONARY.
* `examples` ([String](/fr/reference/data-types/index)) — Exemples d’utilisation.
* `introduced_in` ([String](/fr/reference/data-types/index)) — La version de ClickHouse dans laquelle le layout a été introduit pour la première fois, au format major.minor.
* `related` ([Array(String)](/fr/reference/data-types/index)) — Les noms des layouts de dictionnaire associés.

{/*AUTOGENERATED_END*/}

<div id="example">
  ## Exemple
</div>

```sql title="Query"
SELECT name, is_complex, syntax
FROM system.dictionary_layouts
WHERE name IN ('flat', 'hashed', 'complex_key_hashed')
ORDER BY name
```

```text title="Response"
┌─name───────────────┬─is_complex─┬─syntax───────────────────────────────────────────────────┐
│ complex_key_hashed │          1 │ LAYOUT(COMPLEX_KEY_HASHED())                             │
│ flat               │          0 │ LAYOUT(FLAT([INITIAL_ARRAY_SIZE n] [MAX_ARRAY_SIZE n]))  │
│ hashed             │          0 │ LAYOUT(HASHED())                                         │
└────────────────────┴────────────┴──────────────────────────────────────────────────────────┘
```
