<div id="description">
  ## Description
</div>

Contient la liste des types d’index de saut de données pris en charge par le serveur, ainsi que la documentation intégrée pour chacun d’eux. Un type d’index de saut de données est spécifié dans le `TYPE` d’une déclaration `INDEX` d’une requête `CREATE TABLE` et permet à ClickHouse d’ignorer les granules qui ne peuvent pas satisfaire la condition d’une requête.

Notez que cette table répertorie les *types* d’index disponibles, tandis que [`system.data_skipping_indices`](/fr/reference/system-tables/data_skipping_indices) répertorie les instances d’index définies sur des tables existantes.

<div id="columns">
  ## Colonnes
</div>

{/*AUTOGENERATED_START*/}

* `name` ([String](/fr/reference/data-types/index)) — Le nom du type d’index de saut de données, tel qu’il est spécifié dans le TYPE d’une déclaration INDEX.
* `description` ([String](/fr/reference/data-types/index)) — Une description générale du rôle du type d’index de saut de données.
* `syntax` ([String](/fr/reference/data-types/index)) — La manière dont l’index est déclaré dans une clause INDEX d’une requête CREATE TABLE.
* `examples` ([String](/fr/reference/data-types/index)) — Exemples d’utilisation.
* `introduced_in` ([String](/fr/reference/data-types/index)) — La version de ClickHouse dans laquelle le type d’index a été introduit pour la première fois, sous la forme major.minor.
* `related` ([Array(String)](/fr/reference/data-types/index)) — Les noms des types d’index de saut de données associés.

{/*AUTOGENERATED_END*/}

<div id="example">
  ## Exemple
</div>

```sql title="Query"
SELECT name, syntax
FROM system.data_skipping_index_types
WHERE name IN ('minmax', 'set')
ORDER BY name
```

```text title="Response"
┌─name───┬─syntax───────────────────────────────────────┐
│ minmax │ INDEX name expr TYPE minmax GRANULARITY n     │
│ set    │ INDEX name expr TYPE set(max_rows) GRANULARITY n │
└────────┴──────────────────────────────────────────────┘
```
