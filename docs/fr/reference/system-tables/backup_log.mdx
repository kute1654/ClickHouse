<div id="description">
  ## Description
</div>

Contient des entrées de journal avec des informations sur les opérations `BACKUP` et `RESTORE`.

<div id="columns">
  ## Colonnes
</div>

{/*AUTOGENERATED_START*/}

* `hostname` ([LowCardinality(String)](/fr/reference/data-types/lowcardinality)) — Nom d’hôte du serveur qui exécute la requête.
* `event_date` ([Date](/fr/reference/data-types/date)) — Date de l’entrée.
* `event_time` ([DateTime](/fr/reference/data-types/datetime)) — Heure de l’entrée.
* `event_time_microseconds` ([DateTime64(6)](/fr/reference/data-types/datetime64)) — Heure de l’entrée avec une précision à la microseconde.
* `id` ([String](/fr/reference/data-types/string)) — Identifiant de l’opération de sauvegarde ou de restauration.
* `name` ([String](/fr/reference/data-types/string)) — Nom du stockage de sauvegarde (contenu de la clause FROM ou TO).
* `base_backup_name` ([String](/fr/reference/data-types/string)) — Nom de la sauvegarde de base dans le cas d’une sauvegarde incrémentale.
* `query_id` ([String](/fr/reference/data-types/string)) — ID d’une requête associée à une opération de sauvegarde.
* `status` ([Enum8(&#39;CREATING&#95;BACKUP&#39; = 0, &#39;BACKUP&#95;CREATED&#39; = 1, &#39;BACKUP&#95;FAILED&#39; = 2, &#39;RESTORING&#39; = 3, &#39;RESTORED&#39; = 4, &#39;RESTORE&#95;FAILED&#39; = 5, &#39;BACKUP&#95;CANCELLED&#39; = 6, &#39;RESTORE&#95;CANCELLED&#39; = 7)](/fr/reference/data-types/enum)) — Statut de l’opération.
* `error` ([String](/fr/reference/data-types/string)) — Message d’erreur de l’opération en échec (chaîne vide pour les opérations réussies).
* `start_time` ([DateTime64(6)](/fr/reference/data-types/datetime64)) — Heure de début de l’opération.
* `end_time` ([DateTime64(6)](/fr/reference/data-types/datetime64)) — Heure de fin de l’opération.
* `num_files` ([UInt64](/fr/reference/data-types/int-uint)) — Nombre de fichiers stockés dans la sauvegarde.
* `total_size` ([UInt64](/fr/reference/data-types/int-uint)) — Taille totale des fichiers stockés dans la sauvegarde.
* `num_entries` ([UInt64](/fr/reference/data-types/int-uint)) — Nombre d’entrées dans la sauvegarde, c.-à-d. le nombre de fichiers dans le dossier si la sauvegarde est stockée sous forme de dossier, ou le nombre de fichiers dans l’archive si la sauvegarde est stockée sous forme d’archive. Ce n’est pas la même chose que `num_files` s’il s’agit d’une sauvegarde incrémentale ou si elle contient des fichiers vides ou des doublons. L’assertion suivante est toujours vraie : `num_entries ≤ num_files`.
* `uncompressed_size` ([UInt64](/fr/reference/data-types/int-uint)) — Taille non compressée de la sauvegarde.
* `compressed_size` ([UInt64](/fr/reference/data-types/int-uint)) — Taille compressée de la sauvegarde. Si la sauvegarde n’est pas stockée sous forme d’archive, elle est égale à uncompressed&#95;size.
* `files_read` ([UInt64](/fr/reference/data-types/int-uint)) — Nombre de fichiers lus pendant l’opération de restauration.
* `bytes_read` ([UInt64](/fr/reference/data-types/int-uint)) — Taille totale des fichiers lus pendant l’opération de restauration.

{/*AUTOGENERATED_END*/}
