<div id="description">
  ## Description
</div>

Contient des informations sur les tables pour lesquelles la commande drop table a été exécutée, mais dont les données n’ont pas encore été nettoyées.

<div id="columns">
  ## Colonnes
</div>

{/*AUTOGENERATED_START*/}

* `index` ([UInt32](/fr/reference/data-types/index)) — Index dans la file d’attente marked&#95;dropped&#95;tables.
* `database` ([String](/fr/reference/data-types/index)) — Nom de la base de données.
* `table` ([String](/fr/reference/data-types/index)) — Nom de la table.
* `uuid` ([UUID](/fr/reference/data-types/index)) — UUID de la table.
* `engine` ([String](/fr/reference/data-types/index)) — Nom du moteur de la table.
* `metadata_dropped_path` ([String](/fr/reference/data-types/index)) — Chemin du fichier de métadonnées de la table dans le répertoire metadata&#95;dropped.
* `table_dropped_time` ([DateTime](/fr/reference/data-types/index)) — Heure à laquelle la prochaine tentative de suppression des données de la table est planifiée. Il s’agit généralement de l’heure à laquelle la table a été supprimée, plus `database_atomic_delay_before_drop_table_sec`.

{/*AUTOGENERATED_END*/}

<div id="example">
  ## Exemple
</div>

L’exemple suivant montre comment obtenir des informations sur `dropped_tables`.

```sql
SELECT *
FROM system.dropped_tables\G
```

```text
Row 1:
──────

```
