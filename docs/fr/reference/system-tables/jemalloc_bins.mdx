<div id="description">
  ## Description
</div>

Contient des informations sur les allocations mémoire effectuées par l’allocateur jemalloc dans différentes classes de taille (bins), agrégées à partir de toutes les arenas.
Ces statistiques peuvent ne pas être totalement exactes en raison du cache local par thread dans jemalloc.

<div id="columns">
  ## Colonnes
</div>

{/*AUTOGENERATED_START*/}

* `index` ([UInt16](/fr/reference/data-types/index)) — Indice du bin, ordonné par taille.
* `large` ([UInt8](/fr/reference/data-types/index)) — Vaut True pour les grandes allocations et False pour les petites.
* `size` ([UInt64](/fr/reference/data-types/index)) — Taille des allocations dans ce bin.
* `allocations` ([Int64](/fr/reference/data-types/index)) — Nombre d’allocations.
* `deallocations` ([Int64](/fr/reference/data-types/index)) — Nombre de désallocations.
* `nregs` ([Int64](/fr/reference/data-types/index)) — Nombre de régions par slab.
* `curslabs` ([Int64](/fr/reference/data-types/index)) — Nombre actuel de slabs.
* `curregs` ([Int64](/fr/reference/data-types/index)) — Nombre actuel de régions pour cette classe de taille.

{/*AUTOGENERATED_END*/}

<div id="example">
  ## Exemple
</div>

Recherchez les tailles d’allocation qui contribuent le plus à la consommation totale actuelle de mémoire.

```sql
SELECT
    *,
    allocations - deallocations AS active_allocations,
    size * active_allocations AS allocated_bytes
FROM system.jemalloc_bins
WHERE allocated_bytes > 0
ORDER BY allocated_bytes DESC
LIMIT 10
```
