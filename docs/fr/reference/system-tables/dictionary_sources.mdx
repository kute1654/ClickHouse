<div id="description">
  ## Description
</div>

Contient la liste des sources de dictionnaire prises en charge par le serveur, ainsi que la documentation intégrée pour chaque source. Une source de dictionnaire détermine d’où sont chargées les données du dictionnaire ; elle est spécifiée dans la clause `SOURCE` d’une requête `CREATE DICTIONARY`.

<div id="columns">
  ## Colonnes
</div>

{/*AUTOGENERATED_START*/}

* `name` ([String](/fr/reference/data-types/index)) — Le nom de la source de dictionnaire, tel qu’il est indiqué dans la clause SOURCE.
* `description` ([String](/fr/reference/data-types/index)) — Une description générale du rôle de la source de dictionnaire.
* `syntax` ([String](/fr/reference/data-types/index)) — La structure de la clause SOURCE utilisée pour spécifier la source. Notez que certaines sources sont soumises au contrôle d’accès lorsqu’un dictionnaire est créé à partir d’une requête DDL (par opposition à un fichier de configuration du serveur) ; consultez la `description` de chaque source pour plus de détails.
* `examples` ([String](/fr/reference/data-types/index)) — Exemples d’utilisation.
* `introduced_in` ([String](/fr/reference/data-types/index)) — La version de ClickHouse dans laquelle la source a été introduite pour la première fois, sous la forme major.minor.
* `related` ([Array(String)](/fr/reference/data-types/index)) — Les noms des sources de dictionnaire associées.

{/*AUTOGENERATED_END*/}

<div id="example">
  ## Exemple
</div>

```sql title="Query"
SELECT name, syntax
FROM system.dictionary_sources
WHERE name IN ('clickhouse', 'file')
ORDER BY name
```

```text title="Response"
┌─name───────┬─syntax─────────────────────────────────────────────────────────────────────────┐
│ clickhouse │ SOURCE(CLICKHOUSE(host 'host' port 9000 user 'default' password '' db 'db' table 'table')) │
│ file       │ SOURCE(FILE(path '/path/to/file' format 'CSV'))                                  │
└────────────┴────────────────────────────────────────────────────────────────────────────────┘
```
