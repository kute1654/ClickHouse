<div id="description">
  ## Description
</div>

Contient la liste de tous les formats, avec des indicateurs précisant si chaque format convient à l’entrée/sortie et s’il prend en charge la parallélisation.

<div id="columns">
  ## Colonnes
</div>

{/*AUTOGENERATED_START*/}

* `name` ([String](/fr/reference/data-types/index)) — Nom du format.
* `is_input` ([UInt8](/fr/reference/data-types/index)) — Indicateur indiquant si le format est adapté à l’entrée de données.
* `is_output` ([UInt8](/fr/reference/data-types/index)) — Indicateur indiquant si le format est adapté à la sortie de données.
* `supports_parallel_parsing` ([UInt8](/fr/reference/data-types/index)) — Indicateur indiquant si le format prend en charge l’analyse en parallèle.
* `supports_parallel_formatting` ([UInt8](/fr/reference/data-types/index)) — Indicateur indiquant si le format prend en charge le formatage en parallèle.
* `is_tty_friendly` ([UInt8](/fr/reference/data-types/index)) — Indicateur indiquant si le format s’affiche généralement correctement dans le terminal. Pour les autres formats, le CLI demandera une confirmation avant d’afficher la sortie.
* `content_type` ([String](/fr/reference/data-types/index)) — Content-Type HTTP correspondant au format de sortie. Peut dépendre des paramètres de format actuels.
* `supports_random_access` ([UInt8](/fr/reference/data-types/index)) — Indicateur indiquant si le format prend en charge l’accès aléatoire en entrée.
* `has_schema_inference` ([UInt8](/fr/reference/data-types/index)) — Le format peut déterminer dynamiquement le schéma à partir des données (soit à partir de l’en-tête ou des métadonnées intégrés, soit à partir des données elles-mêmes).
* `has_external_schema` ([UInt8](/fr/reference/data-types/index)) — Le format a soit un schéma fixe, soit accepte un schéma prédéfini dans son propre format.
* `prefers_large_blocks` ([UInt8](/fr/reference/data-types/index)) — Le format écrit des blocks plus volumineux en sortie et génère des blocks plus volumineux en entrée.
* `supports_append` ([UInt8](/fr/reference/data-types/index)) — Il est possible d’ajouter des données à un fichier unique avec ce format.
* `supports_subsets_of_columns` ([UInt8](/fr/reference/data-types/index)) — Le format d’entrée peut détecter lorsque certaines colonnes sont omises.
* `description` ([String](/fr/reference/data-types/index)) — Description générale du format.
* `examples` ([String](/fr/reference/data-types/index)) — Exemples d’utilisation.
* `introduced_in` ([String](/fr/reference/data-types/index)) — Version de ClickHouse dans laquelle le format a été introduit pour la première fois, sous la forme major.minor.
* `related` ([Array(String)](/fr/reference/data-types/index)) — Noms des formats associés.

{/*AUTOGENERATED_END*/}
