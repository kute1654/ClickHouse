<div id="description">
  ## Description
</div>

Contient des informations sur les insertions asynchrones. Chaque entrée représente une requête d’insertion mise en tampon dans une requête d’insertion asynchrone.

Pour activer la journalisation, configurez les paramètres dans la section [asynchronous&#95;insert&#95;log](/fr/reference/settings/server-settings/settings#asynchronous_insert_log).

La fréquence de vidage des données est définie par le paramètre `flush_interval_milliseconds` de la section des paramètres du serveur [asynchronous&#95;insert&#95;log](/fr/reference/settings/server-settings/settings#asynchronous_insert_log). Pour forcer le vidage, utilisez la requête [SYSTEM FLUSH LOGS](/fr/reference/statements/system#flush-logs).

ClickHouse ne supprime pas automatiquement les données de la table. Consultez [Introduction](/fr/reference/system-tables/overview#system-tables-introduction) pour plus de détails.

<div id="columns">
  ## Colonnes
</div>

{/*AUTOGENERATED_START*/}

* `hostname` ([LowCardinality(String)](/fr/reference/data-types/lowcardinality)) — Nom d’hôte du serveur qui exécute la requête.
* `event_date` ([Date](/fr/reference/data-types/date)) — Date à laquelle l’async insert a eu lieu.
* `event_time` ([DateTime](/fr/reference/data-types/datetime)) — Date et heure auxquelles l’async insert a fini de s’exécuter.
* `event_time_microseconds` ([DateTime64(6)](/fr/reference/data-types/datetime64)) — Date et heure auxquelles l’async insert a fini de s’exécuter, avec une précision à la microseconde.
* `query` ([String](/fr/reference/data-types/string)) — Chaîne de la requête.
* `database` ([LowCardinality(String)](/fr/reference/data-types/lowcardinality)) — Nom de la base de données à laquelle appartient la table.
* `table` ([LowCardinality(String)](/fr/reference/data-types/lowcardinality)) — Nom de la table.
* `format` ([LowCardinality(String)](/fr/reference/data-types/lowcardinality)) — Nom du format.
* `query_id` ([String](/fr/reference/data-types/string)) — ID de la requête initiale.
* `bytes` ([UInt64](/fr/reference/data-types/int-uint)) — Nombre d’octets insérés.
* `rows` ([UInt64](/fr/reference/data-types/int-uint)) — Nombre de lignes insérées.
* `exception` ([String](/fr/reference/data-types/string)) — Message d’exception.
* `status` ([Enum8(&#39;Ok&#39; = 0, &#39;ParsingError&#39; = 1, &#39;FlushError&#39; = 2)](/fr/reference/data-types/enum)) — Statut de l’insertion. Valeurs : &#39;Ok&#39; = 0 — Insertion réussie, &#39;ParsingError&#39; = 1 — Exception lors de l’analyse des données, &#39;FlushError&#39; = 2 — Exception lors du flush des données.
* `data_kind` ([Enum8(&#39;Parsed&#39; = 0, &#39;Preprocessed&#39; = 1)](/fr/reference/data-types/enum)) — État des données. Valeurs : &#39;Parsed&#39; et &#39;Preprocessed&#39;.
* `flush_time` ([DateTime](/fr/reference/data-types/datetime)) — Date et heure auxquelles le flush a eu lieu.
* `flush_time_microseconds` ([DateTime64(6)](/fr/reference/data-types/datetime64)) — Date et heure auxquelles le flush a eu lieu, avec une précision à la microseconde.
* `flush_query_id` ([String](/fr/reference/data-types/string)) — ID de la requête de flush.
* `timeout_milliseconds` ([UInt64](/fr/reference/data-types/int-uint)) — Timeout adaptatif calculé pour cette entrée.

{/*AUTOGENERATED_END*/}
