<div id="description">
  ## Description
</div>

Contient des informations sur les parts des tables [MergeTree](/fr/reference/engines/table-engines/mergetree-family/mergetree) supprimées dans [system.dropped&#95;tables](/fr/reference/system-tables/dropped_tables)

Le schéma de cette table est le même que celui de [system.parts](/fr/reference/system-tables/parts)

<div id="columns">
  ## Colonnes
</div>

{/*AUTOGENERATED_START*/}

* `partition` ([String](/fr/reference/data-types/index)) — Identifiant de partition (représentation sous forme de chaîne dérivée de l’expression de la clé de partition de la table).
* `name` ([String](/fr/reference/data-types/index)) —
  Nom de la part de données. La structure de nommage des parts permet de déterminer de nombreux aspects des données, ainsi que les modèles d’ingestion et de fusion. Le format de nommage des parts est le suivant :

```text
<partition_id>_<minimum_block_number>_<maximum_block_number>_<level>_<data_version>
```

* Définitions :
  * `partition_id` - identifie la clé de partition
  * `minimum_block_number` - identifie le numéro de bloc minimum dans la part. ClickHouse fusionne toujours des blocs continus
  * `maximum_block_number` - identifie le numéro de bloc maximum dans la part
  * `level` - est incrémenté d’une unité à chaque fusion supplémentaire de la part. Un niveau de 0 indique qu’il s’agit d’une nouvelle part qui n’a pas encore été fusionnée. Il est important de garder à l’esprit que toutes les parts dans ClickHouse sont toujours immuables
  * `data_version` - valeur facultative, incrémentée lorsqu’une part est mutée (là encore, les données mutées sont toujours écrites uniquement dans une nouvelle part, puisque les parts sont immuables)

* `uuid` ([UUID](/fr/reference/data-types/index)) — L’UUID de la partie de données.

* `part_type` ([String](/fr/reference/data-types/index)) — Le format de stockage de la partie de données. Valeurs possibles : `Wide` — chaque colonne est stockée dans un fichier distinct, `Compact` — toutes les colonnes sont stockées dans un seul fichier. Le format de stockage des données est contrôlé par les paramètres `min_bytes_for_wide_part` et `min_rows_for_wide_part` de la table MergeTree.

* `active` ([UInt8](/fr/reference/data-types/index)) — Indicateur signalant si la partie de données est active. Si une partie de données est active, elle est utilisée dans une table. Sinon, elle est sur le point d’être supprimée. Les parties de données inactives apparaissent après des opérations de fusion et de mutation.

* `marks` ([UInt64](/fr/reference/data-types/index)) — Le nombre de marques. Pour obtenir le nombre approximatif de lignes dans une partie de données, multipliez les marques par la granularité de l’index (généralement 8192) (cette indication ne fonctionne pas avec la granularité adaptative).

* `rows` ([UInt64](/fr/reference/data-types/index)) — Le nombre de lignes.

* `files` ([UInt64](/fr/reference/data-types/index)) — Le nombre de fichiers dans la partie de données.

* `bytes_on_disk` ([UInt64](/fr/reference/data-types/index)) — Taille totale, en octets, de tous les fichiers de la partie de données.

* `data_compressed_bytes` ([UInt64](/fr/reference/data-types/index)) — Taille totale des données compressées dans la partie de données. Les fichiers auxiliaires (par exemple, les fichiers de marques) ne sont pas inclus.

* `data_uncompressed_bytes` ([UInt64](/fr/reference/data-types/index)) — Taille totale des données non compressées dans la partie de données. Les fichiers auxiliaires (par exemple, les fichiers de marques) ne sont pas inclus.

* `primary_key_size` ([UInt64](/fr/reference/data-types/index)) — La quantité de mémoire (en octets) utilisée par les valeurs de clé primaire dans le fichier primary.idx/cidx sur disque.

* `marks_bytes` ([UInt64](/fr/reference/data-types/index)) — La taille du fichier de marques.

* `secondary_indices_compressed_bytes` ([UInt64](/fr/reference/data-types/index)) — Taille totale des données compressées des index secondaires dans la partie de données. Les fichiers auxiliaires (par exemple, les fichiers de marques) ne sont pas inclus.

* `secondary_indices_uncompressed_bytes` ([UInt64](/fr/reference/data-types/index)) — Taille totale des données non compressées des index secondaires dans la partie de données. Les fichiers auxiliaires (par exemple, les fichiers de marques) ne sont pas inclus.

* `secondary_indices_marks_bytes` ([UInt64](/fr/reference/data-types/index)) — La taille du fichier de marques pour les index secondaires.

* `modification_time` ([DateTime](/fr/reference/data-types/index)) — L’heure à laquelle le répertoire contenant la partie de données a été modifié. Cela correspond généralement à l’heure de création de la partie de données.

* `remove_time` ([DateTime](/fr/reference/data-types/index)) — L’heure à laquelle la partie de données est devenue inactive.

* `refcount` ([UInt32](/fr/reference/data-types/index)) — Le nombre d’utilisations de la partie de données. Une valeur supérieure à 2 indique que la partie de données est utilisée dans des requêtes ou des fusions.

* `min_date` ([Date](/fr/reference/data-types/index)) — La valeur minimale de la clé de date dans la partie de données.

* `max_date` ([Date](/fr/reference/data-types/index)) — La valeur maximale de la clé de date dans la partie de données.

* `min_time` ([DateTime](/fr/reference/data-types/index)) — La valeur minimale de la clé de date et d’heure dans la partie de données.

* `max_time` ([DateTime](/fr/reference/data-types/index)) — La valeur maximale de la clé de date et d’heure dans la partie de données.

* `partition_id` ([String](/fr/reference/data-types/index)) — Identifiant de la partition.

* `min_block_number` ([Int64](/fr/reference/data-types/index)) — Le nombre minimal de parties de données qui composent la partie actuelle après fusion.

* `max_block_number` ([Int64](/fr/reference/data-types/index)) — Le nombre maximal de parties de données qui composent la partie actuelle après fusion.

* `level` ([UInt32](/fr/reference/data-types/index)) — Profondeur de l’arborescence de fusion. Zéro signifie que la partie actuelle a été créée par insert, et non par fusion d’autres parties.

* `data_version` ([UInt64](/fr/reference/data-types/index)) — Nombre utilisé pour déterminer quelles mutations doivent être appliquées à la part de données (mutations avec une version supérieure à data&#95;version).

* `primary_key_bytes_in_memory` ([UInt64](/fr/reference/data-types/index)) — Quantité de mémoire (en octets) utilisée par les valeurs de clé primaire. Vaut 0 lorsque `primary_key_lazy_load` est activé et que la clé n’est pas chargée. Lorsqu’elle est non nulle, les octets se trouvent dans la part elle-même et sont comptabilisés dans `jemalloc.mergetree_arena.active_bytes`. Ils ne sont JAMAIS comptés dans `PrimaryIndexCacheBytes` — ces deux cas s’excluent mutuellement pour chaque part : un index se trouve soit dans la part (cette métrique), soit dans le `PrimaryIndexCache` partagé (l’autre), selon `primary_key_lazy_load` et `use_primary_key_cache`.

* `primary_key_bytes_in_memory_allocated` ([UInt64](/fr/reference/data-types/index)) — Quantité de mémoire (en octets) réservée aux valeurs de clé primaire. Vaut 0 lorsque `primary_key_lazy_load` est activé et que la clé n’est pas chargée. Lorsqu’elle est non nulle, elle est incluse dans `jemalloc.mergetree_arena.active_bytes`. Voir la note sur `primary_key_bytes_in_memory` pour la relation avec `PrimaryIndexCacheBytes`.

* `index_granularity_bytes_in_memory` ([UInt64](/fr/reference/data-types/index)) — Quantité de mémoire (en octets) utilisée par les valeurs de granularité d’index (vaut 0 dans le cas de primary&#95;key&#95;lazy&#95;load=1 et use&#95;primary&#95;key&#95;cache=1). Lorsqu’elle est non nulle, les octets appartiennent à la part et sont comptabilisés dans `jemalloc.mergetree_arena.active_bytes`.

* `index_granularity_bytes_in_memory_allocated` ([UInt64](/fr/reference/data-types/index)) — Quantité de mémoire (en octets) réservée aux valeurs de granularité d’index (vaut 0 dans le cas de primary&#95;key&#95;lazy&#95;load=1 et use&#95;primary&#95;key&#95;cache=1). Lorsqu’elle est non nulle, elle est incluse dans `jemalloc.mergetree_arena.active_bytes`.

* `is_frozen` ([UInt8](/fr/reference/data-types/index)) — Indicateur signalant l’existence d’une sauvegarde des données de la partition. 1 : la sauvegarde existe. 0 : la sauvegarde n’existe pas. Pour plus de détails, voir FREEZE PARTITION.

* `database` ([String](/fr/reference/data-types/index)) — Nom de la base de données.

* `table` ([String](/fr/reference/data-types/index)) — Nom de la table.

* `engine` ([String](/fr/reference/data-types/index)) — Nom du moteur de table sans paramètres.

* `disk_name` ([String](/fr/reference/data-types/index)) — Nom du disque qui stocke la part de données.

* `path` ([String](/fr/reference/data-types/index)) — Chemin absolu vers le dossier contenant les fichiers de la part de données.

* `hash_of_all_files` ([String](/fr/reference/data-types/index)) — sipHash128 des fichiers compressés.

* `hash_of_uncompressed_files` ([String](/fr/reference/data-types/index)) — sipHash128 des fichiers non compressés (fichiers avec marks, fichier d’index, etc.).

* `uncompressed_hash_of_compressed_files` ([String](/fr/reference/data-types/index)) — sipHash128 des données contenues dans les fichiers compressés, comme si elles n’étaient pas compressées.

* `delete_ttl_info_min` ([DateTime](/fr/reference/data-types/index)) — Valeur minimale de la clé de date et d’heure pour la règle TTL DELETE.

* `delete_ttl_info_max` ([DateTime](/fr/reference/data-types/index)) — Valeur maximale de la clé de date et d’heure pour la règle TTL DELETE.

* `move_ttl_info.expression` ([Array(String)](/fr/reference/data-types/index)) — Tableau d’expressions. Chaque expression définit une règle TTL MOVE.

* `move_ttl_info.min` ([Array(DateTime)](/fr/reference/data-types/index)) — Tableau de valeurs de date et d’heure. Chaque élément décrit la valeur minimale de la clé pour une règle TTL MOVE.

* `move_ttl_info.max` ([Array(DateTime)](/fr/reference/data-types/index)) — Tableau de valeurs de date et d’heure. Chaque élément décrit la valeur maximale de la clé pour une règle TTL MOVE.

* `default_compression_codec` ([String](/fr/reference/data-types/index)) — Nom du codec utilisé pour compresser cette part de données (lorsqu’aucun codec explicite n’est défini pour les colonnes).

* `recompression_ttl_info.expression` ([Array(String)](/fr/reference/data-types/index)) — Expression TTL.

* `recompression_ttl_info.min` ([Array(DateTime)](/fr/reference/data-types/index)) — Valeur minimale de l’expression TTL calculée dans cette part. Permet de déterminer s’il existe au moins une ligne dont le TTL a expiré.

* `recompression_ttl_info.max` ([Array(DateTime)](/fr/reference/data-types/index)) — La valeur maximale de l’expression TTL calculée dans cette part. Sert à déterminer si toutes les lignes ont un TTL expiré.

* `group_by_ttl_info.expression` ([Array(String)](/fr/reference/data-types/index)) — L’expression TTL.

* `group_by_ttl_info.min` ([Array(DateTime)](/fr/reference/data-types/index)) — La valeur minimale de l’expression TTL calculée dans cette part. Sert à déterminer si au moins une ligne a un TTL expiré.

* `group_by_ttl_info.max` ([Array(DateTime)](/fr/reference/data-types/index)) — La valeur maximale de l’expression TTL calculée dans cette part. Sert à déterminer si toutes les lignes ont un TTL expiré.

* `rows_where_ttl_info.expression` ([Array(String)](/fr/reference/data-types/index)) — L’expression TTL.

* `rows_where_ttl_info.min` ([Array(DateTime)](/fr/reference/data-types/index)) — La valeur minimale de l’expression TTL calculée dans cette part. Sert à déterminer si au moins une ligne a un TTL expiré.

* `rows_where_ttl_info.max` ([Array(DateTime)](/fr/reference/data-types/index)) — La valeur maximale de l’expression TTL calculée dans cette part. Sert à déterminer si toutes les lignes ont un TTL expiré.

* `projections` ([Array(String)](/fr/reference/data-types/index)) — La liste des noms de projections calculés pour cette part.

* `visible` ([UInt8](/fr/reference/data-types/index)) — Indicateur signalant si cette part est visible pour les requêtes SELECT.

* `creation_tid` ([Tuple(UInt64, UInt64, UUID)](/fr/reference/data-types/index)) — ID de la transaction qui a créé ou essaie de créer cet objet.

* `removal_tid_lock` ([UInt64](/fr/reference/data-types/index)) — Hachage de removal&#95;tid, utilisé pour verrouiller un objet en vue de sa suppression.

* `removal_tid` ([Tuple(UInt64, UInt64, UUID)](/fr/reference/data-types/index)) — ID de la transaction qui a supprimé ou essaie de supprimer cet objet

* `creation_csn` ([UInt64](/fr/reference/data-types/index)) — CSN de la transaction qui a créé cet objet

* `removal_csn` ([UInt64](/fr/reference/data-types/index)) — CSN de la transaction qui a supprimé cet objet

* `has_lightweight_delete` ([UInt8](/fr/reference/data-types/index)) — Indicateur signalant si la part possède un masque de lightweight delete.

* `last_removal_attempt_time` ([DateTime](/fr/reference/data-types/index)) — Dernière tentative du serveur pour supprimer cette part.

* `removal_state` ([String](/fr/reference/data-types/index)) — L’état actuel du processus de suppression de la part.
