<div id="description">
  ## Description
</div>

Contient des entrées de journal relatives à différentes opérations de stockage de blob, comme les téléversements et les suppressions.

<div id="columns">
  ## Colonnes
</div>

{/*AUTOGENERATED_START*/}

* `hostname` ([LowCardinality(String)](/fr/reference/data-types/lowcardinality)) — Nom d’hôte du serveur exécutant la requête.
* `event_date` ([Date](/fr/reference/data-types/date)) — Date de l’événement.
* `event_time` ([DateTime](/fr/reference/data-types/datetime)) — Heure de l’événement.
* `event_time_microseconds` ([DateTime64(6)](/fr/reference/data-types/datetime64)) — Heure de l’événement avec une précision à la microseconde.
* `event_type` ([Enum8(&#39;Upload&#39; = 1, &#39;Delete&#39; = 2, &#39;MultiPartUploadCreate&#39; = 3, &#39;MultiPartUploadWrite&#39; = 4, &#39;MultiPartUploadComplete&#39; = 5, &#39;MultiPartUploadAbort&#39; = 6, &#39;Read&#39; = 7)](/fr/reference/data-types/enum)) — Type de l’événement. Valeurs possibles : &#39;Upload&#39;, &#39;Delete&#39;, &#39;MultiPartUploadCreate&#39;, &#39;MultiPartUploadWrite&#39;, &#39;MultiPartUploadComplete&#39;, &#39;MultiPartUploadAbort&#39;, &#39;Read&#39;
* `query_id` ([String](/fr/reference/data-types/string)) — Identifiant de la requête associée à l’événement, le cas échéant.
* `thread_id` ([UInt64](/fr/reference/data-types/int-uint)) — Identifiant du thread effectuant l’opération.
* `thread_name` ([String](/fr/reference/data-types/string)) — Nom du thread effectuant l’opération.
* `disk_name` ([LowCardinality(String)](/fr/reference/data-types/lowcardinality)) — Nom du disque associé.
* `bucket` ([String](/fr/reference/data-types/string)) — Nom du bucket.
* `remote_path` ([String](/fr/reference/data-types/string)) — Chemin d’accès à la ressource distante.
* `local_path` ([String](/fr/reference/data-types/string)) — Chemin d’accès au fichier de métadonnées sur le système local, qui fait référence à la ressource distante.
* `data_size` ([UInt64](/fr/reference/data-types/int-uint)) — Taille des données impliquées dans l’événement de téléversement.
* `elapsed_microseconds` ([UInt64](/fr/reference/data-types/int-uint)) — Temps écoulé de l’opération, en microsecondes.
* `error_code` ([Int32](/fr/reference/data-types/int-uint)) — Code d’erreur de l’opération. 0 s’il n’y a pas eu d’erreur.
* `error` ([String](/fr/reference/data-types/string)) — Message d’erreur associé à l’événement, le cas échéant.

{/*AUTOGENERATED_END*/}

<div id="example">
  ## Exemple
</div>

Supposons qu&#39;une opération de stockage de blob téléverse un fichier et qu&#39;un événement soit consigné :

```sql
SELECT * FROM system.blob_storage_log WHERE query_id = '7afe0450-504d-4e4b-9a80-cd9826047972' ORDER BY event_date, event_time_microseconds \G
```

```text
Row 1:
──────
hostname:                clickhouse.eu-central1.internal
event_date:              2023-10-31
event_time:              2023-10-31 16:03:40
event_time_microseconds: 2023-10-31 16:03:40.481437
event_type:              Upload
query_id:                7afe0450-504d-4e4b-9a80-cd9826047972
thread_id:               2381740
disk_name:               disk_s3
bucket:                  bucket1
remote_path:             rrr/kxo/tbnqtrghgtnxkzgtcrlutwuslgawe
local_path:              store/654/6549e8b3-d753-4447-8047-d462df6e6dbe/tmp_insert_all_1_1_0/checksums.txt
data_size:               259
error:
```

Dans cet exemple, l’opération de téléversement a été associée à la requête `INSERT` avec l’ID `7afe0450-504d-4e4b-9a80-cd9826047972`. Le fichier local de métadonnées `store/654/6549e8b3-d753-4447-8047-d462df6e6dbe/tmp_insert_all_1_1_0/checksums.txt` pointe vers le chemin distant `rrr/kxo/tbnqtrghgtnxkzgtcrlutwuslgawe` dans le bucket `bucket1` sur le disque `disk_s3`, avec une taille de 259 octets.
