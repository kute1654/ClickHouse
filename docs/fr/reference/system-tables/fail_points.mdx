<div id="description">
  ## Description
</div>

Contient la liste de tous les failpoints disponibles enregistrés sur le serveur, ainsi que leur type et leur état d’activation actuel.

Les failpoints peuvent être activés et désactivés au moment de l’exécution à l’aide des instructions `SYSTEM ENABLE FAILPOINT` et `SYSTEM DISABLE FAILPOINT`.

<div id="columns">
  ## Colonnes
</div>

{/*AUTOGENERATED_START*/}

* `name` ([String](/fr/reference/data-types/index)) — Nom du failpoint.
* `type` ([Enum8(&#39;once&#39; = 0, &#39;regular&#39; = 1, &#39;pauseable&#95;once&#39; = 2, &#39;pauseable&#39; = 3)](/fr/reference/data-types/index)) — Type de failpoint : &#39;once&#39; se déclenche une seule fois puis se désactive automatiquement, &#39;regular&#39; se déclenche à chaque fois, &#39;pauseable&#95;once&#39; bloque l’exécution une seule fois, &#39;pauseable&#39; bloque l’exécution à chaque fois jusqu’à ce qu’elle reprenne.
* `enabled` ([UInt8](/fr/reference/data-types/index)) — Indique si le failpoint est actuellement activé (1) ou désactivé (0).

{/*AUTOGENERATED_END*/}

<div id="example">
  ## Exemple
</div>

```sql title="Query"
SYSTEM ENABLE FAILPOINT replicated_merge_tree_insert_retry_pause;
SELECT * FROM system.fail_points WHERE enabled = 1
```
