<div id="description">
  ## Description
</div>

Cette table contient des informations sur les parties de projection des tables de la famille MergeTree.

<div id="columns">
  ## Colonnes
</div>

{/*AUTOGENERATED_START*/}

* `partition` ([String](/fr/reference/data-types/index)) — Le nom de la partition.
* `name` ([String](/fr/reference/data-types/index)) — Nom de la part de données.
* `part_type` ([String](/fr/reference/data-types/index)) — Le format de stockage de la part de données. Valeurs possibles : Wide (un fichier par colonne) et Compact (un seul fichier pour toutes les colonnes).
* `parent_name` ([String](/fr/reference/data-types/index)) — Le nom de la part de données source (parente).
* `parent_uuid` ([UUID](/fr/reference/data-types/index)) — L’UUID de la part de données source (parente).
* `parent_part_type` ([String](/fr/reference/data-types/index)) — Le format de stockage de la part de données source (parente).
* `active` ([UInt8](/fr/reference/data-types/index)) — Indicateur signalant si la part de données est active. Si une part de données est active, elle est utilisée dans une table. Sinon, elle est sur le point d’être supprimée. Les parts de données inactives apparaissent après des opérations de fusion et de mutation.
* `marks` ([UInt64](/fr/reference/data-types/index)) — Le nombre de marks. Pour obtenir le nombre approximatif de lignes dans une part de données, multipliez les marks par la granularité de l’index (généralement 8192) (cette indication ne fonctionne pas pour la granularité adaptative).
* `rows` ([UInt64](/fr/reference/data-types/index)) — Le nombre de lignes.
* `bytes_on_disk` ([UInt64](/fr/reference/data-types/index)) — Taille totale de tous les fichiers de la part de données en octets.
* `data_compressed_bytes` ([UInt64](/fr/reference/data-types/index)) — Taille totale des données compressées dans la part de données. Tous les fichiers auxiliaires (par exemple, les fichiers de marks) ne sont pas inclus.
* `data_uncompressed_bytes` ([UInt64](/fr/reference/data-types/index)) — Taille totale des données non compressées dans la part de données. Tous les fichiers auxiliaires (par exemple, les fichiers de marks) ne sont pas inclus.
* `marks_bytes` ([UInt64](/fr/reference/data-types/index)) — La taille du fichier de marks.
* `parent_marks` ([UInt64](/fr/reference/data-types/index)) — Le nombre de marks dans la part source (parente).
* `parent_rows` ([UInt64](/fr/reference/data-types/index)) — Le nombre de lignes dans la part source (parente).
* `parent_bytes_on_disk` ([UInt64](/fr/reference/data-types/index)) — Taille totale de tous les fichiers de la part de données source (parente) en octets.
* `parent_data_compressed_bytes` ([UInt64](/fr/reference/data-types/index)) — Taille totale des données compressées dans la part de données source (parente).
* `parent_data_uncompressed_bytes` ([UInt64](/fr/reference/data-types/index)) — Taille totale des données non compressées dans la part de données source (parente).
* `parent_marks_bytes` ([UInt64](/fr/reference/data-types/index)) — La taille du fichier de marks dans la part de données source (parente).
* `modification_time` ([DateTime](/fr/reference/data-types/index)) — L’heure à laquelle le répertoire contenant la part de données a été modifié. Cela correspond généralement à l’heure de création de la part de données.
* `remove_time` ([DateTime](/fr/reference/data-types/index)) — L’heure à laquelle la part de données est devenue inactive.
* `refcount` ([UInt32](/fr/reference/data-types/index)) — Le nombre d’endroits où la part de données est utilisée. Une valeur supérieure à 2 indique que la part de données est utilisée dans des requêtes ou des fusions.
* `min_date` ([Date](/fr/reference/data-types/index)) — La valeur minimale de la clé de date dans la part de données.
* `max_date` ([Date](/fr/reference/data-types/index)) — La valeur maximale de la clé de date dans la part de données.
* `min_time` ([DateTime](/fr/reference/data-types/index)) — La valeur minimale de la clé de date et d’heure dans la part de données.
* `max_time` ([DateTime](/fr/reference/data-types/index)) — La valeur maximale de la clé de date et d’heure dans la partie de données.
* `partition_id` ([String](/fr/reference/data-types/index)) — ID de la partition.
* `min_block_number` ([Int64](/fr/reference/data-types/index)) — Le nombre minimal de parties de données qui composent la partie actuelle après fusion.
* `max_block_number` ([Int64](/fr/reference/data-types/index)) — Le nombre maximal de parties de données qui composent la partie actuelle après fusion.
* `level` ([UInt32](/fr/reference/data-types/index)) — Profondeur de l’arbre de fusion. Zéro signifie que la partie actuelle a été créée par insertion et non par fusion d’autres parties.
* `data_version` ([UInt64](/fr/reference/data-types/index)) — Nombre utilisé pour déterminer quelles mutations doivent être appliquées à la partie de données (mutations avec une version supérieure à data&#95;version).
* `primary_key_bytes_in_memory` ([UInt64](/fr/reference/data-types/index)) — La quantité de mémoire (en octets) utilisée par les valeurs de la clé primaire.
* `primary_key_bytes_in_memory_allocated` ([UInt64](/fr/reference/data-types/index)) — La quantité de mémoire (en octets) réservée aux valeurs de la clé primaire.
* `is_frozen` ([UInt8](/fr/reference/data-types/index)) — Indicateur montrant qu’une sauvegarde des données de la partition existe. 1, la sauvegarde existe. 0, la sauvegarde n’existe pas.
* `database` ([String](/fr/reference/data-types/index)) — Nom de la base de données.
* `table` ([String](/fr/reference/data-types/index)) — Nom de la table.
* `engine` ([String](/fr/reference/data-types/index)) — Nom du moteur de table sans paramètres.
* `disk_name` ([String](/fr/reference/data-types/index)) — Nom du disque qui stocke la partie de données.
* `path` ([String](/fr/reference/data-types/index)) — Chemin absolu vers le dossier contenant les fichiers de la partie de données.
* `hash_of_all_files` ([String](/fr/reference/data-types/index)) — sipHash128 des fichiers compressés.
* `hash_of_uncompressed_files` ([String](/fr/reference/data-types/index)) — sipHash128 des fichiers non compressés (fichiers avec marks, fichier d’index, etc.).
* `uncompressed_hash_of_compressed_files` ([String](/fr/reference/data-types/index)) — sipHash128 des données des fichiers compressés, comme si elles n’étaient pas compressées.
* `delete_ttl_info_min` ([DateTime](/fr/reference/data-types/index)) — La valeur minimale de la clé de date et d’heure pour la règle TTL DELETE.
* `delete_ttl_info_max` ([DateTime](/fr/reference/data-types/index)) — La valeur maximale de la clé de date et d’heure pour la règle TTL DELETE.
* `move_ttl_info.expression` ([Array(String)](/fr/reference/data-types/index)) — Tableau d’expressions. Chaque expression définit une règle TTL MOVE.
* `move_ttl_info.min` ([Array(DateTime)](/fr/reference/data-types/index)) — Tableau de valeurs de date et d’heure. Chaque élément décrit la valeur minimale de la clé pour une règle TTL MOVE.
* `move_ttl_info.max` ([Array(DateTime)](/fr/reference/data-types/index)) — Tableau de valeurs de date et d’heure. Chaque élément décrit la valeur maximale de la clé pour une règle TTL MOVE.
* `default_compression_codec` ([String](/fr/reference/data-types/index)) — Le nom du codec utilisé pour compresser cette partie de données (lorsqu’il n’existe pas de codec explicite pour les colonnes).
* `recompression_ttl_info.expression` ([Array(String)](/fr/reference/data-types/index)) — L’expression TTL.
* `recompression_ttl_info.min` ([Array(DateTime)](/fr/reference/data-types/index)) — La valeur minimale de l’expression TTL calculée dans cette partie. Elle sert à déterminer si au moins une ligne a un TTL expiré.
* `recompression_ttl_info.max` ([Array(DateTime)](/fr/reference/data-types/index)) — Valeur maximale de l’expression TTL calculée dans cette partie. Utilisée pour déterminer si toutes les lignes ont un TTL expiré.
* `group_by_ttl_info.expression` ([Array(String)](/fr/reference/data-types/index)) — L’expression TTL.
* `group_by_ttl_info.min` ([Array(DateTime)](/fr/reference/data-types/index)) — Valeur minimale de l’expression TTL calculée dans cette partie. Utilisée pour déterminer si au moins une ligne a un TTL expiré.
* `group_by_ttl_info.max` ([Array(DateTime)](/fr/reference/data-types/index)) — Valeur maximale de l’expression TTL calculée dans cette partie. Utilisée pour déterminer si toutes les lignes ont un TTL expiré.
* `rows_where_ttl_info.expression` ([Array(String)](/fr/reference/data-types/index)) — L’expression TTL.
* `rows_where_ttl_info.min` ([Array(DateTime)](/fr/reference/data-types/index)) — Valeur minimale de l’expression TTL calculée dans cette partie. Utilisée pour déterminer si au moins une ligne a un TTL expiré.
* `rows_where_ttl_info.max` ([Array(DateTime)](/fr/reference/data-types/index)) — Valeur maximale de l’expression TTL calculée dans cette partie. Utilisée pour déterminer si toutes les lignes ont un TTL expiré.
* `is_broken` ([UInt8](/fr/reference/data-types/index)) — Indique si la partie de projection est corrompue
* `exception_code` ([Int32](/fr/reference/data-types/index)) — Message d’exception expliquant l’état corrompu de la partie de projection
* `exception` ([String](/fr/reference/data-types/index)) — Code d’exception expliquant l’état corrompu de la partie de projection
