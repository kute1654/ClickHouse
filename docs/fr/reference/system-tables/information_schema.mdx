`INFORMATION_SCHEMA` (ou : `information_schema`) est une base de données système qui fournit une [vue indépendante du SGBD](https://en.wikipedia.org/wiki/Information_schema), quoique seulement partiellement normalisée, des métadonnées des objets de base de données. Les vues de `INFORMATION_SCHEMA` sont généralement moins complètes que les tables système classiques, mais les outils peuvent les utiliser pour obtenir des informations de base de manière inter-SGBD. La structure et le contenu des vues de `INFORMATION_SCHEMA` sont conçus pour évoluer de manière rétrocompatible, c’est-à-dire que seules de nouvelles fonctionnalités sont ajoutées, sans modification ni suppression des fonctionnalités existantes. Du point de vue de l’implémentation interne, les vues de `INFORMATION_SCHEMA` correspondent généralement à des tables système classiques telles que [system.columns](/fr/reference/system-tables/columns), [system.databases](/fr/reference/system-tables/databases) et [system.tables](/fr/reference/system-tables/tables).

```sql
SHOW TABLES FROM INFORMATION_SCHEMA;

-- or:
SHOW TABLES FROM information_schema;
```

```text

│ KEY_COLUMN_USAGE        │
```

`INFORMATION_SCHEMA` contient les vues suivantes :

* [COLUMNS](#columns)
* [KEY&#95;COLUMN&#95;USAGE](#key_column_usage)
* [REFERENTIAL&#95;CONSTRAINTS](#referential_constraints)
* [SCHEMATA](#schemata)
* [STATISTICS](#statistics)
* [TABLES](#tables)
* [VIEWS](#views)

Des vues équivalentes non sensibles à la casse, par exemple `INFORMATION_SCHEMA.columns`, sont fournies pour des raisons de compatibilité avec d&#39;autres bases de données. Il en va de même pour toutes les colonnes de ces vues : des variantes en minuscules (par exemple `table_name`) et en majuscules (`TABLE_NAME`) sont disponibles.

<div id="schemata">
  ## SCHEMATA
</div>

Contient les colonnes lues à partir de la table système [system.databases](/fr/reference/system-tables/databases), ainsi que des colonnes qui ne sont pas prises en charge dans ClickHouse ou qui n’ont pas de sens (toujours `NULL`), mais qui sont requises par la norme.

Colonnes :

* `catalog_name` ([String](/fr/reference/data-types/string)) — Le nom de la base de données.
* `schema_name` ([String](/fr/reference/data-types/string)) — Le nom de la base de données.
* `schema_owner` ([String](/fr/reference/data-types/string)) — Nom du propriétaire du schéma, toujours `'default'`.
* `default_character_set_catalog` ([Nullable](/fr/reference/data-types/nullable)([String](/fr/reference/data-types/string))) — `NULL`, non pris en charge.
* `default_character_set_schema` ([Nullable](/fr/reference/data-types/nullable)([String](/fr/reference/data-types/string))) — `NULL`, non pris en charge.
* `default_character_set_name` ([Nullable](/fr/reference/data-types/nullable)([String](/fr/reference/data-types/string))) — `NULL`, non pris en charge.
* `sql_path` ([Nullable](/fr/reference/data-types/nullable)([String](/fr/reference/data-types/string))) — `NULL`, non pris en charge.

**Exemple**

```sql title="Query"
SELECT catalog_name,
       schema_name,
       schema_owner,
       default_character_set_catalog,
       default_character_set_schema,
       default_character_set_name,
       sql_path
FROM information_schema.schemata
WHERE schema_name ILIKE 'information_schema' 
LIMIT 1 
FORMAT Vertical;
```

<div id="tables">
  ## TABLES
</div>

Contient les colonnes lues à partir de la table système [system.tables](/fr/reference/system-tables/tables).

Colonnes :

* `table_catalog` ([String](/fr/reference/data-types/string)) — Le nom de la base de données dans laquelle se trouve la table.
* `table_schema` ([String](/fr/reference/data-types/string)) — Le nom de la base de données dans laquelle se trouve la table.
* `table_name` ([String](/fr/reference/data-types/string)) — Le nom de la table.
* `table_type` ([String](/fr/reference/data-types/string)) — Le type de table. Valeurs possibles :
  * `BASE TABLE`
  * `VIEW`
  * `FOREIGN TABLE`
  * `LOCAL TEMPORARY`
  * `SYSTEM VIEW`
* `table_rows` ([Nullable](/fr/reference/data-types/nullable)([UInt64](/fr/reference/data-types/int-uint))) — Le nombre total de lignes. NULL s&#39;il n&#39;a pas pu être déterminé.
* `data_length` ([Nullable](/fr/reference/data-types/nullable)([UInt64](/fr/reference/data-types/int-uint))) — La taille des données sur disque. NULL si elle n&#39;a pas pu être déterminée.
* `index_length` ([Nullable](/fr/reference/data-types/nullable)([UInt64](/fr/reference/data-types/int-uint))) — La taille totale de la clé primaire, des index secondaires et de toutes les marks.
* `table_collation` ([Nullable](/fr/reference/data-types/nullable)([String](/fr/reference/data-types/string))) — La collation par défaut de la table. Toujours `utf8mb4_0900_ai_ci`.
* `table_comment` ([Nullable](/fr/reference/data-types/nullable)([String](/fr/reference/data-types/string))) — Le commentaire utilisé lors de la création de la table.

**Exemple**

```sql title="Query"
SELECT table_catalog, 
       table_schema, 
       table_name, 
       table_type, 
       table_collation, 
       table_comment
FROM INFORMATION_SCHEMA.TABLES
WHERE (table_schema = currentDatabase() OR table_schema = '')
  AND table_name NOT LIKE '%inner%'
LIMIT 1 
FORMAT Vertical;
```

```text title="Response"
Row 1:
──────
table_catalog:   default
table_schema:    default
table_name:      describe_example
table_type:      BASE TABLE
table_collation: utf8mb4_0900_ai_ci
table_comment:   
```

<div id="views">
  ## VIEWS
</div>

Contient les colonnes issues de la table système [system.tables](/fr/reference/system-tables/tables), lorsque le moteur de table [View](/fr/reference/engines/table-engines/special/view) est utilisé.

Colonnes :

{/*AUTOGENERATED_START*/}

* `table_catalog` ([String](/fr/reference/data-types/string)) — Nom de la base de données dans laquelle se trouve la table.
* `table_schema` ([String](/fr/reference/data-types/string)) — Nom de la base de données dans laquelle se trouve la table.
* `table_name` ([String](/fr/reference/data-types/string)) — Nom de la table.
* `view_definition` ([String](/fr/reference/data-types/string)) — Requête `SELECT` de la vue.
* `check_option` ([String](/fr/reference/data-types/string)) — `NONE`, aucune vérification.
* `is_updatable` ([Enum8](/fr/reference/data-types/enum)) — `NO`, la vue ne peut pas être mise à jour.
* `is_insertable_into` ([Enum8](/fr/reference/data-types/enum)) — Indique si la vue créée est [matérialisée](/fr/reference/statements/create/view#materialized-view). Valeurs possibles :
  * `NO` — La vue créée n’est pas matérialisée.
  * `YES` — La vue créée est matérialisée.
* `is_trigger_updatable` ([Enum8](/fr/reference/data-types/enum)) — `NO`, le déclencheur ne peut pas être mis à jour.
* `is_trigger_deletable` ([Enum8](/fr/reference/data-types/enum)) — `NO`, le déclencheur ne peut pas être supprimé.
* `is_trigger_insertable_into` ([Enum8](/fr/reference/data-types/enum)) — `NO`, aucune donnée ne peut être insérée dans le déclencheur.

{/*AUTOGENERATED_END*/}

**Exemple**

```sql title="Query"
CREATE VIEW v (n Nullable(Int32), f Float64) AS SELECT n, f FROM t;
CREATE MATERIALIZED VIEW mv ENGINE = Null AS SELECT * FROM system.one;
SELECT table_catalog,
       table_schema,
       table_name,
       view_definition,
       check_option,
       is_updatable,
       is_insertable_into,
       is_trigger_updatable,
       is_trigger_deletable,
       is_trigger_insertable_into
FROM information_schema.views
WHERE table_schema = currentDatabase() 
LIMIT 1
FORMAT Vertical;
```

```text title="Response"
Row 1:
──────
table_catalog:              default
table_schema:               default
table_name:                 mv
view_definition:            SELECT * FROM system.one
check_option:               NONE
is_updatable:               NO
is_insertable_into:         YES
is_trigger_updatable:       NO
is_trigger_deletable:       NO
is_trigger_insertable_into: NO
```

<div id="key_column_usage">
  ## KEY_COLUMN_USAGE
</div>

Contient les colonnes de la table système [system.tables](/fr/reference/system-tables/tables) qui sont soumises à des contraintes.

Colonnes :

* `constraint_catalog` ([String](/fr/reference/data-types/string)) — Actuellement inutilisé. Toujours `def`.
* `constraint_schema` ([String](/fr/reference/data-types/string)) — Le nom du schéma (base de données) auquel appartient la contrainte.
* `constraint_name` ([Nullable](/fr/reference/data-types/nullable)([String](/fr/reference/data-types/string))) — Le nom de la contrainte.
* `table_catalog` ([String](/fr/reference/data-types/string)) — Actuellement inutilisé. Toujours `def`.
* `table_schema` ([String](/fr/reference/data-types/string)) — Le nom du schéma (base de données) auquel appartient la table.
* `table_name` ([String](/fr/reference/data-types/string)) — Le nom de la table à laquelle la contrainte s&#39;applique.
* `column_name` ([Nullable](/fr/reference/data-types/nullable)([String](/fr/reference/data-types/string))) — Le nom de la colonne à laquelle la contrainte s&#39;applique.
* `ordinal_position` ([UInt32](/fr/reference/data-types/int-uint)) — Actuellement inutilisé. Toujours `1`.
* `position_in_unique_constraint` ([Nullable](/fr/reference/data-types/nullable)([UInt32](/fr/reference/data-types/int-uint))) — Actuellement inutilisé. Toujours `NULL`.
* `referenced_table_schema` ([Nullable](/fr/reference/data-types/nullable)([String](/fr/reference/data-types/string))) — Actuellement inutilisé. Toujours NULL.
* `referenced_table_name` ([Nullable](/fr/reference/data-types/nullable)([String](/fr/reference/data-types/string))) — Actuellement inutilisé. Toujours NULL.
* `referenced_column_name` ([Nullable](/fr/reference/data-types/nullable)([String](/fr/reference/data-types/string))) — Actuellement inutilisé. Toujours NULL.

**Exemple**

```sql title="Query"
CREATE TABLE test (i UInt32, s String) ENGINE MergeTree ORDER BY i;
SELECT constraint_catalog,
       constraint_schema,
       constraint_name,
       table_catalog,
       table_schema,
       table_name,
       column_name,
       ordinal_position,
       position_in_unique_constraint,
       referenced_table_schema,
       referenced_table_name,
       referenced_column_name
FROM information_schema.key_column_usage 
WHERE table_name = 'test' 
FORMAT Vertical;
```
