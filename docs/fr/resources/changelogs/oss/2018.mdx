<div id="clickhouse-release-18-16">
  ## Version 18.16 de ClickHouse
</div>

<div id="clickhouse-release-18-16-1-2018-12-21">
  ### Version de ClickHouse 18.16.1, 2018-12-21
</div>

<div id="bug-fixes">
  #### Correction de bug :
</div>

* Correction d’une erreur à l’origine de problèmes lors de la mise à jour de dictionnaires à partir d’une source ODBC. [#3825](https://github.com/ClickHouse/ClickHouse/issues/3825), [#3829](https://github.com/ClickHouse/ClickHouse/issues/3829)
* La compilation JIT des fonctions d’agrégation fonctionne désormais avec les colonnes LowCardinality. [#3838](https://github.com/ClickHouse/ClickHouse/issues/3838)

<div id="improvements">
  #### Améliorations :
</div>

* Ajout du paramètre `low_cardinality_allow_in_native_format` (activé par défaut). Lorsqu’il est désactivé, les colonnes LowCardinality sont converties en colonnes ordinaires pour les requêtes SELECT, et des colonnes ordinaires sont attendues pour les requêtes INSERT. [#3879](https://github.com/ClickHouse/ClickHouse/pull/3879)

<div id="build-improvements">
  #### Améliorations de la compilation :
</div>

* Correctifs pour la compilation sur macOS et ARM.

<div id="clickhouse-release-18-16-0-2018-12-14">
  ### Version de ClickHouse 18.16.0, 2018-12-14
</div>

<div id="new-features">
  #### Nouvelles fonctionnalités :
</div>

* Les expressions `DEFAULT` sont évaluées pour les champs manquants lors du chargement des données dans des formats d’entrée semi-structurés (`JSONEachRow`, `TSKV`). Cette fonctionnalité est activée via le paramètre `insert_sample_with_metadata`. [#3555](https://github.com/ClickHouse/ClickHouse/pull/3555)
* La requête `ALTER TABLE` prend désormais en charge l’action `MODIFY ORDER BY` pour modifier la clé de tri lors de l’ajout ou de la suppression d’une colonne d’une table. Cela est utile pour les tables de la famille `MergeTree` qui effectuent des opérations supplémentaires lors de la fusion en fonction de cette clé de tri, comme `SummingMergeTree`, `AggregatingMergeTree`, etc. [#3581](https://github.com/ClickHouse/ClickHouse/pull/3581) [#3755](https://github.com/ClickHouse/ClickHouse/pull/3755)
* Pour les tables de la famille `MergeTree`, vous pouvez désormais spécifier une clé de tri (`ORDER BY`) et un index (`PRIMARY KEY`) distincts. La clé de tri peut être plus longue que l’index. [#3581](https://github.com/ClickHouse/ClickHouse/pull/3581)
* Ajout de la fonction de table `hdfs` et du moteur de table `HDFS` pour l’import et l’export de données vers HDFS. [chenxing-xc](https://github.com/ClickHouse/ClickHouse/pull/3617)
* Ajout de fonctions pour travailler avec le base64 : `base64Encode`, `base64Decode`, `tryBase64Decode`. [Alexander Krasheninnikov](https://github.com/ClickHouse/ClickHouse/pull/3350)
* Vous pouvez désormais utiliser un paramètre pour configurer la précision de la fonction d’agrégation `uniqCombined` (sélection du nombre de cellules HyperLogLog). [#3406](https://github.com/ClickHouse/ClickHouse/pull/3406)
* Ajout de la table `system.contributors`, qui contient les noms de tous les contributeurs ayant effectué des commits dans ClickHouse. [#3452](https://github.com/ClickHouse/ClickHouse/pull/3452)
* Ajout de la possibilité d’omettre la partition dans la requête `ALTER TABLE ... FREEZE` afin de sauvegarder toutes les partitions en une seule fois. [#3514](https://github.com/ClickHouse/ClickHouse/pull/3514)
* Ajout des fonctions `dictGet` et `dictGetOrDefault`, qui ne nécessitent pas de spécifier le type de la valeur de retour. Le type est déterminé automatiquement à partir de la description du dictionnaire. [Amos Bird](https://github.com/ClickHouse/ClickHouse/pull/3564)
* Vous pouvez désormais spécifier des commentaires pour une colonne dans la description de la table et les modifier avec `ALTER`. [#3377](https://github.com/ClickHouse/ClickHouse/pull/3377)
* La lecture est désormais prise en charge pour les tables de type `Join` avec des clés simples. [Amos Bird](https://github.com/ClickHouse/ClickHouse/pull/3728)
* Vous pouvez désormais spécifier les options `join_use_nulls`, `max_rows_in_join`, `max_bytes_in_join` et `join_overflow_mode` lors de la création d’une table de type `Join`. [Amos Bird](https://github.com/ClickHouse/ClickHouse/pull/3728)
* Ajout de la fonction `joinGet`, qui permet d’utiliser une table de type `Join` comme un dictionnaire. [Amos Bird](https://github.com/ClickHouse/ClickHouse/pull/3728)
* Ajout des colonnes `partition_key`, `sorting_key`, `primary_key` et `sampling_key` à la table `system.tables` afin de fournir des informations sur les clés de la table. [#3609](https://github.com/ClickHouse/ClickHouse/pull/3609)
* Ajout des colonnes `is_in_partition_key`, `is_in_sorting_key`, `is_in_primary_key` et `is_in_sampling_key` à la table `system.columns`. [#3609](https://github.com/ClickHouse/ClickHouse/pull/3609)
* Ajout des colonnes `min_time` et `max_time` à la table `system.parts`. Ces colonnes sont renseignées lorsque la clé de partitionnement est une expression composée de colonnes `DateTime`. [Emmanuel Donin de Rosière](https://github.com/ClickHouse/ClickHouse/pull/3800)

<div id="improvements">
  #### Améliorations :
</div>

* Le serveur n’écrit pas les fichiers de configuration prétraités dans le répertoire `/etc/clickhouse-server/`. Il les enregistre à la place dans le répertoire `preprocessed_configs` sous `path`. Cela signifie que le répertoire `/etc/clickhouse-server/` n’est pas accessible en écriture à l’utilisateur `clickhouse`, ce qui améliore la sécurité. [#2443](https://github.com/ClickHouse/ClickHouse/pull/2443)
* L’option `min_merge_bytes_to_use_direct_io` est définie par défaut sur 10 GiB. Une fusion qui crée de grandes parts de tables de la famille MergeTree sera effectuée en mode `O_DIRECT`, ce qui évite une éviction excessive du cache de pages. [#3504](https://github.com/ClickHouse/ClickHouse/pull/3504)
* Démarrage du serveur accéléré en présence d’un très grand nombre de tables. [#3398](https://github.com/ClickHouse/ClickHouse/pull/3398)
* Ajout d’un pool de connexions et de `Keep-Alive` HTTP pour les connexions entre répliques. [#3594](https://github.com/ClickHouse/ClickHouse/pull/3594)
* Si la syntaxe de la requête est invalide, le code `400 Bad Request` est renvoyé par l’interface `HTTP` (`500` était renvoyé auparavant). [31bc680a](https://github.com/ClickHouse/ClickHouse/commit/31bc680ac5f4bb1d0360a8ba4696fa84bb47d6ab)
* L’option `join_default_strictness` est définie par défaut sur `ALL` pour des raisons de compatibilité. [120e2cbe](https://github.com/ClickHouse/ClickHouse/commit/120e2cbe2ff4fbad626c28042d9b28781c805afe)
* Suppression de la journalisation vers `stderr` depuis la bibliothèque `re2` pour les expressions régulières invalides ou complexes. [#3723](https://github.com/ClickHouse/ClickHouse/pull/3723)
* Ajout, pour le moteur de table `Kafka`, de vérifications des abonnements avant de commencer la lecture depuis Kafka, ainsi que du paramètre kafka&#95;max&#95;block&#95;size pour la table. [Marek Vavruša](https://github.com/ClickHouse/ClickHouse/pull/3396)
* Les fonctions `cityHash64`, `farmHash64`, `metroHash64`, `sipHash64`, `halfMD5`, `murmurHash2_32`, `murmurHash2_64`, `murmurHash3_32` et `murmurHash3_64` fonctionnent désormais avec n’importe quel nombre d’arguments et avec des arguments sous forme de tuples. [#3451](https://github.com/ClickHouse/ClickHouse/pull/3451) [#3519](https://github.com/ClickHouse/ClickHouse/pull/3519)
* La fonction `arrayReverse` fonctionne désormais avec tous les types de tableaux. [73e3a7b6](https://github.com/ClickHouse/ClickHouse/commit/73e3a7b662161d6005e7727d8a711b930386b871)
* Ajout d’un paramètre facultatif : la taille du créneau pour la fonction `timeSlots`. [Kirill Shvakov](https://github.com/ClickHouse/ClickHouse/pull/3724)
* Pour `FULL` et `RIGHT JOIN`, le paramètre `max_block_size` est utilisé pour un flux de données non jointes provenant de la table de droite. [Amos Bird](https://github.com/ClickHouse/ClickHouse/pull/3699)
* Ajout du paramètre de ligne de commande `--secure` dans `clickhouse-benchmark` et `clickhouse-performance-test` pour activer TLS. [#3688](https://github.com/ClickHouse/ClickHouse/pull/3688) [#3690](https://github.com/ClickHouse/ClickHouse/pull/3690)
* Conversion de type lorsque la structure d’une table de type `Buffer` ne correspond pas à celle de la table de destination. [Vitaly Baranov](https://github.com/ClickHouse/ClickHouse/pull/3603)
* Ajout de l’option `tcp_keep_alive_timeout` pour activer les paquets keep-alive après la période d’inactivité spécifiée. [#3441](https://github.com/ClickHouse/ClickHouse/pull/3441)
* Suppression des guillemets inutiles autour des valeurs de la clé de partition dans la table `system.parts` si elle se compose d’une seule colonne. [#3652](https://github.com/ClickHouse/ClickHouse/pull/3652)
* La fonction modulo fonctionne avec les types de données `Date` et `DateTime`. [#3385](https://github.com/ClickHouse/ClickHouse/pull/3385)
* Ajout de synonymes pour les fonctions `POWER`, `LN`, `LCASE`, `UCASE`, `REPLACE`, `LOCATE`, `SUBSTR` et `MID`. [#3774](https://github.com/ClickHouse/ClickHouse/pull/3774) [#3763](https://github.com/ClickHouse/ClickHouse/pull/3763) Certains noms de fonctions sont insensibles à la casse afin d’assurer la compatibilité avec le standard SQL. Ajout également du sucre syntaxique `SUBSTRING(expr FROM start FOR length)` pour la compatibilité avec SQL. [#3804](https://github.com/ClickHouse/ClickHouse/pull/3804)
* Ajout de la possibilité d’utiliser `mlock` sur les pages mémoire correspondant au code exécutable de `clickhouse-server` afin d’éviter qu’il ne soit expulsé de la mémoire. Cette fonctionnalité est désactivée par défaut. [#3553](https://github.com/ClickHouse/ClickHouse/pull/3553)
* Amélioration des performances lors de la lecture avec `O_DIRECT` (lorsque l’option `min_bytes_to_use_direct_io` est activée). [#3405](https://github.com/ClickHouse/ClickHouse/pull/3405)
* Amélioration des performances de la fonction `dictGet...OrDefault` avec un argument de clé constant et un argument par défaut non constant. [Amos Bird](https://github.com/ClickHouse/ClickHouse/pull/3563)
* La fonction `firstSignificantSubdomain` traite désormais les domaines `gov`, `mil` et `edu`. [Igor Hatarist](https://github.com/ClickHouse/ClickHouse/pull/3601) Performances améliorées. [#3628](https://github.com/ClickHouse/ClickHouse/pull/3628)
* Possibilité de spécifier des variables d’environnement personnalisées pour démarrer `clickhouse-server` à l’aide du script `SYS-V init.d` en définissant `CLICKHOUSE_PROGRAM_ENV` dans `/etc/default/clickhouse`.
  [Pavlo Bashynskyi](https://github.com/ClickHouse/ClickHouse/pull/3612)
* Correction du code de retour du script d’initialisation de clickhouse-server. [#3516](https://github.com/ClickHouse/ClickHouse/pull/3516)
* La table `system.metrics` inclut désormais la métrique `VersionInteger`, et `system.build_options` inclut la ligne `VERSION_INTEGER`, qui contient la forme numérique de la version de ClickHouse, par exemple `18016000`. [#3644](https://github.com/ClickHouse/ClickHouse/pull/3644)
* Suppression de la possibilité de comparer le type `Date` à un nombre afin d’éviter des erreurs potentielles comme `date = 2018-12-17`, où les guillemets autour de la date sont omis par erreur. [#3687](https://github.com/ClickHouse/ClickHouse/pull/3687)
* Correction du comportement des fonctions avec état comme `rowNumberInAllBlocks`. Elles renvoyaient auparavant un résultat supérieur d’une unité, car elles démarraient pendant l’analyse de la requête. [Amos Bird](https://github.com/ClickHouse/ClickHouse/pull/3729)
* Si le fichier `force_restore_data` ne peut pas être supprimé, un message d’erreur s’affiche. [Amos Bird](https://github.com/ClickHouse/ClickHouse/pull/3794)

<div id="build-improvements-4">
  #### Améliorations du build :
</div>

* L’outil `pbuilder` est utilisé pour les builds. Le processus de build est presque entièrement indépendant de l’environnement de l’hôte de build.
* Un seul build est utilisé pour différentes versions d’OS. Les paquets et les binaires ont été rendus compatibles avec un large éventail de systèmes Linux.
* Ajout du paquet `clickhouse-test`. Il peut être utilisé pour exécuter des tests fonctionnels.
* L’archive source peut désormais être publiée dans le dépôt. Elle peut être utilisée pour reproduire le build sans passer par GitHub.
* Ajout d’une intégration limitée avec Travis CI. En raison des limites de temps de build dans Travis, seul le build de débogage est testé et seul un sous-ensemble limité de tests est exécuté.
* Ajout de la prise en charge de `Cap'n'Proto` dans le build par défaut.
* Changement du format des sources de la documentation, de `Restricted Text` à `Markdown`.
* Ajout de la prise en charge de `systemd` (Vladimir Smirnov). Elle est désactivée par défaut en raison d’une incompatibilité avec certaines images d’OS et peut être activée manuellement.
* Pour la génération dynamique de code, `clang` et `lld` sont intégrés au binaire `clickhouse`. Ils peuvent également être invoqués via `clickhouse clang` et `clickhouse lld`.
* Suppression de l’utilisation des extensions GNU dans le code. Activation de l’option `-Wextra`. Lors d’un build avec `clang`, la bibliothèque par défaut est `libc++` au lieu de `libstdc++`.
* Extraction des bibliothèques `clickhouse_parsers` et `clickhouse_common_io` pour accélérer les builds de divers outils.

<div id="backward-incompatible-changes-5">
  #### Changements non rétrocompatibles :
</div>

* Le format des marks dans les tables de type `Log` contenant des colonnes `Nullable` a été modifié de façon non rétrocompatible. Si vous avez de telles tables, vous devez les convertir au type `TinyLog` avant de démarrer la nouvelle version du serveur. Pour ce faire, remplacez `ENGINE = Log` par `ENGINE = TinyLog` dans le fichier `.sql` correspondant du répertoire `metadata`. Si votre table ne comporte pas de colonnes `Nullable` ou si son type n’est pas `Log`, vous n’avez rien à faire.
* Le paramètre `experimental_allow_extended_storage_definition_syntax` a été supprimé. Cette fonctionnalité est désormais activée par défaut.
* La fonction `runningIncome` a été renommée en `runningDifferenceStartingWithFirstvalue` afin d’éviter toute confusion.
* La syntaxe `FROM ARRAY JOIN arr` a été supprimée lorsque ARRAY JOIN est indiqué directement après FROM sans table (Amos Bird).
* Le format `BlockTabSeparated`, utilisé uniquement à des fins de démonstration, a été supprimé.
* Le format d’état des fonctions d’agrégation `varSamp`, `varPop`, `stddevSamp`, `stddevPop`, `covarSamp`, `covarPop`, `corr` a été modifié. Si vous avez stocké les états de ces fonctions d’agrégation dans des tables (à l’aide du type de données `AggregateFunction` ou de vues matérialisées avec les états correspondants), veuillez écrire à feedback@clickhouse.com.
* Dans les versions précédentes du serveur, il existait une fonctionnalité non documentée : si une fonction d’agrégation dépend de paramètres, vous pouviez quand même la spécifier sans paramètres dans le type de données AggregateFunction. Exemple : `AggregateFunction(quantiles, UInt64)` au lieu de `AggregateFunction(quantiles(0.5, 0.9), UInt64)`. Cette fonctionnalité a disparu. Bien qu’elle n’ait pas été documentée, nous prévoyons de la prendre à nouveau en charge dans de futures versions.
* Les types de données Enum ne peuvent pas être utilisés dans les fonctions d’agrégation min/max. Cette possibilité sera rétablie dans la prochaine version.

<div id="please-note-when-upgrading">
  #### Veuillez noter lors de la mise à niveau :
</div>

* Lors d’une rolling update sur un cluster, lorsqu’une partie des répliques exécute encore l’ancienne version de ClickHouse et l’autre la nouvelle, la réplication s’interrompt temporairement et le message `unknown parameter 'shard'` apparaît dans le journal. La réplication reprendra une fois que toutes les répliques du cluster auront été mises à jour.
* Si différentes versions de ClickHouse s’exécutent sur les serveurs du cluster, il est possible que les requêtes distribuées utilisant les fonctions suivantes produisent des résultats incorrects : `varSamp`, `varPop`, `stddevSamp`, `stddevPop`, `covarSamp`, `covarPop`, `corr`. Vous devez mettre à jour tous les nœuds du cluster.
