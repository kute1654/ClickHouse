<div id="backward-incompatible-changes">
  ## Changements non rétrocompatibles
</div>

<div id="data-type-changes">
  ### Changements des types de données
</div>

* Introduction du type `Geometry`. Prise en charge de la lecture des formats `WKB` et `WKT`. Dans les versions précédentes, le type `Geometry` était un alias de `String`, mais il s’agit désormais d’un type à part entière. [#83344](https://github.com/ClickHouse/ClickHouse/pull/83344) ([Konstantin Vedernikov](https://github.com/scanhex12)).
* Suppression du type Object obsolète. [#85718](https://github.com/ClickHouse/ClickHouse/pull/85718) ([Pavel Kruglov](https://github.com/Avogar)).
* Échappement des noms de fichiers créés pour les sous-colonnes du type Variant dans les data parts Wide des tables MergeTree. Ce changement rompt la compatibilité avec les anciennes tables utilisant les types de données Variant/Dynamic/JSON. Il corrige le stockage des types contenant des caractères spéciaux dans Variant (par exemple DateTime avec un fuseau horaire spécifique contenant `\`). L’échappement peut être désactivé en modifiant le paramètre MergeTree `escape_variant_subcolumn_filenames` (pour conserver la compatibilité, désactivez ce paramètre dans la config de MergeTree ou définissez le paramètre `compatibility` sur la version précédente avant la mise à niveau). Résout [#69590](https://github.com/ClickHouse/ClickHouse/issues/69590). [#87300](https://github.com/ClickHouse/ClickHouse/pull/87300) ([Pavel Kruglov](https://github.com/Avogar)).

<div id="query-and-function-changes">
  ### Modifications des requêtes et des fonctions
</div>

* ALTER MODIFY COLUMN exige désormais un `DEFAULT` explicite lors de la conversion de colonnes Nullable en types non Nullable. Auparavant, de telles instructions ALTER pouvaient se bloquer avec des erreurs cannot convert null to not null ; désormais, les NULL sont remplacés par l&#39;expression par défaut de la colonne. Résout [#5985](https://github.com/ClickHouse/ClickHouse/issues/5985). [#84770](https://github.com/ClickHouse/ClickHouse/pull/84770) ([Vladimir Cherkasov](https://github.com/vdimir)).
* Suppression des paramètres `allow_not_comparable_types_in_order_by`/`allow_not_comparable_types_in_comparison_functions`. Autoriser des types non comparables dans ORDER BY ou dans des fonctions de comparaison peut entraîner des erreurs logiques et des résultats inattendus. Résout [#90028](https://github.com/ClickHouse/ClickHouse/issues/90028). [#90527](https://github.com/ClickHouse/ClickHouse/pull/90527) ([Pavel Kruglov](https://github.com/Avogar)).
* Correction des fonctions bitShiftLeft et bitShiftRight afin qu&#39;elles renvoient 0 ou une valeur vide dans le cas d&#39;un décalage exactement égal à la taille du type. [#91943](https://github.com/ClickHouse/ClickHouse/pull/91943) ([Pablo Marcos](https://github.com/pamarcos)).
* Le tokenizer ngram ne renverra plus de ngrams plus courts que la longueur N avec laquelle il a été configuré. Text Search ne renverra aucune ligne lorsque les tokens de recherche sont vides. [#89757](https://github.com/ClickHouse/ClickHouse/pull/89757) ([George Larionov](https://github.com/george-larionov)).

<div id="storage-and-index-changes">
  ### Changements liés au stockage et aux index
</div>

* Suppression de la fonctionnalité obsolète `LIVE VIEW`. Si vous utilisez `LIVE VIEW`, il ne sera pas possible de mettre à niveau vers la nouvelle version. [#88706](https://github.com/ClickHouse/ClickHouse/pull/88706) ([Alexey Milovidov](https://github.com/alexey-milovidov)).
* Il est interdit de créer plusieurs disques `plain-rewritable` sur un même chemin partagé de stockage objet, car cela peut entraîner un comportement indéfini en cas de collision entre différentes transactions du stockage des métadonnées. [#89038](https://github.com/ClickHouse/ClickHouse/pull/89038) ([Mikhail Artemenko](https://github.com/Michicosun)).
* Il est désormais interdit de créer des tables `MergeTree` spéciales (telles que `ReplacingMergeTree`, `CollapsingMergeTree`, etc.) avec une clé `ORDER BY` vide, car le comportement de fusion de ces tables n&#39;est pas défini. Si vous devez malgré tout créer une telle table, activez le paramètre `allow_suspicious_primary_key`. [#91569](https://github.com/ClickHouse/ClickHouse/pull/91569) ([Anton Popov](https://github.com/CurtizJ)).
* Plusieurs corrections ont été apportées autour des index implicites. Le schéma affiché ou stocké (métadonnées de Keeper) n&#39;inclura pas les index implicites, comme ceux créés par les paramètres `add_minmax_index_for_numeric_columns` ou `add_minmax_index_for_string_columns`. Cela peut provoquer des erreurs de métadonnées lorsqu&#39;une table ReplicatedMergeTree est créée ou mise à jour dans une version plus récente alors qu&#39;une réplique existe dans une version plus ancienne. [#91429](https://github.com/ClickHouse/ClickHouse/pull/91429) ([Raúl Marín](https://github.com/Algunenano)).

<div id="settings-and-configuration-changes">
  ### Modifications des paramètres et de la configuration
</div>

* Prise en charge du balisage des exceptions dans les réponses de résultats HTTP afin de permettre aux clients d’analyser les exceptions de façon plus fiable. Résout [#75175](https://github.com/ClickHouse/ClickHouse/issues/75175). Le paramètre `http_write_exception_in_output_format` est désactivé par défaut pour garantir la cohérence entre les formats. [#88818](https://github.com/ClickHouse/ClickHouse/pull/88818) ([Kaviraj Kanagaraj](https://github.com/kavirajk)).
* Corrige la priorité des paramètres SASL du moteur Kafka. Les paramètres SASL définis au niveau de la table dans les requêtes CREATE TABLE remplacent désormais correctement les paramètres spécifiques au `consumer`/`producer` provenant des fichiers de configuration. [#89401](https://github.com/ClickHouse/ClickHouse/pull/89401) ([János Benjamin Antal](https://github.com/antaljanosbenjamin)).
* Les paramètres de configuration ACME `refresh_certificates_task_interval` ont été renommés en `refresh_certificates_task_interval_seconds`, et `refresh_certificates_before` en `refresh_certificates_before_seconds`. Le paramètre `refresh_certificates_task_interval_seconds` attend désormais une valeur en secondes. [#92211](https://github.com/ClickHouse/ClickHouse/pull/92211) ([Konstantin Bogdanov](https://github.com/thevar1able)).
* Signale la désactivation des arguments positionnels dans les projections comme un changement non rétrocompatible. En outre, introduit le paramètre `enable_positional_arguments_for_projections` afin de permettre une mise à niveau sûre d’un cluster ClickHouse lorsque des arguments positionnels sont présents dans les projections. [#92007](https://github.com/ClickHouse/ClickHouse/pull/92007) ([Dmitry Novik](https://github.com/novikd)).

<div id="client-changes">
  ### Modifications du client
</div>

* Mise à jour de clickhouse-client pour qu&#39;il renvoie un code de sortie non nul (159 - TIMEOUT&#95;EXCEEDED) lorsqu&#39;une requête dépasse le délai d&#39;attente à cause de `receive_timeout`. Auparavant, ces dépassements de délai renvoyaient le code de sortie 0 (succès), ce qui compliquait la détection de ces échecs par les scripts et les outils d&#39;automatisation. [#91432](https://github.com/ClickHouse/ClickHouse/pull/91432) ([Sav](https://github.com/sberss)).

<div id="statistics-format-changes">
  ### Modifications du format des statistiques
</div>

* Lors du passage d’une colonne de `String` à `Nullable(String)`, aucune mutation ne sera effectuée sur les données. En revanche, pour la fonction d’agrégation `uniq`, une structure de données différente est utilisée : pour une colonne nullable, `AggregateFunctionNull` sera utilisé avec un agrégateur `uniq` imbriqué. `AggregateFunctionNull` sérialisera un indicateur booléen supplémentaire. Cela rendra le fichier de statistiques incompatible. Le format des statistiques a changé ; le serveur plantera si des statistiques sont encore dans l’ancien format. Pour éviter cette exception, exécutez `ALTER TABLE [db.]table MATERIALIZE STATISTICS ALL` afin de régénérer les statistiques. [#90311](https://github.com/ClickHouse/ClickHouse/pull/90311) ([Han Fei](https://github.com/hanfei1991)).

Résout [#89282](https://github.com/ClickHouse/ClickHouse/issues/89282). [#89432](https://github.com/ClickHouse/ClickHouse/pull/89432) ([yanglongwei](https://github.com/ylw510)).

<div id="sql-and-query-features">
  ### Fonctionnalités SQL et de requête
</div>

* Ajout de l’option `allow_reentry` à la fonction d’agrégation `windowFunnel`. Lorsqu’elle est activée avec `strict_order`, elle ignore les événements qui ne respectent pas l’ordre au lieu d’interrompre l’analyse du funnel. Cela permet de gérer les parcours utilisateur avec des actualisations (A-&gt;A-&gt;B) ou des retours en arrière dans la navigation (A-&gt;B-&gt;A-&gt;C) sans sous-estimer les taux de conversion. [#86916](https://github.com/ClickHouse/ClickHouse/pull/86916) ([Lee ChaeRok](https://github.com/LeeChaeRok)).
* L’utilisateur peut désormais ajouter un nouvel argument `preprocessor` lors de la construction d’un index textuel. Cet argument est une expression arbitraire qui transforme chaque document avant la tokenization. [#88272](https://github.com/ClickHouse/ClickHouse/pull/88272) ([Jimmy Aguilar Mena](https://github.com/Ergus)).
* Ajout de la prise en charge de `LIMIT` et `OFFSET` fractionnaires pour sélectionner une fraction d’une table. Clôt [#81892](https://github.com/ClickHouse/ClickHouse/issues/81892). [#88755](https://github.com/ClickHouse/ClickHouse/pull/88755) ([Ahmed Gouda](https://github.com/0xgouda)).
* Ajout du paramètre `into_outfile_create_parent_directories` pour créer automatiquement les répertoires parents pour INTO OUTFILE, ce qui évite les erreurs lorsque les chemins de sortie n’existent pas. Résout [#88610](https://github.com/ClickHouse/ClickHouse/issues/88610). [#88795](https://github.com/ClickHouse/ClickHouse/pull/88795) ([Saksham](https://github.com/Saksham10-11)).
* Ajout du paramètre `type_json_skip_invalid_typed_paths` pour désactiver les exceptions lors des insertions/conversions de type vers le type JSON lorsque le JSON d’entrée ne peut pas être converti en chemins typés explicites dans le type JSON. Bascule alors vers la valeur null/zéro du chemin typé. Clôt [#86917](https://github.com/ClickHouse/ClickHouse/issues/86917). [#89886](https://github.com/ClickHouse/ClickHouse/pull/89886) ([Max Justus Spransy](https://github.com/maxjustus)).

[#88786](https://github.com/ClickHouse/ClickHouse/pull/88786) ([Elmi Ahmadov](https://github.com/ahmadov)).
* Comme les posting lists sont celles qui occupent le plus d’espace, leur mise en cache améliore les performances lors d’exécutions successives. [#88912](https://github.com/ClickHouse/ClickHouse/pull/88912) ([Elmi Ahmadov](https://github.com/ahmadov)).
* Optimisation des recherches répétées dans le dictionnaire inverse en effectuant des recherches plus rapides dans un ensemble précalculé de valeurs de clé possibles. Résout [#7968](https://github.com/ClickHouse/ClickHouse/issues/7968). [#88971](https://github.com/ClickHouse/ClickHouse/pull/88971) ([Nihal Z. Miaji](https://github.com/nihalzp)).

* Les utilisateurs devraient constater une latence plus faible lors de l’analyse des index lorsque de grands index `minmax` (des millions de granules) sont présents. [#90428](https://github.com/ClickHouse/ClickHouse/pull/90428) ([Shankar Iyer](https://github.com/shankar-iyer)).
