<div id="backward-incompatible-changes">
  ## Changements incompatibles avec les versions précédentes
</div>

* Le format de sortie Parquet convertit les colonnes Date et DateTime en types date/heure pris en charge par Parquet, au lieu de les écrire sous forme de nombres bruts. DateTime devient DateTime64(3) (au lieu de UInt32) ; le paramètre `output_format_parquet_datetime_as_uint32` rétablit l’ancien comportement. Date devient Date32 (au lieu de UInt16). [#70950](https://github.com/ClickHouse/ClickHouse/pull/70950) ([Michael Kolupaev](https://github.com/al13n321)).
* Par défaut, les types comparables (comme JSON/Object/AggregateFunction) ne sont plus autorisés dans ORDER BY ni dans les fonctions de comparaison `less/greater/equal/etc`. [#73276](https://github.com/ClickHouse/ClickHouse/pull/73276) ([Pavel Kruglov](https://github.com/Avogar)).
* `JSONEachRowWithProgress` écrira désormais la progression à chaque mise à jour. Dans les versions précédentes, la progression n’était affichée qu’après chaque bloc de résultat, ce qui la rendait inutile. Le mode d’affichage de la progression change : les valeurs nulles ne seront plus affichées. Gardez à l’esprit que la progression est envoyée même lorsqu’elle se produit fréquemment. Cela peut générer un volume de trafic important. Gardez également à l’esprit que la progression n’est pas écrite lorsque la sortie est compressée. Cela résout [#70800](https://github.com/ClickHouse/ClickHouse/issues/70800). [#73834](https://github.com/ClickHouse/ClickHouse/pull/73834) ([Alexey Milovidov](https://github.com/alexey-milovidov)).
* La source de dictionnaire `mysql` n’exécute plus la requête `SHOW TABLE STATUS`, car elle n’apporte aucune valeur pour les tables InnoDB, y compris sur les versions récentes de MySQL. Cela résout [#72636](https://github.com/ClickHouse/ClickHouse/issues/72636). Ce changement est rétrocompatible, mais il est classé dans cette catégorie afin que vous ayez plus de chances de le remarquer. [#73914](https://github.com/ClickHouse/ClickHouse/pull/73914) ([Alexey Milovidov](https://github.com/alexey-milovidov)).
* Les tables `Merge` unifieront la structure des tables sous-jacentes en utilisant l’union de leurs colonnes et en déduisant des types communs. Cela résout [#64864](https://github.com/ClickHouse/ClickHouse/issues/64864). Cela résout [#35307](https://github.com/ClickHouse/ClickHouse/issues/35307). Dans certains cas, ce changement peut être rétro-incompatible. Par exemple, lorsqu’il n’existe aucun type commun entre les tables, mais qu’une conversion vers le type de la première table reste possible, comme dans le cas de UInt64 et Int64, ou entre n’importe quel type numérique et String. Si vous souhaitez revenir à l’ancien comportement, définissez `merge_table_max_tables_to_look_for_schema_inference` sur `1` ou `compatibility` sur `24.12` ou une version antérieure. [#73956](https://github.com/ClickHouse/ClickHouse/pull/73956) ([Alexey Milovidov](https://github.com/alexey-milovidov)).
* Les requêtes `CHECK TABLE` nécessitent désormais un privilège `CHECK` distinct. Dans les versions précédentes, il suffisait de disposer du privilège `SHOW TABLES` pour exécuter ces requêtes. Or, une requête `CHECK TABLE` peut être coûteuse, et les limites habituelles de complexité des requêtes `SELECT` ne s’y appliquent pas. Cela pouvait entraîner un risque de DoS. [#74471](https://github.com/ClickHouse/ClickHouse/pull/74471) ([Alexey Milovidov](https://github.com/alexey-milovidov)).
* Vérifie que toutes les colonnes d’une vue matérialisée correspondent à celles de la table cible si `allow_materialized_view_with_bad_select` vaut `false`. [#74481](https://github.com/ClickHouse/ClickHouse/pull/74481) ([Christoph Wurm](https://github.com/cwurm)).
* La fonction `h3ToGeo()` renvoie désormais les résultats dans l’ordre `(lat, lon)` (qui est l’ordre standard des fonctions géométriques). Les utilisateurs qui souhaitent conserver l’ancien ordre des résultats `(lon, lat)` peuvent définir le paramètre `h3togeo_lon_lat_result_order = true`. [#74719](https://github.com/ClickHouse/ClickHouse/pull/74719) ([Manish Gill](https://github.com/mgill25)).
* Ajout des formats `JSONCompactEachRowWithProgress` et `JSONCompactStringsEachRowWithProgress`. Suite de [#69989](https://github.com/ClickHouse/ClickHouse/issues/69989). `JSONCompactWithNames` et `JSONCompactWithNamesAndTypes` ne produisent plus &quot;totals&quot; — il s’agissait apparemment d’une erreur d’implémentation. [#75037](https://github.com/ClickHouse/ClickHouse/pull/75037) ([Alexey Milovidov](https://github.com/alexey-milovidov)).
* La valeur par défaut de `format_alter_operations_with_parentheses` passe à true afin de lever toute ambiguïté dans la liste des commandes alter (voir https://github.com/ClickHouse/ClickHouse/pull/59532). Cela casse la réplication avec les clusters antérieurs à 24.3. Si vous mettez à niveau un cluster utilisant des versions plus anciennes, désactivez ce paramètre dans la configuration du serveur ou mettez d’abord à niveau vers 24.3. [#75302](https://github.com/ClickHouse/ClickHouse/pull/75302) ([Raúl Marín](https://github.com/Algunenano)).
* Interdit `TRUNCATE DATABASE` pour les bases de données répliquées. [#76651](https://github.com/ClickHouse/ClickHouse/pull/76651) ([Bharat Nallan](https://github.com/bharatnc)).
* Désactive les répliques parallèles par défaut lorsque l’analyzer est désactivé, quel que soit le paramètre `compatibility`. Il reste possible de modifier ce comportement en définissant explicitement `parallel_replicas_only_with_analyzer` sur `false`. [#77115](https://github.com/ClickHouse/ClickHouse/pull/77115) ([Igor Nikonov](https://github.com/devcrafter)).
* Il n’est plus possible d’utiliser `NaN` ou `inf` comme valeurs flottantes pour les paramètres. [#77546](https://github.com/ClickHouse/ClickHouse/pull/77546) ([Yarik Briukhovetskyi](https://github.com/yariks5s)).
* Corrige les cas où `dateTrunc` est utilisé avec des arguments `Date`/`DateTime` négatifs. [#77622](https://github.com/ClickHouse/ClickHouse/pull/77622) ([Yarik Briukhovetskyi](https://github.com/yariks5s)).
* L’ancienne intégration MongoDB a été supprimée. Le paramètre du serveur `use_legacy_mongodb_integration` est devenu obsolète et n’a désormais plus aucun effet. [#77895](https://github.com/ClickHouse/ClickHouse/pull/77895) ([Robert Schulze](https://github.com/rschu1ze)).
* Améliore la validation de SummingMergeTree afin d’ignorer l’agrégation des colonnes utilisées dans les clés de partitionnement ou de tri. [#78022](https://github.com/ClickHouse/ClickHouse/pull/78022) ([Pervakov Grigorii](https://github.com/GrigoryPervakov)).

<div id="new-features">
  ## Nouvelles fonctionnalités
</div>

* Ajout d’un cache en mémoire pour les granules désérialisés des index de saut. Cela devrait accélérer les requêtes répétées qui utilisent des index de saut. La taille du nouveau cache est contrôlée par les paramètres serveur `skipping_index_cache_size` et `skipping_index_cache_max_entries`. La motivation initiale de ce cache était les index de similarité vectorielle, qui sont désormais beaucoup plus rapides. [#70102](https://github.com/ClickHouse/ClickHouse/pull/70102) ([Robert Schulze](https://github.com/rschu1ze)).
* Nouvelle implémentation du Userspace Page Cache, qui permet de mettre les données en cache dans la mémoire du processus au lieu de s’appuyer sur le cache de pages de l’OS. Elle est utile lorsque les données sont stockées sur un système de fichiers virtuel distant sans bénéficier du cache du système de fichiers local. [#70509](https://github.com/ClickHouse/ClickHouse/pull/70509) ([Michael Kolupaev](https://github.com/al13n321)).
* Ajout d’un paramètre pour interroger les tables Iceberg à un timestamp spécifique. [#71072](https://github.com/ClickHouse/ClickHouse/pull/71072) ([Brett Hoerner](https://github.com/bretthoerner)).
* Implémentation de l’élagage de partitions pour les tables Iceberg lors des opérations de partitionnement par transformation liées au temps dans Iceberg. [#72044](https://github.com/ClickHouse/ClickHouse/pull/72044) ([Daniil Ivanik](https://github.com/divanik)).
* Ajout de la possibilité de créer par défaut des index min-max (de saut) pour les colonnes gérées par MergeTree à l’aide des paramètres `enable_minmax_index_for_all_numeric_columns` (pour les colonnes numériques) et `enable_minmax_index_for_all_string_columns` (pour les colonnes de type chaîne). Pour l’instant, les deux paramètres sont désactivés, il n’y a donc encore aucun changement de comportement. [#72090](https://github.com/ClickHouse/ClickHouse/pull/72090) ([Smita Kulkarni](https://github.com/SmitaRKulkarni)).
* Ajout de la fonction d’agrégation `sequenceMatchEvents`, qui renvoie les timestamps des événements correspondants pour la plus longue chaîne d’événements d’un motif. [#72349](https://github.com/ClickHouse/ClickHouse/pull/72349) ([UnamedRus](https://github.com/UnamedRus)).
* Les instructions `SELECT` et `VIEW` prennent désormais en charge les alias, par ex. `SELECT b FROM (SELECT number, number*2 FROM numbers(2)) AS x (a, b);`. Cela permet à la requête TPC-H 15 de s’exécuter sans modification. [#72480](https://github.com/ClickHouse/ClickHouse/pull/72480) ([Yarik Briukhovetskyi](https://github.com/yariks5s)).
* Ajout d’un nouveau paramètre `enable_adaptive_memory_spill_scheduler` qui permet à plusieurs grace JOIN dans une même requête de surveiller leur empreinte mémoire combinée et de déclencher de manière adaptative le déversement vers un stockage externe afin d’éviter MEMORY&#95;LIMIT&#95;EXCEEDED. [#72728](https://github.com/ClickHouse/ClickHouse/pull/72728) ([lgbo](https://github.com/lgbo-ustc)).
* Ajout de la fonction `arrayNormalizedGini`. [#72823](https://github.com/ClickHouse/ClickHouse/pull/72823) ([flynn](https://github.com/ucasfl)).
* Prise en charge des types de données Decimal à faible cardinalité, correction de [#72256](https://github.com/ClickHouse/ClickHouse/issues/72256). [#72833](https://github.com/ClickHouse/ClickHouse/pull/72833) ([zhanglistar](https://github.com/zhanglistar)).
* Lorsque `min_age_to_force_merge_seconds` et `min_age_to_force_merge_on_partition_only` sont tous deux activés, la fusion des parts ignorera la limite maximale en octets. [#73656](https://github.com/ClickHouse/ClickHouse/pull/73656) ([Kai Zhu](https://github.com/nauu)).
* Prise en charge de la lecture des valeurs HALF&#95;FLOAT depuis Apache Arrow/Parquet/ORC (elles sont lues en Float32). Cela corrige [#72960](https://github.com/ClickHouse/ClickHouse/issues/72960). Gardez à l’esprit que le half float IEEE-754 n’est pas identique à BFloat16. Corrige [#73835](https://github.com/ClickHouse/ClickHouse/issues/73835). [#73836](https://github.com/ClickHouse/ClickHouse/pull/73836) ([Alexey Milovidov](https://github.com/alexey-milovidov)).
* La table `system.trace_log` contiendra deux nouvelles colonnes, `symbols` et `lines`, contenant une stack trace symbolisée. Cela facilite la collecte et l’export des informations de profilage. Ce comportement est contrôlé par la valeur de configuration du serveur `symbolize` dans `trace_log` et est activé par défaut. [#73896](https://github.com/ClickHouse/ClickHouse/pull/73896) ([Alexey Milovidov](https://github.com/alexey-milovidov)).
* Ajout d’une nouvelle fonction, `generateSerialID`, qui permet de générer des nombres auto-incrémentés dans les tables. Suite de [#64310](https://github.com/ClickHouse/ClickHouse/issues/64310) par [kazalika](https://github.com/kazalika). Cela clôt [#62485](https://github.com/ClickHouse/ClickHouse/issues/62485). [#73950](https://github.com/ClickHouse/ClickHouse/pull/73950) ([Alexey Milovidov](https://github.com/alexey-milovidov)).
* Ajout de la syntaxe `query1 PARALLEL WITH query2 PARALLEL WITH query3 ... PARALLEL WITH queryN`. Cela signifie que les sous-requêtes `{query1, query2, ... queryN}` peuvent s’exécuter en parallèle les unes par rapport aux autres (et c’est préférable). [#73983](https://github.com/ClickHouse/ClickHouse/pull/73983) ([Vitaly Baranov](https://github.com/vitlibar)).
* Désormais, l’interface Play affiche une barre de progression pendant l’exécution des requêtes. Elle permet d’annuler les requêtes. Elle affiche le nombre total d’enregistrements ainsi que des informations détaillées sur la vitesse. La table peut être affichée de manière incrémentale dès l’arrivée des données. La compression HTTP est activée. Le rendu de la table est devenu plus rapide. L’en-tête de la table reste désormais fixe. Il est possible de sélectionner des cellules et de s’y déplacer avec les touches fléchées. Correction d’un problème où le contour de la cellule sélectionnée la faisait paraître plus petite. Les cellules ne s’agrandissent plus au survol de la souris, mais uniquement lors de la sélection. Le moment où arrêter le rendu des données entrantes est désormais décidé côté client plutôt que côté serveur. Les groupes de chiffres dans les nombres sont mis en évidence. L’apparence générale a été revue et est devenue plus marquée. L’interface vérifie si le serveur est joignable ainsi que la validité des identifiants, puis affiche la version du serveur et sa durée de fonctionnement. L’icône de cloud a un contour dans toutes les polices, y compris dans Safari. Les grands entiers dans les types de données imbriqués seront mieux affichés. `inf/nan` sera affiché correctement. Les types de données seront affichés au survol d’un en-tête de colonne. [#74204](https://github.com/ClickHouse/ClickHouse/pull/74204) ([Alexey Milovidov](https://github.com/alexey-milovidov)).
* Ajout de la possibilité de créer par défaut des index min-max (skipping) pour les colonnes gérées par MergeTree à l’aide des paramètres `add_minmax_index_for_numeric_columns` (pour les colonnes numériques) et `add_minmax_index_for_string_columns` (pour les colonnes de type chaîne). Pour l’instant, ces deux paramètres sont désactivés, il n’y a donc pas encore de changement de comportement. [#74266](https://github.com/ClickHouse/ClickHouse/pull/74266) ([Smita Kulkarni](https://github.com/SmitaRKulkarni)).
* Ajout des champs `script_query_number` et `script_line_number` à `system.query_log`, à ClientInfo dans le native protocol, ainsi qu’aux journaux du serveur. Cela clôt [#67542](https://github.com/ClickHouse/ClickHouse/issues/67542). Merci à [pinsvin00](https://github.com/pinsvin00) d’avoir lancé cette fonctionnalité plus tôt dans [#68133](https://github.com/ClickHouse/ClickHouse/issues/68133). [#74477](https://github.com/ClickHouse/ClickHouse/pull/74477) ([Alexey Milovidov](https://github.com/alexey-milovidov)).
* Ajout de la prise en charge de l’opérateur moins pour DateTime64, afin d’autoriser la soustraction entre des valeurs DateTime64, ainsi qu’entre des valeurs DateTime. [#74482](https://github.com/ClickHouse/ClickHouse/pull/74482) ([Li Yin](https://github.com/liyinsg)).
* Prise en charge du moteur de table DeltaLake pour AzureBlobStorage. Corrige [#68043](https://github.com/ClickHouse/ClickHouse/issues/68043). [#74541](https://github.com/ClickHouse/ClickHouse/pull/74541) ([Smita Kulkarni](https://github.com/SmitaRKulkarni)).
* Ajout du paramètre bind&#95;host pour définir l’adresse IP source des connexions du clickhouse client. [#74741](https://github.com/ClickHouse/ClickHouse/pull/74741) ([Todd Yocum](https://github.com/toddyocum)).
* Ajout de la possibilité d’appliquer immédiatement, pendant l’exécution des requêtes `SELECT` et juste après leur soumission, des mutations non terminées (non matérialisées par un processus en arrière-plan). Cela peut être activé en définissant `apply_mutations_on_fly`. [#74877](https://github.com/ClickHouse/ClickHouse/pull/74877) ([Anton Popov](https://github.com/CurtizJ)).
* Correction de certains cas auparavant inattendus où les arguments DateTime de `toStartOfInterval` sont négatifs. Cela a été fait en implémentant une nouvelle fonction appelée toStartOfIntervalAllowNegative, qui fait globalement la même chose mais ne renvoie que des valeurs Date32/DateTime64. [#74933](https://github.com/ClickHouse/ClickHouse/pull/74933) ([Yarik Briukhovetskyi](https://github.com/yariks5s)).
* Une nouvelle fonction initialQueryStartTime a été ajoutée. Elle renvoie l’heure de début de la requête en cours. La valeur est identique sur tous les shards lors d’une distributed query. [#75087](https://github.com/ClickHouse/ClickHouse/pull/75087) ([Roman Lomonosov](https://github.com/lomik)).
* Introduction de parametrized&#95;view&#95;parameters dans system.tables. Closes https://github.com/clickhouse/clickhouse/issues/66756. [#75112](https://github.com/ClickHouse/ClickHouse/pull/75112) ([NamNguyenHoai](https://github.com/NamHoaiNguyen)).
* Possibilité de modifier un database comment. Closes [#73351](https://github.com/ClickHouse/ClickHouse/issues/73351) ### Entrée de documentation pour les changements visibles par l’utilisateur. [#75622](https://github.com/ClickHouse/ClickHouse/pull/75622) ([NamNguyenHoai](https://github.com/NamHoaiNguyen)).
* Ajout de la possibilité d’utiliser ATTACH sur des tables sans couche database (évite le contournement par UUID). [#75788](https://github.com/ClickHouse/ClickHouse/pull/75788) ([Azat Khuzhin](https://github.com/azat)).
* Ajout du server setting `concurrent_threads_scheduler`, qui régit la manière dont les CPU slots sont répartis entre les requêtes concurrentes. Il peut être défini sur `round_robin` (comportement précédent) ou `fair_round_robin` pour corriger le problème de répartition inéquitable du CPU entre les INSERT et les SELECT. [#75949](https://github.com/ClickHouse/ClickHouse/pull/75949) ([Sergei Trifonov](https://github.com/serxa)).
* Restauration du codec QPL, qui avait été supprimé dans la v24.10 en raison de problèmes de licence. [#76021](https://github.com/ClickHouse/ClickHouse/pull/76021) ([Konstantin Bogdanov](https://github.com/thevar1able)).
* Ajout de la fonction `arraySymmetricDifference`. Elle renvoie tous les éléments de plusieurs arguments Array qui ne sont pas présents dans tous les arguments. Example : `SELECT arraySymmetricDifference([1, 2], [2, 3])` renvoie `[1, 3]`. (issue [#61673](https://github.com/ClickHouse/ClickHouse/issues/61673)). [#76231](https://github.com/ClickHouse/ClickHouse/pull/76231) ([Filipp Abapolov](https://github.com/pheepa)).
* Ajout de la fonction d’agrégation `estimatecompressionratio` - voir [#70801](https://github.com/ClickHouse/ClickHouse/issues/70801). [#76661](https://github.com/ClickHouse/ClickHouse/pull/76661) ([Tariq Almawash](https://github.com/talmawash)).
* Les profile events `FilterTransformPassedRows` et `FilterTransformPassedBytes` afficheront le nombre de rows et d’octets filtrés pendant l’exécution de la requête. [#76662](https://github.com/ClickHouse/ClickHouse/pull/76662) ([Onkar Deshpande](https://github.com/onkar)).
* Ajout de la hash function keccak256, couramment utilisée dans les implémentations blockchain, en particulier dans les systèmes basés sur l’EVM. [#76669](https://github.com/ClickHouse/ClickHouse/pull/76669) ([Arnaud Briche](https://github.com/arnaudbriche)).
* Scram SHA256 &amp; mise à jour de l’authentification wire de Postgres. [#76839](https://github.com/ClickHouse/ClickHouse/pull/76839) ([scanhex12](https://github.com/scanhex12)).
* Cette fonctionnalité ajoute la possibilité de définir une liste de headers transmis depuis les headers de la requête client vers l’authentificateur http externe. [#77054](https://github.com/ClickHouse/ClickHouse/pull/77054) ([inv2004](https://github.com/inv2004)).
* Prise en charge de `IcebergMetadataFilesCache`, qui stocke les fichiers manifestes/la liste et metadata.json dans un cache unique. [#77156](https://github.com/ClickHouse/ClickHouse/pull/77156) ([Han Fei](https://github.com/hanfei1991)).
* Ajout des fonctions `arrayLevenshteinDistance`, `arrayLevenshteinDistanceWeighted` et `arraySimilarity`. [#77187](https://github.com/ClickHouse/ClickHouse/pull/77187) ([Mikhail f. Shiryaev](https://github.com/Felixoid)).
* Ajout de trois nouvelles fonctions : `icebergTruncate` conformément à la spécification https://iceberg.apache.org/spec/#truncate-transform-details, `toYearNumSinceEpoch` et `toMonthNumSinceEpoch`. Prise en charge de la transformation `truncate` dans le partition pruning pour le moteur `Iceberg`. [#77403](https://github.com/ClickHouse/ClickHouse/pull/77403) ([alesapin](https://github.com/alesapin)).
* Permet à l&#39;utilisateur d&#39;interroger l&#39;état d&#39;une table Iceberg tel qu&#39;il existait à un moment antérieur. [#77439](https://github.com/ClickHouse/ClickHouse/pull/77439) ([Daniil Ivanik](https://github.com/divanik)).
* Ajout de l&#39;ordonnancement des CPU slots pour les workloads, voir https://clickhouse.com/docs/operations/workload-scheduling#cpu&#95;scheduling pour plus de détails. [#77595](https://github.com/ClickHouse/ClickHouse/pull/77595) ([Sergei Trifonov](https://github.com/serxa)).
* La fonction `hasAll()` peut désormais tirer parti des skipping indices plein texte tokenbf&#95;v1 et ngrambf&#95;v1. [#77662](https://github.com/ClickHouse/ClickHouse/pull/77662) ([UnamedRus](https://github.com/UnamedRus)).
* Le type de données `JSON` est prêt pour la production. Voir https://jsonbench.com/. Les types de données `Dynamic` et `Varaint` sont prêts pour la production. [#77785](https://github.com/ClickHouse/ClickHouse/pull/77785) ([Alexey Milovidov](https://github.com/alexey-milovidov)).
* Ajout d&#39;un cache en mémoire pour les vector similarity indexes désérialisés. Cela devrait accélérer les requêtes répétées de recherche approximative des plus proches voisins (ANN). La taille du nouveau cache est contrôlée par les server settings `vector_similarity_index_cache_size` et `vector_similarity_index_cache_max_entries`. Cette fonctionnalité remplace le cache de skipping index des versions antérieures. [#77905](https://github.com/ClickHouse/ClickHouse/pull/77905) ([Shankar Iyer](https://github.com/shankar-iyer)).
* Ajout des fonctions sparseGrams et sparseGramsHashes, avec leurs versions UTF8. Auteur : [scanhex12](https://github.com/scanhex12). [#78176](https://github.com/ClickHouse/ClickHouse/pull/78176) ([Pervakov Grigorii](https://github.com/GrigoryPervakov)).
* Introduction de la fonction `toInterval`. Cette fonction accepte 2 arguments (value et unit) et convertit la valeur vers un type `Interval` spécifique. [#78723](https://github.com/ClickHouse/ClickHouse/pull/78723) ([Andrew Davis](https://github.com/pulpdrew)).

  ## Fonctionnalités expérimentales

* Ajoute la prise en charge [d’Unity Catalog](https://www.databricks.com/product/unity-catalog) pour les tables DeltaLake sur AWS S3 et le système de fichiers local. [#76988](https://github.com/ClickHouse/ClickHouse/pull/76988) ([alesapin](https://github.com/alesapin)).

<div id="performance-improvements">
  ## Améliorations des performances
</div>

* Optimise les performances avec la projection différée afin d’éviter de lire les colonnes inutilisées. [#55518](https://github.com/ClickHouse/ClickHouse/pull/55518) ([Xiaozhe Yu](https://github.com/wudidapaopao)).
* Compare désormais d’abord les lignes en s’appuyant sur les colonnes les plus susceptibles d’être différentes. [#63780](https://github.com/ClickHouse/ClickHouse/pull/63780) ([UnamedRus](https://github.com/UnamedRus)).
* Optimise le format d’entrée RowBinary. Résout [#63805](https://github.com/ClickHouse/ClickHouse/issues/63805). [#65059](https://github.com/ClickHouse/ClickHouse/pull/65059) ([Pavel Kruglov](https://github.com/Avogar)).
