يساعدك هذا الدليل على ترحيل شيفرة pandas الحالية إلى DataStore لتحسين الأداء مع الحفاظ على التوافق.

<div id="one-line">
  ## الترحيل بسطر واحد
</div>

أبسط طريقة للترحيل هي تغيير سطر الاستيراد:

```python
# Before (pandas)
import pandas as pd

# After (DataStore)
from chdb import datastore as pd
```

هذا كل ما في الأمر! يعمل معظم كود pandas كما هو.

<div id="step-by-step">
  ## الترحيل خطوة بخطوة
</div>

<Steps>
  <Step>
    ### تثبيت chDB

    ```bash
    pip install "chdb>=4.0"
    ```
  </Step>

  <Step>
    ### عدّل عبارة الاستيراد

    ```python
    # عدّل هذا:
    import pandas as pd

    # إلى هذا:
    from chdb import datastore as pd
    ```
  </Step>

  <Step>
    ### اختبر الشيفرة البرمجية

    شغّل الشيفرة الحالية. تعمل معظم العمليات كما هي:

    ```python
    from chdb import datastore as pd

    # كل هذه تعمل بالطريقة نفسها
    df = pd.read_csv("data.csv")
    result = df[df['age'] > 25]
    grouped = df.groupby('city')['salary'].mean()
    df.to_csv("output.csv")
    ```
  </Step>

  <Step>
    ### تعامل مع الاختلافات

    يختلف سلوك بعض العمليات. راجع [أبرز الاختلافات](#differences) أدناه.
  </Step>
</Steps>

</div>


</div>

```python

```

<div id="differences">
  ## الاختلافات الرئيسية
</div>

<div id="lazy">
  ### 1. التقييم مؤجل التنفيذ
</div>

تتّسم عمليات DataStore بالتقييم مؤجل التنفيذ، فهي لا تُنفَّذ حتى تصبح النتائج مطلوبة.

**pandas:**

```python
# Executes immediately
result = df[df['age'] > 25]
print(type(result))  # pandas.DataFrame
```

**DataStore:**

```python
# Builds query, doesn't execute yet
result = ds[ds['age'] > 25]
print(type(result))  # DataStore (lazy)

# Executes when you need the data
print(result)        # Triggers execution
df = result.to_df()  # Triggers execution
```

<div id="return-types">
  ### 2. أنواع القيم المعادة
</div>

| العملية           | ما يعيده pandas | ما يعيده DataStore        |
| ----------------- | --------------- | ------------------------- |
| `df['col']`       | Series          | ColumnExpr (مؤجل التنفيذ) |
| `df[['a', 'b']]`  | DataFrame       | DataStore (مؤجل التنفيذ)  |
| `df[condition]`   | DataFrame       | DataStore (مؤجل التنفيذ)  |
| `df.groupby('x')` | GroupBy         | LazyGroupBy               |

<div id="no-inplace">
  ### 3. لا توجد معلمة inplace
</div>

لا يدعم DataStore المعلمة `inplace=True`. استخدم دائمًا القيمة المُعادة:

**pandas:**

```python
df.drop(columns=['col'], inplace=True)
```

**DataStore:**

```python
ds = ds.drop(columns=['col'])  # Assign the result
```

<div id="comparing">
  ### 4. مقارنة DataStores
</div>

لا يتعرّف pandas على كائنات DataStore، لذا استخدِم `to_pandas()` للمقارنة:

```python
# This may not work as expected
df == ds  # pandas doesn't know DataStore

# Do this instead
df.equals(ds.to_pandas())
```

<div id="row-order">
  ### 5. ترتيب الصفوف
</div>

قد لا يحافظ DataStore على ترتيب الصفوف في مصادر الملفات (مثل قواعد بيانات SQL). استخدم فرزًا صريحًا:

```python
# pandas preserves order
df = pd.read_csv("data.csv")

# DataStore - use sort for guaranteed order
ds = pd.read_csv("data.csv")
ds = ds.sort('id')  # Explicit ordering
```

<div id="patterns">
  ## أنماط الترحيل
</div>

<div id="pattern-1">
  ### النمط 1: القراءة-التحليل-الكتابة
</div>

```python
# pandas
import pandas as pd
df = pd.read_csv("data.csv")
result = df[df['amount'] > 100].groupby('category')['amount'].sum()
result.to_csv("output.csv")

# DataStore - same code works!
from chdb import datastore as pd
df = pd.read_csv("data.csv")
result = df[df['amount'] > 100].groupby('category')['amount'].sum()
result.to_csv("output.csv")
```

<div id="pattern-2">
  ### النمط 2: DataFrame مع عمليات pandas
</div>

إذا كنت بحاجة إلى إمكانات خاصة بـ pandas، فحوِّل في النهاية:

```python
from chdb import datastore as pd

# Fast DataStore operations
ds = pd.read_csv("large_data.csv")
ds = ds.filter(ds['date'] >= '2024-01-01')
ds = ds.filter(ds['amount'] > 100)

# Convert to pandas for specific features
df = ds.to_df()
df_pivoted = df.pivot_table(...)  # pandas-specific
```

<div id="pattern-3">
  ### النمط 3: سير عمل مختلط
</div>

```python
from chdb import datastore as pd
import pandas

# Start with DataStore for fast filtering
ds = pd.read_csv("huge_file.csv")  # 10M rows
ds = ds.filter(ds['year'] == 2024)  # Fast SQL filter
ds = ds.select('col1', 'col2', 'col3')  # Column pruning

# Convert for pandas-specific operations
df = ds.to_df()  # Now only ~100K rows
result = df.apply(complex_custom_function)  # pandas
```

<div id="performance">
  ## مقارنة الأداء
</div>

يُعد DataStore أسرع بكثير عند التعامل مع مجموعات البيانات الكبيرة:

| العملية          | pandas  | DataStore | مقدار التسريع |
| ---------------- | ------- | --------- | ------------- |
| GroupBy count    | 347ms   | 17ms      | **19.93x**    |
| مسار معالجة معقد | 2,047ms | 380ms     | **5.39x**     |
| تصفية+فرز+Head   | 1,537ms | 350ms     | **4.40x**     |
| GroupBy agg      | 406ms   | 141ms     | **2.88x**     |

*اختبار أداء على 10M صفوف*

<div id="troubleshooting">
  ## استكشاف أخطاء الترحيل وإصلاحها
</div>

<div id="issue-op">
  ### المشكلة: العملية لا تعمل
</div>

قد لا تكون بعض العمليات في pandas مدعومة. تحقّق مما يلي:

1. هل العملية مُدرجة في [قائمة التوافق](/ar/products/chdb/datastore/pandas-compat)؟
2. جرّب التحويل إلى pandas أولًا: `ds.to_df().operation()`

<div id="issue-results">
  ### المشكلة: نتائج مختلفة
</div>

فعِّل تسجيل تصحيح الأخطاء لفهم ما يحدث:

```python
from chdb.datastore.config import config
config.enable_debug()

# View the SQL being generated
ds.filter(ds['x'] > 10).explain()
```

<div id="issue-slow">
  ### المشكلة: بطء الأداء
</div>

تحقق من نمط التنفيذ:

```python
# Bad: Multiple small executions
for i in range(1000):
    result = ds.filter(ds['id'] == i).to_df()

# Good: Single execution
result = ds.filter(ds['id'].isin(ids)).to_df()
```

<div id="issue-types">
  ### المشكلة: عدم تطابق الأنواع
</div>

قد يستنتج DataStore الأنواع بصورة مختلفة:

```python
# Check types
print(ds.dtypes)

# Force conversion
ds['col'] = ds['col'].astype('int64')
```

<div id="week-1">
  ### الأسبوع الأول: اختبار التوافق

```python
# Keep both imports
import pandas as pd
from chdb import datastore as ds

pdf = pd.read_csv("data.csv")
dsf = ds.read_csv("data.csv")

# Verify they match
assert pdf.equals(dsf.to_pandas())
```

<div id="week-2">
  ### الأسبوع 2: حوّل البرامج النصية البسيطة
</div>

* لا تستخدم دوال apply مخصّصة

<div id="week-3">
  ### الأسبوع الثالث: التعامل مع الحالات المعقدة
</div>

بالنسبة للبرامج النصية التي تتضمن دوال مخصّصة:

```python
from chdb import datastore as pd

# Let DataStore handle the heavy lifting
ds = pd.read_csv("data.csv")
ds = ds.filter(ds['year'] == 2024)  # SQL

# Convert for custom work
df = ds.to_df()
result = df.apply(my_custom_function)
```

<div id="faq-both">
  ### هل يمكنني استخدام كلٍّ من pandas وDataStore؟
</div>

نعم! يمكنك التنقّل بينهما بحرية:

```python
from chdb import datastore as ds
import pandas as pd

# DataStore to pandas
df = ds_result.to_pandas()

# pandas to DataStore  
ds = ds.DataFrame(pd_result)
```

<div id="faq-tests">
  ### هل ستظل اختباراتي تنجح؟
</div>

من المفترض أن تنجح معظم الاختبارات. أما اختبارات المقارنة، فحوِّلها إلى pandas:

```python
def test_my_function():
    result = my_function()
    expected = pd.DataFrame(...)
    pd.testing.assert_frame_equal(result.to_pandas(), expected)
```

<div id="faq-jupyter">
  ### هل يمكنني استخدام DataStore في Jupyter؟

نعم! يعمل DataStore في دفاتر ملاحظات Jupyter:

```python
from chdb import datastore as pd

ds = pd.read_csv("data.csv")
ds.head()  # Displays nicely in Jupyter
```
