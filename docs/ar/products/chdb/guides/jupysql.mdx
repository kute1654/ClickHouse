<div id="setup">
  ## الإعداد
</div>

لنبدأ أولًا بإنشاء بيئة افتراضية:

```bash
python -m venv .venv
source .venv/bin/activate
```

ثم سنقوم بتثبيت JupySQL وIPython وJupyter Lab:

```bash
pip install jupysql ipython jupyterlab
```

يمكننا استخدام JupySQL في IPython، ويمكننا تشغيله بتنفيذ:

```bash
ipython
```

أو في Jupyter Lab، عبر تشغيل:

```bash
jupyter lab
```

<Note>
  إذا كنت تستخدم Jupyter Lab، فستحتاج إلى إنشاء دفتر قبل متابعة بقية الدليل.
</Note>

<div id="downloading-a-dataset">
  ## تنزيل مجموعة بيانات
</div>

سنستخدم إحدى مجموعات بيانات [tennis&#95;atp الخاصة بـ Jeff Sackmann](https://github.com/JeffSackmann/tennis_atp)، والتي تتضمن بيانات وصفية عن اللاعبين وتصنيفاتهم بمرور الوقت.
لنبدأ بتنزيل ملفات التصنيف:

```python
from urllib.request import urlretrieve
```

```python
files = ['00s', '10s', '20s', '70s', '80s', '90s', 'current']
base = "https://raw.githubusercontent.com/JeffSackmann/tennis_atp/master"
for file in files:
  _ = urlretrieve(
    f"{base}/atp_rankings_{file}.csv",
    f"atp_rankings_{file}.csv",
  )
```

<div id="configuring-chdb-and-jupysql">
  ## إعداد chDB وJupySQL
</div>

بعد ذلك، لنستورد الوحدة `dbapi` الخاصة بـ chDB:

```python
from chdb import dbapi
```

وسننشىء اتصالًا بـ chDB.
ستُحفَظ أي بيانات نُخزّنها بشكل دائم في المجلد `atp.chdb`:

```python
conn = dbapi.connect(path="atp.chdb")
```

لنحمّل الآن الأمر السحري `sql` وننشئ اتصالًا بـ chDB:

```python
%load_ext sql
%sql conn --alias chdb
```

بعد ذلك، سنعرض حدّ العرض كي لا تُقتطع نتائج الاستعلامات:

```python
%config SqlMagic.displaylimit = None
```

## الاستعلام عن البيانات في ملفات CSV

نزّلنا مجموعة من الملفات ذات البادئة `atp_rankings`.
لنستخدم عبارة `DESCRIBE` لفهم المخطط:

```python
%%sql
DESCRIBE file('atp_rankings*.csv')
SETTINGS describe_compact_output=1,
         schema_inference_make_columns_nullable=0
```

```text
|     name     |  type |
+--------------+-------+
| ranking_date | Int64 |
|     rank     | Int64 |

```

يمكننا أيضًا كتابة استعلام `SELECT` مباشرةً على هذه الملفات لمعرفة شكل البيانات:

```python
%sql SELECT * FROM file('atp_rankings*.csv') LIMIT 1
```

<div id="querying-data-in-csv-files">
  ## استيراد ملفات CSV إلى chDB
</div>

سنخزّن الآن البيانات من ملفات CSV هذه في جدول.
لا تحفظ قاعدة البيانات الافتراضية البيانات على القرص، لذا نحتاج أولًا إلى إنشاء قاعدة بيانات أخرى:

```python
%sql CREATE DATABASE atp
```

والآن سننشئ جدولًا باسم `rankings`، وسيُستمد مخططه من بنية البيانات في ملفات CSV:

```python
%%sql
CREATE TABLE atp.rankings
ENGINE=MergeTree
ORDER BY ranking_date AS
SELECT * REPLACE (
  toDate(parseDateTime32BestEffort(toString(ranking_date))) AS ranking_date
)
FROM file('atp_rankings*.csv')
SETTINGS schema_inference_make_columns_nullable=0
```

لنتحقق سريعًا من البيانات في جدولنا:

```python
%sql SELECT * FROM atp.rankings LIMIT 10
```

```text
+--------------+------+--------+--------+
|  2000-01-10  |  1   | 101736 |  4135  |

|  2000-01-10  |  3   | 101948 |  2419  |

+--------------+------+--------+--------+
```

يبدو هذا جيدًا - فالنتيجة، كما هو متوقع، هي نفسها عند الاستعلام من ملفات CSV مباشرةً.

سنتبع العملية نفسها مع البيانات الوصفية للاعب.
هذه المرة، توجد جميع البيانات في ملف CSV واحد، لذا لننزّل هذا الملف:

```python
_ = urlretrieve(
    f"{base}/atp_players.csv",
    "atp_players.csv",
)
```

ثم أنشئ جدولًا باسم `players` استنادًا إلى محتويات ملف CSV.
وسنُهيِّئ أيضًا الحقل `dob` ليصبح من النوع `Date32`.

> في ClickHouse، لا يدعم النوع `Date` إلا التواريخ بدءًا من عام 1970. وبما أن العمود `dob` يحتوي على تواريخ تسبق عام 1970، فسنستخدم النوع `Date32` بدلًا منه.

```python
%%sql
CREATE TABLE atp.players
Engine=MergeTree
ORDER BY player_id AS
SELECT * REPLACE (
  makeDate32(
    toInt32OrNull(substring(toString(dob), 1, 4)),
    toInt32OrNull(substring(toString(dob), 5, 2)),
    toInt32OrNull(substring(toString(dob), 7, 2))
  )::Nullable(Date32) AS dob
)
FROM file('atp_players.csv')
SETTINGS schema_inference_make_columns_nullable=0
```

بعد اكتمال التشغيل، يمكننا إلقاء نظرة على البيانات التي استوردناها:

```python
%sql SELECT * FROM atp.players LIMIT 10
```

<div id="importing-csv-files-into-chdb">
  ## الاستعلام باستخدام chDB
</div>

اكتملت عملية إدخال البيانات، والآن حان وقت الجزء الممتع: الاستعلام عن البيانات!

يحصل لاعبو التنس على نقاط بناءً على مستوى أدائهم في البطولات التي يشاركون فيها.
تُحتسب نقاط كل لاعب على أساس فترة متحركة تمتد 52 أسبوعًا.
سنكتب استعلامًا يعثر على الحد الأقصى للنقاط التي جمعها كل لاعب، إلى جانب ترتيبه في ذلك الوقت:

```python
%%sql
SELECT name_first, name_last,
       max(points) as maxPoints,
       argMax(rank, points) as rank,
       argMax(ranking_date, points) as date
FROM atp.players
JOIN atp.rankings ON rankings.player = players.player_id
GROUP BY ALL
ORDER BY maxPoints DESC
LIMIT 10
```

<div id="querying-chdb">
  ## حفظ الاستعلامات
</div>

يمكن حفظ الاستعلامات باستخدام المعلَمة `--save` في السطر نفسه الذي يظهر فيه الأمر السحري `%%sql`.
وتعني المعلَمة `--no-execute` أنه سيتم تخطي تنفيذ الاستعلام.

```python
%%sql --save best_points --no-execute
SELECT name_first, name_last,
       max(points) as maxPoints,
       argMax(rank, points) as rank,
       argMax(ranking_date, points) as date
FROM atp.players
JOIN atp.rankings ON rankings.player = players.player_id
GROUP BY ALL
ORDER BY maxPoints DESC
```

عند تشغيل استعلام محفوظ، سيُحوَّل إلى تعبير جدول مشترك (CTE) قبل تنفيذه.
في الاستعلام التالي، نحسب الحد الأقصى للنقاط التي حققها اللاعبون حين كانوا في المركز الأول:

```python
%sql select * FROM best_points WHERE rank=1
```

<div id="saving-queries">
  ## الاستعلام باستخدام المعلمات
</div>

يمكننا أيضًا استخدام المعلمات في استعلاماتنا.
المعلمات ليست سوى متغيرات عادية:

```python
rank = 10
```

وبعد ذلك، يمكننا استخدام صيغة `{{variable}}` في الاستعلام.
يعثر الاستعلام التالي على اللاعبين الذين كانت لديهم أقل مدة، محسوبة بالأيام، بين أول مرة دخلوا فيها ضمن أفضل 10 وآخر مرة دخلوا فيها ضمن أفضل 10:

```python
%%sql
SELECT name_first, name_last,
       MIN(ranking_date) AS earliest_date,
       MAX(ranking_date) AS most_recent_date,
       most_recent_date - earliest_date AS days,
       1 + (days/7) AS weeks
FROM atp.rankings
JOIN atp.players ON players.player_id = rankings.player
WHERE rank <= {{rank}}
GROUP BY ALL
ORDER BY days
LIMIT 10
```

<div id="querying-with-parameters">
  ## رسم المدرجات التكرارية
</div>

يوفّر JupySQL أيضًا إمكانات محدودة لرسم المخططات.
يمكننا إنشاء مخططات صندوقية أو مدرجات تكرارية.

سننشئ مدرجًا تكراريًا، ولكن دعونا أولًا نكتب (ونحفظ) استعلامًا يحسب المراكز ضمن أفضل 100 التي حققها كل لاعب.
وسيكون بإمكاننا استخدامه لإنشاء مدرج تكراري يحصي عدد اللاعبين الذين حققوا كل مركز:

```python
%%sql --save players_per_rank --no-execute
select distinct player, rank
FROM atp.rankings
WHERE rank <= 100
```

يمكننا بعد ذلك إنشاء مدرج تكراري بتنفيذ ما يلي:

```python
from sql.ggplot import ggplot, geom_histogram, aes

plot = (
  ggplot(
    table="players_per_rank",
    with_="players_per_rank",
    mapping=aes(x="rank", fill="#69f0ae", color="#fff"),
  ) + geom_histogram(bins=100)
)
```
