أعِد تعيين كلمات مرور مستخدمي ClickHouse باستخدام Private API.

<div id="prerequisites">
  ## المتطلبات الأساسية
</div>

* أن يكون ClickHouse Private API مثبّتًا ويمكن الوصول إليه (على سبيل المثال، عبر إعادة توجيه المنفذ إلى `http://localhost:8080/`)
* نشر عنقود ClickHouse واحد على الأقل (يستخدم هذا الدليل `default-xx-01`)

<Steps>
  <Step>
    ## 1. إنشاء تجزئة لكلمة المرور

    جزّئ كلمة المرور الجديدة باستخدام SHA-256 ثم شفّر النتيجة بترميز base64:

    ```ini
    PASSWORD='My super secret p@$$w0rd'
    HASHED_PASSWORD=$(echo -n "$PASSWORD" | shasum -a 256 | awk '{printf $1}' | base64)
    ```
  </Step>

  <Step>
    ## 2. إنشاء جسم الطلب

    أنشئ ملف JSON يتضمن تجزئة كلمة المرور:

    ```json
    {
      "user_hashed_password": "<base64-encoded-sha256-hash>",
      "hashing_function": "sha256",
      "username": "default"
    }
    ```

    احفظه باسم `password_reset.json`.

    | الحقل                  | مطلوب | الافتراضي | الوصف                                                    |
    | ---------------------- | ----- | --------- | -------------------------------------------------------- |
    | `user_hashed_password` | نعم   | --        | قيمة التجزئة المشفّرة بترميز Base64 لكلمة المرور الجديدة |
    | `hashing_function`     | لا    | `sha256`  | خوارزمية التجزئة المستخدمة                               |
    | `username`             | لا    | `default` | اسم المستخدم المطلوب إعادة تعيين كلمة المرور له          |
  </Step>

  <Step>
    ## 3. إرسال الطلب

    ```bash
    curl -X POST "http://localhost:8080/api/v1/instances/default-xx-01/reset-user-password" \
        -H "Content-Type: application/json" \
        --data-binary "@password_reset.json"
    ```

    استجابة ناجحة:

    ```json
    {
      "message": "User password reset successfully"
    }
    ```
  </Step>

  <Step>
    ## 4. التحقق من التغيير

    يسري مفعول إعادة تعيين كلمة المرور بعد أن يعالج المشغّل هذا التغيير. تحقّق من ذلك بالاتصال بالعنقود باستخدام كلمة المرور الجديدة.

    يمكنك أيضًا مراقبة العملية عبر نقطة نهاية الحالة:

    ```bash
    curl "http://localhost:8080/api/v1/instances/default-xx-01/status"
    ```
  </Step>
</Steps>

<div id="hydra-limitation">
  ## قيود Hydra
</div>

**لا يمكن** إعادة تعيين كلمات المرور على المثيلات الفرعية في Hydra. ولإعادة تعيين كلمة المرور لمثيل فرعي في Hydra، أعد تعيينها على المثيل الأصل بدلًا من ذلك.
