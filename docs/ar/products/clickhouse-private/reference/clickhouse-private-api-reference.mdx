ClickHouse Private API هو مكوّن اختياري مستقل لإدارة عناقيد ClickHouse Private. ويوفّر نقاط نهاية REST للنسخ الاحتياطية، والتوسعة الرأسية، وإعادة تعيين كلمات المرور، ومراقبة حالة العناقيد.

***

<div id="configuration-options">
  ## خيارات التهيئة
</div>

يُثبَّت الـ API عبر مخطط Helm (`helm/airgap-management`).

| قيمة Helm                    | النوع      | الافتراضي    | الوصف                                            |
| ---------------------------- | ---------- | ------------ | ------------------------------------------------ |
| `image.repository`           | سلسلة نصية | --           | مستودع ECR/GAR لصورة Private API                 |
| `image.tag`                  | سلسلة نصية | --           | وسم الصورة المطلوب نشرها                         |
| `api.port`                   | عدد صحيح   | `8080`       | المنفذ الذي تستمع عليه الـ API                   |
| `api.basicAuth.enabled`      | منطقي      | `false`      | تمكين مصادقة HTTP الأساسية                       |
| `api.basicAuth.username`     | سلسلة نصية | `"admin"`    | اسم المستخدم للمصادقة الأساسية                   |
| `api.basicAuth.password`     | سلسلة نصية | `"password"` | كلمة المرور للمصادقة الأساسية                    |
| `serviceAccount.enabled`     | منطقي      | `true`       | إنشاء حساب خدمة للـ API                          |
| `serviceAccount.annotations` | خريطة      | `{}`         | التعليقات التوضيحية لحساب الخدمة (مثلًا لـ IRSA) |

***

<div id="authentication">
  ## المصادقة
</div>

بشكل افتراضي، تكون المصادقة الأساسية معطّلة. في بيئات الإنتاج، فعّلها عبر Helm:

```text
--set-json='api.basicAuth.enabled=true'
--set-json='api.basicAuth.username="admin"'
--set-json='api.basicAuth.password="YOUR_SECURE_PASSWORD"'
```

عند التمكين، ضمّن بيانات الاعتماد في الطلبات:

```bash
curl -u admin:YOUR_SECURE_PASSWORD http://localhost:8080/api/v1/...
```

<div id="resource-quantity-format">
  ## صيغة كميات موارد Kubernetes
</div>

تقبل واجهة برمجة التطبيقات صيغ كميات موارد Kubernetes في حقول CPU والذاكرة.

<div id="cpu-quantities">
  ### كميات CPU
</div>

| التنسيق   | المثال   | المعنى           |
| --------- | -------- | ---------------- |
| عدد صحيح  | `"4"`    | 4 أنوية CPU      |
| عدد عشري  | `"2.5"`  | 2.5 أنوية CPU    |
| ميلي نواة | `"500m"` | 0.5 من أنوية CPU |

<div id="memory-quantities">
  ### مقادير الذاكرة
</div>

| الصيغة            | المثال     | المعنى         |
| ----------------- | ---------- | -------------- |
| ثنائية (موصى بها) | `"16Gi"`   | 16 جيبيبايت    |
| ثنائية            | `"4096Mi"` | 4096 ميبيبايت  |
| عشرية             | `"16G"`    | 16 غيغابايت    |
| عشرية             | `"16000M"` | 16000 ميغابايت |

يُوصى باستخدام الوحدات الثنائية (Ki وMi وGi) للذاكرة بما يتوافق مع اصطلاحات Kubernetes.

<div id="backups">
  ### النسخ الاحتياطية
</div>

<div id="create-backup">
  #### إنشاء نسخة احتياطية
</div>

`POST /api/v1/backups?instance_id={instance_id}`

ينشئ نسخة احتياطية جديدة لعنقود ClickHouse المحدد.

**معلمات الاستعلام:**

| المعلّمة      | النوع  | مطلوب | الوصف                                                   |
| ------------- | ------ | ----- | ------------------------------------------------------- |
| `instance_id` | string | نعم   | اسم عنقود ClickHouse (على سبيل المثال، `default-xx-01`) |

**محتوى الطلب:**

```json
{
    "incremental": false,
    "baseBackupUuid": "uuid-of-previous-backup",
    "databases": ["mydb"],
    "tables": ["mydb2.table"]
}
```

| الحقل            | النوع            | مطلوب | الوصف                                                                                                                            |
| ---------------- | ---------------- | ----- | -------------------------------------------------------------------------------------------------------------------------------- |
| `incremental`    | boolean          | لا    | `false` للنسخة الاحتياطية الكاملة، و`true` للنسخة الاحتياطية التزايدية. القيمة الافتراضية هي `false`.                            |
| `baseBackupUuid` | string           | لا    | معرّف UUID للنسخة الاحتياطية السابقة (مطلوب عندما تكون قيمة `incremental` هي `true`)                                             |
| `databases`      | array of strings | لا    | قواعد بيانات محددة لأخذ نسخة احتياطية منها. اتركه فارغًا لنسخ جميع قواعد البيانات احتياطيًا.                                     |
| `tables`         | array of strings | لا    | جداول محددة لأخذ نسخة احتياطية منها. يجب استخدام الأسماء المؤهلة بالكامل (`db.table`). اتركه فارغًا لنسخ جميع الجداول احتياطيًا. |

**الاستجابة:** `201 Created` -- يعيد كائن النسخة الاحتياطية المُنشأ، متضمّنًا معرّف UUID.

**السلوك:** يُنشئ مورد `Backup` مخصصًا في مساحة اسم عنقود ClickHouse. يراقب المُشغِّل كائنات `Backup` الجديدة وينفّذ عبارة SQL ذات الصلة بالنسخ الاحتياطي، مع متابعة الحالة عبر جداول النظام.

***

<div id="get-backup">
  #### الحصول على نسخة احتياطية
</div>

`GET /api/v1/backups/{backup_id}?instance_id={instance_id}`

يعيد حالة وتفاصيل نسخة احتياطية محددة.

**معلمات المسار:**

| المعلمة     | النوع  | مطلوب | الوصف                         |
| ----------- | ------ | ----- | ----------------------------- |
| `backup_id` | string | نعم   | UUID الخاص بالنسخة الاحتياطية |

**معلمات الاستعلام:**

| المعلمة       | النوع  | مطلوب | الوصف                |
| ------------- | ------ | ----- | -------------------- |
| `instance_id` | string | نعم   | اسم عنقود ClickHouse |

**الاستجابة:** `200 OK` -- يعيد كائن النسخة الاحتياطية. ويشير الحقل `status.state` إلى حالة الإكمال (على سبيل المثال، `Ready` عند اكتمالها).

***

<div id="list-backups">
  #### عرض النسخ الاحتياطية
</div>

`GET /api/v1/backups?instance_id={instance_id}`

يسرد جميع النسخ الاحتياطية للمجموعة المحددة.

**معلمات الاستعلام:**

| المعلمة             | النوع   | مطلوب | الوصف                                              |
| ------------------- | ------- | ----- | -------------------------------------------------- |
| `instance_id`       | string  | نعم   | اسم عنقود ClickHouse                              |
| `status__state__eq` | string  | لا    | التصفية حسب حالة النسخة الاحتياطية (مثلًا، `Ready`) |
| `sort`              | string  | لا    | حقل الفرز (مثلًا، `status__finishTime`)             |
| `limit`             | integer | لا    | الحد الأقصى لعدد النتائج المطلوب إرجاعها            |

**الاستجابة:** `200 OK` -- يعيد مصفوفة من كائنات النسخ الاحتياطية.

**مثال:** اعثر على معرّف UUID لآخر نسخة احتياطية ناجحة:

```text
GET /api/v1/backups?instance_id=default-xx-01&status__state__eq=Ready&sort=status__finishTime&limit=1
```

<div id="restore-backup">
  #### استعادة نسخة احتياطية
</div>

`POST /api/v1/backups/{backup_id}/restore?instance_id={instance_id}&target_instance_id={target_instance_id}`

يستعيد نسخة احتياطية إلى عنقود ClickHouse مستهدف. وينفّذ `RESTORE ALL` للنسخة الاحتياطية.

**معلمات المسار:**

| المعلمة     | النوع  | مطلوب | الوصف                                    |
| ----------- | ------ | ----- | ---------------------------------------- |
| `backup_id` | string | نعم   | UUID للنسخة الاحتياطية المطلوب استعادتها |

**معلمات الاستعلام:**

| المعلمة              | النوع  | مطلوب | الوصف                                                      |
| -------------------- | ------ | ----- | ---------------------------------------------------------- |
| `instance_id`        | string | نعم   | اسم عنقود ClickHouse المصدر (حيث أُنشئت النسخة الاحتياطية) |
| `target_instance_id` | string | نعم   | اسم عنقود ClickHouse المستهدف (الذي ستتم الاستعادة إليه)   |

**القيود:** - الاستعادة على المثيل نفسه (`instance_id == target_instance_id`) **غير مسموح بها** لأسباب تتعلق بالسلامة. - يُوصى بالاستعادة إلى عنقود جديد لتجنّب زيادة الحمل على العنقود الأصلي.

**الاستجابة:** `202 Accepted` -- يُرجع معرّفات عمليات الاستعادة.

<div id="vertical-scaling">
  ### التحجيم الرأسي
</div>

<div id="scale-cluster">
  #### توسيع العنقود
</div>

`POST /api/v1/instances/{instance_id}/scale`

يُوسّع عنقود ClickHouse رأسيًا من خلال ضبط موارد CPU والذاكرة.

**معلمات المسار:**

| المعلمة       | النوع  | مطلوب | الوصف                |
| ------------- | ------ | ----- | -------------------- |
| `instance_id` | string | نعم   | اسم عنقود ClickHouse |

**نص الطلب:**

```json
{
  "vertical": {
    "resources": {
      "cpu": "4",
      "memory": "16Gi"
    }
  }
}
```

| الحقل                       | النوع  | مطلوب | الوصف                                      |
| --------------------------- | ------ | ----- | ------------------------------------------ |
| `vertical.resources.cpu`    | string | لا*   | تخصيص CPU بصيغة كميات موارد Kubernetes     |
| `vertical.resources.memory` | string | لا*   | تخصيص الذاكرة بصيغة كميات موارد Kubernetes |

*يجب توفير `cpu` أو `memory` واحد على الأقل.

**نسبة CPU إلى الذاكرة:** تفرض واجهة API **نسبة 1:4** (نواة CPU واحدة لكل 4 GiB من الذاكرة) مع هامش 5%. إذا تم توفير مورد واحد فقط، تُحتسب القيمة الأخرى تلقائيًا للحفاظ على هذه النسبة.

**السلوك:** يُحدِّث `ServerPodPolicy` في المورد المخصص `ClickHouseCluster`. ويؤدي المشغّل إلى **إعادة تشغيل متدرجة** لـ StatefulSets مع تخصيص الموارد الجديد.

**مهم:** بخلاف ClickHouse Cloud، لا يستخدم ClickHouse Private **التحجيم بأسلوب Make Before Break (MBB)**. ويتسبب التحجيم الرأسي في إعادة تشغيل متدرجة قد تعطل الخدمة مؤقتًا.

**الاستجابة:** `201 Created`

<div id="password-reset">
  ### إعادة تعيين كلمة المرور
</div>

<div id="reset-user-password">
  #### إعادة تعيين كلمة مرور المستخدم
</div>

`POST /api/v1/instances/{instance_id}/reset-user-password`

يعيد تعيين كلمة المرور لمستخدم في عنقود ClickHouse.

**معلمات المسار:**

| المعلمة       | النوع  | مطلوب | الوصف                |
| ------------- | ------ | ----- | -------------------- |
| `instance_id` | string | نعم   | اسم عنقود ClickHouse |

**نص الطلب:**

```json
{
  "user_hashed_password": "<base64-encoded-sha256-hash>",
  "hashing_function": "sha256",
  "username": "default"
}
```

| الحقل                  | النوع  | مطلوب | الافتراضي   | الوصف                                           |
| ---------------------- | ------ | ----- | ----------- | ----------------------------------------------- |
| `user_hashed_password` | string | نعم   | --          | تجزئة كلمة المرور الجديدة المرمّزة بصيغة Base64 |
| `hashing_function`     | string | لا    | `"sha256"`  | خوارزمية التجزئة المستخدمة                      |
| `username`             | string | لا    | `"default"` | اسم المستخدم الذي ستُعاد تعيين كلمة المرور له   |

**إنشاء تجزئة كلمة المرور:**

```ini
PASSWORD='My super secret p@$$w0rd'
HASHED_PASSWORD=$(echo -n "$PASSWORD" | shasum -a 256 | awk '{printf $1}' | base64)
```

**القيود:** - لا **يمكن** إعادة تعيين كلمة المرور على مثيلات Hydra الفرعية. أجرِ إعادة التعيين على المثيل الأصل بدلًا من ذلك.

**الاستجابة:**

```json
{
  "message": "User password reset successfully"
}
```

**السلوك:** يُحدِّث الحقل `CustomerAccount` في المورد المخصّص `ClickhouseCluster`. يطبّق المشغّل تغيير بيانات الاعتماد على العنقود. يسري مفعول كلمة المرور بعد أن يعالج المشغّل هذا التغيير.

<div id="cluster-status">
  ### حالة العنقود
</div>

<div id="get-cluster-status">
  #### الحصول على حالة العنقود
</div>

`GET /api/v1/instances/{instance_id}/status`

يُرجع الحالة الحالية لعنقود ClickHouse المحدد.

**معلمات المسار:**

| المَعلمة      | النوع  | مطلوب | الوصف                |
| ------------- | ------ | ----- | -------------------- |
| `instance_id` | string | نعم   | اسم عنقود ClickHouse |

**الاستجابة:**

```json
{
  "state": "Running",
  "previousState": "Provisioning",
  "message": "Cluster is healthy",
  "stateProvidedBy": "operator"
}
```

| الحقل             | النوع | الوصف                                      |
| ----------------- | ----- | ------------------------------------------ |
| `state`           | نصي   | الحالة الحالية لعنقود ClickHouse           |
| `previousState`   | نصي   | الحالة التي انتقل منها العنقود             |
| `message`         | نصي   | تفاصيل سبب انتقال العنقود من حالة إلى أخرى |
| `stateProvidedBy` | نصي   | العملية المسؤولة عن إحداث انتقال الحالة    |

**حالات الاستخدام:** - التحقق من اكتمال عملية التوسع الرأسي. - تأكيد تطبيق إعادة تعيين كلمة المرور على مستوى العنقود بأكمله. - مراقبة تقدّم تهيئة العنقود.
