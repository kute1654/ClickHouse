<div id="iam-service-account-requirements">
  ## متطلبات IAM / حساب الخدمة
</div>

<div id="aws">
  ### AWS
</div>

<div id="ecr-pull-role">
  #### دور السحب من ECR
</div>

* **اسم الدور:** `ClickHouseAirgapECRPullRole`
* **الغرض:** سحب الصور من مستودع ECR الخاص بـ ClickHouse
* **الأذونات:** `ecr:GetDownloadUrlForLayer`, `ecr:BatchGetImage`, `ecr:BatchCheckLayerAvailability`, `ecr:DescribeImages`, `ecr:ListImages`, `ecr:GetAuthorizationToken`
* **الإجراء:** زوّد فريق ClickHouse بـ ARN الخاص بالدور

<div id="clickhouse-serverkeeper-s3-role">
  #### دور S3 لخادم ClickHouse/Keeper
</div>

* **دور واحد لكل عنقود ClickHouse تم توفيره**
* **نمط التسمية:** `CH-S3-$NAME-$REGION-$ORDINAL-Role`
* `$NAME`: اسم العنقود (مثلًا: `default-xx-01`)
* `$REGION`: اختصار المنطقة (مثلًا: `uw2` بدلًا من `us-west-2`)
* `$ORDINAL`: محجوز، ويُضبط على `00`
* مثال: `CH-S3-default-xx-01-uw2-00-Role`
* **الأذونات:** `s3:*` و`s3:ListBucket` على مورد الـ bucket
* **سياسة الثقة:** IRSA عبر `sts:AssumeRoleWithWebIdentity` باستخدام موفّر OIDC لعنقود EKS

<div id="ebs-csi-driver-role">
  #### دور EBS CSI Driver
</div>

* **اسم المثال:** `ClickHouse_EksEbsCsiDriverRole`
* **السياسة المُدارة:** `arn:aws:iam::aws:policy/service-role/AmazonEBSCSIDriverPolicy`
* **سياسة الثقة:** IRSA مع موفّر OIDC لعنقود EKS

<div id="gcp">
  ### GCP
</div>

<div id="deployer-service-account">
  #### حساب خدمة النشر
</div>

| الدور                                   | الغرض                                                      |
| --------------------------------------- | ---------------------------------------------------------- |
| `roles/compute.networkAdmin`            | إنشاء/إدارة شبكات VPC والشبكات الفرعية وNAT وأجهزة التوجيه |
| `roles/container.admin`                 | إنشاء/إدارة مجموعات GKE ومجموعات العُقد                    |
| `roles/storage.admin`                   | إنشاء/إدارة حاويات GCS                                     |
| `roles/iam.serviceAccountAdmin`         | إنشاء/إدارة حسابات الخدمة                                  |
| `roles/iam.serviceAccountUser`          | استخدام حسابات الخدمة مع Workload Identity                 |
| `roles/iam.workloadIdentityUser`        | تهيئة ارتباطات Workload Identity                           |
| `roles/artifactregistry.admin`          | إنشاء/إدارة مستودعات Artifact Registry                     |
| `roles/resourcemanager.projectIamAdmin` | إدارة سياسات IAM على المشروع                               |

<div id="clickhouse-workload-identity-service-account">
  #### حساب الخدمة الخاص بـ Workload Identity في ClickHouse
</div>

* **حساب خدمة GCP:** `clickhouse@$PROJECT.iam.gserviceaccount.com`
* **حساب خدمة K8s (يدوي):** `clickhouse` في مساحة الأسماء `ns-$CLUSTER_NAME`
* **حساب خدمة K8s (أنشأه Helm):** `ch-$CLUSTER_NAME-sa` في مساحة الأسماء `ns-$CLUSTER_NAME`
* **الوسم التوضيحي لحساب خدمة K8s:** `iam.gke.io/gcp-service-account=<GSA_EMAIL>`
* **عضو ربط IAM:** `serviceAccount:$PROJECT.svc.id.goog[<namespace>/<ksa>]`
* **أذونات حاوية التخزين:** `roles/storage.objectAdmin`, `roles/storage.legacyBucketReader`
* **أذونات Artifact Registry:** `roles/artifactregistry.reader`

<div id="bare-metal">
  ### الخوادم المادية
</div>

* **بيانات اعتماد AWS:** مفتاح الوصول والمفتاح السري لسحب صور ECR
* **بيانات اعتماد MinIO:** مفتاح وصول ومفتاح سري منفصلان لتخزين متوافق مع S3
* **أسرار سحب الصور:** سرّ `docker-registry` المسمى `all-icr-io` في مساحة الاسم `clickhouse-operator-system` ومساحات أسماء العنقود
* **حساب الخدمة:** `default` (من دون IRSA؛ تُمرَّر بيانات الاعتماد عبر متغيرات البيئة)

| المكوّن       | الوصف                          | مثال      |
| ------------- | ------------------------------ | --------- |
| `$DESCRIPTOR` | اسم وصفي (أحرف فقط)            | `default` |
| `$LETTERS`    | محجوز؛ أي حرفين                | `xx`      |
| `$ORDINAL`    | رقم ترتيبي متزايد يبدأ من `01` | `01`      |

**اسم المثال:** `default-xx-01`

يجب أن يكون اسم العنقود **فريدًا داخل عنقود Kubernetes المستهدف**، ويُستخدم في: - حيز أسماء Kubernetes: `ns-$CLUSTER_NAME` - موارد Kubernetes: مسبوقة بـ `c-$CLUSTER_NAME` - أسماء أدوار IAM ‏(AWS): `CH-S3-$CLUSTER_NAME-$REGION-$ORDINAL-Role` - بادئات مفاتيح S3/GCS: `ch-s3-<uuid>` (فريدة لكل عنقود)
