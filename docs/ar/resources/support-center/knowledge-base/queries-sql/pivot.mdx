<div id="introduction">
  ## مقدمة
</div>

<div id="understanding-aggregate-function-combinators">
  ## فهم مُعدِّلات الدوال التجميعية
</div>

لنبدأ بمثال بسيط. سنستخدم [clickhouse-local](/ar/concepts/features/tools-and-utilities/clickhouse-local)، ويمكنك تشغيله بتنفيذ الأمر التالي:

```bash
clickhouse -m --output_format_pretty_row_numbers=0
```

يستدعي الاستعلام التالي الدالة `sumMap`، التي تأخذ قيمة من النوع `Map` وتجمع قيم كل مفتاح:

```sql
SELECT sumMap(map('ClickHouse', 1, 'ClickBench', 2));
```

```text
┌─sumMap(map('ClickHouse', 1, 'ClickBench', 2))─┐
│ {'ClickBench':2,'ClickHouse':1}               │
└───────────────────────────────────────────────┘
```

هذا المثال ليس مثيرًا للاهتمام كثيرًا، لأنه يعيد قيمة `Map` نفسها التي مررناها.
لنستدعِ الآن `sumMap` على عدة صفوف من قيم `Map`;

```sql
WITH values AS (
  SELECT map('ClickHouse', 3) AS value
  UNION ALL
  SELECT map('ClickBench', 2, 'ClickHouse', 4) AS value
)
SELECT sumMap(value)
FROM values;
```

```text
┌─sumMap(value)───────────────────┐
│ {'ClickBench':2,'ClickHouse':7} │
└─────────────────────────────────┘
```

ظهر المفتاح `ClickHouse` في كلا الصفَّين، وجُمِعت قيمه. أمّا المفتاح `ClickBench` فلم يظهر إلا في صف واحد، لذا تُجمَع له قيمة واحدة فقط، فتكون النتيجة هي تلك القيمة نفسها!

يمكننا أيضًا استخدام `maxMap` للعثور على القيم العظمى لكل مفتاح:

```sql
WITH values AS (
  SELECT map('ClickHouse', 3) AS value
  UNION ALL
  SELECT map('ClickBench', 2, 'ClickHouse', 4) AS value
)
SELECT maxMap(value)
FROM values;
```

```text
┌─maxMap(value)───────────────────┐
│ {'ClickBench':2,'ClickHouse':4} │
└─────────────────────────────────┘
```

أو يمكننا استخدام `avgMap` لحساب متوسط القيم لكل مفتاح:

```sql
WITH values AS (
  SELECT map('ClickHouse', 3) AS value
  UNION ALL
  SELECT map('ClickBench', 2, 'ClickHouse', 4) AS value
)
SELECT avgMap(value)
FROM values;
```

```text
┌─avgMap(value)─────────────────────┐
│ {'ClickBench':2,'ClickHouse':3.5} │
└───────────────────────────────────┘
```

نأمل أن تكون قد تكوّنت لديك الآن فكرة عن كيفية عمل مُركِّبات الدوال هذه.

<div id="grouping-and-aggregating-by-decade">
  ### التجميع والعمليات التجميعية حسب العقد الزمني
</div>

لنحسب الأسعار الوسيطة، مجمّعة حسب المقاطعة لكل عقد زمني في مجموعة البيانات:

```sql
WITH year(toStartOfInterval(date, toIntervalYear(10))) AS year
SELECT
    county,
    medianMap(map(year, price)) AS medianPrices
FROM uk.uk_price_paid
GROUP BY ALL
ORDER BY max(price) DESC
LIMIT 10;
```

<div id="combining-multiple-aggregations">
  ## دمج عمليات تجميع متعددة
</div>

وإذا أردنا العثور على أعلى سعر لكل عقد زمني، فيمكننا فعل ذلك باستخدام الدالة `maxMap` التي رأيناها سابقًا:

```sql
WITH year(toStartOfInterval(date, toIntervalYear(10))) AS year
SELECT
    county,
    medianMap(map(year, price)) AS medianPrices,

FROM uk.uk_price_paid
WHERE year >= 2010
GROUP BY ALL
ORDER BY max(price) DESC
```

<div id="applying-functions-to-map-values">
  ## تطبيق الدوال على قيم Map
</div>

بدلًا من ذلك، يمكننا احتساب متوسط السعر باستخدام `avgMap`.
تحتوي هذه القيم على عدد كبير من الخانات العشرية، ويمكننا تبسيطها باستخدام الدالة [`mapApply`](/ar/reference/functions/regular-functions/tuple-map-functions#mapApply) لاستدعاء الدالة [`floor`](/ar/reference/functions/regular-functions/rounding-functions#floor) على كل قيمة في Map:

```sql
WITH year(toStartOfInterval(date, toIntervalYear(10))) AS year
SELECT
    county,
    medianMap(map(year, price)) AS medianPrices,
    mapApply((k, v) -> (k, floor(v)), avgMap(map(year, price))) AS avgPrices
FROM uk.uk_price_paid
WHERE year >= 2010
GROUP BY ALL
ORDER BY max(price) DESC
LIMIT 10;
```

<div id="flexible-grouping-counties-districts-and-postcodes">
  ## تجميع مرن: المقاطعات والمناطق والرموز البريدية
</div>

لنجرّب التجميع حسب بعض الحقول المختلفة.

```sql
WHERE year >= 2010
GROUP BY ALL
```
