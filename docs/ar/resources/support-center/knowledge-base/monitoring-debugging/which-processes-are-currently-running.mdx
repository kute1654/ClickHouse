<div id="check-what-code-is-currently-running-on-a-server">
  ## تحقّق من الشيفرة التي تعمل حاليًا على الخادم
</div>

يحتوي ClickHouse على مصحّح أخطاء مضمّن وإمكانات فحص داخلي.

على سبيل المثال، يمكنك الحصول على تتبعات المكدس لكل خيط تنفيذ في الخادم أثناء التشغيل من خلال الاستعلام عن جدول `system.stack_trace`:

```sql
SELECT
    count(),
    arrayStringConcat(arrayMap(x -> concat(demangle(addressToSymbol(x)), '\n    ', addressToLine(x)), trace), '\n') AS sym
FROM system.stack_trace
GROUP BY trace
ORDER BY count() DESC
LIMIT 10
FORMAT Vertical
SETTINGS allow_introspection_functions = 1;
```

ستُظهر نتيجة الاستعلام المواضع في الشفرة المصدرية لـ ClickHouse التي تكون فيها سلاسل التنفيذ قيد التشغيل أو في حالة انتظار. (ستحتاج إلى ضبط `allow_introspection_functions` على `1` لتمكين [دوال الفحص الداخلي](/ar/reference/functions/regular-functions/introspection).) تبدو الاستجابة كما يلي:

<Note>
  إذا ثبّتَّ ClickHouse من **.deb/.rpm/.tgz**، فيمكنك أيضًا تثبيت الحزمة التي تتضمن معلومات التصحيح لرؤية أرقام الأسطر من الشيفرة المصدرية:

  ```bash
  sudo apt install clickhouse-common-static-dbg
  ```

  إذا كنت قد ثبّتَّ ClickHouse كملف تنفيذي واحد، فهو يتضمن بالفعل معلومات التصحيح.
</Note>
