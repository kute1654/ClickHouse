<div id="description">
  ## الوصف
</div>

يتضمن معلومات عن إعدادات الجلسة للمستخدم الحالي.

<div id="columns">
  ## الأعمدة
</div>

{/*AUTOGENERATED_START*/}

* `name` ([String](/ar/reference/data-types/index)) — اسم الإعداد.
* `value` ([String](/ar/reference/data-types/index)) — قيمة الإعداد.
* `changed` ([UInt8](/ar/reference/data-types/index)) — يوضّح ما إذا كان الإعداد قد عُرِّف صراحةً في `config` أو غُيِّر صراحةً.
* `description` ([String](/ar/reference/data-types/index)) — وصف مختصر للإعداد.
* `min` ([Nullable(String)](/ar/reference/data-types/index)) — الحد الأدنى لقيمة الإعداد، إذا كانت هناك قيمة محددة عبر القيود. وإذا لم تكن هناك قيمة دنيا للإعداد، فستكون القيمة NULL.
* `max` ([Nullable(String)](/ar/reference/data-types/index)) — الحد الأقصى لقيمة الإعداد، إذا كانت هناك قيمة محددة عبر القيود. وإذا لم تكن هناك قيمة قصوى للإعداد، فستكون القيمة NULL.
* `disallowed_values` ([Array(String)](/ar/reference/data-types/index)) — قائمة القيم غير المسموح بها
* `readonly` ([UInt8](/ar/reference/data-types/index)) — يوضّح ما إذا كان بإمكان المستخدم الحالي تغيير الإعداد: 0 — يمكن للمستخدم الحالي تغيير الإعداد، 1 — لا يمكن للمستخدم الحالي تغيير الإعداد.
* `type` ([String](/ar/reference/data-types/index)) — نوع القيمة التي يمكن تعيينها لهذا الإعداد.
* `default` ([String](/ar/reference/data-types/index)) — قيمة default للإعداد.
* `alias_for` ([String](/ar/reference/data-types/index)) — اسم الإعداد الأصلي إذا كان الإعداد اسمًا مستعارًا لإعداد آخر.
* `is_obsolete` ([UInt8](/ar/reference/data-types/index)) — يوضّح ما إذا كان الإعداد متقادِمًا.
* `tier` ([Enum8(&#39;Production&#39; = 0, &#39;Obsolete&#39; = 4, &#39;Experimental&#39; = 8, &#39;Beta&#39; = 12)](/ar/reference/data-types/index)) —
  مستوى الدعم لهذه الميزة. تُنظَّم ميزات ClickHouse ضمن مستويات تختلف بحسب حالتها الحالية
  من حيث التطوير والتوقعات الممكنة عند استخدامها:

- PRODUCTION: الميزة مستقرة وآمنة للاستخدام، ولا توجد مشكلات في تفاعلها مع ميزات PRODUCTION الأخرى.
- BETA: الميزة مستقرة وآمنة. أما نتيجة استخدامها مع ميزات أخرى فغير معروفة، ولا يمكن ضمان صحتها. نرحب بالاختبارات والتقارير.
- EXPERIMENTAL: الميزة لا تزال قيد التطوير. وهي مخصصة فقط للمطورين ومتحمسي ClickHouse. قد تعمل أو لا تعمل، وقد تُزال في أي وقت.
- OBSOLETE: لم تعد مدعومة. إما أنها أُزيلت بالفعل أو ستُزال في الإصدارات المستقبلية.

{/*AUTOGENERATED_END*/}

<div id="example">
  ## مثال
</div>

يوضح المثال التالي كيفية الحصول على معلومات حول الإعدادات التي تتضمن أسماؤها `min_i`.

```sql
SELECT *
FROM system.settings
WHERE name LIKE '%min_insert_block_size_%'
FORMAT Vertical
```

قد يكون استخدام `WHERE changed` مفيدًا، على سبيل المثال، عندما تريد التحقق مما يلي:

* ما إذا كانت الإعدادات في ملفات التكوين قد حُمّلت بشكل صحيح وما إذا كانت مستخدمة.
* الإعدادات التي تغيّرت في الجلسة الحالية.

```sql
SELECT * FROM system.settings WHERE changed AND name='load_balancing'
```
