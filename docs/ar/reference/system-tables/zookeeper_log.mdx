<div id="description">
  ## الوصف
</div>

يحتوي هذا الجدول على معلومات حول معلمات الطلب المرسل إلى خادم ZooKeeper والاستجابة الصادرة عنه.

بالنسبة إلى الطلبات، لا تُعبَّأ إلا الأعمدة التي تحتوي على معلمات الطلب، بينما تُملأ الأعمدة المتبقية بالقيم الافتراضية (`0` أو `NULL`). وعند وصول الاستجابة، تُضاف بياناتها إلى الأعمدة الأخرى.

<div id="columns">
  ## الأعمدة
</div>

{/*AUTOGENERATED_START*/}

* `hostname` ([LowCardinality(String)](/ar/reference/data-types/lowcardinality)) — اسم مضيف الخادم الذي ينفّذ الاستعلام.
* `type` ([Enum8(&#39;Request&#39; = 1, &#39;Response&#39; = 2, &#39;Finalize&#39; = 3)](/ar/reference/data-types/enum)) — نوع الحدث في عميل ZooKeeper. ويمكن أن تكون له إحدى القيم التالية: Request — تم إرسال الطلب، Response — تم استلام الاستجابة، Finalize — فُقد الاتصال ولم يتم استلام أي استجابة.
* `event_date` ([Date](/ar/reference/data-types/date)) — التاريخ الذي وقع فيه الحدث.
* `event_time` ([DateTime64(6)](/ar/reference/data-types/datetime64)) — التاريخ والوقت اللذان وقع فيهما الحدث.
* `thread_id` ([UInt64](/ar/reference/data-types/int-uint)) — معرّف الخيط الذي نفّذ هذا الطلب.
* `query_id` ([String](/ar/reference/data-types/string)) — معرّف الاستعلام الذي نُفِّذ هذا الطلب ضمن نطاقه.
* `address` ([IPv6](/ar/reference/data-types/ipv6)) — عنوان IP لخادم ZooKeeper المستخدم لإجراء الطلب.
* `port` ([UInt16](/ar/reference/data-types/int-uint)) — منفذ خادم ZooKeeper المستخدم لإجراء الطلب.
* `session_id` ([Int64](/ar/reference/data-types/int-uint)) — معرّف الجلسة الذي يعيّنه خادم ZooKeeper لكل اتصال.
* `duration_microseconds` ([UInt64](/ar/reference/data-types/int-uint)) — الوقت الذي استغرقه ZooKeeper لتنفيذ الطلب.
* `xid` ([Int64](/ar/reference/data-types/int-uint)) — معرّف الطلب داخل الجلسة. ويكون هذا عادةً رقم طلب تسلسليًا. وهو نفسه في صف الطلب وصف الاستجابة/الإنهاء المقترن به.
* `has_watch` ([UInt8](/ar/reference/data-types/int-uint)) — ما إذا تم تعيين watch لهذا الطلب.
* `op_num` ([Enum16(&#39;Close&#39; = -11, &#39;Error&#39; = -1, &#39;Watch&#39; = 0, &#39;Create&#39; = 1, &#39;Remove&#39; = 2, &#39;Exists&#39; = 3, &#39;Get&#39; = 4, &#39;Set&#39; = 5, &#39;GetACL&#39; = 6, &#39;SetACL&#39; = 7, &#39;SimpleList&#39; = 8, &#39;Sync&#39; = 9, &#39;Heartbeat&#39; = 11, &#39;List&#39; = 12, &#39;Check&#39; = 13, &#39;Multi&#39; = 14, &#39;Create2&#39; = 15, &#39;Reconfig&#39; = 16, &#39;CheckWatch&#39; = 17, &#39;RemoveWatch&#39; = 18, &#39;MultiRead&#39; = 22, &#39;Auth&#39; = 100, &#39;SetWatch&#39; = 101, &#39;SetWatch2&#39; = 105, &#39;AddWatch&#39; = 106, &#39;FilteredList&#39; = 500, &#39;CheckNotExists&#39; = 501, &#39;CreateIfNotExists&#39; = 502, &#39;RemoveRecursive&#39; = 503, &#39;CheckStat&#39; = 504, &#39;TryRemove&#39; = 505, &#39;FilteredListWithStatsAndData&#39; = 506, &#39;ListRecursive&#39; = 507, &#39;SessionID&#39; = 997)](/ar/reference/data-types/enum)) — نوع الطلب أو الاستجابة.
* `path` ([String](/ar/reference/data-types/string)) — مسار عقدة ZooKeeper المحددة في الطلب، أو سلسلة فارغة إذا كان الطلب لا يتطلب تحديد مسار.
* `data` ([String](/ar/reference/data-types/string)) — البيانات المكتوبة إلى عقدة ZooKeeper (في طلبي SET و CREATE — ما أراد الطلب كتابته، وفي الاستجابة لطلب GET — ما تمت قراءته)، أو سلسلة فارغة.
* `is_ephemeral` ([UInt8](/ar/reference/data-types/int-uint)) — ما إذا كانت عقدة ZooKeeper تُنشأ كعقدة مؤقتة.
* `is_sequential` ([UInt8](/ar/reference/data-types/int-uint)) — ما إذا كانت عقدة ZooKeeper تُنشأ كعقدة تسلسلية.
* `version` ([Nullable(Int32)](/ar/reference/data-types/nullable)) — إصدار عقدة ZooKeeper الذي يتوقعه الطلب عند التنفيذ. هذا مدعوم في طلبات CHECK وSET وREMOVE (وتكون القيمة ذات الصلة -1 إذا كان الطلب لا يتحقق من الإصدار، أو NULL في الطلبات الأخرى التي لا تدعم التحقق من الإصدار).
* `requests_size` ([UInt32](/ar/reference/data-types/int-uint)) — عدد الطلبات المضمّنة في الطلب المتعدد (وهو طلب خاص يتكوّن من عدة طلبات عادية متتالية ويُنفَّذها بصورة ذرّية). ستكون لجميع الطلبات المضمّنة في الطلب المتعدد قيمة xid نفسها.
* `request_idx` ([UInt32](/ar/reference/data-types/int-uint)) — رقم الطلب المضمَّن في الطلب المتعدد (بالنسبة إلى الطلب المتعدد — 0، ثم بالتتابع بدءًا من 1).
* `zxid` ([Int64](/ar/reference/data-types/int-uint)) — معرّف معاملة ZooKeeper. وهو الرقم التسلسلي الذي يصدره خادم ZooKeeper استجابةً لطلب نُفِّذ بنجاح (0 إذا لم يُنفَّذ الطلب/أعاد خطأً/لم يكن `client` يعرف ما إذا كان الطلب قد نُفِّذ).
* `error` ([Nullable(Enum8(&#39;ZNOWATCHER&#39; = -121, &#39;ZNOTREADONLY&#39; = -119, &#39;ZSESSIONMOVED&#39; = -118, &#39;ZNOTHING&#39; = -117, &#39;ZCLOSING&#39; = -116, &#39;ZAUTHFAILED&#39; = -115, &#39;ZINVALIDACL&#39; = -114, &#39;ZINVALIDCALLBACK&#39; = -113, &#39;ZSESSIONEXPIRED&#39; = -112, &#39;ZNOTEMPTY&#39; = -111, &#39;ZNODEEXISTS&#39; = -110, &#39;ZNOCHILDRENFOREPHEMERALS&#39; = -108, &#39;ZBADVERSION&#39; = -103, &#39;ZNOAUTH&#39; = -102, &#39;ZNONODE&#39; = -101, &#39;ZAPIERROR&#39; = -100, &#39;ZOUTOFMEMORY&#39; = -10, &#39;ZINVALIDSTATE&#39; = -9, &#39;ZBADARGUMENTS&#39; = -8, &#39;ZOPERATIONTIMEOUT&#39; = -7, &#39;ZUNIMPLEMENTED&#39; = -6, &#39;ZMARSHALLINGERROR&#39; = -5, &#39;ZCONNECTIONLOSS&#39; = -4, &#39;ZDATAINCONSISTENCY&#39; = -3, &#39;ZRUNTIMEINCONSISTENCY&#39; = -2, &#39;ZSYSTEMERROR&#39; = -1, &#39;ZOK&#39; = 0))](/ar/reference/data-types/nullable)) — رمز الخطأ. يمكن أن يأخذ قيَمًا كثيرة، وفيما يلي بعضٌ منها فقط: ZOK — تم تنفيذ الطلب بنجاح، ZCONNECTIONLOSS — فُقد الاتصال، ZOPERATIONTIMEOUT — انتهت مهلة تنفيذ الطلب، ZSESSIONEXPIRED — انتهت صلاحية الجلسة، NULL — اكتمل الطلب.
* `watch_type` ([Nullable(Enum8(&#39;NOTWATCHING&#39; = -2, &#39;SESSION&#39; = -1, &#39;CREATED&#39; = 1, &#39;DELETED&#39; = 2, &#39;CHANGED&#39; = 3, &#39;CHILD&#39; = 4))](/ar/reference/data-types/nullable)) — نوع حدث watch (بالنسبة إلى الاستجابات التي يكون فيها op&#95;num = Watch)، أما بقية الاستجابات فقيمته: NULL.
* `watch_state` ([Nullable(Enum16(&#39;AUTH&#95;FAILED&#39; = -113, &#39;EXPIRED&#95;SESSION&#39; = -112, &#39;CONNECTING&#39; = 1, &#39;ASSOCIATING&#39; = 2, &#39;CONNECTED&#39; = 3, &#39;READONLY&#39; = 5, &#39;NOTCONNECTED&#39; = 999))](/ar/reference/data-types/nullable)) — حالة حدث watch (بالنسبة إلى الاستجابات التي يكون فيها op&#95;num = Watch)، أما بقية الاستجابات فقيمته: NULL.
* `path_created` ([String](/ar/reference/data-types/string)) — المسار إلى عقدة ZooKeeper التي أُنشئت (بالنسبة إلى الاستجابات لطلب CREATE)، وقد يختلف عن المسار إذا كانت العقدة قد أُنشئت كعقدة sequential.
* `stat_czxid` ([Int64](/ar/reference/data-types/int-uint)) — قيمة zxid للتغيير الذي تسبب في إنشاء عقدة ZooKeeper هذه.
* `stat_mzxid` ([Int64](/ar/reference/data-types/int-uint)) — معرّف `zxid` للتغيير الذي أُجري آخر مرة على عقدة ZooKeeper هذه.
* `stat_pzxid` ([Int64](/ar/reference/data-types/int-uint)) — معرّف transaction للتغيير الذي أُجري آخر مرة على العقد الفرعية لعقدة ZooKeeper هذه.
* `stat_version` ([Int32](/ar/reference/data-types/int-uint)) — عدد التغييرات التي أُجريت على بيانات عقدة ZooKeeper هذه.
* `stat_cversion` ([Int32](/ar/reference/data-types/int-uint)) — عدد التغييرات التي أُجريت على العقد الفرعية لعقدة ZooKeeper هذه.
* `stat_dataLength` ([Int32](/ar/reference/data-types/int-uint)) — طول حقل البيانات في عقدة ZooKeeper هذه.
* `stat_numChildren` ([Int32](/ar/reference/data-types/int-uint)) — عدد العقد الفرعية لعقدة ZooKeeper هذه.
* `children` ([Array(String)](/ar/reference/data-types/array)) — قائمة عقد ZooKeeper الفرعية (في استجابات طلب LIST).

{/*AUTOGENERATED_END*/}
