<div id="description">
  ## الوصف
</div>

يتضمن قائمة بمصادر القواميس التي يدعمها الخادم، إلى جانب الوثائق المضمّنة لكل مصدر. ويحدّد مصدر القاموس الجهة التي تُحمَّل منها بيانات القاموس؛ ويُحدَّد ذلك في عبارة `SOURCE` من استعلام `CREATE DICTIONARY`.

<div id="columns">
  ## الأعمدة
</div>

{/*AUTOGENERATED_START*/}

* `name` ([String](/ar/reference/data-types/index)) — اسم مصدر القاموس، كما هو محدد في عبارة SOURCE.
* `description` ([String](/ar/reference/data-types/index)) — وصف عام لما يقوم به مصدر القاموس.
* `syntax` ([String](/ar/reference/data-types/index)) — صيغة عبارة SOURCE المستخدمة لتحديد المصدر. لاحظ أن بعض المصادر تخضع لضوابط الوصول عند إنشاء قاموس من استعلام DDL (بخلاف ملف إعدادات الخادم)؛ راجع `description` الخاص بكل مصدر على حدة للاطلاع على التفاصيل.
* `examples` ([String](/ar/reference/data-types/index)) — أمثلة على الاستخدام.
* `introduced_in` ([String](/ar/reference/data-types/index)) — إصدار ClickHouse الذي ظهر فيه المصدر لأول مرة، بصيغة major.minor.
* `related` ([Array(String)](/ar/reference/data-types/index)) — أسماء مصادر القاموس ذات الصلة.

{/*AUTOGENERATED_END*/}

<div id="example">
  ## مثال
</div>

```sql title="Query"
SELECT name, syntax
FROM system.dictionary_sources
WHERE name IN ('clickhouse', 'file')
ORDER BY name
```

```text title="Response"
┌─name───────┬─syntax─────────────────────────────────────────────────────────────────────────┐
│ clickhouse │ SOURCE(CLICKHOUSE(host 'host' port 9000 user 'default' password '' db 'db' table 'table')) │
│ file       │ SOURCE(FILE(path '/path/to/file' format 'CSV'))                                  │
└────────────┴────────────────────────────────────────────────────────────────────────────────┘
```
