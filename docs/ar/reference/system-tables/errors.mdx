<div id="description">
  ## الوصف
</div>

يتضمن رموز الخطأ مع عدد مرات حدوثها.

لعرض جميع رموز الخطأ الممكنة، بما في ذلك تلك التي لم تحدث، اضبط الإعداد [system&#95;events&#95;show&#95;zero&#95;values](/ar/reference/settings/session-settings#system_events_show_zero_values) على 1.

<div id="columns">
  ## الأعمدة
</div>

{/*AUTOGENERATED_START*/}

* `name` ([String](/ar/reference/data-types/index)) — اسم الخطأ (errorCodeToName).
* `code` ([Int32](/ar/reference/data-types/index)) — رمز الخطأ.
* `value` ([UInt64](/ar/reference/data-types/index)) — عدد المرات التي حدث فيها هذا الخطأ.
* `last_error_time` ([DateTime](/ar/reference/data-types/index)) — الوقت الذي وقع فيه آخر خطأ.
* `last_error_message` ([String](/ar/reference/data-types/index)) — رسالة آخر خطأ.
* `last_error_format_string` ([String](/ar/reference/data-types/index)) — سلسلة التنسيق لآخر خطأ.
* `last_error_trace` ([Array(UInt64)](/ar/reference/data-types/index)) — تتبّع مكدس يمثّل قائمة بالعناوين الفعلية التي توجد عندها الأساليب المُستدعاة.
* `remote` ([UInt8](/ar/reference/data-types/index)) — استثناء بعيد (أي تم استلامه أثناء أحد الاستعلامات الموزعة).
* `query_id` ([String](/ar/reference/data-types/index)) — معرّف الاستعلام الذي تسبّب في الخطأ (إن كان متاحًا).

{/*AUTOGENERATED_END*/}

<Note>
  قد تزداد قيم العدادات لبعض الأخطاء أثناء تنفيذ الاستعلام بنجاح. لا يُنصح باستخدام هذا الجدول لأغراض مراقبة الخادم ما لم تكن متأكدًا من أن الخطأ المقابل لا يمكن أن يكون إيجابية كاذبة.
</Note>

<div id="example">
  ## مثال
</div>

```sql title="Query"
SELECT name, code, value
FROM system.errors
WHERE value > 0
ORDER BY code ASC
LIMIT 1

┌─name─────────────┬─code─┬─value─┐
│ CANNOT_OPEN_FILE │   76 │     1 │
└──────────────────┴──────┴───────┘
```

```sql title="Response"
WITH arrayMap(x -> demangle(addressToSymbol(x)), last_error_trace) AS all
SELECT name, arrayStringConcat(all, '\n') AS res
FROM system.errors
LIMIT 1
SETTINGS allow_introspection_functions=1\G
```
