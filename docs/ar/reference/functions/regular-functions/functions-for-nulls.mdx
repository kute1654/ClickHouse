<div id="assumeNotNull">
  ## assumeNotNull
</div>

قُدِّمت في: v1.1.0

تعيد القيمة غير `Nullable` المقابلة لقيمة من النوع [`Nullable`](/ar/reference/data-types/nullable).
إذا كانت القيمة الأصلية هي `NULL`، فقد تُعاد نتيجة غير محددة.

انظر أيضًا: الدالتان [`ifNull`](#ifNull) و[`coalesce`](#coalesce).

**الصيغة**

```sql
assumeNotNull(x)
```

**الوسيطات**

* `x` — القيمة الأصلية لأي نوع Nullable. [`Nullable(T)`](/ar/reference/data-types/nullable)

**القيمة المُعادة**

يعيد قيمة غير قابلة لأن تكون `NULL` إذا لم تكن القيمة الأصلية `NULL`، وإلا فيُعيد قيمة عشوائية إذا كانت قيمة الإدخال `NULL`. [`Any`](/ar/reference/data-types/index)

**أمثلة**

**مثال على الاستخدام**

```sql title=Query
CREATE TABLE t_null (x Int8, y Nullable(Int8))
ENGINE=MergeTree()
ORDER BY x;

INSERT INTO t_null VALUES (1, NULL), (2, 3);

SELECT assumeNotNull(y) FROM table;
SELECT toTypeName(assumeNotNull(y)) FROM t_null;
```

```response title=Response
┌─assumeNotNull(y)─┐
│                0 │
│                3 │
└──────────────────┘
┌─toTypeName(assumeNotNull(y))─┐
│ Int8                         │
│ Int8                         │
└──────────────────────────────┘
```

<div id="coalesce">
  ## coalesce
</div>

أُضيفت في: v1.1.0

تُرجِع الوسيطة الأولى غير `NULL` من اليسار.

**الصياغة**

```sql
coalesce(x[, y, ...])
```

**الوسائط**

* `x[, y, ...]` — أي عدد من الوسائط من نوع غير مركب. يجب أن تكون جميع الوسائط من أنواع بيانات متوافقة فيما بينها. [`Any`](/ar/reference/data-types/index)

**القيمة المعادة**

تعيد أول وسيط غير `NULL`، وإلا فتعيد `NULL` إذا كانت جميع الوسائط `NULL`. [`Any`](/ar/reference/data-types/index) أو [`NULL`](/ar/reference/syntax#null)

**أمثلة**

**مثال على الاستخدام**

```sql title=Query
-- Consider a list of contacts that may specify multiple ways to contact a customer.

CREATE TABLE aBook
(
    name String,
    mail Nullable(String),
    phone Nullable(String),
    telegram Nullable(UInt32)
)
ENGINE = MergeTree
ORDER BY tuple();

INSERT INTO aBook VALUES ('client 1', NULL, '123-45-67', 123), ('client 2', NULL, NULL, NULL);

-- The mail and phone fields are of type String, but the telegram field is UInt32 so it needs to be converted to String.

-- Get the first available contact method for the customer from the contact list

SELECT name, coalesce(mail, phone, CAST(telegram,'Nullable(String)')) FROM aBook;
```

```response title=Response
┌─name─────┬─coalesce(mail, phone, CAST(telegram, 'Nullable(String)'))─┐
│ client 1 │ 123-45-67                                                 │
│ client 2 │ ᴺᵁᴸᴸ                                                      │
└──────────┴───────────────────────────────────────────────────────────┘
```

<div id="firstNonDefault">
  ## firstNonDefault
</div>

استُحدث في: v25.9.0

يعيد أول قيمة غير افتراضية من مجموعة من المعاملات

**البنية**

```sql
firstNonDefault(arg1[, arg2[ ...]])
```

**الوسائط**

* `arg1` — الوسيط الأول المطلوب التحقق منه - `arg2` — الوسيط الثاني المطلوب التحقق منه - `...` — وسائط إضافية مطلوب التحقق منها

**القيمة المُعادة**

نوع النتيجة هو النوع الأعلى لجميع الوسائط

**أمثلة**

**الأعداد الصحيحة**

```sql title=Query
SELECT firstNonDefault(0, 1, 2)
```

```response title=Response
1
```

**سلاسل نصية**

```sql title=Query
SELECT firstNonDefault('', 'hello', 'world')
```

```response title=Response
'hello'
```

**قيم NULL**

```sql title=Query
SELECT firstNonDefault(NULL, 0 :: UInt8, 1 :: UInt8)
```

```response title=Response
1
```

**صفر من النوع Nullable**

```sql title=Query
SELECT firstNonDefault(NULL, 0 :: Nullable(UInt8), 1 :: Nullable(UInt8))
```

```response title=Response
0
```

<div id="ifNull">
  ## ifNull
</div>

أُضيف في: v1.1.0

يعيد قيمة بديلة إذا كانت الوسيطة الأولى `NULL`.

**الصياغة**

```sql
ifNull(x, alt)
```

**المعاملات**

* `x` — القيمة المطلوب التحقق من كونها `NULL`. [`Any`](/ar/reference/data-types/index)
* `alt` — القيمة التي تُرجعها الدالة إذا كانت `x` تساوي `NULL`. [`Any`](/ar/reference/data-types/index)

**القيمة المُعادة**

تُرجع قيمة `x` إذا لم تكن `NULL`، وإلا فتُرجع `alt`. [`Any`](/ar/reference/data-types/index)

**أمثلة**

**مثال على الاستخدام**

```sql title=Query
SELECT ifNull('a', 'b'), ifNull(NULL, 'b');
```

```response title=Response
┌─ifNull('a', 'b')─┬─ifNull(NULL, 'b')─┐
│ a                │ b                 │
└──────────────────┴───────────────────┘
```

<div id="isNotNull">
  ## isNotNull
</div>

أُضيف في: v1.1.0

يتحقق مما إذا كانت الوسيطة لا تساوي `NULL`.

انظر أيضًا: العامل [`IS NOT NULL`](/ar/reference/operators/index#is_not_null).

**البنية**

```sql
isNotNull(x)
```

**المعاملات**

* `x` — قيمة من نوع بيانات غير مركّب. [`Any`](/ar/reference/data-types/index)

**القيمة المُعادة**

تُرجع `1` إذا كانت `x` ليست `NULL`، وإلا فتُرجع `0`. [`UInt8`](/ar/reference/data-types/int-uint)

**أمثلة**

**مثال على الاستخدام**

```sql title=Query
CREATE TABLE t_null
(
  x Int32,
  y Nullable(Int32)
)
ENGINE = MergeTree
ORDER BY tuple();

INSERT INTO t_null VALUES (1, NULL), (2, 3);

SELECT x FROM t_null WHERE isNotNull(y);
```

```response title=Response
┌─x─┐
│ 2 │
└───┘
```

<div id="isNullable">
  ## isNullable
</div>

قُدِّم في: v22.7.0

يتحقق مما إذا كان نوع بيانات المعامل هو `Nullable` (أي إنه يسمح بقيم `NULL`).

**البنية**

```sql
isNullable(x)
```

**الوسائط**

* `x` — قيمة من أي نوع بيانات. [`Any`](/ar/reference/data-types/index)

**القيمة المعادة**

تعيد `1` إذا كانت `x` من نوع البيانات `Nullable`، وإلا فتعيد `0`. [`UInt8`](/ar/reference/data-types/int-uint)

**أمثلة**

**مثال للاستخدام**

```sql title=Query
CREATE TABLE tab (
    ordinary_col UInt32,
    nullable_col Nullable(UInt32)
)
ENGINE = MergeTree
ORDER BY tuple();
INSERT INTO tab (ordinary_col, nullable_col) VALUES (1,1), (2, 2), (3,3);
SELECT isNullable(ordinary_col), isNullable(nullable_col) FROM tab;
```

```response title=Response
┌───isNullable(ordinary_col)──┬───isNullable(nullable_col)──┐
│                           0 │                           1 │
│                           0 │                           1 │
│                           0 │                           1 │
└─────────────────────────────┴─────────────────────────────┘
```

```sql title=Query
CREATE TABLE t_null
```

```response title=Response

│ 2 │
└───┘
```

<div id="nullIf">
  ## nullIf
</div>

أُضيفت في: v1.1.0

تُرجِع `NULL` إذا تساوى الوسيطان.

**الصياغة**

```sql
nullIf(x, y)
```

**المعاملات**

* `x` — القيمة الأولى. [`Any`](/ar/reference/data-types/index)
* `y` — القيمة الثانية. [`Any`](/ar/reference/data-types/index)

**القيمة المُعادة**

يعيد `NULL` إذا كان المعاملان متساويين، وإلا فيعيد المعامل الأول. [`NULL`](/ar/reference/syntax#null) أو [`Nullable(x)`](/ar/reference/data-types/nullable)

**أمثلة**

**مثال على الاستخدام**

```sql title=Query
SELECT nullIf(1, 1), nullIf(1, 2);
```

```response title=Response
┌─nullIf(1, 1)─┬─nullIf(1, 2)─┐
│         ᴺᵁᴸᴸ │            1 │
└──────────────┴──────────────┘
```

<div id="toNullable">
  ## toNullable
</div>

أُضيف في: v1.1.0

يحوّل نوع الوسيطة المُقدَّمة إلى `Nullable`.

**الصيغة**

```sql
toNullable(x)
```

**الوسائط**

* `x` — قيمة من أي نوع غير مركب. [`Any`](/ar/reference/data-types/index)

**القيمة المُعادة**

تعيد قيمة الإدخال، ولكن بنوع `Nullable`. [`Nullable(Any)`](/ar/reference/data-types/nullable)

**أمثلة**

**مثال على الاستخدام**

```sql title=Query
SELECT toTypeName(10), toTypeName(toNullable(10));
```

```response title=Response
┌─toTypeName(10)─┬─toTypeName(toNullable(10))─┐
│ UInt8          │ Nullable(UInt8)            │
└────────────────┴────────────────────────────┘
```
