<div id="grouping">
  ## GROUPING
</div>

يُعدّ [ROLLUP](/ar/reference/statements/select/group-by#rollup-modifier) و[CUBE](/ar/reference/statements/select/group-by#cube-modifier) معدّلين لـ GROUP BY. وكلاهما يحسب المجاميع الفرعية. يأخذ ROLLUP قائمة مرتّبة من الأعمدة، على سبيل المثال `(day, month, year)`، ويحسب المجاميع الفرعية عند كل مستوى من مستويات التجميع ثم المجموع الكلي. أما CUBE فيحسب المجاميع الفرعية لجميع التركيبات الممكنة للأعمدة المحددة. وتحدّد GROUPING أيّ الصفوف التي يعيدها ROLLUP أو CUBE هي تجميعات عليا، وأيّها صفوف كان سيعيدها GROUP BY من دون تعديل.

تأخذ الدالة GROUPING عدة أعمدة كوسيط، وتُرجع قناع بتات.

* تشير `1` إلى أن الصف الذي يعيده معدّل `ROLLUP` أو `CUBE` لـ `GROUP BY` هو مجموع فرعي
* تشير `0` إلى أن الصف الذي يعيده `ROLLUP` أو `CUBE` هو صف ليس مجموعًا فرعيًا

<div id="grouping-sets">
  ## GROUPING SETS
</div>

افتراضيًا، يحسب المُعدِّل CUBE المجاميع الفرعية لجميع التركيبات الممكنة للأعمدة المُمرَّرة إلى CUBE. ويتيح لك GROUPING SETS تحديد التركيبات التي تريد حسابها تحديدًا.

يُعدّ تحليل البيانات الهرمية حالة استخدام مناسبة لمُعدِّلات ROLLUP وCUBE وGROUPING SETS. المثال هنا عبارة عن جدول يحتوي على بيانات حول توزيعة Linux وإصدارها المثبَّتين عبر مركزي بيانات. وقد يكون من المفيد عرض البيانات بحسب التوزيعة والإصدار والموقع.

<div id="load-sample-data">
  ### تحميل بيانات تجريبية
</div>

```sql
CREATE TABLE servers ( datacenter VARCHAR(255),
                         distro VARCHAR(255) NOT NULL,
                         version VARCHAR(50) NOT NULL,
                         quantity INT
                       )
                        ORDER BY (datacenter, distro, version)
```

<div id="comparing-cube-with-grouping-sets">
  ### مقارنة CUBE مع GROUPING SETS
</div>

يوفّر CUBE في الاستعلام التالي، `CUBE(datacenter,distro,version)`، تدرجًا هرميًا قد لا يكون منطقيًا. فلا معنى للنظر إلى إصدار عبر التوزيعتين، لأن Arch وRHEL لا يتبعان دورة إصدارات واحدة ولا معايير تسمية الإصدارات نفسها. ويُعد مثال GROUPING SETS الوارد بعد هذا المثال أنسب، لأنه يجمع `distro` و`version` في المجموعة نفسها.

```sql
SELECT
   datacenter,
   distro,
   version,
   SUM(quantity)
FROM
   servers
GROUP BY
   CUBE(datacenter,distro,version)
ORDER BY
   datacenter,
   distro;
```
