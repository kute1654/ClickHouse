يتضمن ClickStack مكتبة من قوالب لوحات المعلومات الجاهزة التي تتيح لك رؤية فورية للمقاييس الشائعة للبنية التحتية والتطبيقات.

<div id="browsing-templates">
  ## استعراض القوالب المتاحة
</div>

لاستعراض مكتبة القوالب المدمجة، انتقل إلى **لوحات المعلومات** وانقر على **استعراض قوالب لوحات المعلومات**.

<Image img="/images/use-cases/observability/browse-dashboard-template.webp" alt="زر استعراض قوالب لوحات المعلومات" size="lg" />

سيؤدي ذلك إلى فتح معرض القوالب، حيث تُنظَّم القوالب حسب الفئة. انقر على **استيراد** لبدء عملية استيراد هذا القالب.

<Image img="/images/use-cases/observability/dashboard-template-gallery.webp" alt="معرض قوالب لوحات المعلومات" size="lg" />

<div id="importing-a-template">
  ## استيراد قالب
</div>

لاستيراد قالب، يجب ضبط مصدر بيانات لكل تصوّر في لوحة المعلومات. حدِّد مصدر بيانات من القائمة المنسدلة لكل تصوّر، ثم انقر على `Finish Import`.

<Image img="/images/use-cases/observability/import-dashboard-template.webp" alt="استيراد قالب لوحة المعلومات" size="lg" />

<div id="pre-built-templates">
  ## القوالب المُعدّة مسبقًا
</div>

<div id="otel-runtime-metrics">
  ### مقاييس وقت التشغيل لـ OTel
</div>

صُممت قوالب OTel Runtime Metrics المضمنة للتطبيقات المزوّدة بـ [مقاييس وقت تشغيل OpenTelemetry](https://opentelemetry.io/docs/specs/semconv/runtime/).

| القالب                      | الوصف                                                                                  |
| --------------------------- | -------------------------------------------------------------------------------------- |
| **.NET Runtime Metrics**    | عمليات GC، وحجم heap، واستخدام thread pool، وعدد assemblies لتطبيقات .NET              |
| **Go Runtime Metrics**      | عدد Goroutines، وأزمنة توقّف GC، واستخدام heap، وإحصاءات الذاكرة لتطبيقات Go           |
| **JVM Runtime Metrics**     | ذاكرة heap وnon-heap، ومدة GC، وعدد threads، وتحميل classes للتطبيقات المستندة إلى JVM |
| **Node.js Runtime Metrics** | تأخر event loop، واستخدام heap، واستخدام CPU، وذاكرة V8 لتطبيقات Node.js               |

ملاحظات:

* يُضبط كل قالب باستخدام [تصفية مخصّصة](/ar/clickstack/features/dashboards/overview#custom-filters) للخدمات التي تتطابق فيها سمة المورد [`telemetry.sdk.language`](https://opentelemetry.io/docs/specs/semconv/registry/attributes/telemetry/#telemetry-sdk-language) مع بيئة التشغيل الخاصة بلوحة المعلومات.
  * قد تحتاج البيئات التي تستخدم مخططات مخصّصة لجداول المقاييس في ClickHouse إلى تعديل هذه التصفية للاستعلام عن العمودين الصحيحين: اسم الخدمة وسمات المورد.
  * في البيئات ذات الحجم الكبير، يمكن تقليل أزمنة تحميل التصفية من خلال [تجسيد](/ar/clickstack/managing/performance-tuning#materialize-frequently-queried-attributes) العمود `ResourceAttributes['telemetry.sdk.language']`.
* تعتمد القوالب أحدث OTel Semantic Conventions وقت نشرها، ويجري تحديثها دوريًا مع تحديث مواصفة OTel. وبالنسبة إلى الخدمات المزوّدة بإصدارات أقدم من OTel SDKs، فقد يلزم [تحرير](/ar/clickstack/features/dashboards/overview#dashboards-editing-visualizations) التصورات للإشارة إلى أسماء مقاييس أقدم.
