على غرار [Star Schema Benchmark (SSB)](/ar/get-started/sample-datasets/star-schema)، يستند TPC-DS إلى [TPC-H](/ar/get-started/sample-datasets/tpch)، لكنه اتخذ المسار المعاكس، أي زاد عدد عمليات الربط المطلوبة عبر تخزين البيانات في مخطط ندفة ثلجية معقد (24 جدولًا بدلًا من 8).
توزيع البيانات غير متوازن (على سبيل المثال، التوزيع الطبيعي وتوزيع بواسون).
ويتضمن 99 استعلامًا للتقارير واستعلامات مخصّصة مع استبدالات عشوائية.

**المراجع**

* [The Making of TPC-DS](https://dl.acm.org/doi/10.5555/1182635.1164217) (Nambiar)، 2006

<div id="data-generation-and-import">
  ## توليد البيانات واستيرادها
</div>

أولاً، استنسخ مستودع TPC-DS ثم قم بتجميع مولِّد البيانات:

```bash
git clone https://github.com/gregrahn/tpcds-kit.git
cd tpcds-kit/tools
make
```

بعد ذلك، أنشئ البيانات. تُحدِّد المَعلمة `-scale` معامل القياس.

```bash
./dsdgen -scale 1
```

الآن أنشئ الجداول في ClickHouse. تتوفر تعريفات الجداول في [`init.sql`](https://github.com/ClickHouse/ClickHouse/blob/master/tests/benchmarks/tpc-ds/init.sql) في مستودع ClickHouse.

يمكن استيراد البيانات كما يلي:

clickhouse-client --format_csv_delimiter '|' --query "INSERT INTO catalog_returns FORMAT CSV" < catalog_returns.dat
clickhouse-client --format_csv_delimiter '|' --query "INSERT INTO catalog_sales FORMAT CSV" < catalog_sales.dat

<div id="queries">
  ## الاستعلامات
</div>

يمكن العثور على استعلامات TPC-DS التسعة والتسعين [هنا](https://github.com/ClickHouse/ClickHouse/tree/master/tests/benchmarks/tpc-ds/queries) ضمن مستودع ClickHouse.

للحصول على سلوك متوافق مع معيار SQL والنتائج المتوقعة، طبّق الإعدادات الواردة في [`settings.json`](https://github.com/ClickHouse/ClickHouse/blob/master/tests/benchmarks/tpc-ds/settings.json).
راجع ملف [README](https://github.com/ClickHouse/ClickHouse/blob/master/tests/benchmarks/tpc-ds/README.md) للاطلاع على المشكلات المعروفة والملاحظات المتعلقة ببعض الاستعلامات.

**صحة النتائج**

تتوافق نتائج الاستعلامات مع النتائج الرسمية ما لم يُذكر خلاف ذلك. وقد توجد فروق طفيفة في الدقة، وهو ما تسمح به مواصفة TPC-DS.
