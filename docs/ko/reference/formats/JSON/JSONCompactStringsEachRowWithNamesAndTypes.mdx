<div id="description">
  ## 설명
</div>

`JSONCompactEachRow` 포맷과의 차이점은 [TabSeparatedWithNamesAndTypes](/ko/reference/formats/TabSeparated/TabSeparatedRawWithNamesAndTypes)와 마찬가지로 컬럼 이름과 타입을 담은 2개의 헤더 행도 출력한다는 점입니다.

</div>

<div id="format-settings">
  ## 포맷 설정
</div>

<Note>
  설정 [input&#95;format&#95;with&#95;names&#95;use&#95;header](/ko/reference/settings/formats#input_format_with_names_use_header)가 1로 설정되면,
  입력 데이터의 컬럼이 이름을 기준으로 테이블의 컬럼에 매핑되며, 설정 [input&#95;format&#95;skip&#95;unknown&#95;fields](/ko/reference/settings/formats#input_format_skip_unknown_fields)가 1로 설정된 경우 알 수 없는 이름의 컬럼은 건너뜁니다.
  그렇지 않으면 첫 번째 행을 건너뜁니다.
</Note>

<Note>
  설정 [input&#95;format&#95;with&#95;types&#95;use&#95;header](/ko/reference/settings/formats#input_format_with_types_use_header)가 1로 설정되면,
  입력 데이터의 타입을 테이블의 해당 컬럼 타입과 비교합니다. 그렇지 않으면 두 번째 행을 건너뜁니다.
</Note>
