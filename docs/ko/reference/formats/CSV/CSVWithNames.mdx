<div id="description">
  ## 설명
</div>

[TabSeparatedWithNames](/ko/reference/formats/TabSeparated/TabSeparatedWithNames)와 마찬가지로 컬럼 이름이 포함된 헤더 행도 출력합니다.

<div id="example-usage">
  ## 사용 예시
</div>

<div id="inserting-data">
  ### 데이터 삽입
</div>

<Tip>
  [version](https://github.com/ClickHouse/ClickHouse/releases) 23.1부터 ClickHouse는 `CSV` 포맷 사용 시 CSV 파일의 헤더를 자동으로 감지하므로 `CSVWithNames` 또는 `CSVWithNamesAndTypes`를 사용할 필요가 없습니다.
</Tip>


테이블을 생성하세요:

```sql
CREATE TABLE football
(
    `date` Date,
    `season` Int16,
    `home_team` LowCardinality(String),
    `away_team` LowCardinality(String),
    `home_team_goals` Int8,
    `away_team_goals` Int8
)
ENGINE = MergeTree
ORDER BY (date, home_team);
```

`CSVWithNames` 포맷을 사용해 데이터를 삽입합니다:

```sql
INSERT INTO football FROM INFILE 'football.csv' FORMAT CSVWithNames;
```

</div>

`CSVWithNames` 포맷을 사용해 데이터를 읽습니다:

```sql
SELECT *
FROM football
FORMAT CSVWithNames
```

<div id="format-settings">
  ## 포맷 설정
</div>

<Note>
  [`input_format_with_names_use_header`](/ko/reference/settings/formats#input_format_with_names_use_header) 설정이 `1`로 지정되면,
  입력 데이터의 컬럼이 이름을 기준으로 테이블의 컬럼에 매핑되며, [input&#95;format&#95;skip&#95;unknown&#95;fields](/ko/reference/settings/formats#input_format_skip_unknown_fields) 설정이 `1`이면 알 수 없는 이름의 컬럼은 건너뜁니다.
  그렇지 않으면 첫 번째 행을 건너뜁니다.
</Note>
