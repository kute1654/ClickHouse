<div id="description">
  ## 설명
</div>

행을 이스케이프 처리하지 않고 기록한다는 점에서 [`TabSeparated`](/ko/reference/formats/TabSeparated/TabSeparated) 포맷과 다릅니다.

<Note>
  이 포맷으로 파싱할 때는 각 필드에 탭 또는 줄바꿈 문자를 사용할 수 없습니다.
</Note>

`TabSeparatedRaw` 포맷과 `RawBlob` 포맷의 비교는 다음을 참조하십시오: [Raw Formats Comparison](/ko/reference/formats/RawBLOB#raw-formats-comparison)

```sql

FROM football
FORMAT TabSeparatedRaw
```
