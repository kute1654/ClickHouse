<div id="description">
  ## 설명
</div>

`Pretty` 포맷은 데이터를 유니코드 아트 표 형태로 출력하며,
터미널에 색상을 표시하기 위해 ANSI 이스케이프 시퀀스를 사용합니다.
표의 전체 격자를 그리며, 각 행은 터미널에서 두 줄을 차지합니다.
각 결과 블록은 각각 별도의 표로 출력됩니다.
이 방식은 결과를 버퍼링하지 않고 블록을 출력하기 위해 필요합니다(모든 값의 표시 너비를 미리 계산하려면 버퍼링이 필요합니다).

[NULL](/ko/reference/syntax)은 `ᴺᵁᴸᴸ`로 출력됩니다.

<div id="example-usage">
  ## 사용 예시
</div>

예시([`PrettyCompact`](/ko/reference/formats/Pretty/PrettyCompact) 포맷):

```sql title="Query"
SELECT * FROM t_null
```

```response title="Response"
┌─x─┬────y─┐
│ 1 │ ᴺᵁᴸᴸ │
└───┴──────┘
```

`Pretty` 포맷에서는 행이 어떤 경우에도 이스케이프되지 않습니다. 다음은 [`PrettyCompact`](/ko/reference/formats/Pretty/PrettyCompact) 포맷의 예시입니다:

```sql title="Query"
SELECT 'String with \'quotes\' and \t character' AS Escaping_test
```

```response title="Response"
┌─Escaping_test────────────────────────┐
│ String with 'quotes' and      character │
└──────────────────────────────────────┘
```

터미널에 너무 많은 데이터가 출력되지 않도록, 처음 `10,000`개 행만 출력됩니다.
행 수가 `10,000`개 이상이면 &quot;Showed first 10 000&quot; 메시지가 출력됩니다.

<Note>
  이 포맷은 쿼리 결과를 출력하는 데만 적합하며, 데이터를 파싱하는 용도로는 적합하지 않습니다.
</Note>

Pretty 형식은 합계 값(`WITH TOTALS`를 사용하는 경우)과 극값(&#39;extremes&#39;가 1로 설정된 경우) 출력도 지원합니다.
이 경우 합계 값과 극값은 주요 데이터 뒤에 별도의 테이블로 출력됩니다.
이는 [`PrettyCompact`](/ko/reference/formats/Pretty/PrettyCompact) 포맷을 사용하는 다음 예시에서 확인할 수 있습니다:

```sql title="Query"
SELECT EventDate, count() AS c 
FROM test.hits 
GROUP BY EventDate 
WITH TOTALS 
ORDER BY EventDate 
FORMAT PrettyCompact
```

```response title="Response"
┌──EventDate─┬───────c─┐
│ 2014-03-17 │ 1406958 │
│ 2014-03-18 │ 1383658 │
│ 2014-03-19 │ 1405797 │
│ 2014-03-20 │ 1353623 │
│ 2014-03-21 │ 1245779 │
│ 2014-03-22 │ 1031592 │
│ 2014-03-23 │ 1046491 │
└────────────┴─────────┘

합계:
┌──EventDate─┬───────c─┐
│ 1970-01-01 │ 8873898 │
└────────────┴─────────┘

극값:
┌──EventDate─┬───────c─┐
│ 2014-03-17 │ 1031592 │
│ 2014-03-23 │ 1406958 │
└────────────┴─────────┘
```
