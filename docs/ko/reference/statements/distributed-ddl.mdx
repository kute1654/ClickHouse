기본적으로 `CREATE`, `DROP`, `ALTER`, `RENAME` 쿼리는 쿼리가 실행된 현재 서버에만 영향을 줍니다. 클러스터 구성에서는 `ON CLUSTER` 절을 사용해 이러한 쿼리를 분산 방식으로 실행할 수 있습니다.

예를 들어, 다음 쿼리는 `cluster`의 각 호스트에 `all_hits` `Distributed` 테이블을 생성합니다:

```sql
CREATE TABLE IF NOT EXISTS all_hits ON CLUSTER cluster (p Date, i Int32) ENGINE = Distributed(cluster, default, hits)
```

이 쿼리를 올바르게 실행하려면 각 호스트에 동일한 클러스터(cluster) 정의가 있어야 합니다(구성 동기화를 간소화하려면 ZooKeeper의 치환 기능을 사용할 수 있습니다). 또한 ZooKeeper 서버에도 연결되어 있어야 합니다.

현재 일부 호스트를 사용할 수 없더라도, 쿼리의 로컬 버전은 결국 클러스터의 각 호스트에서 실행됩니다.

<Warning>
  단일 호스트 내에서 쿼리가 실행되는 순서는 보장됩니다.
</Warning>
