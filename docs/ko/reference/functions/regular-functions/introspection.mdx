이 장에 설명된 함수를 사용하면 쿼리 프로파일링을 위해 [ELF](https://en.wikipedia.org/wiki/Executable_and_Linkable_Format)와 [DWARF](https://en.wikipedia.org/wiki/DWARF)를 검사할 수 있습니다.

<Note>
  이 함수는 느리며 보안 측면에서 고려가 필요할 수 있습니다.
</Note>

인트로스펙션 함수가 제대로 작동하려면 다음이 필요합니다.

* `clickhouse-common-static-dbg` 패키지를 설치합니다.

* [allow&#95;introspection&#95;functions](/ko/reference/settings/session-settings#allow_introspection_functions) 설정을 1로 지정합니다.

  보안상의 이유로 인트로스펙션 함수는 기본적으로 비활성화되어 있습니다.

ClickHouse는 프로파일러 보고서를 [trace&#95;log](/ko/reference/system-tables/trace_log) 시스템 테이블에 저장합니다. 테이블과 프로파일러가 올바르게 구성되어 있는지 확인하십시오.

<div id="addressToLine">
  ## addressToLine
</div>

도입 버전: v20.1.0

ClickHouse 서버 프로세스 내부의 가상 메모리 주소를 ClickHouse 소스 코드의 파일 이름과 줄 번호로 변환합니다.

<Note>
  이 함수는 느리며 보안상 문제가 될 수 있습니다.
</Note>

이 내부 검사 함수를 활성화하려면:

* `clickhouse-common-static-dbg` 패키지를 설치합니다.
* [`allow_introspection_functions`](/ko/reference/settings/session-settings#allow_introspection_functions) 설정을 `1`로 지정합니다.

**구문**

```sql
addressToLine(address_of_binary_instruction)
```

**인수**

* `address_of_binary_instruction` — 실행 중인 프로세스의 명령어 주소입니다. [`UInt64`](/ko/reference/data-types/int-uint)

**반환 값**

소스 코드 파일명과 줄 번호를 콜론으로 구분해 반환합니다. 예를 들면 `/build/obj-x86_64-linux-gnu/../src/Common/ThreadPool.cpp:199`와 같습니다. 디버그 정보를 찾을 수 없으면 바이너리 이름을 반환하며, 주소가 유효하지 않으면 빈 문자열을 반환합니다. [`String`](/ko/reference/data-types/string)

**예시**

**`trace_log` 시스템 테이블에서 첫 번째 문자열 조회하기**

<div id="addressToLineWithInlines">
  ## addressToLineWithInlines
</div>

도입 버전: v22.2.0

`addressToLine`과 유사하지만, 모든 인라인 함수가 포함된 배열을 반환합니다.
이 때문에 `addressToLine`보다 더 느립니다.

이 내부 검사 함수를 활성화하려면 다음을 수행하십시오.

* `clickhouse-common-static-dbg` 패키지를 설치합니다.
* [`allow_introspection_functions`](/ko/reference/settings/session-settings#allow_introspection_functions) 설정을 `1`로 지정합니다.

**구문**

```sql
addressToLineWithInlines(address_of_binary_instruction)
```

**인수**

* `address_of_binary_instruction` — 실행 중인 프로세스에 있는 명령어의 주소입니다. [`UInt64`](/ko/reference/data-types/int-uint)

**반환 값**

첫 번째 요소가 콜론으로 구분된 소스 코드 파일 이름과 줄 번호인 배열을 반환합니다. 두 번째, 세 번째 이후의 요소에는 인라인 함수의 소스 코드 파일 이름, 줄 번호, 함수 이름이 나열됩니다. 디버그 정보를 찾을 수 없으면 바이너리 이름만 포함된 단일 요소 배열이 반환되며, 주소가 유효하지 않으면 빈 배열이 반환됩니다. [`Array(String)`](/ko/reference/data-types/array)

**예시**

**주소에 함수를 적용하는 예시**

<div id="demangle">
  ## demangle
</div>

도입 버전: v20.1.0
