<div id="groupConcat">
  ## groupConcat
</div>

도입 버전: v24.8.0

문자열 그룹으로부터 연결된 문자열(concatenated string)을 계산합니다. 필요에 따라 구분자로 분리할 수 있으며, 요소의 최대 개수를 제한할 수도 있습니다.

<Note>
  `limit` 없이 `delimiter`만 지정하는 경우, `delimiter`는 첫 번째 매개변수여야 합니다. `delimiter`와 `limit`를 모두 지정하는 경우에는 `delimiter`가 `limit`보다 앞에 와야 합니다.

  또한 매개변수와 인수에 서로 다른 구분자를 지정한 경우에는 인수에 지정된 구분자만 사용됩니다.
</Note>

**구문**

```sql
groupConcat[(delimiter [, limit])](expression)
```

**별칭**: `group_concat`, `string_agg`

**매개변수**

* `delimiter` — 연결된 값들을 구분하는 데 사용할 문자열입니다. 이 매개변수는 선택 사항이며, 지정하지 않으면 기본값은 빈 문자열입니다. [`String`](/ko/reference/data-types/string)
* `limit` — 연결할 요소의 최대 개수를 지정하는 양의 정수입니다. 요소가 이보다 많으면 초과한 요소는 무시됩니다. 이 매개변수는 선택 사항입니다. [`UInt*`](/ko/reference/data-types/int-uint)

**인수**

* `expression` — 연결할 문자열을 반환하는 표현식 또는 컬럼 이름입니다. [`String`](/ko/reference/data-types/string)
* `delimiter` — 연결된 값들을 구분하는 데 사용할 문자열입니다. 이 인수는 선택 사항이며, 지정하지 않으면 기본값은 빈 문자열 또는 매개변수에 지정된 구분자입니다. [`String`](/ko/reference/data-types/string)

**반환 값**

컬럼 또는 표현식의 값을 연결한 문자열을 반환합니다. 그룹에 요소가 없거나 null 요소만 있고, 함수에서 null 값만 있는 경우의 처리 방식을 지정하지 않으면 결과는 null 값을 갖는 널 허용 문자열입니다. [`String`](/ko/reference/data-types/string)

**예시**

**구분자 없이 사용하는 기본 사용법**

```sql title=Query
SELECT groupConcat(Name) FROM Employees;
```

```response title=Response
JohnJaneBob
```

**구분자로 쉼표 사용하기 (매개변수 구문)**

```sql title=Query
SELECT groupConcat(', ')(Name) FROM Employees;
```

```response title=Response
John, Jane, Bob
```

**구분자로 쉼표 사용(인수 구문)**

```sql title=Query
SELECT groupConcat(Name, ', ') FROM Employees;
```

```response title=Response
John, Jane, Bob
```

**연결할 요소 수 제한**

```sql title=Query
SELECT groupConcat(', ', 2)(Name) FROM Employees;
```

```response title=Response
John, Jane
```
