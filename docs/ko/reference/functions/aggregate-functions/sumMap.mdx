<div id="sumMappedArrays">
  ## sumMappedArrays
</div>

도입 버전: v1.1.0

`key` 배열에 지정된 키를 기준으로 하나 이상의 `value` 배열을 합산합니다. 반환값은 배열의 튜플이며, 먼저 정렬된 순서의 키 배열이 오고, 이어서 각 키에 해당하는 값들을 오버플로우 없이 합산한 배열이 반환됩니다.

<Note>
  * 키 배열과 값 배열의 튜플을 전달하는 것은 키 배열과 값 배열을 각각 전달하는 것과 동일합니다.
  * 합산되는 각 행에서 `key`와 모든 `value` 배열의 요소 수는 같아야 합니다.
</Note>

**구문**

```sql
sumMappedArrays(key, value1 [, value2, ...])
sumMappedArrays(Tuple(key, value1 [, value2, ...]))
```

**인수**

* `key` — 키 배열입니다. [`배열`](/ko/reference/data-types/array)
* `value1, value2, ...` — 각 키별로 합산할 값 배열입니다. [`배열`](/ko/reference/data-types/array)

**반환 값**

배열로 이루어진 튜플을 반환합니다. 첫 번째 배열에는 키가 정렬된 순서로 들어 있으며, 그 뒤에는 각 키에 대해 합산된 값이 들어 있는 배열들이 이어집니다. [`Tuple`](/ko/reference/data-types/tuple)

**예시**

**Nested 타입의 기본 사용법**

```sql title=Query
CREATE TABLE sum_map(
    date Date,
    timeslot DateTime,
    statusMap Nested(
        status UInt16,
        requests UInt64
    ),
    statusMapTuple Tuple(Array(Int32), Array(Int32))
) ENGINE = Memory;

INSERT INTO sum_map VALUES
    ('2000-01-01', '2000-01-01 00:00:00', [1, 2, 3], [10, 10, 10], ([1, 2, 3], [10, 10, 10])),
    ('2000-01-01', '2000-01-01 00:00:00', [3, 4, 5], [10, 10, 10], ([3, 4, 5], [10, 10, 10])),
    ('2000-01-01', '2000-01-01 00:01:00', [4, 5, 6], [10, 10, 10], ([4, 5, 6], [10, 10, 10])),
    ('2000-01-01', '2000-01-01 00:01:00', [6, 7, 8], [10, 10, 10], ([6, 7, 8], [10, 10, 10]));

SELECT
    timeslot,
    sumMappedArrays(statusMap.status, statusMap.requests),
    sumMappedArrays(statusMapTuple)
FROM sum_map
GROUP BY timeslot;
```

```response title=Response
┌────────────timeslot─┬─sumMappedArrays(statusMap.status, statusMap.requests)─┬─sumMappedArrays(statusMapTuple)─────────┐
│ 2000-01-01 00:00:00 │ ([1,2,3,4,5],[10,10,20,10,10])                        │ ([1,2,3,4,5],[10,10,20,10,10])          │
│ 2000-01-01 00:01:00 │ ([4,5,6,7,8],[10,10,20,10,10])                        │ ([4,5,6,7,8],[10,10,20,10,10])          │
└─────────────────────┴───────────────────────────────────────────────────────┴─────────────────────────────────────────┘
```

**여러 값이 있는 배열 예시**

```sql title=Query
CREATE TABLE multi_metrics(
    date Date,
    browser_metrics Nested(
        browser String,
        impressions UInt32,
        clicks UInt32
    )
)
ENGINE = Memory;

INSERT INTO multi_metrics VALUES
    ('2000-01-01', ['Firefox', 'Chrome'], [100, 200], [10, 25]),
    ('2000-01-01', ['Chrome', 'Safari'], [150, 50], [20, 5]),
    ('2000-01-01', ['Firefox', 'Edge'], [80, 40], [8, 4]);

SELECT
    sumMappedArrays(browser_metrics.browser, browser_metrics.impressions, browser_metrics.clicks) AS result
FROM multi_metrics;
```

```response title=Response
┌─result────────────────────────────────────────────────────────────────────────┐
│ (['Chrome', 'Edge', 'Firefox', 'Safari'], [350, 40, 180, 50], [45, 4, 18, 5]) │
└───────────────────────────────────────────────────────────────────────────────┘
-- 이 예시에서:
-- 결과 튜플은 세 개의 배열을 포함합니다
-- 첫 번째 배열: 정렬된 순서의 키(브라우저 이름)
-- 두 번째 배열: 각 브라우저의 총 노출 수
-- 세 번째 배열: 각 브라우저의 총 클릭 수
```
