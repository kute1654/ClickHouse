<div id="timeSeriesInstantRateToGrid">
  ## timeSeriesInstantRateToGrid
</div>

도입 버전: v25.6.0

timestamp와 값의 쌍으로 이루어진 시계열 데이터를 받아, 시작 timestamp, 종료 timestamp, 간격으로 정의된 규칙적인 시간 그리드에서 이 데이터로부터 [PromQL과 유사한 irate](https://prometheus.io/docs/prometheus/latest/querying/functions/#irate)를 계산하는 집계 함수입니다. 그리드의 각 지점에서는 지정된 시간 윈도우 내의 샘플을 사용해 `irate`를 계산합니다.

<Warning>
  이 함수는 실험적 기능이므로, `allow_experimental_ts_to_grid_aggregate_function=true`를 설정하여 활성화하십시오.
</Warning>

**구문**

```sql
timeSeriesInstantRateToGrid(start_timestamp, end_timestamp, grid_step, staleness)(timestamp, value)
```

**매개변수**

* `start_timestamp` — 그리드의 시작점을 지정합니다. [`UInt32`](/ko/reference/data-types/int-uint) 또는 [`DateTime`](/ko/reference/data-types/datetime)
* `end_timestamp` — 그리드의 끝점을 지정합니다. [`UInt32`](/ko/reference/data-types/int-uint) 또는 [`DateTime`](/ko/reference/data-types/datetime)
* `grid_step` — 초 단위 그리드 간격을 지정합니다. [`UInt32`](/ko/reference/data-types/int-uint)
* `staleness` — 고려할 샘플이 초 단위로 얼마나 오래되어도 되는지의 최대값을 지정합니다. 이 staleness 윈도우는 좌측 개방, 우측 폐구간입니다. [`UInt32`](/ko/reference/data-types/int-uint)

**인수**

* `timestamp` — 샘플의 타임스탬프입니다. 개별 값 또는 배열일 수 있습니다. [`UInt32`](/ko/reference/data-types/int-uint) 또는 [`DateTime`](/ko/reference/data-types/datetime) 또는 [`Array(UInt32)`](/ko/reference/data-types/array) 또는 [`Array(DateTime)`](/ko/reference/data-types/array)
* `value` — 타임스탬프에 해당하는 시계열 값입니다. 개별 값 또는 배열일 수 있습니다. [`Float*`](/ko/reference/data-types/float) 또는 [`Array(Float*)`](/ko/reference/data-types/array)

**반환 값**

지정된 그리드에 대한 irate 값을 반환합니다. 반환되는 배열에는 각 시간 그리드 지점마다 값이 하나씩 포함됩니다. 특정 그리드 지점의 순간 rate 값을 계산하기에 윈도우 내 샘플이 충분하지 않으면 해당 값은 NULL입니다. [`Array(Nullable(Float64))`](/ko/reference/data-types/array)

**예시**

**개별 타임스탬프-값 쌍을 사용하는 기본 사용법**

```sql title=Query
WITH
    -- 참고: 140과 190 사이의 간격은 window 매개변수에 따라 ts = 150, 165, 180의 값이 어떻게 채워지는지 보여줍니다
    [110, 120, 130, 140, 190, 200, 210, 220, 230]::Array(DateTime) AS timestamps,
    [1, 1, 3, 4, 5, 5, 8, 12, 13]::Array(Float32) AS values, -- 위 타임스탬프에 대응하는 값 배열
    90 AS start_ts,       -- 타임스탬프 그리드의 시작
    90 + 120 AS end_ts,   -- 타임스탬프 그리드의 끝
    15 AS step_seconds,   -- 타임스탬프 그리드의 간격
    45 AS window_seconds  -- "staleness" 윈도우
SELECT timeSeriesInstantRateToGrid(start_ts, end_ts, step_seconds, window_seconds)(timestamp, value)
FROM
(
    -- 이 서브쿼리는 타임스탬프와 값의 배열을 `timestamp`, `value` 행으로 변환합니다
    SELECT
        arrayJoin(arrayZip(timestamps, values)) AS ts_and_val,
        ts_and_val.1 AS timestamp,
        ts_and_val.2 AS value
);
```

```response title=Response
┌─timeSeriesInstantRateToGrid(start_ts, end_ts, step_seconds, window_seconds)(timestamp, value)─┐
│ [NULL,NULL,0,0.2,0.1,0.1,NULL,NULL,0.3]                                                      │
└──────────────────────────────────────────────────────────────────────────────────────────────┘
```

**배열 인수 사용하기**

```sql title=Query
-- 동일한 크기의 배열로 여러 timestamp와 값 샘플을 전달하는 것도 가능합니다
WITH
    [110, 120, 130, 140, 190, 200, 210, 220, 230]::Array(DateTime) AS timestamps,
    [1, 1, 3, 4, 5, 5, 8, 12, 13]::Array(Float32) AS values,
    90 AS start_ts,
    90 + 120 AS end_ts,
    15 AS step_seconds,
    45 AS window_seconds
SELECT timeSeriesInstantRateToGrid(start_ts, end_ts, step_seconds, window_seconds)(timestamps, values);
```

```response title=Response
┌─timeSeriesInstantRateToGrid(start_ts, end_ts, step_seconds, window_seconds)(timestamps, values)─┐
│ [NULL,NULL,0,0.2,0.1,0.1,NULL,NULL,0.3]                                                        │
└────────────────────────────────────────────────────────────────────────────────────────────────┘
```
