<div id="contingency">
  ## contingency
</div>

도입 버전: v22.1.0

`contingency` 함수는 테이블의 두 컬럼 사이의 연관성을 측정하는 값인 [분할계수](https://en.wikipedia.org/wiki/Contingency_table#Cram%C3%A9r's_V_and_the_contingency_coefficient_C)를 계산합니다.
계산 방식은 [`cramersV`](/ko/reference/functions/aggregate-functions/cramersV) 함수와 비슷하지만, 제곱근 내 분모가 다릅니다.

**구문**

```sql
contingency(column1, column2)
```

**인수**

* `column1` — 비교할 첫 번째 컬럼입니다. [`Any`](/ko/reference/data-types/index)
* `column2` — 비교할 두 번째 컬럼입니다. [`Any`](/ko/reference/data-types/index)

**반환 값**

0과 1 사이의 값을 반환합니다. 결과가 클수록 두 컬럼 간의 연관성이 더 높습니다. [`Float64`](/ko/reference/data-types/float)

**예시**

**cramersV와의 비교**

```sql title=Query
SELECT
    cramersV(a, b),
    contingency(a, b)
FROM
(
    SELECT
        number % 10 AS a,
        number % 4 AS b
    FROM
        numbers(150)
);
```

```response title=Response
┌─────cramersV(a, b)─┬──contingency(a, b)─┐
│ 0.5798088336225178 │ 0.708607540104077  │
└────────────────────┴────────────────────┘
```
