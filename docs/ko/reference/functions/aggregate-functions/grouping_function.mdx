<div id="grouping">
  ## GROUPING
</div>

[ROLLUP](/ko/reference/statements/select/group-by#rollup-modifier)과 [CUBE](/ko/reference/statements/select/group-by#cube-modifier)는 GROUP BY의 수정자입니다. 둘 다 소계를 계산합니다. ROLLUP은 `(day, month, year)`와 같이 정렬된 컬럼 목록을 받아, 집계의 각 수준에 대한 소계와 마지막으로 전체 합계를 계산합니다. CUBE는 지정된 컬럼의 가능한 모든 조합에 대한 소계를 계산합니다. GROUPING은 ROLLUP 또는 CUBE가 반환한 행 중 어떤 것이 상위 집계 행인지, 그리고 어떤 것이 수정되지 않은 GROUP BY에서 반환되는 행인지를 식별합니다.

GROUPING 함수는 여러 컬럼을 인수로 받아 비트마스크를 반환합니다.

* `1`은 `GROUP BY`에 `ROLLUP` 또는 `CUBE` 수정자를 적용해 반환된 행이 소계임을 나타냅니다
* `0`은 `ROLLUP` 또는 `CUBE`가 반환한 행이 소계가 아님을 나타냅니다

<div id="grouping-sets">
  ## GROUPING SETS
</div>

기본적으로 CUBE 수정자는 CUBE에 전달된 컬럼의 가능한 모든 조합에 대해 소계를 계산합니다. GROUPING SETS를 사용하면 계산할 조합을 구체적으로 지정할 수 있습니다.

계층형 데이터 분석에는 ROLLUP, CUBE, GROUPING SETS 수정자가 적합합니다. 여기의 예제는 두 개의 데이터 센터에 설치된 Linux 배포판과 해당 배포판의 버전에 대한 데이터를 담고 있는 테이블입니다. 이 데이터를 배포판, 버전, 위치별로 살펴보면 유용합니다.

<div id="load-sample-data">
  ### 샘플 데이터 불러오기
</div>

```sql
CREATE TABLE servers ( datacenter VARCHAR(255),
                         distro VARCHAR(255) NOT NULL,
                         version VARCHAR(50) NOT NULL,
                         quantity INT
                       )
                        ORDER BY (datacenter, distro, version)
```

<div id="comparing-cube-with-grouping-sets">
  ### CUBE와 GROUPING SETS 비교
</div>

다음 쿼리의 CUBE인 `CUBE(datacenter,distro,version)`는 그다지 적절하지 않은 계층 구조를 만듭니다. 두 배포판 전체에 걸쳐 `version`을 살펴보는 것은 의미가 없습니다(Arch와 RHEL은 릴리스 주기나 버전 명명 기준이 서로 다르기 때문입니다). 이어지는 GROUPING SETS 예시는 `distro`와 `version`을 같은 세트로 그룹화하므로 더 적절합니다.

```sql
SELECT
   datacenter,
   distro,
   version,
   SUM(quantity)
FROM
   servers
GROUP BY
   CUBE(datacenter,distro,version)
ORDER BY
   datacenter,
   distro;
```

```sql
SELECT
    datacenter,
    distro,
    version,
    SUM(quantity)
FROM servers
GROUP BY
    GROUPING SETS (
        (datacenter, distro, version),
        (datacenter, distro))
```
