<div id="description">
  ## 설명
</div>

원격 객체용 파일 시스템 캐시에 있는 모든 항목의 정보를 포함합니다.

<div id="columns">
  ## 컬럼
</div>

{/*AUTOGENERATED_START*/}

* `cache_name` ([String](/ko/reference/data-types/index)) — 캐시 객체의 이름입니다. `SYSTEM DESCRIBE FILESYSTEM CACHE <name>`, `SYSTEM DROP FILESYSTEM CACHE <name>` 명령에서 사용할 수 있습니다.
* `cache_base_path` ([String](/ko/reference/data-types/index)) — 모든 캐시 파일(`cache_name`으로 식별되는 캐시의 파일)이 저장되는 기본 디렉터리 경로입니다.
* `cache_path` ([String](/ko/reference/data-types/index)) — 소스 파일의 file segment에 해당하는 특정 캐시 파일의 경로입니다.
* `key` ([String](/ko/reference/data-types/index)) — file segment의 캐시 키입니다.
* `file_segment_range_begin` ([UInt64](/ko/reference/data-types/index)) — file segment 범위의 시작에 해당하는 오프셋입니다.
* `file_segment_range_end` ([UInt64](/ko/reference/data-types/index)) — file segment 범위의 끝(포함)에 해당하는 오프셋입니다.
* `size` ([UInt64](/ko/reference/data-types/index)) — file segment의 크기입니다.
* `state` ([String](/ko/reference/data-types/index)) — file segment 상태입니다(DOWNLOADED, DOWNLOADING, PARTIALLY&#95;DOWNLOADED, ...).
* `finished_download_time` ([DateTime](/ko/reference/data-types/index)) — file segment 다운로드가 완료된 시각입니다.
* `cache_hits` ([UInt64](/ko/reference/data-types/index)) — 해당 file segment의 캐시 적중 횟수입니다.
* `references` ([UInt64](/ko/reference/data-types/index)) — 해당 file segment에 대한 참조 수입니다. 값이 1이면 현재 아무도 이를 사용하고 있지 않음을 의미합니다(유일한 참조는 캐시 저장소 자체에만 있습니다).
* `downloaded_size` ([UInt64](/ko/reference/data-types/index)) — file segment의 다운로드된 크기입니다.
* `kind` ([String](/ko/reference/data-types/index)) — file segment 종류입니다(&#39;Temporary data in cache&#39;의 일부로 추가된 file segment를 구분하는 데 사용됩니다).
* `unbound` ([UInt8](/ko/reference/data-types/index)) — 내부 구현 플래그입니다.
* `user_id` ([String](/ko/reference/data-types/index)) — file segment를 생성한 사용자의 사용자 id입니다.
* `segment_type` ([String](/ko/reference/data-types/index)) — 세그먼트 유형입니다. 데이터 파일(`.json`, `.txt` 등)과 데이터 파일(`.bin`, 마크 파일)을 구분하는 데 사용됩니다.
* `file_size` ([Nullable(UInt64)](/ko/reference/data-types/index)) — 현재 file segment가 속한 파일의 크기입니다.

{/*AUTOGENERATED_END*/}
