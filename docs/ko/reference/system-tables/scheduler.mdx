<div id="description">
  ## 설명
</div>

로컬 server에 있는 [스케줄링 노드](/ko/concepts/features/configuration/server-config/workload-scheduling#hierarchy)의 정보와 상태를 담고 있습니다.
이 table은 모니터링에 사용할 수 있습니다. 각 스케줄링 노드마다 하나의 행이 있습니다.

<div id="columns">
  ## 컬럼
</div>

{/*AUTOGENERATED_START*/}

* `resource` ([String](/ko/reference/data-types/index)) — 리소스 이름
* `path` ([String](/ko/reference/data-types/index)) — 이 리소스 스케줄링 계층 구조에서 해당 스케줄링 노드까지의 경로
* `type` ([String](/ko/reference/data-types/index)) — 스케줄링 노드의 유형
* `weight` ([Float64](/ko/reference/data-types/index)) — 노드의 가중치이며, `fair` 유형의 상위 노드에서 사용됩니다.
* `priority` ([Int64](/ko/reference/data-types/index)) — 노드의 우선순위이며, &#39;priority&#39; 유형의 상위 노드에서 사용됩니다(값이 낮을수록 우선순위가 높음).
* `is_active` ([UInt8](/ko/reference/data-types/index)) — 이 노드가 현재 활성 상태인지 여부입니다. 즉, 큐에서 꺼낼 리소스 요청이 있고 제약 조건이 충족된 상태를 의미합니다.
* `active_children` ([UInt64](/ko/reference/data-types/index)) — 활성 상태인 하위 노드의 수입니다.
* `dequeued_requests` ([UInt64](/ko/reference/data-types/index)) — 이 노드에서 큐에서 꺼낸 리소스 요청의 총수입니다.
* `canceled_requests` ([UInt64](/ko/reference/data-types/index)) — 이 노드에서 취소된 리소스 요청의 총수입니다.
* `dequeued_cost` ([Int64](/ko/reference/data-types/index)) — 이 노드에서 큐에서 꺼낸 모든 요청 비용(예: 바이트 단위 크기)의 합계입니다.
* `throughput` ([Float64](/ko/reference/data-types/index)) — 현재 평균 처리량(초당 큐에서 꺼낸 비용)입니다.
* `canceled_cost` ([Int64](/ko/reference/data-types/index)) — 이 노드에서 취소된 모든 요청 비용(예: 바이트 단위 크기)의 합계입니다.
* `busy_periods` ([UInt64](/ko/reference/data-types/index)) — 이 노드가 비활성화된 총횟수입니다.
* `vruntime` ([Nullable(Float64)](/ko/reference/data-types/index)) — `fair` 노드의 하위 노드에만 적용됩니다. SFQ 알고리즘이 max-min 공정 방식으로 다음에 처리할 하위 노드를 선택할 때 사용하는 노드의 가상 런타임입니다.
* `system_vruntime` ([Nullable(Float64)](/ko/reference/data-types/index)) — `fair` 노드에만 적용됩니다. 마지막으로 처리된 리소스 요청의 `vruntime`을 나타내는 가상 런타임입니다. 하위 노드를 활성화할 때 `vruntime`의 새 값으로 사용됩니다.
* `queue_length` ([Nullable(UInt64)](/ko/reference/data-types/index)) — `fifo` 노드에만 적용됩니다. 현재 큐에 있는 리소스 요청 수입니다.
* `queue_cost` ([Nullable(Int64)](/ko/reference/data-types/index)) — `fifo` 노드에만 적용됩니다. 현재 큐에 있는 모든 요청 비용(예: 바이트 단위 크기)의 합계입니다.
* `budget` ([Nullable(Int64)](/ko/reference/data-types/index)) — `fifo` 노드에만 적용됩니다. 새 리소스 요청에 사용할 수 있는 &#39;비용 단위&#39; 수입니다. 리소스 요청의 예상 비용과 실제 비용이 일치하지 않는 경우(예: 읽기/쓰기 실패 후) 나타날 수 있습니다.
* `is_satisfied` ([Nullable(UInt8)](/ko/reference/data-types/index)) — 제약 노드에만 적용됩니다(예: `inflight_limit`). 이 노드의 모든 제약 조건이 충족되면 `1`입니다.
* `inflight_requests` ([Nullable(Int64)](/ko/reference/data-types/index)) — `inflight_limit` 노드에만 적용됩니다. 이 노드에서 큐에서 꺼낸 리소스 요청 중 현재 활용 상태에 있는 요청 수입니다.
* `inflight_cost` ([Nullable(Int64)](/ko/reference/data-types/index)) — `inflight_limit` 노드에만 적용됩니다. 이 노드에서 큐에서 꺼낸 리소스 요청 중 현재 활용 상태에 있는 모든 요청 비용(예: 바이트)의 합계입니다.
* `max_requests` ([Nullable(Int64)](/ko/reference/data-types/index)) — `inflight_limit` 노드에만 적용됩니다. 제약 조건 위반을 유발하는 `inflight_requests`의 상한입니다.
* `max_cost` ([Nullable(Int64)](/ko/reference/data-types/index)) — `inflight_limit` 노드에만 적용됩니다. 제약 조건 위반을 유발하는 `inflight_cost`의 상한입니다.
* `max_speed` ([Nullable(Float64)](/ko/reference/data-types/index)) — `bandwidth_limit` 노드에만 적용됩니다. 초당 토큰 수 기준 대역폭의 상한입니다.
* `max_burst` ([Nullable(Float64)](/ko/reference/data-types/index)) — `bandwidth_limit` 노드에만 적용됩니다. token-bucket throttler에서 사용할 수 있는 토큰의 상한입니다.
* `throttling_us` ([Nullable(Int64)](/ko/reference/data-types/index)) — `bandwidth_limit` 노드에만 적용됩니다. 이 노드가 스로틀링 상태였던 총 마이크로초 수입니다.
* `tokens` ([Nullable(Float64)](/ko/reference/data-types/index)) — `bandwidth_limit` 노드에만 적용됩니다. 현재 token-bucket throttler에서 사용할 수 있는 토큰 수입니다.

{/*AUTOGENERATED_END*/}
