<div id="description">
  ## 설명
</div>

[MergeTree](/ko/reference/engines/table-engines/mergetree-family/mergetree) 테이블의 파트에 관한 정보를 포함합니다.

각 행은 하나의 데이터 파트를 나타냅니다.

<div id="columns">
  ## 컬럼
</div>

{/*AUTOGENERATED_START*/}

* `partition` ([String](/ko/reference/data-types/index)) — 파티션 식별자(테이블 파티션 키 표현식에서 파생된 문자열 표현).
* `name` ([String](/ko/reference/data-types/index)) —
  데이터 파트의 이름입니다. 파트 명명 구조를 사용하면 데이터, 수집, 머지 패턴의 여러 특성을 파악할 수 있습니다. 파트 명명 포맷은 다음과 같습니다:

```text
<partition_id>_<minimum_block_number>_<maximum_block_number>_<level>_<data_version>
```

* 정의:
  * `partition_id` - 파티션 키를 식별합니다
  * `minimum_block_number` - 파트의 최소 block 번호를 식별합니다. ClickHouse는 항상 연속된 block을 머지합니다
  * `maximum_block_number` - 파트의 최대 block 번호를 식별합니다
  * `level` - 파트에서 추가 머지가 발생할 때마다 1씩 증가합니다. level이 0이면 아직 머지되지 않은 새 파트임을 나타냅니다. ClickHouse의 모든 파트는 항상 변경 불가능하다는 점을 기억하는 것이 중요합니다
  * `data_version` - 선택적 값으로, 파트가 변경되면 증가합니다(즉, 파트는 변경 불가능하므로 변경된 데이터는 항상 새 파트에만 기록됩니다)

* `uuid` ([UUID](/ko/reference/data-types/index)) — 데이터 파트의 UUID입니다.

* `part_type` ([String](/ko/reference/data-types/index)) — 데이터 파트의 저장 포맷입니다. 가능한 값: `Wide` — 각 컬럼이 별도의 파일에 저장됨, `Compact` — 모든 컬럼이 하나의 파일에 저장됨. 데이터 저장 포맷은 MergeTree 테이블의 `min_bytes_for_wide_part` 및 `min_rows_for_wide_part` 설정으로 제어됩니다.

* `active` ([UInt8](/ko/reference/data-types/index)) — 데이터 파트가 활성 상태인지 나타내는 플래그입니다. 데이터 파트가 활성 상태이면 테이블에서 사용됩니다. 그렇지 않으면 곧 삭제됩니다. 비활성 데이터 파트는 머지 및 변경 작업 후에 나타납니다.

* `marks` ([UInt64](/ko/reference/data-types/index)) — 마크 수입니다. 데이터 파트의 대략적인 행 수를 구하려면 마크 수에 index granularity(일반적으로 8192)를 곱하십시오(이 방법은 적응형 세분화 수준에는 적용되지 않습니다).

* `rows` ([UInt64](/ko/reference/data-types/index)) — 행 수입니다.

* `files` ([UInt64](/ko/reference/data-types/index)) — 데이터 파트에 있는 파일 수입니다.

* `bytes_on_disk` ([UInt64](/ko/reference/data-types/index)) — 디스크에 있는 모든 데이터 파트 파일의 총 크기(바이트)입니다.

* `data_compressed_bytes` ([UInt64](/ko/reference/data-types/index)) — 데이터 파트의 압축된 데이터 총 크기입니다. 모든 보조 파일(예: 마크 파일)은 포함되지 않습니다.

* `data_uncompressed_bytes` ([UInt64](/ko/reference/data-types/index)) — 데이터 파트의 비압축 데이터 총 크기입니다. 모든 보조 파일(예: 마크 파일)은 포함되지 않습니다.

* `primary_key_size` ([UInt64](/ko/reference/data-types/index)) — 디스크의 primary.idx/cidx 파일에서 기본 키 값이 사용하는 메모리 양(바이트)입니다.

* `marks_bytes` ([UInt64](/ko/reference/data-types/index)) — 마크 파일의 크기입니다.

* `secondary_indices_compressed_bytes` ([UInt64](/ko/reference/data-types/index)) — 데이터 파트에서 보조 인덱스의 압축된 데이터 총 크기입니다. 모든 보조 파일(예: 마크 파일)은 포함되지 않습니다.

* `secondary_indices_uncompressed_bytes` ([UInt64](/ko/reference/data-types/index)) — 데이터 파트에서 보조 인덱스의 비압축 데이터 총 크기입니다. 모든 보조 파일(예: 마크 파일)은 포함되지 않습니다.

* `secondary_indices_marks_bytes` ([UInt64](/ko/reference/data-types/index)) — 보조 인덱스용 마크 파일의 크기입니다.

* `modification_time` ([DateTime](/ko/reference/data-types/index)) — 데이터 파트가 있는 디렉터리가 수정된 시각입니다. 일반적으로 데이터 파트 생성 시각에 해당합니다.

* `remove_time` ([DateTime](/ko/reference/data-types/index)) — 데이터 파트가 비활성 상태가 된 시각입니다.

* `refcount` ([UInt32](/ko/reference/data-types/index)) — 데이터 파트가 사용되는 위치 수입니다. 값이 2보다 크면 해당 데이터 파트가 쿼리 또는 머지에 사용되고 있음을 나타냅니다.

* `min_date` ([Date](/ko/reference/data-types/index)) — 데이터 파트에 있는 날짜 키의 최솟값입니다.

* `max_date` ([Date](/ko/reference/data-types/index)) — 데이터 파트에 있는 날짜 키의 최댓값입니다.

* `min_time` ([DateTime](/ko/reference/data-types/index)) — 데이터 파트에 있는 날짜 및 시간 키의 최솟값입니다.

* `max_time` ([DateTime](/ko/reference/data-types/index)) — 데이터 파트에 있는 날짜 및 시간 키의 최댓값입니다.

* `partition_id` ([String](/ko/reference/data-types/index)) — 파티션 ID입니다.

* `min_block_number` ([Int64](/ko/reference/data-types/index)) — 머지 후 현재 파트를 구성하는 데이터 파트들의 최소 번호입니다.

* `max_block_number` ([Int64](/ko/reference/data-types/index)) — 머지 후 현재 파트를 구성하는 데이터 파트들의 최대 번호입니다.

* `level` ([UInt32](/ko/reference/data-types/index)) — 머지 트리의 깊이입니다. 0은 현재 파트가 다른 파트를 머지해서 생성된 것이 아니라 삽입으로 생성되었음을 의미합니다.

* `data_version` ([UInt64](/ko/reference/data-types/index)) — 데이터 파트에 적용할 뮤테이션을 결정하는 데 사용되는 번호입니다(`data_version`보다 버전이 높은 뮤테이션).

* `primary_key_bytes_in_memory` ([UInt64](/ko/reference/data-types/index)) — 기본 키 값에 사용되는 메모리 양(바이트)입니다. `primary_key_lazy_load`가 활성화되어 있고 키가 로드되지 않은 경우 0이 됩니다. 0이 아닌 경우 이 바이트는 파트 자체에 속하며 `jemalloc.mergetree_arena.active_bytes`에 포함됩니다. 이 값은 `PrimaryIndexCacheBytes`에 절대 포함되지 않습니다. 파트별로 두 값은 서로 배타적이기 때문입니다. `primary_key_lazy_load`와 `use_primary_key_cache` 설정에 따라 인덱스는 파트 내부(이 메트릭) 또는 공유 `PrimaryIndexCache`(다른 메트릭) 중 한 곳에만 존재합니다.

* `primary_key_bytes_in_memory_allocated` ([UInt64](/ko/reference/data-types/index)) — 기본 키 값에 예약된 메모리 양(바이트)입니다. `primary_key_lazy_load`가 활성화되어 있고 키가 로드되지 않은 경우 0이 됩니다. 0이 아닌 경우 `jemalloc.mergetree_arena.active_bytes`에 포함됩니다. `PrimaryIndexCacheBytes`와의 관계는 `primary_key_bytes_in_memory` 항목의 참고 사항을 확인하십시오.

* `index_granularity_bytes_in_memory` ([UInt64](/ko/reference/data-types/index)) — 인덱스 세분화 수준 값에 사용되는 메모리 양(바이트)입니다(`primary_key_lazy_load=1`이고 `use_primary_key_cache=1`인 경우 0이 됨). 0이 아닌 경우 이 바이트는 파트에 속하며 `jemalloc.mergetree_arena.active_bytes`에 포함됩니다.

* `index_granularity_bytes_in_memory_allocated` ([UInt64](/ko/reference/data-types/index)) — 인덱스 세분화 수준 값에 예약된 메모리 양(바이트)입니다(`primary_key_lazy_load=1`이고 `use_primary_key_cache=1`인 경우 0이 됨). 0이 아닌 경우 `jemalloc.mergetree_arena.active_bytes`에 포함됩니다.

* `is_frozen` ([UInt8](/ko/reference/data-types/index)) — 파티션 데이터 Backup이 존재함을 나타내는 플래그입니다. 1이면 Backup이 존재합니다. 0이면 Backup이 존재하지 않습니다. 자세한 내용은 FREEZE PARTITION을 참조하십시오.

* `database` ([String](/ko/reference/data-types/index)) — 데이터베이스 이름입니다.

* `table` ([String](/ko/reference/data-types/index)) — 테이블 이름입니다.

* `engine` ([String](/ko/reference/data-types/index)) — 매개변수를 제외한 테이블 엔진 이름입니다.

* `disk_name` ([String](/ko/reference/data-types/index)) — 데이터 파트를 저장하는 디스크 이름입니다.

* `path` ([String](/ko/reference/data-types/index)) — 데이터 파트 파일이 있는 폴더의 절대 경로입니다.

* `hash_of_all_files` ([String](/ko/reference/data-types/index)) — 압축된 파일의 sipHash128입니다.

* `hash_of_uncompressed_files` ([String](/ko/reference/data-types/index)) — 압축되지 않은 파일(마크 파일, 인덱스 파일 등)의 sipHash128입니다.

* `uncompressed_hash_of_compressed_files` ([String](/ko/reference/data-types/index)) — 압축 파일을 압축 해제한 것으로 간주했을 때 그 안의 데이터에 대한 sipHash128입니다.

* `delete_ttl_info_min` ([DateTime](/ko/reference/data-types/index)) — TTL DELETE 규칙에 대한 날짜 및 시간 키의 최솟값입니다.

* `delete_ttl_info_max` ([DateTime](/ko/reference/data-types/index)) — TTL DELETE 규칙에 대한 날짜 및 시간 키의 최댓값입니다.

* `move_ttl_info.expression` ([Array(String)](/ko/reference/data-types/index)) — 표현식의 배열입니다. 각 표현식은 TTL MOVE 규칙을 정의합니다.

* `move_ttl_info.min` ([Array(DateTime)](/ko/reference/data-types/index)) — 날짜 및 시간 값의 배열입니다. 각 요소는 TTL MOVE 규칙의 최소 키 값을 나타냅니다.

* `move_ttl_info.max` ([Array(DateTime)](/ko/reference/data-types/index)) — 날짜 및 시간 값의 배열입니다. 각 요소는 TTL MOVE 규칙의 최대 키 값을 나타냅니다.

* `default_compression_codec` ([String](/ko/reference/data-types/index)) — 이 데이터 파트를 압축하는 데 사용되는 코덱 이름입니다(컬럼에 명시적으로 지정된 코덱이 없는 경우).

* `recompression_ttl_info.expression` ([Array(String)](/ko/reference/data-types/index)) — TTL 표현식입니다.

* `recompression_ttl_info.min` ([Array(DateTime)](/ko/reference/data-types/index)) — 이 파트에서 계산된 TTL 표현식의 최솟값입니다. TTL이 만료된 행이 하나 이상 있는지 확인하는 데 사용됩니다.

* `recompression_ttl_info.max` ([Array(DateTime)](/ko/reference/data-types/index)) — 이 파트 내에서 계산된 TTL 표현식의 최댓값입니다. 모든 행의 TTL이 만료되었는지 파악하는 데 사용됩니다.

* `group_by_ttl_info.expression` ([Array(String)](/ko/reference/data-types/index)) — TTL 표현식입니다.

* `group_by_ttl_info.min` ([Array(DateTime)](/ko/reference/data-types/index)) — 이 파트 내에서 계산된 TTL 표현식의 최솟값입니다. TTL이 만료된 행이 하나 이상 있는지 파악하는 데 사용됩니다.

* `group_by_ttl_info.max` ([Array(DateTime)](/ko/reference/data-types/index)) — 이 파트 내에서 계산된 TTL 표현식의 최댓값입니다. 모든 행의 TTL이 만료되었는지 파악하는 데 사용됩니다.

* `rows_where_ttl_info.expression` ([Array(String)](/ko/reference/data-types/index)) — TTL 표현식입니다.

* `rows_where_ttl_info.min` ([Array(DateTime)](/ko/reference/data-types/index)) — 이 파트 내에서 계산된 TTL 표현식의 최솟값입니다. TTL이 만료된 행이 하나 이상 있는지 파악하는 데 사용됩니다.

* `rows_where_ttl_info.max` ([Array(DateTime)](/ko/reference/data-types/index)) — 이 파트 내에서 계산된 TTL 표현식의 최댓값입니다. 모든 행의 TTL이 만료되었는지 파악하는 데 사용됩니다.

* `projections` ([Array(String)](/ko/reference/data-types/index)) — 이 파트에 대해 계산된 프로젝션 이름 목록입니다.

* `visible` ([UInt8](/ko/reference/data-types/index)) — 이 파트가 SELECT 쿼리에서 표시되는지 여부를 나타내는 플래그입니다.

* `creation_tid` ([Tuple(UInt64, UInt64, UUID)](/ko/reference/data-types/index)) — 이 객체를 생성했거나 생성하려는 트랜잭션의 ID입니다.

* `removal_tid_lock` ([UInt64](/ko/reference/data-types/index)) — `removal_tid`의 해시로, 삭제를 위해 객체를 잠그는 데 사용됩니다.

* `removal_tid` ([Tuple(UInt64, UInt64, UUID)](/ko/reference/data-types/index)) — 이 객체를 삭제했거나 삭제하려는 트랜잭션의 ID입니다.

* `creation_csn` ([UInt64](/ko/reference/data-types/index)) — 이 객체를 생성한 트랜잭션의 CSN입니다.

* `removal_csn` ([UInt64](/ko/reference/data-types/index)) — 이 객체를 삭제한 트랜잭션의 CSN입니다.

* `has_lightweight_delete` ([UInt8](/ko/reference/data-types/index)) — 이 파트에 경량한 삭제 마스크가 있는지 여부를 나타내는 플래그입니다.

* `last_removal_attempt_time` ([DateTime](/ko/reference/data-types/index)) — 서버가 이 파트를 삭제하려고 마지막으로 시도한 시각입니다.

* `removal_state` ([String](/ko/reference/data-types/index)) — 파트 삭제 프로세스의 현재 상태입니다.
