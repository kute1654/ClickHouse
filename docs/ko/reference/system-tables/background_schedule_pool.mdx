<div id="description">
  ## 설명
</div>

백그라운드 스케줄 풀의 작업에 관한 정보를 담고 있습니다. 백그라운드 스케줄 풀은 분산 전송, 버퍼 플러시, 메시지 브로커 작업과 같은 주기적인 작업을 실행하는 데 사용됩니다.

<div id="columns">
  ## 컬럼
</div>

{/*AUTOGENERATED_START*/}

* `pool` ([String](/ko/reference/data-types/index)) — 풀 이름입니다. 가능한 값: `schedule` — 범용 스케줄 풀, `buffer_flush` — Buffer 테이블 데이터를 플러시하는 풀, `distributed` — 분산 테이블 작업용 풀, `message_broker` — 메시지 브로커 작업용 풀입니다.
* `database` ([String](/ko/reference/data-types/index)) — 데이터베이스 이름입니다.
* `table` ([String](/ko/reference/data-types/index)) — 테이블 이름입니다.
* `table_uuid` ([UUID](/ko/reference/data-types/index)) — 테이블 UUID입니다.
* `query_id` ([String](/ko/reference/data-types/index)) — 쿼리 ID입니다(현재 실행 중인 경우). 참고: 이는 실제 쿼리가 아니라 `system.text_log`에서 로그를 매칭하기 위해 무작위로 생성된 ID입니다.
* `elapsed_ms` ([UInt64](/ko/reference/data-types/index)) — 작업 실행 시간입니다(현재 실행 중인 경우).
* `log_name` ([String](/ko/reference/data-types/index)) — 작업의 로그 이름입니다.
* `deactivated` ([UInt8](/ko/reference/data-types/index)) — 작업이 비활성화되었는지 여부입니다(비활성화된 작업은 풀에서 제거되므로 항상 false입니다).
* `scheduled` ([UInt8](/ko/reference/data-types/index)) — 작업이 실행되도록 예약되었는지 여부입니다.
* `delayed` ([UInt8](/ko/reference/data-types/index)) — 작업이 지연되어 예약되었는지 여부입니다.
* `executing` ([UInt8](/ko/reference/data-types/index)) — 작업이 현재 실행 중인지 여부입니다.

{/*AUTOGENERATED_END*/}
