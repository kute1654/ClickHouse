<div id="description">
  ## 설명
</div>

모든 포맷의 목록과 함께 각 포맷이 입력/출력에 적합한지, 병렬화를 지원하는지를 나타내는 플래그를 포함합니다.

<div id="columns">
  ## 컬럼
</div>

{/*AUTOGENERATED_START*/}

* `name` ([String](/ko/reference/data-types/index)) — 포맷 이름입니다.
* `is_input` ([UInt8](/ko/reference/data-types/index)) — 포맷이 데이터 입력에 적합한지 나타내는 플래그입니다.
* `is_output` ([UInt8](/ko/reference/data-types/index)) — 포맷이 데이터 출력에 적합한지 나타내는 플래그입니다.
* `supports_parallel_parsing` ([UInt8](/ko/reference/data-types/index)) — 포맷이 병렬 파싱을 지원하는지 나타내는 플래그입니다.
* `supports_parallel_formatting` ([UInt8](/ko/reference/data-types/index)) — 포맷이 병렬 포맷팅을 지원하는지 나타내는 플래그입니다.
* `is_tty_friendly` ([UInt8](/ko/reference/data-types/index)) — 포맷이 일반적으로 터미널에서 잘 표시되는지 나타내는 플래그입니다. 다른 포맷의 경우 출력 전에 CLI가 확인을 요청합니다.
* `content_type` ([String](/ko/reference/data-types/index)) — 출력 포맷에 해당하는 HTTP Content-Type입니다. 현재 포맷 설정에 따라 달라질 수 있습니다.
* `supports_random_access` ([UInt8](/ko/reference/data-types/index)) — 포맷이 입력에서 랜덤 액세스를 지원하는지 나타내는 플래그입니다.
* `has_schema_inference` ([UInt8](/ko/reference/data-types/index)) — 포맷이 데이터에서 스키마를 동적으로 추론할 수 있음을 나타냅니다(내장된 헤더/메타데이터 또는 데이터 조각으로부터).
* `has_external_schema` ([UInt8](/ko/reference/data-types/index)) — 포맷에 고정된 스키마가 있거나 자체 포맷으로 사전 정의된 스키마를 받을 수 있음을 나타냅니다.
* `prefers_large_blocks` ([UInt8](/ko/reference/data-types/index)) — 이 포맷은 출력 시 더 큰 블록을 기록하고 입력 시 더 큰 블록을 생성합니다.
* `supports_append` ([UInt8](/ko/reference/data-types/index)) — 이 포맷을 사용하면 단일 파일에 데이터를 추가할 수 있습니다.
* `supports_subsets_of_columns` ([UInt8](/ko/reference/data-types/index)) — 입력 포맷이 일부 컬럼이 생략된 경우를 인식할 수 있음을 나타냅니다.
* `description` ([String](/ko/reference/data-types/index)) — 해당 포맷에 대한 개괄적인 설명입니다.
* `examples` ([String](/ko/reference/data-types/index)) — 사용 예시입니다.
* `introduced_in` ([String](/ko/reference/data-types/index)) — 이 포맷이 처음 도입된 ClickHouse 버전이며, 형식은 major.minor입니다.
* `related` ([Array(String)](/ko/reference/data-types/index)) — 관련된 포맷의 이름입니다.

{/*AUTOGENERATED_END*/}
