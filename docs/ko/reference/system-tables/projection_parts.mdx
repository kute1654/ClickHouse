<div id="description">
  ## 설명
</div>

이 테이블에는 MergeTree 엔진 계열 테이블의 프로젝션 파트 정보가 포함되어 있습니다.

<div id="columns">
  ## 컬럼
</div>

{/*AUTOGENERATED_START*/}

* `partition` ([String](/ko/reference/data-types/index)) — 파티션 이름입니다.
* `name` ([String](/ko/reference/data-types/index)) — 데이터 파트의 이름입니다.
* `part_type` ([String](/ko/reference/data-types/index)) — 데이터 파트의 저장 포맷입니다. 가능한 값: Wide (컬럼별 파일 1개) 및 Compact (모든 컬럼에 대한 단일 파일).
* `parent_name` ([String](/ko/reference/data-types/index)) — 원본(상위) 데이터 파트의 이름입니다.
* `parent_uuid` ([UUID](/ko/reference/data-types/index)) — 원본(상위) 데이터 파트의 UUID입니다.
* `parent_part_type` ([String](/ko/reference/data-types/index)) — 원본(상위) 데이터 파트의 저장 포맷입니다.
* `active` ([UInt8](/ko/reference/data-types/index)) — 데이터 파트가 활성 상태인지 나타내는 플래그입니다. 데이터 파트가 활성 상태이면 테이블에서 사용됩니다. 그렇지 않으면 곧 삭제됩니다. 비활성 데이터 파트는 머지 및 변경 작업 후에 나타납니다.
* `marks` ([UInt64](/ko/reference/data-types/index)) — 마크 수입니다. 데이터 파트의 대략적인 행 수를 구하려면 마크 수에 인덱스 세분화 수준(index granularity)(일반적으로 8192)을 곱하십시오(이 방법은 적응형 세분화 수준에는 적용되지 않습니다).
* `rows` ([UInt64](/ko/reference/data-types/index)) — 행 수입니다.
* `bytes_on_disk` ([UInt64](/ko/reference/data-types/index)) — 모든 데이터 파트 파일의 총 크기(바이트)입니다.
* `data_compressed_bytes` ([UInt64](/ko/reference/data-types/index)) — 데이터 파트 내 압축된 데이터의 총 크기입니다. 모든 보조 파일(예: 마크 파일)은 포함되지 않습니다.
* `data_uncompressed_bytes` ([UInt64](/ko/reference/data-types/index)) — 데이터 파트 내 비압축 데이터의 총 크기입니다. 모든 보조 파일(예: 마크 파일)은 포함되지 않습니다.
* `marks_bytes` ([UInt64](/ko/reference/data-types/index)) — 마크 파일의 크기입니다.
* `parent_marks` ([UInt64](/ko/reference/data-types/index)) — 원본(상위) 파트의 마크 수입니다.
* `parent_rows` ([UInt64](/ko/reference/data-types/index)) — 원본(상위) 파트의 행 수입니다.
* `parent_bytes_on_disk` ([UInt64](/ko/reference/data-types/index)) — 모든 원본(상위) 데이터 파트 파일의 총 크기(바이트)입니다.
* `parent_data_compressed_bytes` ([UInt64](/ko/reference/data-types/index)) — 원본(상위) 데이터 파트 내 압축된 데이터의 총 크기입니다.
* `parent_data_uncompressed_bytes` ([UInt64](/ko/reference/data-types/index)) — 원본(상위) 데이터 파트 내 비압축 데이터의 총 크기입니다.
* `parent_marks_bytes` ([UInt64](/ko/reference/data-types/index)) — 원본(상위) 데이터 파트의 마크 파일 크기입니다.
* `modification_time` ([DateTime](/ko/reference/data-types/index)) — 데이터 파트가 있는 디렉터리가 수정된 시각입니다. 일반적으로 데이터 파트가 생성된 시각에 해당합니다.
* `remove_time` ([DateTime](/ko/reference/data-types/index)) — 데이터 파트가 비활성 상태가 된 시각입니다.
* `refcount` ([UInt32](/ko/reference/data-types/index)) — 데이터 파트가 사용되는 위치의 수입니다. 값이 2보다 크면 해당 데이터 파트가 쿼리 또는 머지에 사용되고 있음을 나타냅니다.
* `min_date` ([Date](/ko/reference/data-types/index)) — 데이터 파트 내 날짜 키의 최솟값입니다.
* `max_date` ([Date](/ko/reference/data-types/index)) — 데이터 파트 내 날짜 키의 최댓값입니다.
* `min_time` ([DateTime](/ko/reference/data-types/index)) — 데이터 파트 내 날짜 및 시간 키의 최솟값입니다.
* `max_time` ([DateTime](/ko/reference/data-types/index)) — 데이터 파트의 날짜 및 시간 키 최댓값입니다.
* `partition_id` ([String](/ko/reference/data-types/index)) — 파티션 ID입니다.
* `min_block_number` ([Int64](/ko/reference/data-types/index)) — 머지 후 현재 파트를 구성하는 데이터 파트의 최소 번호입니다.
* `max_block_number` ([Int64](/ko/reference/data-types/index)) — 머지 후 현재 파트를 구성하는 데이터 파트의 최대 번호입니다.
* `level` ([UInt32](/ko/reference/data-types/index)) — 머지 트리의 깊이입니다. 0은 현재 파트가 다른 파트들을 머지해서 생성된 것이 아니라 삽입으로 생성되었음을 의미합니다.
* `data_version` ([UInt64](/ko/reference/data-types/index)) — 데이터 파트에 적용할 뮤테이션을 결정하는 데 사용되는 번호입니다 (`data_version`보다 높은 버전의 뮤테이션).
* `primary_key_bytes_in_memory` ([UInt64](/ko/reference/data-types/index)) — 기본 키 값이 사용하는 메모리 크기(바이트)입니다.
* `primary_key_bytes_in_memory_allocated` ([UInt64](/ko/reference/data-types/index)) — 기본 키 값에 예약된 메모리 크기(바이트)입니다.
* `is_frozen` ([UInt8](/ko/reference/data-types/index)) — 파티션 데이터 백업의 존재 여부를 나타내는 플래그입니다. 1이면 백업이 존재합니다. 0이면 백업이 존재하지 않습니다.
* `database` ([String](/ko/reference/data-types/index)) — 데이터베이스 이름입니다.
* `table` ([String](/ko/reference/data-types/index)) — 테이블 이름입니다.
* `engine` ([String](/ko/reference/data-types/index)) — 매개변수를 제외한 테이블 엔진 이름입니다.
* `disk_name` ([String](/ko/reference/data-types/index)) — 데이터 파트를 저장하는 디스크 이름입니다.
* `path` ([String](/ko/reference/data-types/index)) — 데이터 파트 파일이 있는 폴더의 절대 경로입니다.
* `hash_of_all_files` ([String](/ko/reference/data-types/index)) — 압축된 파일의 sipHash128입니다.
* `hash_of_uncompressed_files` ([String](/ko/reference/data-types/index)) — 압축되지 않은 파일(마크 파일, 인덱스 파일 등)의 sipHash128입니다.
* `uncompressed_hash_of_compressed_files` ([String](/ko/reference/data-types/index)) — 압축된 파일을 압축 해제한 것으로 가정했을 때 해당 데이터의 sipHash128입니다.
* `delete_ttl_info_min` ([DateTime](/ko/reference/data-types/index)) — TTL DELETE 규칙에 대한 날짜 및 시간 키의 최솟값입니다.
* `delete_ttl_info_max` ([DateTime](/ko/reference/data-types/index)) — TTL DELETE 규칙에 대한 날짜 및 시간 키의 최댓값입니다.
* `move_ttl_info.expression` ([Array(String)](/ko/reference/data-types/index)) — 표현식의 배열입니다. 각 표현식은 TTL 이동 규칙을 정의합니다.
* `move_ttl_info.min` ([Array(DateTime)](/ko/reference/data-types/index)) — 날짜 및 시간 값의 배열입니다. 각 요소는 TTL 이동 규칙의 최소 키 값을 나타냅니다.
* `move_ttl_info.max` ([Array(DateTime)](/ko/reference/data-types/index)) — 날짜 및 시간 값의 배열입니다. 각 요소는 TTL 이동 규칙의 최대 키 값을 나타냅니다.
* `default_compression_codec` ([String](/ko/reference/data-types/index)) — 이 데이터 파트를 압축하는 데 사용되는 코덱 이름입니다 (컬럼에 명시적인 코덱이 없는 경우).
* `recompression_ttl_info.expression` ([Array(String)](/ko/reference/data-types/index)) — TTL 표현식입니다.
* `recompression_ttl_info.min` ([Array(DateTime)](/ko/reference/data-types/index)) — 이 파트 내에서 계산된 TTL 표현식의 최솟값입니다. 만료된 TTL을 가진 행이 하나 이상 있는지 확인하는 데 사용됩니다.
* `recompression_ttl_info.max` ([Array(DateTime)](/ko/reference/data-types/index)) — 이 파트에서 계산된 TTL 표현식의 최댓값입니다. TTL이 만료된 모든 행이 포함되어 있는지 확인하는 데 사용됩니다.
* `group_by_ttl_info.expression` ([Array(String)](/ko/reference/data-types/index)) — TTL 표현식입니다.
* `group_by_ttl_info.min` ([Array(DateTime)](/ko/reference/data-types/index)) — 이 파트에서 계산된 TTL 표현식의 최솟값입니다. TTL이 만료된 행이 하나 이상 포함되어 있는지 확인하는 데 사용됩니다.
* `group_by_ttl_info.max` ([Array(DateTime)](/ko/reference/data-types/index)) — 이 파트에서 계산된 TTL 표현식의 최댓값입니다. TTL이 만료된 모든 행이 포함되어 있는지 확인하는 데 사용됩니다.
* `rows_where_ttl_info.expression` ([Array(String)](/ko/reference/data-types/index)) — TTL 표현식입니다.
* `rows_where_ttl_info.min` ([Array(DateTime)](/ko/reference/data-types/index)) — 이 파트에서 계산된 TTL 표현식의 최솟값입니다. TTL이 만료된 행이 하나 이상 포함되어 있는지 확인하는 데 사용됩니다.
* `rows_where_ttl_info.max` ([Array(DateTime)](/ko/reference/data-types/index)) — 이 파트에서 계산된 TTL 표현식의 최댓값입니다. TTL이 만료된 모든 행이 포함되어 있는지 확인하는 데 사용됩니다.
* `is_broken` ([UInt8](/ko/reference/data-types/index)) — 프로젝션 파트가 손상되었는지 여부입니다.
* `exception_code` ([Int32](/ko/reference/data-types/index)) — 프로젝션 파트의 손상 상태를 설명하는 예외 메시지입니다.
* `exception` ([String](/ko/reference/data-types/index)) — 프로젝션 파트의 손상 상태를 설명하는 예외 코드입니다.
