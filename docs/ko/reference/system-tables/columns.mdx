<div id="description">
  ## 설명
</div>

모든 테이블의 컬럼 정보를 포함합니다.

이 테이블을 사용하면 여러 테이블에 대해 한 번에 [DESCRIBE TABLE](/ko/reference/statements/describe-table) 쿼리와 유사한 정보를 확인할 수 있습니다.

[temporary tables](/ko/reference/statements/create/table#temporary-tables)의 컬럼은 해당 테이블이 생성된 session에서만 `system.columns`에 표시됩니다. 이 컬럼들은 `database` 필드가 비어 있는 상태로 표시됩니다.

<div id="columns">
  ## 컬럼
</div>

{/*AUTOGENERATED_START*/}

* `database` ([String](/ko/reference/data-types/index)) — 데이터베이스 이름.
* `table` ([String](/ko/reference/data-types/index)) — 테이블 이름.
* `name` ([String](/ko/reference/data-types/index)) — 컬럼 이름.
* `type` ([String](/ko/reference/data-types/index)) — 컬럼 타입.
* `position` ([UInt64](/ko/reference/data-types/index)) — 테이블에서 1부터 시작하는 컬럼의 순번입니다.
* `default_kind` ([String](/ko/reference/data-types/index)) — 기본값의 표현식 유형(DEFAULT, MATERIALIZED, ALIAS)이며, 정의되지 않은 경우 빈 문자열입니다.
* `default_expression` ([String](/ko/reference/data-types/index)) — 기본값의 표현식이며, 정의되지 않은 경우 빈 문자열입니다.
* `data_compressed_bytes` ([UInt64](/ko/reference/data-types/index)) — 압축된 데이터의 크기(바이트)입니다.
* `data_uncompressed_bytes` ([UInt64](/ko/reference/data-types/index)) — 압축 해제된 데이터의 크기(바이트)입니다.
* `marks_bytes` ([UInt64](/ko/reference/data-types/index)) — 마크의 크기(바이트)입니다.
* `comment` ([String](/ko/reference/data-types/index)) — 컬럼에 대한 설명이며, 정의되지 않은 경우 빈 문자열입니다.
* `is_in_partition_key` ([UInt8](/ko/reference/data-types/index)) — 해당 컬럼이 파티션 표현식에 포함되는지를 나타내는 플래그입니다.
* `is_in_sorting_key` ([UInt8](/ko/reference/data-types/index)) — 해당 컬럼이 정렬 키 표현식에 포함되는지를 나타내는 플래그입니다.
* `is_in_primary_key` ([UInt8](/ko/reference/data-types/index)) — 해당 컬럼이 기본 키 표현식에 포함되는지를 나타내는 플래그입니다.
* `is_in_sampling_key` ([UInt8](/ko/reference/data-types/index)) — 해당 컬럼이 샘플링 키 표현식에 포함되는지를 나타내는 플래그입니다.
* `compression_codec` ([String](/ko/reference/data-types/index)) — 압축 코덱 이름입니다.
* `character_octet_length` ([Nullable(UInt64)](/ko/reference/data-types/index)) — 바이너리 데이터, 문자 데이터, 텍스트 데이터 및 이미지의 최대 길이(바이트)입니다. ClickHouse에서는 FixedString 데이터 타입에만 의미가 있습니다. 그 외에는 NULL 값이 반환됩니다.
* `numeric_precision` ([Nullable(UInt64)](/ko/reference/data-types/index)) — 근사 숫자 데이터, 정확한 숫자 데이터, 정수 데이터 또는 통화 데이터의 정밀도입니다. ClickHouse에서는 정수 타입의 경우 비트 폭이고 Decimal 타입의 경우 소수 정밀도입니다. 그 외에는 NULL 값이 반환됩니다.
* `numeric_precision_radix` ([Nullable(UInt64)](/ko/reference/data-types/index)) — 근사 숫자 데이터, 정확한 숫자 데이터, 정수 데이터 또는 통화 데이터 정밀도의 수 체계 밑수입니다. ClickHouse에서는 정수 타입의 경우 2이고 Decimal 타입의 경우 10입니다. 그 외에는 NULL 값이 반환됩니다.
* `numeric_scale` ([Nullable(UInt64)](/ko/reference/data-types/index)) — 근사 숫자 데이터, 정확한 숫자 데이터, 정수 데이터 또는 통화 데이터의 스케일입니다. ClickHouse에서는 Decimal 타입에만 의미가 있습니다. 그 외에는 NULL 값이 반환됩니다.
* `datetime_precision` ([Nullable(UInt64)](/ko/reference/data-types/index)) — DateTime64 데이터 타입의 Decimal 정밀도입니다. 다른 데이터 타입의 경우 NULL 값이 반환됩니다.
* `serialization_hint` ([Nullable(String)](/ko/reference/data-types/index)) — 통계에 따라 삽입 시 컬럼의 직렬화 방식을 선택하기 위한 힌트입니다.
* `statistics` ([String](/ko/reference/data-types/index)) — 이 컬럼에 생성된 통계의 타입입니다.

{/*AUTOGENERATED_END*/}
