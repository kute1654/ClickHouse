<div id="description">
  ## 설명
</div>

로깅 항목을 포함합니다. 이 테이블에 기록되는 로깅 수준은 `text_log.level` 서버 설정으로 제한할 수 있습니다.

<div id="columns">
  ## 컬럼
</div>

{/*AUTOGENERATED_START*/}

* `hostname` ([LowCardinality(String)](/ko/reference/data-types/lowcardinality)) — 쿼리를 실행하는 서버의 호스트명입니다.
* `event_date` ([Date](/ko/reference/data-types/date)) — 로그 항목의 날짜입니다.
* `event_time` ([DateTime](/ko/reference/data-types/datetime)) — 로그 항목의 시간입니다.
* `event_time_microseconds` ([DateTime64(6)](/ko/reference/data-types/datetime64)) — 마이크로초 정밀도의 로그 항목 시간입니다.
* `thread_name` ([LowCardinality(String)](/ko/reference/data-types/lowcardinality)) — 로깅이 수행된 스레드의 이름입니다.
* `thread_id` ([UInt64](/ko/reference/data-types/int-uint)) — OS 스레드 ID입니다.
* `level` ([Enum8(&#39;Fatal&#39; = 1, &#39;Critical&#39; = 2, &#39;Error&#39; = 3, &#39;Warning&#39; = 4, &#39;Notice&#39; = 5, &#39;Information&#39; = 6, &#39;Debug&#39; = 7, &#39;Trace&#39; = 8, &#39;Test&#39; = 9)](/ko/reference/data-types/enum)) — 항목 수준입니다. 가능한 값: 1 또는 &#39;Fatal&#39;, 2 또는 &#39;Critical&#39;, 3 또는 &#39;Error&#39;, 4 또는 &#39;Warning&#39;, 5 또는 &#39;Notice&#39;, 6 또는 &#39;Information&#39;, 7 또는 &#39;Debug&#39;, 8 또는 &#39;Trace&#39;.
* `query_id` ([String](/ko/reference/data-types/string)) — 쿼리의 ID입니다.
* `logger_name` ([LowCardinality(String)](/ko/reference/data-types/lowcardinality)) — 로거의 이름입니다(예: DDLWorker).
* `message` ([String](/ko/reference/data-types/string)) — 메시지 자체입니다.
* `revision` ([UInt32](/ko/reference/data-types/int-uint)) — ClickHouse 리비전입니다.
* `source_file` ([LowCardinality(String)](/ko/reference/data-types/lowcardinality)) — 로깅이 수행된 소스 파일입니다.
* `source_line` ([UInt64](/ko/reference/data-types/int-uint)) — 로깅이 수행된 소스 줄 번호입니다.
* `message_format_string` ([LowCardinality(String)](/ko/reference/data-types/lowcardinality)) — 메시지를 포맷하는 데 사용된 포맷 문자열입니다.
* `value1` ([String](/ko/reference/data-types/string)) — 메시지를 포맷하는 데 사용된 인수 1입니다.
* `value2` ([String](/ko/reference/data-types/string)) — 메시지를 포맷하는 데 사용된 인수 2입니다.
* `value3` ([String](/ko/reference/data-types/string)) — 메시지를 포맷하는 데 사용된 인수 3입니다.
* `value4` ([String](/ko/reference/data-types/string)) — 메시지를 포맷하는 데 사용된 인수 4입니다.
* `value5` ([String](/ko/reference/data-types/string)) — 메시지를 포맷하는 데 사용된 인수 5입니다.
* `value6` ([String](/ko/reference/data-types/string)) — 메시지를 포맷하는 데 사용된 인수 6입니다.
* `value7` ([String](/ko/reference/data-types/string)) — 메시지를 포맷하는 데 사용된 인수 7입니다.
* `value8` ([String](/ko/reference/data-types/string)) — 메시지를 포맷하는 데 사용된 인수 8입니다.
* `value9` ([String](/ko/reference/data-types/string)) — 메시지를 포맷하는 데 사용된 인수 9입니다.
* `value10` ([String](/ko/reference/data-types/string)) — 메시지를 포맷하는 데 사용된 인수 10입니다.

{/*AUTOGENERATED_END*/}
