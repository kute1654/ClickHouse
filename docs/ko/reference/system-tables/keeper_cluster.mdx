<div id="description">
  ## 설명
</div>

이 노드가 인프로세스 ClickHouse Keeper를 실행하도록 구성되지 않은 경우 이 테이블은 존재하지 않습니다. 이 테이블에는 Raft 클러스터의 각 멤버당 1개의 행이 있으며, 정적 클러스터 토폴로지(Raft 구성 기준)와 로컬 노드 자체의 로그 위치를 함께 보여줍니다.

각 노드는 자기 자신의 `server_id`와 일치하는 행에 정확히 하나의 `last_log_index` 값만 채웁니다. 피어의 로그 위치는 리더에서만 추적되며, 이 정보는 클러스터 전체에서 대칭적으로 보이지 않으므로 여기에는 표시되지 않습니다.

<div id="columns">
  ## 컬럼
</div>

{/*AUTOGENERATED_START*/}

* `server_id` ([Int32](/ko/reference/data-types/int-uint)) — 이 멤버의 Raft server id입니다.
* `host` ([String](/ko/reference/data-types/string)) — endpoint에서 파싱한 host입니다(마지막 콜론 앞의 접두사).
* `endpoint` ([String](/ko/reference/data-types/string)) — 구성된 원시 Raft endpoint입니다(`host:port`).
* `is_observer` ([Bool](/ko/reference/data-types/boolean)) — 이 구성원이 `<can_become_leader>false</can_become_leader>`로 구성된 비투표 observer이면 `true`입니다.
* `priority` ([Int32](/ko/reference/data-types/int-uint)) — 이 구성원의 Raft 우선순위이며, leader election 중에는 값이 높을수록 우선됩니다.
* `is_leader` ([Bool](/ko/reference/data-types/boolean)) — 이 구성원이 현재 Raft leader이면 `true`입니다.
* `is_self` ([Bool](/ko/reference/data-types/boolean)) — 이 행이 로컬 Keeper 노드를 나타내면 `true`입니다.
* `last_log_index` ([Nullable(UInt64)](/ko/reference/data-types/int-uint)) — 이 노드의 로컬 Raft 로그 저장소에서 가장 높은 로그 인덱스입니다. 현재 노드에 해당하는 행(`is_self = true`)에만 채워지며, 다른 행에서는 `NULL`입니다.

{/*AUTOGENERATED_END*/}

```sql
SELECT * FROM system.keeper_cluster ORDER BY server_id;
```
