<div id="description">
  ## 설명
</div>

`BACKUP` 및 `RESTORE` 작업에 대한 정보가 담긴 로그 항목이 포함됩니다.

<div id="columns">
  ## 컬럼
</div>

{/*AUTOGENERATED_START*/}

* `hostname` ([LowCardinality(String)](/ko/reference/data-types/lowcardinality)) — 쿼리를 실행하는 서버의 호스트명입니다.
* `event_date` ([Date](/ko/reference/data-types/date)) — 기록 날짜입니다.
* `event_time` ([DateTime](/ko/reference/data-types/datetime)) — 기록 시각입니다.
* `event_time_microseconds` ([DateTime64(6)](/ko/reference/data-types/datetime64)) — 마이크로초 정밀도의 기록 시각입니다.
* `id` ([String](/ko/reference/data-types/string)) — 백업 또는 복원 작업의 식별자입니다.
* `name` ([String](/ko/reference/data-types/string)) — 백업 저장소의 이름입니다(`FROM` 또는 `TO` 절의 내용).
* `base_backup_name` ([String](/ko/reference/data-types/string)) — 증분 백업인 경우 기준 백업의 이름입니다.
* `query_id` ([String](/ko/reference/data-types/string)) — 백업 작업과 연관된 쿼리 ID입니다.
* `status` ([Enum8(&#39;CREATING&#95;BACKUP&#39; = 0, &#39;BACKUP&#95;CREATED&#39; = 1, &#39;BACKUP&#95;FAILED&#39; = 2, &#39;RESTORING&#39; = 3, &#39;RESTORED&#39; = 4, &#39;RESTORE&#95;FAILED&#39; = 5, &#39;BACKUP&#95;CANCELLED&#39; = 6, &#39;RESTORE&#95;CANCELLED&#39; = 7)](/ko/reference/data-types/enum)) — 작업 상태입니다.
* `error` ([String](/ko/reference/data-types/string)) — 실패한 작업의 오류 메시지입니다(성공한 작업의 경우 빈 문자열).
* `start_time` ([DateTime64(6)](/ko/reference/data-types/datetime64)) — 작업 시작 시각입니다.
* `end_time` ([DateTime64(6)](/ko/reference/data-types/datetime64)) — 작업 종료 시각입니다.
* `num_files` ([UInt64](/ko/reference/data-types/int-uint)) — 백업에 저장된 파일 수입니다.
* `total_size` ([UInt64](/ko/reference/data-types/int-uint)) — 백업에 저장된 파일의 총 크기입니다.
* `num_entries` ([UInt64](/ko/reference/data-types/int-uint)) — 백업의 엔트리 수입니다. 즉, 백업이 폴더로 저장된 경우 폴더 안의 파일 수이고, 아카이브로 저장된 경우 아카이브 안의 파일 수입니다. 증분 백업이거나 빈 파일 또는 중복 파일이 포함된 경우 `num_files`와 같지 않습니다. 다음 조건은 항상 참입니다: `num_entries ≤ num_files`.
* `uncompressed_size` ([UInt64](/ko/reference/data-types/int-uint)) — 백업의 압축되지 않은 크기입니다.
* `compressed_size` ([UInt64](/ko/reference/data-types/int-uint)) — 백업의 압축된 크기입니다. 백업이 아카이브로 저장되지 않은 경우 `uncompressed_size`와 같습니다.
* `files_read` ([UInt64](/ko/reference/data-types/int-uint)) — 복원 작업 중 읽은 파일 수입니다.
* `bytes_read` ([UInt64](/ko/reference/data-types/int-uint)) — 복원 작업 중 읽은 파일의 총 크기입니다.

{/*AUTOGENERATED_END*/}
