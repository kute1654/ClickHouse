<div id="description">
  ## 설명
</div>

구성 파일에서 사용할 수 있는 클러스터와 해당 클러스터에 속한 서버 정보를 포함합니다.

<div id="Columns">
  ## 컬럼
</div>

{/*AUTOGENERATED_START*/}

* `cluster` ([String](/ko/reference/data-types/index)) — 클러스터 이름입니다.
* `shard_num` ([UInt32](/ko/reference/data-types/index)) — 클러스터 내 세그먼트 번호이며, 1부터 시작합니다.
* `shard_name` ([String](/ko/reference/data-types/index)) — 클러스터 내 세그먼트 이름입니다.
* `shard_weight` ([UInt32](/ko/reference/data-types/index)) — 데이터를 쓸 때 해당 세그먼트의 상대 가중치입니다.
* `internal_replication` ([UInt8](/ko/reference/data-types/index)) — 이 호스트가 자체적으로 데이터를 복제할 수 있는 앙상블의 일부인지 나타내는 플래그입니다.
* `replica_num` ([UInt32](/ko/reference/data-types/index)) — 세그먼트 내 레플리카 번호이며, 1부터 시작합니다.
* `host_name` ([String](/ko/reference/data-types/index)) — 구성에 지정된 호스트 이름입니다.
* `host_address` ([String](/ko/reference/data-types/index)) — DNS에서 가져온 호스트 IP 주소입니다.
* `port` ([UInt16](/ko/reference/data-types/index)) — 서버에 연결할 때 사용할 포트입니다.
* `is_local` ([UInt8](/ko/reference/data-types/index)) — 해당 호스트가 로컬 호스트인지 나타내는 플래그입니다.
* `user` ([String](/ko/reference/data-types/index)) — 서버에 연결할 때 사용할 사용자 이름입니다.
* `default_database` ([String](/ko/reference/data-types/index)) — 기본 데이터베이스 이름입니다.
* `errors_count` ([UInt32](/ko/reference/data-types/index)) — 이 호스트가 레플리카에 도달하지 못한 횟수입니다.
* `slowdowns_count` ([UInt32](/ko/reference/data-types/index)) — 헤지드 요청으로 연결을 설정할 때 레플리카 변경으로 이어진 지연 발생 횟수입니다.
* `estimated_recovery_time` ([UInt32](/ko/reference/data-types/index)) — 레플리카 오류 횟수가 0으로 초기화되어 정상 상태로 간주될 때까지 남은 시간(초)입니다.
* `database_shard_name` ([String](/ko/reference/data-types/index)) — `Replicated` 데이터베이스 세그먼트의 이름입니다(`Replicated` 데이터베이스에 속한 클러스터의 경우).
* `database_replica_name` ([String](/ko/reference/data-types/index)) — `Replicated` 데이터베이스 레플리카의 이름입니다(`Replicated` 데이터베이스에 속한 클러스터의 경우).
* `is_shared_catalog_cluster` ([UInt8](/ko/reference/data-types/index)) — 클러스터가 Shared Catalog에 속하는지 나타내는 Bool 값입니다.
* `is_active` ([Nullable(UInt8)](/ko/reference/data-types/index)) — Replicated 데이터베이스 레플리카의 상태입니다(Replicated 데이터베이스에 속한 클러스터의 경우). 1은 &#39;레플리카가 온라인 상태&#39;, 0은 &#39;레플리카가 오프라인 상태&#39;, NULL은 &#39;알 수 없음&#39;을 의미합니다.
* `unsynced_after_recovery` ([Nullable(UInt8)](/ko/reference/data-types/index)) — 레플리카를 생성하거나 복구한 후 Replicated 데이터베이스 레플리카의 복제 지연이 max&#95;replication&#95;lag&#95;to&#95;enqueue보다 큰지 나타냅니다.
* `replication_lag` ([Nullable(UInt32)](/ko/reference/data-types/index)) — `Replicated` 데이터베이스 레플리카의 복제 지연입니다(Replicated 데이터베이스에 속한 클러스터의 경우).
* `recovery_time` ([Nullable(UInt64)](/ko/reference/data-types/index)) — `Replicated` 데이터베이스 레플리카의 복구 시간입니다(Replicated 데이터베이스에 속한 클러스터의 경우). 단위는 밀리초입니다.

{/*AUTOGENERATED_END*/}
