<div id="description">
  ## 설명
</div>

각 오류 코드가 발생한 횟수를 포함합니다.

발생하지 않은 오류 코드까지 포함해 가능한 모든 오류 코드를 표시하려면 설정 [system&#95;events&#95;show&#95;zero&#95;values](/ko/reference/settings/session-settings#system_events_show_zero_values)를 1로 설정하십시오.

<div id="columns">
  ## 컬럼
</div>

{/*AUTOGENERATED_START*/}

* `name` ([String](/ko/reference/data-types/index)) — 오류 이름(errorCodeToName)입니다.
* `code` ([Int32](/ko/reference/data-types/index)) — 오류 코드 번호입니다.
* `value` ([UInt64](/ko/reference/data-types/index)) — 이 오류가 발생한 횟수입니다.
* `last_error_time` ([DateTime](/ko/reference/data-types/index)) — 마지막 오류가 발생한 시간입니다.
* `last_error_message` ([String](/ko/reference/data-types/index)) — 마지막 오류 메시지입니다.
* `last_error_format_string` ([String](/ko/reference/data-types/index)) — 마지막 오류의 포맷 문자열입니다.
* `last_error_trace` ([Array(UInt64)](/ko/reference/data-types/index)) — 호출된 메서드가 저장된 물리 주소 목록을 나타내는 스택 트레이스입니다.
* `remote` ([UInt8](/ko/reference/data-types/index)) — 원격 예외입니다(즉, 분산 쿼리 중 하나에서 수신된 예외).
* `query_id` ([String](/ko/reference/data-types/index)) — 오류를 일으킨 쿼리의 ID입니다(있는 경우).

{/*AUTOGENERATED_END*/}

<Note>
  일부 오류의 카운터는 쿼리가 성공적으로 실행되는 중에도 증가할 수 있습니다. 해당 오류가 오탐(false positive)이 아님을 확신할 수 없는 경우에는 서버 모니터링 용도로 이 테이블을 사용하지 않는 것이 좋습니다.
</Note>

<div id="example">
  ## 예시
</div>

```sql title="Query"
SELECT name, code, value
FROM system.errors
WHERE value > 0
ORDER BY code ASC
LIMIT 1

┌─name─────────────┬─code─┬─value─┐
│ CANNOT_OPEN_FILE │   76 │     1 │
└──────────────────┴──────┴───────┘
```

```sql title="Response"
WITH arrayMap(x -> demangle(addressToSymbol(x)), last_error_trace) AS all
SELECT name, arrayStringConcat(all, '\n') AS res
FROM system.errors
LIMIT 1
SETTINGS allow_introspection_functions=1\G
```
