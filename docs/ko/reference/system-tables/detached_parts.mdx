<div id="purpose">
  ## 설명
</div>

[MergeTree](/ko/reference/engines/table-engines/mergetree-family/mergetree) 테이블의 분리된 파트에 관한 정보를 포함합니다. `reason` 컬럼은 파트가 분리된 이유를 나타냅니다.

사용자가 분리한 파트의 경우 `reason`은 비어 있습니다. 이러한 파트는 [ALTER TABLE ATTACH PARTITION|PART](/ko/reference/statements/alter/partition#attach-partitionpart) 명령으로 ATTACH할 수 있습니다.

다른 컬럼에 대한 설명은 [system.parts](/ko/reference/system-tables/parts)를 참조하십시오.

파트 이름이 올바르지 않으면 일부 컬럼의 값이 `NULL`일 수 있습니다. 이러한 파트는 [ALTER TABLE DROP DETACHED PART](/ko/reference/statements/alter/partition#drop-detached-partitionpart)로 삭제할 수 있습니다.

<div id="columns">
  ## 컬럼
</div>

{/*AUTOGENERATED_START*/}

* `database` ([String](/ko/reference/data-types/index)) — 이 파트가 속한 데이터베이스의 이름입니다.
* `table` ([String](/ko/reference/data-types/index)) — 이 파트가 속한 테이블의 이름입니다.
* `partition_id` ([Nullable(String)](/ko/reference/data-types/index)) — 이 파트가 속한 파티션의 식별자입니다.
* `name` ([String](/ko/reference/data-types/index)) — 이 파트의 이름입니다.
* `bytes_on_disk` ([UInt64](/ko/reference/data-types/index)) — 모든 데이터 파트 파일의 총 크기(바이트)입니다.
* `modification_time` ([DateTime](/ko/reference/data-types/index)) — 데이터 파트가 있는 디렉터리가 수정된 시간입니다. 일반적으로 분리가 수행된 시점에 해당합니다.
* `disk` ([String](/ko/reference/data-types/index)) — 이 데이터 파트를 저장하는 디스크의 이름입니다.
* `path` ([String](/ko/reference/data-types/index)) — 디스크에서 이 데이터 파트 파일이 위치한 경로입니다.
* `reason` ([Nullable(String)](/ko/reference/data-types/index)) — 이 파트가 분리된 이유에 대한 설명입니다.
* `min_block_number` ([Nullable(Int64)](/ko/reference/data-types/index)) — 머지 후 현재 파트를 구성하는 데이터 파트들의 최소 번호입니다.
* `max_block_number` ([Nullable(Int64)](/ko/reference/data-types/index)) — 머지 후 현재 파트를 구성하는 데이터 파트들의 최대 번호입니다.
* `level` ([Nullable(UInt32)](/ko/reference/data-types/index)) — 머지 트리의 깊이입니다. 0은 현재 파트가 다른 파트들을 머지한 결과가 아니라 삽입으로 생성되었음을 의미합니다.
