<div id="description">
  ## 설명
</div>

비동기 삽입에 대한 정보를 포함합니다. 각 항목은 비동기 삽입 query에 버퍼링된 삽입 쿼리 하나를 나타냅니다.

로깅을 시작하려면 [asynchronous&#95;insert&#95;log](/ko/reference/settings/server-settings/settings#asynchronous_insert_log) 섹션에서 매개변수를 구성하십시오.

데이터 플러시 주기는 [asynchronous&#95;insert&#95;log](/ko/reference/settings/server-settings/settings#asynchronous_insert_log) 서버 설정 섹션의 `flush_interval_milliseconds` 매개변수에서 설정됩니다. 플러시를 강제로 수행하려면 [SYSTEM FLUSH LOGS](/ko/reference/statements/system#flush-logs) 쿼리를 사용하십시오.

ClickHouse는 테이블의 데이터를 자동으로 삭제하지 않습니다. 자세한 내용은 [Introduction](/ko/reference/system-tables/overview#system-tables-introduction)을 참조하십시오.

<div id="columns">
  ## 컬럼
</div>

{/*AUTOGENERATED_START*/}

* `hostname` ([LowCardinality(String)](/ko/reference/data-types/lowcardinality)) — 쿼리를 실행하는 서버의 호스트명입니다.
* `event_date` ([Date](/ko/reference/data-types/date)) — 비동기 삽입가 발생한 날짜입니다.
* `event_time` ([DateTime](/ko/reference/data-types/datetime)) — 비동기 삽입 실행이 완료된 날짜 및 시간입니다.
* `event_time_microseconds` ([DateTime64(6)](/ko/reference/data-types/datetime64)) — 비동기 삽입 실행이 마이크로초 정밀도로 완료된 날짜 및 시간입니다.
* `query` ([String](/ko/reference/data-types/string)) — 쿼리 문자열입니다.
* `database` ([LowCardinality(String)](/ko/reference/data-types/lowcardinality)) — 테이블이 속한 데이터베이스 이름입니다.
* `table` ([LowCardinality(String)](/ko/reference/data-types/lowcardinality)) — 테이블 이름입니다.
* `format` ([LowCardinality(String)](/ko/reference/data-types/lowcardinality)) — 포맷 이름입니다.
* `query_id` ([String](/ko/reference/data-types/string)) — 초기 쿼리의 ID입니다.
* `bytes` ([UInt64](/ko/reference/data-types/int-uint)) — 삽입된 바이트 수입니다.
* `rows` ([UInt64](/ko/reference/data-types/int-uint)) — 삽입된 행 수입니다.
* `exception` ([String](/ko/reference/data-types/string)) — 예외 메시지입니다.
* `status` ([Enum8(&#39;Ok&#39; = 0, &#39;ParsingError&#39; = 1, &#39;FlushError&#39; = 2)](/ko/reference/data-types/enum)) — 삽입 상태입니다. 값: &#39;Ok&#39; = 0 — 삽입 성공, &#39;ParsingError&#39; = 1 — 데이터 파싱 중 예외 발생, &#39;FlushError&#39; = 2 — 데이터 플러시 중 예외 발생.
* `data_kind` ([Enum8(&#39;Parsed&#39; = 0, &#39;Preprocessed&#39; = 1)](/ko/reference/data-types/enum)) — 데이터 상태입니다. 값: &#39;Parsed&#39; 및 &#39;Preprocessed&#39;입니다.
* `flush_time` ([DateTime](/ko/reference/data-types/datetime)) — 플러시가 발생한 날짜 및 시간입니다.
* `flush_time_microseconds` ([DateTime64(6)](/ko/reference/data-types/datetime64)) — 플러시가 마이크로초 정밀도로 발생한 날짜 및 시간입니다.
* `flush_query_id` ([String](/ko/reference/data-types/string)) — 플러시 쿼리의 ID입니다.
* `timeout_milliseconds` ([UInt64](/ko/reference/data-types/int-uint)) — 이 항목에 대해 계산된 적응형 timeout입니다.

{/*AUTOGENERATED_END*/}
