<div id="description">
  ## 설명
</div>

이 테이블은 이 노드가 in-process ClickHouse Keeper를 실행하도록 구성되지 않은 경우 존재하지 않습니다. 이 테이블에는 in-process Keeper log store에서 추적하는 각 Raft changelog 파일(`changelog_<from>_<to>.bin[.zstd]`)마다 1개의 행이 있으며, 현재 데이터를 추가 중인 활성 파일도 포함됩니다.

<div id="columns">
  ## 컬럼
</div>

{/*AUTOGENERATED_START*/}

* `from_log_index` ([UInt64](/ko/reference/data-types/int-uint)) — 이 파일에 있는 첫 번째 Raft 로그 인덱스입니다(포함).
* `to_log_index` ([UInt64](/ko/reference/data-types/int-uint)) — 파일 이름이 나타내는 마지막 Raft 로그 인덱스입니다(포함). 활성 파일에서는 이것이 회전 대상이므로 `last_entry_index`보다 클 수 있습니다.
* `last_entry_index` ([Nullable(UInt64)](/ko/reference/data-types/int-uint)) — 이 파일에 실제로 추가된 가장 큰 로그 인덱스입니다. 활성 파일이 아직 어떤 항목도 받지 않았거나 파일이 손상된 경우 `NULL`입니다.
* `entries` ([UInt64](/ko/reference/data-types/int-uint)) — 이 파일에 추가된 항목 수입니다. `ifNull(last_entry_index - from_log_index + 1, 0)`의 `ALIAS`입니다.
* `path` ([String](/ko/reference/data-types/string)) — 디스크상의 파일 경로입니다.
* `disk_name` ([String](/ko/reference/data-types/string)) — 이 파일이 저장된 디스크의 이름입니다.
* `size_bytes` ([UInt64](/ko/reference/data-types/int-uint)) — 디스크상의 파일 크기입니다.
* `modification_time` ([DateTime](/ko/reference/data-types/datetime)) — 파일이 마지막으로 수정된 시간입니다.
* `is_compressed` ([Bool](/ko/reference/data-types/boolean)) — 파일 payload가 zstd로 압축되어 있습니다.
* `active` ([Bool](/ko/reference/data-types/boolean)) — 현재 이 파일에 계속 추가되고 있습니다.
* `is_broken` ([Bool](/ko/reference/data-types/boolean)) — 시작 시 끝부분 레코드가 손상된 것으로 확인되었습니다.

{/*AUTOGENERATED_END*/}

예시:

```sql
SELECT from_log_index, to_log_index, entries, path, active FROM system.keeper_changelogs ORDER BY from_log_index;
```

```text
┌─from_log_index─┬─to_log_index─┬─entries─┬─path───────────────────────────┬─active─┐
│              1 │         1000 │    1000 │ changelog_1_1000.bin.zstd      │ false  │
│           1001 │         2000 │     537 │ changelog_1001_2000.bin.zstd   │ true   │
└────────────────┴──────────────┴─────────┴────────────────────────────────┴────────┘
```
