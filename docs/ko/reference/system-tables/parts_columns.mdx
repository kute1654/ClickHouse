<div id="description">
  ## 설명
</div>

[MergeTree](/ko/reference/engines/table-engines/mergetree-family/mergetree) 테이블의 파트와 컬럼 정보를 포함합니다.
각 행은 하나의 데이터 파트를 나타냅니다.

<div id="columns">
  ## 컬럼
</div>

{/*AUTOGENERATED_START*/}

* `partition` ([String](/ko/reference/data-types/index)) — 파티션 이름입니다.
* `name` ([String](/ko/reference/data-types/index)) — 데이터 파트의 이름입니다.
* `uuid` ([UUID](/ko/reference/data-types/index)) — 파트의 UUID입니다.
* `part_type` ([String](/ko/reference/data-types/index)) — 데이터 파트의 저장 포맷입니다. 가능한 값: Wide — 각 컬럼이 파일 시스템의 별도 파일에 저장됩니다. Compact — 모든 컬럼이 파일 시스템의 하나의 파일에 저장됩니다.
* `active` ([UInt8](/ko/reference/data-types/index)) — 데이터 파트가 활성 상태인지 나타내는 플래그입니다. 데이터 파트가 활성 상태이면 테이블에서 사용됩니다. 그렇지 않으면 삭제됩니다. 비활성 데이터 파트는 머지 후에도 남아 있습니다.
* `marks` ([UInt64](/ko/reference/data-types/index)) — 마크 수입니다. 데이터 파트의 대략적인 행 수를 구하려면 marks에 index granularity(일반적으로 8192)를 곱하십시오(이 값은 적응형 세분화 수준에서는 적용되지 않습니다).
* `rows` ([UInt64](/ko/reference/data-types/index)) — 행 수입니다.
* `bytes_on_disk` ([UInt64](/ko/reference/data-types/index)) — 모든 데이터 파트 파일의 전체 크기(바이트)입니다.
* `data_compressed_bytes` ([UInt64](/ko/reference/data-types/index)) — 데이터 파트의 압축된 데이터 전체 크기입니다. 모든 보조 파일(예: 마크가 포함된 파일)은 포함되지 않습니다.
* `data_uncompressed_bytes` ([UInt64](/ko/reference/data-types/index)) — 데이터 파트의 압축되지 않은 데이터 전체 크기입니다. 모든 보조 파일(예: 마크가 포함된 파일)은 포함되지 않습니다.
* `marks_bytes` ([UInt64](/ko/reference/data-types/index)) — 마크가 저장된 파일의 크기입니다.
* `modification_time` ([DateTime](/ko/reference/data-types/index)) — 데이터 파트가 있는 디렉터리가 수정된 시각입니다. 일반적으로 데이터 파트가 생성된 시각과 일치합니다.
* `remove_time` ([DateTime](/ko/reference/data-types/index)) — 데이터 파트가 비활성 상태가 된 시각입니다.
* `refcount` ([UInt32](/ko/reference/data-types/index)) — 데이터 파트가 사용되는 위치의 수입니다. 값이 2보다 크면 해당 데이터 파트가 쿼리 또는 머지에 사용되고 있음을 의미합니다.
* `min_date` ([Date](/ko/reference/data-types/index)) — 파티션 키(partition key)에 포함된 경우 Date 컬럼의 최솟값입니다.
* `max_date` ([Date](/ko/reference/data-types/index)) — 파티션 키에 포함된 경우 Date 컬럼의 최댓값입니다.
* `min_time` ([DateTime](/ko/reference/data-types/index)) — 파티션 키에 포함된 경우 DateTime 컬럼의 최솟값입니다.
* `max_time` ([DateTime](/ko/reference/data-types/index)) — 파티션 키에 포함된 경우 DateTime 컬럼의 최댓값입니다.
* `partition_id` ([String](/ko/reference/data-types/index)) — 파티션 ID입니다.
* `min_block_number` ([Int64](/ko/reference/data-types/index)) — 머지 후 현재 파트를 구성하는 데이터 파트의 최소 번호입니다.
* `max_block_number` ([Int64](/ko/reference/data-types/index)) — 머지 후 현재 파트를 구성하는 데이터 파트의 최대 번호입니다.
* `level` ([UInt32](/ko/reference/data-types/index)) — 머지 트리의 깊이입니다. 0은 현재 파트가 다른 파트를 머지해서 생성된 것이 아니라 삽입으로 생성되었음을 의미합니다.
* `data_version` ([UInt64](/ko/reference/data-types/index)) — 데이터 파트에 어떤 뮤테이션을 적용해야 하는지 판단하는 데 사용되는 번호입니다(`data_version`보다 높은 버전의 뮤테이션).
* `primary_key_bytes_in_memory` ([UInt64](/ko/reference/data-types/index)) — 기본 키 값이 사용하는 메모리 양(바이트)입니다.
* `primary_key_bytes_in_memory_allocated` ([UInt64](/ko/reference/data-types/index)) — 기본 키 값을 위해 예약된 메모리 양(바이트)입니다.
* `database` ([String](/ko/reference/data-types/index)) — 데이터베이스 이름입니다.
* `table` ([String](/ko/reference/data-types/index)) — 테이블 이름입니다.
* `engine` ([String](/ko/reference/data-types/index)) — 매개변수를 제외한 테이블 엔진 이름입니다.
* `disk_name` ([String](/ko/reference/data-types/index)) — 데이터 파트를 저장하는 디스크 이름입니다.
* `path` ([String](/ko/reference/data-types/index)) — 데이터 파트 파일이 들어 있는 폴더의 절대 경로입니다.
* `column` ([String](/ko/reference/data-types/index)) — 컬럼 이름입니다.
* `type` ([String](/ko/reference/data-types/index)) — 컬럼 타입입니다.
* `column_position` ([UInt64](/ko/reference/data-types/index)) — 테이블에서 1부터 시작하는 컬럼의 순서 위치입니다.
* `default_kind` ([String](/ko/reference/data-types/index)) — 기본값의 표현식 유형(DEFAULT, MATERIALIZED, ALIAS)이며, 정의되지 않은 경우 빈 문자열입니다.
* `default_expression` ([String](/ko/reference/data-types/index)) — 기본값 표현식이며, 정의되지 않은 경우 빈 문자열입니다.
* `column_bytes_on_disk` ([UInt64](/ko/reference/data-types/index)) — 디스크에 저장된 컬럼의 전체 크기(바이트)입니다.
* `column_data_compressed_bytes` ([UInt64](/ko/reference/data-types/index)) — 컬럼의 압축된 데이터 전체 크기(바이트)입니다.
* `column_data_uncompressed_bytes` ([UInt64](/ko/reference/data-types/index)) — 컬럼의 압축 해제된 데이터 전체 크기(바이트)입니다.
* `column_marks_bytes` ([UInt64](/ko/reference/data-types/index)) — 컬럼 마크의 크기(바이트)입니다.
* `column_modification_time` ([Nullable(DateTime)](/ko/reference/data-types/index)) — 컬럼이 마지막으로 수정된 시간입니다.
* `column_ttl_min` ([Nullable(DateTime)](/ko/reference/data-types/index)) — 컬럼의 계산된 TTL 표현식 최솟값입니다.
* `column_ttl_max` ([Nullable(DateTime)](/ko/reference/data-types/index)) — 컬럼의 계산된 TTL 표현식 최댓값입니다.
* `statistics` ([Array(String)](/ko/reference/data-types/index)) — 컬럼의 통계입니다.
* `estimates.min` ([Nullable(String)](/ko/reference/data-types/index)) — 컬럼의 추정 최솟값입니다.
* `estimates.max` ([Nullable(String)](/ko/reference/data-types/index)) — 컬럼의 추정 최댓값입니다.
* `estimates.cardinality` ([Nullable(UInt64)](/ko/reference/data-types/index)) — 컬럼의 추정 카디널리티입니다.
* `estimates.null_count` ([Nullable(UInt64)](/ko/reference/data-types/index)) — 컬럼의 추정 NULL 값 개수입니다.
* `serialization_kind` ([String](/ko/reference/data-types/index)) — 컬럼 직렬화 종류입니다.
* `substreams` ([Array(String)](/ko/reference/data-types/index)) — 컬럼이 직렬화되는 하위 스트림 이름입니다.
* `filenames` ([Array(String)](/ko/reference/data-types/index)) — 컬럼의 각 하위 스트림에 해당하는 파일 이름입니다.
* `subcolumns.names` ([Array(String)](/ko/reference/data-types/index)) — 컬럼의 서브컬럼 이름
* `subcolumns.types` ([Array(String)](/ko/reference/data-types/index)) — 컬럼의 서브컬럼 타입
* `subcolumns.serializations` ([Array(String)](/ko/reference/data-types/index)) — 컬럼의 서브컬럼 직렬화 유형
* `subcolumns.bytes_on_disk` ([Array(UInt64)](/ko/reference/data-types/index)) — 각 서브컬럼의 크기(바이트)
* `subcolumns.data_compressed_bytes` ([Array(UInt64)](/ko/reference/data-types/index)) — 각 서브컬럼의 압축된 데이터 크기(바이트)
* `subcolumns.data_uncompressed_bytes` ([Array(UInt64)](/ko/reference/data-types/index)) — 각 서브컬럼의 압축 해제된 데이터 크기(바이트)
* `subcolumns.marks_bytes` ([Array(UInt64)](/ko/reference/data-types/index)) — 컬럼의 각 서브컬럼에 대한 마크 크기(바이트)

{/*AUTOGENERATED_END*/}
