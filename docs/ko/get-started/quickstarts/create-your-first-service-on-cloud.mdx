<div id="what-youll-build">
  ## 구축할 내용
</div>

이 빠른 시작에서는 첫 번째 ClickHouse Cloud 서비스를 생성하고, Cloud Console과 기본 제공 SQL 콘솔에 익숙해지며, 실제 데이터를 쿼리할 수 있도록 예시 데이터셋을 로드합니다.
이 과정을 마치면 데이터가 로드된 실행 중인 ClickHouse Cloud 서비스가 준비되어, 이후 빠른 시작 시리즈를 진행하는 기반으로 활용할 수 있습니다.

<Steps titleSize="h3">
  <Step>
    ### ClickHouse Cloud 가입하기

    [console.clickhouse.cloud](https://console.clickhouse.cloud/)로 이동합니다. 로그인 페이지가 표시됩니다. 하단의 &quot;아직 계정이 없으신가요?&quot; 옆에 있는 **Register**를 클릭하세요.

    <Frame caption="새 계정을 만들려면 Register를 클릭하세요">
      <img src="/images/get-started/quickstarts/signup-page.webp" alt="Register 링크가 있는 ClickHouse Cloud 로그인 페이지" />
    </Frame>

    등록 페이지에서 Google 또는 Microsoft 계정으로 가입하거나 회사 이메일을 입력하세요. 신용 카드는 필요하지 않습니다. 무료 체험을 통해 $300의 크레딧이 제공되며, 30일 동안 ClickHouse Cloud를 이용해 볼 수 있습니다.

    <Frame caption="Google, Microsoft 또는 회사 이메일로 가입하세요">
      <img src="/images/get-started/quickstarts/enter-details.webp" alt="ClickHouse Cloud 등록 페이지" />
    </Frame>

    이메일로 가입하는 경우 비밀번호를 설정하라는 메시지가 표시됩니다. 비밀번호는 다음 요구 사항을 충족해야 합니다:

    * 최소 12자
    * 다음 항목 중 최소 3가지: 소문자, 대문자, 숫자, 특수 문자(예: `!@#$%^&*`)

    <Frame caption="비밀번호를 입력하고 이용 약관에 동의하세요">
      <img src="/images/get-started/quickstarts/enter-email.webp" alt="비밀번호 입력 필드가 있는 ClickHouse Cloud 이메일 등록 페이지" />
    </Frame>

    이용 약관 및 개인정보 처리방침에 동의한 다음 **Sign up**을 클릭하세요. 이메일에서 확인 링크를 찾아 **Verify email**을 클릭하여 계정을 활성화하세요.

    확인이 완료되면 새 이메일과 비밀번호로 로그인하세요.
  </Step>

  <Step>
    ### 새 서비스 만들기

    처음 로그인하면 바로 서비스 구성 페이지로 이동합니다. 다음과 같이 설정하십시오.

    | Setting        | Recommendation                                                   |
    | -------------- | ---------------------------------------------------------------- |
    | **서비스 이름**     | 서비스의 용도가 잘 드러나는 이름을 지정합니다(예: `My first service`)                 |
    | **클라우드 제공업체**  | 선호에 따라 AWS, GCP 또는 Azure를 선택합니다                                  |
    | **Region**     | 지연 시간을 최소화하려면 가장 가까운 리전을 선택합니다                                   |
    | **메모리 및 스케일링** | 이 빠른 시작에서는 **Mini 12GB**(레플리카 1개)를 선택합니다. 학습용으로 가장 비용 효율적인 옵션입니다 |

    **서비스 생성**을 클릭해 서비스를 프로비저닝합니다.

    <Frame caption="클라우드 서비스 구성">
      <img src="/images/get-started/quickstarts/service-setup.webp" alt="ClickHouse Cloud 서비스 구성 페이지" />
    </Frame>

    서비스 홈 페이지로 이동하면 처음에는 상태가 **Provisioning**으로 표시됩니다. 일반적으로 1분 이내에 완료됩니다. 완료되면 상태가 **Running**으로 변경됩니다.

    <Frame caption="서비스가 실행 중">
      <img src="/images/get-started/quickstarts/service-home.webp" alt="Running 상태가 표시된 ClickHouse Cloud 서비스 홈 페이지" />
    </Frame>
  </Step>

  <Step>
    ### 예시 데이터셋 로드하기

    왼쪽 사이드바에서 **Data sources**를 클릭하세요. 서비스로 데이터를 가져오는 여러 옵션이 표시됩니다.

    <Frame caption="데이터 로드 옵션이 있는 Data sources 페이지">
      <img src="/images/get-started/quickstarts/data-sources.webp" alt="ClickHouse Cloud 데이터 소스 페이지" />
    </Frame>

    사용 가능한 예시 데이터셋을 찾아보려면 **Add sample data**를 클릭하세요. 세 가지 옵션이 표시됩니다: Cell Towers, UK Property Price Paid, New York Taxi Data. **UK Property Price Paid** 데이터셋에서 **Get started**를 클릭하세요.

    <Frame caption="UK Property Price Paid 데이터셋 선택">
      <img src="/images/get-started/quickstarts/example-datasets.webp" alt="ClickHouse Cloud 예시 데이터셋 선택" />
    </Frame>

    테이블 스키마와 함께 데이터 미리보기가 표시됩니다. 데이터를 서비스로 로드하려면 **Import dataset**을 클릭하세요.

    <Frame caption="데이터를 미리 보고 Import dataset 클릭">
      <img src="/images/get-started/quickstarts/import-dataset.webp" alt="UK 부동산 데이터가 포함된 ClickHouse Cloud 데이터셋 가져오기 미리보기" />
    </Frame>

    가져오기는 1~2분 정도 걸립니다. 완료되면 바로 데이터를 쿼리할 수 있습니다.
  </Step>

  <Step>
    ### 데이터 쿼리하기

    왼쪽 사이드바에서 **SQL 콘솔**을 클릭해 내장 쿼리 편집기를 여십시오. 여기에서 브라우저에서 직접 SQL 쿼리를 작성하고 실행할 수 있으며, 별도의 외부 도구는 필요하지 않습니다.

    영국 부동산 가격 데이터를 살펴볼 수 있도록 몇 가지 쿼리를 시도해 보십시오.

    지금까지 판매된 부동산 중 가장 비싼 부동산을 찾아보겠습니다:

    ```sql
    SELECT
        town,
        district,
        county,
        price,
        date
    FROM pp_complete
    ORDER BY price DESC
    LIMIT 5;
    ```

    연도별 평균 판매가를 구합니다:

    ```sql
    SELECT
        toYear(date) AS year,
        round(avg(price)) AS avg_price,
        formatReadableQuantity(count()) AS sales
    FROM pp_complete
    GROUP BY year
    ORDER BY year DESC;
    ```

    이러한 쿼리는 몇 초 만에 수백만 개의 행을 스캔합니다 - 이는 ClickHouse의 열 지향 스토리지와 압축 덕분입니다.
  </Step>
</Steps>
