이 가이드에서는 OpenTelemetry를 사용해 소규모 Node.js 애플리케이션을 계측하고 해당 애플리케이션의 logs, 메트릭, traces를 Managed ClickStack으로 전송하는 방법을 설명합니다. 백엔드는 애플리케이션 소스 코드를 변경하지 않고 계측됩니다.

[HackerNews Analyzer](https://github.com/ClickHouse/hn-news-analyzer)는 공개 ClickHouse 데모에 호스팅된 HackerNews 데이터셋에 쿼리를 실행하는 Node.js 앱입니다. 모든 차트, 테이블, 검색 상자는 실제 ClickHouse 쿼리로 동작하므로 모든 상호작용에서 trace가 생성되며, 그 주요 스팬은 백엔드에서 ClickHouse로 보내는 HTTPS 호출입니다.

이 가이드는 [OpenTelemetry Collector 설정하기](/ko/clickstack/managed-onboarding/setting-up-your-opentelemetry-collector)를 이미 완료했고, 이 애플리케이션을 실행하는 머신에서 접근할 수 있는 ClickStack collector가 실행 중이라고 가정합니다. **해당 collector의 OTLP endpoint**와 배포 시 설정한 `OTLP_AUTH_TOKEN`을 반드시 기록해 두십시오.

<div id="prerequisites">
  ## 사전 요구 사항
</div>

* 이 머신에서 접근할 수 있는 ClickStack collector. 아직 배포하지 않았다면 먼저 [OpenTelemetry Collector 설정하기](/ko/clickstack/managed-onboarding/setting-up-your-opentelemetry-collector)를 따르십시오.
* 해당 collector의 OTLP endpoint와 여기에 설정한 `OTLP_AUTH_TOKEN`.
* Node 18+와 npm.
