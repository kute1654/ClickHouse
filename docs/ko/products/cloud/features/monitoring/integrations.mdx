<div id="direct-datadog">
  ## Datadog 직접 통합
</div>

Datadog는 시스템 테이블을 직접 쿼리하는 에이전트용 ClickHouse 모니터링 플러그인을 제공합니다. 이 통합은 `clusterAllReplicas` 기능을 통해 클러스터를 인식하는 포괄적인 데이터베이스 모니터링을 제공합니다.

<Warning>
  **ClickHouse Cloud에는 권장되지 않음**

  시스템 테이블을 쿼리하는 Datadog 에이전트 직접 통합은 비용 최적화를 위한 유휴 상태 동작과 클라우드 프록시 계층의 운영상 제약과 호환되지 않으므로 ClickHouse Cloud 배포에는 권장되지 않습니다.
</Warning>

대신 Datadog [Agent](https://docs.datadoghq.com/agent/?tab=Linux)와 [OpenMetrics 통합](https://docs.datadoghq.com/integrations/openmetrics/)을 사용해 ClickHouse Cloud Prometheus 엔드포인트에서 메트릭을 수집하십시오. 이 방식은 서비스의 유휴 상태 전환 동작을 고려하면서 모니터링 workload와 운영 workload를 분리해 유지합니다. 구성 방법은 [Datadog의 Prometheus 및 OpenMetrics 통합 문서](https://docs.datadoghq.com/integrations/openmetrics/)를 참조하십시오.

Prometheus 엔드포인트 설정에 대한 자세한 내용은 [Prometheus 통합 페이지](/ko/products/cloud/features/monitoring/prometheus#integrating-with-datadog)를 참조하십시오.

<div id="billing-usage-api">
  ## 청구 및 사용량 API
</div>

청구 및 사용량 API를 사용하면 Cloud 조직의 청구 및 사용량 기록에 프로그래밍 방식으로 액세스할 수 있습니다. 사용자 지정 비용 모니터링 대시보드를 구축하거나 청구 데이터를 기존 재무 보고 워크플로에 통합할 때 유용합니다.

전체 API 참조는 [Billing API 문서](/ko/api-reference/organization/get-list-of-available-organizations#tag/Billing)를 참조하십시오.
