ClickHouse는 속도와 안정성으로 잘 알려져 있지만, 최적의 성능은
일정한 운영 범위 내에서 발휘됩니다. 예를 들어 테이블,
데이터베이스 또는 파트가 너무 많으면 성능에 부정적인 영향을 줄 수 있습니다. 이를 방지하기 위해 ClickHouse
Cloud는 여러 운영 측면에 걸쳐 한도를 적용합니다.
이러한 제한 기준의 세부 내용은 아래와 같습니다.

<Tip>
  이러한 제한 기준 중 하나에 도달했다면,
  사용 사례가 최적화되지 않은 방식으로 구현되었을 가능성이 있습니다. 지원 팀에 문의하시면
  제한 기준을 초과하지 않도록 사용 사례를 개선하는 데 도움을 드리거나,
  필요한 경우 이를 통제된 방식으로 늘릴 수 있는 방법도 함께 검토해 드리겠습니다.
</Tip>

| 항목                         | 한도                                                                                                           |
| -------------------------- | ------------------------------------------------------------------------------------------------------------ |
| **데이터베이스**                 | 1000                                                                                                         |
| **테이블**                    | 5000                                                                                                         |
| **컬럼**                     | ∼1000 (compact보다 wide format 권장)                                                                             |
| **파티션**                    | 50k                                                                                                          |
| **파트**                     | 10k ([`max_parts_in_total`](/ko/products/cloud/guides/cloud-compatibility#max_parts_in_total-10000) setting 참조) |
| **파트 크기**                  | 150 GB                                                                                                       |
| **조직당 서비스 수**              | 20 (소프트 한도)                                                                                                  |
| **웨어하우스당 서비스 수**           | 5 (소프트 한도)                                                                                                   |
| **서비스당 레플리카 수**            | 20 (소프트 한도)                                                                                                  |
| **낮은 카디널리티**               | 10k 이하                                                                                                       |
| **테이블의 기본 키(primary key)** | 데이터를 충분히 필터링할 수 있는 4~5개                                                                                      |
| **쿼리 동시성**                 | 1000 (레플리카당)                                                                                                 |
| **배치 수집**                  | 1M을 초과하는 항목은 시스템이 1M 행 블록으로 분할합니다                                                                            |

<Note>
  Single Replica Services의 경우 최대 데이터베이스 수는
  100개로 제한되며, 최대 테이블 수는 500개로 제한됩니다. 또한 Basic Tier Services의 스토리지는
  1 TB로 제한됩니다.
</Note>
