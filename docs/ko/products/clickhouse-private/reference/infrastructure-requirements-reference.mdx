<div id="operator-node-group-x86">
  #### Operator 노드 그룹 (x86)
</div>

| 속성         | 값                                                     |
| ---------- | ----------------------------------------------------- |
| AMI        | `AL2023_x86_64`                                       |
| 디스크 크기     | 20 GB                                                 |
| 최소 인스턴스 크기 | xlarge                                                |
| 인스턴스 유형    | x86 호환 유형이면 모두 가능                                     |
| 비고         | 이미 x86과 호환되며 최소 xlarge 요건을 충족한다면 기존 노드 그룹을 사용할 수 있습니다 |

**모든 AWS 노드는 인증을 위해 [IMDS](https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/instancedata-data-retrieval.html)가 필요합니다.**

Keeper 및 서버 노드 그룹의 경우, 여러 AZ에 걸쳐 클러스터 오토스케일러를 지원하려면 **AZ별로 노드 그룹을 하나씩 생성하십시오**.

오토스케일러 태그(클러스터 오토스케일러를 사용하는 경우): - `k8s.io/cluster-autoscaler/enabled: true` - `k8s.io/cluster-autoscaler/$CLUSTER_NAME: owned`

<div id="iam-service-account-requirements">
  ## IAM / 서비스 계정 요구 사항
</div>

<div id="aws">
  ### AWS
</div>

<div id="ecr-pull-role">
  #### ECR Pull 역할
</div>

* **역할 이름:** `ClickHouseAirgapECRPullRole`
* **목적:** ClickHouse 비공개 ECR에서 이미지를 가져옵니다
* **권한:** `ecr:GetDownloadUrlForLayer`, `ecr:BatchGetImage`, `ecr:BatchCheckLayerAvailability`, `ecr:DescribeImages`, `ecr:ListImages`, `ecr:GetAuthorizationToken`
* **조치:** 역할 ARN을 ClickHouse 팀에 제공하십시오

<div id="clickhouse-serverkeeper-s3-role">
  #### ClickHouse 서버/Keeper S3 역할
</div>

* **프로비저닝된 ClickHouse 클러스터마다 역할 1개**
* **명명 규칙:** `CH-S3-$NAME-$REGION-$ORDINAL-Role`
* `$NAME`: 클러스터 이름(예: `default-xx-01`)
* `$REGION`: 축약된 리전(예: `us-west-2`는 `uw2`)
* `$ORDINAL`: 예약 필드이며, `00`으로 설정
* 예시: `CH-S3-default-xx-01-uw2-00-Role`
* **권한:** 버킷 리소스에 대한 `s3:*`, `s3:ListBucket`
* **신뢰 정책:** EKS 클러스터의 OIDC 공급자를 사용해 `sts:AssumeRoleWithWebIdentity`로 IRSA 구성

<div id="ebs-csi-driver-role">
  #### EBS CSI Driver 역할
</div>

* **예시 이름:** `ClickHouse_EksEbsCsiDriverRole`
* **관리형 정책:** `arn:aws:iam::aws:policy/service-role/AmazonEBSCSIDriverPolicy`
* **신뢰 정책:** EKS 클러스터의 OIDC 공급자를 사용하는 IRSA

<div id="deployer-service-account">
  #### 배포용 서비스 계정
</div>

<div id="cluster-naming-conventions">
  ## 클러스터 명명 규칙
</div>

ClickHouse 클러스터의 명명 규칙은 모든 플랫폼에서 동일하게 적용됩니다:

```text
$DESCRIPTOR-$LETTERS-$ORDINAL
```

| 구성 요소         | 설명                       | 예시        |
| ------------- | ------------------------ | --------- |
| `$DESCRIPTOR` | 설명용 이름(영문자만 사용)          | `default` |
| `$LETTERS`    | 예약됨; 임의의 영문자 2개          | `xx`      |
| `$ORDINAL`    | `01`부터 시작해 순차적으로 증가하는 번호 | `01`      |

**예시 이름:** `default-xx-01`

클러스터 이름은 **대상 Kubernetes 클러스터 내에서 고유해야 하며** 다음에 사용됩니다: - Kubernetes 네임스페이스: `ns-$CLUSTER_NAME` - Kubernetes 리소스: `c-$CLUSTER_NAME` 프리픽스 사용 - IAM 역할 이름(AWS): `CH-S3-$CLUSTER_NAME-$REGION-$ORDINAL-Role` - S3/GCS 키 프리픽스: `ch-s3-<uuid>` (클러스터별로 고유)
