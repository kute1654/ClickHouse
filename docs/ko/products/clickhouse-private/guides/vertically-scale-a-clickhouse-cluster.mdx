Private API를 사용하여 ClickHouse 클러스터의 CPU와 메모리 리소스를 조정합니다.

<div id="prerequisites">
  ## 사전 요구 사항
</div>

* ClickHouse Private API가 설치되어 있고 액세스할 수 있어야 합니다(예: `http://localhost:8080/`로 포트 포워딩).
* 최소 1개의 ClickHouse 클러스터가 배포되어 있어야 합니다(이 가이드에서는 `default-xx-01`을 사용합니다).

<Steps>
  <Step>
    ## 1. 스케일 요청 생성

    원하는 리소스 할당을 지정한 JSON 파일을 생성합니다:

    ```json
    {
      "vertical": {
        "resources": {
          "cpu": "4",
          "memory": "16Gi"
        }
      }
    }
    ```

    이 파일을 `scale.json`으로 저장합니다.
  </Step>

  <Step>
    ## 2. 요청 전송

    ```bash
    curl -X POST "http://localhost:8080/api/v1/instances/default-xx-01/scale" \
        -H "Content-Type: application/json" \
        --data-binary "@scale.json"
    ```
  </Step>

  <Step>
    ## 3. CPU:메모리 비율 강제 적용

    API는 **1:4 CPU 대 메모리 비율**(CPU 코어 1개당 메모리 4 GiB)을 5% 허용 오차와 함께 강제 적용합니다.

    리소스 유형 하나만 지정하면 API가 나머지 하나를 자동으로 계산합니다. 예를 들어, `"memory": "16Gi"`만 지정하면 CPU는 4코어로 설정됩니다.

    **허용되는 리소스 수량 형식:**

    | 유형  | 예시                                        |
    | --- | ----------------------------------------- |
    | CPU | `"4"` (코어), `"2.5"` (소수), `"500m"` (밀리코어) |
    | 메모리 | `"16Gi"` (기비바이트), `"4096Mi"` (메비바이트)      |

    메모리에는 바이너리 단위(Ki, Mi, Gi)를 사용하는 것을 권장합니다.
  </Step>

  <Step>
    ## 4. 스케일링 진행 상황 모니터링

    스케일링 작업을 추적하려면 클러스터 상태를 확인합니다:

    ```bash
    curl "http://localhost:8080/api/v1/instances/default-xx-01/status"
    ```

    응답에는 다음 항목이 포함됩니다: - `state`: 현재 클러스터 상태 - `previousState`: 전환 이전 상태 - `message`: 상태 전환에 대한 세부 정보

    클러스터가 다시 Running 상태로 돌아오면 스케일링이 완료된 것입니다.
  </Step>
</Steps>

<div id="rolling-restart-behavior">
  ## 롤링 재시작 동작
</div>

API는 `ClickhouseCluster` 사용자 지정 리소스의 `ServerPodPolicy`를 업데이트합니다. 그러면 연산자가 새 리소스 할당을 적용해 StatefulSet을 롤링 재시작합니다.

> **중요:** ClickHouse Cloud와 달리 ClickHouse Private는 [Make Before Break (MBB) scaling](https://clickhouse.com/docs/manage/scaling#how-scaling-works-in-clickhouse-cloud)을 사용하지 않습니다. 수직 스케일링 중에는 롤링 재시작 과정에서 서비스가 잠시 중단될 수 있습니다.
