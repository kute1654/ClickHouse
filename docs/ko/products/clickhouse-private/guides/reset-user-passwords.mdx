Private API를 사용해 ClickHouse 사용자 비밀번호를 재설정합니다.

<div id="prerequisites">
  ## 사전 요구사항
</div>

* ClickHouse Private API가 설치되어 있고 접근할 수 있어야 합니다(예: `http://localhost:8080/`로 포트 포워딩).
* 최소 1개의 ClickHouse 클러스터가 배포되어 있어야 합니다(이 가이드에서는 `default-xx-01`을 사용합니다).

<Steps>
  <Step>
    ## 1. 해시된 비밀번호 생성

    새 비밀번호를 SHA-256으로 해시한 뒤, 결과를 base64로 인코딩합니다.

    ```ini
    PASSWORD='My super secret p@$$w0rd'
    HASHED_PASSWORD=$(echo -n "$PASSWORD" | shasum -a 256 | awk '{printf $1}' | base64)
    ```
  </Step>

  <Step>
    ## 2. 요청 본문 생성

    해시된 비밀번호를 포함한 JSON 파일을 생성합니다.

    ```json
    {
      "user_hashed_password": "<base64-encoded-sha256-hash>",
      "hashing_function": "sha256",
      "username": "default"
    }
    ```

    이 파일을 `password_reset.json`으로 저장합니다.

    | 필드                     | 필수  | 기본값       | 설명                    |
    | ---------------------- | --- | --------- | --------------------- |
    | `user_hashed_password` | 예   | --        | 새 비밀번호의 base64 인코딩 해시 |
    | `hashing_function`     | 아니요 | `sha256`  | 사용할 해시 알고리즘           |
    | `username`             | 아니요 | `default` | 비밀번호를 재설정할 사용자 이름     |
  </Step>

  <Step>
    ## 3. 요청 전송

    ```bash
    curl -X POST "http://localhost:8080/api/v1/instances/default-xx-01/reset-user-password" \
        -H "Content-Type: application/json" \
        --data-binary "@password_reset.json"
    ```

    성공 시 응답은 다음과 같습니다.

    ```json
    {
      "message": "User password reset successfully"
    }
    ```
  </Step>

  <Step>
    ## 4. 변경 사항 확인

    비밀번호 재설정은 오퍼레이터가 변경 사항을 처리한 후 적용됩니다. 새 비밀번호로 클러스터에 연결해 확인합니다.

    상태 엔드포인트를 통해 작업 상태를 모니터링할 수도 있습니다.

    ```bash
    curl "http://localhost:8080/api/v1/instances/default-xx-01/status"
    ```
  </Step>
</Steps>

<div id="hydra-limitation">
  ## Hydra 제한 사항
</div>

Hydra 하위 인스턴스에서는 비밀번호를 재설정할 **수 없습니다**. Hydra 하위 인스턴스의 비밀번호를 재설정하려면 상위 인스턴스에서 재설정하십시오.
