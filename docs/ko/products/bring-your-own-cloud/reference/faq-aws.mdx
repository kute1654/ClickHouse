<div id="faq">
  ## FAQ
</div>

<div id="compute">
  ### 컴퓨트
</div>

<AccordionGroup>
  <Accordion title="이 EKS 클러스터 하나에서 여러 서비스를 생성할 수 있습니까?">
    예. AWS 계정과 리전 조합별로 인프라는 한 번만 프로비저닝하면 됩니다.
  </Accordion>

  <Accordion title="BYOC는 어느 리전을 지원합니까?">
    BYOC는 ClickHouse Cloud와 동일한 [리전](/ko/products/cloud/reference/supported-regions#aws-regions)을 지원합니다.
  </Accordion>

  <Accordion title="추가 리소스 오버헤드가 있습니까? ClickHouse 인스턴스 외의 서비스를 실행하려면 어떤 리소스가 필요합니까?">
    ClickHouse 인스턴스(ClickHouse 서버 및 ClickHouse Keeper) 외에도 `clickhouse-operator`, `aws-cluster-autoscaler`, Istio 등의 서비스와 모니터링 스택을 실행합니다.

    현재 이러한 워크로드를 실행하기 위해 전용 노드 그룹에서 m5.xlarge 노드 3개를 사용하고 있으며, 각 AZ에 1개씩 배치되어 있습니다.
  </Accordion>
</AccordionGroup>

<div id="network-and-security">
  ### 네트워크 및 보안
</div>

<AccordionGroup>
  <Accordion title="설치 중 설정한 권한을 설치 완료 후 취소할 수 있습니까?">
    현재로서는 불가능합니다.
  </Accordion>

  <Accordion title="문제 해결을 위해 ClickHouse 엔지니어가 고객 인프라에 접근할 때 적용할 향후 보안 제어 기능을 검토하고 있습니까?">
    예. 고객이 엔지니어의 클러스터 접근을 승인할 수 있도록 하는 고객 제어 메커니즘을 로드맵에 포함하고 있습니다. 현재 엔지니어는 클러스터에 대한 적시 접근 권한을 얻기 위해 내부 에스컬레이션 절차를 거쳐야 합니다. 이 과정은 기록되며 보안 팀의 감사를 받습니다.
  </Accordion>

  <Accordion title="생성되는 VPC IP 범위의 크기는 얼마입니까?">
    기본적으로 BYOC VPC에는 `10.0.0.0/16`을 사용합니다. 향후 스케일링 가능성을 고려해 최소 /22를 예약하는 것을 권장하지만,
    크기를 제한하려는 경우 최대 30개의 서버 파드로
    제한될 가능성이 높다면 /23을 사용하는 것도 가능합니다.
  </Accordion>

  <Accordion title="유지 관리 빈도를 결정할 수 있습니까?">
    유지 관리 기간을 예약하려면 지원팀에 문의하십시오. 최소 주 1회의 업데이트 일정이 적용됩니다.
  </Accordion>
</AccordionGroup>

<div id="uptime-slas">
  ### 업타임 SLA
</div>

<AccordionGroup>
  <Accordion title="ClickHouse는 BYOC에 대한 업타임 SLA를 제공합니까?">
    아니요. 데이터 플레인은 고객의 클라우드 환경에서 호스팅되므로 서비스 가용성은 ClickHouse가 제어할 수 없는 리소스에 따라 달라집니다. 따라서 ClickHouse는 BYOC 배포에 대해 공식적인 업타임 SLA를 제공하지 않습니다. 추가로 궁금한 사항이 있으면 support@clickhouse.com으로 문의하십시오.
  </Accordion>
</AccordionGroup>
