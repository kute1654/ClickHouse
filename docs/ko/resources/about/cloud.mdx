ClickHouse Cloud는 널리 사용되는 오픈소스 OLAP 데이터베이스인 ClickHouse를 만든 개발팀이 제공하는 클라우드 서비스입니다.
[무료 평가판 시작하기](https://console.clickhouse.cloud/signUp)를 통해 ClickHouse Cloud를 직접 체험해 볼 수 있습니다.

<div id="clickhouse-cloud-benefits">
  ## ClickHouse Cloud의 장점
</div>

ClickHouse Cloud를 사용할 때 얻을 수 있는 주요 장점은 다음과 같습니다:

* **빠른 가치 실현**: 클러스터 규모를 산정하거나 스케일링할 필요 없이 즉시 구축을 시작할 수 있습니다.
* **원활한 스케일링**: 자동 스케일링이 가변적인 워크로드에 맞게 조정되므로 최대 사용량에 대비해 과도하게 프로비저닝할 필요가 없습니다.
* **서버리스 운영**: 규모 산정, 스케일링, 보안, 안정성, 업그레이드는 ClickHouse Cloud가 처리하므로 운영 부담을 줄일 수 있습니다.
* **투명한 가격 정책**: 리소스 예약과 스케일링 제어를 활용하면서 실제 사용한 만큼만 비용을 지불합니다.
* **총소유비용 절감**: 뛰어난 가격 대비 성능과 낮은 관리 부담을 제공합니다.
* **폭넓은 에코시스템**: 선호하는 데이터 커넥터, 시각화 도구, SQL 및 언어 클라이언트를 그대로 활용할 수 있습니다.

<div id="oss-vs-clickhouse-cloud">
  ## ClickHouse Cloud는 어떤 버전의 ClickHouse를 사용합니까?
</div>

ClickHouse Cloud는 수정 사항, 새로운 기능, 성능 개선이 포함된 최신 버전으로 서비스를 주기적으로 업그레이드합니다. 핵심 데이터베이스 버전이 오픈 소스로 공개된 후에는 클라우드 스테이징 환경에서 추가 검증을 수행하며, 일반적으로 프로덕션에 배포되기까지 6~8주가 소요됩니다. 배포는 클라우드 서비스 제공업체, 서비스 유형, 리전별로 단계적으로 진행됩니다.

특정 릴리스 채널을 구독하여 ClickHouse Cloud 서비스의 업그레이드 일정을 지정할 수 있습니다. 예를 들어, 일반 릴리스 일정에 앞서 업데이트를 받기 위한 [&quot;Fast&quot; Release Channel](/ko/products/cloud/features/admin-features/upgrades#fast-release-channel-early-upgrades)과 업그레이드를 연기하기 위한 [&quot;Slow&quot; Release Channel](/ko/products/cloud/features/admin-features/upgrades#slow-release-channel-deferred-upgrades)을 비롯해, 더 세분화된 일정 옵션도 제공합니다.

하위 호환성 보장을 포함한 ClickHouse Cloud의 업그레이드 프로세스 개요는 [Upgrades](/ko/products/cloud/features/admin-features/upgrades) 참고 문서를 확인하십시오.
