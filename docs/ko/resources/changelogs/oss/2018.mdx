<div id="clickhouse-release-18-16">
  ## ClickHouse 릴리스 18.16
</div>

<div id="clickhouse-release-18-16-1-2018-12-21">
  ### ClickHouse 릴리스 18.16.1, 2018-12-21
</div>

<div id="bug-fixes">
  #### 버그 수정:
</div>

* ODBC 소스를 사용하는 딕셔너리 업데이트 시 문제가 발생하던 오류를 수정했습니다. [#3825](https://github.com/ClickHouse/ClickHouse/issues/3825), [#3829](https://github.com/ClickHouse/ClickHouse/issues/3829)
* 이제 집계 함수의 JIT 컴파일이 LowCardinality 컬럼에서도 작동합니다. [#3838](https://github.com/ClickHouse/ClickHouse/issues/3838)

<div id="improvements">
  #### 개선 사항:
</div>

* `low_cardinality_allow_in_native_format` 설정이 추가되었습니다(기본적으로 활성화됨). 비활성화하면 SELECT 쿼리에서는 LowCardinality 컬럼이 일반 컬럼으로 변환되며, INSERT 쿼리에서는 일반 컬럼을 사용해야 합니다. [#3879](https://github.com/ClickHouse/ClickHouse/pull/3879)

<div id="build-improvements">
  #### 빌드 개선 사항:
</div>

* macOS 및 ARM 빌드 관련 수정.

<div id="clickhouse-release-18-16-0-2018-12-14">
  ### ClickHouse 릴리스 18.16.0, 2018-12-14
</div>

<div id="new-features">
  #### 새로운 기능:
</div>

* 반정형 입력 형식(`JSONEachRow`, `TSKV`)으로 데이터를 로드할 때 누락된 필드에 대해서는 `DEFAULT` 표현식이 평가됩니다. 이 기능은 `insert_sample_with_metadata` 설정으로 활성화됩니다. [#3555](https://github.com/ClickHouse/ClickHouse/pull/3555)
* 이제 `ALTER TABLE` 쿼리에서 테이블 컬럼을 추가하거나 제거할 때 정렬 키를 변경하는 `MODIFY ORDER BY` 작업을 사용할 수 있습니다. 이는 `SummingMergeTree`, `AggregatingMergeTree` 등과 같이 이 정렬 키를 기준으로 머지 시 추가 작업을 수행하는 `MergeTree` 계열 테이블에 유용합니다. [#3581](https://github.com/ClickHouse/ClickHouse/pull/3581) [#3755](https://github.com/ClickHouse/ClickHouse/pull/3755)
* 이제 `MergeTree` 계열 테이블에서는 서로 다른 정렬 키(`ORDER BY`)와 인덱스(`PRIMARY KEY`)를 지정할 수 있습니다. 정렬 키는 인덱스보다 더 길 수 있습니다. [#3581](https://github.com/ClickHouse/ClickHouse/pull/3581)
* HDFS로 데이터를 가져오고 내보내기 위한 `hdfs` 테이블 함수와 `HDFS` 테이블 엔진이 추가되었습니다. [chenxing-xc](https://github.com/ClickHouse/ClickHouse/pull/3617)
* base64를 처리하기 위한 `base64Encode`, `base64Decode`, `tryBase64Decode` 함수가 추가되었습니다. [Alexander Krasheninnikov](https://github.com/ClickHouse/ClickHouse/pull/3350)
* 이제 `uniqCombined` 집계 함수의 precision을 설정하는 매개변수(HyperLogLog cell 수 선택)를 사용할 수 있습니다. [#3406](https://github.com/ClickHouse/ClickHouse/pull/3406)
* ClickHouse에 커밋한 모든 사람의 이름이 포함된 `system.contributors` 테이블이 추가되었습니다. [#3452](https://github.com/ClickHouse/ClickHouse/pull/3452)
* 모든 파티션을 한 번에 백업할 수 있도록 `ALTER TABLE ... FREEZE` 쿼리에서 파티션 지정을 생략할 수 있게 되었습니다. [#3514](https://github.com/ClickHouse/ClickHouse/pull/3514)
* 반환값의 유형을 지정할 필요가 없는 `dictGet` 및 `dictGetOrDefault` 함수가 추가되었습니다. 유형은 딕셔너리 설명을 바탕으로 자동으로 결정됩니다. [Amos Bird](https://github.com/ClickHouse/ClickHouse/pull/3564)
* 이제 테이블 설명에서 컬럼 comment를 지정하고 `ALTER`로 변경할 수 있습니다. [#3377](https://github.com/ClickHouse/ClickHouse/pull/3377)
* 단순 key를 사용하는 `Join` 유형 테이블에 대한 읽기가 지원됩니다. [Amos Bird](https://github.com/ClickHouse/ClickHouse/pull/3728)
* 이제 `Join` 유형 테이블을 생성할 때 `join_use_nulls`, `max_rows_in_join`, `max_bytes_in_join`, `join_overflow_mode` 옵션을 지정할 수 있습니다. [Amos Bird](https://github.com/ClickHouse/ClickHouse/pull/3728)
* `Join` 유형 테이블을 딕셔너리처럼 사용할 수 있게 해주는 `joinGet` 함수가 추가되었습니다. [Amos Bird](https://github.com/ClickHouse/ClickHouse/pull/3728)
* 테이블 키 정보를 제공하기 위해 `system.tables` 테이블에 `partition_key`, `sorting_key`, `primary_key`, `sampling_key` 컬럼이 추가되었습니다. [#3609](https://github.com/ClickHouse/ClickHouse/pull/3609)
* `system.columns` 테이블에 `is_in_partition_key`, `is_in_sorting_key`, `is_in_primary_key`, `is_in_sampling_key` 컬럼이 추가되었습니다. [#3609](https://github.com/ClickHouse/ClickHouse/pull/3609)
* `system.parts` 테이블에 `min_time` 및 `max_time` 컬럼이 추가되었습니다. 이 컬럼들은 파티셔닝 키가 `DateTime` 컬럼으로 이루어진 표현식일 때 채워집니다. [Emmanuel Donin de Rosière](https://github.com/ClickHouse/ClickHouse/pull/3800)

<div id="bug-fixes-1">
  #### 버그 수정 사항:
</div>

* `LowCardinality` 데이터 타입의 수정 및 성능 개선. `LowCardinality(Nullable(...))`를 사용하는 `GROUP BY`. `extremes` 값 조회. 고차 함수 처리. `LEFT ARRAY JOIN`. 분산 `GROUP BY`. `Array`를 반환하는 함수. `ORDER BY` 실행. `Distributed` 테이블에 쓰기 (nicelulu). `Native` 프로토콜을 구현하는 이전 클라이언트의 `INSERT` 쿼리에 대한 하위 호환성. `JOIN`에 대한 `LowCardinality` 지원. 단일 스트림에서 작업할 때 성능 개선. [#3823](https://github.com/ClickHouse/ClickHouse/pull/3823) [#3803](https://github.com/ClickHouse/ClickHouse/pull/3803) [#3799](https://github.com/ClickHouse/ClickHouse/pull/3799) [#3769](https://github.com/ClickHouse/ClickHouse/pull/3769) [#3744](https://github.com/ClickHouse/ClickHouse/pull/3744) [#3681](https://github.com/ClickHouse/ClickHouse/pull/3681) [#3651](https://github.com/ClickHouse/ClickHouse/pull/3651) [#3649](https://github.com/ClickHouse/ClickHouse/pull/3649) [#3641](https://github.com/ClickHouse/ClickHouse/pull/3641) [#3632](https://github.com/ClickHouse/ClickHouse/pull/3632) [#3568](https://github.com/ClickHouse/ClickHouse/pull/3568) [#3523](https://github.com/ClickHouse/ClickHouse/pull/3523) [#3518](https://github.com/ClickHouse/ClickHouse/pull/3518)
* `select_sequential_consistency` 옵션의 동작 방식을 수정했습니다. 이전에는 이 설정이 활성화되면 새 파티션에 쓰기 시작한 뒤 불완전한 결과가 반환되는 경우가 있었습니다. [#2863](https://github.com/ClickHouse/ClickHouse/pull/2863)
* DDL `ON CLUSTER` 쿼리와 `ALTER UPDATE/DELETE`를 실행할 때 데이터베이스가 올바르게 지정되도록 수정되었습니다. [#3772](https://github.com/ClickHouse/ClickHouse/pull/3772) [#3460](https://github.com/ClickHouse/ClickHouse/pull/3460)
* VIEW 내부의 서브쿼리에서 데이터베이스가 올바르게 지정되도록 수정했습니다. [#3521](https://github.com/ClickHouse/ClickHouse/pull/3521)
* `VersionedCollapsingMergeTree`에서 `FINAL`과 함께 사용할 때의 `PREWHERE` 버그를 수정했습니다. [7167bfd7](https://github.com/ClickHouse/ClickHouse/commit/7167bfd7b365538f7a91c4307ad77e552ab4e8c1)
* 이제 테이블 잠금이 해제되기를 기다리느라 아직 시작되지 않은 쿼리를 취소하는 데 `KILL QUERY`를 사용할 수 있습니다. [#3517](https://github.com/ClickHouse/ClickHouse/pull/3517)
* 자정에 시계를 뒤로 돌린 경우 날짜 및 시간 계산을 수정했습니다(이 현상은 이란에서 발생하며, 모스크바에서는 1981년부터 1983년까지 발생했습니다). 이전에는 이로 인해 시간이 필요 이상으로 하루 일찍 재설정되었고, 텍스트 포맷에서 날짜와 시간이 잘못 표시되는 문제도 있었습니다. [#3819](https://github.com/ClickHouse/ClickHouse/pull/3819)
* 데이터베이스를 생략한 `VIEW` 및 서브쿼리의 일부 경우에서 발생하던 버그를 수정했습니다. [Winter Zhang](https://github.com/ClickHouse/ClickHouse/pull/3521)
* 내부 `MATERIALIZED VIEW`를 잠그지 않아 `MATERIALIZED VIEW`를 동시에 읽고 삭제할 때 발생하던 경쟁 상태 문제를 수정했습니다. [#3404](https://github.com/ClickHouse/ClickHouse/pull/3404) [#3694](https://github.com/ClickHouse/ClickHouse/pull/3694)
* 오류 `Lock handler cannot be nullptr.`를 해결했습니다. [#3689](https://github.com/ClickHouse/ClickHouse/pull/3689)
* `compile_expressions` 옵션이 활성화된 경우의 쿼리 처리 문제가 수정되었습니다(기본적으로 활성화되어 있습니다). `now` 함수와 같은 비결정적 상수 표현식은 더 이상 전개되지 않습니다. [#3457](https://github.com/ClickHouse/ClickHouse/pull/3457)
* `toDecimal32/64/128` 함수에서 상수가 아닌 scale 인수를 지정할 때 발생하던 충돌 문제가 수정되었습니다.
* `Values` 포맷에서 `NULL` 요소를 포함한 배열을 `Nullable` 없이 `Array` 타입인 컬럼에 삽입하려고 할 때 발생하던 오류를 수정했습니다(`input_format_values_interpret_expressions` = 1인 경우). [#3487](https://github.com/ClickHouse/ClickHouse/pull/3487) [#3503](https://github.com/ClickHouse/ClickHouse/pull/3503)
* ZooKeeper를 사용할 수 없을 때 `DDLWorker`에서 오류가 계속 로깅되던 문제를 수정했습니다. [8f50c620](https://github.com/ClickHouse/ClickHouse/commit/8f50c620334988b28018213ec0092fe6423847e2)
* `Date` 및 `DateTime` 타입 인수를 받는 `quantile*` 함수의 반환 타입을 수정했습니다. [#3580](https://github.com/ClickHouse/ClickHouse/pull/3580)
* 표현식 없이 단순 별칭만 지정한 경우 `WITH` 절의 문제를 수정했습니다. [#3570](https://github.com/ClickHouse/ClickHouse/pull/3570)
* `enable_optimize_predicate_expression`이 활성화된 상태에서, 이름이 지정된 서브쿼리와 수식된 컬럼 이름이 포함된 쿼리의 처리 문제를 수정했습니다. [Winter Zhang](https://github.com/ClickHouse/ClickHouse/pull/3588)
* materialized view 사용 시 발생하던 `Attempt to attach to nullptr thread group` 오류를 수정했습니다. [Marek Vavruša](https://github.com/ClickHouse/ClickHouse/pull/3623)
* 일부 잘못된 인수를 `arrayReverse` 함수에 전달할 때 발생하던 충돌을 수정했습니다. [73e3a7b6](https://github.com/ClickHouse/ClickHouse/commit/73e3a7b662161d6005e7727d8a711b930386b871)
* `extractURLParameter` 함수의 버퍼 오버플로우를 수정했습니다. 성능을 개선했습니다. 0바이트를 포함하는 문자열을 올바르게 처리하도록 했습니다. [141e9799](https://github.com/ClickHouse/ClickHouse/commit/141e9799e49201d84ea8e951d1bed4fb6d3dacb5)
* `lowerUTF8` 및 `upperUTF8` 함수의 버퍼 오버플로우를 수정했습니다. `FixedString` 유형 인수에 대해 이러한 함수를 실행할 수 있는 기능을 제거했습니다. [#3662](https://github.com/ClickHouse/ClickHouse/pull/3662)
* `MergeTree` 테이블 삭제 시 드물게 발생하던 race condition을 수정했습니다. [#3680](https://github.com/ClickHouse/ClickHouse/pull/3680)
* `Buffer` 테이블을 읽는 동시에 대상 테이블에 대해 `ALTER` 또는 `DROP`을 수행할 때 발생하던 경쟁 조건 문제를 수정했습니다. [#3719](https://github.com/ClickHouse/ClickHouse/pull/3719)
* `max_temporary_non_const_columns` 제한을 초과했을 때 발생하던 segfault를 수정했습니다. [#3788](https://github.com/ClickHouse/ClickHouse/pull/3788)

<div id="improvements">
  #### 개선 사항:
</div>

* 서버는 처리된 설정 파일을 `/etc/clickhouse-server/` 디렉터리에 쓰지 않습니다. 대신 `path` 내의 `preprocessed_configs` 디렉터리에 저장합니다. 즉, `/etc/clickhouse-server/` 디렉터리에는 `clickhouse` 사용자의 쓰기 권한이 없으므로 보안이 향상됩니다. [#2443](https://github.com/ClickHouse/ClickHouse/pull/2443)
* `min_merge_bytes_to_use_direct_io` 옵션은 기본적으로 10 GiB로 설정됩니다. MergeTree 엔진 계열 테이블에서 큰 파트를 생성하는 머지는 `O_DIRECT` 모드로 수행되며, 이를 통해 과도한 페이지 캐시 제거를 방지합니다. [#3504](https://github.com/ClickHouse/ClickHouse/pull/3504)
* 테이블 수가 매우 많을 때 서버 시작 속도를 개선했습니다. [#3398](https://github.com/ClickHouse/ClickHouse/pull/3398)
* 레플리카 간 연결에 연결 풀과 HTTP `Keep-Alive`가 추가되었습니다. [#3594](https://github.com/ClickHouse/ClickHouse/pull/3594)
* 쿼리 구문이 올바르지 않으면 `HTTP` 인터페이스에서는 `400 Bad Request` 코드가 반환됩니다(이전에는 `500`이 반환되었습니다). [31bc680a](https://github.com/ClickHouse/ClickHouse/commit/31bc680ac5f4bb1d0360a8ba4696fa84bb47d6ab)
* `join_default_strictness` 옵션의 기본값은 호환성을 위해 `ALL`로 설정되어 있습니다. [120e2cbe](https://github.com/ClickHouse/ClickHouse/commit/120e2cbe2ff4fbad626c28042d9b28781c805afe)
* 유효하지 않거나 복잡한 정규식에 대해 `re2` 라이브러리가 `stderr`로 출력하던 로깅을 제거했습니다. [#3723](https://github.com/ClickHouse/ClickHouse/pull/3723)
* `Kafka` 테이블 엔진에 다음이 추가되었습니다. Kafka에서 읽기를 시작하기 전에 subscription을 확인하는 검사와 테이블용 kafka&#95;max&#95;block&#95;size 설정입니다. [Marek Vavruša](https://github.com/ClickHouse/ClickHouse/pull/3396)
* 이제 `cityHash64`, `farmHash64`, `metroHash64`, `sipHash64`, `halfMD5`, `murmurHash2_32`, `murmurHash2_64`, `murmurHash3_32`, `murmurHash3_64` 함수는 인수 개수와 관계없이 사용할 수 있으며, 튜플 형식의 인수도 지원합니다. [#3451](https://github.com/ClickHouse/ClickHouse/pull/3451) [#3519](https://github.com/ClickHouse/ClickHouse/pull/3519)
* `arrayReverse` 함수는 이제 모든 타입의 배열을 지원합니다. [73e3a7b6](https://github.com/ClickHouse/ClickHouse/commit/73e3a7b662161d6005e7727d8a711b930386b871)
* 선택적 매개변수가 추가되었습니다. `timeSlots` 함수의 슬롯 크기를 지정하는 매개변수입니다. [Kirill Shvakov](https://github.com/ClickHouse/ClickHouse/pull/3724)
* `FULL` 및 `RIGHT JOIN`에서는 오른쪽 테이블(table)의 조인되지 않은 데이터 스트림에 `max_block_size` 설정이 사용됩니다. [Amos Bird](https://github.com/ClickHouse/ClickHouse/pull/3699)
* TLS를 활성화할 수 있도록 `clickhouse-benchmark` 및 `clickhouse-performance-test`에 `--secure` 명령줄 매개변수를 추가했습니다. [#3688](https://github.com/ClickHouse/ClickHouse/pull/3688) [#3690](https://github.com/ClickHouse/ClickHouse/pull/3690)
* 대상 테이블 구조와 일치하지 않을 때 `Buffer` 유형의 테이블에 대한 형 변환. [Vitaly Baranov](https://github.com/ClickHouse/ClickHouse/pull/3603)
* 지정된 시간 동안 비활성 상태가 지속되면 keep-alive 패킷을 활성화하는 `tcp_keep_alive_timeout` 옵션이 추가되었습니다. [#3441](https://github.com/ClickHouse/ClickHouse/pull/3441)
* 파티션 키가 단일 컬럼으로 이루어진 경우 `system.parts` 테이블에서 값에 대한 불필요한 따옴표를 제거했습니다. [#3652](https://github.com/ClickHouse/ClickHouse/pull/3652)
* 모듈로 함수는 `Date` 및 `DateTime` 데이터 타입에서 사용할 수 있습니다. [#3385](https://github.com/ClickHouse/ClickHouse/pull/3385)
* `POWER`, `LN`, `LCASE`, `UCASE`, `REPLACE`, `LOCATE`, `SUBSTR`, `MID` 함수의 별칭을 추가했습니다. [#3774](https://github.com/ClickHouse/ClickHouse/pull/3774) [#3763](https://github.com/ClickHouse/ClickHouse/pull/3763) SQL 표준과의 호환성을 위해 일부 함수 이름은 대소문자를 구분하지 않습니다. SQL과의 호환성을 위해 `SUBSTRING(expr FROM start FOR length)` 구문 설탕을 추가했습니다. [#3804](https://github.com/ClickHouse/ClickHouse/pull/3804)

* `LIMIT BY` 절에서 임의의 표현식을 사용할 수 있습니다. 이전에는 `SELECT` 결과로 생성된 컬럼만 사용할 수 있었습니다.
* `IN`을 프라이머리 키 컬럼의 표현식 튜플에 적용하면 `MergeTree` 테이블의 인덱스가 사용됩니다. 예시: `WHERE (UserID, EventDate) IN ((123, '2000-01-01'), ...)` (Anastasiya Tsarkova).
* 클러스터 간 복사와 데이터 세그먼트 재분할(베타)을 위한 `clickhouse-copier` 도구가 추가되었습니다.
* consistent hash 함수 `yandexConsistentHash`, `jumpConsistentHash`, `sumburConsistentHash`가 추가되었습니다. 이후 리샤딩 시 네트워크 트래픽을 줄이기 위해 이를 샤딩 키로 사용할 수 있습니다.
* 추가된 함수: `arrayAny`, `arrayAll`, `hasAny`, `hasAll`, `arrayIntersect`, `arrayResize`.
* `arrayCumSum` 함수를 추가했습니다(Javi Santana).
* 다양한 포맷의 텍스트가 포함된 문자열에서 DateTime을 읽을 수 있도록 `parseDateTimeBestEffort`, `parseDateTimeBestEffortOrZero`, `parseDateTimeBestEffortOrNull` 함수를 추가했습니다.
* 업데이트 중에 외부 딕셔너리에서 데이터를 부분적으로 다시 로드할 수 있습니다(지정된 필드의 값이 이전 다운로드 때보다 큰 레코드만 로드) (Arsen Hakobyan).
* `cluster` 테이블 함수가 추가되었습니다. 예시: `cluster(cluster_name, db, table)`. `remote` 테이블 함수는 클러스터 이름이 식별자로 지정된 경우 이를 첫 번째 인수로 받을 수 있습니다.
* `remote` 및 `cluster` 테이블 함수는 `INSERT` 쿼리에서 사용할 수 있습니다.
* `create_table_query` 및 `engine_full` 가상 컬럼이 `system.tables` 테이블에 추가되었습니다. `metadata_modification_time` 컬럼은 가상 컬럼입니다.
* `system.tables`and`system.databases` 테이블에 `data_path` 및 `metadata_path` 컬럼이 추가되었고, `system.parts` 및 `system.parts_columns` 테이블에는 `path` 컬럼이 추가되었습니다.
* `system.part_log` 테이블에 머지 관련 정보가 추가되었습니다.
* `system.query_log` 테이블에 임의의 파티셔닝 키를 사용할 수 있게 되었습니다 (Kirill Shvakov).
* 이제 `SHOW TABLES` 쿼리에도 임시 테이블이 표시됩니다. `system.tables`에 임시 테이블과 `is_temporary` 컬럼을 추가했습니다 (zhang2014).
* `DROP TEMPORARY TABLE` 및 `EXISTS TEMPORARY TABLE` 쿼리가 추가되었습니다(zhang2014).
* 임시 테이블용 `SHOW CREATE TABLE` 지원(zhang2014).
* 내부 프로세스에서 사용되는 설정용 `system_profile` 구성 매개변수가 추가되었습니다.
* `MongoDB` Dictionaries에서 `object_id`를 속성으로 로드하는 기능이 추가되었습니다(Pavel Litvinenko).
* `MongoDB` 소스를 사용하는 외부 딕셔너리에서 데이터를 로드할 때 `null`을 기본값으로 읽도록 했습니다(Pavel Litvinenko).
* 작은따옴표 없이 Unix 타임스탬프에서 `Values` 포맷으로 `DateTime` 값 읽기.
* `remote` 테이블 함수는 일부 레플리카에 요청한 테이블이 없을 때 failover를 지원합니다.
* `clickhouse-server`를 실행할 때 명령줄에서 구성 관련 설정을 덮어쓸 수 있습니다. 예시: `clickhouse-server -- --logger.level=information`.
* `FixedString` 인수에 대해 `empty` 함수를 구현했습니다. 문자열이 null 바이트로만 이루어진 경우 이 함수는 1을 반환합니다(zhang2014).
* `listen_try` 구성 매개변수가 추가되어, 일부 주소에서 수신 대기할 수 없더라도 종료하지 않고 listen 주소 중 최소 하나에서 수신 대기할 수 있습니다(IPv4 또는 IPv6 지원이 비활성화된 시스템에서 유용합니다).
* `VersionedCollapsingMergeTree` 테이블 엔진이 추가되었습니다.
* `library` 딕셔너리 소스에서 행과 임의의 숫자 타입을 지원합니다.
* `MergeTree` 테이블은 기본 키(primary key) 없이도 사용할 수 있지만, `ORDER BY tuple()`를 지정해야 합니다.
* 인수가 `NULL`이 아닌 경우 `널 허용` 유형은 `CAST`를 통해 `널 허용`이 아닌 유형으로 변환할 수 있습니다.
* `VIEW`에 대해서도 `RENAME TABLE`을 수행할 수 있습니다.
* `throwIf` 함수를 추가했습니다.
* `odbc_default_field_size` 옵션이 추가되었습니다. 이 옵션을 사용하면 ODBC 소스에서 로드되는 값의 최대 크기를 늘릴 수 있습니다(기본값은 1024입니다).
* `system.processes` 테이블과 `SHOW PROCESSLIST`에는 이제 `is_cancelled` 및 `peak_memory_usage` 컬럼이 추가되었습니다.

<div id="improvements">
  #### 개선 사항:
</div>

* 결과에 대한 제한과 쿼터는 더 이상 `INSERT SELECT` 쿼리 또는 `SELECT` 서브쿼리의 중간 데이터에 적용되지 않습니다.
* 서버 시작 시 `Replicated` 테이블의 상태를 확인할 때 `force_restore_data`가 잘못 트리거되는 일이 줄었습니다.
* `allow_distributed_ddl` 옵션이 추가되었습니다.
* 비결정적 함수는 `MergeTree` 테이블 키의 표현식에 사용할 수 없습니다.
* `config.d` 디렉터리의 치환 값이 포함된 파일은 알파벳순으로 로드됩니다.
* 요소 중 하나가 빈 배열인 상수 다차원 배열에서 `arrayElement` 함수의 성능이 개선되었습니다. 예시: `[[1], []][x]`.
* 매우 큰 치환 값(예: 매우 긴 IP 네트워크 목록)이 포함된 설정 파일을 사용할 때 서버가 이제 더 빠르게 시작됩니다.
* 쿼리를 실행할 때 테이블 값 함수는 한 번만 실행됩니다. 이전에는 `remote` 및 `mysql` 테이블 값 함수가 원격 서버에서 테이블 구조를 가져오기 위해 동일한 쿼리를 두 번 실행했습니다.
* 문서 생성기로 `MkDocs`를 사용합니다.
* 다른 컬럼의 `DEFAULT`/`MATERIALIZED` 표현식이 의존하는 테이블 컬럼을 삭제하려고 하면 예외가 발생합니다 (zhang2014).
* `Float` 데이터 타입에서 빈 줄을 숫자 0으로 파싱할 수 있는 기능이 추가되었습니다. 이 기능은 이전에도 제공되었지만 1.1.54342 릴리스에서 사라졌습니다.
* `Enum` 값은 `min`, `max`, `sum` 및 일부 다른 함수에서 사용할 수 있습니다. 이 경우 해당 숫자 값이 사용됩니다. 이 기능은 이전에도 제공되었지만 1.1.54337 릴리스에서 사라졌습니다.
* 별칭을 재귀적으로 확장한 뒤 AST 크기를 제한하는 `max_expanded_ast_elements`가 추가되었습니다.

<div id="bug-fixes">
  #### 버그 수정:
</div>

* 불필요한 컬럼이 서브쿼리에서 잘못 제거되거나, `UNION ALL`이 포함된 서브쿼리에서는 제거되지 않던 문제를 수정했습니다.
* `ReplacingMergeTree` 테이블의 머지에서 발생하던 버그를 수정했습니다.
* `Distributed` 테이블의 동기 삽입(`insert_distributed_sync = 1`) 관련 문제를 수정했습니다.
* 서브쿼리에 중복 컬럼이 있을 때 `FULL` 및 `RIGHT JOIN`을 특정 방식으로 사용하면 발생하던 segfault를 수정했습니다.
* `replace_running_query` 및 `KILL QUERY`를 특정 방식으로 사용하면 발생하던 segfault를 수정했습니다.
* `system.dictionaries` 테이블에서 `source` 및 `last_exception` 컬럼의 순서를 바로잡았습니다.
* `DROP DATABASE` 쿼리가 메타데이터 파일을 삭제하지 않던 버그를 수정했습니다.
* `Dictionary` 데이터베이스에 대한 `DROP DATABASE` 쿼리를 수정했습니다.
* 항목 수가 1억 개를 초과할 때 `uniqHLL12` 및 `uniqCombined` 함수의 정밀도가 낮아지던 문제를 수정했습니다 (Alex Bocharov).
* `INSERT` 쿼리에서 기본 명시 표현식을 동시에 계산해야 할 때 암시적 기본값 계산이 잘못되던 문제를 수정했습니다 (zhang2014).
* 드물게 `MergeTree` 테이블에 대한 쿼리가 끝나지 않던 문제를 수정했습니다 (chenxing-xc).
* 모든 세그먼트가 로컬일 때 `Distributed` 테이블에 대해 `CHECK` 쿼리를 실행하면 발생하던 크래시를 수정했습니다 (chenxing.xc).
* 정규식을 사용하는 함수에서 발생하던 경미한 성능 회귀를 수정했습니다.
* 복잡한 표현식에서 다차원 배열을 생성할 때 발생하던 성능 회귀를 수정했습니다.
* 메타데이터가 포함된 `.sql` 파일에 불필요한 `FORMAT` 섹션이 추가로 나타날 수 있던 버그를 수정했습니다.
* 명시적으로 지정된 테이블을 참조하는 `MATERIALIZED VIEW`를 삭제하려고 할 때 `max_table_size_to_drop` 제한이 적용되던 버그를 수정했습니다.
* 이전 클라이언트와의 비호환성 문제를 수정했습니다 (이전 클라이언트에 이해할 수 없는 `DateTime('timezone')` 유형의 데이터가 간헐적으로 전송되던 문제가 있었습니다).
* `ALTER`로 추가되었지만 이전 파티션에서는 비어 있는 구조의 `Nested` 컬럼 요소를 읽을 때, 해당 컬럼의 조건이 `PREWHERE`로 이동하면 발생하던 버그를 수정했습니다.
* `Merge` 테이블에 대한 쿼리에서 가상 `_table` 컬럼으로 테이블을 필터링할 때 발생하던 버그를 수정했습니다.
* `Distributed` 테이블에서 `ALIAS` 컬럼을 사용할 때 발생하던 버그를 수정했습니다.
* `quantile` 계열의 집계 함수가 포함된 쿼리에서 동적 컴파일이 불가능하던 버그를 수정했습니다.
* 많은 수의 테이블이 있는 `Merge` 테이블을 사용할 때와 `GLOBAL` 서브쿼리를 사용할 때 매우 드물게 발생하던 쿼리 실행 파이프라인의 race condition을 수정했습니다.
* 여러 인수를 사용하는 집계 함수와 함께 `arrayReduce` 함수에 크기가 서로 다른 배열을 전달할 때 발생하던 크래시를 수정했습니다.
* `MATERIALIZED VIEW`에서 `UNION ALL`이 포함된 쿼리 사용을 금지했습니다.
* 서버 시작 시 `part_log` 시스템 테이블 초기화 중 발생하던 오류를 수정했습니다 (기본적으로 `part_log`는 비활성화되어 있습니다).

<div id="new-features">
  #### 새로운 기능:
</div>

* 테이블에서 다차원 배열과 튜플(`Tuple` 데이터 타입) 저장을 지원하도록 추가했습니다.
* `DESCRIBE` 및 `INSERT` 쿼리에서 테이블 함수를 지원합니다. `DESCRIBE`에서 서브쿼리 지원도 추가했습니다. 예시: `DESC TABLE remote('host', default.hits)`; `DESC TABLE (SELECT 1)`; `INSERT INTO TABLE FUNCTION remote('host', default.hits)`. `INSERT INTO`뿐 아니라 `INSERT INTO TABLE`도 지원합니다.
* 시간대 지원을 개선했습니다. `DateTime` 데이터 타입에는 형식에서 파싱 및 포맷팅에 사용할 시간대를 지정할 수 있습니다. 예시: `DateTime('Asia/Istanbul')`. `DateTime` 인수에 대해 함수에서 시간대를 지정하면 반환 타입이 해당 시간대를 유지하며, 값도 예상대로 표시됩니다.
* `toTimeZone`, `timeDiff`, `toQuarter`, `toRelativeQuarterNum` 함수를 추가했습니다. `toRelativeHour`/`Minute`/`Second` 함수는 `Date` 타입 값을 인수로 받을 수 있습니다. `now` 함수 이름은 대소문자를 구분합니다.
* `toStartOfFifteenMinutes` 함수를 추가했습니다(Kirill Shvakov).
* 쿼리 포맷팅을 위한 `clickhouse format` 도구를 추가했습니다.
* `format_schema_path` 구성 매개변수를 추가했습니다(Marek Vavruša). 이 매개변수는 `Cap'n Proto` 포맷의 스키마를 지정하는 데 사용됩니다. 스키마 파일은 지정된 디렉터리에만 둘 수 있습니다.
* 외부 딕셔너리 및 모델 구성에 사용할 config substitutions(`incl` 및 `conf.d`) 지원을 추가했습니다(Pavel Yakunin).
* `system.settings` 테이블에 문서용 컬럼을 추가했습니다(Kirill Shvakov).
* `MergeTree` 테이블의 각 데이터 파트에 있는 컬럼 크기 정보를 제공하는 `system.parts_columns` 테이블을 추가했습니다.
* 로드된 `CatBoost` 머신 러닝 모델 정보를 제공하는 `system.models` 테이블을 추가했습니다.
* 원격 데이터베이스에 접근하기 위한 `mysql` 및 `odbc` 테이블 함수와, 이에 대응하는 `MySQL` 및 `ODBC` 테이블 엔진을 추가했습니다. 이 기능은 베타 단계입니다.
* `groupArray` 집계 함수에 `AggregateFunction` 타입 인수를 전달할 수 있는 기능을 추가했습니다(즉, 일부 집계 함수 상태의 배열을 생성할 수 있습니다).
* 다양한 집계 함수 combinator 조합에 대한 제한을 제거했습니다. 예를 들어 동작이 서로 다른 `avgForEachIf`와 `avgIfForEach` 집계 함수를 모두 사용할 수 있습니다.
* `-ForEach` 집계 함수 combinator를 여러 인수를 받는 집계 함수에도 사용할 수 있도록 확장했습니다.
* 함수가 non-`Nullable` 결과를 반환하는 경우에도 `Nullable` 인수에 대한 집계 함수를 지원하도록 추가했습니다(Silviu Caragea의 기여 포함). 예시: `groupArray`, `groupUniqArray`, `topK`.
* `clickhouse-client`용 `max_client_network_bandwidth`를 추가했습니다(Kirill Shvakov).
* `readonly = 2` 설정이 있는 사용자는 TEMPORARY 테이블로 작업할 수 있습니다(CREATE, DROP, INSERT...) (Kirill Shvakov).
* `Kafka` 엔진에서 여러 consumer를 사용할 수 있도록 지원을 추가했습니다. `Kafka`의 구성 옵션도 확장했습니다(Marek Vavruša).
* `intExp3` 및 `intExp4` 함수를 추가했습니다.
* `sumKahan` 집계 함수를 추가했습니다.
* * Number*가 숫자 타입인 `to \* Number\* OrNull` 함수를 추가했습니다.
* `INSERT SELECT` 쿼리에 대한 `WITH` 절 지원을 추가했습니다(작성자: zhang2014).
* `http_connection_timeout`, `http_send_timeout`, `http_receive_timeout` 설정을 추가했습니다. 특히 이 설정은 복제를 위해 데이터 파트를 다운로드할 때 사용됩니다. 이 설정을 변경하면 네트워크에 과부하가 걸렸을 때 더 빠르게 failover할 수 있습니다.
* `Null` 타입 테이블에 대한 `ALTER` 지원을 추가했습니다(Anastasiya Tsarkova).
* `reinterpretAsString` 함수를 메모리에 연속적으로 저장되는 모든 데이터 타입에 대해 사용할 수 있도록 확장했습니다.
* `clickhouse-local` 도구에 `--silent` 옵션을 추가했습니다. 이 옵션은 stderr에 쿼리 실행 정보를 출력하지 않습니다.
* 월 및/또는 일이 두 자리가 아니라 한 자리로 지정된 포맷의 텍스트에서 `Date` 타입 값을 읽을 수 있도록 지원을 추가했습니다(Amos Bird).

<div id="performance-optimizations">
  #### 성능 최적화:
</div>

* 문자열 인수를 사용하는 집계 함수 `min`, `max`, `any`, `anyLast`, `anyHeavy`, `argMin`, `argMax`의 성능이 개선되었습니다.
* `isInfinite`, `isFinite`, `isNaN`, `roundToExp2` 함수의 성능이 개선되었습니다.
* 텍스트 포맷의 `Date` 및 `DateTime` 유형 값에 대한 파싱 및 포맷 성능이 개선되었습니다.
* 부동소수점 수의 파싱 성능과 정밀도가 개선되었습니다.
* 좌측 및 우측 파트에 `USING`에 포함되지 않은 동일한 이름의 컬럼이 있는 경우 `JOIN`의 메모리 사용량이 줄어들었습니다.
* 계산 안정성을 낮추는 대신 집계 함수 `varSamp`, `varPop`, `stddevSamp`, `stddevPop`, `covarSamp`, `covarPop`, `corr`의 성능이 개선되었습니다. 기존 함수는 `varSampStable`, `varPopStable`, `stddevSampStable`, `stddevPopStable`, `covarSampStable`, `covarPopStable`, `corrStable`라는 이름으로 사용할 수 있습니다.

<div id="bug-fixes">
  #### 버그 수정:
</div>

* `DROP` 또는 `DETACH PARTITION` 쿼리 실행 후 데이터 중복 제거가 제대로 동작하지 않던 문제를 수정했습니다. 이전 버전에서는 파티션을 삭제한 뒤 동일한 데이터를 다시 삽입해도, 삽입된 블록이 중복으로 간주되어 처리되지 않았습니다.
* `POPULATE` 가 포함된 `CREATE MATERIALIZED VIEW` 쿼리에서 `WHERE` 절이 잘못 해석될 수 있던 버그를 수정했습니다.
* `zookeeper_servers` 구성에서 `root_path` 매개변수 사용과 관련된 버그를 수정했습니다.
* `Date` 인수를 `toStartOfDay` 에 전달할 때 예기치 않은 결과가 나오던 문제를 수정했습니다.
* 결과가 이전 연도로 넘어가는 경우 `addMonths` 및 `subtractMonths` 함수와 `INTERVAL n MONTH` 산술 처리의 문제를 수정했습니다.
* `DISTINCT`, `JOIN`, `uniq` 집계 함수와 외부 딕셔너리에서 `UUID` 데이터 타입 지원이 누락되어 있던 문제를 추가했습니다 (Evgeniy Ivanov). `UUID` 지원은 아직 완전하지 않습니다.
* 행의 합이 0이 되는 경우 `SummingMergeTree` 의 동작을 수정했습니다.
* `Kafka` 엔진에 여러 수정 사항을 적용했습니다 (Marek Vavruša).
* `Join` 테이블 엔진의 잘못된 동작을 수정했습니다 (Amos Bird).
* FreeBSD 및 OS X에서 allocator의 잘못된 동작을 수정했습니다.
* 이제 `extractAll` 함수가 빈 일치도 지원합니다.
* `openssl` 대신 `libressl` 을 사용할 수 없게 하던 오류를 수정했습니다.
* 임시 테이블에 대한 `CREATE TABLE AS SELECT` 쿼리의 문제를 수정했습니다.
* 복제 큐 업데이트의 원자성 부족 문제를 수정했습니다. 이 문제로 인해 서버가 재시작될 때까지 레플리카 간 동기화가 맞지 않을 수 있었습니다.
* `gcd`, `lcm`, `modulo` (`%` 연산자)에서 발생할 수 있는 오버플로우를 수정했습니다 (Maks Skorokhod).
* 이제 `umask` 를 변경한 후 `-preprocessed` 파일이 생성됩니다 (`umask` 는 구성에서 변경할 수 있습니다).
* 사용자 지정 파티션 키를 사용할 때 parts의 백그라운드 검사(`MergeTreePartChecker`)에서 발생하던 버그를 수정했습니다.
* 형식에서 튜플(`Tuple` 데이터 타입의 값) 파싱을 수정했습니다.
* `multiIf`, `array` 및 일부 다른 함수에 호환되지 않는 타입이 전달되었을 때 표시되는 오류 메시지를 개선했습니다.
* `Nullable` 타입 지원을 재설계했습니다. 서버 충돌로 이어질 수 있는 버그를 수정했습니다. 또한 `NULL` 지원과 관련된 거의 모든 다른 버그도 수정했습니다. 예: INSERT SELECT에서의 잘못된 타입 변환, HAVING 및 PREWHERE에서의 Nullable 지원 부족, `join_use_nulls` 모드, `OR` 연산자의 인수로 사용되는 Nullable 타입 등입니다.
* 데이터 타입의 내부 의미 체계와 관련된 여러 버그를 수정했습니다. 예시: `SummingMergeTree` 에서 `Enum` 타입 필드가 불필요하게 합산되던 문제, `Pretty` 포맷에서 `Enum` 타입 정렬 문제 등입니다.
* 복합 컬럼의 허용된 조합에 대한 검사를 더 엄격하게 했습니다.
* `FixedString` 데이터 타입에 매우 큰 매개변수를 지정할 때 발생하던 오버플로우를 수정했습니다.
* 일반적인 경우에 발생하던 `topK` 집계 함수의 버그를 수정했습니다.
* `-Array` combinator가 있는 집계 함수의 n항 변형 인수에서 배열 크기 동일성 검사가 누락되어 있던 문제를 수정했습니다.
* `clickhouse-client` 의 `--pager` 관련 버그를 수정했습니다 (author: ks1322).
* `exp10` 함수의 정밀도를 수정했습니다.
* 문서와의 컴플라이언스를 더 잘 맞추도록 `visitParamExtract` 함수의 동작을 수정했습니다.
* 잘못된 데이터 타입이 지정되었을 때 발생하던 충돌을 수정했습니다.
* 모든 컬럼이 상수인 경우 `DISTINCT` 동작을 수정했습니다.
* 튜플 요소 인덱스로 복잡한 상수 표현식을 사용하는 `tupleElement` 함수에서 쿼리 포맷팅 문제를 수정했습니다.
* `range_hashed` 딕셔너리에 대한 `Dictionary` 테이블의 버그를 수정했습니다.
* `FULL` 및 `RIGHT JOIN` 결과에 과도한 행이 포함되던 버그를 수정했습니다 (Amos Bird).
* 구성 다시 로드 중 `config.d` 디렉터리에서 임시 파일을 생성하고 제거할 때 발생하던 서버 충돌을 수정했습니다.
* `SYSTEM DROP DNS CACHE` 쿼리의 문제를 수정했습니다. 캐시는 플러시되었지만 클러스터 노드 주소는 업데이트되지 않았습니다.
* 뷰 아래에 있는 테이블에 대해 `DETACH TABLE` 을 실행한 후 `MATERIALIZED VIEW` 의 동작을 수정했습니다 (Marek Vavruša).

<div id="build-improvements">
  #### 빌드 개선 사항:
</div>

* 빌드에는 `pbuilder` 도구를 사용합니다. 빌드 프로세스는 빌드 호스트 환경에 거의 완전히 독립적입니다.
* 단일 빌드를 서로 다른 OS 버전에 사용합니다. 패키지와 바이너리는 광범위한 Linux 시스템에서 호환되도록 개선되었습니다.
* `clickhouse-test` 패키지를 추가했습니다. 기능 테스트를 실행하는 데 사용할 수 있습니다.
* 이제 소스 tarball을 리포지토리에 게시할 수 있습니다. GitHub를 사용하지 않고도 빌드를 재현하는 데 사용할 수 있습니다.
* Travis CI와의 제한적인 통합을 추가했습니다. Travis의 빌드 시간 제한 때문에 디버그 빌드만 테스트하고, 제한된 일부 테스트만 실행합니다.
* 기본 빌드에 `Cap'n'Proto` 지원을 추가했습니다.
* 문서 소스의 포맷을 `Restricted Text`에서 `Markdown`으로 변경했습니다.
* `systemd` 지원을 추가했습니다(Vladimir Smirnov). 일부 OS 이미지와 호환되지 않아 기본적으로 비활성화되어 있으며, 수동으로 활성화할 수 있습니다.
* 동적 코드 생성을 위해 `clang`과 `lld`를 `clickhouse` 바이너리에 내장했습니다. 또한 `clickhouse clang` 및 `clickhouse lld`로 호출할 수도 있습니다.
* 코드에서 GNU 확장 기능 사용을 제거했습니다. `-Wextra` 옵션을 활성화했습니다. `clang`으로 빌드할 때는 기본값으로 `libstdc++` 대신 `libc++`를 사용합니다.
* 다양한 도구의 빌드 속도를 높이기 위해 `clickhouse_parsers` 및 `clickhouse_common_io` 라이브러리를 분리했습니다.

<div id="backward-incompatible-changes">
  #### 하위 호환되지 않는 변경 사항:
</div>

* `Nullable` 컬럼을 포함하는 `Log` 타입 테이블의 마크 포맷이 하위 호환되지 않도록 변경되었습니다. 이러한 테이블이 있는 경우, 새 server 버전을 시작하기 전에 `TinyLog` 타입으로 변환해야 합니다. 이를 위해 `metadata` 디렉터리의 해당 `.sql` 파일에서 `ENGINE = Log`를 `ENGINE = TinyLog`로 대체하십시오. 테이블에 `Nullable` 컬럼이 없거나 테이블 타입이 `Log`가 아니라면 별도의 작업은 필요하지 않습니다.
* `experimental_allow_extended_storage_definition_syntax` 설정이 제거되었습니다. 이제 이 기능은 기본적으로 활성화됩니다.
* 혼동을 방지하기 위해 `runningIncome` 함수의 이름이 `runningDifferenceStartingWithFirstvalue`로 변경되었습니다.
* 테이블 없이 FROM 바로 뒤에 ARRAY JOIN을 지정하는 `FROM ARRAY JOIN arr` 구문이 제거되었습니다 (Amos Bird).
* 데모 목적으로만 사용되던 `BlockTabSeparated` 포맷이 제거되었습니다.
* 집계 함수 `varSamp`, `varPop`, `stddevSamp`, `stddevPop`, `covarSamp`, `covarPop`, `corr`의 state 포맷이 변경되었습니다. 이러한 집계 함수의 state를 테이블에 저장하고 있는 경우(`AggregateFunction` 데이터 타입 또는 해당 state를 사용하는 materialized view 사용), feedback@clickhouse.com으로 문의해 주십시오.
* 이전 server 버전에는 문서화되지 않은 기능이 있었습니다. 집계 함수가 매개변수에 의존하는 경우에도 AggregateFunction 데이터 타입에서는 매개변수 없이 지정할 수 있었습니다. 예시: `AggregateFunction(quantiles, UInt64)` (`AggregateFunction(quantiles(0.5, 0.9), UInt64)` 대신). 이 기능은 더 이상 사용할 수 없습니다. 문서화되지는 않았지만, 향후 릴리스에서 다시 지원할 계획입니다.
* enum 데이터 타입은 min/max 집계 함수에 사용할 수 없습니다. 이 기능은 다음 릴리스에서 다시 제공될 예정입니다.

<div id="please-note-when-upgrading">
  #### 업그레이드 시 유의 사항:
</div>

* 클러스터에서 롤링 업데이트를 수행할 때 일부 레플리카는 이전 버전의 ClickHouse를 실행하고 다른 레플리카는 새 버전을 실행하는 동안에는 복제가 일시적으로 중지되며, 로그에 `unknown parameter 'shard'` 메시지가 표시됩니다. 클러스터의 모든 레플리카 업데이트가 완료되면 복제가 다시 계속됩니다.
* 클러스터 서버에서 서로 다른 버전의 ClickHouse가 실행 중이면 다음 함수를 사용하는 분산 쿼리의 결과가 올바르지 않을 수 있습니다: `varSamp`, `varPop`, `stddevSamp`, `stddevPop`, `covarSamp`, `covarPop`, `corr`. 클러스터의 모든 노드를 업데이트해야 합니다.
