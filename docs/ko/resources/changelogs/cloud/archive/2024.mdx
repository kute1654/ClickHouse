<div id="marketplace-subscription-organization-attachment">
  ### Marketplace subscription 조직 연결
</div>

이제 새 marketplace subscription을 기존 ClickHouse Cloud 조직에 연결할 수 있습니다. marketplace subscription 가입을 완료하고 ClickHouse Cloud로 리디렉션되면, 기존에 생성한 조직을 새 marketplace subscription에 연결할 수 있습니다. 이후부터 조직의 리소스는 marketplace를 통해 청구됩니다.

<div id="force-openapi-key-expiration">
  ### OpenAPI Key 만료 강제 설정
</div>

이제 API Key의 만료 옵션을 제한하여 만료되지 않는 OpenAPI Key를 생성할 수 없도록 할 수 있습니다. 조직에서 이러한 제한을 활성화하려면 ClickHouse Cloud Support 팀에 문의하십시오.

<div id="custom-emails-for-notifications">
  ### 알림용 사용자 지정 이메일
</div>

이제 조직 관리자는 특정 알림에 추가 수신자로 이메일 주소를 더 추가할 수 있습니다. 알림을 별칭(alias)으로 보내거나, ClickHouse Cloud 사용자는 아니지만 조직 내 다른 사용자에게 보내야 하는 경우에 유용합니다. 이를 설정하려면 Cloud Console의 알림 설정으로 이동한 다음, 이메일 알림을 받을 이메일 주소를 편집하십시오.

<div id="byoc-beta">
  ### BYOC (베타)
</div>

AWS용 Bring Your Own Cloud를 이제 베타로 이용할 수 있습니다. 이 배포 모드를 사용하면 자체 AWS 계정에서 ClickHouse Cloud를 배포하고 실행할 수 있습니다. 현재 11개 이상의 AWS 리전에서 배포를 지원하며, 지원 리전은 곧 더 확대될 예정입니다. 액세스하려면 [지원팀에 문의하십시오](https://clickhouse.com/support/program). 이 배포는 대규모 환경에만 제공된다는 점에 유의하십시오.

<div id="postgres-change-data-capture-cdc-connector-in-clickpipes">
  ### ClickPipes의 Postgres 변경 데이터 캡처(Change Data Capture, CDC) 커넥터
</div>

이 바로 사용할 수 있는 통합을 사용하면 몇 번의 클릭만으로 Postgres 데이터베이스를 ClickHouse Cloud로 복제하고, ClickHouse를 활용해 초고속 분석을 수행할 수 있습니다. 이 커넥터는 Postgres의 지속적 복제와 일회성 마이그레이션 모두에 사용할 수 있습니다.

<div id="query-api-endpoints-ga">
  ### Query API endpoint (일반 제공)
</div>

ClickHouse Cloud에서 Query API Endpoints가 일반 제공(GA) 단계에 도달했음을 알려드립니다. Query API Endpoints를 사용하면 몇 번만 클릭하여 저장된 쿼리를 위한 RESTful API 엔드포인트를 만들고, 언어별 클라이언트나 복잡한 인증 처리를 거치지 않고도 애플리케이션에서 바로 데이터를 활용할 수 있습니다. 최초 출시 이후 다음과 같은 여러 개선 사항이 적용되었습니다.

* 특히 콜드 스타트 시 엔드포인트 지연 시간 감소
* 엔드포인트 RBAC 제어 강화
* CORS 허용 도메인 구성 가능
* 결과 스트리밍
* 모든 ClickHouse 호환 출력 형식 지원

이러한 개선 사항과 더불어, 기존 프레임워크를 활용해 ClickHouse Cloud 서비스에서 임의의 SQL 쿼리를 실행할 수 있는 범용 Query API 엔드포인트도 발표합니다. 범용 엔드포인트는 서비스 설정 페이지에서 활성화하고 구성할 수 있습니다.

시작하려면 [Query API Endpoints documentation](/ko/products/cloud/guides/sql-console/query-endpoints)을 참조하십시오.

<Image img="/images/cloud/reference/api_endpoints.webp" size="lg" alt="여러 설정이 표시된 API 엔드포인트 구성 화면의 ClickHouse Cloud 인터페이스" border />

<div id="vector-search-using-vector-similarity-indexes-early-access">
  ### 벡터 유사성 인덱스를 사용한 벡터 검색(얼리 액세스)
</div>

근사 벡터 검색을 위한 벡터 유사성 인덱스를 얼리 액세스로 제공하게 되었음을 알려드립니다.

ClickHouse는 이미 [거리 함수]https://clickhouse.com/blog/reinvent-2024-product-announcements#vector-search-using-vector-similarity-indexes-early-access)와 선형 스캔 수행 기능을 통해 벡터 기반 사용 사례를 폭넓게 지원하고 있습니다. 또한 최근에는 [usearch](https://github.com/unum-cloud/usearch) 라이브러리와 Hierarchical Navigable Small Worlds(HNSW) 근사 최근접 이웃 검색 알고리즘을 기반으로 하는 실험적 [근사 벡터 검색](/ko/reference/engines/table-engines/mergetree-family/annindexes) 방식도 추가했습니다.

시작하려면 [얼리 액세스 대기자 명단에 등록하세요](https://clickhouse.com/cloud/vector-search-index-waitlist).

<div id="clickhouse-connect-python-and-clickhouse-kafka-connect-users">

* ClickHouse-Connect (Java): &gt; 0.8.6

<div id="clickhouse-cloud-now-offers-hipaa-ready-services-in-beta-for-gcp">
  ### ClickHouse Cloud에서 이제 GCP용 HIPAA 대응 서비스가 베타로 제공됩니다
</div>

보호 대상 건강 정보(PHI)에 대해 한층 강화된 보안을 원하는 고객은 이제 [Google Cloud Platform (GCP)](https://cloud.google.com/)에서 ClickHouse Cloud 이용을 시작할 수 있습니다. ClickHouse는 [HIPAA 보안 규칙](https://www.hhs.gov/hipaa/for-professionals/security/index.html)에서 규정한 관리적, 물리적, 기술적 보호 조치를 구현했으며, 이제 각 사용 사례와 워크로드에 따라 적용할 수 있는 구성 가능한 보안 설정을 제공합니다. 사용 가능한 보안 설정에 대한 자세한 내용은 [보안 기능 페이지](/ko/products/cloud/features/security)를 참조하십시오.

이 서비스는 GCP `us-central-1`에서 **Dedicated** service 유형 고객에게 제공되며, Business Associate Agreement(BAA)가 필요합니다. 이 기능에 대한 액세스를 요청하거나 추가 GCP, AWS, Azure 리전의 대기자 명단에 등록하려면 [sales](mailto:sales@clickhouse.com) 또는 [support](https://clickhouse.com/support/program)에 문의하십시오.

<div id="compute-compute-separation-is-now-in-private-preview-for-gcp-and-azure">
  ### 컴퓨트-컴퓨트 분리가 이제 GCP 및 Azure에서 비공개 프리뷰로 제공됩니다
</div>

최근 AWS용 컴퓨트-컴퓨트 분리 비공개 프리뷰를 발표했습니다. 이제 GCP 및 Azure에서도 사용할 수 있습니다.

컴퓨트-컴퓨트 분리를 사용하면 특정 서비스를 읽기-쓰기 서비스 또는 읽기 전용 서비스로 지정할 수 있으므로, 애플리케이션의 비용과 성능을 최적화할 수 있는 최적의 컴퓨트 구성을 설계할 수 있습니다. 자세한 내용은 [문서](/ko/products/cloud/features/infrastructure/warehouses)를 참조하십시오.

<div id="self-service-mfa-recovery-codes">
  ### 셀프서비스 MFA 복구 코드
</div>

다중 요소 인증(MFA)을 사용하는 고객은 휴대전화를 분실했거나 토큰을 실수로 삭제한 경우에 사용할 수 있는 복구 코드를 이제 받을 수 있습니다. 처음으로 MFA를 등록하는 고객에게는 설정 과정에서 이 코드가 제공됩니다. 이미 MFA를 사용 중인 고객은 기존 MFA 토큰을 제거한 후 새 토큰을 추가하여 복구 코드를 받을 수 있습니다.

<div id="clickpipes-update-custom-certificates-latency-insights-and-more">
  ### ClickPipes 업데이트: 사용자 지정 인증서, 지연 시간 인사이트 등
</div>

ClickHouse 서비스에 데이터를 수집하는 가장 쉬운 방법인 ClickPipes의 최신 업데이트를 소개합니다. 이번 새 기능은 데이터 수집에 대한 제어력을 높이고 성능 메트릭을 더욱 잘 파악할 수 있도록 설계되었습니다.

*Kafka용 사용자 지정 인증서*

이제 ClickPipes for Kafka는 SASL 및 공개 SSL/TLS를 사용하는 Kafka 브로커용 사용자 지정 인증서를 지원합니다. ClickPipe 설정 중 SSL Certificate 섹션에서 자체 인증서를 쉽게 업로드할 수 있으므로 Kafka에 더 안전하게 연결할 수 있습니다.

*Kafka 및 Kinesis용 지연 시간 메트릭 소개*

성능 가시성은 매우 중요합니다. 이제 ClickPipes는 지연 시간 그래프를 제공하여 메시지가 생성된 시점(Kafka 토픽 또는 Kinesis 스트림)부터 ClickHouse Cloud에 수집되기까지의 시간을 파악할 수 있습니다. 이 새 메트릭을 통해 데이터 파이프라인의 성능을 더 면밀하게 모니터링하고 그에 맞춰 최적화할 수 있습니다.

<Image img="/images/cloud/reference/oct-4-latency-insights.webp" size="lg" alt="데이터 수집 성능을 위한 지연 시간 메트릭 그래프를 보여주는 ClickPipes 인터페이스" border />

<br />

*Kafka 및 Kinesis용 스케일링 제어(비공개 베타)*

고처리량 워크로드에서는 데이터 볼륨과 지연 시간 요구 사항을 충족하기 위해 추가 리소스가 필요할 수 있습니다. ClickPipes를 위한 수평 스케일링을 Cloud Console에서 직접 사용할 수 있도록 도입하고 있습니다. 이 기능은 현재 비공개 베타로 제공되며, 요구 사항에 따라 리소스를 더욱 효과적으로 스케일링할 수 있습니다. 베타 참여를 원하면 [support](https://clickhouse.com/support/program)에 문의하십시오.

*Kafka 및 Kinesis용 원시 메시지 수집*

이제 Kafka 또는 Kinesis 메시지 전체를 파싱하지 않고 그대로 수집할 수 있습니다. 이제 ClickPipes는 전체 메시지를 하나의 String 컬럼에 매핑할 수 있도록 `_raw_message` [가상 컬럼](/ko/integrations/clickpipes/kafka/reference#kafka-virtual-columns)을 지원합니다. 이를 통해 필요에 따라 원시 데이터를 유연하게 활용할 수 있습니다.

* 처음부터: Kafka 토픽의 맨 처음부터 데이터 읽기를 시작합니다. 이 옵션은 모든 과거 데이터를 다시 처리해야 하는 경우에 적합합니다.

* 타임스탬프부터: 특정 타임스탬프 시점 또는 그 이후에 생성된 메시지부터 데이터 읽기를 시작합니다. 이 기능을 사용하면 더 정밀하게 제어할 수 있으며, 정확한 시점부터 처리를 재개할 수 있습니다.

ClickHouse Cloud는 [수평 스케일링](/ko/products/cloud/features/autoscaling/overview#how-scaling-works-in-clickhouse-cloud), 즉 서비스에 동일한 크기의 레플리카를 추가할 수 있는 기능을 지원합니다. 수평 스케일링은 성능과 병렬 처리 능력을 향상시켜 동시 쿼리 처리를 지원합니다. 이전에는 레플리카를 추가하려면 ClickHouse Cloud 콘솔이나 API를 사용해야 했습니다. 이제 Terraform을 사용해 서비스의 레플리카를 추가하거나 제거할 수 있으므로, 필요에 따라 ClickHouse 서비스를 프로그래밍 방식으로 스케일링할 수 있습니다.

자세한 내용은 [ClickHouse Terraform 프로바이더](https://registry.terraform.io/providers/ClickHouse/clickhouse/latest/docs)를 참조하십시오.
