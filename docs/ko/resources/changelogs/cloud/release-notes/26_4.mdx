<div id="backward-incompatible-changes">
  ## 하위 호환되지 않는 변경 사항
</div>

<div id="query-and-syntax-changes">
  ### 쿼리 및 구문 변경
</div>

* `IN` 연산자는 이제 `Bool` 타입에 대해 정확한 값 의미론을 사용합니다. 즉, 집합의 `0`과 `1`만 `Bool` 값과 일치합니다. 이전에는 `IN` 집합의 `255`보다 큰 숫자 값이 잘못해서 true로 보정되어 `SELECT CAST(1, 'Bool') IN (256)`이 `1`을 반환했습니다. 이제는 올바르게 `0`을 반환합니다. [#93115](https://github.com/ClickHouse/ClickHouse/pull/93115) ([Ashrith Bandla](https://github.com/ashrithb)).
* `NOT` 연산자 우선순위를 SQL 표준에 맞게 수정했습니다. 이제 `NOT`은 `IS NULL`, `BETWEEN`, `LIKE` 및 산술 연산자보다 느슨하게 결합됩니다. 예를 들어 `NOT (x) IS NULL`은 이제 `(NOT x) IS NULL`이 아니라 `NOT (x IS NULL)`로 해석됩니다. 이 변경으로 인해 이전의(표준이 아닌) 동작에 의존하던 쿼리 결과가 달라질 수 있습니다. [#97680](https://github.com/ClickHouse/ClickHouse/pull/97680) ([Alexey Milovidov](https://github.com/alexey-milovidov)).
* 이제 `WITH` 표현식 목록 요소에서 `SELECT`를 인용 부호 없는 식별자로 사용할 수 없습니다. [#101059](https://github.com/ClickHouse/ClickHouse/pull/101059) ([Aruj Bansal](https://github.com/arujbansal)).
* `IN` 연산자는 이제 복합 타입(`Tuple`, `Array`, `Map`) 내부에서 손실이 발생하는 `Decimal` 변환을 거부하므로, 동작이 최상위 스칼라 비교와 일관되게 됩니다. 이전에는 정밀도 검사가 최상위 스칼라 값에만 적용되었습니다. 예를 들어 `CAST('33.3', 'Decimal64(1)') IN (33.33)`은 올바르게 `0`을 반환했지만 `CAST(['33.3'], 'Array(Decimal64(1))') IN ([33.33])`은 잘못해서 `1`을 반환했습니다. 이제 두 경우 모두 올바르게 `0`을 반환합니다. [#101812](https://github.com/ClickHouse/ClickHouse/pull/101812) ([Nihal Z. Miaji](https://github.com/nihalzp)).

<div id="data-type-changes">
  ### 데이터 타입 변경 사항
</div>

* 이제 데이터 타입 직렬화 버전이 중첩된 데이터 타입에도 전파됩니다. 예를 들어 String 직렬화 버전 `with_size_stream`은 이전에는 최상위 String 컬럼과 Tuple 요소에만 적용되었지만, 이제는 Array, Map, Variant, JSON 등 중첩 타입 내부의 모든 String 타입에도 적용됩니다. 이는 설정 `propagate_types_serialization_versions_to_nested_types`로 제어되며, 현재 기본적으로 활성화되어 있습니다. 이 변경 이후 새로 생성된 데이터 파트는 이전 버전에서 읽을 수 없지만, 기존 파트는 새 버전에서 읽을 수 있습니다. 업그레이드는 안전하지만 다운그레이드는 안전하지 않습니다. [#94859](https://github.com/ClickHouse/ClickHouse/pull/94859) ([Pavel Kruglov](https://github.com/Avogar)).
* `QBit` 바이너리 직렬화의 엔디언을 리틀 엔디언 포맷을 사용하도록 수정했습니다. [#101378](https://github.com/ClickHouse/ClickHouse/pull/101378) ([Raufs Dunamalijevs](https://github.com/rienath)).

<div id="storage-and-index-changes">
  ### 스토리지 및 인덱스 변경 사항
</div>

* 여러 컬럼으로 구성된 정렬 키를 사용하는 프로젝션이 올바르게 인식되도록 일반 프로젝션의 메타데이터를 수정했습니다. [#91352](https://github.com/ClickHouse/ClickHouse/pull/91352) ([Amos Bird](https://github.com/amosbird)).
* 스킵 인덱스 파일이 `replace_long_file_name_to_hash` 설정을 따르지 않던 문제를 수정했습니다. 이 문제로 인해 이름이 긴 인덱스에서 &quot;File name too long&quot; 오류가 발생하고 인덱스를 읽지 못하는 문제가 있었습니다. 이제 스킵 인덱스 파일 이름은 컬럼 파일과 마찬가지로 `max_file_name_length`를 초과하면 해시 처리됩니다. 이는 하위 호환됩니다(새 서버는 기존 파트를 읽을 수 있습니다). 그러나 다운그레이드 시(또는 롤링 업그레이드 중 구버전 서버에서는) 이름이 긴 인덱스가 무시될 수 있습니다. [#97128](https://github.com/ClickHouse/ClickHouse/pull/97128) ([Raúl Marín](https://github.com/Algunenano)).

<div id="removed-features">
  ### 제거된 기능
</div>

* `hypothesis` 스킵 인덱스 유형을 제거했습니다. 이 기능은 잘 알려지지 않은 실험적 기능으로, 실제 활용도도 제한적이었습니다. 이제 `INDEX ... TYPE hypothesis`로 테이블을 생성하면 오류가 발생합니다. [#96874](https://github.com/ClickHouse/ClickHouse/pull/96874) ([Alexey Milovidov](https://github.com/alexey-milovidov)).
* 실험적 `detectProgrammingLanguage` 함수를 제거했습니다. [#99567](https://github.com/ClickHouse/ClickHouse/pull/99567) ([Alexey Milovidov](https://github.com/alexey-milovidov)).

<div id="settings-and-configuration-changes">
  ### 설정 및 구성 변경 사항
</div>

* `mysql_datatypes_support_level`의 기본값이 빈 값에서 `decimal,datetime64,date2Date32`로 변경되었습니다. 이로써 기본적으로 MySQL `DATE`는 `Date32`로, `DECIMAL`/`NUMERIC`은 `Decimal`로, 그리고 정밀도가 지정된 `DATETIME`/`TIMESTAMP`는 `DateTime64`로 올바르게 매핑됩니다. 이전에는 MySQL `DATE` 컬럼이 `Date`로 매핑되었는데, `Date`는 1970-01-01 이전 날짜를 표현할 수 없어 데이터 손상이 발생했습니다. [#97716](https://github.com/ClickHouse/ClickHouse/pull/97716) ([Alexey Milovidov](https://github.com/alexey-milovidov)).

<div id="insert-and-deduplication-changes">
  ### 삽입 및 중복 제거 변경 사항
</div>

* 이제 async insert가 기본적으로 활성화됩니다 — ClickHouse는 기본적으로 모든 작은 삽입을 배치로 묶어 처리합니다. 이는 `compatibility` 설정으로 제어됩니다. `compatibility`를 26.2보다 낮은 버전으로 설정하면 이전 기본값인 `false`로 복원됩니다. async inserts는 여러 수준에서 켜거나 끌 수 있습니다. 구성의 사용자 profile, session, 쿼리 또는 MergeTree table 수준에서 설정할 수 있습니다. [#97590](https://github.com/ClickHouse/ClickHouse/pull/97590) ([Sema Checherinda](https://github.com/CheSema)).

<div id="system-table-changes">
  ### 시스템 테이블 변경 사항
</div>

* `system.query_log`의 `skip_indexes` 컬럼 변경을 되돌렸습니다. [#101018](https://github.com/ClickHouse/ClickHouse/pull/101018) ([Alexey Milovidov](https://github.com/alexey-milovidov)).

<div id="other-breaking-changes">
  ### 기타 호환되지 않는 변경 사항
</div>

* `mergeTreeAnalyzeIndexes{,UUID}`는 이제 정규 표현식 대신 파트 이름 배열을 받습니다. 정규 표현식은 느리기 때문입니다(실험적 기능). [#98474](https://github.com/ClickHouse/ClickHouse/pull/98474) ([Azat Khuzhin](https://github.com/azat)).
* H3 라이브러리가 v4로 업데이트되어 길이, 면적 및 기타 메트릭 계산의 정밀도가 향상되었습니다. 새 결과가 이전 결과와 다르므로, 이 변경 사항은 하위 호환되지 않습니다. [#100348](https://github.com/ClickHouse/ClickHouse/pull/100348) ([Alexey Milovidov](https://github.com/alexey-milovidov)).
* `Merge` 테이블이 가상 컬럼(virtual columns)을 처리하는 방식이 변경되었습니다. 기본 테이블에 `_table` 또는 `_database`가 있으면 해당 컬럼은 스토리지에서 읽고, 그렇지 않으면 읽기 단계 이후 표현식 단계를 통해 채웁니다. [#101742](https://github.com/ClickHouse/ClickHouse/pull/101742) ([Mikhail Artemenko](https://github.com/Michicosun)).

<div id="new-features">
  ## 새 기능
</div>

<div id="functions">
  ### 함수
</div>

* 자연 정렬용 키를 위한 `naturalSortKey(s)` 함수를 추가했습니다. [#90322](https://github.com/ClickHouse/ClickHouse/pull/90322) ([Nazarii Piontko](https://github.com/nazarii-piontko)).
* 각 지연값에 대해 숫자 배열의 정규화된 자기상관을 계산하는 `arrayAutocorrelation(arr [, max_lag])` 함수를 추가했습니다. 정수, 부동소수점, Decimal 배열 타입을 지원합니다. [#94776](https://github.com/ClickHouse/ClickHouse/pull/94776) ([Wenyu Chen](https://github.com/wenyuchen96)).
* 이제 `Map`과 유사하게 경로 존재 여부를 확인할 수 있도록 JSON 타입에서 `has()` 함수가 지원됩니다. [#96927](https://github.com/ClickHouse/ClickHouse/pull/96927) ([DQ](https://github.com/il9ue)).
* 텍스트 인덱스에서 직접 데이터를 읽는 `mergeTreeTextIndex(database, table, index)` 테이블 함수를 추가했습니다. 내부 검사에 활용하거나 텍스트 인덱스 데이터에 대해 집계를 수행하는 데 사용할 수 있습니다. [#97003](https://github.com/ClickHouse/ClickHouse/pull/97003) ([Anton Popov](https://github.com/CurtizJ)).
* `obfuscateQuery` SQL 함수를 추가했습니다. [#98305](https://github.com/ClickHouse/ClickHouse/pull/98305) ([Xuewei Wang](https://github.com/Sallery-X)).
* 유니코드 대소문자 접기와 발음 구별 기호 제거를 위한 `caseFoldUTF8` 및 `removeDiacriticsUTF8` 함수를 추가했습니다. [#98973](https://github.com/ClickHouse/ClickHouse/pull/98973) ([George Larionov](https://github.com/george-larionov)).
* 이제 `printf` 함수가 상수가 아닌 포맷 문자열을 지원하므로, 컬럼 값에 따라 각 행마다 서로 다른 포맷 패턴을 사용할 수 있습니다. [#98991](https://github.com/ClickHouse/ClickHouse/pull/98991) ([Yash](https://github.com/Onyx2406)).
* 텍스트 문자열에서 검색어가 나타나는 부분을 HTML 태그(기본값 `<em>`/`</em>`)로 감싸는 `highlight()` 함수를 추가했습니다. ASCII 대소문자 구분 없는 일치, 겹치는 일치 항목의 자동 병합, 사용자 지정 여는/닫는 태그를 지원합니다. [#99131](https://github.com/ClickHouse/ClickHouse/pull/99131) ([Peng](https://github.com/fastio)).
* NFKC 정규화와 대소문자 접기를 결합한 NFKC&#95;Casefold 유니코드 정규화를 위한 `normalizeUTF8NFKCCasefold` 문자열 함수를 추가했습니다. [#99276](https://github.com/ClickHouse/ClickHouse/pull/99276) ([George Larionov](https://github.com/george-larionov)).
* 전문 검색 인덱스용 `unicode_word` 토크나이저와 `tokens` 함수를 추가했습니다. 유니코드 단어 경계 규칙을 사용해 텍스트를 분할합니다. ASCII 단어는 연결 문자(밑줄, 콜론, 점, 작은따옴표)를 포함해 구성되며, ASCII 이외의 유니코드 문자는 단일 문자 토큰이 됩니다. 구성 가능한 불용어의 기본값은 일반적인 CJK 문장 부호입니다. [#99357](https://github.com/ClickHouse/ClickHouse/pull/99357) ([Amos Bird](https://github.com/amosbird)).
* 2차원 배열(행렬)을 받아 전치하는 `arrayTranspose` 함수를 추가했습니다. [#101214](https://github.com/ClickHouse/ClickHouse/pull/101214) ([Vitaly Baranov](https://github.com/vitlibar)).
* 구문 검색(연속된 토큰 시퀀스)을 위한 `hasPhrase` 함수(별칭 `matchPhrase`)를 추가했습니다. 검색은 브루트포스 방식이며 아직 텍스트 인덱스에서는 지원되지 않습니다. [#101997](https://github.com/ClickHouse/ClickHouse/pull/101997) ([Elmi Ahmadov](https://github.com/ahmadov)).
* 벡터 유사성 인덱스에서 지원되는 거리 함수로 `dotProduct`를 추가했습니다. 이제 모든 거리 함수에 대해 정렬 방향을 검증하여 잘못된 결과가 아무 경고 없이 반환되는 일을 방지합니다. [#102254](https://github.com/ClickHouse/ClickHouse/pull/102254) ([Renzo](https://github.com/RenzoMXD)).
* 이제 `stem` 함수가 `String`, `FixedString`, `Array([Fixed]String)`, `Nullable`, `LowCardinality`, `Const` 컬럼의 모든 단어와 토큰에 대해 어간 추출을 수행할 수 있습니다. [#99137](https://github.com/ClickHouse/ClickHouse/pull/99137) ([Jimmy Aguilar Mena](https://github.com/Ergus)).
* `stem` 함수는 이제 실험적 기능이 아닙니다(이전에는 `allow_experimental_nlp_functions`를 활성화해야 했습니다). [#102399](https://github.com/ClickHouse/ClickHouse/pull/102399) ([Jimmy Aguilar Mena](https://github.com/Ergus)).
* `JSONAllValues` 함수가 추가되었습니다. 이 함수는 `JSON` 컬럼의 모든 값을 `Array(String)`으로 반환하며, 값은 텍스트 형식으로 직렬화되고 경로 이름 순으로 정렬됩니다. 또한 `JSON` 컬럼의 `JSONAllValues` 표현식에 대한 텍스트 인덱스 지원도 추가되었습니다. `JSONAllValues(json_column)`에 텍스트 인덱스를 생성하면 `JSON` 서브컬럼에 대한 쿼리 필터링(예: `json_column.key1 = 'value'`)에 자동으로 사용됩니다. [#100730](https://github.com/ClickHouse/ClickHouse/pull/100730) ([Anton Popov](https://github.com/CurtizJ)).
* `EXTRACT` 연산자에 PostgreSQL 호환 단위 `EPOCH`, `DOW`, `DOY`, `ISODOW`, `ISOYEAR`, `WEEK`, `CENTURY`, `DECADE`, `MILLENNIUM`이 추가되었습니다. 또한 이전에 오류를 발생시키던 `EXTRACT(WEEK FROM date)`도 수정되었습니다. [#100274](https://github.com/ClickHouse/ClickHouse/pull/100274) ([Alexey Milovidov](https://github.com/alexey-milovidov)).
* SQL 표준 `OVERLAY` 함수 구문에 대해 parser sugar를 통한 `toDaysInMonth()` 함수 지원이 추가되었습니다. `overlay` 함수는 이미 존재하며, 이번 변경으로 `PLACING`, `FROM`, `FOR`를 구분자로 사용하는 키워드 기반 형식도 지원됩니다. [#101681](https://github.com/ClickHouse/ClickHouse/pull/101681) ([Desel72](https://github.com/Desel72)).
* 이제 `Date` 및 `Date32` 값을 `+` 연산자를 사용해 `Time` 및 `Time64` 값과 더할 수 있으며, 그 결과로 `DateTime` 또는 `DateTime64`가 생성됩니다. 예를 들어 `SELECT toDate('2024-01-15') + toTime('14:30:25')`는 `2024-01-15 14:30:25`를 반환합니다. 결과는 세션 시간대에서 계산되며, 범위를 벗어난 결과는 `date_time_overflow_behavior` 설정에 따라 처리됩니다. [#102421](https://github.com/ClickHouse/ClickHouse/pull/102421) ([Nihal Z. Miaji](https://github.com/nihalzp)).

<div id="sql-and-query-features">
  ### SQL 및 쿼리 기능
</div>

* 매개변수의 유형이 `Nullable`이고 값이 지정되지 않은 경우, 이제 해당 값은 `NULL`로 간주됩니다. [#93869](https://github.com/ClickHouse/ClickHouse/pull/93869) ([Vikash Kumar](https://github.com/vikashkumar2020)).
* 이제 ClickHouse는 min/max 통계를 기반으로 `SELECT` 쿼리에서 전체 데이터 파트를 걸러낼 수 있습니다. [#94140](https://github.com/ClickHouse/ClickHouse/pull/94140) ([zoomxi](https://github.com/zoomxi)).
* Materialized CTEs 지원이 추가되었습니다. 이 기능은 쿼리 실행 중 CTE를 한 번만 평가하고 결과를 임시 테이블에 저장합니다. [#94849](https://github.com/ClickHouse/ClickHouse/pull/94849) ([Dmitry Novik](https://github.com/novikd)).
* 이제 일부 함수는 SQL에서 괄호 없이 사용할 수 있습니다. [#95949](https://github.com/ClickHouse/ClickHouse/pull/95949) ([Aly Kafoury](https://github.com/AlyHKafoury)).
* `JSON` 유형에 결합 서브컬럼 구문 `json.@path`가 추가되었습니다. 경로에 스칼라가 있으면 리터럴 값을 `Dynamic`으로 반환하고, 경로에 중첩 객체가 있으면 하위 객체를 `Dynamic`으로 반환하며, 경로가 없으면 `NULL`을 반환합니다. [#98788](https://github.com/ClickHouse/ClickHouse/pull/98788) ([Pavel Kruglov](https://github.com/Avogar)).
* 서브쿼리 표현식용 `SOME` 키워드가 추가되었습니다. 동작은 `ANY`와 동일합니다. [#99842](https://github.com/ClickHouse/ClickHouse/pull/99842) ([Artem Kytkin](https://github.com/Vinceent)).
* `SET session_timezone`의 별칭으로 `SET TIME ZONE 'tz'` 지원이 추가되었습니다. [#99883](https://github.com/ClickHouse/ClickHouse/pull/99883) ([phulv94](https://github.com/phulv94)).
* SQL 표준 `VALUES` 절을 `FROM`의 테이블 표현식으로 사용할 수 있도록 지원이 추가되었습니다. 예: `SELECT * FROM (VALUES (1, 'a'), (2, 'b')) AS t(id, val)`. [#100143](https://github.com/ClickHouse/ClickHouse/pull/100143) ([Desel72](https://github.com/Desel72)).
* `TO` 범위 한정자가 포함된 SQL 표준 복합 인터벌 리터럴 지원이 추가되었습니다. 예: `INTERVAL '1:30' HOUR TO MINUTE`. 내부적으로는 인터벌의 합으로 분해됩니다. [#100453](https://github.com/ClickHouse/ClickHouse/pull/100453) ([Desel72](https://github.com/Desel72)).
* ordered 및 unordered 모드용 `SYSTEM FLUSH OBJECT STORAGE QUEUE db.table PATH 'x'` 명령이 추가되었습니다. [#100709](https://github.com/ClickHouse/ClickHouse/pull/100709) ([Bharat Nallan](https://github.com/bharatnc)).

<div id="table-engines-and-storage">
  ### 테이블 엔진 및 스토리지
</div>

* Keeper 기반 스냅샷 진행 상태 추적과 함께 `Paimon` 테이블 엔진에 대한 증분 읽기 지원이 추가되었으며, `paimon_target_snapshot_id`를 통한 특정 스냅샷 델타 읽기도 포함됩니다. [#93655](https://github.com/ClickHouse/ClickHouse/pull/93655) ([XiaoBinMu](https://github.com/Binnn-MX)).
* `DatabaseReplicated`에 보조 ZooKeeper 지원이 추가되었습니다. [#95590](https://github.com/ClickHouse/ClickHouse/pull/95590) ([RinChanNOW](https://github.com/RinChanNOWWW)).
* Iceberg v3 나노초 타임스탬프 타입(`timestamp_ns` 및 `timestamptz_ns`) 읽기 지원이 추가되었으며, 각각 `DateTime64(9)` 및 `DateTime64(9, 'UTC')`로 매핑됩니다. 이를 통해 ClickHouse는 스키마에 이러한 타입이 포함된 Iceberg 데이터를 읽을 수 있습니다. 다만 이는 Iceberg v3 전체를 지원한다는 의미는 아니며, 이러한 타입으로 Iceberg v3에 쓰기를 지원하는 것도 아닙니다. [#97132](https://github.com/ClickHouse/ClickHouse/pull/97132) ([Dmitry Kovalev](https://github.com/dk-github)).
* Iceberg 테이블용 `ALTER TABLE ... EXECUTE expire_snapshots('<timestamp>')`가 추가되었습니다. [#97904](https://github.com/ClickHouse/ClickHouse/pull/97904) ([murphy-4o](https://github.com/murphy-4o)).
* 메타데이터만 읽기 위해 파일을 다시 다운로드할 필요를 없애 읽기 성능을 개선하는 Parquet 메타데이터용 새 SLRU 캐시가 추가되었습니다. [#98140](https://github.com/ClickHouse/ClickHouse/pull/98140) ([Grant Holly](https://github.com/grantholly-clickhouse)).
* `JSONAllPaths`를 사용하는 JSON 컬럼에 대해 `bloom_filter`, `tokenbf_v1`, `ngrambf_v1`, `text`(역색인) 인덱스 타입과 함께 MergeTree 스킵 인덱스 지원이 추가되어, 각 granule에 존재하는 JSON 경로 집합을 기준으로 granule 스키핑이 가능해졌습니다. [#98886](https://github.com/ClickHouse/ClickHouse/pull/98886) ([Pavel Kruglov](https://github.com/Avogar)).
* 삽입 순서에 따라 데이터를 재구성하는 `commit_order` 프로젝션 인덱스가 추가되었습니다. [#99004](https://github.com/ClickHouse/ClickHouse/pull/99004) ([Mikhail Artemenko](https://github.com/Michicosun)).
* MergeTree의 `Map` 컬럼에 버킷 직렬화가 추가되었습니다(`map_serialization_version = 'with_buckets'`). 키는 해시 기반 버킷으로 분할되므로 단일 키(`m['key']`)를 읽을 때 전체 컬럼 대신 하나의 버킷만 읽게 되어, 맵 크기에 따라 단일 키 조회 성능이 2~49배 향상됩니다. 버킷 수와 버킷팅 전략은 새로운 MergeTree 설정 `map_serialization_version`, `max_buckets_in_map`, `map_buckets_strategy`, `map_buckets_coefficient`, `map_buckets_min_avg_size`로 제어할 수 있습니다. `map_serialization_version_for_zero_level_parts` 설정은 삽입 시 쓰기 오버헤드를 피하기 위해 기본 직렬화를 유지하고, 병합된 파트에는 버킷을 사용하도록 합니다. `mapKeys`/`mapValues`에 대한 스킵 인덱스는 버킷 직렬화와 함께 지원되며, 버킷을 사용할 수 있을 때 `optimize_functions_to_subcolumns`는 `m['key']`를 키별 서브컬럼 읽기로 재작성합니다. [#99200](https://github.com/ClickHouse/ClickHouse/pull/99200) ([Pavel Kruglov](https://github.com/Avogar)).

<div id="settings-and-configuration">
  ### 설정 및 구성
</div>

* 테이블을 읽기 전용으로 표시해 삽입과 수정을 방지하는 `table_readonly` MergeTree 설정이 추가되었습니다. [#97652](https://github.com/ClickHouse/ClickHouse/pull/97652) ([Alexey Milovidov](https://github.com/alexey-milovidov)).
* `use_partition_pruning` 설정(별칭 `use_partition_key`)이 추가되었습니다. 파티션 키 기반 파티션 프루닝을 비활성화하려면 이 값을 `false`로 설정하십시오. [#97888](https://github.com/ClickHouse/ClickHouse/pull/97888) ([Nihal Z. Miaji](https://github.com/nihalzp)).
* 이제 `<protocols>`의 각 `type=http` 항목에서 별도의 `<http_handlers_*>` 구성 섹션을 가리키는 사용자 지정 `<handlers>` 키를 지정할 수 있으므로, 포트별로 서로 다른 HTTP 라우팅 규칙을 적용할 수 있습니다. [#98414](https://github.com/ClickHouse/ClickHouse/pull/98414) ([Amos Bird](https://github.com/amosbird)).
* `replicated_fetches_min_part_level` 및 `replicated_fetches_min_part_level_timeout_seconds`의 두 MergeTree 설정이 추가되었습니다. 이 설정을 사용하면 레플리카가 피어에서 새로 삽입된(머지되지 않은) 파트를 가져오는 작업을 건너뛸 수 있어, 대량 수집 중 복제 오버헤드를 줄일 수 있습니다. [#98625](https://github.com/ClickHouse/ClickHouse/pull/98625) ([tanner-bruce](https://github.com/tanner-bruce)).
* `restore_access_entities_with_current_grants` 서버 설정이 추가되었습니다. 이 설정을 활성화하면 복원된 사용자/역할의 권한 부여 범위가 복원을 수행하는 사용자가 부여할 수 있는 권한으로 제한되며(`GRANT CURRENT GRANTS`와 동일한 의미), `ACCESS_DENIED` 오류로 실패하지 않습니다. [#98795](https://github.com/ClickHouse/ClickHouse/pull/98795) ([pufit](https://github.com/pufit)).
* `skip_unavailable_shards`가 활성화된 경우 조용히 건너뛸 수 있는 세그먼트 수를 제한하는 `max_skip_unavailable_shards_num` 및 `max_skip_unavailable_shards_ratio` 설정이 추가되었습니다. 사용 불가능한 세그먼트의 수 또는 비율이 구성된 임계값을 초과하면, 조용히 불완전한 결과를 반환하는 대신 예외가 발생합니다. [#99369](https://github.com/ClickHouse/ClickHouse/pull/99369) ([Alexey Milovidov](https://github.com/alexey-milovidov)).
* 공개 ClickHouse 서비스를 오남용으로부터 보호하기 위해 정규화된 쿼리 해시 기반 쿼터가 구현되었습니다. `NORMALIZED_QUERY_HASH`는 쿼터 키 유형으로 지원되며, 고유한 각 정규화된 쿼리마다 별도의 쿼터 버킷을 사용하므로 `CREATE QUOTA q KEYED BY normalized_query_hash`는 서로 다른 각 쿼리를 독립적으로 추적합니다. `QUERIES_PER_NORMALIZED_HASH`는 쿼터 리소스 유형으로 지원되며, 인터벌 내에서 단일 정규화된 쿼리의 최대 실행 횟수를 제한하므로 `MAX queries_per_normalized_hash = 100`은 하나의 쿼리 패턴이 100회를 초과해 실행되지 않도록 합니다. [#99586](https://github.com/ClickHouse/ClickHouse/pull/99586) ([Alexey Milovidov](https://github.com/alexey-milovidov)).
* 이제 `auto_statistics_types` MergeTree 설정의 기본값이 `minmax, uniq`이므로, 새 테이블의 적합한 모든 컬럼에 대해 minmax 및 uniq 통계가 자동으로 생성됩니다. 또한 `materialize_statistics_on_insert`의 기본값이 `false`로 변경되어, 통계는 삽입 시점이 아니라 머지 중에 생성되므로 삽입 오버헤드를 줄일 수 있습니다. 이전 동작으로 되돌리려면 `materialize_statistics_on_insert = 1`로 설정하십시오. [#101275](https://github.com/ClickHouse/ClickHouse/pull/101275) ([Han Fei](https://github.com/hanfei1991)).
* 레플리카 간 복제 지연으로 인한 누락 데이터를 줄이기 위해 materialized view 종속성 체인용 `prefer_dependency_replica` 갱신 설정이 추가되었습니다. [#101591](https://github.com/ClickHouse/ClickHouse/pull/101591) ([Seva Potapov](https://github.com/seva-potapov)).

* 이제 `users_xml` 구성에서 여러 인증 메서드를 지정할 수 있습니다. [#91998](https://github.com/ClickHouse/ClickHouse/pull/91998) ([Flip-Liquid](https://github.com/Flip-Liquid)).

* `allow_nullable_tuple_in_extracted_subcolumns` 설정이 추가되었습니다. 이 설정은 `Tuple`, `Variant`, `Dynamic`, `JSON`에서 추출한 `Tuple(...)` 서브컬럼을 `Nullable(Tuple(...))`로 반환할지(누락된 행은 `NULL`) 또는 `Tuple(...)`로 반환할지(누락된 행은 기본 tuple 값) 제어합니다. 기본적으로 비활성화되어 있으며, server를 재시작해야만 변경할 수 있습니다. [#97299](https://github.com/ClickHouse/ClickHouse/pull/97299) ([Nihal Z. Miaji](https://github.com/nihalzp)).
* 이제 `type_json_allow_duplicated_key_with_literal_and_nested_object`가 기본적으로 활성화되어, ClickHouse가 원본 데이터 `{"a" : 42, "a.b" : 42}`에서 생성할 수 있는 `{"a" : 42, "a" : {"b" : 42}}`와 같은 JSON을 파싱할 때 발생하는 중복 키 관련 오류를 방지합니다. [#97487](https://github.com/ClickHouse/ClickHouse/pull/97487) ([Han Fei](https://github.com/hanfei1991)).
* `enable_parallel_replicas`와 `automatic_parallel_replicas_mode`의 관계를 명확히 했습니다. 쿼리는 `enable_parallel_replicas > 0`인 경우에만 병렬 레플리카를 사용할 수 있습니다. `automatic_parallel_replicas_mode=1`이면 수집된 통계를 기반으로 계획 단계에서 결정되며, `automatic_parallel_replicas_mode=0`이면 통계와 관계없이 지원되는 모든 쿼리에 병렬 레플리카가 사용됩니다. 병렬 레플리카를 사용하는 Distributed insert-select는 항상 `automatic_parallel_replicas_mode=0`인 것처럼 동작합니다. [#97517](https://github.com/ClickHouse/ClickHouse/pull/97517) ([Nikita Taranov](https://github.com/nickitat)).
* impersonate 기능([EXECUTE AS target&#95;user](/ko/reference/statements/execute_as))이 기본적으로 활성화되었습니다. [#97870](https://github.com/ClickHouse/ClickHouse/pull/97870) ([Vitaly Baranov](https://github.com/vitlibar)).
* `access_control_improvements.disallow_config_defined_profiles_for_sql_defined_users` 설정이 추가되었습니다(기본적으로 허용됨). 이 설정은 SQL로 정의된 사용자에 대해 구성에서 정의된 settings profiles(`default` profile 제외)을 사용하는 것을 금지합니다. * `Variant` 및 `Dynamic`의 타입 불일치 시 동작을 제어하는 설정이 추가되었습니다(`throw` 또는 null 반환). [#99085](https://github.com/ClickHouse/ClickHouse/pull/99085) ([Bharat Nallan](https://github.com/bharatnc)).

  ### S3/Azure/객체 스토리지 수정 사항

  * `CachedOnDiskReadBufferFromFile::readBigAt`에서 발생하던 예외를 수정했습니다. [#97890](https://github.com/ClickHouse/ClickHouse/pull/97890) ([Kseniia Sumarokova](https://github.com/kssenii)).
  * 글롭 패턴(예: `file('dir/**', 'LineAsString')`)으로 파일을 쿼리할 때 디렉터리에 끊어진 심볼릭 링크가 있으면 처리되지 않은 파일 시스템 예외로 인해 크래시가 발생하던 문제를 수정했습니다. 이제 끊어진 심볼릭 링크는 자동으로 건너뜁니다. [#98143](https://github.com/ClickHouse/ClickHouse/pull/98143) ([Mark Andreev](https://github.com/mrk-andreev)).
  * ArrowStream 리더가 처음 몇 바이트를 매우 큰 메타데이터 길이로 잘못 해석해, Arrow가 아닌 데이터를 읽을 때(예: 명시적 포맷 없이 `url()` 또는 `file()`에서 읽는 JSON) 포맷 자동 감지 중 과도한 메모리 사용량(~514 MiB)이 발생하던 문제를 수정했습니다. [#98893](https://github.com/ClickHouse/ClickHouse/pull/98893) ([Konstantin Bogdanov](https://github.com/thevar1able)).
  * `urlCluster` 또는 유사한 클러스터 테이블 함수를 사용하는 쿼리에서 &quot;ReadBuffer is canceled&quot; 예외를 유발할 수 있던 race condition을 수정했습니다. [#98955](https://github.com/ClickHouse/ClickHouse/pull/98955) ([Alexey Milovidov](https://github.com/alexey-milovidov)).
  * 글롭 패턴과 함께 `url()` 테이블 함수를 통해 존재하지 않는 압축 파일을 읽을 때 오해를 불러일으키던 &quot;inflate failed: buffer error&quot; 메시지를 수정했습니다. [#99034](https://github.com/ClickHouse/ClickHouse/pull/99034) ([Alexey Milovidov](https://github.com/alexey-milovidov)).
  * S3 중괄호 확장 글롭 패턴에서 발생하던 N+1 `HeadObject` 호출을 수정했습니다. [#99219](https://github.com/ClickHouse/ClickHouse/pull/99219) ([Konstantin Bogdanov](https://github.com/thevar1able)).
  * Hive 파티셔닝 경로에서 앞에 0이 붙은 숫자로 인해 오류가 발생하던 경우를 수정했습니다. [#99458](https://github.com/ClickHouse/ClickHouse/pull/99458) ([Yarik Briukhovetskyi](https://github.com/yariks5s)).
  * `O_DIRECT`를 사용할 때 `AsynchronousReadBufferFromFileDescriptor`에서 seek 처리가 잘못되던 문제를 수정했습니다. [#99678](https://github.com/ClickHouse/ClickHouse/pull/99678) ([Pavel Kruglov](https://github.com/Avogar)).
  * HEAD 요청 응답에 `Content-Length` 헤더가 없을 경우(예: GCS의 압축 해제 트랜스코딩 때문) ClickHouse가 파일을 건너뛸 수 있던 버그를 수정했습니다. [#99971](https://github.com/ClickHouse/ClickHouse/pull/99971) ([Yarik Briukhovetskyi](https://github.com/yariks5s)).
  * Azure blob storage 디스크가 구성되어 있지만 엔드포인트에 일시적으로 연결할 수 없는 경우(예: DNS 실패)에도 이제 서버가 시작에 실패하지 않습니다. [#100701](https://github.com/ClickHouse/ClickHouse/pull/100701) ([Raúl Marín](https://github.com/Algunenano)).
  * 연결 풀 use-after-free로 인해 `s3Cluster`와 분산 쿼리에서 발생하던 segfault를 수정했습니다. [#100837](https://github.com/ClickHouse/ClickHouse/pull/100837) ([Konstantin Bogdanov](https://github.com/thevar1able)).
  * S3 `Client::~Client` destructor에서 예외가 전파되어 서버가 종료되던 문제를 수정했습니다. [#101798](https://github.com/ClickHouse/ClickHouse/pull/101798) ([Gagan Dhakrey](https://github.com/gagandhakrey)).
  * 재연결 중 host와 port가 올바르게 기록되지 않던 문제를 수정했습니다. [#102280](https://github.com/ClickHouse/ClickHouse/pull/102280) ([Grant Holly](https://github.com/grantholly-clickhouse)).

  ### 보안 및 접근 제어 버그 수정

  * `loop` 테이블 함수가 `inner_storage->read()`를 직접 호출하면서, 행 정책, 컬럼 수준 권한, 기타 보안 검사가 적용되는 인터프리터 계층을 우회하던 문제를 수정했습니다. 이로 인해 행 정책으로 제한된 사용자도 `loop(table)`을 통해 모든 행을 읽을 수 있었습니다. [#97682](https://github.com/ClickHouse/ClickHouse/pull/97682) ([pufit](https://github.com/pufit)).
  * 스칼라 `file()` 및 `DESCRIBE TABLE file()`에 대해 `READ ON FILE` 검사를 강제합니다. [#98115](https://github.com/ClickHouse/ClickHouse/pull/98115) ([Nikolay Degterinsky](https://github.com/evillique)).
  * HTTP Basic Auth에서 패딩이 없는 base64 자격 증명도 허용합니다. 일부 HTTP 클라이언트는 `Authorization: Basic` 헤더의 끝에 있는 `=` 패딩을 생략하는데, 이전에는 이로 인해 authentication이 실패했습니다. [#98392](https://github.com/ClickHouse/ClickHouse/pull/98392) ([Amos Bird](https://github.com/amosbird)).
  * `SHOW_COLUMNS` 권한 없이도 localhost를 가리키는 `remote()`, `remoteSecure()`, `cluster()`, 또는 `clusterAllReplicas()`를 통해 임의의 테이블을 `DESCRIBE`할 수 있게 했던 RBAC 우회를 수정했습니다. [#98669](https://github.com/ClickHouse/ClickHouse/pull/98669) ([pufit](https://github.com/pufit)).
  * `system.grants`의 `access_object` 컬럼에서 `URL` 및 `S3` 권한 부여의 정규식 매개변수가 누락되던 문제를 수정했습니다. [#98987](https://github.com/ClickHouse/ClickHouse/pull/98987) ([DQ](https://github.com/il9ue)).
  * 필요한 소스 접근 권한 없이도 테이블 함수(`mysql()`, `postgresql()`, `sqlite()`, `arrowFlight()`, `jdbc()`, `odbc()` 등)에 대해 `DESCRIBE TABLE` 또는 `CREATE TABLE AS`를 사용해 테이블 구조를 확인할 수 있게 했던 RBAC 우회를 수정했습니다. 원격 서버에서 스키마를 추론하는 함수의 경우, 이로 인해 권한 없이 아웃바운드 연결(SSRF)을 트리거할 수도 있었습니다. [#99122](https://github.com/ClickHouse/ClickHouse/pull/99122) ([pufit](https://github.com/pufit)).
  * 분산 DDL 쿼리에 대해 DDL worker에서 설정 제약 조건 값을 허용 범위로 제한합니다. [#99317](https://github.com/ClickHouse/ClickHouse/pull/99317) ([Pablo Marcos](https://github.com/pamarcos)).
  * TOTP authentication과 관련된 사소한 문제를 수정했습니다. 빈 비밀번호와 함께 사용하는 `--one-time-password` CLI 옵션, 그리고 `<digits>` 및 `<period>` 구성 값 검증이 대상입니다. [#99322](https://github.com/ClickHouse/ClickHouse/pull/99322) ([Vladimir Cherkasov](https://github.com/vdimir)).
  * 이제 JDBC, ODBC 및 NATS 연결 문자열의 자격 증명은 쿼리 로그와 `SHOW CREATE` 출력에서 마스킹됩니다. URI 형식의 연결 문자열에서는 비밀번호 부분만 마스킹됩니다. `nats_token` 설정도 이제 마스킹됩니다. [#99344](https://github.com/ClickHouse/ClickHouse/pull/99344) ([János Benjamin Antal](https://github.com/antaljanosbenjamin)).
  * 로컬 파일에서 Google 자격 증명을 읽는 기능을 금지했습니다. 파일 경로를 알고 있으면 다른 자격 증명까지 읽을 수 있어 안전하지 않았기 때문입니다. [#99584](https://github.com/ClickHouse/ClickHouse/pull/99584) ([Konstantin Vedernikov](https://github.com/scanhex12)).
  * 인라인 DDL 매개변수에서 MySQL 딕셔너리 소스가 `RemoteHostFilter`를 우회하던 문제를 수정했습니다. [#99720](https://github.com/ClickHouse/ClickHouse/pull/99720) ([Shaohua Wang](https://github.com/tiandiwonder)).

  * HTTP 핸들러에서 `query_id`를 정제해 `X-ClickHouse-Query-Id` 헤더를 통한 응답 헤더 CRLF 주입을 방지합니다. [#100500](https://github.com/ClickHouse/ClickHouse/pull/100500) ([Pablo Marcos](https://github.com/pamarcos)).
