Há cenários em que pode ser útil ressincronizar tabelas específicas de um pipe. Alguns exemplos de casos de uso incluem alterações significativas de schema no MySQL ou alguma remodelagem de dados no ClickHouse.

Embora a ressincronização de tabelas individuais com um clique ainda esteja em desenvolvimento, este guia mostra as etapas para fazer isso hoje no ClickPipe do MySQL.

<div id="removing-table">
  ### 1. Remova a tabela do pipe
</div>

Para isso, siga o [guia de remoção de tabela](/pt-BR/integrations/clickpipes/mysql/remove-table).

<div id="truncate-drop-table">
  ### 2. Fazer TRUNCATE ou remover a tabela no ClickHouse
</div>

Esta etapa serve para evitar a duplicação de dados quando essa tabela for adicionada novamente na próxima etapa. Você pode fazer isso acessando a aba **SQL Console** no ClickHouse Cloud e executando uma consulta.
Observe que há uma validação para bloquear a adição da tabela caso ela já exista no ClickHouse e não esteja vazia.

<div id="add-table-again">
  ### 3. Adicione novamente a tabela ao ClickPipe
</div>

Isso pode ser feito seguindo o [guia para adicionar a tabela](/pt-BR/integrations/clickpipes/mysql/add-table).
