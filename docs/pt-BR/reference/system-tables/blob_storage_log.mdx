<div id="description">
  ## Descrição
</div>

Contém registros de log com informações sobre várias operações de armazenamento de blobs, como uploads e exclusões.

<div id="columns">
  ## Colunas
</div>

{/*AUTOGENERATED_START*/}

* `hostname` ([LowCardinality(String)](/pt-BR/reference/data-types/lowcardinality)) — Hostname do servidor que executa a consulta.
* `event_date` ([Date](/pt-BR/reference/data-types/date)) — Data do evento.
* `event_time` ([DateTime](/pt-BR/reference/data-types/datetime)) — Horário do evento.
* `event_time_microseconds` ([DateTime64(6)](/pt-BR/reference/data-types/datetime64)) — Horário do evento com precisão de microssegundos.
* `event_type` ([Enum8(&#39;Upload&#39; = 1, &#39;Delete&#39; = 2, &#39;MultiPartUploadCreate&#39; = 3, &#39;MultiPartUploadWrite&#39; = 4, &#39;MultiPartUploadComplete&#39; = 5, &#39;MultiPartUploadAbort&#39; = 6, &#39;Read&#39; = 7)](/pt-BR/reference/data-types/enum)) — Tipo do evento. Valores possíveis: &#39;Upload&#39;, &#39;Delete&#39;, &#39;MultiPartUploadCreate&#39;, &#39;MultiPartUploadWrite&#39;, &#39;MultiPartUploadComplete&#39;, &#39;MultiPartUploadAbort&#39;, &#39;Read&#39;
* `query_id` ([String](/pt-BR/reference/data-types/string)) — Identificador da consulta associada ao evento, se houver.
* `thread_id` ([UInt64](/pt-BR/reference/data-types/int-uint)) — Identificador da thread que executa a operação.
* `thread_name` ([String](/pt-BR/reference/data-types/string)) — Nome da thread que executa a operação.
* `disk_name` ([LowCardinality(String)](/pt-BR/reference/data-types/lowcardinality)) — Nome do disco associado.
* `bucket` ([String](/pt-BR/reference/data-types/string)) — Nome do bucket.
* `remote_path` ([String](/pt-BR/reference/data-types/string)) — Caminho para o recurso remoto.
* `local_path` ([String](/pt-BR/reference/data-types/string)) — Caminho para o arquivo de metadados no sistema local, que faz referência ao recurso remoto.
* `data_size` ([UInt64](/pt-BR/reference/data-types/int-uint)) — Tamanho dos dados envolvidos no evento de upload.
* `elapsed_microseconds` ([UInt64](/pt-BR/reference/data-types/int-uint)) — Tempo decorrido da operação, em microssegundos.
* `error_code` ([Int32](/pt-BR/reference/data-types/int-uint)) — Código de erro da operação. 0 se não houve erro.
* `error` ([String](/pt-BR/reference/data-types/string)) — Mensagem de erro associada ao evento, se houver.

{/*AUTOGENERATED_END*/}

<div id="example">
  ## Exemplo
</div>

Suponha que uma operação no armazenamento de blobs faça upload de um arquivo e que um evento seja registrado:

```sql
SELECT * FROM system.blob_storage_log WHERE query_id = '7afe0450-504d-4e4b-9a80-cd9826047972' ORDER BY event_date, event_time_microseconds \G
```

```text
Row 1:
──────
hostname:                clickhouse.eu-central1.internal
event_date:              2023-10-31
event_time:              2023-10-31 16:03:40
event_time_microseconds: 2023-10-31 16:03:40.481437
event_type:              Upload
query_id:                7afe0450-504d-4e4b-9a80-cd9826047972
thread_id:               2381740
disk_name:               disk_s3
bucket:                  bucket1
remote_path:             rrr/kxo/tbnqtrghgtnxkzgtcrlutwuslgawe
local_path:              store/654/6549e8b3-d753-4447-8047-d462df6e6dbe/tmp_insert_all_1_1_0/checksums.txt
data_size:               259
error:
```

Neste exemplo, a operação de upload foi associada à consulta `INSERT` com o ID `7afe0450-504d-4e4b-9a80-cd9826047972`. O arquivo local de metadados `store/654/6549e8b3-d753-4447-8047-d462df6e6dbe/tmp_insert_all_1_1_0/checksums.txt` faz referência ao caminho remoto `rrr/kxo/tbnqtrghgtnxkzgtcrlutwuslgawe` no bucket `bucket1` no disco `disk_s3`, com tamanho de 259 bytes.
