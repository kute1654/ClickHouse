<div id="description">
  ## Descrição
</div>

Contém informações sobre o número de eventos ocorridos no sistema. Por exemplo, na tabela, é possível ver quantas consultas `SELECT` foram processadas desde o início do servidor ClickHouse.

<div id="columns">
  ## Colunas
</div>

{/*AUTOGENERATED_START*/}

* `event` ([String](/pt-BR/reference/data-types/index)) — Nome do evento.
* `value` ([UInt64](/pt-BR/reference/data-types/index)) — Número de ocorrências do evento.
* `description` ([String](/pt-BR/reference/data-types/index)) — Descrição do evento.

{/*AUTOGENERATED_END*/}

<div id="example">
  ## Exemplo
</div>

```sql
SELECT * FROM system.events LIMIT 5
```

┌─event─────────────────────────────────┬─value─┬─description────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────┐

Does not include queries that failed to parse or were rejected due to AST size limits, quota limits or limits on the number of simultaneously running queries. Does not include sockets.                                                                                                                                                                             │

  ### DataAfterMergeDiffersFromReplica
</div>

Número de vezes em que os dados após a mesclagem não são byte a byte idênticos aos dados em outras réplicas. Pode haver vários motivos:

1. Uso de uma versão mais recente da biblioteca de compressão após a atualização do servidor.
2. Uso de outro método de compressão.
3. Algoritmo de compressão não determinístico (altamente improvável).
4. Algoritmo de mesclagem não determinístico devido a um erro lógico no código.
5. Corrupção de dados na memória devido a um bug no código.
6. Corrupção de dados na memória devido a um problema de hardware.
7. Modificação manual dos dados de origem após a inicialização do servidor.
8. Modificação manual das somas de verificação armazenadas no ZooKeeper.
9. Configurações relacionadas ao formato das partes, como &#39;enable&#95;mixed&#95;granularity&#95;parts&#39;, são diferentes entre as réplicas.
   O servidor detectou essa situação com sucesso e baixará a parte mesclada da réplica para garantir um resultado byte a byte idêntico.
