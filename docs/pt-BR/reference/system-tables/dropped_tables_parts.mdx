<div id="description">
  ## Descrição
</div>

Contém informações sobre partes de tabelas [MergeTree](/pt-BR/reference/engines/table-engines/mergetree-family/mergetree) removidas em [system.dropped&#95;tables](/pt-BR/reference/system-tables/dropped_tables)

O esquema desta tabela é o mesmo de [system.parts](/pt-BR/reference/system-tables/parts)

<div id="columns">
  ## Colunas
</div>

{/*AUTOGENERATED_START*/}

* `partition` ([String](/pt-BR/reference/data-types/index)) — Identificador da partição (representação textual derivada da expressão da chave de partição da tabela).
* `name` ([String](/pt-BR/reference/data-types/index)) —
  Nome da parte de dados. A estrutura de nomenclatura da parte pode ser usada para determinar muitos aspectos dos padrões de dados, de ingestão e de merge. O formato de nomenclatura da parte é o seguinte:

```text
<partition_id>_<minimum_block_number>_<maximum_block_number>_<level>_<data_version>
```

* Definições:
  * `partition_id` - identifica a chave de partição
  * `minimum_block_number` - identifica o número mínimo do bloco na parte. O ClickHouse sempre faz merge de blocos contínuos
  * `maximum_block_number` - identifica o número máximo do bloco na parte
  * `level` - incrementado em um a cada merge adicional na parte. Um nível 0 indica que esta é uma nova parte que ainda não passou por merge. É importante lembrar que todas as partes no ClickHouse são sempre imutáveis
  * `data_version` - valor opcional, incrementado quando uma parte é modificada (novamente, os dados modificados sempre são gravados apenas em uma nova parte, já que as partes são imutáveis)

* `uuid` ([UUID](/pt-BR/reference/data-types/index)) — O UUID da parte de dados.

* `part_type` ([String](/pt-BR/reference/data-types/index)) — O formato de armazenamento da parte de dados. Valores possíveis: `Wide` — cada coluna é armazenada em um arquivo separado, `Compact` — todas as colunas são armazenadas em um único arquivo. O formato de armazenamento dos dados é controlado pelas configurações `min_bytes_for_wide_part` e `min_rows_for_wide_part` da tabela MergeTree.

* `active` ([UInt8](/pt-BR/reference/data-types/index)) — Indicador que mostra se a parte de dados está ativa. Se uma parte de dados estiver ativa, ela será usada em uma tabela. Caso contrário, está prestes a ser excluída. Partes de dados inativas aparecem após operações de mesclagem e mutação.

* `marks` ([UInt64](/pt-BR/reference/data-types/index)) — O número de marcas. Para obter o número aproximado de linhas em uma parte de dados, multiplique as marcas pela granularidade do índice (geralmente 8192) (esta dica não funciona para granularidade adaptativa).

* `rows` ([UInt64](/pt-BR/reference/data-types/index)) — O número de linhas.

* `files` ([UInt64](/pt-BR/reference/data-types/index)) — O número de arquivos na parte de dados.

* `bytes_on_disk` ([UInt64](/pt-BR/reference/data-types/index)) — Tamanho total de todos os arquivos da parte de dados em bytes.

* `data_compressed_bytes` ([UInt64](/pt-BR/reference/data-types/index)) — Tamanho total dos dados comprimidos na parte de dados. Todos os arquivos auxiliares (por exemplo, arquivos com marcas) não estão incluídos.

* `data_uncompressed_bytes` ([UInt64](/pt-BR/reference/data-types/index)) — Tamanho total dos dados não comprimidos na parte de dados. Todos os arquivos auxiliares (por exemplo, arquivos com marcas) não estão incluídos.

* `primary_key_size` ([UInt64](/pt-BR/reference/data-types/index)) — A quantidade de memória (em bytes) usada pelos valores da chave primária no arquivo primary.idx/cidx em disco.

* `marks_bytes` ([UInt64](/pt-BR/reference/data-types/index)) — O tamanho do arquivo com marcas.

* `secondary_indices_compressed_bytes` ([UInt64](/pt-BR/reference/data-types/index)) — Tamanho total dos dados comprimidos dos índices secundários na parte de dados. Todos os arquivos auxiliares (por exemplo, arquivos com marcas) não estão incluídos.

* `secondary_indices_uncompressed_bytes` ([UInt64](/pt-BR/reference/data-types/index)) — Tamanho total dos dados não comprimidos dos índices secundários na parte de dados. Todos os arquivos auxiliares (por exemplo, arquivos com marcas) não estão incluídos.

* `secondary_indices_marks_bytes` ([UInt64](/pt-BR/reference/data-types/index)) — O tamanho do arquivo com marcas dos índices secundários.

* `modification_time` ([DateTime](/pt-BR/reference/data-types/index)) — O momento em que o diretório com a parte de dados foi modificado. Isso normalmente corresponde ao momento de criação da parte de dados.

* `remove_time` ([DateTime](/pt-BR/reference/data-types/index)) — O momento em que a parte de dados se tornou inativa.

* `refcount` ([UInt32](/pt-BR/reference/data-types/index)) — O número de locais em que a parte de dados é usada. Um valor maior que 2 indica que a parte de dados é usada em consultas ou mesclagens.

* `min_date` ([Date](/pt-BR/reference/data-types/index)) — O valor mínimo da chave de data na parte de dados.

* `max_date` ([Date](/pt-BR/reference/data-types/index)) — O valor máximo da chave de data na parte de dados.

* `min_time` ([DateTime](/pt-BR/reference/data-types/index)) — O valor mínimo da chave de data e hora na parte de dados.

* `max_time` ([DateTime](/pt-BR/reference/data-types/index)) — O valor máximo da chave de data e hora na parte de dados.

* `partition_id` ([String](/pt-BR/reference/data-types/index)) — ID da partição.

* `min_block_number` ([Int64](/pt-BR/reference/data-types/index)) — O número mínimo de partes de dados que compõem a parte atual após a mesclagem.

* `max_block_number` ([Int64](/pt-BR/reference/data-types/index)) — O número máximo de partes de dados que compõem a parte atual após a mesclagem.

* `level` ([UInt32](/pt-BR/reference/data-types/index)) — Profundidade da árvore de mesclagem. Zero significa que a parte atual foi criada por inserção, e não pela mesclagem de outras partes.

* `data_version` ([UInt64](/pt-BR/reference/data-types/index)) — Número usado para determinar quais mutações devem ser aplicadas à parte de dados (mutações com uma versão maior que data&#95;version).

* `primary_key_bytes_in_memory` ([UInt64](/pt-BR/reference/data-types/index)) — A quantidade de memória (em bytes) usada pelos valores da chave primária. Será 0 quando `primary_key_lazy_load` estiver habilitado e a chave não estiver carregada. Quando diferente de zero, os bytes ficam na própria parte e são contabilizados em `jemalloc.mergetree_arena.active_bytes`. Eles NUNCA são contabilizados em `PrimaryIndexCacheBytes` — essas opções são mutuamente exclusivas por parte: um índice fica ou na parte (esta métrica) ou no `PrimaryIndexCache` compartilhado (a outra), dependendo de `primary_key_lazy_load` e `use_primary_key_cache`.

* `primary_key_bytes_in_memory_allocated` ([UInt64](/pt-BR/reference/data-types/index)) — A quantidade de memória (em bytes) reservada para os valores da chave primária. Será 0 quando `primary_key_lazy_load` estiver habilitado e a chave não estiver carregada. Quando diferente de zero, é incluída em `jemalloc.mergetree_arena.active_bytes`. Veja a observação em `primary_key_bytes_in_memory` sobre a relação com `PrimaryIndexCacheBytes`.

* `index_granularity_bytes_in_memory` ([UInt64](/pt-BR/reference/data-types/index)) — A quantidade de memória (em bytes) usada pelos valores de granularidade do índice (será 0 no caso de primary&#95;key&#95;lazy&#95;load=1 e use&#95;primary&#95;key&#95;cache=1). Quando diferente de zero, os bytes pertencem à parte e são contabilizados em `jemalloc.mergetree_arena.active_bytes`.

* `index_granularity_bytes_in_memory_allocated` ([UInt64](/pt-BR/reference/data-types/index)) — A quantidade de memória (em bytes) reservada para os valores de granularidade do índice (será 0 no caso de primary&#95;key&#95;lazy&#95;load=1 e use&#95;primary&#95;key&#95;cache=1). Quando diferente de zero, é incluída em `jemalloc.mergetree_arena.active_bytes`.

* `is_frozen` ([UInt8](/pt-BR/reference/data-types/index)) — Indicador de que existe um backup dos dados da partição. 1, o backup existe. 0, o backup não existe. Para mais detalhes, consulte FREEZE PARTITION.

* `database` ([String](/pt-BR/reference/data-types/index)) — Nome do banco de dados.

* `table` ([String](/pt-BR/reference/data-types/index)) — Nome da tabela.

* `engine` ([String](/pt-BR/reference/data-types/index)) — Nome do mecanismo da tabela sem parâmetros.

* `disk_name` ([String](/pt-BR/reference/data-types/index)) — Nome do disco que armazena a parte de dados.

* `path` ([String](/pt-BR/reference/data-types/index)) — Caminho absoluto para a pasta com os arquivos da parte de dados.

* `hash_of_all_files` ([String](/pt-BR/reference/data-types/index)) — sipHash128 dos arquivos comprimidos.

* `hash_of_uncompressed_files` ([String](/pt-BR/reference/data-types/index)) — sipHash128 dos arquivos não comprimidos (arquivos com marcas, arquivo de índice etc.).

* `uncompressed_hash_of_compressed_files` ([String](/pt-BR/reference/data-types/index)) — sipHash128 dos dados nos arquivos comprimidos como se não estivessem comprimidos.

* `delete_ttl_info_min` ([DateTime](/pt-BR/reference/data-types/index)) — O valor mínimo da chave de data e hora para a regra TTL DELETE.

* `delete_ttl_info_max` ([DateTime](/pt-BR/reference/data-types/index)) — O valor máximo da chave de data e hora para a regra TTL DELETE.

* `move_ttl_info.expression` ([Array(String)](/pt-BR/reference/data-types/index)) — Array de expressões. Cada expressão define uma regra TTL MOVE.

* `move_ttl_info.min` ([Array(DateTime)](/pt-BR/reference/data-types/index)) — Array de valores de data e hora. Cada elemento descreve o valor mínimo da chave para uma regra TTL MOVE.

* `move_ttl_info.max` ([Array(DateTime)](/pt-BR/reference/data-types/index)) — Array de valores de data e hora. Cada elemento descreve o valor máximo da chave para uma regra TTL MOVE.

* `default_compression_codec` ([String](/pt-BR/reference/data-types/index)) — O nome do codec usado para comprimir esta parte de dados (quando não há codec explícito para as colunas).

* `recompression_ttl_info.expression` ([Array(String)](/pt-BR/reference/data-types/index)) — A expressão TTL.

* `recompression_ttl_info.min` ([Array(DateTime)](/pt-BR/reference/data-types/index)) — O valor mínimo da expressão TTL calculada nesta parte. É usado para determinar se há pelo menos uma linha com TTL expirado.

* `recompression_ttl_info.max` ([Array(DateTime)](/pt-BR/reference/data-types/index)) — O valor máximo da expressão TTL calculada nesta parte. Usado para determinar se todas as linhas têm TTL expirado.

* `group_by_ttl_info.expression` ([Array(String)](/pt-BR/reference/data-types/index)) — A expressão TTL.

* `group_by_ttl_info.min` ([Array(DateTime)](/pt-BR/reference/data-types/index)) — O valor mínimo da expressão TTL calculada nesta parte. Usado para determinar se há pelo menos uma linha com TTL expirado.

* `group_by_ttl_info.max` ([Array(DateTime)](/pt-BR/reference/data-types/index)) — O valor máximo da expressão TTL calculada nesta parte. Usado para determinar se todas as linhas têm TTL expirado.

* `rows_where_ttl_info.expression` ([Array(String)](/pt-BR/reference/data-types/index)) — A expressão TTL.

* `rows_where_ttl_info.min` ([Array(DateTime)](/pt-BR/reference/data-types/index)) — O valor mínimo da expressão TTL calculada nesta parte. Usado para determinar se há pelo menos uma linha com TTL expirado.

* `rows_where_ttl_info.max` ([Array(DateTime)](/pt-BR/reference/data-types/index)) — O valor máximo da expressão TTL calculada nesta parte. Usado para determinar se todas as linhas têm TTL expirado.

* `projections` ([Array(String)](/pt-BR/reference/data-types/index)) — A lista de nomes das projeções calculadas para esta parte.

* `visible` ([UInt8](/pt-BR/reference/data-types/index)) — flag que indica se esta parte está visível para consultas SELECT.

* `creation_tid` ([Tuple(UInt64, UInt64, UUID)](/pt-BR/reference/data-types/index)) — ID da transação que criou/está tentando criar este objeto.

* `removal_tid_lock` ([UInt64](/pt-BR/reference/data-types/index)) — Hash de removal&#95;tid, usado para bloquear um objeto para remoção.

* `removal_tid` ([Tuple(UInt64, UInt64, UUID)](/pt-BR/reference/data-types/index)) — ID da transação que removeu/está tentando remover este objeto

* `creation_csn` ([UInt64](/pt-BR/reference/data-types/index)) — CSN da transação que criou este objeto

* `removal_csn` ([UInt64](/pt-BR/reference/data-types/index)) — CSN da transação que removeu este objeto

* `has_lightweight_delete` ([UInt8](/pt-BR/reference/data-types/index)) — A flag que indica se a parte tem uma máscara de exclusão leve.

* `last_removal_attempt_time` ([DateTime](/pt-BR/reference/data-types/index)) — A última vez em que o servidor tentou excluir esta parte.

* `removal_state` ([String](/pt-BR/reference/data-types/index)) — O estado atual do processo de remoção da parte.
