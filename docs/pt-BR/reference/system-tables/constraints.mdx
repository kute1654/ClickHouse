<div id="description">
  ## Descrição
</div>

Contém informações sobre as restrições existentes em todas as tabelas.

As restrições definidas em [tabelas temporárias](/pt-BR/reference/statements/create/table#temporary-tables) são visíveis em `system.constraints` apenas na sessão em que foram criadas. Elas são exibidas com o campo `database` vazio.

<div id="columns">
  ## Colunas
</div>

{/*AUTOGENERATED_START*/}

* `database` ([String](/pt-BR/reference/data-types/index)) — Nome do banco de dados.
* `table` ([String](/pt-BR/reference/data-types/index)) — Nome da tabela.
* `name` ([String](/pt-BR/reference/data-types/index)) — Nome da restrição.
* `type` ([Enum8(&#39;CHECK&#39; = 0, &#39;ASSUME&#39; = 1)](/pt-BR/reference/data-types/index)) — Tipo da restrição.
* `expression` ([String](/pt-BR/reference/data-types/index)) — Expressão da restrição.

{/*AUTOGENERATED_END*/}

<div id="example">
  ## Exemplo
</div>

```sql
SELECT * FROM system.constraints LIMIT 2 FORMAT Vertical;
```

```text
Row 1:
──────
database:    default
table:       hits
name:        check_hits
type:        CHECK
expression:  CounterID > 0

Row 2:
──────
database:    default
table:       hits
name:        assume_positive
type:        ASSUME
expression:  WatchID > 0
```
