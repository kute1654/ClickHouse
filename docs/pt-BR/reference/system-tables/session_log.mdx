<div id="description">
  ## Descrição
</div>

Contém informações sobre todos os eventos de login e logout, bem-sucedidos e malsucedidos.

<div id="columns">
  ## Colunas
</div>

{/*AUTOGENERATED_START*/}

* `hostname` ([LowCardinality(String)](/pt-BR/reference/data-types/lowcardinality)) — Hostname do servidor que executa a consulta.
* `type` ([Enum8(&#39;LoginFailure&#39; = 0, &#39;LoginSuccess&#39; = 1, &#39;Logout&#39; = 2)](/pt-BR/reference/data-types/enum)) — Resultado de login/logout. Valores possíveis: LoginFailure — Erro de login. LoginSuccess — Login bem-sucedido. Logout — Saída do sistema.
* `auth_id` ([UUID](/pt-BR/reference/data-types/uuid)) — ID de autenticação, que é um UUID gerado automaticamente toda vez que o usuário faz login.
* `session_id` ([String](/pt-BR/reference/data-types/string)) — ID da sessão enviado pelo cliente via interface HTTP.
* `event_date` ([Date](/pt-BR/reference/data-types/date)) — Data de login/logout.
* `event_time` ([DateTime](/pt-BR/reference/data-types/datetime)) — Horário de login/logout.
* `event_time_microseconds` ([DateTime64(6)](/pt-BR/reference/data-types/datetime64)) — Horário de início do login/logout com precisão de microssegundos.
* `user` ([Nullable(String)](/pt-BR/reference/data-types/nullable)) — Nome do usuário.
* `auth_type` ([Nullable(Enum8(&#39;NO&#95;PASSWORD&#39; = 0, &#39;PLAINTEXT&#95;PASSWORD&#39; = 1, &#39;SHA256&#95;PASSWORD&#39; = 2, &#39;DOUBLE&#95;SHA1&#95;PASSWORD&#39; = 3, &#39;LDAP&#39; = 4, &#39;KERBEROS&#39; = 5, &#39;SSL&#95;CERTIFICATE&#39; = 6, &#39;BCRYPT&#95;PASSWORD&#39; = 7, &#39;SSH&#95;KEY&#39; = 8, &#39;HTTP&#39; = 9, &#39;JWT&#39; = 10, &#39;SCRAM&#95;SHA256&#95;PASSWORD&#39; = 11, &#39;NO&#95;AUTHENTICATION&#39; = 12))](/pt-BR/reference/data-types/nullable)) — O tipo de autenticação.
* `profiles` ([Array(LowCardinality(String))](/pt-BR/reference/data-types/array)) — A lista de perfis configurados para todos os roles e/ou usuários.
* `roles` ([Array(LowCardinality(String))](/pt-BR/reference/data-types/array)) — A lista de roles aos quais o perfil é aplicado.
* `settings` ([Array(Tuple(LowCardinality(String), String))](/pt-BR/reference/data-types/array)) — Configurações alteradas quando o cliente fez login/logout.
* `client_address` ([IPv6](/pt-BR/reference/data-types/ipv6)) — O endereço IP usado para fazer login/logout.
* `client_port` ([UInt16](/pt-BR/reference/data-types/int-uint)) — A porta do cliente usada para fazer login/logout.
* `interface` ([Enum8(&#39;TCP&#39; = 1, &#39;HTTP&#39; = 2, &#39;gRPC&#39; = 3, &#39;MySQL&#39; = 4, &#39;PostgreSQL&#39; = 5, &#39;Local&#39; = 6, &#39;TCP&#95;Interserver&#39; = 7, &#39;Prometheus&#39; = 8, &#39;Background&#39; = 9)](/pt-BR/reference/data-types/enum)) — A interface a partir da qual o login foi iniciado.
* `client_hostname` ([String](/pt-BR/reference/data-types/string)) — O hostname da máquina cliente em que o clickhouse-client ou outro cliente TCP é executado.
* `client_name` ([String](/pt-BR/reference/data-types/string)) — O nome do clickhouse-client ou de outro cliente TCP.
* `client_revision` ([UInt32](/pt-BR/reference/data-types/int-uint)) — Revisão do clickhouse-client ou de outro cliente TCP.
* `client_version_major` ([UInt32](/pt-BR/reference/data-types/int-uint)) — A versão principal do clickhouse-client ou de outro cliente TCP.
* `client_version_minor` ([UInt32](/pt-BR/reference/data-types/int-uint)) — A versão secundária do clickhouse-client ou de outro cliente TCP.
* `client_version_patch` ([UInt32](/pt-BR/reference/data-types/int-uint)) — Componente de patch da versão do clickhouse-client ou de outro cliente TCP.
* `failure_reason` ([String](/pt-BR/reference/data-types/string)) — A mensagem de exceção contendo o motivo da falha de login/logout.
* `certificate_subjects` ([Array(LowCardinality(String))](/pt-BR/reference/data-types/array)) — A lista de subjects (Common Name e Subject Alternative Names) do certificado TLS do cliente apresentado na conexão, no formato &#39;CN:...&#39; / &#39;SAN:...&#39;. Vazia se nenhum certificado foi apresentado.
* `certificate_serial` ([LowCardinality(String)](/pt-BR/reference/data-types/lowcardinality)) — Número de série do certificado TLS do cliente. Vazio se nenhum certificado foi apresentado.
* `certificate_issuer` ([LowCardinality(String)](/pt-BR/reference/data-types/lowcardinality)) — Emissor do certificado TLS do cliente. Vazio se nenhum certificado foi apresentado.
* `certificate_not_before` ([Nullable(DateTime64)](/pt-BR/reference/data-types/nullable)) — Horário a partir do qual o certificado TLS do cliente é válido. NULL se nenhum certificado foi apresentado.
* `certificate_not_after` ([Nullable(DateTime64)](/pt-BR/reference/data-types/nullable)) — Horário após o qual o certificado TLS do cliente expira. NULL se nenhum certificado foi apresentado.

{/*AUTOGENERATED_END*/}
