<div id="description">
  ## Descrição
</div>

Esta tabela de sistema é usada para implementar a consulta `SHOW PROCESSLIST`.

<div id="columns">
  ## Colunas
</div>

{/*AUTOGENERATED_START*/}

* `is_initial_query` ([UInt8](/pt-BR/reference/data-types/index)) — Se esta consulta vem diretamente do usuário ou foi emitida pelo servidor ClickHouse no contexto da execução distribuída de consultas.
* `user` ([String](/pt-BR/reference/data-types/index)) — O usuário que fez a consulta. Tenha em mente que, para processamento distribuído, as consultas são enviadas para servidores remotos com o usuário `default`. O campo contém o username de uma consulta específica, não de uma consulta iniciada por esta consulta.
* `query_id` ([String](/pt-BR/reference/data-types/index)) — ID da consulta, se definido.
* `address` ([IPv6](/pt-BR/reference/data-types/index)) — O endereço IP a partir do qual a consulta foi feita. O mesmo vale para processamento distribuído. Para rastrear de onde uma consulta distribuída foi originalmente feita, consulte system.processes no servidor solicitante da consulta.
* `port` ([UInt16](/pt-BR/reference/data-types/index)) — A porta do cliente a partir da qual a consulta foi feita.
* `initial_user` ([String](/pt-BR/reference/data-types/index)) — Nome do usuário que executou a consulta inicial (para execução distribuída de consultas).
* `initial_query_id` ([String](/pt-BR/reference/data-types/index)) — ID da consulta inicial (para execução distribuída de consultas).
* `initial_address` ([IPv6](/pt-BR/reference/data-types/index)) — Endereço IP a partir do qual a consulta pai foi iniciada.
* `initial_port` ([UInt16](/pt-BR/reference/data-types/index)) — A porta do cliente usada para fazer a consulta pai.
* `interface` ([UInt8](/pt-BR/reference/data-types/index)) — A interface usada para enviar a consulta. TCP = 1, HTTP = 2, GRPC = 3, MYSQL = 4, POSTGRESQL = 5, LOCAL = 6, TCP&#95;INTERSERVER = 7.
* `os_user` ([String](/pt-BR/reference/data-types/index)) — Username do sistema operacional que executa o clickhouse-client.
* `client_hostname` ([String](/pt-BR/reference/data-types/index)) — Hostname da máquina cliente em que o clickhouse-client ou outro cliente TCP é executado.
* `client_name` ([String](/pt-BR/reference/data-types/index)) — O nome do clickhouse-client ou de outro cliente TCP.
* `client_agent` ([LowCardinality(String)](/pt-BR/reference/data-types/index)) — O agente de codificação por IA que invocou o cliente (por exemplo, `claude-code`, `cursor`), detectado a partir de variáveis de ambiente. Vazio se nenhum agente for detectado.
* `client_revision` ([UInt64](/pt-BR/reference/data-types/index)) — Revisão do clickhouse-client ou de outro cliente TCP.
* `client_version_major` ([UInt64](/pt-BR/reference/data-types/index)) — Versão principal do clickhouse-client ou de outro cliente TCP.
* `client_version_minor` ([UInt64](/pt-BR/reference/data-types/index)) — Versão secundária do clickhouse-client ou de outro cliente TCP.
* `client_version_patch` ([UInt64](/pt-BR/reference/data-types/index)) — Componente de patch da versão do clickhouse-client ou de outro cliente TCP.
* `http_method` ([UInt8](/pt-BR/reference/data-types/index)) — Método HTTP que iniciou a consulta. Valores possíveis: 0 — A consulta foi iniciada pela interface TCP. 1 — O método GET foi usado. 2 — O método POST foi usado.
* `http_user_agent` ([String](/pt-BR/reference/data-types/index)) — Cabeçalho HTTP UserAgent enviado na consulta HTTP.
* `http_referer` ([String](/pt-BR/reference/data-types/index)) — Cabeçalho HTTP Referer enviado na consulta HTTP (contém um endereço absoluto ou parcial da página que faz a consulta).
* `forwarded_for` ([String](/pt-BR/reference/data-types/index)) — Cabeçalho HTTP X-Forwarded-For enviado na consulta HTTP.
* `quota_key` ([String](/pt-BR/reference/data-types/index)) — A quota key especificada na configuração de quotas (consulte keyed).
* `distributed_depth` ([UInt64](/pt-BR/reference/data-types/index)) — O número de vezes que a consulta foi retransmitida internamente entre nós do servidor.
* `elapsed` ([Float64](/pt-BR/reference/data-types/index)) — O tempo, em segundos, desde o início da execução da solicitação.
* `is_cancelled` ([UInt8](/pt-BR/reference/data-types/index)) — Indica se a consulta foi cancelada.
* `is_all_data_sent` ([UInt8](/pt-BR/reference/data-types/index)) — Indica se todos os dados foram enviados ao client (ou seja, se a consulta foi concluída no servidor).
* `read_rows` ([UInt64](/pt-BR/reference/data-types/index)) — O número de linhas lidas da tabela. Para processamento distribuído, no servidor solicitante, este é o total de todos os servidores remotos.
* `read_bytes` ([UInt64](/pt-BR/reference/data-types/index)) — O número de bytes não comprimidos lidos da tabela. Para processamento distribuído, no servidor solicitante, este é o total de todos os servidores remotos.
* `total_rows_approx` ([UInt64](/pt-BR/reference/data-types/index)) — A estimativa do número total de linhas que devem ser lidas. Para processamento distribuído, no servidor solicitante, este é o total de todos os servidores remotos. Esse valor pode ser atualizado durante o processamento da solicitação, quando novas fontes a serem processadas se tornam conhecidas.
* `written_rows` ([UInt64](/pt-BR/reference/data-types/index)) — A quantidade de linhas gravadas no armazenamento.
* `written_bytes` ([UInt64](/pt-BR/reference/data-types/index)) — A quantidade de bytes gravados no armazenamento.
* `memory_usage` ([Int64](/pt-BR/reference/data-types/index)) — Quantidade de RAM usada pela consulta. Pode não incluir alguns tipos de memória dedicada.
* `peak_memory_usage` ([Int64](/pt-BR/reference/data-types/index)) — O pico atual de uso de memória.
* `query` ([String](/pt-BR/reference/data-types/index)) — O texto da consulta. Para INSERT, não inclui os dados a serem inseridos.
* `normalized_query_hash` ([UInt64](/pt-BR/reference/data-types/index)) — Um valor de hash numérico, idêntico para consultas que diferem apenas nos valores dos literais.
* `query_kind` ([String](/pt-BR/reference/data-types/index)) — O tipo da consulta: SELECT, INSERT etc.
* `thread_ids` ([Array(UInt64)](/pt-BR/reference/data-types/index)) — A lista de identificadores de todas as threads que participaram desta consulta.
* `peak_threads_usage` ([UInt64](/pt-BR/reference/data-types/index)) — Número máximo de threads simultâneas executando a consulta.
* `ProfileEvents` ([Map(LowCardinality(String), UInt64)](/pt-BR/reference/data-types/index)) — ProfileEvents calculados para esta consulta.
* `Settings` ([Map(LowCardinality(String), LowCardinality(String))](/pt-BR/reference/data-types/index)) — A lista de user-level settings modificadas.
* `current_database` ([String](/pt-BR/reference/data-types/index)) — O nome do banco de dados atual.
* `is_internal` ([UInt8](/pt-BR/reference/data-types/index)) — Indica se é uma consulta auxiliar executada internamente.

{/*AUTOGENERATED_END*/}
