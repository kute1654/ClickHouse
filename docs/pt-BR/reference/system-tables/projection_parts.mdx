<div id="description">
  ## Descrição
</div>

Esta tabela contém informações sobre as partes de projeção de tabelas da família MergeTree.

<div id="columns">
  ## Colunas
</div>

{/*AUTOGENERATED_START*/}

* `partition` ([String](/pt-BR/reference/data-types/index)) — O nome da partição.
* `name` ([String](/pt-BR/reference/data-types/index)) — Nome da parte de dados.
* `part_type` ([String](/pt-BR/reference/data-types/index)) — O formato de armazenamento da parte de dados. Valores possíveis: Wide (um arquivo por coluna) e Compact (um único arquivo para todas as colunas).
* `parent_name` ([String](/pt-BR/reference/data-types/index)) — O nome da parte de dados de origem (pai).
* `parent_uuid` ([UUID](/pt-BR/reference/data-types/index)) — O UUID da parte de dados de origem (pai).
* `parent_part_type` ([String](/pt-BR/reference/data-types/index)) — O formato de armazenamento da parte de dados de origem (pai).
* `active` ([UInt8](/pt-BR/reference/data-types/index)) — Indicador que mostra se a parte de dados está ativa. Se uma parte de dados estiver ativa, ela é usada em uma tabela. Caso contrário, está prestes a ser excluída. Partes de dados inativas aparecem após operações de merge e mutação.
* `marks` ([UInt64](/pt-BR/reference/data-types/index)) — O número de marcas. Para obter o número aproximado de linhas em uma parte de dados, multiplique as marcas pela granularidade do índice (geralmente 8192) (essa dica não funciona para granularidade adaptativa).
* `rows` ([UInt64](/pt-BR/reference/data-types/index)) — O número de linhas.
* `bytes_on_disk` ([UInt64](/pt-BR/reference/data-types/index)) — Tamanho total, em bytes, de todos os arquivos da parte de dados.
* `data_compressed_bytes` ([UInt64](/pt-BR/reference/data-types/index)) — Tamanho total dos dados comprimidos na parte de dados. Todos os arquivos auxiliares (por exemplo, arquivos com marcas) não são incluídos.
* `data_uncompressed_bytes` ([UInt64](/pt-BR/reference/data-types/index)) — Tamanho total dos dados não comprimidos na parte de dados. Todos os arquivos auxiliares (por exemplo, arquivos com marcas) não são incluídos.
* `marks_bytes` ([UInt64](/pt-BR/reference/data-types/index)) — O tamanho do arquivo com marcas.
* `parent_marks` ([UInt64](/pt-BR/reference/data-types/index)) — O número de marcas na parte de origem (pai).
* `parent_rows` ([UInt64](/pt-BR/reference/data-types/index)) — O número de linhas na parte de origem (pai).
* `parent_bytes_on_disk` ([UInt64](/pt-BR/reference/data-types/index)) — Tamanho total, em bytes, de todos os arquivos da parte de dados de origem (pai).
* `parent_data_compressed_bytes` ([UInt64](/pt-BR/reference/data-types/index)) — Tamanho total dos dados comprimidos na parte de dados de origem (pai).
* `parent_data_uncompressed_bytes` ([UInt64](/pt-BR/reference/data-types/index)) — Tamanho total dos dados não comprimidos na parte de dados de origem (pai).
* `parent_marks_bytes` ([UInt64](/pt-BR/reference/data-types/index)) — O tamanho do arquivo com marcas na parte de dados de origem (pai).
* `modification_time` ([DateTime](/pt-BR/reference/data-types/index)) — O momento em que o diretório com a parte de dados foi modificado. Isso normalmente corresponde ao momento de criação da parte de dados.
* `remove_time` ([DateTime](/pt-BR/reference/data-types/index)) — O momento em que a parte de dados se tornou inativa.
* `refcount` ([UInt32](/pt-BR/reference/data-types/index)) — O número de locais em que a parte de dados é usada. Um valor maior que 2 indica que a parte de dados é usada em consultas ou merges.
* `min_date` ([Date](/pt-BR/reference/data-types/index)) — O valor mínimo da chave de data na parte de dados.
* `max_date` ([Date](/pt-BR/reference/data-types/index)) — O valor máximo da chave de data na parte de dados.
* `min_time` ([DateTime](/pt-BR/reference/data-types/index)) — O valor mínimo da chave de data e hora na parte de dados.
* `max_time` ([DateTime](/pt-BR/reference/data-types/index)) — O valor máximo da chave de data e hora na parte de dados.
* `partition_id` ([String](/pt-BR/reference/data-types/index)) — ID da partição.
* `min_block_number` ([Int64](/pt-BR/reference/data-types/index)) — O número mínimo de partes de dados que compõem a parte atual após a mesclagem.
* `max_block_number` ([Int64](/pt-BR/reference/data-types/index)) — O número máximo de partes de dados que compõem a parte atual após a mesclagem.
* `level` ([UInt32](/pt-BR/reference/data-types/index)) — Profundidade da árvore de mesclagem. Zero significa que a parte atual foi criada por inserção, e não pela mesclagem de outras partes.
* `data_version` ([UInt64](/pt-BR/reference/data-types/index)) — Número usado para determinar quais mutações devem ser aplicadas à parte de dados (mutações com versão superior a data&#95;version).
* `primary_key_bytes_in_memory` ([UInt64](/pt-BR/reference/data-types/index)) — A quantidade de memória (em bytes) usada pelos valores da chave primária.
* `primary_key_bytes_in_memory_allocated` ([UInt64](/pt-BR/reference/data-types/index)) — A quantidade de memória (em bytes) reservada para os valores da chave primária.
* `is_frozen` ([UInt8](/pt-BR/reference/data-types/index)) — Indicador de que existe um backup dos dados da partição. 1: o backup existe. 0: o backup não existe.
* `database` ([String](/pt-BR/reference/data-types/index)) — Nome do banco de dados.
* `table` ([String](/pt-BR/reference/data-types/index)) — Nome da tabela.
* `engine` ([String](/pt-BR/reference/data-types/index)) — Nome do engine da tabela sem parâmetros.
* `disk_name` ([String](/pt-BR/reference/data-types/index)) — Nome do disco que armazena a parte de dados.
* `path` ([String](/pt-BR/reference/data-types/index)) — Caminho absoluto para a pasta com os arquivos da parte de dados.
* `hash_of_all_files` ([String](/pt-BR/reference/data-types/index)) — sipHash128 dos arquivos comprimidos.
* `hash_of_uncompressed_files` ([String](/pt-BR/reference/data-types/index)) — sipHash128 dos arquivos não comprimidos (arquivos com marcas, arquivo de índice etc.).
* `uncompressed_hash_of_compressed_files` ([String](/pt-BR/reference/data-types/index)) — sipHash128 dos dados nos arquivos comprimidos, como se estivessem descomprimidos.
* `delete_ttl_info_min` ([DateTime](/pt-BR/reference/data-types/index)) — O valor mínimo da chave de data e hora para a regra TTL DELETE.
* `delete_ttl_info_max` ([DateTime](/pt-BR/reference/data-types/index)) — O valor máximo da chave de data e hora para a regra TTL DELETE.
* `move_ttl_info.expression` ([Array(String)](/pt-BR/reference/data-types/index)) — Array de expressões. Cada expressão define uma regra TTL MOVE.
* `move_ttl_info.min` ([Array(DateTime)](/pt-BR/reference/data-types/index)) — Array de valores de data e hora. Cada elemento descreve o valor mínimo da chave para uma regra TTL MOVE.
* `move_ttl_info.max` ([Array(DateTime)](/pt-BR/reference/data-types/index)) — Array de valores de data e hora. Cada elemento descreve o valor máximo da chave para uma regra TTL MOVE.
* `default_compression_codec` ([String](/pt-BR/reference/data-types/index)) — O nome do codec usado para comprimir esta parte de dados (caso não haja codec explícito para colunas).
* `recompression_ttl_info.expression` ([Array(String)](/pt-BR/reference/data-types/index)) — A expressão TTL.
* `recompression_ttl_info.min` ([Array(DateTime)](/pt-BR/reference/data-types/index)) — O valor mínimo da expressão TTL calculada nesta parte. Usado para determinar se há pelo menos uma linha com TTL expirado.
* `recompression_ttl_info.max` ([Array(DateTime)](/pt-BR/reference/data-types/index)) — O valor máximo da expressão TTL calculada nesta parte. Usado para entender se temos todas as linhas com TTL expirado.
* `group_by_ttl_info.expression` ([Array(String)](/pt-BR/reference/data-types/index)) — A expressão TTL.
* `group_by_ttl_info.min` ([Array(DateTime)](/pt-BR/reference/data-types/index)) — O valor mínimo da expressão TTL calculada nesta parte. Usado para entender se temos pelo menos uma linha com TTL expirado.
* `group_by_ttl_info.max` ([Array(DateTime)](/pt-BR/reference/data-types/index)) — O valor máximo da expressão TTL calculada nesta parte. Usado para entender se temos todas as linhas com TTL expirado.
* `rows_where_ttl_info.expression` ([Array(String)](/pt-BR/reference/data-types/index)) — A expressão TTL.
* `rows_where_ttl_info.min` ([Array(DateTime)](/pt-BR/reference/data-types/index)) — O valor mínimo da expressão TTL calculada nesta parte. Usado para entender se temos pelo menos uma linha com TTL expirado.
* `rows_where_ttl_info.max` ([Array(DateTime)](/pt-BR/reference/data-types/index)) — O valor máximo da expressão TTL calculada nesta parte. Usado para entender se temos todas as linhas com TTL expirado.
* `is_broken` ([UInt8](/pt-BR/reference/data-types/index)) — Indica se a parte da projeção está com falha
* `exception_code` ([Int32](/pt-BR/reference/data-types/index)) — Mensagem de exceção que explica o estado com falha da parte da projeção
* `exception` ([String](/pt-BR/reference/data-types/index)) — Código de exceção que explica o estado com falha da parte da projeção
