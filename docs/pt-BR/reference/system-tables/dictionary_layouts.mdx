<div id="description">
  ## Descrição
</div>

Contém a lista de layouts de dicionário compatíveis com o servidor, bem como a documentação embutida de cada layout. Um layout de dicionário determina como um dicionário é armazenado na memória (ou em disco) e como a busca nele é realizada; ele é especificado na cláusula `LAYOUT` de uma consulta `CREATE DICTIONARY`.

<div id="columns">
  ## Colunas
</div>

{/*AUTOGENERATED_START*/}

* `name` ([String](/pt-BR/reference/data-types/index)) — O nome do layout de dicionário, conforme especificado na cláusula LAYOUT.
* `is_complex` ([UInt8](/pt-BR/reference/data-types/index)) — Indica se o layout exige uma chave complexa (uma chave composta por vários atributos ou de um tipo não inteiro).
* `description` ([String](/pt-BR/reference/data-types/index)) — Uma descrição geral do que o layout de dicionário faz.
* `syntax` ([String](/pt-BR/reference/data-types/index)) — Como o layout é especificado na cláusula LAYOUT de uma consulta CREATE DICTIONARY.
* `examples` ([String](/pt-BR/reference/data-types/index)) — Exemplos de uso.
* `introduced_in` ([String](/pt-BR/reference/data-types/index)) — A versão do ClickHouse em que o layout foi introduzido pela primeira vez, no formato major.minor.
* `related` ([Array(String)](/pt-BR/reference/data-types/index)) — Os nomes de layouts de dicionário relacionados.

{/*AUTOGENERATED_END*/}

<div id="example">
  ## Exemplo
</div>

```sql title="Query"
SELECT name, is_complex, syntax
FROM system.dictionary_layouts
WHERE name IN ('flat', 'hashed', 'complex_key_hashed')
ORDER BY name
```

```text title="Response"
┌─name───────────────┬─is_complex─┬─syntax───────────────────────────────────────────────────┐
│ complex_key_hashed │          1 │ LAYOUT(COMPLEX_KEY_HASHED())                             │
│ flat               │          0 │ LAYOUT(FLAT([INITIAL_ARRAY_SIZE n] [MAX_ARRAY_SIZE n]))  │
│ hashed             │          0 │ LAYOUT(HASHED())                                         │
└────────────────────┴────────────┴──────────────────────────────────────────────────────────┘
```
