<div id="purpose">
  ## Descrição
</div>

Contém informações sobre partes desanexadas de tabelas [MergeTree](/pt-BR/reference/engines/table-engines/mergetree-family/mergetree). A coluna `reason` especifica por que a parte foi desanexada.

Para partes desanexadas pelo usuário, o motivo fica em branco. Essas partes podem ser anexadas com o comando [ALTER TABLE ATTACH PARTITION|PART](/pt-BR/reference/statements/alter/partition#attach-partitionpart).

Para a descrição das outras colunas, consulte [system.parts](/pt-BR/reference/system-tables/parts).

Se o nome da parte for inválido, os valores de algumas colunas podem ser `NULL`. Essas partes podem ser excluídas com [ALTER TABLE DROP DETACHED PART](/pt-BR/reference/statements/alter/partition#drop-detached-partitionpart).

<div id="columns">
  ## Colunas
</div>

{/*AUTOGENERATED_START*/}

* `database` ([String](/pt-BR/reference/data-types/index)) — O nome do banco de dados ao qual esta parte pertence.
* `table` ([String](/pt-BR/reference/data-types/index)) — O nome da tabela à qual esta parte pertence.
* `partition_id` ([Nullable(String)](/pt-BR/reference/data-types/index)) — O identificador da partição à qual esta parte pertence.
* `name` ([String](/pt-BR/reference/data-types/index)) — O nome da parte.
* `bytes_on_disk` ([UInt64](/pt-BR/reference/data-types/index)) — O tamanho total, em bytes, de todos os arquivos da parte de dados.
* `modification_time` ([DateTime](/pt-BR/reference/data-types/index)) — A data e hora em que o diretório com a parte de dados foi modificado. Isso geralmente corresponde ao momento em que a desanexação ocorreu.
* `disk` ([String](/pt-BR/reference/data-types/index)) — O nome do disco que armazena esta parte de dados.
* `path` ([String](/pt-BR/reference/data-types/index)) — O caminho no disco até o arquivo desta parte de dados.
* `reason` ([Nullable(String)](/pt-BR/reference/data-types/index)) — A explicação do motivo pelo qual esta parte foi desanexada.
* `min_block_number` ([Nullable(Int64)](/pt-BR/reference/data-types/index)) — O menor número das partes de dados que compõem a parte atual após a mesclagem.
* `max_block_number` ([Nullable(Int64)](/pt-BR/reference/data-types/index)) — O maior número das partes de dados que compõem a parte atual após a mesclagem.
* `level` ([Nullable(UInt32)](/pt-BR/reference/data-types/index)) — Profundidade da árvore de mesclagem. Zero significa que a parte atual foi criada por insert, e não pela mesclagem de outras partes.

{/*AUTOGENERATED_END*/}
