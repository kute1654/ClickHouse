<div id="description">
  ## Descrição
</div>

Contém a lista dos índices de omissão de dados suportados pelo servidor, juntamente com a documentação incorporada de cada tipo. Um índice de omissão de dados é especificado no `TYPE` de uma declaração `INDEX` em uma consulta `CREATE TABLE` e permite ao ClickHouse ignorar grânulos que não podem corresponder à condição de uma consulta.

Observe que esta tabela lista os *tipos* de índice disponíveis, enquanto [`system.data_skipping_indices`](/pt-BR/reference/system-tables/data_skipping_indices) lista as instâncias de índice definidas em tabelas existentes.

<div id="columns">
  ## Colunas
</div>

{/*AUTOGENERATED_START*/}

* `name` ([String](/pt-BR/reference/data-types/index)) — O nome do tipo de índice de data skipping, conforme especificado no TYPE de uma declaração INDEX.
* `description` ([String](/pt-BR/reference/data-types/index)) — Uma descrição geral do que o tipo de índice de data skipping faz.
* `syntax` ([String](/pt-BR/reference/data-types/index)) — Como o índice é declarado em uma cláusula INDEX de uma consulta CREATE TABLE.
* `examples` ([String](/pt-BR/reference/data-types/index)) — Exemplos de uso.
* `introduced_in` ([String](/pt-BR/reference/data-types/index)) — A versão do ClickHouse na qual o tipo de índice foi introduzido pela primeira vez, no formato major.minor.
* `related` ([Array(String)](/pt-BR/reference/data-types/index)) — Os nomes de tipos relacionados de índice de data skipping.

{/*AUTOGENERATED_END*/}

<div id="example">
  ## Exemplo
</div>

```sql title="Query"
SELECT name, syntax
FROM system.data_skipping_index_types
WHERE name IN ('minmax', 'set')
ORDER BY name
```

```text title="Response"
┌─name───┬─syntax───────────────────────────────────────┐
│ minmax │ INDEX name expr TYPE minmax GRANULARITY n     │
│ set    │ INDEX name expr TYPE set(max_rows) GRANULARITY n │
└────────┴──────────────────────────────────────────────┘
```
