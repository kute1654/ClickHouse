<div id="description">
  ## Descrição
</div>

Contém entradas de log com informações sobre as operações `BACKUP` e `RESTORE`.

<div id="columns">
  ## Colunas
</div>

{/*AUTOGENERATED_START*/}

* `hostname` ([LowCardinality(String)](/pt-BR/reference/data-types/lowcardinality)) — Hostname do servidor que executa a consulta.
* `event_date` ([Date](/pt-BR/reference/data-types/date)) — Data do registro.
* `event_time` ([DateTime](/pt-BR/reference/data-types/datetime)) — Hora do registro.
* `event_time_microseconds` ([DateTime64(6)](/pt-BR/reference/data-types/datetime64)) — Hora do registro com precisão de microssegundos.
* `id` ([String](/pt-BR/reference/data-types/string)) — Identificador da operação de backup ou restauração.
* `name` ([String](/pt-BR/reference/data-types/string)) — Nome do armazenamento de backup (o conteúdo da cláusula FROM ou TO).
* `base_backup_name` ([String](/pt-BR/reference/data-types/string)) — Nome do backup base no caso de um backup incremental.
* `query_id` ([String](/pt-BR/reference/data-types/string)) — ID de uma consulta associada a uma operação de backup.
* `status` ([Enum8(&#39;CREATING&#95;BACKUP&#39; = 0, &#39;BACKUP&#95;CREATED&#39; = 1, &#39;BACKUP&#95;FAILED&#39; = 2, &#39;RESTORING&#39; = 3, &#39;RESTORED&#39; = 4, &#39;RESTORE&#95;FAILED&#39; = 5, &#39;BACKUP&#95;CANCELLED&#39; = 6, &#39;RESTORE&#95;CANCELLED&#39; = 7)](/pt-BR/reference/data-types/enum)) — Status da operação.
* `error` ([String](/pt-BR/reference/data-types/string)) — Mensagem de erro da operação com falha (string vazia para operações bem-sucedidas).
* `start_time` ([DateTime64(6)](/pt-BR/reference/data-types/datetime64)) — Hora de início da operação.
* `end_time` ([DateTime64(6)](/pt-BR/reference/data-types/datetime64)) — Hora de término da operação.
* `num_files` ([UInt64](/pt-BR/reference/data-types/int-uint)) — Número de arquivos armazenados no backup.
* `total_size` ([UInt64](/pt-BR/reference/data-types/int-uint)) — Tamanho total dos arquivos armazenados no backup.
* `num_entries` ([UInt64](/pt-BR/reference/data-types/int-uint)) — Número de entradas no backup, isto é, o número de arquivos dentro da pasta, se o backup for armazenado como pasta, ou o número de arquivos dentro do arquivo, se o backup for armazenado como arquivo. Não é o mesmo que `num_files` se for um backup incremental ou se contiver arquivos vazios ou duplicados. O seguinte é sempre verdadeiro: `num_entries ≤ num_files`.
* `uncompressed_size` ([UInt64](/pt-BR/reference/data-types/int-uint)) — Tamanho não compactado do backup.
* `compressed_size` ([UInt64](/pt-BR/reference/data-types/int-uint)) — Tamanho compactado do backup. Se o backup não for armazenado como arquivo, ele será igual a uncompressed&#95;size.
* `files_read` ([UInt64](/pt-BR/reference/data-types/int-uint)) — Número de arquivos lidos durante a operação de restauração.
* `bytes_read` ([UInt64](/pt-BR/reference/data-types/int-uint)) — Tamanho total dos arquivos lidos durante a operação de restauração.

{/*AUTOGENERATED_END*/}
