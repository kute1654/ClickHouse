<div id="description">
  ## Descrição
</div>

Contém informações sobre todos os esquemas de arquivos em cache.

<div id="columns">
  ## Colunas
</div>

{/*AUTOGENERATED_START*/}

* `storage` ([String](/pt-BR/reference/data-types/index)) — Nome do armazenamento: File, URL, S3 ou HDFS.
* `source` ([String](/pt-BR/reference/data-types/index)) — Origem do arquivo.
* `format` ([String](/pt-BR/reference/data-types/index)) — Nome do formato.
* `additional_format_info` ([String](/pt-BR/reference/data-types/index)) — Informações adicionais necessárias para identificar o esquema. Por exemplo, configurações específicas do formato.
* `registration_time` ([DateTime](/pt-BR/reference/data-types/index)) — Timestamp em que o esquema foi adicionado ao cache.
* `schema` ([Nullable(String)](/pt-BR/reference/data-types/index)) — Esquema em cache.
* `number_of_rows` ([Nullable(UInt64)](/pt-BR/reference/data-types/index)) — Número de linhas no arquivo no formato especificado. É usado para manter em cache o `count()` trivial de arquivos de dados e o número de linhas dos metadados durante a inferência de esquema.
* `schema_inference_mode` ([Nullable(String)](/pt-BR/reference/data-types/index)) — Modo de inferência de esquema.

{/*AUTOGENERATED_END*/}

<div id="example">
  ## Exemplo
</div>

Digamos que temos um arquivo `data.jsonl` com o seguinte conteúdo:

```json
{"id" :  1, "age" :  25, "name" :  "Josh", "hobbies" :  ["football", "cooking", "music"]}
{"id" :  2, "age" :  19, "name" :  "Alan", "hobbies" :  ["tennis", "art"]}
{"id" :  3, "age" :  32, "name" :  "Lana", "hobbies" :  ["fitness", "reading", "shopping"]}
{"id" :  4, "age" :  47, "name" :  "Brayan", "hobbies" :  ["movies", "skydiving"]}
```

<Tip>
  Coloque `data.jsonl` no diretório `user_files_path`. Você pode localizar esse diretório
  nos arquivos de configuração do ClickHouse. O padrão é:

  ```sql
  <user_files_path>/var/lib/clickhouse/user_files/</user_files_path>
  ```
</Tip>

Abra o `clickhouse-client` e execute a consulta `DESCRIBE`:

```sql
DESCRIBE file('data.jsonl') SETTINGS input_format_try_infer_integers=0;
```

```response
┌─name────┬─type────────────────────┬─default_type─┬─default_expression─┬─comment─┬─codec_expression─┬─ttl_expression─┐
│ id      │ Nullable(Float64)       │              │                    │         │                  │                │
│ age     │ Nullable(Float64)       │              │                    │         │                  │                │
│ name    │ Nullable(String)        │              │                    │         │                  │                │
│ hobbies │ Array(Nullable(String)) │              │                    │         │                  │                │
└─────────┴─────────────────────────┴──────────────┴────────────────────┴─────────┴──────────────────┴────────────────┘
```

Vamos ver o conteúdo da tabela `system.schema_inference_cache`:

```sql
SELECT *
FROM system.schema_inference_cache
FORMAT Vertical
```

```response
Row 1:
──────
storage:                File
source:                 /home/droscigno/user_files/data.jsonl
format:                 JSONEachRow
additional_format_info: schema_inference_hints=, max_rows_to_read_for_schema_inference=25000, schema_inference_make_columns_nullable=true, try_infer_integers=false, try_infer_dates=true, try_infer_datetimes=true, try_infer_numbers_from_strings=true, read_bools_as_numbers=true, try_infer_objects=false
registration_time:      2022-12-29 17:49:52
schema:                 id Nullable(Float64), age Nullable(Float64), name Nullable(String), hobbies Array(Nullable(String))
```
