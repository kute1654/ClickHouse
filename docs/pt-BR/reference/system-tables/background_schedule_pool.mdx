<div id="description">
  ## Descrição
</div>

Contém informações sobre tarefas em pools de agendamento em segundo plano. Esses pools de agendamento em segundo plano são usados para executar tarefas periódicas, como envios distribuídos, flushes de buffer e operações do broker de mensagens.

<div id="columns">
  ## Colunas
</div>

{/*AUTOGENERATED_START*/}

* `pool` ([String](/pt-BR/reference/data-types/index)) — Nome do pool. Valores possíveis: `schedule` — pool de agendamento de uso geral, `buffer_flush` — pool para descarregar dados da tabela Buffer, `distributed` — pool para operações de tabela distribuída, `message_broker` — pool para operações do broker de mensagens.
* `database` ([String](/pt-BR/reference/data-types/index)) — Nome do banco de dados.
* `table` ([String](/pt-BR/reference/data-types/index)) — Nome da tabela.
* `table_uuid` ([UUID](/pt-BR/reference/data-types/index)) — UUID da tabela.
* `query_id` ([String](/pt-BR/reference/data-types/index)) — ID da consulta (se estiver em execução no momento). Observação: esta não é uma consulta real, mas um ID gerado aleatoriamente para corresponder logs em `system.text_log`.
* `elapsed_ms` ([UInt64](/pt-BR/reference/data-types/index)) — Tempo de execução da tarefa (se estiver em execução no momento).
* `log_name` ([String](/pt-BR/reference/data-types/index)) — Nome do log da tarefa.
* `deactivated` ([UInt8](/pt-BR/reference/data-types/index)) — Indica se a tarefa está desativada (sempre falso, já que tarefas desativadas são removidas do pool).
* `scheduled` ([UInt8](/pt-BR/reference/data-types/index)) — Indica se a tarefa está agendada para execução.
* `delayed` ([UInt8](/pt-BR/reference/data-types/index)) — Indica se a tarefa está agendada com atraso.
* `executing` ([UInt8](/pt-BR/reference/data-types/index)) — Indica se a tarefa está sendo executada no momento.

{/*AUTOGENERATED_END*/}
