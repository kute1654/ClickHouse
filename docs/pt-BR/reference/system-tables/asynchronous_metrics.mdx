<div id="description">
  ## Descrição
</div>

Contém métricas calculadas periodicamente em segundo plano. Por exemplo, a quantidade de RAM em uso.

<div id="columns">
  ## Colunas
</div>

{/*AUTOGENERATED_START*/}

* `metric` ([String](/pt-BR/reference/data-types/string)) — Nome da métrica.
* `value` ([Float64](/pt-BR/reference/data-types/float)) — Valor da métrica.
* `description` ([String](/pt-BR/reference/data-types/string) - Descrição da métrica)

{/*AUTOGENERATED_END*/}

<div id="metric-descriptions">
  ## Descrições das métricas
</div>

As descrições abaixo são geradas a partir do código-fonte em C++ por `utils/generate-async-metrics-docs`. A fonte única de verdade é o literal de string ao lado de cada registro de métrica em `src/Common/AsynchronousMetrics.cpp`, `src/Interpreters/ServerAsynchronousMetrics.cpp` e `src/Coordination/KeeperAsynchronousMetrics.cpp`. Os nomes de métricas que incluem um sufixo variável (por disco, por CPU, por interface, ...) são exibidos com um placeholder `*name*`; o servidor em execução os informa com o sufixo concreto substituído.

<div id="blockactivetime_name">
  ### BlockActiveTime_*name*
</div>

Tempo, em segundos, durante o qual o dispositivo de bloco teve solicitações de E/S na fila. Esta é uma métrica de todo o sistema; inclui todos os processos na máquina host, não apenas o clickhouse-server. Origem: `/sys/block`. Consulte https://www.kernel.org/doc/Documentation/block/stat.txt

<div id="blockactivetimeperop_name">
  ### BlockActiveTimePerOp_*name*
</div>

Semelhante às métricas `BlockActiveTime`, mas o valor é dividido pelo número de operações de E/S para medir o tempo por operação.

<div id="blockdiscardbytes_name">
  ### BlockDiscardBytes_*name*
</div>

Número de bytes descartados no dispositivo de bloco. Essas operações são relevantes para SSDs. Esta é uma métrica de todo o sistema; ela inclui todos os processos na máquina host, não apenas o clickhouse-server. Origem: `/sys/block`. Veja https://www.kernel.org/doc/Documentation/block/stat.txt

Esta é uma métrica de todo o sistema; inclui todos os processos na máquina host, não apenas o clickhouse-server. Fonte: `/sys/block`. Consulte https://www.kernel.org/doc/Documentation/block/stat.txt

Esta é uma métrica de todo o sistema; ela inclui todos os processos na máquina host, não apenas o clickhouse-server. Fonte: `/sys/block`. Veja https://www.kernel.org/doc/Documentation/block/stat.txt

<div id="blockqueuetimeperop_name">

Semelhante às métricas `BlockQueueTime`, mas o valor é dividido pelo número de operações de E/S para obter o tempo por operação.

<div id="blockwritebytes_name">

Número de bytes gravados no dispositivo de bloco. <div id="cgroupmaxcpu">
  ### CGroupMaxCPU
</div>

O número máximo de núcleos de CPU segundo os CGroups.

<div id="cgroupmemorytotal">
  ### CGroupMemoryTotal
</div>

A quantidade total de memória no cgroup, em bytes. Se esse valor for zero, o limite será o mesmo que OSMemoryTotal.

<div id="cgroupmemoryused">
  ### CGroupMemoryUsed
</div>

A quantidade de memória usada no cgroup, em bytes. No cgroup v2, isso corresponde a anon + sock + memória de kernel não recuperável; no cgroup v1, corresponde a RSS. Em ambos os casos, o cache de páginas do SO do kernel (cache baseado em arquivo) é excluído.

<div id="cgroupmemoryusedwithoutpagecache">
  ### CGroupMemoryUsedWithoutPageCache
</div>

A quantidade de memória usada no cgroup, em bytes, excluindo o cache de páginas em espaço do usuário do ClickHouse. Isso corresponde a CGroupMemoryUsed menos o tamanho do cache de páginas em espaço do usuário. Quando o cache de páginas em espaço do usuário está desabilitado, esse valor é igual a CGroupMemoryUsed.

<div id="cgroupsystemtime">
  ### CGroupSystemTime
</div>

A proporção do tempo em que o núcleo da CPU estava executando código do kernel do sistema operacional (system).

<div id="cgroupsystemtimenormalized">
  ### CGroupSystemTimeNormalized
</div>

O valor é semelhante a `CGroupSystemTime`, mas é dividido pelo número de núcleos de CPU disponíveis para ser medido no intervalo [0..1], independentemente da quantidade de núcleos. Isso permite calcular a média dos valores dessa métrica em vários servidores de um cluster, mesmo que o número de núcleos não seja uniforme, e ainda assim obter a métrica média de utilização de recursos. Se especificada, a cota de CPU do cgroup dividida pelo seu período pode ser usada em vez do número real de núcleos de CPU e, nesse caso, o valor dessa métrica pode exceder 1 em alguns momentos.

<div id="cgroupusertime">
  ### CGroupUserTime
</div>

A proporção de tempo em que o núcleo da CPU estava executando código em espaço de usuário. Isso também inclui o tempo em que a CPU ficou subutilizada por motivos internos à própria CPU (carregamentos de memória, stalls de pipeline, previsões incorretas de desvio, execução de outro núcleo SMT).

<div id="cgroupusertimenormalized">
  ### CGroupUserTimeNormalized
</div>

O valor é semelhante a `CGroupUserTime`, mas é dividido pelo número de núcleos de CPU disponíveis, para ser medido no intervalo [0..1] independentemente da quantidade de núcleos. Isso permite calcular a média dos valores dessa métrica em vários servidores de um cluster, mesmo que o número de núcleos não seja uniforme, e ainda assim obter a métrica média de utilização de recursos. Se especificada, a CPU quota do cgroup dividida pelo respectivo período pode ser usada no lugar do número real de núcleos de CPU e, nesse caso, o valor dessa métrica pode exceder 1 em determinados momentos.
