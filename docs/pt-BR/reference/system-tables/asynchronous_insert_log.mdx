<div id="description">
  ## Descrição
</div>

Contém informações sobre inserções assíncronas. Cada entrada representa uma consulta de inserção que foi armazenada em buffer por uma consulta de inserção assíncrona.

Para ativar o logging, configure os parâmetros na seção [asynchronous&#95;insert&#95;log](/pt-BR/reference/settings/server-settings/settings#asynchronous_insert_log).

O intervalo de flush dos dados é definido no parâmetro `flush_interval_milliseconds` da seção de configurações do servidor [asynchronous&#95;insert&#95;log](/pt-BR/reference/settings/server-settings/settings#asynchronous_insert_log). Para forçar o flush, use a consulta [SYSTEM FLUSH LOGS](/pt-BR/reference/statements/system#flush-logs).

O ClickHouse não exclui dados da tabela automaticamente. Consulte [Introdução](/pt-BR/reference/system-tables/overview#system-tables-introduction) para mais detalhes.

<div id="columns">
  ## Colunas
</div>

{/*AUTOGENERATED_START*/}

* `hostname` ([LowCardinality(String)](/pt-BR/reference/data-types/lowcardinality)) — Hostname do servidor que executa a consulta.
* `event_date` ([Date](/pt-BR/reference/data-types/date)) — A data em que a inserção assíncrona ocorreu.
* `event_time` ([DateTime](/pt-BR/reference/data-types/datetime)) — A data e a hora em que a inserção assíncrona terminou de ser executada.
* `event_time_microseconds` ([DateTime64(6)](/pt-BR/reference/data-types/datetime64)) — A data e a hora em que a inserção assíncrona terminou de ser executada, com precisão de microssegundos.
* `query` ([String](/pt-BR/reference/data-types/string)) — Texto da consulta.
* `database` ([LowCardinality(String)](/pt-BR/reference/data-types/lowcardinality)) — O nome do banco de dados em que a tabela está.
* `table` ([LowCardinality(String)](/pt-BR/reference/data-types/lowcardinality)) — Nome da tabela.
* `format` ([LowCardinality(String)](/pt-BR/reference/data-types/lowcardinality)) — Nome do formato.
* `query_id` ([String](/pt-BR/reference/data-types/string)) — ID da consulta inicial.
* `bytes` ([UInt64](/pt-BR/reference/data-types/int-uint)) — Número de bytes inseridos.
* `rows` ([UInt64](/pt-BR/reference/data-types/int-uint)) — Número de linhas inseridas.
* `exception` ([String](/pt-BR/reference/data-types/string)) — Mensagem da exceção.
* `status` ([Enum8(&#39;Ok&#39; = 0, &#39;ParsingError&#39; = 1, &#39;FlushError&#39; = 2)](/pt-BR/reference/data-types/enum)) — Status do insert. Valores: &#39;Ok&#39; = 0 — Insert bem-sucedido, &#39;ParsingError&#39; = 1 — Exceção ao analisar os dados, &#39;FlushError&#39; = 2 — Exceção ao fazer flush dos dados.
* `data_kind` ([Enum8(&#39;Parsed&#39; = 0, &#39;Preprocessed&#39; = 1)](/pt-BR/reference/data-types/enum)) — O status dos dados. Valores: &#39;Parsed&#39; e &#39;Preprocessed&#39;.
* `flush_time` ([DateTime](/pt-BR/reference/data-types/datetime)) — A data e a hora em que o flush ocorreu.
* `flush_time_microseconds` ([DateTime64(6)](/pt-BR/reference/data-types/datetime64)) — A data e a hora em que o flush ocorreu, com precisão de microssegundos.
* `flush_query_id` ([String](/pt-BR/reference/data-types/string)) — ID da consulta de flush.
* `timeout_milliseconds` ([UInt64](/pt-BR/reference/data-types/int-uint)) — O timeout adaptativo calculado para esta entrada.

{/*AUTOGENERATED_END*/}
