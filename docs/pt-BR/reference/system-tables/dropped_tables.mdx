<div id="description">
  ## Descrição
</div>

Contém informações sobre tabelas nas quais foi executado o comando DROP TABLE, mas cuja limpeza de dados ainda não foi realizada.

<div id="columns">
  ## Colunas
</div>

{/*AUTOGENERATED_START*/}

* `index` ([UInt32](/pt-BR/reference/data-types/index)) — Índice na fila marked&#95;dropped&#95;tables.
* `database` ([String](/pt-BR/reference/data-types/index)) — Nome do banco de dados.
* `table` ([String](/pt-BR/reference/data-types/index)) — Nome da tabela.
* `uuid` ([UUID](/pt-BR/reference/data-types/index)) — UUID da tabela.
* `engine` ([String](/pt-BR/reference/data-types/index)) — Nome do mecanismo da tabela.
* `metadata_dropped_path` ([String](/pt-BR/reference/data-types/index)) — Caminho do arquivo de metadados da tabela no diretório metadata&#95;dropped.
* `table_dropped_time` ([DateTime](/pt-BR/reference/data-types/index)) — Hora em que está agendada a próxima tentativa de remover os dados da tabela. Normalmente, é a hora em que a tabela foi removida mais `database_atomic_delay_before_drop_table_sec`.

{/*AUTOGENERATED_END*/}

  ## Exemplo
</div>

O exemplo a seguir mostra como obter informações sobre `dropped_tables`.

```sql
SELECT *
FROM system.dropped_tables\G
```
