<div id="description">
  ## Descrição
</div>

Contém a lista de fontes de dicionário suportadas pelo servidor, juntamente com a documentação embutida de cada fonte. Uma fonte de dicionário determina de onde os dados do dicionário são carregados; ela é especificada na cláusula `SOURCE` de uma consulta `CREATE DICTIONARY`.

<div id="columns">
  ## Colunas
</div>

{/*AUTOGENERATED_START*/}

* `name` ([String](/pt-BR/reference/data-types/index)) — O nome da fonte de dicionário, conforme especificado na cláusula SOURCE.
* `description` ([String](/pt-BR/reference/data-types/index)) — Uma descrição geral do que a fonte de dicionário faz.
* `syntax` ([String](/pt-BR/reference/data-types/index)) — A estrutura da cláusula SOURCE usada para especificar a origem. Observe que algumas origens estão sujeitas a controle de acesso quando um dicionário é criado a partir de uma consulta DDL (em vez de um arquivo de configuração do servidor); consulte a `description` da origem específica para mais detalhes.
* `examples` ([String](/pt-BR/reference/data-types/index)) — Exemplos de uso.
* `introduced_in` ([String](/pt-BR/reference/data-types/index)) — A versão do ClickHouse em que a origem foi introduzida pela primeira vez, no formato major.minor.
* `related` ([Array(String)](/pt-BR/reference/data-types/index)) — Os nomes de fontes de dicionário relacionadas.

{/*AUTOGENERATED_END*/}

<div id="example">
  ## Exemplo
</div>

```sql title="Query"
SELECT name, syntax
FROM system.dictionary_sources
WHERE name IN ('clickhouse', 'file')
ORDER BY name
```

```text title="Response"
┌─name───────┬─syntax─────────────────────────────────────────────────────────────────────────┐
│ clickhouse │ SOURCE(CLICKHOUSE(host 'host' port 9000 user 'default' password '' db 'db' table 'table')) │
│ file       │ SOURCE(FILE(path '/path/to/file' format 'CSV'))                                  │
└────────────┴────────────────────────────────────────────────────────────────────────────────┘
```
