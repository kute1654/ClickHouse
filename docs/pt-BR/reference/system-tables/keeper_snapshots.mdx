<div id="description">
  ## Descrição
</div>

Esta tabela não existe se este nó não estiver configurado para executar um ClickHouse Keeper in-process. Ela contém uma linha para cada arquivo de snapshot do Raft acompanhado pela máquina de estados do Keeper in-process, incluindo snapshots que estão sendo recebidos no momento do líder.

<div id="columns">
  ## Colunas
</div>

{/*AUTOGENERATED_START*/}

* `last_log_index` ([UInt64](/pt-BR/reference/data-types/int-uint)) — Último índice de log abrangido pelo snapshot.
* `path` ([String](/pt-BR/reference/data-types/string)) — Caminho do arquivo de snapshot no disco.
* `disk_name` ([String](/pt-BR/reference/data-types/string)) — Nome do disco que armazena o snapshot.
* `size_bytes` ([UInt64](/pt-BR/reference/data-types/int-uint)) — Tamanho do arquivo de snapshot no disco.
* `last_modified_at` ([DateTime](/pt-BR/reference/data-types/datetime)) — Hora da última modificação do arquivo de snapshot.
* `is_received` ([Bool](/pt-BR/reference/data-types/boolean)) — `true` se o snapshot estiver sendo recebido no momento do líder. Para essas linhas, `size_bytes` e `last_modified_at` refletem o arquivo parcial conforme foi gravado até agora e podem indicar um valor menor que o real.
* `exists_on_disk` ([Bool](/pt-BR/reference/data-types/boolean)) — Indica se o arquivo de snapshot está presente no disco no momento. Sempre `true` para snapshots finalizados (`is_received` = `false`), a menos que o arquivo tenha sido removido externamente ou esteja corrompido; pode ser `false` para snapshots em recebimento (`is_received` = `true`) antes que qualquer byte seja gravado.

{/*AUTOGENERATED_END*/}

Exemplo:

```sql
SELECT * FROM system.keeper_snapshots ORDER BY last_log_index;
```

└────────────────┴───────────────────────────────┴───────────┴────────────┴─────────────────────┴─────────────┴────────────────┘
