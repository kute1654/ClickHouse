`INFORMATION_SCHEMA` (ou `information_schema`) é um banco de dados de sistema que fornece uma [visão independente de SGBD](https://en.wikipedia.org/wiki/Information_schema) (em certa medida) padronizada dos metadados de objetos de banco de dados. As views em `INFORMATION_SCHEMA` geralmente são inferiores às tabelas de sistema normais, mas ferramentas podem usá-las para obter informações básicas de forma compatível entre diferentes SGBDs. Espera-se que a estrutura e o conteúdo das views em `INFORMATION_SCHEMA` evoluam de maneira retrocompatível, ou seja, apenas novas funcionalidades sejam adicionadas, sem que as funcionalidades existentes sejam alteradas ou removidas. Em termos de implementação interna, as views em `INFORMATION_SCHEMA` normalmente mapeiam para tabelas de sistema normais, como [system.columns](/pt-BR/reference/system-tables/columns), [system.databases](/pt-BR/reference/system-tables/databases) e [system.tables](/pt-BR/reference/system-tables/tables).

```sql
SHOW TABLES FROM INFORMATION_SCHEMA;

-- or:
SHOW TABLES FROM information_schema;
```

```text

│ COLUMNS                 │
│ KEY_COLUMN_USAGE        │
│ REFERENTIAL_CONSTRAINTS │
│ SCHEMATA                │
| STATISTICS              |
│ TABLES                  │
│ VIEWS                   │
│ columns                 │
│ key_column_usage        │
│ referential_constraints │
│ schemata                │
| statistics              |
│ tables                  │

└─────────────────────────┘
```

`INFORMATION_SCHEMA` contém as seguintes views:

* [COLUMNS](#columns)
* [KEY&#95;COLUMN&#95;USAGE](#key_column_usage)
* [REFERENTIAL&#95;CONSTRAINTS](#referential_constraints)
* [SCHEMATA](#schemata)
* [STATISTICS](#statistics)
* [TABLES](#tables)
* [VIEWS](#views)

Views equivalentes sem distinção entre maiúsculas e minúsculas, por exemplo `INFORMATION_SCHEMA.columns`, são fornecidas por motivos de compatibilidade com outros bancos de dados. O mesmo se aplica a todas as colunas dessas views: são fornecidas variantes tanto em minúsculas (por exemplo, `table_name`) quanto em maiúsculas (`TABLE_NAME`).

<div id="schemata">
  ## SCHEMATA
</div>

Contém colunas lidas da tabela de sistema [system.databases](/pt-BR/reference/system-tables/databases) e colunas que não são suportadas no ClickHouse ou não fazem sentido (sempre `NULL`), mas são exigidas pelo padrão.

Colunas:

* `catalog_name` ([String](/pt-BR/reference/data-types/string)) — O nome do banco de dados.
* `schema_name` ([String](/pt-BR/reference/data-types/string)) — O nome do banco de dados.
* `schema_owner` ([String](/pt-BR/reference/data-types/string)) — Nome do proprietário do schema, sempre `'default'`.
* `default_character_set_catalog` ([Nullable](/pt-BR/reference/data-types/nullable)([String](/pt-BR/reference/data-types/string))) — `NULL`, não suportado.
* `default_character_set_schema` ([Nullable](/pt-BR/reference/data-types/nullable)([String](/pt-BR/reference/data-types/string))) — `NULL`, não suportado.
* `default_character_set_name` ([Nullable](/pt-BR/reference/data-types/nullable)([String](/pt-BR/reference/data-types/string))) — `NULL`, não suportado.
* `sql_path` ([Nullable](/pt-BR/reference/data-types/nullable)([String](/pt-BR/reference/data-types/string))) — `NULL`, não suportado.

**Exemplo**

```sql title="Query"
SELECT catalog_name,
       schema_name,
       schema_owner,
       default_character_set_catalog,
       default_character_set_schema,
       default_character_set_name,
       sql_path
FROM information_schema.schemata
WHERE schema_name ILIKE 'information_schema' 
LIMIT 1 
FORMAT Vertical;
```

<div id="tables">
  ## TABLES
</div>

Contém as colunas lidas da tabela de sistema [system.tables](/pt-BR/reference/system-tables/tables).

Colunas:

* `table_catalog` ([String](/pt-BR/reference/data-types/string)) — O nome do banco de dados em que a tabela está localizada.
* `table_schema` ([String](/pt-BR/reference/data-types/string)) — O nome do banco de dados em que a tabela está localizada.
* `table_name` ([String](/pt-BR/reference/data-types/string)) — Nome da tabela.
* `table_type` ([String](/pt-BR/reference/data-types/string)) — Tipo da tabela. Valores possíveis:
  * `BASE TABLE`
  * `VIEW`
  * `FOREIGN TABLE`
  * `LOCAL TEMPORARY`
  * `SYSTEM VIEW`
* `table_rows` ([Nullable](/pt-BR/reference/data-types/nullable)([UInt64](/pt-BR/reference/data-types/int-uint))) — O número total de linhas. NULL se não puder ser determinado.
* `data_length` ([Nullable](/pt-BR/reference/data-types/nullable)([UInt64](/pt-BR/reference/data-types/int-uint))) — O tamanho dos dados em disco. NULL se não puder ser determinado.
* `index_length` ([Nullable](/pt-BR/reference/data-types/nullable)([UInt64](/pt-BR/reference/data-types/int-uint))) — O tamanho total da chave primária, dos índices secundários e de todas as marcas.
* `table_collation` ([Nullable](/pt-BR/reference/data-types/nullable)([String](/pt-BR/reference/data-types/string))) — A collation padrão da tabela. Sempre `utf8mb4_0900_ai_ci`.
* `table_comment` ([Nullable](/pt-BR/reference/data-types/nullable)([String](/pt-BR/reference/data-types/string))) — O comentário usado ao criar a tabela.

**Exemplo**

```sql title="Query"
SELECT table_catalog, 
       table_schema, 
       table_name, 
       table_type, 
       table_collation, 
       table_comment
FROM INFORMATION_SCHEMA.TABLES
WHERE (table_schema = currentDatabase() OR table_schema = '')
  AND table_name NOT LIKE '%inner%'
LIMIT 1 
FORMAT Vertical;
```

```text title="Response"
Row 1:
──────
table_catalog:   default
table_schema:    default
table_name:      describe_example
table_type:      BASE TABLE
table_collation: utf8mb4_0900_ai_ci
table_comment:   
```

<div id="views">
  ## VIEWS
</div>

Contém colunas lidas da tabela de sistema [system.tables](/pt-BR/reference/system-tables/tables), quando o motor de tabela [View](/pt-BR/reference/engines/table-engines/special/view) é usado.

Colunas:

{/*AUTOGENERATED_START*/}

* `table_catalog` ([String](/pt-BR/reference/data-types/string)) — O nome do banco de dados em que a tabela está localizada.
* `table_schema` ([String](/pt-BR/reference/data-types/string)) — O nome do banco de dados em que a tabela está localizada.
* `table_name` ([String](/pt-BR/reference/data-types/string)) — Nome da tabela.
* `view_definition` ([String](/pt-BR/reference/data-types/string)) — consulta `SELECT` da view.
* `check_option` ([String](/pt-BR/reference/data-types/string)) — `NONE`, sem verificação.
* `is_updatable` ([Enum8](/pt-BR/reference/data-types/enum)) — `NO`, a view não é atualizável.
* `is_insertable_into` ([Enum8](/pt-BR/reference/data-types/enum)) — Mostra se a view criada é [materializada](/pt-BR/reference/statements/create/view#materialized-view). Valores possíveis:
  * `NO` — A view criada não é materializada.
  * `YES` — A view criada é materializada.
* `is_trigger_updatable` ([Enum8](/pt-BR/reference/data-types/enum)) — `NO`, o gatilho não é atualizável.
* `is_trigger_deletable` ([Enum8](/pt-BR/reference/data-types/enum)) — `NO`, o gatilho não pode ser excluído.
* `is_trigger_insertable_into` ([Enum8](/pt-BR/reference/data-types/enum)) — `NO`, nenhum dado é inserido no gatilho.

{/*AUTOGENERATED_END*/}

**Exemplo**

```sql title="Query"
CREATE VIEW v (n Nullable(Int32), f Float64) AS SELECT n, f FROM t;
CREATE MATERIALIZED VIEW mv ENGINE = Null AS SELECT * FROM system.one;
SELECT table_catalog,
       table_schema,
       table_name,
       view_definition,
       check_option,
       is_updatable,
       is_insertable_into,
       is_trigger_updatable,
       is_trigger_deletable,
       is_trigger_insertable_into
FROM information_schema.views
WHERE table_schema = currentDatabase() 
LIMIT 1
FORMAT Vertical;
```

```text title="Response"
Row 1:
──────
table_catalog:              default
table_schema:               default
table_name:                 mv
view_definition:            SELECT * FROM system.one
check_option:               NONE
is_updatable:               NO
is_insertable_into:         YES
is_trigger_updatable:       NO
is_trigger_deletable:       NO
is_trigger_insertable_into: NO
```

<div id="key_column_usage">
  ## KEY_COLUMN_USAGE
</div>

Contém colunas da tabela de sistema [system.tables](/pt-BR/reference/system-tables/tables) às quais se aplicam restrições.

Colunas:

* `constraint_catalog` ([String](/pt-BR/reference/data-types/string)) — Atualmente sem uso. Sempre `def`.
* `constraint_schema` ([String](/pt-BR/reference/data-types/string)) — O nome do schema (banco de dados) ao qual a restrição pertence.
* `constraint_name` ([Nullable](/pt-BR/reference/data-types/nullable)([String](/pt-BR/reference/data-types/string))) — O nome da restrição.
* `table_catalog` ([String](/pt-BR/reference/data-types/string)) — Atualmente sem uso. Sempre `def`.
* `table_schema` ([String](/pt-BR/reference/data-types/string)) — O nome do schema (banco de dados) ao qual a tabela pertence.
* `table_name` ([String](/pt-BR/reference/data-types/string)) — O nome da tabela à qual a restrição se aplica.
* `column_name` ([Nullable](/pt-BR/reference/data-types/nullable)([String](/pt-BR/reference/data-types/string))) — O nome da coluna à qual a restrição se aplica.
* `ordinal_position` ([UInt32](/pt-BR/reference/data-types/int-uint)) — Atualmente sem uso. Sempre `1`.
* `position_in_unique_constraint` ([Nullable](/pt-BR/reference/data-types/nullable)([UInt32](/pt-BR/reference/data-types/int-uint))) — Atualmente sem uso. Sempre `NULL`.
* `referenced_table_schema` ([Nullable](/pt-BR/reference/data-types/nullable)([String](/pt-BR/reference/data-types/string))) — Atualmente sem uso. Sempre NULL.
* `referenced_table_name` ([Nullable](/pt-BR/reference/data-types/nullable)([String](/pt-BR/reference/data-types/string))) — Atualmente sem uso. Sempre NULL.
* `referenced_column_name` ([Nullable](/pt-BR/reference/data-types/nullable)([String](/pt-BR/reference/data-types/string))) — Atualmente sem uso. Sempre NULL.

**Exemplo**

```sql title="Query"
CREATE TABLE test (i UInt32, s String) ENGINE MergeTree ORDER BY i;
SELECT constraint_catalog,
       constraint_schema,
       constraint_name,
       table_catalog,
       table_schema,
       table_name,
       column_name,
       ordinal_position,
       position_in_unique_constraint,
       referenced_table_schema,
       referenced_table_name,
       referenced_column_name
FROM information_schema.key_column_usage 
WHERE table_name = 'test' 
FORMAT Vertical;
```
