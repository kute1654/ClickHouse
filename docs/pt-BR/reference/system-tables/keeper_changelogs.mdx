<div id="description">
  ## Descrição
</div>

Esta tabela não existe se este nó não estiver configurado para executar um ClickHouse Keeper no próprio processo. Ela contém uma linha por arquivo de changelog do Raft (`changelog_<from>_<to>.bin[.zstd]`) acompanhado pelo armazenamento de logs do Keeper no próprio processo, incluindo o arquivo ativo ao qual novas entradas estão sendo acrescentadas no momento.

<div id="columns">
  ## Colunas
</div>

{/*AUTOGENERATED_START*/}

* `from_log_index` ([UInt64](/pt-BR/reference/data-types/int-uint)) — Primeiro índice de log do Raft no arquivo (inclusivo).
* `to_log_index` ([UInt64](/pt-BR/reference/data-types/int-uint)) — Último índice de log do Raft abrangido pelo nome do arquivo (inclusivo). No arquivo ativo, este é o alvo da rotação e pode estar à frente de `last_entry_index`.
* `last_entry_index` ([Nullable(UInt64)](/pt-BR/reference/data-types/int-uint)) — Maior índice de log efetivamente anexado a este arquivo. `NULL` se o arquivo ativo ainda não tiver recebido nenhuma entrada ou se o arquivo estiver danificado.
* `entries` ([UInt64](/pt-BR/reference/data-types/int-uint)) — Número de entradas anexadas a este arquivo. `ALIAS` para `ifNull(last_entry_index - from_log_index + 1, 0)`.
* `path` ([String](/pt-BR/reference/data-types/string)) — Caminho do arquivo no disco.
* `disk_name` ([String](/pt-BR/reference/data-types/string)) — Nome do disco que armazena o arquivo.
* `size_bytes` ([UInt64](/pt-BR/reference/data-types/int-uint)) — Tamanho do arquivo no disco.
* `modification_time` ([DateTime](/pt-BR/reference/data-types/datetime)) — Data e hora da última modificação do arquivo.
* `is_compressed` ([Bool](/pt-BR/reference/data-types/boolean)) — O payload do arquivo é comprimido com zstd.
* `active` ([Bool](/pt-BR/reference/data-types/boolean)) — Este arquivo está recebendo novas entradas no momento.
* `is_broken` ([Bool](/pt-BR/reference/data-types/boolean)) — Foi encontrado um registro final corrompido na inicialização.

{/*AUTOGENERATED_END*/}

Exemplo:

```sql
SELECT from_log_index, to_log_index, entries, path, active FROM system.keeper_changelogs ORDER BY from_log_index;
```

```text
┌─from_log_index─┬─to_log_index─┬─entries─┬─path───────────────────────────┬─active─┐
│              1 │         1000 │    1000 │ changelog_1_1000.bin.zstd      │ false  │
│           1001 │         2000 │     537 │ changelog_1001_2000.bin.zstd   │ true   │
└────────────────┴──────────────┴─────────┴────────────────────────────────┴────────┘
```
