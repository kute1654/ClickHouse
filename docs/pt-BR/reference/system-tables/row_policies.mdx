<div id="description">
  ## Descrição
</div>

Contém filtros para uma tabela específica, bem como uma lista de roles e/ou usuários aos quais esta ROW POLICY deve ser aplicada.

<div id="columns">
  ## Colunas
</div>

{/*AUTOGENERATED_START*/}

* `name` ([String](/pt-BR/reference/data-types/index)) — Nome de uma ROW POLICY.
* `short_name` ([String](/pt-BR/reference/data-types/index)) — Nome curto de uma ROW POLICY. Os nomes das ROW POLICY são compostos, por exemplo: myfilter ON mydb.mytable. Aqui, &#39;myfilter ON mydb.mytable&#39; é o nome da ROW POLICY, e &#39;myfilter&#39; é seu nome curto.
* `database` ([String](/pt-BR/reference/data-types/index)) — nome do banco de dados.
* `table` ([String](/pt-BR/reference/data-types/index)) — Nome da tabela. Fica vazio se a política for para o banco de dados.
* `id` ([UUID](/pt-BR/reference/data-types/index)) — ID da ROW POLICY.
* `storage` ([String](/pt-BR/reference/data-types/index)) — Nome do diretório em que a ROW POLICY é armazenada.
* `select_filter` ([Nullable(String)](/pt-BR/reference/data-types/index)) — Expressão usada para filtragem em consultas SELECT.
* `is_restrictive` ([UInt8](/pt-BR/reference/data-types/index)) — Mostra se a ROW POLICY restringe o acesso às linhas. Valor: • 0 — A ROW POLICY é definida com a cláusula `AS PERMISSIVE`, • 1 — A ROW POLICY é definida com a cláusula `AS RESTRICTIVE`.
* `apply_to_all` ([UInt8](/pt-BR/reference/data-types/index)) — Mostra que as ROW POLICY estão definidas para todos os role e/ou usuários.
* `apply_to_list` ([Array(String)](/pt-BR/reference/data-types/index)) — Lista dos role e/ou usuários aos quais as ROW POLICY são aplicadas.
* `apply_to_except` ([Array(String)](/pt-BR/reference/data-types/index)) — As ROW POLICY são aplicadas a todos os role e/ou usuários, exceto aos listados.

{/*AUTOGENERATED_END*/}
