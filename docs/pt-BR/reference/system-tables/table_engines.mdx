<div id="description">
  ## Descrição
</div>

Contém a descrição dos motores de tabela compatíveis com o servidor e informações sobre o suporte aos seus recursos.

Esta tabela contém as seguintes colunas (o tipo da coluna é mostrado entre parênteses):

{/*AUTOGENERATED_START*/}

* `name` ([String](/pt-BR/reference/data-types/index)) — O nome do motor de tabela.
* `supports_settings` ([UInt8](/pt-BR/reference/data-types/index)) — Sinalizador que indica se o motor de tabela oferece suporte à cláusula SETTINGS.
* `supports_skipping_indices` ([UInt8](/pt-BR/reference/data-types/index)) — Sinalizador que indica se o motor de tabela oferece suporte a skipping indices.
* `supports_projections` ([UInt8](/pt-BR/reference/data-types/index)) — Sinalizador que indica se o motor de tabela oferece suporte a projeções.
* `supports_sort_order` ([UInt8](/pt-BR/reference/data-types/index)) — Sinalizador que indica se o motor de tabela oferece suporte às cláusulas PARTITION&#95;BY, PRIMARY&#95;KEY, ORDER&#95;BY e SAMPLE&#95;BY.
* `supports_ttl` ([UInt8](/pt-BR/reference/data-types/index)) — Sinalizador que indica se o motor de tabela oferece suporte a TTL.
* `supports_replication` ([UInt8](/pt-BR/reference/data-types/index)) — Sinalizador que indica se o motor de tabela oferece suporte à replicação de dados.
* `supports_deduplication` ([UInt8](/pt-BR/reference/data-types/index)) — Sinalizador que indica se o motor de tabela oferece suporte à desduplicação de dados.
* `supports_parallel_insert` ([UInt8](/pt-BR/reference/data-types/index)) — Sinalizador que indica se o motor de tabela oferece suporte a inserções paralelas (consulte a configuração max&#95;insert&#95;threads).
* `description` ([String](/pt-BR/reference/data-types/index)) — Uma descrição do que o motor de tabela faz. Para motores que têm uma página de documentação dedicada, este campo contém o corpo completo em Markdown dessa página; para os demais motores, traz um resumo conciso.
* `syntax` ([String](/pt-BR/reference/data-types/index)) — Como o motor de tabela é usado ao criar uma tabela. Para a maioria dos motores, trata-se da cláusula ENGINE de uma consulta CREATE TABLE, mas alguns motores (como os vários tipos de views ou o motor Loop) são usados de outras formas, como CREATE VIEW ou um SELECT a partir de uma table function.
* `examples` ([String](/pt-BR/reference/data-types/index)) — Exemplos de uso.
* `introduced_in` ([String](/pt-BR/reference/data-types/index)) — A versão do ClickHouse em que o motor de tabela foi introduzido pela primeira vez, no formato major.minor.
* `related` ([Array(String)](/pt-BR/reference/data-types/index)) — Os nomes dos motores de tabela relacionados.

{/*AUTOGENERATED_END*/}

<div id="example">
  ## Exemplo
</div>

```sql title="Query"
SELECT
    name,
    supports_settings,
    supports_skipping_indices,
    supports_sort_order,
    supports_ttl,
    supports_replication,
    supports_deduplication,
    supports_parallel_insert
FROM system.table_engines
WHERE name IN ('Kafka', 'MergeTree', 'ReplicatedCollapsingMergeTree')
```
