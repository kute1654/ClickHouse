<div id="description">
  ## Descrição
</div>

Contém uma lista de todos os formatos, com indicadores de se um formato é adequado para entrada/saída e se oferece suporte à paralelização.

<div id="columns">
  ## Colunas
</div>

{/*AUTOGENERATED_START*/}

* `name` ([String](/pt-BR/reference/data-types/index)) — Nome do formato.
* `is_input` ([UInt8](/pt-BR/reference/data-types/index)) — Flag que indica se o formato é adequado para entrada de dados.
* `is_output` ([UInt8](/pt-BR/reference/data-types/index)) — Flag que indica se o formato é adequado para saída de dados.
* `supports_parallel_parsing` ([UInt8](/pt-BR/reference/data-types/index)) — Flag que indica se o formato oferece suporte a parsing paralelo.
* `supports_parallel_formatting` ([UInt8](/pt-BR/reference/data-types/index)) — Flag que indica se o formato oferece suporte a formatação paralela.
* `is_tty_friendly` ([UInt8](/pt-BR/reference/data-types/index)) — Flag que indica se o formato geralmente é exibido corretamente no terminal. Para outros formatos, a CLI solicitará confirmação antes de gerar a saída.
* `content_type` ([String](/pt-BR/reference/data-types/index)) — HTTP Content-Type correspondente ao formato de saída. Pode depender das configurações atuais do formato.
* `supports_random_access` ([UInt8](/pt-BR/reference/data-types/index)) — Flag que indica se o formato oferece suporte a acesso aleatório na entrada.
* `has_schema_inference` ([UInt8](/pt-BR/reference/data-types/index)) — O formato pode determinar dinamicamente o esquema a partir dos dados (seja de header/metadata embutidos ou do trecho de dados).
* `has_external_schema` ([UInt8](/pt-BR/reference/data-types/index)) — O formato tem um esquema fixo ou aceita um esquema predefinido em seu próprio formato.
* `prefers_large_blocks` ([UInt8](/pt-BR/reference/data-types/index)) — O formato gravará blocos maiores na saída e gerará blocos maiores na entrada.
* `supports_append` ([UInt8](/pt-BR/reference/data-types/index)) — É possível acrescentar dados a um único arquivo com este formato.
* `supports_subsets_of_columns` ([UInt8](/pt-BR/reference/data-types/index)) — O formato de entrada pode reconhecer quando determinadas colunas são omitidas.
* `description` ([String](/pt-BR/reference/data-types/index)) — Uma descrição geral do formato.
* `examples` ([String](/pt-BR/reference/data-types/index)) — Exemplos de uso.
* `introduced_in` ([String](/pt-BR/reference/data-types/index)) — A versão do ClickHouse na qual o formato foi introduzido pela primeira vez, no formato major.minor.
* `related` ([Array(String)](/pt-BR/reference/data-types/index)) — Os nomes de formatos relacionados.

{/*AUTOGENERATED_END*/}
