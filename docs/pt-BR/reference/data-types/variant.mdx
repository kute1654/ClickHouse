Este tipo representa uma uniГЈo de outros tipos de dados. O tipo `Variant(T1, T2, ..., TN)` significa que cada linha desse tipo
tem um valor do tipo `T1`, `T2`, ... ou `TN`, ou de nenhum deles (valor `NULL`).

A ordem dos tipos aninhados nГЈo importa: Variant(T1, T2) = Variant(T2, T1).
Os tipos aninhados podem ser quaisquer tipos, exceto Nullable(...), LowCardinality(Nullable(...)) e Variant(...).

<Note>
  NГЈo Г© recomendado usar tipos semelhantes como variantes (por exemplo, tipos numГ©ricos diferentes como `Variant(UInt32, Int64)` ou tipos de data diferentes como `Variant(Date, DateTime)`),
  porque trabalhar com valores desses tipos pode gerar ambiguidades. Por padrГЈo, a criaГ§ГЈo desse tipo `Variant` gerarГЎ uma exceГ§ГЈo, mas isso pode ser habilitado usando a configuraГ§ГЈo `allow_suspicious_variant_types`
</Note>

  ## Comparando valores do tipo Variant

Valores de um tipo `Variant` sГі podem ser comparados com valores do mesmo tipo `Variant`.

Por padrГЈo, os operadores de comparaГ§ГЈo usam a [implementaГ§ГЈo padrГЈo para Variant](#functions-with-variant-arguments),
