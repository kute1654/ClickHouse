Os motores de tabela da família MergeTree são o núcleo dos recursos de armazenamento de dados do ClickHouse. Eles oferecem a maior parte dos recursos de resiliência e recuperação de dados de alto desempenho: armazenamento colunar, particionamento personalizado, índice primário esparso, índices secundários para ignorar dados, etc.

O motor de tabela base [MergeTree](/pt-BR/reference/engines/table-engines/mergetree-family/mergetree) pode ser considerado o motor de tabela padrão para instâncias de nó único do ClickHouse, porque é versátil e prático para uma ampla variedade de casos de uso.

Para uso em produção, [ReplicatedMergeTree](/pt-BR/reference/engines/table-engines/mergetree-family/replication) é a melhor opção, porque adiciona alta disponibilidade a todos os recursos do motor MergeTree comum. Um bônus é a desduplicação automática na ingestão de dados, para que o software possa tentar novamente com segurança caso haja algum problema de rede durante o insert.

Todos os outros motores da família MergeTree adicionam funcionalidades extras para casos de uso específicos. Normalmente, isso é implementado como manipulação adicional de dados em segundo plano.

A principal desvantagem dos motores MergeTree é que eles são relativamente pesados. Portanto, o padrão é não ter muitos deles. Se você precisar de muitas tabelas pequenas, por exemplo para dados temporários, considere a [família de motores Log](/pt-BR/reference/engines/table-engines/log-family/index).
