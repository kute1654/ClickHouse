Você pode usar as funções descritas neste capítulo para inspecionar [ELF](https://en.wikipedia.org/wiki/Executable_and_Linkable_Format) e [DWARF](https://en.wikipedia.org/wiki/DWARF) no profiling de consultas.

<Note>
  Essas funções são lentas e podem ter implicações de segurança.
</Note>

Para o funcionamento correto das funções de introspecção:

* Instale o pacote `clickhouse-common-static-dbg`.

* Defina a configuração [allow&#95;introspection&#95;functions](/pt-BR/reference/settings/session-settings#allow_introspection_functions) como 1.

  Por motivos de segurança, as funções de introspecção ficam desabilitadas por padrão.

O ClickHouse salva relatórios do profiler na tabela de sistema [trace&#95;log](/pt-BR/reference/system-tables/trace_log). Certifique-se de que a tabela e o profiler estejam configurados corretamente.

<div id="addressToLine">
  ## addressToLine
</div>

Introduzido em: v20.1.0

Converte um endereço de memória virtual dentro do processo do servidor ClickHouse em um nome de arquivo e um número de linha no código-fonte do ClickHouse.

<Note>
  Esta função é lenta e pode trazer implicações de segurança.
</Note>

Para habilitar esta função de introspecção:

* Instale o pacote `clickhouse-common-static-dbg`.
* Defina a configuração [`allow_introspection_functions`](/pt-BR/reference/settings/session-settings#allow_introspection_functions) como `1`.

**Sintaxe**

```sql
addressToLine(address_of_binary_instruction)
```

**Argumentos**

* `address_of_binary_instruction` — Endereço da instrução em um processo em execução. [`UInt64`](/pt-BR/reference/data-types/int-uint)

**Valor retornado**

Retorna um nome de arquivo de código-fonte e um número de linha separados por dois-pontos, por exemplo, `/build/obj-x86_64-linux-gnu/../src/Common/ThreadPool.cpp:199`. Retorna o nome de um binário se nenhuma informação de depuração for encontrada; caso contrário, retorna uma string vazia se o endereço não for válido. [`String`](/pt-BR/reference/data-types/string)

**Exemplos**

**Selecionando a primeira string da tabela de sistema `trace_log`**

```sql title=Query
SET allow_introspection_functions=1;
SELECT * FROM system.trace_log LIMIT 1 \G;
```


**Como obter o nome do arquivo do código-fonte e o número da linha de um único endereço**

<div id="addressToLineWithInlines">
  ## addressToLineWithInlines
</div>

Introduzido em: v22.2.0

Semelhante a `addressToLine`, mas retorna um Array com todas as funções embutidas.
Por isso, é mais lento que `addressToLine`.

Para habilitar esta função de introspecção:

* Instale o pacote `clickhouse-common-static-dbg`.
* Defina a configuração [`allow_introspection_functions`](/pt-BR/reference/settings/session-settings#allow_introspection_functions) como `1`.

**Sintaxe**

```sql
addressToLineWithInlines(address_of_binary_instruction)
```

**Argumentos**

* `address_of_binary_instruction` — O endereço de uma instrução em um processo em execução. [`UInt64`](/pt-BR/reference/data-types/int-uint)

**Valor retornado**

Retorna um array cujo primeiro elemento é o nome do arquivo de código-fonte e o número da linha, separados por dois-pontos. O segundo, o terceiro etc. elementos listam os nomes dos arquivos de código-fonte, os números das linhas e os nomes das funções inline. Se nenhuma informação de depuração for encontrada, será retornado um array com um único elemento igual ao nome do binário; caso contrário, será retornado um array vazio se o endereço não for válido. [`Array(String)`](/pt-BR/reference/data-types/array)

**Exemplos**

**Aplicando a função a um endereço**

```sql title=Query
SET allow_introspection_functions=1;
SELECT addressToLineWithInlines(531055181::UInt64);
```

<div id="isMergeTreePartCoveredBy">

