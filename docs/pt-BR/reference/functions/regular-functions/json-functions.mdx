<div id="types-of-functions">
  ## Tipos de funções JSON
</div>

Há dois conjuntos de funções para analisar de JSON:

* [`simpleJSON*` (`visitParam*`)](#simplejson-visitparam-functions), feito para fazer o analisar de um subconjunto limitado de JSON com extrema rapidez.
* [`JSONExtract*`](#jsonextract-functions), feito para fazer o analisar de JSON comum.

<div id="simplejson-visitparam-functions">
  ### funções `simpleJSON` (visitParam)
</div>

O ClickHouse tem funções especiais para trabalhar com JSON simplificado. Todas essas funções JSON se baseiam em pressupostos rígidos sobre como o JSON pode ser. Elas procuram fazer o mínimo possível para executar a tarefa o mais rápido possível.

Os seguintes pressupostos são adotados:

1. O nome do campo (argumento da função) deve ser uma constante.
2. O nome do campo deve estar codificado de forma canônica no JSON. Por exemplo: `simpleJSONHas('{"abc":"def"}', 'abc') = 1`, mas `simpleJSONHas('{"\\u0061\\u0062\\u0063":"def"}', 'abc') = 0`
3. Os campos são procurados em qualquer nível de aninhamento, sem distinção. Se houver vários campos correspondentes, a primeira ocorrência será usada.
4. O JSON não contém caracteres de espaço fora de literais de string.

<div id="jsonextract-functions">
  ### Funções JSONExtract
</div>

Essas funções são baseadas em [simdjson](https://github.com/lemire/simdjson) e foram projetadas para necessidades mais complexas de análise de JSON.

<div id="case-insensitive-jsonextract-functions">
  ### Funções JSONExtract sem distinção entre maiúsculas e minúsculas
</div>

Essas funções fazem correspondência de chaves em ASCII sem distinção entre maiúsculas e minúsculas ao extrair valores de objetos JSON.
Elas funcionam de forma idêntica às equivalentes com distinção entre maiúsculas e minúsculas, exceto pelo fato de que as chaves do objeto são correspondidas sem considerar o uso de maiúsculas e minúsculas.
Quando várias chaves correspondem com grafias diferentes entre maiúsculas e minúsculas, a primeira correspondência é retornada.

<Note>
  Essas funções podem ter desempenho inferior ao das equivalentes com distinção entre maiúsculas e minúsculas, portanto, use as funções JSONExtract regulares, se possível.
</Note>

<div id="JSONAllPaths">
  ## JSONAllPaths
</div>

Introduzido em: v24.8.0

Retorna a lista de todos os caminhos armazenados em cada linha de uma coluna JSON.

**Sintaxe**

```sql
JSONAllPaths(json)
```

**Argumentos**

* `json` — coluna JSON. [`JSON`](/pt-BR/reference/data-types/newjson)

**Valor retornado**

Retorna um array com todos os caminhos da coluna JSON. [`Array(String)`](/pt-BR/reference/data-types/array)

**Exemplos**

**Exemplo de uso**

```sql title=Query
CREATE TABLE test (json JSON(max_dynamic_paths=1)) ENGINE = Memory;
INSERT INTO test FORMAT JSONEachRow {"json" : {"a" : 42}}, {"json" : {"b" : "Hello"}}, {"json" : {"a" : [1, 2, 3], "c" : "2020-01-01"}}
SELECT json, JSONAllPaths(json) FROM test;
```

```response title=Response
┌─json─────────────────────────────────┬─JSONAllPaths(json)─┐
│ {"a":"42"}                           │ ['a']              │
│ {"b":"Hello"}                        │ ['b']              │
│ {"a":["1","2","3"],"c":"2020-01-01"} │ ['a','c']          │
└──────────────────────────────────────┴────────────────────┘
```

<div id="JSONExtract">
  ## JSONExtract
</div>

Introduzido em: v19.14.0

Analisa JSON e extrai um valor do tipo de dado ClickHouse especificado.

**Sintaxe**

```sql
JSONExtract(json[, indices_or_keys, ...], return_type)
```

**Argumentos**

* `json` — String JSON a ser processada. [`String`](/pt-BR/reference/data-types/string)
* `indices_or_keys` — Uma lista de zero ou mais argumentos, cada um dos quais pode ser uma string ou um inteiro. [`String`](/pt-BR/reference/data-types/string) ou [`(U)Int*`](/pt-BR/reference/data-types/int-uint)
* `return_type` — Tipo de dados do ClickHouse a ser retornado. [`String`](/pt-BR/reference/data-types/string)

**Valor retornado**

Retorna um valor do tipo de dados do ClickHouse especificado, se possível; caso contrário, retorna o valor padrão desse tipo.

**Exemplos**

**Exemplo de uso**

```sql title=Query
SELECT JSONExtract('{"a": "hello", "b": [-100, 200.0, 300]}', 'Tuple(String, Array(Float64))') AS res;
```

```response title=Response
┌─res──────────────────────────────┐
│ ('hello',[-100,200,300])         │
└──────────────────────────────────┘
```
