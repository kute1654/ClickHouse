<div id="h3-index">
  ## H3 Index
</div>

[H3](https://h3geo.org/) é um sistema de indexação geográfica em que a superfície da Terra é dividida em uma grade de células hexagonais uniformes. Esse sistema é hierárquico, ou seja, cada hexágono no nível superior (&quot;pai&quot;) pode ser dividido em sete outros hexágonos uniformes, porém menores (&quot;filhos&quot;), e assim por diante.

O nível da hierarquia é chamado de `resolution` e pode assumir um valor de `0` a `15`, em que `0` é o nível `base`, com as células maiores e menos detalhadas.

Um par de latitude e longitude pode ser transformado em um índice H3 de 64 bits, que identifica uma célula da grade.

O índice H3 é usado principalmente para agrupar localizações e realizar outras manipulações geoespaciais.

A descrição completa do sistema H3 está disponível no [site de Engenharia da Uber](https://www.uber.com/blog/h3/).

<div id="h3isvalid">
  ## h3IsValid
</div>

Verifica se o número é um índice [H3](#h3-index) válido.

**Sintaxe**

```sql
h3IsValid(h3index)
```

**Parâmetro**

* `h3index` — Número do índice hexagonal. [UInt64](/pt-BR/reference/data-types/int-uint).

**Valores retornados**

* 1 — O número é um índice H3 válido. [UInt8](/pt-BR/reference/data-types/int-uint).
* 0 — O número não é um índice H3 válido. [UInt8](/pt-BR/reference/data-types/int-uint).

**Exemplo**

```sql title="Query"
SELECT h3IsValid(630814730351855103) AS h3IsValid;
```

```text title="Response"
┌─h3IsValid─┐
│         1 │
└───────────┘
```

<div id="h3getresolution">
  ## h3GetResolution
</div>

Define a resolução do índice [H3](#h3-index) informado.

**Sintaxe**

```sql
h3GetResolution(h3index)
```

**Parâmetro**

* `h3index` — Número do índice hexagonal. [UInt64](/pt-BR/reference/data-types/int-uint).

**Valores retornados**

* Resolução do índice. Intervalo: `[0, 15]`. [UInt8](/pt-BR/reference/data-types/int-uint).
* Se o índice não for válido, a função retornará um valor aleatório. Use [h3IsValid](#h3isvalid) para verificar o índice. [UInt8](/pt-BR/reference/data-types/int-uint).

**Exemplo**

```sql title="Query"
SELECT h3GetResolution(639821929606596015) AS resolution;
```

```text title="Response"
┌─resolution─┐
│         14 │
└────────────┘
```

<div id="geotoh3">
  ## geoToH3
</div>

Retorna [H3](#h3-index) o índice do ponto `(lat, lon)` com a resolução especificada.

**Sintaxe**

```sql
geoToH3(lat, lon, resolution)
```

**Argumentos**

* `lat` — Latitude. [Float64](/pt-BR/reference/data-types/float).
* `lon` — Longitude. [Float64](/pt-BR/reference/data-types/float).
* `resolution` — Resolução do índice. Intervalo: `[0, 15]`. [UInt8](/pt-BR/reference/data-types/int-uint).

**Valores retornados**

* Número do índice hexagonal. [UInt64](/pt-BR/reference/data-types/int-uint).
* 0 em caso de erro. [UInt64](/pt-BR/reference/data-types/int-uint).

Observação: no ClickHouse v25.4 ou anterior, `geoToH3()` recebe os valores na ordem `(lon, lat)`. A partir do ClickHouse v25.5, os valores de entrada seguem a ordem `(lat, lon)`. O comportamento anterior pode ser restaurado usando a configuração `geotoh3_argument_order = 'lon_lat'`.

**Exemplo**

```sql title="Query"
SELECT geoToH3(55.71290588, 37.79506683, 15) AS h3Index;
```

```text title="Response"
┌────────────h3Index─┐
│ 644325524701193974 │
└────────────────────┘
```

Observação: no ClickHouse v24.12 ou anterior, `h3ToGeo()` retorna os valores na ordem `(lon, lat)`. A partir do ClickHouse v25.1, os valores retornados ficam na ordem `(lat, lon)`. O comportamento anterior pode ser restaurado usando a configuração `h3togeo_lon_lat_result_order = true`.

**Exemplo**



<div id="h3polygontocellswithcontainment">
  ## h3PolygonToCellsWithContainment
</div>

Retorna os hexágonos (na resolução especificada) que cobrem a geometria fornecida, seja um anel ou um (multi)polígono, usando os modos experimentais de contenção do H3.

Esta função associa as `flags` aos modos de contenção do H3:

* `0` — `CONTAINMENT_CENTER`
* `1` — `CONTAINMENT_FULL`
* `2` — `CONTAINMENT_OVERLAPPING`
* `3` — `CONTAINMENT_OVERLAPPING_BBOX`

**Sintaxe**

```sql
h3PolygonToCellsWithContainment(geometry, resolution, flags)
```

**Argumentos**

* `geometry` pode ser um dos seguintes [Tipos de Dados Geo](/pt-BR/reference/data-types/geo) ou seus respectivos tipos primitivos:
  * [ring](/pt-BR/reference/data-types/geo#ring)
  * [Polygon](/pt-BR/reference/data-types/geo#polygon)
  * [multipolígono](/pt-BR/reference/data-types/geo#multipolygon)
* `resolution` — Resolução do índice. Intervalo: `[0, 15]`. [UInt8](/pt-BR/reference/data-types/int-uint).
* `flags` — Modo de contenção. Intervalo: `[0, 3]`. Use literais inteiros como `0`, `1`, `2` e `3`, ou valores [UInt32](/pt-BR/reference/data-types/int-uint) (por exemplo, `toUInt32(2)`). Os valores são passados para a API H3 como `UInt32`. Outros tipos inteiros nativos são convertidos com exatidão.

**Valores retornados**

* Array de índices H3 de acordo com o modo de contenção selecionado. [Array](/pt-BR/reference/data-types/array)([UInt64](/pt-BR/reference/data-types/int-uint)).

**Exemplo**

```sql title="Query"
SELECT
    h3PolygonToCellsWithContainment([(-122.4089866999972145, 37.813318999983238), (-122.3544736999993603, 37.7198061999978478), (-122.4798767000009008, 37.8151571999998453)], 7, 0) AS center_cells,
```
