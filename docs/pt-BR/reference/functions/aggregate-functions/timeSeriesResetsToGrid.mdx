<div id="timeSeriesResetsToGrid">
  ## timeSeriesResetsToGrid
</div>

Introduzido em: v25.6.0

Função de agregação que recebe dados de série temporal como pares de timestamps e valores e calcula [resets no estilo do PromQL](https://prometheus.io/docs/prometheus/latest/querying/functions/#resets) a partir desses dados em uma grade de tempo regular descrita por timestamp inicial, timestamp final e passo. Para cada ponto da grade, as amostras usadas para calcular `resets` são consideradas na janela de tempo especificada.

<Note>
  Esta função é experimental; habilite-a definindo `allow_experimental_ts_to_grid_aggregate_function=true`.
</Note>

**Sintaxe**

```sql
timeSeriesResetsToGrid(start_timestamp, end_timestamp, grid_step, staleness)(timestamp, value)
```

**Parâmetros**

* `start_timestamp` — Especifica o início da grade. - `end_timestamp` — Especifica o fim da grade. - `grid_step` — Especifica o passo da grade em segundos. - `staleness` — Especifica a &quot;staleness&quot; máxima, em segundos, das amostras consideradas.

**Argumentos**

* `timestamp` — Timestamp da amostra. Pode ser um valor individual ou arrays. - `value` — Valor da série temporal correspondente ao timestamp. Pode ser um valor individual ou arrays.

**Valor retornado**

Valores de `resets` na grade especificada como um `Array(Nullable(Float64))`. O array retornado contém um valor para cada ponto da grade temporal. O valor é NULL se não houver amostras dentro da janela para calcular o valor de resets de um ponto específico da grade.

**Exemplos**

**Calcule os valores de resets na grade [90, 105, 120, 135, 150, 165, 180, 195, 210, 225]**

```sql title=Query
WITH
    -- NOTA: o intervalo entre 130 e 190 é para mostrar como os valores são preenchidos para ts = 180 de acordo com o parâmetro window
    [110, 120, 130, 190, 200, 210, 220, 230]::Array(DateTime) AS timestamps,
    [1, 3, 2, 6, 6, 4, 2, 0]::Array(Float32) AS values, -- array de valores correspondentes aos timestamps acima
    90 AS start_ts,       -- início da grade de timestamps
    90 + 135 AS end_ts,   -- fim da grade de timestamps
    15 AS step_seconds,   -- passo da grade de timestamps
    45 AS window_seconds  -- janela de "staleness"
SELECT timeSeriesResetsToGrid(start_ts, end_ts, step_seconds, window_seconds)(timestamp, value)
FROM
(
    -- Esta subconsulta converte arrays de timestamps e valores em linhas de `timestamp`, `value`
    SELECT
        arrayJoin(arrayZip(timestamps, values)) AS ts_and_val,
        ts_and_val.1 AS timestamp,
        ts_and_val.2 AS value
);
```

```response title=Response
┌─timeSeriesResetsToGrid(start_ts, end_ts, step_seconds, window_seconds)(timestamp, value)─┐
│ [NULL,NULL,0,1,1,1,NULL,0,1,2]                                                           │
└──────────────────────────────────────────────────────────────────────────────────────────┘
```

**Mesma consulta com argumentos do tipo array**

```sql title=Query
WITH
    [110, 120, 130, 190, 200, 210, 220, 230]::Array(DateTime) AS timestamps,
    [1, 3, 2, 6, 6, 4, 2, 0]::Array(Float32) AS values,
    90 AS start_ts,
    90 + 135 AS end_ts,
    15 AS step_seconds,
    45 AS window_seconds
SELECT timeSeriesResetsToGrid(start_ts, end_ts, step_seconds, window_seconds)(timestamps, values);
```

```response title=Response
┌─timeSeriesResetsToGrid(start_ts, end_ts, step_seconds, window_seconds)(timestamp, value)─┐
│ [NULL,NULL,0,1,1,0,NULL,0,1,2]                                                           │
└──────────────────────────────────────────────────────────────────────────────────────────┘
```
