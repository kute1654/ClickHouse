<div id="sumMapWithOverflow">
  ## sumMapWithOverflow
</div>

Introduzido em: v20.1.0

Soma um array `value` de acordo com as chaves especificadas no array `key`. Retorna uma tupla de dois arrays: chaves em ordem crescente e valores somados para as chaves correspondentes.
Difere da funГ§ГЈo [`sumMap`](/pt-BR/reference/functions/aggregate-functions/sumMap) porque faz a soma com overflow, ou seja, retorna para a soma o mesmo tipo de dado do argumento.

<Note>
  * Passar uma tupla de arrays de chave e valor Г© idГӘntico a passar um array de chaves e um array de valores.
  * O nГәmero de elementos em `key` e `value` deve ser o mesmo em cada linha totalizada.
</Note>

**Sintaxe**

```sql
sumMapWithOverflow(key, value)
sumMapWithOverflow(Tuple(key, value))
```

**Argumentos**

* `key` вҖ” Array de chaves. [`Array`](/pt-BR/reference/data-types/array)
* `value` вҖ” Array de valores. [`Array`](/pt-BR/reference/data-types/array)

**Valor retornado**

Retorna uma tupla de dois arrays: chaves em ordem crescente e valores somados para as chaves correspondentes. [`Tuple(Array, Array)`](/pt-BR/reference/data-types/tuple)

**Exemplos**

**Sintaxe de Array demonstrando o comportamento de overflow**

```sql title=Query
CREATE TABLE sum_map(
    date Date,
    timeslot DateTime,
    statusMap Nested(
        status UInt8,
        requests UInt8
    ),
    statusMapTuple Tuple(Array(Int8), Array(Int8))
) ENGINE = Memory;

INSERT INTO sum_map VALUES
    ('2000-01-01', '2000-01-01 00:00:00', [1, 2, 3], [10, 10, 10], ([1, 2, 3], [10, 10, 10])),
    ('2000-01-01', '2000-01-01 00:00:00', [3, 4, 5], [10, 10, 10], ([3, 4, 5], [10, 10, 10])),
    ('2000-01-01', '2000-01-01 00:01:00', [4, 5, 6], [10, 10, 10], ([4, 5, 6], [10, 10, 10])),
    ('2000-01-01', '2000-01-01 00:01:00', [6, 7, 8], [10, 10, 10], ([6, 7, 8], [10, 10, 10]));

SELECT
    timeslot,
    toTypeName(sumMap(statusMap.status, statusMap.requests)),
    toTypeName(sumMapWithOverflow(statusMap.status, statusMap.requests))
FROM sum_map
GROUP BY timeslot;
```
