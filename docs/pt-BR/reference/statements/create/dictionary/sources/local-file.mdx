A origem de arquivo local carrega os dados do dicionário de um arquivo no sistema de arquivos local. Isso é útil para tabelas de consulta pequenas e estáticas que podem ser armazenadas como arquivos simples em formatos como TSV, CSV ou qualquer outro [formato compatível](/pt-BR/reference/formats/index).

Exemplo de configurações:

<Tabs>
  <Tab title="DDL">
    ```sql
    SOURCE(FILE(path './user_files/os.tsv' format 'TabSeparated'))
    ```
  </Tab>

  <Tab title="Arquivo de configuração">
    ```xml
    <source>
      <file>
        <path>/opt/dictionaries/os.tsv</path>
        <format>TabSeparated</format>
      </file>
    </source>
    ```
  </Tab>
</Tabs>

<br />

Campos de configuração:

| Configuração | Descrição                                                                                                 |
| ------------ | --------------------------------------------------------------------------------------------------------- |
| `path`       | O caminho absoluto do arquivo.                                                                            |
| `format`     | O formato do arquivo. Todos os formatos descritos em [Formats](/pt-BR/reference/formats/index) são compatíveis. |

Quando um dicionário com origem `FILE` é criado por meio de um comando DDL (`CREATE DICTIONARY ...`), o arquivo de origem precisa estar localizado no diretório `user_files` para evitar que usuários do DB acessem arquivos arbitrários no nó do ClickHouse.
