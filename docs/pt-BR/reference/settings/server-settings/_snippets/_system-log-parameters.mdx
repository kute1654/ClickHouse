---
title: Parâmetros de log do sistema
---

As configurações a seguir podem ser definidas por subtags:

| Setting                            | Description                                                                                                                                                                           | Default             | Note                                                                                                                                               |
| ---------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------- |
| `database`                         | Nome do banco de dados.                                                                                                                                                               |                     |                                                                                                                                                    |
| `table`                            | Nome da tabela do sistema.                                                                                                                                                            |                     |                                                                                                                                                    |
| `engine`                           | [Definição do motor MergeTree](/pt-BR/reference/engines/table-engines/mergetree-family/mergetree#table_engine-mergetree-creating-a-table) para uma tabela do sistema.                       |                     | Não pode ser usado se `partition_by` ou `order_by` estiver definido. Se não for especificado, `MergeTree` será selecionado por padrão              |
| `partition_by`                     | [Chave de particionamento personalizada](/pt-BR/reference/engines/table-engines/mergetree-family/custom-partitioning-key) para uma tabela do sistema.                                       |                     | Se `engine` for especificado para a tabela do sistema, o parâmetro `partition_by` deverá ser especificado diretamente dentro de &#39;engine&#39;   |
| `ttl`                              | Especifica o [TTL](/pt-BR/reference/engines/table-engines/mergetree-family/mergetree#table_engine-mergetree-ttl) da tabela.                                                                 |                     | Se `engine` for especificado para a tabela do sistema, o parâmetro `ttl` deverá ser especificado diretamente dentro de &#39;engine&#39;            |
| `order_by`                         | [Chave de ordenação personalizada](/pt-BR/reference/engines/table-engines/mergetree-family/mergetree#order_by) para uma tabela do sistema. Não pode ser usada se `engine` estiver definido. |                     | Se `engine` for especificado para a tabela do sistema, o parâmetro `order_by` deverá ser especificado diretamente dentro de &#39;engine&#39;       |
| `storage_policy`                   | Nome da política de armazenamento a ser usada para a tabela (opcional).                                                                                                               |                     | Se `engine` for especificado para a tabela do sistema, o parâmetro `storage_policy` deverá ser especificado diretamente dentro de &#39;engine&#39; |
| `settings`                         | [Parâmetros adicionais](/pt-BR/reference/engines/table-engines/mergetree-family/mergetree#settings) que controlam o comportamento do MergeTree (opcional).                                  |                     | Se `engine` for especificado para a tabela do sistema, o parâmetro `settings` deverá ser especificado diretamente dentro de &#39;engine&#39;       |
| `flush_interval_milliseconds`      | Intervalo para gravar na tabela os dados do buffer em memória.                                                                                                                        | `7500`              |                                                                                                                                                    |
| `max_size_rows`                    | Tamanho máximo, em linhas, para os logs. Quando a quantidade de logs ainda não gravados atinge o tamanho máximo, os logs são gravados em disco.                                       | `1048576`           |                                                                                                                                                    |
| `reserved_size_rows`               | Tamanho de memória pré-alocado, em linhas, para os logs.                                                                                                                              | `8192`              |                                                                                                                                                    |
| `buffer_size_rows_flush_threshold` | Limite para a quantidade de linhas. Se esse limite for atingido, a gravação dos logs em disco será iniciada em segundo plano.                                                         | `max_size_rows / 2` |                                                                                                                                                    |
| `flush_on_crash`                   | Define se os logs devem ser gravados em disco em caso de falha.                                                                                                                       | `false`             |                                                                                                                                                    |

Além disso, a configuração a seguir, no nível do servidor, controla a política de flush padrão para todas as tabelas de log do sistema:

```xml
<default_system_log_flush_policy>
    <skip_alias_columns>true</skip_alias_columns>
</default_system_log_flush_policy>
```

| Configuração         | Descrição                                                                                                                                    | Padrão  |
| -------------------- | -------------------------------------------------------------------------------------------------------------------------------------------- | ------- |
| `skip_alias_columns` | Quando `true`, as colunas ALIAS são omitidas dos esquemas das tabelas de logs do sistema. Necessário para logs do sistema com suporte do S3. | `false` |
