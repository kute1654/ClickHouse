<div id="description">
  ## Descrição
</div>

Semelhante ao formato [`TabSeparated`](/pt-BR/reference/formats/TabSeparated/TabSeparated), mas gera a saída com um valor no formato `name=value`.
Os nomes são escapados da mesma forma que no formato [`TabSeparated`](/pt-BR/reference/formats/TabSeparated/TabSeparated), e o símbolo `=` também é escapado.

```text
SearchPhrase=   count()=8267016
SearchPhrase=bathroom interior design    count()=2166
SearchPhrase=clickhouse     count()=1655
SearchPhrase=2014 spring fashion    count()=1549
SearchPhrase=freeform photos       count()=1480
SearchPhrase=angelina jolie    count()=1245
SearchPhrase=omsk       count()=1112
SearchPhrase=photos of dog breeds    count()=1091
SearchPhrase=curtain designs        count()=1064
SearchPhrase=baku       count()=1000
```

```sql title="Query"
SELECT * FROM t_null FORMAT TSKV
```

```text title="Response"
x=1    y=\N
```

<Note>
  Quando há um grande número de colunas pequenas, este formato é ineficiente, e geralmente não há motivo para usá-lo.
  Ainda assim, ele não é pior do que o formato [`JSONEachRow`](/pt-BR/reference/formats/JSON/JSONEachRow) em termos de eficiência.
</Note>

Na análise, os valores das diferentes colunas podem estar em qualquer ordem.
É aceitável omitir alguns valores, pois eles são tratados como iguais aos respectivos valores padrão.
Nesse caso, zeros e linhas em branco são usados como valores padrão.
Valores complexos que poderiam ser especificados na tabela não são aceitos como padrão.

A análise permite adicionar um campo extra `tskv` sem o sinal de igual nem um valor. Esse campo é ignorado.

Durante a importação, colunas com nomes desconhecidos serão ignoradas,
se a configuração [`input_format_skip_unknown_fields`](/pt-BR/reference/settings/formats#input_format_skip_unknown_fields) estiver definida como `1`.

[NULL](/pt-BR/reference/syntax) é formatado como `\N`.
