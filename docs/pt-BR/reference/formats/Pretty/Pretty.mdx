<div id="description">
  ## Descrição
</div>

O formato `Pretty` exibe os dados como tabelas em arte Unicode,
usando sequências de escape ANSI para mostrar cores no terminal.
Uma grade completa da tabela é desenhada, e cada linha de dados ocupa duas linhas no terminal.
Cada bloco de resultados é exibido como uma tabela separada.
Isso é necessário para que os blocos possam ser exibidos sem armazenar os resultados em buffer (o buffering seria necessário para pré-calcular a largura visível de todos os valores).

[NULL](/pt-BR/reference/syntax) é exibido como `ᴺᵁᴸᴸ`.

<div id="example-usage">
  ## Exemplo de uso
</div>

Exemplo (para o formato [`PrettyCompact`](/pt-BR/reference/formats/Pretty/PrettyCompact)):

```sql title="Query"
SELECT * FROM t_null
```

```response title="Response"
┌─x─┬────y─┐
│ 1 │ ᴺᵁᴸᴸ │
└───┴──────┘
```

As linhas não são escapadas em nenhum dos formatos `Pretty`. O exemplo a seguir é para o formato [`PrettyCompact`](/pt-BR/reference/formats/Pretty/PrettyCompact):

```sql title="Query"
SELECT 'String with \'quotes\' and \t character' AS Escaping_test
```

```response title="Response"
┌─Escaping_test────────────────────────┐
│ String with 'quotes' and      character │
└──────────────────────────────────────┘
```

Para evitar despejar dados demais no terminal, apenas as primeiras `10.000` linhas são exibidas.
Se o número de linhas for maior ou igual a `10.000`, a mensagem &quot;Foram mostradas as primeiras 10 000&quot; será exibida.

<Note>
  Este formato é apropriado apenas para exibir o resultado de uma consulta, mas não para fazer parsing de dados.
</Note>

O formato Pretty oferece suporte à exibição de totais (ao usar `WITH TOTALS`) e de extremos (quando &#39;extremes&#39; está definido como 1).
Nesses casos, os totais e os valores extremos são exibidos após os dados principais, em tabelas separadas.
Isso é mostrado no exemplo a seguir, que usa o formato [`PrettyCompact`](/pt-BR/reference/formats/Pretty/PrettyCompact):

```sql title="Query"
SELECT EventDate, count() AS c 
FROM test.hits 
GROUP BY EventDate 
WITH TOTALS 
ORDER BY EventDate 
FORMAT PrettyCompact
```

```response title="Response"
┌──EventDate─┬───────c─┐
│ 2014-03-17 │ 1406958 │
│ 2014-03-18 │ 1383658 │
│ 2014-03-19 │ 1405797 │
│ 2014-03-20 │ 1353623 │
│ 2014-03-21 │ 1245779 │
│ 2014-03-22 │ 1031592 │
│ 2014-03-23 │ 1046491 │
└────────────┴─────────┘

Totais:
┌──EventDate─┬───────c─┐
│ 1970-01-01 │ 8873898 │
└────────────┴─────────┘

Extremos:
┌──EventDate─┬───────c─┐
│ 2014-03-17 │ 1031592 │
│ 2014-03-23 │ 1406958 │
└────────────┴─────────┘
```
