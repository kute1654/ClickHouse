<div id="description">
  ## Descrição
</div>

Também imprime duas linhas de cabeçalho com nomes de colunas e tipos, semelhante a [TabSeparatedWithNamesAndTypes](/pt-BR/reference/formats/TabSeparated/TabSeparatedWithNamesAndTypes).

<div id="example-usage">
  ## Exemplo de uso
</div>

<div id="inserting-data">
  ### Inserção de dados
</div>

<Tip>
  A partir da [versão](https://github.com/ClickHouse/ClickHouse/releases) 23.1, o ClickHouse detecta automaticamente cabeçalhos em arquivos CSV ao usar o formato `CSV`, portanto não é necessário usar `CSVWithNames` nem `CSVWithNamesAndTypes`.
</Tip>

Crie uma tabela:

```sql
CREATE TABLE football
(
    `date` Date,
    `season` Int16,
    `home_team` LowCardinality(String),
    `away_team` LowCardinality(String),
    `home_team_goals` Int8,
    `away_team_goals` Int8
)
ENGINE = MergeTree
ORDER BY (date, home_team);
```

Insira dados no formato `CSVWithNamesAndTypes`:

```sql
INSERT INTO football FROM INFILE 'football_types.csv' FORMAT CSVWithNamesAndTypes;
```

<div id="reading-data">
  ### Leitura de dados
</div>

Leia os dados usando o formato `CSVWithNamesAndTypes`:

```sql
SELECT *
FROM football
FORMAT CSVWithNamesAndTypes
```

<div id="format-settings">
  ## Configurações de formato
</div>

<Note>
  Se a configuração [input&#95;format&#95;with&#95;names&#95;use&#95;header](/pt-BR/reference/settings/formats#input_format_with_names_use_header) estiver definida como `1`,
  as colunas dos dados de entrada serão mapeadas para as colunas da tabela com base em seus nomes, e colunas com nomes desconhecidos serão ignoradas se a configuração [input&#95;format&#95;skip&#95;unknown&#95;fields](/pt-BR/reference/settings/formats#input_format_skip_unknown_fields) estiver definida como `1`.
  Caso contrário, a primeira linha será ignorada.
</Note>

<Note>
  Se a configuração [input&#95;format&#95;with&#95;types&#95;use&#95;header](/pt-BR/reference/settings/formats#input_format_with_types_use_header) estiver definida como `1`,
  os tipos dos dados de entrada serão comparados com os tipos das colunas correspondentes da tabela. Caso contrário, a segunda linha será ignorada.
</Note>
