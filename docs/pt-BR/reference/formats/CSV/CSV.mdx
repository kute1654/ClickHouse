<div id="description">
  ## Descrição
</div>

Formato Comma Separated Values ([RFC](https://tools.ietf.org/html/rfc4180)).
Na formatação, as linhas são colocadas entre aspas duplas. Aspas duplas dentro de uma string são geradas como duas aspas duplas consecutivas.
Não há outras regras de escape de caracteres.

* Date e date-time são colocados entre aspas duplas.
* Números são gerados sem aspas.
* Os valores são separados por um caractere delimitador, que por padrão é `,`. O caractere delimitador é definido na configuração [format&#95;csv&#95;delimiter](/pt-BR/reference/settings/formats#format_csv_delimiter).
* As linhas são separadas usando o line feed (LF) do Unix.
* Arrays são serializados em CSV da seguinte forma:
  * primeiro, o array é serializado como uma string, como no formato TabSeparated
  * A string resultante é gerada em CSV entre aspas duplas.
* Tuples no formato CSV são serializados como colunas separadas (ou seja, seu aninhamento na tupla é perdido).

```bash
$ clickhouse-client --format_csv_delimiter="|" --query="INSERT INTO test.csv FORMAT CSV" < data.csv
```

<Note>
  Por padrão, o delimitador é `,`
  Consulte a configuração [format&#95;csv&#95;delimiter](/pt-BR/reference/settings/formats#format_csv_delimiter) para mais informações.
</Note>

Durante o parsing, todos os valores podem ser interpretados com ou sem aspas. Há suporte a aspas duplas e simples.

As linhas também podem ser fornecidas sem aspas. Nesse caso, elas são analisadas até o caractere delimitador ou a quebra de linha (CR ou LF).
No entanto, em desacordo com a RFC, ao analisar linhas sem aspas, os espaços e tabulações no início e no fim são ignorados.
Há suporte aos tipos de quebra de linha: Unix (LF), Windows (CR LF) e Mac OS Classic (CR LF).

`NULL` é formatado de acordo com a configuração [format&#95;csv&#95;null&#95;representation](/pt-BR/reference/settings/formats#format_csv_null_representation) (o valor padrão é `\N`).

Nos dados de entrada, os valores `ENUM` podem ser representados como nomes ou como IDs.
Primeiro, tentamos corresponder o valor de entrada ao nome do `ENUM`.
Se isso falhar e o valor de entrada for um número, tentamos corresponder esse número ao ID do `ENUM`.
Se os dados de entrada contiverem apenas IDs de `ENUM`, é recomendável habilitar a configuração [input&#95;format&#95;csv&#95;enum&#95;as&#95;number](/pt-BR/reference/settings/formats#input_format_csv_enum_as_number) para otimizar o parsing de `ENUM`.
