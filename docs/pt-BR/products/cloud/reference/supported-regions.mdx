<div id="private-regions">
  ## Regiões privadas
</div>

<EnterprisePlanFeatureBadge feature="Recurso de regiões privadas" />

Oferecemos regiões privadas para nossos serviços da tier Enterprise. Entre em contato conosco [aqui](https://clickhouse.com/company/contact) para solicitar uma região privada.

Principais considerações sobre regiões privadas:

* Os serviços não têm escalonamento automático; no entanto, há suporte a escalonamento vertical e horizontal manual.
* Os serviços não podem ser inativados.
* A página de status não está disponível para regiões privadas.

Requisitos adicionais podem se aplicar para conformidade com a HIPAA (incluindo a assinatura de um BAA). Observe que, no momento, a HIPAA está disponível apenas para serviços da tier Enterprise

<div id="hipaa-compliant-regions">
  ## Regiões compatíveis com HIPAA
</div>

<EnterprisePlanFeatureBadge feature="HIPAA" support="true" />

Os clientes devem assinar um Business Associate Agreement (BAA) e solicitar o onboarding por meio da equipe de Vendas ou do Suporte para configurar serviços em regiões compatíveis com HIPAA. As regiões a seguir oferecem suporte à conformidade com HIPAA:

* AWS af-south-1 (África do Sul) **Região Privada**
* AWS ca-central-1 (Canadá) **Região Privada**
* AWS eu-central-1 (Frankfurt)
* AWS eu-north-1 (Estocolmo) **Região Privada**
* AWS eu-west-1 (Irlanda)
* AWS eu-west-2 (Londres)
* AWS sa-east-1 (América do Sul) **Região Privada**
* AWS us-east-1 (Virgínia do Norte)
* AWS us-east-2 (Ohio)
* AWS us-west-2 (Oregon)
* GCP europe-west4 (Países Baixos)
* GCP us-central1 (Iowa)
* GCP us-east1 (Carolina do Sul)
