Redefina as senhas de usuários do ClickHouse usando a API privada.

<div id="prerequisites">
  ## Pré-requisitos
</div>

* ClickHouse API privada instalada e acessível (por exemplo, via redirecionamento de porta para `http://localhost:8080/`)
* Pelo menos um cluster do ClickHouse implantado (este guia usa `default-xx-01`)

<Steps>
  <Step>
    ## 1. Gerar um hash da senha

    Gere o hash da nova senha usando SHA-256 e codifique o resultado em base64:

    ```ini
    PASSWORD='My super secret p@$$w0rd'
    HASHED_PASSWORD=$(echo -n "$PASSWORD" | shasum -a 256 | awk '{printf $1}' | base64)
    ```
  </Step>

  <Step>
    ## 2. Criar o corpo da requisição

    Crie um arquivo JSON com o hash da senha:

    ```json
    {
      "user_hashed_password": "<base64-encoded-sha256-hash>",
      "hashing_function": "sha256",
      "username": "default"
    }
    ```

    Salve-o como `password_reset.json`.

    | Campo                  | Obrigatório | Padrão    | Descrição                                  |
    | ---------------------- | ----------- | --------- | ------------------------------------------ |
    | `user_hashed_password` | Sim         | --        | Hash da nova senha codificado em base64    |
    | `hashing_function`     | Não         | `sha256`  | Algoritmo de hash usado                    |
    | `username`             | Não         | `default` | Nome de usuário cuja senha será redefinida |
  </Step>

  <Step>
    ## 3. Enviar a requisição

    ```bash
    curl -X POST "http://localhost:8080/api/v1/instances/default-xx-01/reset-user-password" \
        -H "Content-Type: application/json" \
        --data-binary "@password_reset.json"
    ```

    Uma resposta bem-sucedida:

    ```json
    {
      "message": "User password reset successfully"
    }
    ```
  </Step>

  <Step>
    ## 4. Verificar a alteração

    A redefinição da senha entra em vigor depois que o operator processa a alteração. Verifique conectando-se ao cluster com a nova senha.

    Você também pode monitorar a operação pelo endpoint de status:

    ```bash
    curl "http://localhost:8080/api/v1/instances/default-xx-01/status"
    ```
  </Step>
</Steps>

<div id="hydra-limitation">
  ## Limitação do Hydra
</div>

Não é possível redefinir senhas em instâncias filhas do Hydra. Para redefinir a senha de uma instância filha do Hydra, redefina-a na instância pai.
