O ClickHouse Cloud oferece diferentes canais de lançamento para atender às diferentes necessidades dos usuários em relação à estabilidade, ao acesso a novos recursos e à previsibilidade das atualizações. Cada canal tem um cronograma de atualização próprio e foi pensado para diferentes casos de uso — tanto para usuários que querem acesso imediato a novas versões quanto para aqueles que preferem adiar atualizações para garantir que recebam a versão mais estável.

<div id="release-channel-details">
  ## Detalhes dos canais de lançamento
</div>

<Accordion title="Saiba mais sobre os canais de lançamento">
  | Nome do canal            | Descrição                                                                                                                                                                                                                                               | Principais considerações                                                                                                                         | Tiers compatíveis                            |
  | :----------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | :----------------------------------------------------------------------------------------------------------------------------------------------- | :------------------------------------------- |
  | **Fast (Early Release)** | Recomendado para ambientes não produtivos. Este é o primeiro canal de lançamento para cada upgrade de versão do banco de dados                                                                                                                          | Acesso a novos recursos em troca de estabilidade.<br />Possibilidade de testar releases em ambientes não produtivos antes do upgrade em produção | Basic (padrão)<br />tiers Scale e Enterprise |
  | **Regular**              | Canal de lançamento padrão para todos os serviços com várias réplicas.<br />O rollout nesse canal normalmente começa duas semanas após o início do canal de lançamento Fast.                                                                            | Upgrades padrão/em toda a frota.<br />Os serviços são atualizados gradualmente ao longo de várias semanas                                        | Scale e Enterprise                           |
  | **Slow (Deferred)**      | Recomendado para usuários mais avessos a riscos que desejam que seus serviços sejam atualizados mais perto do fim do cronograma de lançamento.<br />O rollout nesse canal normalmente começa duas semanas após o início do canal de lançamento Regular. | Máxima estabilidade e previsibilidade.<br />Indicado para quem precisa de mais testes das novas releases nos canais Fast/Regular                 | Enterprise                                   |

  <br />

  <br />

  <Note>
    Todos os serviços com réplica única são inscritos automaticamente no canal de lançamento Fast.
  </Note>
</Accordion>

Janelas de upgrade programadas estão disponíveis em todos os canais de lançamento para serviços no tier Enterprise. Esse recurso permite configurar uma janela de horário em um determinado dia da semana para upgrades.

<div id="release-schedule">
  ## Cronograma de lançamento
</div>

<Warning>
  **Entendendo as datas de lançamento**

  As datas mostradas abaixo indicam quando o ClickHouse **inicia o rollout** em cada canal de lançamento, não quando o seu serviço específico será atualizado.

  * Os rollouts são automatizados e ocorrem gradualmente ao longo de várias semanas
  * Serviços com janelas de upgrade programadas são atualizados durante a janela agendada na semana seguinte ao fim do rollout do canal (a partir de segunda-feira às 00:00 UTC)
  * A conclusão do rollout pode ser adiada devido a pausas no rollout (por exemplo, congelamentos em feriados) ou ao monitoramento de integridade

  Para testes antecipados antes dos upgrades em produção, use o canal Fast ou Regular para serviços que não sejam de produção e o canal Slow para serviços de produção.
</Warning>
