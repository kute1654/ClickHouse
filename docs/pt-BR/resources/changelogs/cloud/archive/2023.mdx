<div id="december-18-2023">
  ## 18 de dezembro de 2023
</div>

Este lançamento traz uma nova região no GCP (us-east1), a capacidade de configurar, em autoatendimento, conexões seguras com endpoints, suporte a integrações adicionais, incluindo o DBT 1.7, além de inúmeras correções de bugs e melhorias de segurança.

<div id="general-changes-1">
  ### Alterações gerais
</div>

* O ClickHouse Cloud agora está disponível na região us-east1 (South Carolina) do GCP
* Habilitada a configuração do AWS Private Link e do GCP Private Service Connect via OpenAPI

<div id="console-changes-7">
  ### alteração no Console
</div>

* Login sem interrupções no SQL Console habilitado para quem tem a função Developer
* Fluxo de trabalho simplificado para configurar controles de idling durante o onboarding

<div id="integrations-changes-7">
  ### alteração na integração
</div>

* Conector DBT: adicionado suporte ao DBT até a v1.7
* Metabase: adicionado suporte ao Metabase v0.48
* Conector PowerBI: adicionado suporte para execução no PowerBI Cloud
* Permitir configurar as permissões do usuário interno do ClickPipes
* Kafka Connect
  * Lógica de desduplicação e ingestão de tipos Nullable aprimorada.
  * Adicionado suporte a formatos baseados em texto (CSV, TSV)
* Apache Beam: adicionado suporte aos tipos Boolean e LowCardinality
* cliente Nodejs: adicionado suporte ao formato Parquet

<div id="security-announcements">
  ### Comunicados de segurança
</div>

* Corrigidas 3 vulnerabilidades de segurança - veja o [changelog de segurança](/pt-BR/resources/changelogs/security-changelog) para mais detalhes:
  * CVE 2023-47118 (CVSS 7.0) - uma vulnerabilidade de heap buffer overflow que afeta a interface nativa executada por padrão na porta 9000/tcp
  * CVE-2023-48704 (CVSS 7.0) - uma vulnerabilidade de heap buffer overflow que afeta a interface nativa executada por padrão na porta 9000/tcp
  * CVE 2023-48298 (CVSS 5.9) - uma vulnerabilidade de underflow de inteiro no codec de compressão FPC

<div id="november-22-2023">
  ## 22 de novembro de 2023
</div>

Este lançamento atualiza a versão principal do banco de dados, melhora o fluxo de login e autenticação e adiciona suporte a proxy no Kafka Connect Sink.

<div id="clickhouse-version-upgrade-2">
  ### Atualização da versão do ClickHouse
</div>

* O desempenho na leitura de arquivos Parquet foi drasticamente melhorado. Veja o [blog de lançamento 23.8](https://clickhouse.com/blog/clickhouse-release-23-08) para mais detalhes.
* Foi adicionado suporte à inferência de tipos para JSON. Veja o [blog de lançamento 23.9](https://clickhouse.com/blog/clickhouse-release-23-09) para mais detalhes.
* Foram introduzidas funções poderosas voltadas para analistas, como `ArrayFold`. Veja o [blog de lançamento 23.10](https://clickhouse.com/blog/clickhouse-release-23-10) para mais detalhes.
* **Mudança incompatível com versões anteriores visível ao usuário**: a configuração `input_format_json_try_infer_numbers_from_strings` foi desabilitada por padrão para evitar a inferência de números a partir de strings no formato JSON. Isso pode causar possíveis erros de parsing quando os dados de amostra contêm strings semelhantes a números.
* Dezenas de novos recursos, melhorias de desempenho e correções de bugs. Veja os [changelogs do banco de dados principal](/pt-BR/resources/changelogs/oss/2026) para mais detalhes.

<div id="console-changes-8">
  ### Mudanças no Console
</div>

* Fluxo de login e autenticação aprimorado.
* Sugestões de consulta baseadas em IA aprimoradas para oferecer melhor suporte a esquemas extensos.

<div id="integrations-changes-8">
  ### Alterações nas integrações
</div>

* Kafka Connect Sink: Adicionado suporte a proxy, ao mapeamento `topic-tablename` e à configuração das propriedades de entrega *exactly-once* do Keeper.
* Cliente Node.js: Adicionado suporte ao formato Parquet.
* Metabase: Adicionado suporte à função `datetimeDiff`.
* Cliente Python: Adicionado suporte a caracteres especiais em nomes de colunas. Corrigida a associação do parâmetro de fuso horário.

</div>

* Os serviços Development agora estão disponíveis na AWS nas regiões `ap-south-1` (Mumbai) e `ap-southeast-1` (Singapura)

<div id="console-changes-9">
  ### Alterações no Console
</div>

* Adicionada a possibilidade de configurar opções fiscais detalhadas ao adicionar um cartão de crédito

<div id="integrations-changes-9">
  ### Mudanças nas integrações
</div>

* MySQL
  * Aprimorado o suporte ao Tableau Online e ao QuickSight via MySQL
* Kafka Connector
  * Introduzido um novo StringConverter para oferecer suporte a formatos baseados em texto (CSV, TSV)
  * Adicionado suporte aos tipos de dados Bytes e Decimal
  * Ajustadas as Retryable Exceptions para que agora sejam sempre tentadas novamente (mesmo quando errors.tolerance=all)
* Node.js client
  * Corrigido um problema em que grandes datasets em streaming produziam resultados corrompidos
* Python client
  * Corrigidos timeouts em inserts grandes
  * Corrigido um problema de Date32 no NumPy/Pandas
    ​​- Golang client
  * Corrigida a inserção de um map vazio em coluna JSON, limpeza do buffer de compressão, escape de consulta e panic em zero/nil para IPv4 e IPv6
  * Adicionado watchdog para inserts cancelados
* DBT
  * Aprimorado o suporte a tabela distribuída com testes

* Foram corrigidos um problema de conexão SSL e os mapeamentos de tipos entre o [PowerBI](/pt-BR/integrations/connectors/data-visualization/powerbi-and-clickhouse) e os tipos de dados do ClickHouse

<div id="july-27-2023">

Este lançamento traz a prévia privada do ClickPipes for Kafka, uma nova experiência para carregamento de dados e a possibilidade de carregar um arquivo a partir de uma URL pelo Cloud Console.

Inscreva-se na lista de espera [aqui](https://clickhouse.com/cloud/clickpipes#joinwaitlist).

* cliente Python: corrigidos problemas na compressão de insert e na leitura de LowCardinality
