O ClickHouse oferece suporte experimental a s390x.

<div id="building-clickhouse-for-s390x">
  ## Compilando o ClickHouse para s390x
</div>

Assim como outras plataformas, o s390x compila o OpenSSL como uma biblioteca estática. Se você quiser compilar com OpenSSL dinâmico, precisará passar `-DENABLE_OPENSSL_DYNAMIC=1` ao CMake.

Estas instruções pressupõem que a máquina host seja Linux x86&#95;64/ARM e tenha todas as ferramentas necessárias para compilação nativa, com base nas [instruções de compilação](/pt-BR/resources/develop-contribute/build/build). Também pressupõem que o host seja Ubuntu 22.04, mas as instruções a seguir também devem funcionar no Ubuntu 20.04.

Além de instalar as ferramentas usadas na compilação nativa, os pacotes adicionais a seguir precisam ser instalados:

```bash
apt-get mold
rustup target add s390x-unknown-linux-gnu
```

Para compilar em s390x:

```bash
cmake -DCMAKE_TOOLCHAIN_FILE=cmake/linux/toolchain-s390x.cmake ..
ninja
```

<div id="running">
  ## Executando
</div>

Para emular, você precisará do binário estático do QEMU para modo usuário em s390x. No Ubuntu, ele pode ser instalado com:

```bash
apt-get install binfmt-support binutils-s390x-linux-gnu qemu-user-static
```

Depois de compilado, o binário pode ser executado, por exemplo, com:

```bash
qemu-s390x-static -L /usr/s390x-linux-gnu ./programs/clickhouse local --query "Select 2"
2
```

<div id="debugging">
  ## Depuração
</div>

Instale o LLDB:

```bash
apt-get install lldb-21
```

Para depurar um executável s390x, execute o ClickHouse usando o QEMU em modo de depuração:

```bash
qemu-s390x-static -g 31338 -L /usr/s390x-linux-gnu ./clickhouse
```

<div id="visual-studio-code-integration">
  ## Integração com o Visual Studio Code
</div>

* A extensão [CodeLLDB](https://github.com/vadimcn/vscode-lldb) é necessária para a depuração visual.
* A extensão [Command Variable](https://github.com/rioj7/command-variable) pode ajudar com inicializações dinâmicas ao usar [CMake Variants](https://github.com/microsoft/vscode-cmake-tools/blob/main/docs/variants.md).
* Certifique-se de definir o backend para a sua instalação do LLVM, por exemplo: `"lldb.library": "/usr/lib/x86_64-linux-gnu/liblldb-21.so"`
* Certifique-se de executar o executável `clickhouse` em modo de depuração antes de iniciar. (Também é possível criar uma `preLaunchTask` para automatizar isso)

<div id="example-configurations">
  ### Configurações de exemplo
</div>

<div id="cmake-variantsyaml">
  #### cmake-variants.yaml
</div>

```yaml
buildType:
  default: relwithdebinfo
  choices:
    debug:
      short: Debug
      long: Emit debug information
      buildType: Debug
    release:
      short: Release
      long: Optimize generated code
      buildType: Release
    relwithdebinfo:
      short: RelWithDebInfo
      long: Release with Debug Info
      buildType: RelWithDebInfo
    tsan:
      short: MinSizeRel
      long: Minimum Size Release
      buildType: MinSizeRel

toolchain:
  default: default
  description: Select toolchain
  choices:
    default:
      short: x86_64
      long: x86_64
    s390x:
      short: s390x
      long: s390x
      settings:
        CMAKE_TOOLCHAIN_FILE: cmake/linux/toolchain-s390x.cmake
```

<div id="launchjson">
  #### launch.json
</div>

```json
{
    "version": "0.2.0",
    "configurations": [
        {
            "type": "lldb",
            "request": "custom",
            "name": "(lldb) Launch s390x with qemu",
            "targetCreateCommands": ["target create ${command:cmake.launchTargetPath}"],
            "processCreateCommands": ["gdb-remote 2159"],
            "preLaunchTask": "Run ClickHouse"
        }
    ]
}
```

<div id="settingsjson">
  #### settings.json
</div>

Isso também colocaria builds diferentes em subpastas distintas da pasta `build`.

```json
{
    "cmake.buildDirectory": "${workspaceFolder}/build/${buildKitVendor}-${buildKitVersion}-${variant:toolchain}-${variant:buildType}",
    "lldb.library": "/usr/lib/x86_64-linux-gnu/liblldb-21.so"
}
```

<div id="run-debugsh">
  #### run-debug.sh
</div>

```sh
#! /bin/sh
echo 'Starting debugger session'
cd $1
qemu-s390x-static -g 2159 -L /usr/s390x-linux-gnu $2 $3 $4
```

<div id="tasksjson">
  #### tasks.json
</div>

Define uma tarefa para executar o executável compilado no modo `server`, em uma pasta `tmp` ao lado dos binários, com a configuração em `programs/server/config.xml`.

```json
{
    "version": "2.0.0",
    "tasks": [
        {
            "label": "Run ClickHouse",
            "type": "shell",
            "isBackground": true,
            "command": "${workspaceFolder}/.vscode/run-debug.sh",
            "args": [
                "${command:cmake.launchTargetDirectory}/tmp",
                "${command:cmake.launchTargetPath}",
                "server",
                "--config-file=${workspaceFolder}/programs/server/config.xml"
            ],
            "problemMatcher": [
                {
                    "pattern": [
                        {
                            "regexp": ".",
                            "file": 1,
                            "location": 2,
                            "message": 3
                        }
                    ],
                    "background": {
                        "activeOnStart": true,
                        "beginsPattern": "^Starting debugger session",
                        "endsPattern": ".*"
                    }
                }
            ]
        }
    ]
}
```
