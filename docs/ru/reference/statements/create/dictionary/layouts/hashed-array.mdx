<div id="hashed_array">
  ## hashed_array
</div>

Словарь полностью хранится в памяти. Каждый атрибут хранится в массиве. Ключевой атрибут хранится в виде хеш-таблицы, где значением служит индекс в массиве атрибутов. Словарь может содержать любое количество элементов с любыми идентификаторами. На практике количество ключей может достигать десятков миллионов.

Ключ словаря имеет тип [UInt64](/ru/reference/data-types/int-uint).

Поддерживаются все типы источников. При обновлении данные (из файла или таблицы) считываются целиком.

Пример конфигурации:

<Tabs>
  <Tab title="DDL">
    ```sql
    LAYOUT(HASHED_ARRAY([SHARDS 1]))
    ```
  </Tab>

  <Tab title="файл конфигурации">
    ```xml
    <layout>
      <hashed_array>
      </hashed_array>
    </layout>
    ```
  </Tab>
</Tabs>

<br />

<div id="complex_key_hashed_array">
  ## complex_key_hashed_array
</div>

Этот тип хранилища предназначен для работы с составными [ключами](/ru/reference/statements/create/dictionary/attributes#composite-key). Аналогичен [hashed&#95;array](#hashed_array).

Пример конфигурации:

<Tabs>
  <Tab title="DDL">
    ```sql
    LAYOUT(COMPLEX_KEY_HASHED_ARRAY([SHARDS 1]))
    ```
  </Tab>

  <Tab title="Файл конфигурации">
    ```xml
    <layout>
      <complex_key_hashed_array />
    </layout>
    ```
  </Tab>
</Tabs>
