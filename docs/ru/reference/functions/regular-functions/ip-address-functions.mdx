<div id="IPv4CIDRToRange">
  ## IPv4CIDRToRange
</div>

Добавленный в: v20.1.0

Принимает IPv4-адрес и длину его префикса Classless Inter-Domain Routing (CIDR), а возвращает диапазон адресов подсети в виде кортежа из двух значений IPv4: первого и последнего адресов этой подсети.
Версию для IPv6 см. в [`IPv6CIDRToRange`](#IPv4CIDRToRange).

**Синтаксис**

```sql
IPv4CIDRToRange(ipv4, cidr)
```

**Аргументы**

* `ipv4` — адрес IPv4. [`IPv4`](/ru/reference/data-types/ipv4) или [`String`](/ru/reference/data-types/string)
* `cidr` — значение CIDR. [`UInt8`](/ru/reference/data-types/int-uint)

**Возвращаемое значение**

Возвращает кортеж из двух адресов IPv4, представляющих диапазон подсети. [`Tuple(IPv4, IPv4)`](/ru/reference/data-types/tuple)

**Примеры**

**Пример использования**

```sql title=Query
SELECT IPv4CIDRToRange(toIPv4('192.168.5.2'), 16);
```

```response title=Response
┌─IPv4CIDRToRange(toIPv4('192.168.5.2'), 16)─┐
│ ('192.168.0.0','192.168.255.255')          │
└────────────────────────────────────────────┘
```

<div id="IPv4NumToString">
  ## IPv4NumToString
</div>

Добавленный в: v1.1.0

Преобразует 32-битное целое число в строковое представление IPv4-адреса в точечно-десятичной записи (формат A.B.C.D).
Интерпретирует входное значение в порядке байтов big-endian.

**Синтаксис**

```sql
IPv4NumToString(num)
```

**Псевдонимы**: `INET_NTOA`

**Аргументы**

* `num` — IPv4-адрес в виде числа UInt32. [`UInt32`](/ru/reference/data-types/int-uint)

**Возвращаемое значение**

Возвращает число, представляющее MAC-адрес, или `0`, если формат некорректен. [`String`](/ru/reference/data-types/string)

**Примеры**

**Пример использования**

```sql title=Query
IPv4NumToString(3232235521)
```

```response title=Response
192.168.0.1
```

<div id="IPv6StringToNum">

HEX может быть записан в верхнем или нижнем регистре.

```sql

```

Возвращает IPv6-адрес в двоичном формате. [`FixedString(16)`](/ru/reference/data-types/fixedstring)

**Примеры**

**Базовый пример**

```sql title=Query
SELECT addr, cutIPv6(IPv6StringToNum(addr), 0, 0) FROM (SELECT ['notaddress', '127.0.0.1', '1111::ffff'] AS addr) ARRAY JOIN addr;
```

```response title=Response
┌─addr───────┬─cutIPv6(IPv6StringToNum(addr), 0, 0)─┐
```

<div id="isIPAddressInRange">
  ## isIPAddressInRange
</div>

Добавленный в: v21.4.0

Определяет, принадлежит ли IP-адрес сети, заданной в нотации [Classless Inter-Domain Routing (CIDR)](https://en.wikipedia.org/wiki/Classless_Inter-Domain_Routing).

Эта функция принимает как IPv4-, так и IPv6-адреса (и сети), представленные в виде строк. Она возвращает `0`, если IP-версия адреса и CIDR не совпадают.

**Синтаксис**

```sql
isIPAddressInRange(address, prefix)
```

**Аргументы**

* `address` — IPv4- или IPv6-адрес. [`String`](/ru/reference/data-types/string)
* `prefix` — Сетевой префикс IPv4 или IPv6 в нотации CIDR. [`String`](/ru/reference/data-types/string)

**Возвращаемое значение**

Возвращает `1`, если IP-версия адреса и CIDR совпадают, в противном случае — `0`. [`UInt8`](/ru/reference/data-types/int-uint)

**Примеры**

**IPv4-адрес в диапазоне**

```sql title=Query
SELECT isIPAddressInRange('127.0.0.1', '127.0.0.0/8')
```

```response title=Response
1
```

**IPv4-адрес не входит в диапазон**

```sql title=Query
SELECT isIPAddressInRange('127.0.0.1', 'ffff::/16')
```

```response title=Response
0
```

**IPv6-адрес не входит в диапазон**

```sql title=Query
SELECT isIPAddressInRange('::ffff:192.168.0.1', '::ffff:192.168.0.4/128')
```

```response title=Response
0
```
