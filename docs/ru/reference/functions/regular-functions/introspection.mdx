Вы можете использовать функции, описанные в этой главе, для интроспекции [ELF](https://en.wikipedia.org/wiki/Executable_and_Linkable_Format) и [DWARF](https://en.wikipedia.org/wiki/DWARF) при профилировании запросов.

<Note>
  Эти функции работают медленно и могут создавать риски для безопасности.
</Note>

Для корректной работы функций интроспекции:

* Установите пакет `clickhouse-common-static-dbg`.

* Установите значение настройки [allow&#95;introspection&#95;functions](/ru/reference/settings/session-settings#allow_introspection_functions) равным 1.

  По соображениям безопасности функции интроспекции по умолчанию отключены.

ClickHouse сохраняет отчёты профилировщика в системную таблицу [trace&#95;log](/ru/reference/system-tables/trace_log). Убедитесь, что таблица и профилировщик настроены правильно.

<div id="addressToLine">
  ## addressToLine
</div>

Добавленный в: v20.1.0

Преобразует виртуальный адрес памяти в процессе сервера ClickHouse в имя файла и номер строки в исходном коде ClickHouse.

<Note>
  Эта функция работает медленно и может иметь последствия для безопасности.
</Note>

Чтобы включить эту функцию интроспекции:

* Установите пакет `clickhouse-common-static-dbg`.
* Установите значение настройки [`allow_introspection_functions`](/ru/reference/settings/session-settings#allow_introspection_functions) равным `1`.

**Синтаксис**

```sql
addressToLine(address_of_binary_instruction)
```

**Аргументы**

* `address_of_binary_instruction` — Адрес инструкции в выполняющемся процессе. [`UInt64`](/ru/reference/data-types/int-uint)

**Возвращаемое значение**

Возвращает имя файла исходного кода и номер строки, разделённые двоеточием, например `/build/obj-x86_64-linux-gnu/../src/Common/ThreadPool.cpp:199`. Если отладочная информация не найдена, возвращает имя бинарного файла; если адрес недопустим — пустую строку. [`String`](/ru/reference/data-types/string)

**Примеры**

**Выбор первой строки из системной таблицы `trace_log`**

```sql title=Query
SET allow_introspection_functions=1;
SELECT * FROM system.trace_log LIMIT 1 \G;
```

**Получение имени файла исходного кода и номера строки по одному адресу**

<div id="addressToLineWithInlines">
  ## addressToLineWithInlines
</div>

Добавленный в: v22.2.0

Похожа на `addressToLine`, но возвращает Array со всеми инлайн-функциями.
Поэтому она работает медленнее, чем `addressToLine`.

Чтобы включить эту функцию интроспекции:

* Установите пакет `clickhouse-common-static-dbg`.
* Установите значение `1` для настройки [`allow_introspection_functions`](/ru/reference/settings/session-settings#allow_introspection_functions).

**Синтаксис**

```sql
addressToLineWithInlines(address_of_binary_instruction)
```

**Аргументы**

* `address_of_binary_instruction` — Адрес инструкции в выполняющемся процессе. [`UInt64`](/ru/reference/data-types/int-uint)

**Возвращаемое значение**

Возвращает массив, первый элемент которого содержит имя файла исходного кода и номер строки, разделённые двоеточием. Второй, третий и последующие элементы содержат имена файлов исходного кода, номера строк и имена встроенных inline-функций. Если отладочную информацию найти не удалось, возвращается массив с единственным элементом — именем бинарного файла; если адрес недопустим, возвращается пустой массив. [`Array(String)`](/ru/reference/data-types/array)

**Примеры**

**Применение функции к адресу**

```sql title=Query
SET allow_introspection_functions=1;
```

<div id="isMergeTreePartCoveredBy">
  ## isMergeTreePartCoveredBy
</div>

Добавленный в: v25.6.0

Функция, которая проверяет, покрывается ли часть из первого аргумента частью из второго аргумента.

**Синтаксис**

```sql
isMergeTreePartCoveredBy(nested_part, covering_part)
```

**Базовый пример**

```sql title=Query
WITH 'all_12_25_7_4' AS lhs, 'all_7_100_10_20' AS rhs
SELECT isMergeTreePartCoveredBy(rhs, lhs), isMergeTreePartCoveredBy(lhs, rhs);
```

<div id="logTrace">
  ## logTrace
</div>

Добавленный в: v20.12.0

Выводит в журнал сервера сообщение трассировки для каждого [блока](/ru/resources/develop-contribute/introduction/architecture#block).

**Синтаксис**

```sql
logTrace(message)
```

**Аргументы**

* `message` — Сообщение, которое выводится в журнал сервера. [`const String`](/ru/reference/data-types/string)

**Возвращаемое значение**

Всегда возвращает `0`. [`UInt8`](/ru/reference/data-types/int-uint)

**Примеры**

**Базовый пример**

