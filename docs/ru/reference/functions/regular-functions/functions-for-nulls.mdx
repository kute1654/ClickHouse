<div id="assumeNotNull">
  ## assumeNotNull
</div>

Добавленный в: v1.1.0

Возвращает соответствующее значение без `Nullable` для значения типа [`Nullable`](/ru/reference/data-types/nullable).
Если исходное значение — `NULL`, может быть возвращён произвольный результат.

См. также: функции [`ifNull`](#ifNull) и [`coalesce`](#coalesce).

**Синтаксис**

```sql
assumeNotNull(x)
```

**Аргументы**

* `x` — Исходное значение любого типа `Nullable`. [`Nullable(T)`](/ru/reference/data-types/nullable)

**Возвращаемое значение**

Возвращает значение типа без `Nullable`, если исходное значение не равно `NULL`; в противном случае, если входное значение равно `NULL`, возвращается произвольное значение. [`Any`](/ru/reference/data-types/index)

**Примеры**

**Пример использования**

```sql title=Query
CREATE TABLE t_null (x Int8, y Nullable(Int8))
ENGINE=MergeTree()
ORDER BY x;

INSERT INTO t_null VALUES (1, NULL), (2, 3);

SELECT assumeNotNull(y) FROM table;
SELECT toTypeName(assumeNotNull(y)) FROM t_null;
```

```response title=Response
┌─assumeNotNull(y)─┐
│                0 │
│                3 │
└──────────────────┘
┌─toTypeName(assumeNotNull(y))─┐
│ Int8                         │
│ Int8                         │
└──────────────────────────────┘
```

<div id="coalesce">
  ## coalesce
</div>

Добавленный в: v1.1.0

Возвращает первый слева аргумент, не равный `NULL`.

**Синтаксис**

```sql
coalesce(x[, y, ...])
```

**Аргументы**

* `x[, y, ...]` — Любое количество параметров нескалярного типа. Все параметры должны иметь взаимно совместимые типы данных. [`Any`](/ru/reference/data-types/index)

**Возвращаемое значение**

Возвращает первый аргумент, не равный `NULL`; если все аргументы равны `NULL`, возвращает `NULL`. [`Any`](/ru/reference/data-types/index) или [`NULL`](/ru/reference/syntax#null)

**Примеры**

**Пример использования**

```sql title=Query
-- Consider a list of contacts that may specify multiple ways to contact a customer.

CREATE TABLE aBook
(
    name String,
    mail Nullable(String),
    phone Nullable(String),
    telegram Nullable(UInt32)
)
ENGINE = MergeTree
ORDER BY tuple();

INSERT INTO aBook VALUES ('client 1', NULL, '123-45-67', 123), ('client 2', NULL, NULL, NULL);

-- The mail and phone fields are of type String, but the telegram field is UInt32 so it needs to be converted to String.

-- Get the first available contact method for the customer from the contact list

SELECT name, coalesce(mail, phone, CAST(telegram,'Nullable(String)')) FROM aBook;
```

```response title=Response
┌─name─────┬─coalesce(mail, phone, CAST(telegram, 'Nullable(String)'))─┐
│ client 1 │ 123-45-67                                                 │
│ client 2 │ ᴺᵁᴸᴸ                                                      │
└──────────┴───────────────────────────────────────────────────────────┘
```

<div id="firstNonDefault">
  ## firstNonDefault
</div>

Добавленный в: v25.9.0

Возвращает первое значение, отличное от значения по умолчанию, из набора аргументов

**Синтаксис**

```sql
firstNonDefault(arg1[, arg2[ ...]])
```

**Аргументы**

* `arg1` — Первый аргумент для проверки - `arg2` — Второй аргумент для проверки - `...` — Дополнительные аргументы для проверки

**Возвращаемое значение**

Тип результата — супертип всех аргументов

**Примеры**

**целые числа**

```sql title=Query
SELECT firstNonDefault(0, 1, 2)
```

```response title=Response
1
```

**строки**

```sql title=Query
SELECT firstNonDefault('', 'hello', 'world')
```

```response title=Response
'hello'
```

**NULL-значения**

```sql title=Query
SELECT firstNonDefault(NULL, 0 :: UInt8, 1 :: UInt8)
```

```response title=Response
1
```

**нулевое значение типа Nullable**

```sql title=Query
SELECT firstNonDefault(NULL, 0 :: Nullable(UInt8), 1 :: Nullable(UInt8))
```

```response title=Response
0
```

<div id="ifNull">
  ## ifNull
</div>

Добавленный в: v1.1.0

Возвращает альтернативное значение, если первый аргумент равен `NULL`.

**Синтаксис**

```sql
ifNull(x, alt)
```

**Аргументы**

* `x` — Значение, проверяемое на `NULL`. [`Any`](/ru/reference/data-types/index)
* `alt` — Значение, которое функция возвращает, если `x` равно `NULL`. [`Any`](/ru/reference/data-types/index)

**Возвращаемое значение**

Возвращает значение `x`, если оно не равно `NULL`, иначе — `alt`. [`Any`](/ru/reference/data-types/index)

**Примеры**

**Пример использования**

```sql title=Query
SELECT ifNull('a', 'b'), ifNull(NULL, 'b');
```

```response title=Response
┌─ifNull('a', 'b')─┬─ifNull(NULL, 'b')─┐
│ a                │ b                 │
└──────────────────┴───────────────────┘
```

<div id="isNotNull">
  ## isNotNull
</div>

Добавленный в: v1.1.0

Проверяет, что аргумент не имеет значения `NULL`.

См. также: оператор [`IS NOT NULL`](/ru/reference/operators/index#is_not_null).

**Синтаксис**

```sql
isNotNull(x)
```

**Аргументы**

* `x` — Значение несоставного типа. [`Any`](/ru/reference/data-types/index)

**Возвращаемое значение**

Возвращает `1`, если `x` не равен `NULL`, в противном случае — `0`. [`UInt8`](/ru/reference/data-types/int-uint)

**Примеры**

**Пример использования**

```sql title=Query
CREATE TABLE t_null
(
  x Int32,
  y Nullable(Int32)
)
ENGINE = MergeTree
ORDER BY tuple();

INSERT INTO t_null VALUES (1, NULL), (2, 3);

SELECT x FROM t_null WHERE isNotNull(y);
```

```response title=Response
┌─x─┐
│ 2 │
└───┘
```

<div id="isNullable">
  ## isNullable
</div>

Добавленный в: v22.7.0

Проверяет, имеет ли аргумент тип данных `Nullable` (то есть допускает значения `NULL`).

**Синтаксис**

```sql
isNullable(x)
```

**Аргументы**

* `x` — значение любого типа данных. [`Any`](/ru/reference/data-types/index)

**Возвращаемое значение**

Возвращает `1`, если `x` имеет тип `Nullable`, в противном случае — `0`. [`UInt8`](/ru/reference/data-types/int-uint)

**Примеры**

**Пример использования**

```sql title=Query
CREATE TABLE tab (
    ordinary_col UInt32,
    nullable_col Nullable(UInt32)
)
ENGINE = MergeTree
ORDER BY tuple();
INSERT INTO tab (ordinary_col, nullable_col) VALUES (1,1), (2, 2), (3,3);
SELECT isNullable(ordinary_col), isNullable(nullable_col) FROM tab;
```

```response title=Response
┌───isNullable(ordinary_col)──┬───isNullable(nullable_col)──┐
│                           0 │                           1 │
│                           0 │                           1 │
│                           0 │                           1 │
└─────────────────────────────┴─────────────────────────────┘
```

<div id="nullIf">
  ## nullIf
</div>

Добавленный в: v1.1.0

Возвращает `NULL`, если оба аргумента равны.

**Синтаксис**

```sql
nullIf(x, y)
```

**Аргументы**

* `x` — Первое значение. [`Any`](/ru/reference/data-types/index)
* `y` — Второе значение. [`Any`](/ru/reference/data-types/index)

**Возвращаемое значение**

Возвращает `NULL`, если оба аргумента равны, иначе возвращает первый аргумент. [`NULL`](/ru/reference/syntax#null) или [`Nullable(x)`](/ru/reference/data-types/nullable)

**Примеры**

**Пример использования**

```sql title=Query
SELECT nullIf(1, 1), nullIf(1, 2);
```

```response title=Response
┌─nullIf(1, 1)─┬─nullIf(1, 2)─┐
│         ᴺᵁᴸᴸ │            1 │
└──────────────┴──────────────┘
```
