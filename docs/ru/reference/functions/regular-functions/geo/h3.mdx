<div id="h3-index">
  ## индекс H3
</div>

[H3](https://h3geo.org/) — это система географической индексации, в которой поверхность Земли разбита на сетку из одинаковых шестиугольных ячеек. Эта система иерархична, то есть каждый шестиугольник верхнего уровня (`parent`) может быть разделён на семь таких же, но меньших (`children`), и так далее.

Уровень иерархии называется `resolution` и может принимать значения от `0` до `15`, где `0` — это уровень `base` с самыми большими и самыми грубыми ячейками.

Пару широты и долготы можно преобразовать в 64-битный индекс H3, который идентифицирует ячейку сетки.

Индекс H3 в основном используется для группировки местоположений по бакетам и других геопространственных преобразований.

Полное описание системы H3 доступно на [сайте Uber Engineering](https://www.uber.com/blog/h3/).

<div id="h3isvalid">
  ## h3IsValid
</div>

Проверяет, является ли число корректным индексом [H3](#h3-index).

**Синтаксис**

```sql
h3IsValid(h3index)
```

**Параметр**

* `h3index` — Номер шестиугольного индекса. [UInt64](/ru/reference/data-types/int-uint).

**Возвращаемые значения**

* 1 — Число является корректным индексом H3. [UInt8](/ru/reference/data-types/int-uint).
* 0 — Число не является корректным индексом H3. [UInt8](/ru/reference/data-types/int-uint).

**Пример**

```sql title="Query"
SELECT h3IsValid(630814730351855103) AS h3IsValid;
```

```text title="Response"
┌─h3IsValid─┐
│         1 │
└───────────┘
```

<div id="h3getresolution">
  ## h3GetResolution
</div>

Определяет разрешение для указанного [H3](#h3-index)-индекса.

**Синтаксис**

```sql
h3GetResolution(h3index)
```

**Параметр**

* `h3index` — номер шестиугольного индекса. [UInt64](/ru/reference/data-types/int-uint).

**Возвращаемые значения**

* Разрешение индекса. Диапазон: `[0, 15]`. [UInt8](/ru/reference/data-types/int-uint).
* Если индекс некорректен, функция возвращает случайное значение. Используйте [h3IsValid](#h3isvalid), чтобы проверить индекс. [UInt8](/ru/reference/data-types/int-uint).

**Пример**

```sql title="Query"
SELECT h3GetResolution(639821929606596015) AS resolution;
```

```text title="Response"
┌─resolution─┐
│         14 │
└────────────┘
```

<div id="geotoh3">
  ## geoToH3
</div>

Возвращает индекс точки [H3](#h3-index) `(lat, lon)` с указанным разрешением.

**Синтаксис**

```sql
geoToH3(lat, lon, resolution)
```

**Аргументы**

* `lat` — Широта. [Float64](/ru/reference/data-types/float).
* `lon` — Долгота. [Float64](/ru/reference/data-types/float).
* `resolution` — Разрешение индекса. Диапазон: `[0, 15]`. [UInt8](/ru/reference/data-types/int-uint).

**Возвращаемые значения**

* Номер шестиугольного индекса. [UInt64](/ru/reference/data-types/int-uint).
* 0 в случае ошибки. [UInt64](/ru/reference/data-types/int-uint).

Примечание: в ClickHouse v25.4 и более ранних версиях `geoToH3()` принимает значения в порядке `(lon, lat)`. Начиная с ClickHouse v25.5, входные значения принимаются в порядке `(lat, lon)`. Предыдущее поведение можно восстановить с помощью настройки `geotoh3_argument_order = 'lon_lat'`.

**Пример**

```sql title="Query"
SELECT geoToH3(55.71290588, 37.79506683, 15) AS h3Index;
```

```text title="Response"
┌────────────h3Index─┐
│ 644325524701193974 │
└────────────────────┘
```

<div id="h3togeo">
  ## h3ToGeo
</div>

Возвращает широту и долготу центра, соответствующие указанному индексу [H3](#h3-index).

**Синтаксис**

```sql
h3ToGeo(h3Index)
```

**Аргументы**

* `h3Index` — индекс H3. [UInt64](/ru/reference/data-types/int-uint).

**Возвращаемые значения**

* Кортеж из двух значений: `tuple(lat,lon)`. `lat` — широта. [Float64](/ru/reference/data-types/float). `lon` — долгота. [Float64](/ru/reference/data-types/float).

Примечание: в ClickHouse v24.12 и более ранних версиях `h3ToGeo()` возвращает значения в порядке `(lon, lat)`. Начиная с ClickHouse v25.1, возвращаемые значения идут в порядке `(lat, lon)`. Предыдущее поведение можно восстановить с помощью настройки `h3togeo_lon_lat_result_order = true`.

**Пример**

```sql title="Query"
SELECT h3ToGeo(644325524701193974) AS coordinates;
```

```text title="Response"
┌─coordinates───────────────────────────┐
│ (55.71290243145668,37.79506616830252) │
└───────────────────────────────────────┘
```

<div id="h3polygontocellswithcontainment">
  ## h3PolygonToCellsWithContainment
</div>

Возвращает шестиугольники (с указанным разрешением), покрывающие переданную геометрию — Ring или (multi-)polygon, — с использованием экспериментальных режимов покрытия H3.

Эта функция сопоставляет `flags` с режимами покрытия H3:

* `0` — `CONTAINMENT_CENTER`
* `1` — `CONTAINMENT_FULL`
* `2` — `CONTAINMENT_OVERLAPPING`
* `3` — `CONTAINMENT_OVERLAPPING_BBOX`

**Синтаксис**

```sql
h3PolygonToCellsWithContainment(geometry, resolution, flags)
```

**Аргументы**

* `geometry` может быть одним из следующих [гео-типов данных](/ru/reference/data-types/geo) или их базовых примитивных типов:
  * [Ring](/ru/reference/data-types/geo#ring)
  * [Polygon](/ru/reference/data-types/geo#polygon)
  * [MultiPolygon](/ru/reference/data-types/geo#multipolygon)
* `resolution` — разрешение индекса. Диапазон: `[0, 15]`. [UInt8](/ru/reference/data-types/int-uint).
* `flags` — режим покрытия. Диапазон: `[0, 3]`. Используйте целочисленные литералы, такие как `0`, `1`, `2` и `3`, или значения [UInt32](/ru/reference/data-types/int-uint) (например, `toUInt32(2)`). Значения передаются в H3 API как `UInt32`. Другие встроенные целочисленные типы преобразуются с помощью точного приведения типов.

**Возвращаемые значения**

* Массив индексов H3 в соответствии с выбранным режимом покрытия. [Array](/ru/reference/data-types/array)([UInt64](/ru/reference/data-types/int-uint)).

```sql title="Query"
SELECT
```
