<div id="timeSeriesChangesToGrid">
  ## timeSeriesChangesToGrid
</div>

Добавленный в: v25.6.0

Агрегатная функция, принимающая данные временных рядов в виде пар временных меток и значений и вычисляющая [изменения в стиле PromQL](https://prometheus.io/docs/prometheus/latest/querying/functions/#changes) по этим данным на регулярной временной сетке, задаваемой начальной временной меткой, конечной временной меткой и шагом. Для каждой точки сетки значения для вычисления `changes` берутся в пределах указанного временного окна.

<Note>
  Эта функция экспериментальная; чтобы включить её, задайте `allow_experimental_ts_to_grid_aggregate_function=true`.
</Note>

**Синтаксис**

```sql
timeSeriesChangesToGrid(start_timestamp, end_timestamp, grid_step, staleness)(timestamp, value)
```

**Параметры**

* `start_timestamp` — Задает начало сетки. - `end_timestamp` — Задает конец сетки. - `grid_step` — Задает шаг сетки в секундах. - `staleness` — Задает максимально допустимое устаревание рассматриваемых образцов в секундах.

**Аргументы**

* `timestamp` — Временная метка образца. Может быть отдельным значением или массивом. - `value` — Значение временного ряда, соответствующее временной метке. Может быть отдельным значением или массивом.

**Возвращаемое значение**

Значения `changes` на указанной сетке в виде `Array(Nullable(Float64))`. Возвращаемый массив содержит одно значение для каждой точки временной сетки. Значение равно NULL, если в пределах окна нет образцов для вычисления значения `changes` для конкретной точки сетки.

**Примеры**

**Вычисление значений `changes` на сетке [90, 105, 120, 135, 150, 165, 180, 195, 210, 225]**

```sql title=Query
WITH
    -- ПРИМЕЧАНИЕ: пропуск между 130 и 190 показывает, как значения заполняются для ts = 180 согласно параметру window
    [110, 120, 130, 190, 200, 210, 220, 230]::Array(DateTime) AS timestamps,
    [1, 1, 3, 5, 5, 8, 12, 13]::Array(Float32) AS values, -- массив значений, соответствующих временным меткам выше
    90 AS start_ts,       -- начало сетки временных меток
    90 + 135 AS end_ts,   -- конец сетки временных меток
    15 AS step_seconds,   -- шаг сетки временных меток
    45 AS window_seconds  -- окно "устаревания"
SELECT timeSeriesChangesToGrid(start_ts, end_ts, step_seconds, window_seconds)(timestamp, value)
FROM
(
    -- Этот подзапрос преобразует массивы временных меток и значений в строки `timestamp`, `value`
    SELECT
        arrayJoin(arrayZip(timestamps, values)) AS ts_and_val,
        ts_and_val.1 AS timestamp,
        ts_and_val.2 AS value
);
```

```response title=Response
┌─timeSeriesChangesToGrid(start_ts, end_ts, step_seconds, window_seconds)(timestamp, value)─┐
│ [NULL,NULL,0,1,1,1,NULL,0,1,2]                                                            │
└───────────────────────────────────────────────────────────────────────────────────────────┘
```

**Тот же запрос с аргументами типа Array**

```sql title=Query
WITH
    [110, 120, 130, 190, 200, 210, 220, 230]::Array(DateTime) AS timestamps,
    [1, 1, 3, 5, 5, 8, 12, 13]::Array(Float32) AS values,
    90 AS start_ts,
    90 + 135 AS end_ts,
    15 AS step_seconds,
    45 AS window_seconds
SELECT timeSeriesChangesToGrid(start_ts, end_ts, step_seconds, window_seconds)(timestamps, values);
```

```response title=Response
┌─timeSeriesChangesToGrid(start_ts, end_ts, step_seconds, window_seconds)(timestamp, value)─┐
│ [NULL,NULL,0,1,1,1,NULL,0,1,2]                                                            │
└───────────────────────────────────────────────────────────────────────────────────────────┘
```
