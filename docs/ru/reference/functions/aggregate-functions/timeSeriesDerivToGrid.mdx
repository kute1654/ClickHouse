<div id="timeSeriesDerivToGrid">
  ## timeSeriesDerivToGrid
</div>

Добавленный в: v25.6.0

Агрегатная функция, которая принимает данные временных рядов в виде пар временных меток и значений и вычисляет [производную в стиле PromQL](https://prometheus.io/docs/prometheus/latest/querying/functions/#deriv) по этим данным на регулярной временной сетке, заданной начальной временной меткой, конечной временной меткой и шагом. Для каждой точки сетки при вычислении `deriv` учитываются выборки в пределах указанного временного окна.

<Note>
  Эта функция экспериментальная; чтобы включить её, задайте `allow_experimental_ts_to_grid_aggregate_function=true`.
</Note>

**Синтаксис**

```sql
timeSeriesDerivToGrid(start_timestamp, end_timestamp, grid_step, staleness)(timestamp, value)
```

**Параметры**

* `start_timestamp` — Указывает начало сетки. - `end_timestamp` — Указывает конец сетки. - `grid_step` — Указывает шаг сетки в секундах. - `staleness` — Указывает максимально допустимую «устарелость» рассматриваемых отсчётов в секундах. Окно устарелости представляет собой интервал, открытый слева и закрытый справа.

**Аргументы**

* `timestamp` — Временная метка отсчёта. Может быть отдельным значением или массивом. - `value` — Значение временного ряда, соответствующее временной метке. Может быть отдельным значением или массивом.

**Возвращаемое значение**

Значения `deriv` на указанной сетке в виде `Array(Nullable(Float64))`. Возвращаемый массив содержит по одному значению для каждой точки временной сетки. Значение равно NULL, если в пределах окна недостаточно отсчётов для вычисления значения производной для конкретной точки сетки.

**Примеры**

**Вычисление значений производной на сетке [90, 105, 120, 135, 150, 165, 180, 195, 210]**

```sql title=Query
WITH
    -- ПРИМЕЧАНИЕ: пропуск между 140 и 190 показывает, как заполняются значения для ts = 150, 165, 180 согласно параметру window
    [110, 120, 130, 140, 190, 200, 210, 220, 230]::Array(DateTime) AS timestamps,
    [1, 1, 3, 4, 5, 5, 8, 12, 13]::Array(Float32) AS values, -- массив значений, соответствующих приведённым выше временным меткам
    90 AS start_ts,       -- начало сетки временных меток
    90 + 120 AS end_ts,   -- конец сетки временных меток
    15 AS step_seconds,   -- шаг сетки временных меток
    45 AS window_seconds  -- окно «устаревания»
SELECT timeSeriesDerivToGrid(start_ts, end_ts, step_seconds, window_seconds)(timestamp, value)
FROM
(
    -- Этот подзапрос преобразует массивы временных меток и значений в строки `timestamp`, `value`
    SELECT
        arrayJoin(arrayZip(timestamps, values)) AS ts_and_val,
        ts_and_val.1 AS timestamp,
        ts_and_val.2 AS value
);
```

```response title=Response
┌─timeSeriesDerivToGrid(start_ts, end_ts, step_seconds, window_seconds)(timestamp, value)─┐
│ [NULL,NULL,0,0.1,0.11,0.15,NULL,NULL,0.15]                                              │
└─────────────────────────────────────────────────────────────────────────────────────────┘
```

**Тот же запрос с аргументами в виде массивов**

```sql title=Query
WITH
    [110, 120, 130, 140, 190, 200, 210, 220, 230]::Array(DateTime) AS timestamps,
    [1, 1, 3, 4, 5, 5, 8, 12, 13]::Array(Float32) AS values,
    90 AS start_ts,
    90 + 120 AS end_ts,
    15 AS step_seconds,
    45 AS window_seconds
SELECT timeSeriesDerivToGrid(start_ts, end_ts, step_seconds, window_seconds)(timestamps, values);
```

```response title=Response
┌─timeSeriesDerivToGrid(start_ts, end_ts, step_seconds, window_seconds)(timestamps, values)─┐
│ [NULL,NULL,0,0.1,0.11,0.15,NULL,NULL,0.15]                                                │
└───────────────────────────────────────────────────────────────────────────────────────────┘
```
