<div id="description">
  ## Описание
</div>

Содержит список всех форматов, а также флаги, указывающие, подходит ли тот или иной формат для ввода/вывода и поддерживает ли он распараллеливание.

<div id="columns">
  ## Столбцы
</div>

{/*AUTOGENERATED_START*/}

* `name` ([String](/ru/reference/data-types/index)) — Название формата.
* `is_input` ([UInt8](/ru/reference/data-types/index)) — Флаг, указывающий, подходит ли формат для ввода данных.
* `is_output` ([UInt8](/ru/reference/data-types/index)) — Флаг, указывающий, подходит ли формат для вывода данных.
* `supports_parallel_parsing` ([UInt8](/ru/reference/data-types/index)) — Флаг, указывающий, поддерживает ли формат параллельный разбор.
* `supports_parallel_formatting` ([UInt8](/ru/reference/data-types/index)) — Флаг, указывающий, поддерживает ли формат параллельное форматирование.
* `is_tty_friendly` ([UInt8](/ru/reference/data-types/index)) — Флаг, указывающий, обычно ли формат корректно отображается в терминале. Для других форматов CLI перед выводом запросит подтверждение.
* `content_type` ([String](/ru/reference/data-types/index)) — HTTP Content-Type, соответствующий выходному формату. Может зависеть от текущих настроек формата.
* `supports_random_access` ([UInt8](/ru/reference/data-types/index)) — Флаг, указывающий, поддерживает ли формат произвольный доступ при вводе.
* `has_schema_inference` ([UInt8](/ru/reference/data-types/index)) — Формат может динамически определять схему по данным (либо по встроенному заголовку/метаданным, либо по самому фрагменту данных).
* `has_external_schema` ([UInt8](/ru/reference/data-types/index)) — Формат либо имеет фиксированную схему, либо принимает заранее заданную схему в собственном формате.
* `prefers_large_blocks` ([UInt8](/ru/reference/data-types/index)) — Формат будет записывать более крупные блоки при выводе и формировать более крупные блоки при вводе.
* `supports_append` ([UInt8](/ru/reference/data-types/index)) — Этот формат позволяет дописывать данные в один и тот же файл.
* `supports_subsets_of_columns` ([UInt8](/ru/reference/data-types/index)) — Входной формат может распознавать отсутствие некоторых столбцов.
* `description` ([String](/ru/reference/data-types/index)) — Общее описание формата.
* `examples` ([String](/ru/reference/data-types/index)) — Примеры использования.
* `introduced_in` ([String](/ru/reference/data-types/index)) — Версия ClickHouse, в которой формат был впервые представлен, в формате major.minor.
* `related` ([Array(String)](/ru/reference/data-types/index)) — Названия связанных форматов.
