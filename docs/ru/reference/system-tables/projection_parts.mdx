<div id="description">
  ## Описание
</div>

Эта таблица содержит информацию о частях проекций таблиц семейства MergeTree.

<div id="columns">
  ## Столбцы
</div>

{/*AUTOGENERATED_START*/}

* `partition` ([String](/ru/reference/data-types/index)) — Имя партиции.
* `name` ([String](/ru/reference/data-types/index)) — Имя части данных.
* `part_type` ([String](/ru/reference/data-types/index)) — Формат хранения части данных. Возможные значения: Wide (отдельный файл для каждого столбца) и Compact (один файл для всех столбцов).
* `parent_name` ([String](/ru/reference/data-types/index)) — Имя исходной (родительской) части данных.
* `parent_uuid` ([UUID](/ru/reference/data-types/index)) — UUID исходной (родительской) части данных.
* `parent_part_type` ([String](/ru/reference/data-types/index)) — Формат хранения исходной (родительской) части данных.
* `active` ([UInt8](/ru/reference/data-types/index)) — Флаг, указывающий, активна ли часть данных. Если часть данных активна, она используется в таблице. В противном случае она будет удалена. Неактивные части данных появляются после операций слияния и мутации.
* `marks` ([UInt64](/ru/reference/data-types/index)) — Количество marks. Чтобы получить приблизительное количество строк в части данных, умножьте число marks на гранулярность индекса (обычно 8192) (эта подсказка не работает для adaptive granularity).
* `rows` ([UInt64](/ru/reference/data-types/index)) — Количество строк.
* `bytes_on_disk` ([UInt64](/ru/reference/data-types/index)) — Общий размер всех файлов части данных в байтах.
* `data_compressed_bytes` ([UInt64](/ru/reference/data-types/index)) — Общий размер сжатых данных в части данных. Все вспомогательные файлы (например, файлы с marks) не включаются.
* `data_uncompressed_bytes` ([UInt64](/ru/reference/data-types/index)) — Общий размер несжатых данных в части данных. Все вспомогательные файлы (например, файлы с marks) не включаются.
* `marks_bytes` ([UInt64](/ru/reference/data-types/index)) — Размер файла с marks.
* `parent_marks` ([UInt64](/ru/reference/data-types/index)) — Количество marks в исходной (родительской) части.
* `parent_rows` ([UInt64](/ru/reference/data-types/index)) — Количество строк в исходной (родительской) части.
* `parent_bytes_on_disk` ([UInt64](/ru/reference/data-types/index)) — Общий размер всех файлов исходной (родительской) части данных в байтах.
* `parent_data_compressed_bytes` ([UInt64](/ru/reference/data-types/index)) — Общий размер сжатых данных в исходной (родительской) части данных.
* `parent_data_uncompressed_bytes` ([UInt64](/ru/reference/data-types/index)) — Общий размер несжатых данных в исходной (родительской) части данных.
* `parent_marks_bytes` ([UInt64](/ru/reference/data-types/index)) — Размер файла с marks в исходной (родительской) части данных.
* `modification_time` ([DateTime](/ru/reference/data-types/index)) — Время изменения каталога с частью данных. Обычно оно соответствует времени создания части данных.
* `remove_time` ([DateTime](/ru/reference/data-types/index)) — Время, когда часть данных стала неактивной.
* `refcount` ([UInt32](/ru/reference/data-types/index)) — Количество мест, где используется часть данных. Значение больше 2 указывает на то, что часть данных используется в запросах или слияниях.
* `min_date` ([Date](/ru/reference/data-types/index)) — Минимальное значение ключа даты в части данных.
* `max_date` ([Date](/ru/reference/data-types/index)) — Максимальное значение ключа даты в части данных.
* `min_time` ([DateTime](/ru/reference/data-types/index)) — Минимальное значение ключа даты и времени в части данных.
* `max_time` ([DateTime](/ru/reference/data-types/index)) — Максимальное значение ключа даты и времени в части данных.
* `partition_id` ([String](/ru/reference/data-types/index)) — Идентификатор партиции.
* `min_block_number` ([Int64](/ru/reference/data-types/index)) — Минимальное количество частей данных, из которых состоит текущая часть после слияния.
* `max_block_number` ([Int64](/ru/reference/data-types/index)) — Максимальное количество частей данных, из которых состоит текущая часть после слияния.
* `level` ([UInt32](/ru/reference/data-types/index)) — Глубина дерева слияния. Ноль означает, что текущая часть была создана вставкой, а не в результате слияния других частей.
* `data_version` ([UInt64](/ru/reference/data-types/index)) — Число, используемое для определения того, какие мутации должны быть применены к части данных (мутации с версией выше, чем data&#95;version).
* `primary_key_bytes_in_memory` ([UInt64](/ru/reference/data-types/index)) — Объём памяти (в байтах), используемой значениями первичного ключа.
* `primary_key_bytes_in_memory_allocated` ([UInt64](/ru/reference/data-types/index)) — Объём памяти (в байтах), зарезервированной для значений первичного ключа.
* `is_frozen` ([UInt8](/ru/reference/data-types/index)) — Флаг, показывающий наличие резервной копии данных партиции. 1 — резервная копия существует. 0 — резервная копия не существует.
* `database` ([String](/ru/reference/data-types/index)) — Имя базы данных.
* `table` ([String](/ru/reference/data-types/index)) — Имя таблицы.
* `engine` ([String](/ru/reference/data-types/index)) — Имя движка таблицы без параметров.
* `disk_name` ([String](/ru/reference/data-types/index)) — Имя диска, на котором хранится часть данных.
* `path` ([String](/ru/reference/data-types/index)) — Абсолютный путь к папке с файлами части данных.
* `hash_of_all_files` ([String](/ru/reference/data-types/index)) — sipHash128 сжатых файлов.
* `hash_of_uncompressed_files` ([String](/ru/reference/data-types/index)) — sipHash128 несжатых файлов (файлов с marks, index file и т. д.).
* `uncompressed_hash_of_compressed_files` ([String](/ru/reference/data-types/index)) — sipHash128 данных в сжатых файлах, как если бы они были несжатыми.
* `delete_ttl_info_min` ([DateTime](/ru/reference/data-types/index)) — Минимальное значение ключа даты и времени для правила TTL DELETE.
* `delete_ttl_info_max` ([DateTime](/ru/reference/data-types/index)) — Максимальное значение ключа даты и времени для правила TTL DELETE.
* `move_ttl_info.expression` ([Array(String)](/ru/reference/data-types/index)) — Массив выражений. Каждое выражение определяет правило TTL MOVE.
* `move_ttl_info.min` ([Array(DateTime)](/ru/reference/data-types/index)) — Массив значений даты и времени. Каждый элемент описывает минимальное значение ключа для правила TTL MOVE.
* `move_ttl_info.max` ([Array(DateTime)](/ru/reference/data-types/index)) — Массив значений даты и времени. Каждый элемент описывает максимальное значение ключа для правила TTL MOVE.
* `default_compression_codec` ([String](/ru/reference/data-types/index)) — Имя кодека, используемого для сжатия этой части данных (если для столбцов не задан явный кодек).
* `recompression_ttl_info.expression` ([Array(String)](/ru/reference/data-types/index)) — Выражение TTL.
* `recompression_ttl_info.min` ([Array(DateTime)](/ru/reference/data-types/index)) — Минимальное значение вычисленного TTL-выражения в пределах этой части. Используется, чтобы определить, есть ли хотя бы одна строка с истёкшим TTL.
* `recompression_ttl_info.max` ([Array(DateTime)](/ru/reference/data-types/index)) — Максимальное значение вычисленного TTL-выражения в этой части. Используется, чтобы определить, все ли строки имеют истёкший TTL.
* `group_by_ttl_info.expression` ([Array(String)](/ru/reference/data-types/index)) — TTL-выражение.
* `group_by_ttl_info.min` ([Array(DateTime)](/ru/reference/data-types/index)) — Минимальное значение вычисленного TTL-выражения в этой части. Используется, чтобы определить, есть ли хотя бы одна строка с истёкшим TTL.
* `group_by_ttl_info.max` ([Array(DateTime)](/ru/reference/data-types/index)) — Максимальное значение вычисленного TTL-выражения в этой части. Используется, чтобы определить, все ли строки имеют истёкший TTL.
* `rows_where_ttl_info.expression` ([Array(String)](/ru/reference/data-types/index)) — TTL-выражение.
* `rows_where_ttl_info.min` ([Array(DateTime)](/ru/reference/data-types/index)) — Минимальное значение вычисленного TTL-выражения в этой части. Используется, чтобы определить, есть ли хотя бы одна строка с истёкшим TTL.
* `rows_where_ttl_info.max` ([Array(DateTime)](/ru/reference/data-types/index)) — Максимальное значение вычисленного TTL-выражения в этой части. Используется, чтобы определить, все ли строки имеют истёкший TTL.
* `is_broken` ([UInt8](/ru/reference/data-types/index)) — Повреждена ли часть проекции
* `exception_code` ([Int32](/ru/reference/data-types/index)) — Сообщение об исключении, объясняющее повреждённое состояние части проекции
* `exception` ([String](/ru/reference/data-types/index)) — Код исключения, объясняющий повреждённое состояние части проекции
