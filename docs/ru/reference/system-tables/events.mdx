<div id="description">
  ## Описание
</div>

Содержит сведения о количестве событий, произошедших в системе. Например, в таблице можно увидеть, сколько запросов `SELECT` было обработано с момента запуска сервера ClickHouse.

<div id="columns">
  ## Столбцы
</div>

{/*AUTOGENERATED_START*/}

* `event` ([String](/ru/reference/data-types/index)) — Имя события.
* `value` ([UInt64](/ru/reference/data-types/index)) — Количество произошедших событий.
* `description` ([String](/ru/reference/data-types/index)) — Описание события.

{/*AUTOGENERATED_END*/}

<div id="example">
  ## Пример
</div>

```sql
SELECT * FROM system.events LIMIT 5
```

Может включать внутренние запросы, инициированные самим ClickHouse. Подзапросы не учитываются.                  │

<div id="dataaftermergediffersfromreplica">
  ### Данные после слияния отличаются от данных на реплике
</div>

Количество случаев, когда данные после слияния не являются побайтно идентичными данным на другой реплике. Причин может быть несколько:

1. Использование более новой версии библиотеки сжатия после обновления сервера.
2. Использование другого метода сжатия.
3. Недетерминированный алгоритм сжатия (крайне маловероятно).
4. Недетерминированный алгоритм слияния из-за логической ошибки в коде.
5. Повреждение данных в памяти из-за ошибки в коде.
6. Повреждение данных в памяти из-за аппаратной проблемы.
7. Ручное изменение исходных данных после запуска сервера.
8. Ручное изменение контрольных сумм, хранящихся в ZooKeeper.
9. Настройки, связанные с форматом частей, такие как &#39;enable&#95;mixed&#95;granularity&#95;parts&#39;, различаются на разных репликах.
   Сервер обнаружил эту ситуацию и загрузит слитую часть с реплики, чтобы обеспечить побайтно идентичный результат.
