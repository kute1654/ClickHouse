<div id="description">
  ## Описание
</div>

Содержит статус загрузки, сведения об ошибках и метаданные конфигурации для [пользовательских функций (UDF)](/ru/reference/functions/regular-functions/udf).

<div id="columns">
  ## Столбцы
</div>

{/*AUTOGENERATED_START*/}

* `name` ([String](/ru/reference/data-types/index)) — Имя UDF.
* `load_status` ([Enum8(&#39;Success&#39; = 0, &#39;Failed&#39; = 1)](/ru/reference/data-types/index)) — Статус загрузки. Возможные значения:
  * **Success** — UDF загружена и готова к использованию
  * **Failed** — Не удалось загрузить UDF (подробности см. в поле &#39;loading&#95;error&#95;message&#39;).
* `loading_error_message` ([String](/ru/reference/data-types/index)) — Подробное сообщение об ошибке при сбое загрузки. Пусто, если загрузка прошла успешно.
* `last_successful_update_time` ([Nullable(DateTime)](/ru/reference/data-types/index)) — Временная метка последнего успешного обновления. NULL, если успешных обновлений не было.
* `loading_duration_ms` ([UInt64](/ru/reference/data-types/index)) — Время загрузки UDF в миллисекундах.
* `type` ([Enum8(&#39;executable&#39; = 0, &#39;executable&#95;pool&#39; = 1)](/ru/reference/data-types/index)) — Тип UDF: &#39;executable&#39; (один процесс) или &#39;executable&#95;pool&#39; (пул процессов).
* `command` ([String](/ru/reference/data-types/index)) — Скрипт или команда, выполняемые для этой UDF.
* `format` ([String](/ru/reference/data-types/index)) — Формат данных для ввода-вывода (например, &#39;TabSeparated&#39;, &#39;JSONEachRow&#39;).
* `return_type` ([String](/ru/reference/data-types/index)) — Возвращаемый тип функции (например, &#39;String&#39;, &#39;UInt64&#39;).
* `return_name` ([String](/ru/reference/data-types/index)) — Необязательный идентификатор возвращаемого значения. Пусто, если не задано.
* `argument_types` ([Array(String)](/ru/reference/data-types/index)) — Массив типов аргументов (например, [&#39;String&#39;, &#39;UInt64&#39;]).
* `argument_names` ([Array(String)](/ru/reference/data-types/index)) — Массив имен аргументов. Для аргументов без имени используются пустые строки.
* `max_command_execution_time` ([UInt64](/ru/reference/data-types/index)) — Максимальное время обработки блока данных в секундах. Только для типа &#39;executable&#95;pool&#39;.
* `command_termination_timeout` ([UInt64](/ru/reference/data-types/index)) — Число секунд до отправки SIGTERM процессу команды.
* `command_read_timeout` ([UInt64](/ru/reference/data-types/index)) — Время чтения из stdout команды в Milliseconds.
* `command_write_timeout` ([UInt64](/ru/reference/data-types/index)) — Время записи в stdin команды в Milliseconds.
* `pool_size` ([UInt64](/ru/reference/data-types/index)) — Количество экземпляров процесса команды. Только для типа &#39;executable&#95;pool&#39;.
* `send_chunk_header` ([UInt8](/ru/reference/data-types/index)) — Отправлять ли число строк перед каждым фрагментом данных (булевый тип).
* `execute_direct` ([UInt8](/ru/reference/data-types/index)) — Выполнять ли команду напрямую (1) или через /bin/bash (0).
* `lifetime` ([UInt64](/ru/reference/data-types/index)) — Интервал перезагрузки в секундах. 0 означает, что перезагрузка отключена.
* `deterministic` ([UInt8](/ru/reference/data-types/index)) — Возвращает ли функция один и тот же результат для одинаковых аргументов (булевый тип).

{/*AUTOGENERATED_END*/}

<div id="example">
  ## Пример
</div>

Просмотрите все пользовательские функции (UDF) и их статус загрузки:

```sql
SELECT
    name,
    load_status,
    type,
    command,
    return_type,
    argument_types
FROM system.user_defined_functions
FORMAT Vertical;
```

```response
Row 1:
──────
name:           my_sum_udf
load_status:    Success
type:           executable
command:        /var/lib/clickhouse/user_scripts/sum.py
return_type:    UInt64
argument_types: ['UInt64','UInt64']
```

Найдите пользовательские функции (UDF), завершившиеся с ошибкой:

```sql
SELECT
    name,
    loading_error_message
FROM system.user_defined_functions
WHERE load_status = 'Failed';
```
