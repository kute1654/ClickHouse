<div id="description">
  ## Описание
</div>

Содержит информацию о таблицах, для которых была выполнена команда DROP TABLE, но очистка данных для них ещё не завершена.

<div id="columns">
  ## Столбцы
</div>

{/*AUTOGENERATED_START*/}

* `index` ([UInt32](/ru/reference/data-types/index)) — Индекс в очереди marked&#95;dropped&#95;tables.
* `database` ([String](/ru/reference/data-types/index)) — Имя базы данных.
* `table` ([String](/ru/reference/data-types/index)) — Имя таблицы.
* `uuid` ([UUID](/ru/reference/data-types/index)) — UUID таблицы.
* `engine` ([String](/ru/reference/data-types/index)) — Имя движка таблицы.
* `metadata_dropped_path` ([String](/ru/reference/data-types/index)) — Путь к файлу метаданных таблицы в каталоге metadata&#95;dropped.
* `table_dropped_time` ([DateTime](/ru/reference/data-types/index)) — Время, на которое запланирована следующая попытка удалить данные таблицы. Обычно это время удаления таблицы плюс `database_atomic_delay_before_drop_table_sec`.

{/*AUTOGENERATED_END*/}

<div id="example">
  ## Пример
</div>

В следующем примере показано, как получить сведения о `dropped_tables`.

```sql
SELECT *
FROM system.dropped_tables\G
```
