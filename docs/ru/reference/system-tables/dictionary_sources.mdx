<div id="description">
  ## Описание
</div>

Содержит список источников словарей, поддерживаемых сервером, а также встроенную документацию для каждого источника. Источник словаря определяет, откуда загружаются данные словаря; он указывается в предложении `SOURCE` запроса `CREATE DICTIONARY`.

<div id="columns">
  ## Столбцы
</div>

{/*AUTOGENERATED_START*/}

* `name` ([String](/ru/reference/data-types/index)) — Имя источника словаря, как указано в предложении SOURCE.
* `description` ([String](/ru/reference/data-types/index)) — Общее описание назначения источника словаря.
* `syntax` ([String](/ru/reference/data-types/index)) — Структура предложения SOURCE, используемого для указания источника. Обратите внимание, что некоторые источники подпадают под управление доступом, если словарь создаётся из DDL-запроса (а не из файла конфигурации сервера); подробности см. в `description` конкретного источника.
* `examples` ([String](/ru/reference/data-types/index)) — Примеры использования.
* `introduced_in` ([String](/ru/reference/data-types/index)) — Версия ClickHouse, в которой этот источник был впервые представлен, в формате major.minor.
* `related` ([Array(String)](/ru/reference/data-types/index)) — Имена связанных источников словаря.

{/*AUTOGENERATED_END*/}

<div id="example">
  ## Пример
</div>

```sql title="Query"
SELECT name, syntax
FROM system.dictionary_sources
WHERE name IN ('clickhouse', 'file')
ORDER BY name
```

```text title="Response"
┌─name───────┬─syntax─────────────────────────────────────────────────────────────────────────┐
│ clickhouse │ SOURCE(CLICKHOUSE(host 'host' port 9000 user 'default' password '' db 'db' table 'table')) │
│ file       │ SOURCE(FILE(path '/path/to/file' format 'CSV'))                                  │
└────────────┴────────────────────────────────────────────────────────────────────────────────┘
```
