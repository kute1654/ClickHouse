<div id="description">
  ## Описание
</div>

Содержит сведения о сообщениях, полученных через потоковый движок и разобранных с ошибками. В настоящее время реализовано для Kafka и RabbitMQ.

Логирование включается указанием `dead_letter_queue` для специфичного для движка параметра `handle_error_mode`.

Период сброса данных задается параметром `flush_interval_milliseconds` в разделе настроек сервера [dead&#95;letter&#95;queue](/ru/reference/settings/server-settings/settings#dead_letter_queue). Чтобы принудительно выполнить сброс, используйте запрос [SYSTEM FLUSH LOGS](/ru/reference/statements/system#flush-logs).

ClickHouse не удаляет данные из таблицы автоматически. Подробнее см. в разделе [Introduction](/ru/reference/system-tables/overview#system-tables-introduction).
