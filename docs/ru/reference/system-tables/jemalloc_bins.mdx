<div id="description">
  ## Описание
</div>

Содержит агрегированную по всем аренам информацию о выделениях памяти, выполненных через аллокатор jemalloc в разных классах размеров (bins).
Эта статистика может быть не совсем точной из-за локального кэширования в jemalloc на уровне потоков.

<div id="columns">
  ## Столбцы
</div>

{/*AUTOGENERATED_START*/}

* `index` ([UInt16](/ru/reference/data-types/index)) — Индекс bin, отсортированного по размеру.
* `large` ([UInt8](/ru/reference/data-types/index)) — Принимает значение true для крупных выделений и false для малых.
* `size` ([UInt64](/ru/reference/data-types/index)) — Размер выделений в этом bin.
* `allocations` ([Int64](/ru/reference/data-types/index)) — Количество выделений.
* `deallocations` ([Int64](/ru/reference/data-types/index)) — Количество освобождений памяти.
* `nregs` ([Int64](/ru/reference/data-types/index)) — Количество регионов в slab.
* `curslabs` ([Int64](/ru/reference/data-types/index)) — Текущее количество slab.
* `curregs` ([Int64](/ru/reference/data-types/index)) — Текущее количество регионов для этого класса размера.

{/*AUTOGENERATED_END*/}

<div id="example">
  ## Пример
</div>

Найдите размеры выделений, которые больше всего повлияли на текущее общее использование памяти.

```sql
SELECT
    *,
    allocations - deallocations AS active_allocations,
    size * active_allocations AS allocated_bytes
FROM system.jemalloc_bins
WHERE allocated_bytes > 0
ORDER BY allocated_bytes DESC
LIMIT 10
```
