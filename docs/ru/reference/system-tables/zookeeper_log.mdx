<div id="description">
  ## Описание
</div>

Эта таблица содержит информацию о параметрах запроса к серверу ZooKeeper и ответа от него.

Для запросов заполняются только столбцы с параметрами запроса, а остальные столбцы получают значения по умолчанию (`0` или `NULL`). Когда приходит ответ, данные из него добавляются в остальные столбцы.

<div id="columns">
  ## Столбцы
</div>

{/*AUTOGENERATED_START*/}

* `hostname` ([LowCardinality(String)](/ru/reference/data-types/lowcardinality)) — Имя хоста сервера, выполняющего запрос.
* `type` ([Enum8(&#39;Request&#39; = 1, &#39;Response&#39; = 2, &#39;Finalize&#39; = 3)](/ru/reference/data-types/enum)) — Тип события клиента ZooKeeper. Может принимать одно из следующих значений: Request — запрос был отправлен, Response — ответ был получен, Finalize — соединение потеряно, ответ не был получен.
* `event_date` ([Date](/ru/reference/data-types/date)) — Дата события.
* `event_time` ([DateTime64(6)](/ru/reference/data-types/datetime64)) — дата и время, когда произошло событие.
* `thread_id` ([UInt64](/ru/reference/data-types/int-uint)) — Идентификатор потока, выполнившего этот запрос.
* `query_id` ([String](/ru/reference/data-types/string)) — Идентификатор запроса, в контексте которого был выполнен данный запрос.
* `address` ([IPv6](/ru/reference/data-types/ipv6)) — IP-адрес сервера ZooKeeper, к которому был отправлен запрос.
* `port` ([UInt16](/ru/reference/data-types/int-uint)) — Порт сервера ZooKeeper, использованный для выполнения запроса.
* `session_id` ([Int64](/ru/reference/data-types/int-uint)) — Идентификатор сеанса, который сервер ZooKeeper устанавливает для каждого соединения.
* `duration_microseconds` ([UInt64](/ru/reference/data-types/int-uint)) — Время, затраченное ZooKeeper на выполнение запроса.
* `xid` ([Int64](/ru/reference/data-types/int-uint)) — Идентификатор запроса в рамках сеанса. Обычно это последовательный номер запроса. Он одинаков для строки запроса и соответствующей парной строки ответа/finalize.
* `has_watch` ([UInt8](/ru/reference/data-types/int-uint)) — Признак того, установлено ли наблюдение.
* `op_num` ([Enum16(&#39;Close&#39; = -11, &#39;Error&#39; = -1, &#39;Watch&#39; = 0, &#39;Create&#39; = 1, &#39;Remove&#39; = 2, &#39;Exists&#39; = 3, &#39;Get&#39; = 4, &#39;Set&#39; = 5, &#39;GetACL&#39; = 6, &#39;SetACL&#39; = 7, &#39;SimpleList&#39; = 8, &#39;Sync&#39; = 9, &#39;Heartbeat&#39; = 11, &#39;List&#39; = 12, &#39;Check&#39; = 13, &#39;Multi&#39; = 14, &#39;Create2&#39; = 15, &#39;Reconfig&#39; = 16, &#39;CheckWatch&#39; = 17, &#39;RemoveWatch&#39; = 18, &#39;MultiRead&#39; = 22, &#39;Auth&#39; = 100, &#39;SetWatch&#39; = 101, &#39;SetWatch2&#39; = 105, &#39;AddWatch&#39; = 106, &#39;FilteredList&#39; = 500, &#39;CheckNotExists&#39; = 501, &#39;CreateIfNotExists&#39; = 502, &#39;RemoveRecursive&#39; = 503, &#39;CheckStat&#39; = 504, &#39;TryRemove&#39; = 505, &#39;FilteredListWithStatsAndData&#39; = 506, &#39;ListRecursive&#39; = 507, &#39;SessionID&#39; = 997)](/ru/reference/data-types/enum)) — Тип запроса или тип ответа.
* `path` ([String](/ru/reference/data-types/string)) — путь к узлу ZooKeeper, указанному в запросе, или пустая строка, если в запросе не требуется указывать путь.
* `data` ([String](/ru/reference/data-types/string)) — данные, записанные в узел ZooKeeper (для запросов SET и CREATE — то, что запрос пытался записать; для ответа на запрос GET — то, что было прочитано), либо пустая строка.
* `is_ephemeral` ([UInt8](/ru/reference/data-types/int-uint)) — Создаётся ли узел ZooKeeper как эфемерный узел.
* `is_sequential` ([UInt8](/ru/reference/data-types/int-uint)) — Создаётся ли узел ZooKeeper как последовательный узел.
* `version` ([Nullable(Int32)](/ru/reference/data-types/nullable)) — Версия узла ZooKeeper, которую запрос ожидает при выполнении. Поддерживается для запросов CHECK, SET, REMOVE (значение — -1, если запрос не проверяет версию, или NULL для других запросов, не поддерживающих проверку версии).
* `requests_size` ([UInt32](/ru/reference/data-types/int-uint)) — Количество запросов, входящих в multi request (это специальный запрос, состоящий из нескольких последовательных обычных запросов и выполняемый атомарно). Все запросы, входящие в multi request, имеют одинаковый xid.
* `request_idx` ([UInt32](/ru/reference/data-types/int-uint)) — Номер запроса, включённого в multi request (для multi request — 0, затем по порядку, начиная с 1).
* `zxid` ([Int64](/ru/reference/data-types/int-uint)) — идентификатор транзакции ZooKeeper. Порядковый номер, выдаваемый сервером ZooKeeper в ответ на успешно выполненный запрос (0, если запрос не был выполнен/вернул ошибку/клиент не знает, был ли запрос выполнен).
* `error` ([Nullable(Enum8(&#39;ZNOWATCHER&#39; = -121, &#39;ZNOTREADONLY&#39; = -119, &#39;ZSESSIONMOVED&#39; = -118, &#39;ZNOTHING&#39; = -117, &#39;ZCLOSING&#39; = -116, &#39;ZAUTHFAILED&#39; = -115, &#39;ZINVALIDACL&#39; = -114, &#39;ZINVALIDCALLBACK&#39; = -113, &#39;ZSESSIONEXPIRED&#39; = -112, &#39;ZNOTEMPTY&#39; = -111, &#39;ZNODEEXISTS&#39; = -110, &#39;ZNOCHILDRENFOREPHEMERALS&#39; = -108, &#39;ZBADVERSION&#39; = -103, &#39;ZNOAUTH&#39; = -102, &#39;ZNONODE&#39; = -101, &#39;ZAPIERROR&#39; = -100, &#39;ZOUTOFMEMORY&#39; = -10, &#39;ZINVALIDSTATE&#39; = -9, &#39;ZBADARGUMENTS&#39; = -8, &#39;ZOPERATIONTIMEOUT&#39; = -7, &#39;ZUNIMPLEMENTED&#39; = -6, &#39;ZMARSHALLINGERROR&#39; = -5, &#39;ZCONNECTIONLOSS&#39; = -4, &#39;ZDATAINCONSISTENCY&#39; = -3, &#39;ZRUNTIMEINCONSISTENCY&#39; = -2, &#39;ZSYSTEMERROR&#39; = -1, &#39;ZOK&#39; = 0))](/ru/reference/data-types/nullable)) — Код ошибки. Может принимать множество значений, здесь приведены лишь некоторые из них: ZOK — запрос успешно выполнен, ZCONNECTIONLOSS — соединение потеряно, ZOPERATIONTIMEOUT — истёк тайм-аут выполнения запроса, ZSESSIONEXPIRED — срок действия сеанса истёк, NULL — запрос завершён.
* `watch_type` ([Nullable(Enum8(&#39;NOTWATCHING&#39; = -2, &#39;SESSION&#39; = -1, &#39;CREATED&#39; = 1, &#39;DELETED&#39; = 2, &#39;CHANGED&#39; = 3, &#39;CHILD&#39; = 4))](/ru/reference/data-types/nullable)) — Тип события наблюдения (для ответов с op&#95;num = Watch); для остальных ответов — NULL.
* `watch_state` ([Nullable(Enum16(&#39;AUTH&#95;FAILED&#39; = -113, &#39;EXPIRED&#95;SESSION&#39; = -112, &#39;CONNECTING&#39; = 1, &#39;ASSOCIATING&#39; = 2, &#39;CONNECTED&#39; = 3, &#39;READONLY&#39; = 5, &#39;NOTCONNECTED&#39; = 999))](/ru/reference/data-types/nullable)) — статус события наблюдения (для ответов с op&#95;num = Watch), для остальных ответов: NULL.
* `path_created` ([String](/ru/reference/data-types/string)) — путь к созданному узлу ZooKeeper (в ответах на запрос CREATE); может отличаться от исходного пути, если узел создаётся в последовательном режиме.
* `stat_czxid` ([Int64](/ru/reference/data-types/int-uint)) — zxid изменения, в результате которого был создан этот узел ZooKeeper.
* `stat_mzxid` ([Int64](/ru/reference/data-types/int-uint)) — zxid изменения, которое последним изменило этот узел ZooKeeper.
* `stat_pzxid` ([Int64](/ru/reference/data-types/int-uint)) — Идентификатор транзакции изменения, которая последней изменила дочерние элементы этого узла ZooKeeper.
* `stat_version` ([Int32](/ru/reference/data-types/int-uint)) — Количество изменений данных у этого узла ZooKeeper.
* `stat_cversion` ([Int32](/ru/reference/data-types/int-uint)) — Количество изменений в дочерних узлах этого узла ZooKeeper.
* `stat_dataLength` ([Int32](/ru/reference/data-types/int-uint)) — Длина поля данных этого узла ZooKeeper.
* `stat_numChildren` ([Int32](/ru/reference/data-types/int-uint)) — количество дочерних узлов данного узла ZooKeeper.
* `children` ([Array(String)](/ru/reference/data-types/array)) — список дочерних узлов ZooKeeper (для ответов на запрос LIST).

{/*AUTOGENERATED_END*/}
