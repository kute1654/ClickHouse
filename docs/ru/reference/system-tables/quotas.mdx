<div id="description">
  ## Описание
</div>

Содержит сведения о [квотах](/ru/reference/system-tables/quotas).

<div id="columns">
  ## Столбцы
</div>

{/*AUTOGENERATED_START*/}

* `name` ([String](/ru/reference/data-types/index)) — Имя квоты.
* `id` ([UUID](/ru/reference/data-types/index)) — Идентификатор квоты.
* `storage` ([String](/ru/reference/data-types/index)) — Способ хранения квот. Возможные значения: &#39;users.xml&#39; — если квота настроена в файле users.xml, &#39;disk&#39; — если квота настроена SQL-запросом.
* `keys` ([Array(Enum8(&#39;user&#95;name&#39; = 1, &#39;ip&#95;address&#39; = 2, &#39;forwarded&#95;ip&#95;address&#39; = 3, &#39;client&#95;key&#39; = 4, &#39;normalized&#95;query&#95;hash&#39; = 7))](/ru/reference/data-types/index)) — Ключ определяет, как должна разделяться квота. Если два соединения используют одну и ту же квоту и один и тот же ключ, они совместно расходуют один и тот же объём ресурсов. Значения: [] — Все пользователи используют одну и ту же квоту, [&#39;user&#95;name&#39;] — Соединения с одинаковым именем пользователя используют одну и ту же квоту, [&#39;ip&#95;address&#39;] — Соединения с одного и того же IP-адреса используют одну и ту же квоту. [&#39;client&#95;key&#39;] — Соединения с одинаковым ключом используют одну и ту же квоту. Ключ должен быть явно передан клиентом. При использовании clickhouse-client передайте значение ключа через параметр --quota&#95;key или используйте параметр quota&#95;key в файле конфигурации клиента. При использовании HTTP-интерфейса используйте заголовок X-ClickHouse-Quota, [&#39;user&#95;name&#39;, &#39;client&#95;key&#39;] — Соединения с одинаковым client&#95;key используют одну и ту же квоту. Если ключ не передан клиентом, квота отслеживается по `user_name`, [&#39;client&#95;key&#39;, &#39;ip&#95;address&#39;] — Соединения с одинаковым client&#95;key используют одну и ту же квоту. Если ключ не передан клиентом, квота отслеживается по ip&#95;address.
* `durations` ([Array(UInt32)](/ru/reference/data-types/index)) — Длины временных интервалов в секундах.
* `apply_to_all` ([UInt8](/ru/reference/data-types/index)) — Логическое значение. Показывает, к каким пользователям применяется квота. Значения: 0 — Квота применяется к пользователям, указанным в apply&#95;to&#95;list. 1 — Квота применяется ко всем пользователям, кроме перечисленных в apply&#95;to&#95;except.
* `apply_to_list` ([Array(String)](/ru/reference/data-types/index)) — Список имён пользователей/ролей, к которым должна применяться квота.
* `apply_to_except` ([Array(String)](/ru/reference/data-types/index)) — Список имён пользователей/ролей, к которым квота не должна применяться.
* `ipv4_prefix_bits` ([Nullable(UInt8)](/ru/reference/data-types/index)) — Количество бит префикса для маскирования IPv4-адреса. Соединения из одной и той же подсети используют одну и ту же квоту. Используется только тогда, когда ключ — `ip_address` или `forwarded_ip_address`.
* `ipv6_prefix_bits` ([Nullable(UInt8)](/ru/reference/data-types/index)) — Количество бит префикса для маскирования IPv6-адреса. Соединения из одной и той же подсети используют одну и ту же квоту. Используется только тогда, когда ключ — `ip_address` или `forwarded_ip_address`.
