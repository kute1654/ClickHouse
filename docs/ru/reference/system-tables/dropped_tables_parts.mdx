<div id="description">
  ## Описание
</div>

Содержит сведения о частях удалённых таблиц [MergeTree](/ru/reference/engines/table-engines/mergetree-family/mergetree) из [system.dropped&#95;tables](/ru/reference/system-tables/dropped_tables)

Схема этой таблицы совпадает со схемой [system.parts](/ru/reference/system-tables/parts)

<div id="columns">
  ## Столбцы
</div>

{/*AUTOGENERATED_START*/}

* `partition` ([String](/ru/reference/data-types/index)) — Идентификатор партиции (строковое представление, производное от выражения ключа партиционирования таблицы).
* `name` ([String](/ru/reference/data-types/index)) —
  Имя части данных. По структуре имени части можно определить многие характеристики данных, а также шаблоны приёма и слияния. Формат имени части следующий:

```text
<partition_id>_<minimum_block_number>_<maximum_block_number>_<level>_<data_version>
```

* Определения:
  * `partition_id` - идентифицирует ключ партиционирования
  * `minimum_block_number` - идентифицирует минимальный номер блока в части. ClickHouse всегда объединяет только непрерывные блоки
  * `maximum_block_number` - идентифицирует максимальный номер блока в части
  * `level` - увеличивается на единицу с каждым дополнительным слиянием части. Уровень 0 означает, что это новая часть, которая ещё не подвергалась слиянию. Важно помнить, что все части в ClickHouse всегда неизменяемы
  * `data_version` - необязательное значение, увеличивается при мутации части (опять же, изменённые данные всегда записываются только в новую часть, поскольку части неизменяемы)

* `uuid` ([UUID](/ru/reference/data-types/index)) — UUID части данных.

* `part_type` ([String](/ru/reference/data-types/index)) — Формат хранения части данных. Возможные значения: `Wide` — каждый столбец хранится в отдельном файле, `Compact` — все столбцы хранятся в одном файле. Формат хранения данных задаётся настройками `min_bytes_for_wide_part` и `min_rows_for_wide_part` таблицы семейства MergeTree.

* `active` ([UInt8](/ru/reference/data-types/index)) — Флаг, указывающий, активна ли часть данных. Если часть данных активна, она используется в таблице. В противном случае она скоро будет удалена. Неактивные части данных появляются после операций слияния и мутации.

* `marks` ([UInt64](/ru/reference/data-types/index)) — Количество marks. Чтобы получить примерное количество строк в части данных, умножьте число marks на гранулярность индекса (обычно 8192) (эта подсказка не работает для адаптивной гранулярности).

* `rows` ([UInt64](/ru/reference/data-types/index)) — Количество строк.

* `files` ([UInt64](/ru/reference/data-types/index)) — Количество файлов в части данных.

* `bytes_on_disk` ([UInt64](/ru/reference/data-types/index)) — Общий размер всех файлов части данных в байтах.

* `data_compressed_bytes` ([UInt64](/ru/reference/data-types/index)) — Общий размер сжатых данных в части данных. Все вспомогательные файлы (например, файлы с marks) не включаются.

* `data_uncompressed_bytes` ([UInt64](/ru/reference/data-types/index)) — Общий размер несжатых данных в части данных. Все вспомогательные файлы (например, файлы с marks) не включаются.

* `primary_key_size` ([UInt64](/ru/reference/data-types/index)) — Объём памяти (в байтах), используемой значениями первичного ключа в файле primary.idx/cidx на диске.

* `marks_bytes` ([UInt64](/ru/reference/data-types/index)) — Размер файла с marks.

* `secondary_indices_compressed_bytes` ([UInt64](/ru/reference/data-types/index)) — Общий размер сжатых данных вторичных индексов в части данных. Все вспомогательные файлы (например, файлы с marks) не включаются.

* `secondary_indices_uncompressed_bytes` ([UInt64](/ru/reference/data-types/index)) — Общий размер несжатых данных вторичных индексов в части данных. Все вспомогательные файлы (например, файлы с marks) не включаются.

* `secondary_indices_marks_bytes` ([UInt64](/ru/reference/data-types/index)) — Размер файла с marks для вторичных индексов.

* `modification_time` ([DateTime](/ru/reference/data-types/index)) — Время изменения каталога с частью данных. Обычно оно соответствует времени создания части данных.

* `remove_time` ([DateTime](/ru/reference/data-types/index)) — Время, когда часть данных стала неактивной.

* `refcount` ([UInt32](/ru/reference/data-types/index)) — Количество мест, где используется часть данных. Значение больше 2 указывает на то, что часть данных используется в запросах или слияниях.

* `min_date` ([Date](/ru/reference/data-types/index)) — Минимальное значение ключа даты в части данных.

* `max_date` ([Date](/ru/reference/data-types/index)) — Максимальное значение ключа даты в части данных.

* `min_time` ([DateTime](/ru/reference/data-types/index)) — Минимальное значение ключа даты и времени в части данных.

* `max_time` ([DateTime](/ru/reference/data-types/index)) — Максимальное значение ключа даты и времени в части данных.

* `partition_id` ([String](/ru/reference/data-types/index)) — Идентификатор партиции.

* `min_block_number` ([Int64](/ru/reference/data-types/index)) — Минимальный номер частей данных, из которых после слияния сформирована текущая часть.

* `max_block_number` ([Int64](/ru/reference/data-types/index)) — Максимальный номер частей данных, из которых после слияния сформирована текущая часть.

* `level` ([UInt32](/ru/reference/data-types/index)) — Глубина дерева слияния. Ноль означает, что текущая часть была создана вставкой, а не слиянием других частей.

* `data_version` ([UInt64](/ru/reference/data-types/index)) — Число, используемое для определения того, какие мутации должны быть применены к части данных (мутации с версией выше, чем у data&#95;version).

* `primary_key_bytes_in_memory` ([UInt64](/ru/reference/data-types/index)) — Объем памяти (в байтах), используемый значениями первичного ключа. Будет равен 0, если `primary_key_lazy_load` включен и ключ не загружен. Если значение не равно нулю, эти байты находятся в самой части и учитываются в `jemalloc.mergetree_arena.active_bytes`. Они НИКОГДА не учитываются в `PrimaryIndexCacheBytes` — для каждой части это взаимоисключающие варианты: индекс находится либо в самой части (эта метрика), либо в общем `PrimaryIndexCache` (другая метрика), в зависимости от `primary_key_lazy_load` и `use_primary_key_cache`.

* `primary_key_bytes_in_memory_allocated` ([UInt64](/ru/reference/data-types/index)) — Объем памяти (в байтах), зарезервированный для значений первичного ключа. Будет равен 0, если `primary_key_lazy_load` включен и ключ не загружен. Если значение не равно нулю, включается в `jemalloc.mergetree_arena.active_bytes`. См. примечание к `primary_key_bytes_in_memory` о связи с `PrimaryIndexCacheBytes`.

* `index_granularity_bytes_in_memory` ([UInt64](/ru/reference/data-types/index)) — Объем памяти (в байтах), используемый значениями гранулярности индекса (будет равен 0 в случае primary&#95;key&#95;lazy&#95;load=1 и use&#95;primary&#95;key&#95;cache=1). Если значение не равно нулю, эти байты принадлежат части и учитываются в `jemalloc.mergetree_arena.active_bytes`.

* `index_granularity_bytes_in_memory_allocated` ([UInt64](/ru/reference/data-types/index)) — Объем памяти (в байтах), зарезервированный для значений гранулярности индекса (будет равен 0 в случае primary&#95;key&#95;lazy&#95;load=1 и use&#95;primary&#95;key&#95;cache=1). Если значение не равно нулю, включается в `jemalloc.mergetree_arena.active_bytes`.

* `is_frozen` ([UInt8](/ru/reference/data-types/index)) — Флаг, показывающий наличие резервной копии данных партиции. 1 — резервная копия существует. 0 — резервной копии нет. Подробнее см. FREEZE PARTITION.

* `database` ([String](/ru/reference/data-types/index)) — Имя базы данных.

* `table` ([String](/ru/reference/data-types/index)) — Имя таблицы.

* `engine` ([String](/ru/reference/data-types/index)) — Имя движка таблицы без параметров.

* `disk_name` ([String](/ru/reference/data-types/index)) — Имя диска, на котором хранится часть данных.

* `path` ([String](/ru/reference/data-types/index)) — Абсолютный путь к папке с файлами части данных.

* `hash_of_all_files` ([String](/ru/reference/data-types/index)) — sipHash128 сжатых файлов.

* `hash_of_uncompressed_files` ([String](/ru/reference/data-types/index)) — sipHash128 несжатых файлов (файлов с marks, index file и т. д.).

* `uncompressed_hash_of_compressed_files` ([String](/ru/reference/data-types/index)) — sipHash128 данных в сжатых файлах, как если бы они были несжатыми.

* `delete_ttl_info_min` ([DateTime](/ru/reference/data-types/index)) — Минимальное значение ключа даты и времени для правила TTL DELETE.

* `delete_ttl_info_max` ([DateTime](/ru/reference/data-types/index)) — Максимальное значение ключа даты и времени для правила TTL DELETE.

* `move_ttl_info.expression` ([Array(String)](/ru/reference/data-types/index)) — Массив выражений. Каждое выражение задает правило TTL MOVE.

* `move_ttl_info.min` ([Array(DateTime)](/ru/reference/data-types/index)) — Массив значений даты и времени. Каждый элемент описывает минимальное значение ключа для правила TTL MOVE.

* `move_ttl_info.max` ([Array(DateTime)](/ru/reference/data-types/index)) — Массив значений даты и времени. Каждый элемент описывает максимальное значение ключа для правила TTL MOVE.

* `default_compression_codec` ([String](/ru/reference/data-types/index)) — Имя кодека, используемого для сжатия этой части данных (если для столбцов не задан явный кодек).

* `recompression_ttl_info.expression` ([Array(String)](/ru/reference/data-types/index)) — Выражение TTL.

* `recompression_ttl_info.min` ([Array(DateTime)](/ru/reference/data-types/index)) — Минимальное значение вычисленного выражения TTL в пределах этой части. Используется, чтобы определить, есть ли хотя бы одна строка с истекшим TTL.

* `recompression_ttl_info.max` ([Array(DateTime)](/ru/reference/data-types/index)) — Максимальное значение вычисленного TTL-выражения в этой части. Используется, чтобы понять, истёк ли TTL у всех строк.

* `group_by_ttl_info.expression` ([Array(String)](/ru/reference/data-types/index)) — TTL-выражение.

* `group_by_ttl_info.min` ([Array(DateTime)](/ru/reference/data-types/index)) — Минимальное значение вычисленного TTL-выражения в этой части. Используется, чтобы понять, есть ли хотя бы одна строка с истёкшим TTL.

* `group_by_ttl_info.max` ([Array(DateTime)](/ru/reference/data-types/index)) — Максимальное значение вычисленного TTL-выражения в этой части. Используется, чтобы понять, истёк ли TTL у всех строк.

* `rows_where_ttl_info.expression` ([Array(String)](/ru/reference/data-types/index)) — TTL-выражение.

* `rows_where_ttl_info.min` ([Array(DateTime)](/ru/reference/data-types/index)) — Минимальное значение вычисленного TTL-выражения в этой части. Используется, чтобы понять, есть ли хотя бы одна строка с истёкшим TTL.

* `rows_where_ttl_info.max` ([Array(DateTime)](/ru/reference/data-types/index)) — Максимальное значение вычисленного TTL-выражения в этой части. Используется, чтобы понять, истёк ли TTL у всех строк.

* `projections` ([Array(String)](/ru/reference/data-types/index)) — Список имён проекций, рассчитанных для этой части.

* `visible` ([UInt8](/ru/reference/data-types/index)) — Флаг, указывающий, видима ли эта часть для SELECT-запросов.

* `creation_tid` ([Tuple(UInt64, UInt64, UUID)](/ru/reference/data-types/index)) — ID транзакции, которая создала или пытается создать этот объект.

* `removal_tid_lock` ([UInt64](/ru/reference/data-types/index)) — Хеш removal&#95;tid, используемый для блокировки объекта на удаление.

* `removal_tid` ([Tuple(UInt64, UInt64, UUID)](/ru/reference/data-types/index)) — ID транзакции, которая удалила или пытается удалить этот объект

* `creation_csn` ([UInt64](/ru/reference/data-types/index)) — CSN транзакции, которая создала этот объект

* `removal_csn` ([UInt64](/ru/reference/data-types/index)) — CSN транзакции, которая удалила этот объект

* `has_lightweight_delete` ([UInt8](/ru/reference/data-types/index)) — Флаг, указывающий, есть ли у части маска легковесного удаления.

* `last_removal_attempt_time` ([DateTime](/ru/reference/data-types/index)) — Последнее время, когда сервер пытался удалить эту часть.

* `removal_state` ([String](/ru/reference/data-types/index)) — Текущее состояние процесса удаления части.
