<div id="description">
  ## Описание
</div>

Эта таблица отсутствует, если данный узел не настроен на запуск in-process ClickHouse Keeper. Она содержит по одной строке для каждого файла снимка Raft, отслеживаемого машиной состояний in-process Keeper, включая снимки, которые в данный момент принимаются от лидер.

<div id="columns">
  ## Столбцы
</div>

{/*AUTOGENERATED_START*/}

* `last_log_index` ([UInt64](/ru/reference/data-types/int-uint)) — Последний индекс журнала, охватываемый снимком.
* `path` ([String](/ru/reference/data-types/string)) — Путь к файлу снимка на диске.
* `disk_name` ([String](/ru/reference/data-types/string)) — Имя диска, на котором хранится снимок.
* `size_bytes` ([UInt64](/ru/reference/data-types/int-uint)) — Размер файла снимка на диске.
* `last_modified_at` ([дата и время](/ru/reference/data-types/datetime)) — Время последнего изменения файла снимка.
* `is_received` ([Bool](/ru/reference/data-types/boolean)) — `true`, если снимок в данный момент принимается от лидера. Для таких строк `size_bytes` и `last_modified_at` отражают состояние частично записанного файла на текущий момент и могут быть меньше фактических.
* `exists_on_disk` ([Bool](/ru/reference/data-types/boolean)) — Находится ли файл снимка в данный момент на диске. Для завершённых снимков (`is_received` = `false`) всегда `true`, если только файл не был удалён вне системы или не повреждён; для принимаемых снимков (`is_received` = `true`) может быть `false`, пока не записан ни один байт.

{/*AUTOGENERATED_END*/}

Пример:

```sql
SELECT * FROM system.keeper_snapshots ORDER BY last_log_index;
```

```text
┌─last_log_index─┬─path──────────────────────────┬─disk_name─┬─size_bytes─┬────last_modified_at─┬─is_received─┬─exists_on_disk─┐
│           1000 │ snapshot_1000.bin.zstd        │ default   │      32468 │ 2026-05-22 14:00:00 │ false       │ true           │
│           2000 │ snapshot_2000.bin.zstd        │ default   │      48217 │ 2026-05-22 14:15:00 │ false       │ true           │
└────────────────┴───────────────────────────────┴───────────┴────────────┴─────────────────────┴─────────────┴────────────────┘
```
