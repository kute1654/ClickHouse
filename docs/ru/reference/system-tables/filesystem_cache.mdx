<div id="description">
  ## Описание
</div>

Содержит информацию обо всех записях файлового кэша для удалённых объектов.

<div id="columns">
  ## Столбцы
</div>

{/*AUTOGENERATED_START*/}

* `cache_name` ([String](/ru/reference/data-types/index)) — Имя объекта кэша. Может использоваться в командах `SYSTEM DESCRIBE FILESYSTEM CACHE <name>`, `SYSTEM DROP FILESYSTEM CACHE <name>`
* `cache_base_path` ([String](/ru/reference/data-types/index)) — Путь к базовому каталогу, где хранятся все файлы кэша (кэша, определяемого по `cache_name`).
* `cache_path` ([String](/ru/reference/data-types/index)) — Путь к конкретному файлу кэша, соответствующему сегменту файла в исходном файле
* `key` ([String](/ru/reference/data-types/index)) — Ключ кэша для сегмента файла
* `file_segment_range_begin` ([UInt64](/ru/reference/data-types/index)) — Смещение, соответствующее началу диапазона сегмента файла
* `file_segment_range_end` ([UInt64](/ru/reference/data-types/index)) — Смещение, соответствующее концу диапазона сегмента файла (включительно)
* `size` ([UInt64](/ru/reference/data-types/index)) — Размер сегмента файла
* `state` ([String](/ru/reference/data-types/index)) — Состояние сегмента файла (DOWNLOADED, DOWNLOADING, PARTIALLY&#95;DOWNLOADED, ...)
* `finished_download_time` ([DateTime](/ru/reference/data-types/index)) — Время завершения загрузки сегмента файла.
* `cache_hits` ([UInt64](/ru/reference/data-types/index)) — Число попаданий в кэш для соответствующего сегмента файла
* `references` ([UInt64](/ru/reference/data-types/index)) — Количество ссылок на соответствующий сегмент файла. Значение 1 означает, что в данный момент его никто не использует (единственная существующая ссылка находится в самом хранилище кэша)
* `downloaded_size` ([UInt64](/ru/reference/data-types/index)) — Размер загруженной части сегмента файла
* `kind` ([String](/ru/reference/data-types/index)) — Вид сегмента файла (используется для различения сегментов файла, добавленных как часть &#39;Temporary data in cache&#39;)
* `unbound` ([UInt8](/ru/reference/data-types/index)) — Внутренний флаг реализации
* `user_id` ([String](/ru/reference/data-types/index)) — Идентификатор пользователя, создавшего сегмент файла
* `segment_type` ([String](/ru/reference/data-types/index)) — Тип сегмента. Используется для разделения файлов данных (`.json`, `.txt` и т. д.) и служебных файлов данных (`.bin`, mark files).
* `file_size` ([Nullable(UInt64)](/ru/reference/data-types/index)) — Размер файла, к которому относится текущий сегмент файла
