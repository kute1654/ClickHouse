<div id="description">
  ## Описание
</div>

Содержит сведения о глобальных настройках сервера, заданных в `config.xml`.
В настоящее время в таблице показаны только настройки из первого уровня `config.xml`; вложенные конфигурации не поддерживаются (например, [logger](/ru/reference/settings/server-settings/settings#logger)).

<div id="columns">
  ## Столбцы
</div>

{/*AUTOGENERATED_START*/}

* `name` ([String](/ru/reference/data-types/index)) — Имя настройки сервера.
* `value` ([String](/ru/reference/data-types/index)) — Значение настройки сервера.
* `default` ([String](/ru/reference/data-types/index)) — Значение настройки сервера по умолчанию.
* `changed` ([UInt8](/ru/reference/data-types/index)) — Показывает, указана ли настройка в config.xml
* `description` ([String](/ru/reference/data-types/index)) — Краткое описание настройки сервера.
* `type` ([String](/ru/reference/data-types/index)) — Тип значения настройки сервера.
* `changeable_without_restart` ([Enum8(&#39;No&#39; = 0, &#39;IncreaseOnly&#39; = 1, &#39;DecreaseOnly&#39; = 2, &#39;Yes&#39; = 3)](/ru/reference/data-types/index)) — Показывает, можно ли изменить настройку во время работы.
* `is_obsolete` ([UInt8](/ru/reference/data-types/index)) — Показывает, является ли настройка устаревшей.

{/*AUTOGENERATED_END*/}

<div id="example">
  ## Пример
</div>

В следующем примере показано, как получить информацию о настройках сервера, в имени которых содержится `thread_pool`.

```sql
SELECT *
FROM system.server_settings
WHERE name LIKE '%thread_pool%'
```

Использование `WHERE changed` может быть полезно, например, когда нужно проверить,
правильно ли загружены и применяются ли настройки из файлов конфигурации.

{/* */}

```sql
SELECT * FROM system.server_settings WHERE changed AND name='max_thread_pool_size'
```
