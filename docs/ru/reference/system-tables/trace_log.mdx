<div id="description">
  ## Описание
</div>

Содержит трассировки стека, собранные [выборочным профилировщиком запросов](/ru/concepts/features/performance/troubleshoot/sampling-query-profiler).

ClickHouse создает эту таблицу, когда настроен раздел конфигурации сервера [trace&#95;log](/ru/reference/settings/server-settings/settings#trace_log). См. также настройки: [query&#95;profiler&#95;real&#95;time&#95;period&#95;ns](/ru/reference/settings/session-settings#query_profiler_real_time_period_ns), [query&#95;profiler&#95;cpu&#95;time&#95;period&#95;ns](/ru/reference/settings/session-settings#query_profiler_cpu_time_period_ns), [memory&#95;profiler&#95;step](/ru/reference/settings/session-settings#memory_profiler_step),
[memory&#95;profiler&#95;sample&#95;probability](/ru/reference/settings/session-settings#memory_profiler_sample_probability), [trace&#95;profile&#95;events](/ru/reference/settings/session-settings#trace_profile_events).

Для анализа журналов используйте функции интроспекции `addressToLine`, `addressToLineWithInlines`, `addressToSymbol` и `demangle`.

<div id="columns">
  ## Столбцы
</div>

{/*AUTOGENERATED_START*/}

* `hostname` ([LowCardinality(String)](/ru/reference/data-types/lowcardinality)) — Имя хоста сервера, выполняющего запрос.
* `event_date` ([Date](/ru/reference/data-types/date)) — Дата сэмплирования.
* `event_time` ([DateTime](/ru/reference/data-types/datetime)) — временная метка момента сэмплирования.
* `event_time_microseconds` ([DateTime64(6)](/ru/reference/data-types/datetime64)) — Временная метка момента сэмплирования с точностью до микросекунд.
* `timestamp_ns` ([UInt64](/ru/reference/data-types/int-uint)) — временная метка момента сэмплирования в наносекундах.
* `revision` ([UInt32](/ru/reference/data-types/int-uint)) — ревизия сборки сервера ClickHouse. При подключении к серверу с помощью `clickhouse-client` вы видите строку вида `Connected to ClickHouse server version 19.18.1.`. Это поле содержит `revision`, а не `version` сервера.
* `trace_type` ([Enum8(&#39;Real&#39; = 0, &#39;CPU&#39; = 1, &#39;Memory&#39; = 2, &#39;MemorySample&#39; = 3, &#39;MemoryPeak&#39; = 4, &#39;ProfileEvent&#39; = 5, &#39;JemallocSample&#39; = 6, &#39;MemoryAllocatedWithoutCheck&#39; = 7, &#39;Instrumentation&#39; = 8)](/ru/reference/data-types/enum)) — Тип трассировки: `Real` означает сбор трассировок стека по астрономическому времени. `CPU` означает сбор трассировок стека по времени CPU. `Memory` означает сбор событий выделения и освобождения памяти, когда её выделение превышает следующую водяную метку. `MemorySample` означает сбор случайных событий выделения и освобождения памяти. `MemoryPeak` означает сбор обновлений пикового потребления памяти. `ProfileEvent` означает сбор приращений profile events. `JemallocSample` означает сбор сэмплов jemalloc. `MemoryAllocatedWithoutCheck` означает сбор значительных выделений памяти (&gt;16MiB), который выполняется без учёта каких-либо ограничений памяти (только для разработчиков ClickHouse).`Instrumentation` означает трассы, собранные с помощью инструментирования через XRay.
* `cpu_id` ([UInt64](/ru/reference/data-types/int-uint)) — идентификатор процессора.
* `thread_id` ([UInt64](/ru/reference/data-types/int-uint)) — идентификатор потока.
* `thread_name` ([LowCardinality(String)](/ru/reference/data-types/lowcardinality)) — имя потока.
* `query_id` ([String](/ru/reference/data-types/string)) — идентификатор запроса, который можно использовать для получения сведений о выполнявшемся запросе из системной таблицы query&#95;log.
* `trace` ([Array(UInt64)](/ru/reference/data-types/array)) — трассировка стека на момент сэмплирования. Каждый элемент — виртуальный адрес памяти в процессе сервера ClickHouse.
* `size` ([Int64](/ru/reference/data-types/int-uint)) — для типов trace Memory, MemorySample, MemoryAllocatedWithoutCheck и MemoryPeak это объём выделенной памяти, для остальных типов trace — 0.
* `ptr` ([UInt64](/ru/reference/data-types/int-uint)) — адрес выделенного фрагмента.
* `memory_context` ([Enum8(&#39;Unknown&#39; = -1, &#39;Global&#39; = 0, &#39;User&#39; = 1, &#39;Process&#39; = 2, &#39;Thread&#39; = 3, &#39;Max&#39; = 4)](/ru/reference/data-types/enum)) — контекст трекера памяти (только для Memory/MemoryPeak): контекст `Unknown` не определён для данного trace&#95;type. `Global` обозначает контекст сервера. `User` обозначает контекст пользователя или слияния. `Process` обозначает контекст процесса (то есть запроса). `Thread` обозначает контекст потока (потока конкретного процесса). `Max` — это специальное значение, которое означает, что трекер памяти не заблокирован (для столбца blocked&#95;context).
* `memory_blocked_context` ([Enum8(&#39;Unknown&#39; = -1, &#39;Global&#39; = 0, &#39;User&#39; = 1, &#39;Process&#39; = 2, &#39;Thread&#39; = 3, &#39;Max&#39; = 4)](/ru/reference/data-types/enum)) — Контекст, для которого заблокирован трекер памяти (только для разработчиков ClickHouse): контекст `Unknown` не определён для этого trace&#95;type. `Global` обозначает контекст сервера. `User` обозначает контекст пользователя/слияния. `Process` обозначает контекст процесса (то есть запроса). `Thread` обозначает контекст потока (потока конкретного процесса). `Max` — специальное значение, которое означает, что трекер памяти не заблокирован (для столбца `blocked&#95;context`).
* `event` ([LowCardinality(String)](/ru/reference/data-types/lowcardinality)) — Для типа `trace` `ProfileEvent` это имя обновлённого события профиля, для других типов `trace` это пустая строка.
* `increment` ([Int64](/ru/reference/data-types/int-uint)) — для трассировки типа ProfileEvent это величина приращения события профиля, для остальных типов трассировки — 0.
* `symbols` ([Array(LowCardinality(String))](/ru/reference/data-types/array)) — Если символизация включена, содержит деманглированные имена символов, соответствующие `trace`. Символизацию можно включить или отключить с помощью настройки `symbolize` в разделе `trace_log` файла конфигурации сервера.
* `lines` ([Array(LowCardinality(String))](/ru/reference/data-types/array)) — Если символизация включена, содержит строки с именами файлов и номерами строк, соответствующими `trace`.
* `function_id` ([Nullable(Int32)](/ru/reference/data-types/nullable)) — Для типа trace Instrumentation — идентификатор, присвоенный функции в разделе xray&#95;instr&#95;map ELF-бинарного файла.
* `function_name` ([Nullable(String)](/ru/reference/data-types/nullable)) — Для Instrumentation типа trace — имя функции, в которую добавлена инструментация.
* `handler` ([Nullable(String)](/ru/reference/data-types/nullable)) — Для трассировок типа Instrumentation — обработчик функции с инструментированием.
* `entry_type` ([Nullable(Enum8(&#39;Entry&#39; = 0, &#39;Exit&#39; = 1))](/ru/reference/data-types/nullable)) — Для трассы типа Instrumentation: тип Entry/Exit инструментируемой функции.
* `duration_nanoseconds` ([Nullable(UInt64)](/ru/reference/data-types/nullable)) — для trace-типа Instrumentation: время выполнения функции в наносекундах.

**Псевдонимы:**

* `build_id` — псевдоним для `'E2EEB60ED9503BFFA825B628D480CCDC40B2D416'`.

{/*AUTOGENERATED_END*/}

Символизацию можно включить или отключить с помощью параметра `symbolize` в разделе `trace_log` файла конфигурации сервера.

<div id="chrome-event-trace-format">
  ## Преобразование в формат трассировки событий Chrome
</div>

Данные профилирования можно преобразовать в формат трассировки событий Chrome с помощью следующего запроса. Сохраните запрос в файл `chrome_trace.sql`:

```sql
WITH traces AS (
    SELECT * FROM system.trace_log
    WHERE event_date >= today() AND trace_type = 'Instrumentation' AND handler = 'profile'
    ORDER BY event_time, entry_type
)
SELECT
    format(
        '{{"traceEvents": [{}\n]}}',
        arrayStringConcat(
            groupArray(
                format(
                    '\n{{"name": "{}", "cat": "clickhouse", "ph": "{}", "ts": {}, "pid": 1, "tid": {}, "args": {{"query_id": "{}", "cpu_id": {}, "stack": [{}]}}}},',
                    function_name,
                    if(entry_type = 0, 'B', 'E'),
                    timestamp_ns/1000,
                    toString(thread_id),
                    query_id,
                    cpu_id,
                    arrayStringConcat(arrayMap((x, y) -> concat('"', x, ': ', y, '", '), lines, symbols))
                )
            )
        )
    )
FROM traces;
```

И выполнить его в клиенте ClickHouse, чтобы экспортировать в файл `trace.json`, который затем можно импортировать либо в [Perfetto](https://ui.perfetto.dev/), либо в [speedscope](https://www.speedscope.app/).

```bash
echo $(clickhouse client --query "$(cat chrome_trace.sql)") > trace.json
```

Мы можем опустить часть со стеком, если нужна более компактная, но менее информативная трассировка.
