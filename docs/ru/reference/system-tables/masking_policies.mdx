<div id="description">
  ## Описание
</div>

Содержит сведения обо всех политиках маскирования, определённых в системе.

<div id="columns">
  ## Столбцы
</div>

{/*AUTOGENERATED_START*/}

* `name` ([String](/ru/reference/data-types/string)) — Имя политики маскирования. Полный формат имени: `short_name ON database.table`.
* `short_name` ([String](/ru/reference/data-types/string)) — Короткое имя политики маскирования. Например, если полное имя — `mask_email ON mydb.mytable`, короткое имя — `mask_email`.
* `database` ([String](/ru/reference/data-types/string)) — Имя базы данных.
* `table` ([String](/ru/reference/data-types/string)) — Имя таблицы.
* `id` ([UUID](/ru/reference/data-types/uuid)) — Идентификатор политики маскирования.
* `storage` ([String](/ru/reference/data-types/string)) — Имя каталога, в котором хранится политика маскирования.
* `update_assignments` ([Nullable(String)](/ru/reference/data-types/nullable)) — Присваивания в UPDATE, определяющие, как должны маскироваться данные. Например: `email = '***masked***', phone = '***-***-****'`.
* `where_condition` ([Nullable(String)](/ru/reference/data-types/nullable)) — Необязательное условие WHERE, указывающее, когда следует применять маскирование.
* `priority` ([Int64](/ru/reference/data-types/int-uint)) — Приоритет применения нескольких политик маскирования. Сначала применяются политики с более высоким приоритетом. Значение по умолчанию — 0.
* `apply_to_all` ([UInt8](/ru/reference/data-types/int-uint)) — Показывает, применяется ли политика маскирования ко всем ролям и/или пользователям. 1 — да, 0 — нет.
* `apply_to_list` ([Array(String)](/ru/reference/data-types/array)) — Список ролей и/или пользователей, к которым применяется политика маскирования.
* `apply_to_except` ([Array(String)](/ru/reference/data-types/array)) — Политика маскирования применяется ко всем ролям и/или пользователям, кроме перечисленных. Заполняется только если `apply_to_all` равно 1.

{/*AUTOGENERATED_END*/}
