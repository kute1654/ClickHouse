<div id="description">
  ## Описание
</div>

История `system.histogram_metrics`. Снимок создаётся каждые `collect_interval_milliseconds`; данные сбрасываются на диск.

<div id="columns">
  ## Столбцы
</div>

{/*AUTOGENERATED_START*/}

* `hostname` ([LowCardinality(String)](/ru/reference/data-types/lowcardinality)) — Имя хоста сервера, выполняющего запрос.
* `event_date` ([Date](/ru/reference/data-types/date)) — Дата события.
* `event_time` ([DateTime](/ru/reference/data-types/datetime)) — Время события.
* `event_time_microseconds` ([DateTime64(6)](/ru/reference/data-types/datetime64)) — Время события с точностью до микросекунд.
* `metric` ([LowCardinality(String)](/ru/reference/data-types/lowcardinality)) — Имя метрики.
* `labels` ([Map(LowCardinality(String), LowCardinality(String))](/ru/reference/data-types/map)) — Метки метрик.
* `histogram` ([Map(Float64, UInt64)](/ru/reference/data-types/map)) — Накопительная гистограмма: сопоставляет верхнюю границу бакета с количеством наблюдений ≤ этого значения; включает +inf в качестве последнего бакета.
* `count` ([UInt64](/ru/reference/data-types/int-uint)) — Общее количество наблюдений, равно histogram[+inf].
* `sum` ([Float64](/ru/reference/data-types/float)) — Сумма всех наблюдаемых значений.

{/*AUTOGENERATED_END*/}

<div id="example">
  ## Пример
</div>

```sql
SELECT event_time, metric, labels, histogram
FROM system.histogram_metric_log
WHERE metric = 'keeper_response_time_ms'
ORDER BY event_time DESC
LIMIT 1
FORMAT Vertical;
```
