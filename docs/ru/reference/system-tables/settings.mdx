<div id="description">
  ## Описание
</div>

Содержит сведения о настройках сеанса текущего пользователя.

<div id="columns">
  ## Столбцы
</div>

{/*AUTOGENERATED_START*/}

* `name` ([String](/ru/reference/data-types/index)) — Имя настройки.
* `value` ([String](/ru/reference/data-types/index)) — Значение настройки.
* `changed` ([UInt8](/ru/reference/data-types/index)) — Показывает, была ли настройка явно задана в конфигурации или явно изменена.
* `description` ([String](/ru/reference/data-types/index)) — Краткое описание настройки.
* `min` ([Nullable(String)](/ru/reference/data-types/index)) — Минимальное значение настройки, если оно задано через ограничения. Если минимальное значение для настройки не задано, содержит NULL.
* `max` ([Nullable(String)](/ru/reference/data-types/index)) — Максимальное значение настройки, если оно задано через ограничения. Если максимальное значение для настройки не задано, содержит NULL.
* `disallowed_values` ([Array(String)](/ru/reference/data-types/index)) — Список недопустимых значений
* `readonly` ([UInt8](/ru/reference/data-types/index)) — Показывает, может ли текущий пользователь изменить настройку: 0 — текущий пользователь может изменить настройку, 1 — текущий пользователь не может изменить настройку.
* `type` ([String](/ru/reference/data-types/index)) — Тип значения, которое можно присвоить этой настройке.
* `default` ([String](/ru/reference/data-types/index)) — Значение настройки по умолчанию.
* `alias_for` ([String](/ru/reference/data-types/index)) — Имя исходной настройки, если эта настройка является псевдонимом другой настройки.
* `is_obsolete` ([UInt8](/ru/reference/data-types/index)) — Показывает, является ли настройка устаревшей.
* `tier` ([Enum8(&#39;Production&#39; = 0, &#39;Obsolete&#39; = 4, &#39;Experimental&#39; = 8, &#39;Beta&#39; = 12)](/ru/reference/data-types/index)) —
  Уровень поддержки этой возможности. Возможности ClickHouse распределены по уровням в зависимости от текущего этапа их
  разработки и ожиданий, которые можно связывать с их использованием:

- PRODUCTION: Возможность стабильна, безопасна в использовании и не вызывает проблем при взаимодействии с другими возможностями уровня PRODUCTION.
- BETA: Возможность стабильна и безопасна. Результат её совместного использования с другими возможностями неизвестен, и корректность не гарантируется. Приветствуются тестирование и отчёты.
- EXPERIMENTAL: Возможность находится в разработке. Предназначена только для разработчиков и энтузиастов ClickHouse. Возможность может как работать, так и не работать, и может быть удалена в любой момент.
- OBSOLETE: Больше не поддерживается. Либо уже удалена, либо будет удалена в будущих релизах.

{/*AUTOGENERATED_END*/}

  ## Пример
</div>

Следующий пример показывает, как получить информацию о настройках, название которых содержит `min_i`.

```sql
SELECT *
FROM system.settings
WHERE name LIKE '%min_insert_block_size_%'
FORMAT Vertical
```

Использование `WHERE changed` может быть полезно, например, если вы хотите проверить:

* Правильно ли загружены и применяются ли настройки из файлов конфигурации.
* Какие настройки были изменены в текущем сеансе.

```sql
SELECT * FROM system.settings WHERE changed AND name='load_balancing'
```
