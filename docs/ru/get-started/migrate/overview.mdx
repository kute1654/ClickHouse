Существует несколько вариантов переноса данных в ClickHouse Cloud, в зависимости от того, где они находятся сейчас:

* [Из самоуправляемой версии в Cloud](/ru/get-started/migrate/oss-to-cloud/clickhouse-to-cloud): используйте функцию `remoteSecure` для переноса данных
* [Другая СУБД](/ru/get-started/migrate/other-methods/clickhouse-local-etl): используйте ETL-инструмент [clickhouse-local] вместе с подходящей табличной функцией ClickHouse для вашей текущей СУБД
* [Откуда угодно!](/ru/get-started/migrate/other-methods/etl-tool-to-clickhouse): используйте один из множества популярных инструментов ETL/ELT, которые подключаются к самым разным источникам данных
* [Объектное хранилище](/ru/get-started/migrate/other-methods/object-storage-to-clickhouse): легко выполняйте вставку данных из S3 в ClickHouse

В примере [Миграция из Redshift](/ru/get-started/migrate/redshift/migration-guide) мы показываем три разных способа переноса данных в ClickHouse.
