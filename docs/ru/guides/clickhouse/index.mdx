| [Каскадные materialized views](/ru/concepts/features/materialized-views/cascading-materialized-views)                                                | Руководство о том, как создавать Materialized Views и выстраивать их каскадом, объединяя несколько исходных таблиц в одну целевую таблицу. Содержит пример использования каскадных materialized views для агрегации данных по месяцам и годам для группы доменных имён.                                                                                                                |
| [Отладка проблем с памятью](/ru/concepts/features/performance/troubleshoot/debugging-memory-issues)                                                  | Руководство по отладке проблем с памятью в ClickHouse.                                                                                                                                                                                                                                                                                                                                 |

| [Стратегии дедупликации](/ru/concepts/features/operations/insert/deduplication)                                                                      | Руководство с подробным разбором дедупликации данных — метода удаления повторяющихся строк из базы данных. В нём объясняются отличия от дедупликации на основе первичного ключа в OLTP-системах, подход ClickHouse к дедупликации и способы работы со сценариями дублирования данных в запросах к ClickHouse.                                                                          |
| [Заполнение пропусков в данных временных рядов](/ru/guides/use-cases/real-time-analytics/time-series/time-series-filling-gaps)                       | Руководство, которое даёт представление о возможностях ClickHouse по работе с данными временных рядов, включая методы заполнения пропусков для создания более полного и непрерывного представления данных временных рядов.                                                                                                                                                             |
| [Управление данными с помощью TTL (Time-to-live)](/ru/concepts/features/operations/delete/ttl)                                                       | Руководство, в котором рассматривается использование предложения `WITH FILL` для заполнения пропусков в данных временных рядов. В нём описывается, как заполнять пропуски значениями 0, как задавать начальную точку заполнения, как заполнять пропуски до определённой конечной точки и как интерполировать значения для накопительных вычислений.                                    |
| [Хранимые процедуры и параметры запросов](/ru/guides/clickhouse/data-modelling/stored-procedures-and-prepared-statements)                            | Руководство, объясняющее, что ClickHouse не поддерживает традиционные хранимые процедуры, и предлагающее рекомендуемые альтернативы, включая Пользовательские функции (UDF), параметризованные представления, materialized views и внешнюю оркестрацию. Также рассматриваются параметры запроса для безопасных параметризованных запросов (по аналогии с подготовленными операторами). |
| [Понимание выполнения запросов с помощью Analyzer](/ru/guides/clickhouse/performance-and-monitoring/understanding-query-execution-with-the-analyzer) | Руководство, которое помогает разобраться в выполнении запросов в ClickHouse, знакомя с инструментом Analyzer. В нём объясняется, как Analyzer разбивает запрос на последовательность шагов, позволяя визуализировать и отлаживать весь процесс выполнения для достижения оптимальной производительности.                                                                              |
| [Использование JOIN в ClickHouse](/ru/concepts/features/operations/select/joining-tables)                                                            | Руководство, упрощающее объединение таблиц в ClickHouse. В нём рассматриваются разные типы JOIN (`INNER`, `LEFT`, `RIGHT` и т. д.), лучшие практики эффективного использования JOIN (например, размещение меньших таблиц справа), а также приводятся сведения о внутренних алгоритмах JOIN в ClickHouse, чтобы помочь оптимизировать запросы для сложных связей между данными.         |
