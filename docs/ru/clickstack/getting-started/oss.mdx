Чтобы развернуть **ClickStack с открытым исходным кодом**, где вы сами запускаете ClickHouse и интерфейс ClickStack и управляете ими, мы предоставляем готовые Docker-образы, которые объединяют интерфейс, OpenTelemetry Collector и ClickHouse в одном контейнере, — это упрощает запуск локальной разработки, тестирования и самоупляемых развертываний.

<Tip>
  Также доступна управляемая версия ClickStack — см. [Управляемый ClickStack](/ru/clickstack/getting-started/managed).
</Tip>

Эти образы основаны на официальном Debian-пакете ClickHouse и доступны в нескольких вариантах дистрибуции для разных сценариев использования.

Самый простой вариант — **дистрибутив с одним образом**, в который входят все основные компоненты стека:

* **Интерфейс HyperDX**
* **OpenTelemetry (OTel) Collector**
* **ClickHouse**

Этот универсальный образ позволяет запустить весь стек одной командой, что делает его идеальным для тестирования, экспериментов или быстрого локального развертывания.

<Steps>
  <Step>
    ## Разверните стек с помощью Docker

    Следующая команда запустит OpenTelemetry Collector (на портах 4317 и 4318), интерфейс HyperDX (на порту 8080) и ClickHouse (8123).

    ```shell
    docker run --name clickstack -p 8123:8123 -p 8080:8080 -p 4317:4317 -p 4318:4318 clickhouse/clickstack-all-in-one:latest clickstack
    ```

    <Info>
      **Обновление имени образа**

      Образы ClickStack теперь публикуются как `clickhouse/clickstack-*` (ранее — `docker.hyperdx.io/hyperdx/*`).
    </Info>

    <Tip>
      **Сохранение данных и настроек**

      Чтобы сохранять данные и настройки между перезапусками контейнера, вы можете изменить приведённую выше команду Docker и смонтировать пути `/data/db`, `/var/lib/clickhouse` и `/var/log/clickhouse-server`.

      Например:

      ```shell
      # измените команду, чтобы смонтировать пути
      docker run \
        --name clickstack \
        -p 8123:8123 \
        -p 8080:8080 \
        -p 4317:4317 \
        -p 4318:4318 \
        -v "$(pwd)/.volumes/db:/data/db" \
        -v "$(pwd)/.volumes/ch_data:/var/lib/clickhouse" \
        -v "$(pwd)/.volumes/ch_logs:/var/log/clickhouse-server" \
        clickhouse/clickstack-all-in-one:latest
      ```
    </Tip>
  </Step>

  <Step>
    ## Перейдите в интерфейс ClickStack

    Перейдите по адресу [http://localhost:8080](http://localhost:8080), чтобы открыть интерфейс ClickStack (HyperDX).

    Создайте пользователя, указав имя пользователя и пароль, соответствующие требованиям сложности.

    <Image img="/images/use-cases/observability/hyperdx-login.webp" alt="Интерфейс HyperDX" size="lg" />

    HyperDX автоматически подключится к локальному кластеру и создаст источники данных для логов, трассировок, метрик и сеансов, чтобы вы могли сразу приступить к работе с продуктом.
  </Step>

  <Step>
    ## Изучите продукт

    После развертывания стека попробуйте один из наших примеров датасетов.

    Чтобы продолжить работу с локальным кластером:

    * [Пример датасета](/ru/clickstack/example-datasets/sample-data) - Загрузите пример датасета из нашего публичного демо. Диагностируйте простую проблему.
    * [Локальные файлы и метрики](/ru/clickstack/example-datasets/local-data) - Загружайте локальные файлы и отслеживайте систему в OSX или Linux с помощью локального OpenTelemetry Collector.

    <br />

    Либо вы можете подключиться к демо-кластеру, где можно изучить более крупный датасет:

    * [Удалённый демонстрационный датасет](/ru/clickstack/example-datasets/remote-demo-data) - Изучите демонстрационный датасет в нашем демо-сервисе ClickHouse.
  </Step>
</Steps>

<div id="alternative-deployment-models">
  ## Альтернативные модели развертывания
</div>

<div id="local-mode">
  ### Локальный режим
</div>

Локальный режим позволяет развернуть HyperDX без аутентификации.

**Аутентификация не поддерживается**.

Этот режим предназначен для быстрого тестирования, разработки, демо и отладки, когда аутентификация и сохранение настроек не нужны.

Подробнее об этой модели развертывания см. в разделе [&quot;Только локальный режим&quot;](/ru/clickstack/deployment/local-mode-only).

<div id="hosted-version">
  ### Хостируемая версия
</div>

Вы можете использовать хостируемую версию ClickStack в локальном режиме; она доступна по адресу [play-clickstack.clickhouse.com](https://play-clickstack.clickstack.com).

<div id="self-hosted-version">
  ### Самоуправляемая версия
</div>

<Steps>
  <Step>
    ### Запуск с помощью Docker

    Локальный образ для самоуправляемого развертывания уже включает настроенные OpenTelemetry Collector, интерфейс ClickStack и сервер ClickHouse. Благодаря этому можно легко получать телеметрические данные из ваших приложений и визуализировать их с минимальной дополнительной настройкой. Чтобы начать работу с самоуправляемой версией, просто запустите Docker-контейнер с пробросом соответствующих портов:

    ```shell
    docker run -p 8080:8080 clickhouse/clickstack-local:latest
    ```

    В отличие от образа &quot;Все в одном&quot;, вам не будет предложено создать пользователя, так как **локальный режим не включает аутентификацию**.
  </Step>

  <Step>
    ### Укажите учетные данные для подключения

    Чтобы подключиться к своему **внешнему кластеру ClickHouse**, можно вручную ввести учетные данные для подключения.

    Кроме того, для быстрого знакомства с продуктом можно нажать **Подключиться к демо-серверу**, чтобы получить доступ к предзагруженным датасетам и попробовать ClickStack без какой-либо настройки.

    <Image img="/images/use-cases/observability/hyperdx-2.webp" alt="Учетные данные" size="md" />

    При подключении к демо-серверу вы можете изучить датасет, следуя [инструкциям по демо-набору данных](/ru/clickstack/example-datasets/remote-demo-data).
  </Step>
</Steps>
