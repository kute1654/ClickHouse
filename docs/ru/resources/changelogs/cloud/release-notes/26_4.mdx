<div id="backward-incompatible-changes">
  ## Изменения, нарушающие обратную совместимость
</div>

<div id="query-and-syntax-changes">
  ### Изменения в запросах и синтаксисе
</div>

* Оператор `IN` теперь использует семантику точного совпадения значений для типа `Bool`: значениям `Bool` соответствуют только `0` и `1` в наборе. Ранее числовые значения больше `255` в наборе `IN` ошибочно приводились к true, поэтому `SELECT CAST(1, 'Bool') IN (256)` возвращал `1`. Теперь он корректно возвращает `0`. [#93115](https://github.com/ClickHouse/ClickHouse/pull/93115) ([Ashrith Bandla](https://github.com/ashrithb)).
* Исправлен приоритет оператора `NOT` в соответствии со стандартом SQL: теперь `NOT` имеет более низкий приоритет, чем `IS NULL`, `BETWEEN`, `LIKE` и арифметические операторы. Например, `NOT (x) IS NULL` теперь разбирается как `NOT (x IS NULL)`, а не как `(NOT x) IS NULL`. Это может изменить результаты запросов, которые опирались на прежнее (нестандартное) поведение. [#97680](https://github.com/ClickHouse/ClickHouse/pull/97680) ([Alexey Milovidov](https://github.com/alexey-milovidov)).
* `SELECT` больше нельзя использовать как неэкранированный идентификатор в элементе списка выражений `WITH`. [#101059](https://github.com/ClickHouse/ClickHouse/pull/101059) ([Aruj Bansal](https://github.com/arujbansal)).
* Оператор `IN` теперь отклоняет преобразования `Decimal` с потерей точности внутри составных типов (`Tuple`, `Array`, `Map`), что делает его поведение согласованным со сравнениями скалярных значений верхнего уровня. Ранее проверки точности применялись только к скалярным значениям верхнего уровня: например, `CAST('33.3', 'Decimal64(1)') IN (33.33)` корректно возвращал `0`, а `CAST(['33.3'], 'Array(Decimal64(1))') IN ([33.33])` ошибочно возвращал `1`. Теперь оба случая корректно возвращают `0`. [#101812](https://github.com/ClickHouse/ClickHouse/pull/101812) ([Nihal Z. Miaji](https://github.com/nihalzp)).

<div id="data-type-changes">
  ### Изменения типов данных
</div>

* Версии сериализации типов данных теперь распространяются и на вложенные типы данных. Например, версия сериализации String `with_size_stream` раньше применялась только к столбцам String верхнего уровня и элементам Tuple; теперь она применяется к любому типу String внутри любого вложенного типа, такого как Array, Map, Variant или JSON. Это поведение управляется настройкой MergeTree `propagate_types_serialization_versions_to_nested_types`, которая теперь включена по умолчанию. После этого изменения вновь созданные части данных не смогут читаться старыми версиями, но старые части можно читать в новой версии. Обновления безопасны, а откат на более старую версию — нет. [#94859](https://github.com/ClickHouse/ClickHouse/pull/94859) ([Pavel Kruglov](https://github.com/Avogar)).
* Исправлен порядок байтов в бинарной сериализации `QBit`: теперь используется формат little-endian. [#101378](https://github.com/ClickHouse/ClickHouse/pull/101378) ([Raufs Dunamalijevs](https://github.com/rienath)).

<div id="storage-and-index-changes">
  ### Изменения хранилища и индексов
</div>

* Исправлены метаданные обычных проекций, чтобы проекции с ключами сортировки из нескольких столбцов корректно распознавались. [#91352](https://github.com/ClickHouse/ClickHouse/pull/91352) ([Amos Bird](https://github.com/amosbird)).
* Исправлена проблема с файлами индекса пропуска данных: они не учитывали настройку `replace_long_file_name_to_hash`, из-за чего возникали ошибки &quot;File name too long&quot; и нарушалось чтение индексов с длинными именами. Теперь имена файлов индекса пропуска данных хешируются, если превышают `max_file_name_length`, аналогично файлам столбцов. Это обратно совместимо (новые серверы читают старые части), но при откате на более раннюю версию (или при использовании старых серверов во время поэтапного обновления) индексы с длинными именами могут игнорироваться. [#97128](https://github.com/ClickHouse/ClickHouse/pull/97128) ([Raúl Marín](https://github.com/Algunenano)).

<div id="removed-features">
  ### Удалённые возможности
</div>

* Удалён тип индекса пропуска данных `hypothesis`. Это была малоизвестная экспериментальная возможность, практически не имевшая применения. Создание таблиц с `INDEX ... TYPE hypothesis` теперь приводит к ошибке. [#96874](https://github.com/ClickHouse/ClickHouse/pull/96874) ([Alexey Milovidov](https://github.com/alexey-milovidov)).
* Удалена экспериментальная функция `detectProgrammingLanguage`. [#99567](https://github.com/ClickHouse/ClickHouse/pull/99567) ([Alexey Milovidov](https://github.com/alexey-milovidov)).

<div id="settings-and-configuration-changes">
  ### Изменения настроек и конфигурации
</div>

* Значение `mysql_datatypes_support_level` по умолчанию изменено с пустого на `decimal,datetime64,date2Date32`, что включает корректное сопоставление MySQL `DATE` с `Date32`, `DECIMAL`/`NUMERIC` с `Decimal`, а `DATETIME`/`TIMESTAMP` с указанной точностью — с `DateTime64`. Ранее столбцы MySQL `DATE` сопоставлялись с `Date`, который не может представлять даты до 1970-01-01, что приводило к повреждению данных. [#97716](https://github.com/ClickHouse/ClickHouse/pull/97716) ([Alexey Milovidov](https://github.com/alexey-milovidov)).

<div id="insert-and-deduplication-changes">
  ### Изменения во вставке и дедупликации
</div>

* Async insert теперь включен по умолчанию — ClickHouse по умолчанию объединяет все небольшие вставки в батчи. Это поведение регулируется настройкой `compatibility`: если установить `compatibility` на версию ниже 26.2, будет восстановлено прежнее значение по умолчанию — `false`. Async inserts можно включать и отключать на нескольких уровнях: в профилях пользователей в config, для сеанса, для запроса или для таблицы семейства MergeTree. [#97590](https://github.com/ClickHouse/ClickHouse/pull/97590) ([Sema Checherinda](https://github.com/CheSema)).

<div id="system-table-changes">
  ### Изменения в системных таблицах
</div>

* В `system.query_log` возвращён столбец `skip_indexes`. [#101018](https://github.com/ClickHouse/ClickHouse/pull/101018) ([Alexey Milovidov](https://github.com/alexey-milovidov)).

<div id="other-breaking-changes">
  ### Другие изменения, нарушающие обратную совместимость
</div>

* `mergeTreeAnalyzeIndexes{,UUID}` теперь принимает массив имён частей вместо regexp, поскольку регулярные выражения работают медленно (экспериментальная возможность). [#98474](https://github.com/ClickHouse/ClickHouse/pull/98474) ([Azat Khuzhin](https://github.com/azat)).
* Библиотека H3 была обновлена до v4, что повышает точность вычисления длины, площади и других метрик. Это изменение нарушает обратную совместимость, поскольку новые результаты отличаются от прежних. [#100348](https://github.com/ClickHouse/ClickHouse/pull/100348) ([Alexey Milovidov](https://github.com/alexey-milovidov)).
* Изменён способ обработки виртуальных столбцов таблицей `Merge`. Если базовая таблица содержит `_table` или `_database`, эти столбцы читаются из хранилища; в противном случае они заполняются после этапа чтения на шаге expression. [#101742](https://github.com/ClickHouse/ClickHouse/pull/101742) ([Mikhail Artemenko](https://github.com/Michicosun)).

<div id="new-features">
  ## Новые возможности
</div>

<div id="functions">
  ### Функции
</div>

* Добавлена функция `naturalSortKey(s)` для ключей естественной сортировки. [#90322](https://github.com/ClickHouse/ClickHouse/pull/90322) ([Nazarii Piontko](https://github.com/nazarii-piontko)).
* Добавлена функция `arrayAutocorrelation(arr [, max_lag])`, которая вычисляет нормализованную автокорреляцию числового массива для каждого лага. Поддерживаются массивы целых чисел, чисел с плавающей запятой и Decimal. [#94776](https://github.com/ClickHouse/ClickHouse/pull/94776) ([Wenyu Chen](https://github.com/wenyuchen96)).
* Для JSON type теперь поддерживается функция `has()` для проверки существования path, аналогично `Map`. [#96927](https://github.com/ClickHouse/ClickHouse/pull/96927) ([DQ](https://github.com/il9ue)).
* Добавлена табличная функция `mergeTreeTextIndex(database, table, index)`, которая читает данные напрямую из текстового индекса. Её можно использовать для интроспекции или для выполнения агрегаций по данным текстового индекса. [#97003](https://github.com/ClickHouse/ClickHouse/pull/97003) ([Anton Popov](https://github.com/CurtizJ)).
* Добавлена SQL-функция `obfuscateQuery`. [#98305](https://github.com/ClickHouse/ClickHouse/pull/98305) ([Xuewei Wang](https://github.com/Sallery-X)).
* Добавлены функции `caseFoldUTF8` и `removeDiacriticsUTF8` для преобразования Unicode к единому регистру и удаления диакритических знаков. [#98973](https://github.com/ClickHouse/ClickHouse/pull/98973) ([George Larionov](https://github.com/george-larionov)).
* Функция `printf` теперь поддерживает неконстантные строки формата, что позволяет задавать разный шаблон форматирования для каждой строки на основе значений столбцов. [#98991](https://github.com/ClickHouse/ClickHouse/pull/98991) ([Yash](https://github.com/Onyx2406)).
* Добавлена функция `highlight()`, которая оборачивает вхождения поисковых терминов в текстовой строке HTML-тегами (по умолчанию `<em>`/`</em>`). Поддерживаются ASCII-регистронезависимое сопоставление, автоматическое слияние перекрывающихся совпадений и пользовательские открывающие/закрывающие теги. [#99131](https://github.com/ClickHouse/ClickHouse/pull/99131) ([Peng](https://github.com/fastio)).
* Добавлена строковая функция `normalizeUTF8NFKCCasefold` для Unicode-нормализации NFKC&#95;Casefold, которая объединяет нормализацию NFKC с приведением к единому регистру. [#99276](https://github.com/ClickHouse/ClickHouse/pull/99276) ([George Larionov](https://github.com/george-larionov)).
* Для полнотекстовых индексов и функции `tokens` добавлен токенизатор `unicode_word`. Он разбивает текст по правилам границ слов Unicode: ASCII-слова формируются с использованием символов-соединителей (подчёркивание, двоеточие, точка, одиночная кавычка), а не-ASCII-символы Unicode становятся односимвольными токенами. Настраиваемые стоп-слова по умолчанию соответствуют распространённым знакам пунктуации CJK. [#99357](https://github.com/ClickHouse/ClickHouse/pull/99357) ([Amos Bird](https://github.com/amosbird)).
* Добавлена функция `arrayTranspose`, которая принимает двумерный массив (матрицу) и транспонирует его. [#101214](https://github.com/ClickHouse/ClickHouse/pull/101214) ([Vitaly Baranov](https://github.com/vitlibar)).
* Добавлена функция `hasPhrase` (псевдоним `matchPhrase`) для поиска по фразам (непрерывным последовательностям токенов). Поиск выполняется методом полного перебора и пока не поддерживается текстовым индексом. [#101997](https://github.com/ClickHouse/ClickHouse/pull/101997) ([Elmi Ahmadov](https://github.com/ahmadov)).
* `dotProduct` добавлена как поддерживаемая функция расстояния для индексов векторного сходства. Направление сортировки теперь проверяется для всех функций расстояния, чтобы предотвратить незаметное появление некорректных результатов. [#102254](https://github.com/ClickHouse/ClickHouse/pull/102254) ([Renzo](https://github.com/RenzoMXD)).
* Функция `stem` теперь может выполнять стемминг всех слов/токенов в столбцах `String`, `FixedString`, `Array([Fixed]String)`, `Nullable`, `LowCardinality` и `Const`. [#99137](https://github.com/ClickHouse/ClickHouse/pull/99137) ([Jimmy Aguilar Mena](https://github.com/Ergus)).
* Функция `stem` больше не является экспериментальной (раньше для этого нужно было включить `allow_experimental_nlp_functions`). [#102399](https://github.com/ClickHouse/ClickHouse/pull/102399) ([Jimmy Aguilar Mena](https://github.com/Ergus)).
* Добавлена функция `JSONAllValues`, которая возвращает все значения из столбца `JSON` в виде `Array(String)`; значения сериализуются в текстовое представление и упорядочиваются по именам путей. Также добавлена поддержка текстового индекса для выражения `JSONAllValues` на столбцах `JSON`: если создать текстовый индекс для `JSONAllValues(json_column)`, он будет автоматически использоваться для фильтрации запросов по подстолбцам `JSON` (например, `json_column.key1 = 'value'`). [#100730](https://github.com/ClickHouse/ClickHouse/pull/100730) ([Anton Popov](https://github.com/CurtizJ)).
* В оператор `EXTRACT` добавлены единицы, совместимые с PostgreSQL: `EPOCH`, `DOW`, `DOY`, `ISODOW`, `ISOYEAR`, `WEEK`, `CENTURY`, `DECADE` и `MILLENNIUM`. Также исправлен `EXTRACT(WEEK FROM date)`, который ранее вызывал ошибку. [#100274](https://github.com/ClickHouse/ClickHouse/pull/100274) ([Alexey Milovidov](https://github.com/alexey-milovidov)).
* Добавлена поддержка функции `toDaysInMonth()` через синтаксический сахар парсера для стандартного SQL-синтаксиса функции `OVERLAY`. Функция `overlay` уже существует; это добавляет поддержку формы с ключевыми словами `PLACING`, `FROM` и `FOR` в качестве разделителей. [#101681](https://github.com/ClickHouse/ClickHouse/pull/101681) ([Desel72](https://github.com/Desel72)).
* Значения `Date` и `Date32` теперь можно складывать со значениями `Time` и `Time64` с помощью оператора `+`, получая результат типа `DateTime` или `DateTime64`. Например, `SELECT toDate('2024-01-15') + toTime('14:30:25')` возвращает `2024-01-15 14:30:25`. Результат вычисляется в часовом поясе сеанса, а значения вне допустимого диапазона обрабатываются в соответствии с настройкой `date_time_overflow_behavior`. [#102421](https://github.com/ClickHouse/ClickHouse/pull/102421) ([Nihal Z. Miaji](https://github.com/nihalzp)).

<div id="sql-and-query-features">
  ### SQL и возможности запросов
</div>

* Если параметр имеет тип `Nullable` и не указан, теперь считается, что его значение равно `NULL`. [#93869](https://github.com/ClickHouse/ClickHouse/pull/93869) ([Vikash Kumar](https://github.com/vikashkumar2020)).
* ClickHouse теперь может отсекать целые части данных в запросах `SELECT` на основе статистики min/max. [#94140](https://github.com/ClickHouse/ClickHouse/pull/94140) ([zoomxi](https://github.com/zoomxi)).
* Добавлена поддержка Materialized CTEs, которые вычисляются только один раз во время выполнения запроса, а их результаты сохраняются во временных таблицах. [#94849](https://github.com/ClickHouse/ClickHouse/pull/94849) ([Dmitry Novik](https://github.com/novikd)).
* Некоторые функции теперь можно использовать в SQL без скобок. [#95949](https://github.com/ClickHouse/ClickHouse/pull/95949) ([Aly Kafoury](https://github.com/AlyHKafoury)).
* Для типа `JSON` добавлен комбинированный синтаксис подстолбцов `json.@path`. Он возвращает скалярное значение как `Dynamic`, если по пути находится скаляр, вложенный объект как `Dynamic`, если по пути находится вложенный объект, или `NULL`, если путь отсутствует. [#98788](https://github.com/ClickHouse/ClickHouse/pull/98788) ([Pavel Kruglov](https://github.com/Avogar)).
* Для выражений с подзапросами добавлено ключевое слово `SOME`. Оно работает так же, как `ANY`. [#99842](https://github.com/ClickHouse/ClickHouse/pull/99842) ([Artem Kytkin](https://github.com/Vinceent)).
* Добавлена поддержка `SET TIME ZONE 'tz'` как псевдонима для `SET session_timezone`. [#99883](https://github.com/ClickHouse/ClickHouse/pull/99883) ([phulv94](https://github.com/phulv94)).
* Добавлена поддержка стандартной для SQL конструкции `VALUES` как табличного выражения в `FROM`, например: `SELECT * FROM (VALUES (1, 'a'), (2, 'b')) AS t(id, val)`. [#100143](https://github.com/ClickHouse/ClickHouse/pull/100143) ([Desel72](https://github.com/Desel72)).
* Добавлена поддержка составных литералов interval по стандарту SQL с квалификаторами диапазона `TO`, например `INTERVAL '1:30' HOUR TO MINUTE`; внутри они раскладываются в суммы интервалов. [#100453](https://github.com/ClickHouse/ClickHouse/pull/100453) ([Desel72](https://github.com/Desel72)).
* Добавлена команда `SYSTEM FLUSH OBJECT STORAGE QUEUE db.table PATH 'x'` для режимов ordered и unordered. [#100709](https://github.com/ClickHouse/ClickHouse/pull/100709) ([Bharat Nallan](https://github.com/bharatnc)).

<div id="table-engines-and-storage">
  ### Движки таблиц и хранилище
</div>

* Добавлена поддержка инкрементального чтения для движка таблицы `Paimon` с отслеживанием прогресса снимков через Keeper, включая целевое чтение дельт снимков с помощью `paimon_target_snapshot_id`. [#93655](https://github.com/ClickHouse/ClickHouse/pull/93655) ([XiaoBinMu](https://github.com/Binnn-MX)).
* Добавлена поддержка дополнительного ZooKeeper в `DatabaseReplicated`. [#95590](https://github.com/ClickHouse/ClickHouse/pull/95590) ([RinChanNOW](https://github.com/RinChanNOWWW)).
* Добавлена поддержка чтения типов временных меток Iceberg v3 с наносекундной точностью (`timestamp_ns` и `timestamptz_ns`), сопоставляемых с `DateTime64(9)` и `DateTime64(9, 'UTC')`. Это позволяет ClickHouse читать данные Iceberg с такими типами в схеме; однако это не означает полной поддержки Iceberg v3 и не включает запись в Iceberg v3 с этими типами. [#97132](https://github.com/ClickHouse/ClickHouse/pull/97132) ([Dmitry Kovalev](https://github.com/dk-github)).
* Добавлен `ALTER TABLE ... EXECUTE expire_snapshots('<timestamp>')` для таблиц Iceberg. [#97904](https://github.com/ClickHouse/ClickHouse/pull/97904) ([murphy-4o](https://github.com/murphy-4o)).
* Добавлен новый SLRU-кэш для метаданных Parquet, чтобы повысить производительность чтения и убрать необходимость повторно загружать файлы только ради чтения метаданных. [#98140](https://github.com/ClickHouse/ClickHouse/pull/98140) ([Grant Holly](https://github.com/grantholly-clickhouse)).
* Добавлена поддержка индекса пропуска данных MergeTree для JSON-столбцов с использованием `JSONAllPaths` и типов индексов `bloom_filter`, `tokenbf_v1`, `ngrambf_v1` и `text` (инвертированный), что позволяет пропускать гранулы на основе набора JSON-путей, присутствующих в каждой грануле. [#98886](https://github.com/ClickHouse/ClickHouse/pull/98886) ([Pavel Kruglov](https://github.com/Avogar)).
* Добавлен индекс проекции `commit_order`, который переупорядочивает данные в порядке вставки. [#99004](https://github.com/ClickHouse/ClickHouse/pull/99004) ([Mikhail Artemenko](https://github.com/Michicosun)).
* Добавлена сериализация Map по бакетам для столбцов `Map` в MergeTree (`map_serialization_version = 'with_buckets'`). Ключи разбиваются на бакеты на основе хеша, так что при чтении одного ключа (`m['key']`) считывается только один бакет, а не весь столбец, что даёт ускорение в 2–49 раз для выборки по одному ключу в зависимости от размера map. Количеством бакетов и стратегией бакетизации можно управлять с помощью новых настроек MergeTree `map_serialization_version`, `max_buckets_in_map`, `map_buckets_strategy`, `map_buckets_coefficient` и `map_buckets_min_avg_size`. Настройка `map_serialization_version_for_zero_level_parts` сохраняет базовую сериализацию для вставок, чтобы избежать дополнительных накладных расходов при записи, тогда как для слитых частей используются бакеты. Индексы пропуска данных для `mapKeys`/`mapValues` поддерживаются при сериализации Map по бакетам, а `optimize_functions_to_subcolumns` переписывает `m['key']` в чтение подстолбца для конкретного ключа, когда бакеты доступны. [#99200](https://github.com/ClickHouse/ClickHouse/pull/99200) ([Pavel Kruglov](https://github.com/Avogar)).

* Реализованы квоты по хешу нормализованного запроса для защиты публичных сервисов ClickHouse от злоупотреблений. `NORMALIZED_QUERY_HASH` поддерживается как тип quota key — для каждого уникального нормализованного запроса создаётся отдельный бакет квоты, поэтому `CREATE QUOTA q KEYED BY normalized_query_hash` отслеживает каждый такой запрос отдельно. `QUERIES_PER_NORMALIZED_HASH` поддерживается как тип ресурса квоты — он ограничивает максимальное число выполнений любого отдельного нормализованного запроса в пределах интервала, поэтому `MAX queries_per_normalized_hash = 100` не позволяет одному и тому же шаблону запроса выполняться более 100 раз. Для `materialize_statistics_on_insert` значением по умолчанию теперь является `false`, поэтому статистика строится во время слияний, а не в момент вставки, что снижает накладные расходы на вставку. <div id="storage-and-io-performance">
  ### Производительность хранилища и ввода-вывода
</div>

* Снижена конкуренция за блокировки при операциях только для чтения в таблицах ReplicatedMergeTree с завершёнными мутациями. [#95771](https://github.com/ClickHouse/ClickHouse/pull/95771) ([Eduard Karacharov](https://github.com/korowa)).
* Iceberg теперь может асинхронно заранее заполнять кэш метаданными; это включается настройкой `iceberg_metadata_async_prefetch_period_ms` при создании таблицы. Запросы `SELECT` к таблицам Iceberg теперь могут задавать параметр `iceberg_metadata_staleness_ms`, который позволяет ClickHouse использовать кэшированную версию метаданных, если она свежее указанного порога устаревания; в противном случае для получения последних метаданных выполняется запрос к удалённому каталогу Iceberg. Это позволяет исключить обращения к каталогу Iceberg во время обработки запросов, что даёт заметный прирост производительности. [#96191](https://github.com/ClickHouse/ClickHouse/pull/96191) ([Arsen Muk](https://github.com/arsenmuk)).
* Упорядоченный режим S3Queue теперь использует S3 `ListObjectsV2` `StartAfter`, чтобы избежать повторного перечисления всей истории по префиксу и сократить число вызовов `ListObjects`. [#96370](https://github.com/ClickHouse/ClickHouse/pull/96370) ([Venkata Vineel](https://github.com/vyalamar)).
* Включено параллельное чтение из движка таблицы YTsaurus. [#97343](https://github.com/ClickHouse/ClickHouse/pull/97343) ([MikhailBurdukov](https://github.com/MikhailBurdukov)).
* Улучшена производительность озер данных. В предыдущих версиях чтение из Объектного хранилища не подстраивало размер конвейера под число потоков обработки. [#99548](https://github.com/ClickHouse/ClickHouse/pull/99548) ([Alexey Milovidov](https://github.com/alexey-milovidov)).
* Разрешена предвыборка при чтении удалённого файла через кэш страниц в пространстве пользователя. [#99919](https://github.com/ClickHouse/ClickHouse/pull/99919) ([Alexey Milovidov](https://github.com/alexey-milovidov)).
* Исправлена существенная регрессия производительности INSERT при `deduplicate_insert=enable` (по умолчанию с 26.2): вычисление хеша данных перенесено с этапа squashing в sink, а также использовано батч-хеширование столбцов через `updateHashWithValueRange`, что снизило накладные расходы примерно с ~2.5s до ~0.5s для 5M строк и 22 столбцов. [#101494](https://github.com/ClickHouse/ClickHouse/pull/101494) ([Sema Checherinda](https://github.com/CheSema)).
