<div id="december-20-2022">
  ## 20 декабря 2022 г.
</div>

Этот релиз включает бесшовный вход администраторов в консоль SQL, повышенную производительность при холодных чтениях и улучшенный коннектор Metabase для ClickHouse Cloud.

<div id="console-changes-27">
  ### Изменения в консоли
</div>

* Для пользователей с ролью Admin включён беспрепятственный доступ к консоли SQL
* Ролью по умолчанию для новых приглашённых стала &quot;Administrator&quot;
* Добавлен опрос в процессе онбординга

<div id="reliability-and-performance-3">
  ### Надежность и производительность
</div>

* Добавлен механизм повторных попыток для длительно выполняющихся запросов на вставку, чтобы восстанавливаться после сбоев сети
* Повышена производительность холодных чтений

<div id="integrations-changes-26">
  ### Изменения в интеграциях
</div>

* [Плагин Metabase](/ru/integrations/connectors/data-visualization/metabase-and-clickhouse) получил долгожданное крупное обновление до версии v0.9.1. Теперь он совместим с последней версией Metabase и был тщательно протестирован на совместимость с ClickHouse Cloud.

<div id="december-6-2022---general-availability">
  ## 6 декабря 2022 г. — общая доступность
</div>

ClickHouse Cloud теперь готов к использованию в продакшне: соответствует требованиям SOC 2 Type II, предоставляет SLA по времени безотказной работы для продакшн-нагрузок и публичную страницу статуса. В этот релиз также вошли такие важные новые возможности, как интеграция с AWS Marketplace, консоль SQL — рабочее пространство для исследования данных для пользователей ClickHouse, и ClickHouse Academy — самостоятельное обучение в ClickHouse Cloud. Подробнее читайте в этом [блоге](https://clickhouse.com/blog/clickhouse-cloud-generally-available).

<div id="production-ready">
  ### Готово для продакшна
</div>

* Соответствие требованиям SOC2 Type II (подробности в [блоге](https://clickhouse.com/blog/clickhouse-cloud-is-now-soc-2-type-ii-compliant) и [Trust Center](https://trust.clickhouse.com/))
* Публичная [страница статуса](https://status.clickhouse.com/) для ClickHouse Cloud
* SLA по времени доступности доступен для продакшн-сценариев
* Доступно в [AWS Marketplace](https://aws.amazon.com/marketplace/pp/prodview-jettukeanwrfc)

<div id="major-new-capabilities">

* Представлена консоль SQL — рабочее пространство для исследования данных для пользователей ClickHouse

<div id="general-changes-4">
  ### Общие изменения
</div>

* Пользователи больше не могут изменять настройки `allow_suspicious_low_cardinality_types`, `allow_suspicious_fixed_string_types` и `allow_suspicious_codecs` (по умолчанию — false) из соображений стабильности.

<div id="console-changes-27">
  ### Изменения в консоли

* Для платных клиентов максимальный объем памяти для вертикального масштабирования в режиме самообслуживания увеличен до 720 ГБ
* Улучшен процесс восстановления из резервной копии: теперь в нем можно задавать правила IP Access List и пароль

* Улучшена обработка ошибок при загрузке файлов
* Улучшены сценарии администрирования биллинга

<div id="clickhouse-2210-version-upgrade">
  ### Обновление до версии ClickHouse 22.10
</div>

* Улучшены слияния при работе с объектными хранилищами за счёт повышения порога &quot;too many parts&quot; при наличии большого количества крупных частей (не менее 10 GiB). Это позволяет хранить до PB данных в одной партиции одной таблицы.
* Улучшено управление слиянием с помощью настройки `min_age_to_force_merge_seconds`, которая позволяет выполнять слияние по истечении заданного временного порога.
* Добавлен совместимый с MySQL синтаксис для сброса настроек: `SET setting_name = DEFAULT`.
* Добавлены функции для кодирования по кривой Мортона, хеширования целых чисел Java и генерации случайных чисел.
* Полный список изменений см. в [подробном changelog для 22.10](#clickhouse-2210-version-upgrade).

<div id="october-25-2022">
  ## 25 октября 2022 г.
</div>

Этот релиз значительно снижает потребление вычислительных ресурсов для небольших рабочих нагрузок, уменьшает стоимость вычислений (подробности см. на странице [pricing](https://clickhouse.com/pricing)), повышает стабильность благодаря более удачным значениям по умолчанию и улучшает разделы Billing и Usage в консоли ClickHouse Cloud.

<div id="general-changes-4">
  ### Общие изменения
</div>

* Минимальный объём памяти сервиса снижен до 24G
* Тайм-аут простоя сервиса сокращён с 30 минут до 5 минут

<div id="configuration-changes-3">
  ### Изменения конфигурации
</div>

* Значение max&#95;parts&#95;in&#95;total уменьшено со 100k до 10k. Значение по умолчанию для настройки `max_parts_in_total` у таблиц семейства MergeTree было снижено со 100 000 до 10 000. Причина этого изменения в том, что, как мы заметили, большое количество частей данных, скорее всего, приводит к медленному запуску сервисов в облаке. Большое количество частей обычно указывает на слишком детализированный ключ партиционирования, который чаще всего выбирают случайно, поэтому этого следует избегать. Изменение значения по умолчанию позволит выявлять такие случаи раньше.

<div id="console-changes-27">
  ### Изменения в консоли
</div>

* Добавлена более подробная информация об использовании кредитов в разделе Billing для пользователей пробной версии
* Улучшены всплывающие подсказки и справочный текст, а также добавлена ссылка на страницу с ценами в разделе Usage
* Упрощено переключение параметров фильтрации IP-адресов
* В облачную консоль добавлена кнопка повторной отправки письма с подтверждением
