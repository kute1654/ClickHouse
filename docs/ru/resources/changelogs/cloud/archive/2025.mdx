<div id="december-19-2025">
  ## 19 декабря 2025 г.
</div>

* AWS ap-south-1 теперь поддерживает запуск сервисов, соответствующих требованиям PCI.
* **Закрытая предварительная версия единой пользовательской идентификации**
  Клиенты, которые хотят управлять пользователями базы данных из консоли, могут включить новый метод аутентификации для SQL-консоли.
  Это позволит опробовать новый метод аутентификации, пока мы продолжаем работу над добавлением управления пользователями базы данных в консоль.
* **Неупорядоченный режим теперь доступен в S3 ClickPipes**:
  Теперь клиенты могут загружать данные из Amazon S3 в ClickHouse Cloud в любом порядке для аналитики на основе событий.
  Файлы больше не нужно обрабатывать в лексикографическом порядке. Подробнее — в [блоге](https://clickhouse.com/blog/clickpipes-s3-unordered-mode).
* Коннектор Fivetran недавно перешёл в статус бета. Если вы используете Fivetran и хотите настроить ClickHouse как пункт назначения, ознакомьтесь с этой [документацией](https://fivetran.com/docs/destinations/clickhouse/setup-guide).

<div id="december-12-2025">
  ## 12 декабря 2025 г.
</div>

* **Самостоятельная настройка SAML SSO**

  Теперь клиенты Enterprise могут выполнить настройку SAML в консоли без обращения в поддержку.
  Кроме того, клиенты, использующие SAML, теперь могут задать роль по умолчанию, которая будет назначаться новым пользователям, добавленным через их провайдер идентификации, а также настроить индивидуальные параметры тайм-аута сеанса.
  Подробнее см. в нашей [документации](/ru/products/cloud/guides/security/cloud-access-management/saml-sso-setup).
* **Максимальный размер реплики и ограничения масштабирования в Azure**

  Теперь клиенты могут установить максимальный размер реплики до 356 GiB во всех регионах Azure, кроме `eastus2`, где максимально доступный размер реплики составляет 120 GiB.

<div id="november-7-2025">
  ## 7 ноября 2025
</div>

* Консоль ClickHouse Cloud теперь поддерживает настройку размеров реплик с шагом 1 vCPU и 4 GiB.
  Эти параметры доступны как при настройке нового сервиса, так и при задании минимального и максимального размеров реплик на странице настроек.
* Пользовательские аппаратные профили (доступные на уровне Enterprise) теперь поддерживают автоматический переход в режим простоя.
* ClickHouse Cloud теперь предлагает упрощенный процесс покупки через AWS Marketplace с отдельными вариантами для [оплаты по мере использования](https://aws.amazon.com/marketplace/pp/prodview-p4gwofrqpkltu?sr=0-2\&ref_=beagle\&applicationId=AWSMPContessa) и [контрактов с обязательствами по объему расходов](https://aws.amazon.com/marketplace/pp/prodview-4qyeihstyym2s?sr=0-3\&ref_=beagle\&applicationId=AWSMPContessa).
* В ClickHouse Cloud для пользователей ClickStack теперь доступны оповещения.
  Теперь вы можете создавать оповещения и управлять ими напрямую в интерфейсе HyperDX для журналов, метрик и трассировок без дополнительной настройки, инфраструктуры, отдельных сервисов и конфигурации. Оповещения интегрируются со Slack, PagerDuty и другими сервисами.
  Подробнее см. в [документации по оповещениям](/ru/clickstack/features/alerts).

<div id="october-17-2025">
  ## 17 октября 2025 г.
</div>

* **Мониторинг сервиса — панель использования ресурсов**
  Отображение метрик загрузки CPU и использования памяти изменится: вместо средних значений теперь будет показываться максимальное значение за выбранный период времени, чтобы лучше выявлять случаи недостаточного выделения ресурсов.
  Кроме того, метрика загрузки CPU будет показывать использование CPU на уровне Kubernetes, что точнее соответствует метрике, которую использует автоскейлер ClickHouse Cloud.
* **Внешние бакеты**
  ClickHouse Cloud теперь позволяет экспортировать резервные копии напрямую в аккаунт вашего облачного провайдера.
  Подключите внешний бакет хранилища — AWS S3, Google Cloud Storage или Azure Blob Storage — и управляйте резервными копиями самостоятельно.

<div id="august-29-2025">
  ## 29 августа 2025 г.

* В [ClickHouse Cloud Azure Private Link](/ru/products/cloud/guides/security/connectivity/private-networking/azure-privatelink) для идентификации ресурсов вместо фильтров Resource GUID теперь используются фильтры Resource ID. Вы по-прежнему можете использовать устаревший Resource GUID, поскольку сохраняется обратная совместимость, но мы рекомендуем перейти на фильтры Resource ID. Подробности о переходе см. в [документации](/ru/products/cloud/guides/security/connectivity/private-networking/azure-privatelink#obtaining-private-endpoint-resourceid) по Azure Private Link.

<div id="august-22-2025">
  ## 22 августа 2025 г.
</div>

* **Коннектор ClickHouse для AWS Glue**
  Теперь вы можете использовать официальный [ClickHouse Connector for AWS Glue](/ru/integrations/connectors/data-ingestion/AWS/glue), доступный в [AWS Marketplace](https://aws.amazon.com/marketplace/pp/prodview-eqvmuopqzdg7s). Он использует бессерверный движок AWS Glue на базе Apache
  Spark для извлечения, преобразования и загрузки данных при интеграции ClickHouse с другими источниками данных. Чтобы
  начать, ознакомьтесь с анонсом в [блоге](http://clickhouse.com/blog/clickhouse-connector-aws-glue), где показано, как создавать таблицы, записывать и читать данные между ClickHouse и Spark.
* **Изменение минимального количества реплик в сервисе**
  Сервисы, масштаб которых был увеличен, теперь можно [уменьшить обратно](/ru/products/cloud/features/autoscaling/overview) до одной реплики (раньше минимум составлял 2 реплики). Примечание: сервисы с одной репликой имеют меньшую доступность и не рекомендуются для использования в продакшне.
* ClickHouse Cloud по умолчанию начнет отправлять уведомления о масштабировании сервиса и обновлениях его версии для ролей администраторов. Параметры этих уведомлений можно изменить в настройках уведомлений.

<div id="august-13-2025">
  ## 13 августа 2025 г.

* **ClickPipes for MongoDB CDC теперь доступен в рамках закрытой предварительной версии**
  Теперь вы можете использовать ClickPipes для репликации данных из MongoDB в ClickHouse Cloud за несколько кликов, что позволяет
  выполнять Real-time аналитику без использования внешних ETL-инструментов. Коннектор поддерживает непрерывную
  репликацию, а также разовые миграции и совместим с MongoDB Atlas и самоуправляемыми развертываниями MongoDB.
  Прочитайте [пост в блоге](https://clickhouse.com/blog/mongodb-cdc-clickhouse-preview), чтобы получить обзор коннектора MongoDB CDC, и [зарегистрируйтесь для раннего доступа здесь](https://clickhouse.com/cloud/clickpipes/mongodb-cdc-connector)!

* **Уведомления**: Теперь вы будете получать уведомление в интерфейсе, когда ваш сервис начнет обновляться до новой версии ClickHouse. Дополнительные уведомления по электронной почте и в Slack можно настроить через центр уведомлений.

  * [Исправление ошибки] Селектор &quot;Период времени&quot; для метрик в интерфейсе по умолчанию устанавливался на &quot;24 часа&quot; независимо от выбранного периода времени. Теперь это исправлено, и интерфейс корректно обновляет графики в соответствии с выбранным периодом времени.

**Для ClickPipes теперь доступно вертикальное масштабирование**

[Для стриминговых ClickPipes теперь доступно вертикальное масштабирование](https://clickhouse.com/blog/clickpipes-flexible-scaling-monitoring).
Эта возможность позволяет управлять размером каждой реплики помимо
количества реплик (горизонтального масштабирования). На странице сведений для каждого ClickPipe теперь
также отображается использование CPU и памяти по каждой реплике, что помогает лучше
понимать свои рабочие нагрузки и уверенно планировать операции по изменению размера.

**ClickPipes для MySQL CDC теперь в публичной бете**

Коннектор MySQL CDC в ClickPipes теперь доступен всем в рамках публичной беты. Всего за несколько кликов

<div id="june-11-2025">
  ## 11 июля 2025 г.
</div>

* Новые сервисы теперь хранят метаданные database и table в централизованном **SharedCatalog** —
  новой модели координации и управления жизненным циклом объектов, которая обеспечивает:

  * **DDL в масштабе Cloud**, даже при высоком параллелизме
  * **Надёжное удаление и новые DDL-операции**
  * **Быстрый запуск и выход из спящего режима**, поскольку узлы без сохранения состояния теперь запускаются без зависимости от disk
  * **Вычисления без сохранения состояния как для native-, так и для открытых форматов**, включая Iceberg и Delta Lake

  Подробнее о SharedCatalog читайте в нашем [блоге](https://clickhouse.com/blog/clickhouse-cloud-stateless-compute)

* Теперь можно запускать сервисы, соответствующие требованиям HIPAA, в GCP `europe-west4`

<div id="may-30-2025">
  ## 30 мая 2025
</div>

* Мы рады сообщить, что **ClickPipes для Postgres CDC** в ClickHouse Cloud
  вышел на стадию General Availability. Теперь всего за несколько кликов вы можете реплицировать свои базы данных Postgres
  и получить молниеносную Real-time аналитику. Этот коннектор обеспечивает
  более быструю синхронизацию данных, задержку всего в несколько секунд, автоматическое применение изменений схемы,
  полностью защищенное подключение и многое другое. Подробнее см. в
  [блоге](https://clickhouse.com/blog/postgres-cdc-connector-clickpipes-ga). Чтобы
  начать работу, воспользуйтесь инструкциями [здесь](/ru/integrations/clickpipes/postgres/index).

* Представлены новые улучшения для панелей мониторинга в SQL-консоли:
  * Общий доступ: Вы можете предоставить доступ к своей панели мониторинга участникам команды. Поддерживаются четыре уровня доступа, которые можно настраивать как глобально, так и для отдельных пользователей:
    * *Доступ на запись*: добавление/редактирование визуализаций, изменение настроек обновления, взаимодействие с панелями мониторинга через фильтры.
    * *Владелец*: предоставление общего доступа к панели мониторинга, удаление панели мониторинга и все остальные разрешения пользователя с &quot;доступом на запись&quot;.
    * *Доступ только для чтения*: просмотр панели мониторинга и взаимодействие с ней через фильтры
    * *Нет доступа*: панель мониторинга недоступна для просмотра
  * Для уже созданных панелей мониторинга администраторы организации могут назначать себя их владельцами.
  * Теперь вы можете добавлять таблицу или диаграмму из SQL-консоли на панель мониторинга из представления запроса.

* Представлена панель использования ресурсов, которая показывает, какие
  ресурсы использует сервис в ClickHouse Cloud. На этой панели отображаются

  * Передача данных: графики, показывающие входящий и исходящий трафик данных в ClickHouse Cloud. Подробнее [здесь](/ru/products/cloud/reference/billing/network-data-transfer).

  созданного, чтобы упростить мониторинг сервисов ClickHouse Cloud.
  Этот mix-in использует нашу совместимую с Prometheus конечную точку API, чтобы легко интегрировать
  метрики ClickHouse в существующую конфигурацию Prometheus и Grafana. Он включает
  предварительно настроенную панель мониторинга, которая дает вам представление о состоянии
  и производительности ваших сервисов в реальном времени. Подробнее читайте в [блоге](https://clickhouse.com/blog/monitor-with-new-prometheus-grafana-mix-in).

<div id="april-18-2025">
  ## 18 апреля 2025 г.
</div>

* Представлены новая роль уровня организации **Member** и две новые роли уровня
  сервиса: **Service Admin** и **Service Read Only**.
  **Member** — это роль уровня организации, которая по умолчанию назначается пользователям SAML SSO и
  предоставляет только возможность входа в систему и обновления профиля. Роли **Service Admin**
  и **Service Read Only** для одного или нескольких сервисов могут назначаться пользователям
  с ролями **Member**, **Developer** или **Billing Admin**. Дополнительную информацию
  см. в разделе [&quot;Управление доступом в ClickHouse Cloud&quot;](/ru/products/cloud/guides/security/cloud-access-management/manage-database-users)
* ClickHouse Cloud теперь предлагает сервисы **HIPAA** и **PCI** в следующих регионах
  для клиентов **Enterprise**: AWS eu-central-1, AWS eu-west-2, AWS us-east-2.
* Представлены **пользовательские уведомления для ClickPipes**. Эта возможность отправляет
  автоматические оповещения о сбоях ClickPipes по электронной почте, в интерфейсе ClickHouse Cloud и
  в Slack. Уведомления по электронной почте и в интерфейсе включены по умолчанию и могут
  настраиваться для каждого пайпа. Для **Postgres CDC ClickPipes** оповещения также охватывают
  порог слота репликации (настраивается на вкладке **Settings**), определенные типы ошибок
  и шаги для самостоятельного устранения сбоев.
* **Закрытая предварительная версия MySQL CDC** теперь открыта. Это позволяет клиентам реплицировать базы данных MySQL
  в ClickHouse Cloud за несколько щелчков, обеспечивая быструю аналитику и
  устраняя необходимость во внешних ETL-инструментах. Коннектор поддерживает как непрерывную
  репликацию, так и одноразовые миграции независимо от того, где размещен MySQL: в облаке (RDS,
  Aurora, Cloud SQL, Azure и т. д.) или в собственной инфраструктуре. Вы можете зарегистрироваться в закрытой

<div id="single-replica-services">
  ### Сервисы с одной репликой
</div>

Мы вводим понятие «сервиса с одной репликой» — как отдельного варианта, так и в рамках хранилищ. Как отдельный вариант, сервисы с одной репликой ограничены по размеру и предназначены для небольших тестовых рабочих нагрузок. В рамках хранилищ сервисы с одной репликой можно развертывать в более крупных конфигурациях и использовать для рабочих нагрузок, которым не требуется высокая доступность в крупном масштабе, например для перезапускаемых ETL-задач.

<div id="vertical-auto-scaling-improvements">
  ### Улучшения вертикального автомасштабирования
</div>

Мы внедряем новый механизм вертикального масштабирования для вычислительных реплик, который называем &quot;Make Before Break&quot; (MBB). При таком подходе перед удалением старых реплик добавляется одна или несколько реплик нового размера, что предотвращает потерю мощности во время масштабирования. Устраняя разрыв между удалением существующих реплик и добавлением новых, MBB делает процесс масштабирования более плавным и менее прерывистым. Это особенно полезно в сценариях scale-up, когда высокая загрузка ресурсов приводит к необходимости увеличить мощность, поскольку преждевременное удаление реплик лишь усугубило бы нехватку ресурсов.

<div id="prometheus-integration-improvements">
  ### Улучшения интеграции с Prometheus
</div>

Мы внесли несколько улучшений в интеграцию с Prometheus:

* **Конечная точка на уровне организации**. Мы улучшили интеграцию с Prometheus для ClickHouse Cloud. Помимо метрик на уровне сервиса, API теперь включает конечную точку для **метрик на уровне организации**. Эта новая конечная точка автоматически собирает метрики для всех сервисов в вашей организации, упрощая экспорт метрик в ваш collector Prometheus. Эти метрики можно интегрировать с такими инструментами визуализации, как Grafana и Datadog, чтобы получить более полное представление о производительности вашей организации.

* **Отфильтрованные метрики**. Мы добавили в интеграцию с Prometheus для ClickHouse Cloud поддержку возврата отфильтрованного списка метрик. Эта возможность помогает сократить размер полезной нагрузки ответа, позволяя сосредоточиться на метриках, критически важных для мониторинга состояния вашего сервиса.

  Эта функция доступна через необязательный query parameter в API, что упрощает оптимизацию сбора данных и интеграцию с такими инструментами, как Grafana и Datadog.

  Возможность использования отфильтрованных метрик теперь доступна всем пользователям. Подробнее см. [здесь](/ru/products/cloud/features/monitoring/prometheus).
