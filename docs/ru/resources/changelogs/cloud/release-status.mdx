ClickHouse Cloud предлагает разные каналы релизов, чтобы учитывать различные потребности пользователей в плане стабильности, доступа к новым возможностям и предсказуемости обновлений. У каждого канала свой график обновлений, и каждый из них предназначен для разных сценариев использования — как для пользователей, которым нужен немедленный доступ к новым релизам, так и для тех, кто хочет отложить обновление, чтобы получить максимально стабильную версию релиза.

  ## Подробности о каналах релизов

  | :----------------------- | :--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | :-------------------------------------------------------------------------------------------------------------------------------------------------------- | :------------------------------------------ |
  | **Fast (Early Release)** | Рекомендуется для непродуктивных сред. Это первый канал релизов для каждого обновления версии базы данных                                                                                                                                      | Более ранний доступ к новым возможностям в обмен на стабильность.<br />Возможность протестировать релизы в непродуктивных средах до обновления production | Basic (по умолчанию)<br />Scale, Enterprise |
  | **Regular**              | Канал релизов по умолчанию для всех сервисов с несколькими репликами.<br />Развертывание в этом канале обычно начинается через две недели после запуска канала Fast.                                                                           | Обновления по умолчанию для всего парка сервисов.<br />Сервисы обновляются постепенно в течение нескольких недель                                         | Scale и Enterprise                          |
  | **Slow (Deferred)**      | Рекомендуется пользователям, которые предпочитают минимизировать риски и хотят, чтобы их сервисы обновлялись ближе к концу графика выпусков.<br />Развертывание в этом канале обычно начинается через две недели после запуска канала Regular. | Максимальная стабильность и предсказуемость.<br />Предназначен для тех, кому нужно больше времени на тестирование новых релизов в каналах Fast/Regular    | Enterprise                                  |

  <br />

  <br />

  <Note>
    Все сервисы с одной репликой автоматически подключаются к каналу релизов Fast.
  </Note>
</Accordion>

Запланированные окна обновления доступны во всех каналах релизов для сервисов уровня Enterprise. Эта возможность позволяет настроить временное окно в заданный день недели для обновлений.

<div id="release-schedule">
  ## График релизов
</div>

<Warning>
  **Как понимать даты релизов**

  Указанные ниже даты показывают, когда ClickHouse **начинает поэтапное развертывание** в каждом канале релизов, а не когда будет обновлён именно ваш сервис.

  * Развертывание выполняется автоматически и происходит постепенно в течение нескольких недель
  * Сервисы с настроенными запланированными окнами обновления обновляются в своё окно на неделе после завершения развертывания в канале (начиная с понедельника, 00:00 UTC)
  * Завершение развертывания может быть отложено из-за приостановки развертывания (например, на время праздников) или по результатам мониторинга состояния

  Чтобы заранее протестировать обновления перед их применением в production, используйте канал Fast или Regular для непроизводственных сервисов, а канал Slow — для production-сервисов.
</Warning>
