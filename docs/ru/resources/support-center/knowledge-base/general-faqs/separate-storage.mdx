Короткий ответ — «да».

Объектное хранилище (S3, GCS) можно использовать как эластичное основное хранилище для данных в таблицах ClickHouse. Опубликованы руководства по [S3-backed MergeTree](/ru/integrations/connectors/data-ingestion/AWS/integrating-s3-with-clickhouse) и [GCS-backed MergeTree](/ru/reference/functions/table-functions/gcs). В такой конфигурации на вычислительных узлах локально хранятся только метаданные. В этой схеме можно легко увеличивать и уменьшать вычислительные ресурсы, поскольку дополнительным узлам нужно лишь реплицировать метаданные.
