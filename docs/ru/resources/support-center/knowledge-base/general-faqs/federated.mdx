ClickHouse предлагает наиболее полную поддержку федеративных запросов и гибридного выполнения запросов среди аналитических баз данных.

Он поддерживает выполнение запросов к внешним системам и источникам данных:

* PostgreSQL
* MySQL
* MongoDB
* Redis
* любому источнику данных ODBC
* любому источнику данных JDBC
* любому источнику данных Arrow Flight
* стриминговым источникам данных, таким как Kafka и RabbitMQ
* озерам данных, таким как Iceberg, Delta Lake, Apache Hudi, Apache Paimon
* внешним файлам, размещенным в общем хранилище, таком как AWS S3, GCS, Minio, Cloudflare R2, Azure Blob Storage, Alicloud OSS, Tencent COS, а также в локальном хранилище, во множестве форматов данных

ClickHouse может выполнять JOIN между несколькими разными источниками данных в рамках одного запроса. Он также поддерживает гибридное выполнение запросов, объединяя локальные ресурсы и перенося часть выполнения запроса на удаленные машины.

Примечательно, что ClickHouse может ускорять запросы к внешним источникам данных без перемещения самих данных. Например, агрегационный запрос к MySQL будет выполняться быстрее в ClickHouse, поскольку накладные расходы на перемещение данных компенсируются более быстрым движком запросов.
