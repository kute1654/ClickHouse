В ClickHouse есть отдельные механизмы оптимизации, основанной на стоимости. Например, порядок чтения столбцов определяется стоимостью чтения сжатых диапазонов данных с диска.

ClickHouse также меняет порядок JOIN на основе статистики столбцов, однако это (по состоянию на 2025 год) пока далеко не так развито, как CBE в Postgres, Oracle и MS SQL Server.
