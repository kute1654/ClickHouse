Короткий ответ — «да». В ClickHouse есть несколько механизмов, которые позволяют освобождать место на диске за счёт удаления старых данных. Каждый из них предназначен для разных сценариев.

<div id="ttl">
  ## TTL
</div>

ClickHouse позволяет автоматически удалять значения при выполнении некоторого условия. Это условие задаётся как выражение на основе любых столбцов, обычно это просто фиксированное смещение для столбца с временной меткой.

Ключевое преимущество этого подхода в том, что он не требует никакой внешней системы для запуска: после настройки TTL удаление данных автоматически выполняется в фоновом режиме.

<Note>
  TTL также можно использовать для перемещения данных не только в [/dev/null](https://en.wikipedia.org/wiki/Null_device), но и между разными системами хранения, например с SSD на HDD.
</Note>

Подробнее см. в разделе [настройка TTL](/ru/reference/engines/table-engines/mergetree-family/mergetree#table_engine-mergetree-ttl).

<div id="delete-from">
  ## DELETE FROM
</div>

[DELETE FROM](/ru/reference/statements/delete) позволяет выполнять в ClickHouse стандартные запросы DELETE. Строки, подпадающие под условие фильтра, помечаются как удаленные и исключаются из последующих результирующих наборов. Очистка этих строк происходит асинхронно.

<Note>
  DELETE FROM доступен для общего использования начиная с версии 23.3. В более старых версиях он является экспериментальным и должен быть включен с помощью:

  ```sql
  SET allow_experimental_lightweight_delete = true;
  ```
</Note>

<div id="alter-delete">
  ## ALTER DELETE
</div>

ALTER DELETE удаляет строки с помощью асинхронных батч-операций. В отличие от DELETE FROM, запросы, выполняемые после ALTER DELETE, но до завершения батч-операций, будут включать строки, предназначенные для удаления. Подробнее см. в документации [ALTER DELETE](/ru/reference/statements/alter/delete).

`ALTER DELETE` можно использовать для гибкого удаления старых данных. Если вам нужно делать это регулярно, основным недостатком будет необходимость в наличии внешней системы, отправляющей запрос. Также следует учитывать влияние на производительность, поскольку мутации переписывают части целиком, даже если требуется удалить всего одну строку.

Это наиболее распространённый способ привести систему на базе ClickHouse в соответствие с [GDPR](https://gdpr-info.eu).

Подробнее о [мутациях](/ru/reference/statements/alter/index#mutations).

<div id="drop-partition">
  ## DROP PARTITION
</div>

`ALTER TABLE ... DROP PARTITION` — это экономичный способ удалить целую партицию. Этот оператор не слишком гибок и требует, чтобы схема партиционирования была правильно настроена при создании таблицы, но всё же покрывает большинство типичных сценариев. Как и мутации, для регулярного использования его нужно запускать из внешней системы.

Подробнее см. в разделе [управление партициями](/ru/reference/statements/alter/partition).

<div id="truncate">
  ## TRUNCATE
</div>

Полное удаление всех данных из таблицы — довольно радикальный шаг, но в некоторых случаях это именно то, что вам нужно.

Подробнее см. в разделе [очистка таблицы](/ru/reference/statements/truncate).
