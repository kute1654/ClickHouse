  ## Инструменты автоматической миграции схем для ClickHouse
</div>

Нас часто спрашивают, какой инструмент для миграции схем подходит для ClickHouse и как лучше управлять схемами баз данных в ClickHouse, которые могут со временем меняться?

Для ClickHouse нет стандартного инструмента для миграции схем, но мы составили следующий список (без какого-либо определенного порядка) известных нам инструментов автоматической миграции схем с поддержкой ClickHouse:

* [Goose](https://github.com/pressly/goose)
* [Sqitch](https://sqitch.org/docs/manual/sqitchtutorial-clickhouse/)
* [Atlas](https://atlasgo.io/guides/clickhouse?utm_source=clickhouse\&utm_term=knowledge)
* [Bytebase](https://www.bytebase.com/)
* [Flyway](https://documentation.red-gate.com/fd/supported-databases-and-versions-143754067.html)
* [Liquibase](https://www.liquibase.com/)
* Простой [инструмент от сообщества](https://github.com/VVVi/clickhouse-migrations) под названием `clickhouse-migrations`
* Еще один [инструмент от сообщества](https://github.com/golang-migrate/migrate/tree/master/database/clickhouse), написанный на Go
* [Houseplant](https://houseplant.readthedocs.io)
