<div id="question">
  ## Вопрос
</div>

Как быстро воссоздать таблицу и её данные, используя только копирование и вставку между разными терминалами?

<div id="answer">
  ## Ответ
</div>

Это НЕ рекомендуемый способ миграции данных из одной базы данных в другую, и его НЕ следует использовать для миграции production-данных.

Это лишь простой и быстрый способ заново создать небольшой объём данных при разработке в нескольких средах.

1. Получите оператор CREATE TABLE с помощью `SHOW CREATE table`:

```sql
SHOW CREATE TABLE cookies;

SHOW CREATE TABLE cookies

Query id: 248ec8e2-5bce-45b3-97d9-ed68edf445a5

┌─statement────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────┐
│ CREATE TABLE default.cookies
(
    `id` String,
    `timestamp` DateTime
)
ENGINE = MergeTree
ORDER BY id
SETTINGS index_granularity = 8192 │
└──────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────┘

```

2. Получите выгрузку данных с помощью `FORMAT SQLInsert`

```sql
SELECT * FROM cookies FORMAT SQLInsert;

SELECT *
FROM cookies
FORMAT SQLInsert

Query id: 383759b8-69c0-4561-ab95-f8224abc0071

INSERT INTO table (`id`, `timestamp`) VALUES ('4', '2023-03-15 16:28:46')
, ('2', '2023-03-15 16:28:41')
, ('1', '2023-03-15 16:11:02'), ('1', '2023-03-15 16:11:40'), ('1', '2023-03-15 16:11:48'), ('1', '2023-03-15 16:16:05'), ('2', '2023-03-15 16:11:06'), ('3', '2023-03-15 16:11:12'), ('3', '2023-03-15 16:11:45'), ('3', '2023-03-15 16:16:08'), ('4', '2023-03-15 16:11:14'), ('4', '2023-03-15 16:11:50'), ('4', '2023-03-15 16:16:01'), ('5', '2023-03-15 16:11:18'), ('5', '2023-03-15 16:16:11')
;

15 rows in set. Elapsed: 0.023 sec.
```

Обратите внимание: в пункте 2 нужно заменить имя `table` на фактическое имя таблицы (в этом примере — `cookies`)
