В некоторых случаях может потребоваться повторно синхронизировать отдельные таблицы в пайпе. Например, это может понадобиться при существенных изменениях схемы в Postgres или при изменении модели данных в ClickHouse.

Хотя возможность повторно синхронизировать отдельные таблицы одним нажатием кнопки всё ещё находится в разработке, в этом руководстве показано, как сделать это уже сегодня в Postgres ClickPipe.

<div id="removing-table">
  ### 1. Удалите таблицу из пайпа
</div>

Для этого воспользуйтесь [руководством по удалению таблицы](/ru/integrations/clickpipes/postgres/remove-table).

<div id="truncate-drop-table">
  ### 2. Выполните TRUNCATE или DROP таблицы в ClickHouse
</div>

Этот шаг нужен, чтобы избежать дублирования данных, когда на следующем шаге вы снова добавите эту таблицу. Для этого перейдите на вкладку **SQL Console** в ClickHouse Cloud и выполните запрос.
Обратите внимание: добавление таблицы блокируется, если такая таблица уже существует в ClickHouse и не пуста.

Либо, если вам нужно сохранить старую таблицу, её можно просто переименовать. Это особенно полезно, если таблица очень большая и операция DROP может занять некоторое время.

```sql
RENAME TABLE table_A TO table_A_bak;
```

<div id="add-table-again">
  ### 3. Снова добавьте таблицу в ClickPipe
</div>

Для этого следуйте инструкциям из [руководства по добавлению таблицы](/ru/integrations/clickpipes/postgres/add-table).
