В некоторых сценариях бывает полезно повторно синхронизировать отдельные таблицы в пайпе. Например, это может потребоваться при крупных изменениях схемы в MongoDB или при переработке модели данных в ClickHouse.

Хотя возможность повторной синхронизации отдельных таблиц по нажатию кнопки всё ещё находится в разработке, в этом руководстве описано, как уже сегодня выполнить это в MongoDB ClickPipe.

<div id="removing-table">
  ### 1. Удалите таблицу из пайпа
</div>

Для этого следуйте [руководству по удалению таблицы](/ru/integrations/clickpipes/mongodb/remove-table).

<div id="truncate-drop-table">
  ### 2. Выполните TRUNCATE или удалите таблицу в ClickHouse
</div>

Этот шаг нужен, чтобы избежать дублирования данных, когда на следующем шаге мы снова добавим эту таблицу. Для этого перейдите на вкладку **SQL Console** в ClickHouse Cloud и выполните запрос.
Обратите внимание: добавление таблицы блокируется, если такая таблица уже существует в ClickHouse и не пуста.

<div id="add-table-again">
  ### 3. Снова добавьте таблицу в ClickPipe
</div>

Для этого воспользуйтесь [руководством по добавлению таблицы](/ru/integrations/clickpipes/mongodb/add-table).
