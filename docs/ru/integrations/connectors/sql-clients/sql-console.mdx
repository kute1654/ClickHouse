Консоль SQL — самый быстрый и простой способ просматривать базы данных в ClickHouse Cloud и выполнять к ним запросы. С помощью консоли SQL вы можете:

* Подключаться к сервисам ClickHouse Cloud
* Просматривать, фильтровать и сортировать данные таблиц
* Выполнять запросы и визуализировать результаты всего за несколько кликов
* Делиться запросами с участниками команды и эффективнее работать вместе.

<div id="exploring-tables">
  ## Просмотр таблиц
</div>

<div id="viewing-table-list-and-schema-info">
  ### Просмотр списка таблиц и сведений о схеме
</div>

Обзор таблиц в вашем экземпляре ClickHouse можно найти на левой боковой панели. Чтобы просмотреть таблицы в конкретной базе данных, используйте селектор базы данных в верхней части левой панели.

<Image img="/images/cloud/sqlconsole/table-list-and-schema.webp" size="lg" border alt="Список таблиц и представление схемы с таблицами базы данных на левой боковой панели" />

Таблицы в списке также можно развернуть, чтобы просмотреть столбцы и типы данных.

<Image img="/images/cloud/sqlconsole/view-columns.webp" size="lg" border alt="Развернутая таблица с именами столбцов и типами данных" />

<div id="exploring-table-data">
  ### Просмотр данных таблицы
</div>

Нажмите на таблицу в списке, чтобы открыть её в новой вкладке. В режиме просмотра таблицы данные можно легко просматривать, выделять и копировать. Обратите внимание: при копировании в табличные приложения, такие как Microsoft Excel и Google Sheets, структура и форматирование сохраняются. Переключаться между страницами данных таблицы (по 30 строк на страницу) можно с помощью навигации в нижнем колонтитуле.

<Image img="/images/cloud/sqlconsole/abc.webp" size="lg" border alt="Режим просмотра таблицы с данными, которые можно выделять и копировать" />

<div id="inspecting-cell-data">
  ### Просмотр данных в ячейке
</div>

Инструмент Cell Inspector можно использовать для просмотра больших объёмов данных, содержащихся в одной ячейке. Чтобы открыть его, щёлкните правой кнопкой мыши по ячейке и выберите «Inspect Cell». Содержимое инспектора ячейки можно скопировать, нажав значок копирования в правом верхнем углу.

<Image img="/images/cloud/sqlconsole/inspecting-cell-content.webp" size="lg" border alt="Диалог Cell Inspector, показывающий содержимое выбранной ячейки" />

<div id="filtering-and-sorting-tables">
  ## Фильтрация и сортировка таблиц
</div>

<div id="sorting-a-table">
  ### Сортировка таблицы
</div>

Чтобы отсортировать таблицу в консоли SQL, откройте таблицу и нажмите кнопку &#39;Sort&#39; на панели инструментов. Откроется меню, в котором можно настроить сортировку. Вы можете выбрать столбец, по которому нужно сортировать, и задать порядок сортировки (по возрастанию или по убыванию). Нажмите &#39;Apply&#39; или клавишу Enter, чтобы отсортировать таблицу

<Image img="/images/cloud/sqlconsole/sort-descending-on-column.webp" size="lg" border alt="Диалоговое окно Sort с настройкой сортировки по убыванию для столбца" />

Консоль SQL также позволяет добавить для таблицы несколько сортировок. Снова нажмите кнопку &#39;Sort&#39;, чтобы добавить еще одну сортировку. Обратите внимание: сортировки применяются в том порядке, в котором они отображаются на панели сортировки (сверху вниз). Чтобы удалить сортировку, просто нажмите кнопку &#39;x&#39; рядом с ней.

<div id="filtering-a-table">
  ### Фильтрация таблицы
</div>

Чтобы отфильтровать таблицу в Консоли SQL, откройте таблицу и нажмите кнопку &#39;Filter&#39;. Как и при сортировке, эта кнопка открывает меню, в котором можно настроить фильтр. Вы можете выбрать столбец, по которому нужно фильтровать, и указать необходимые критерии. Консоль SQL автоматически показывает варианты фильтрации, соответствующие типу данных в столбце.

<Image img="/images/cloud/sqlconsole/filter-on-radio-column-equal-gsm.webp" size="lg" border alt="Диалог фильтрации с настройкой фильтра для столбца radio со значением GSM" />

Когда фильтр вас устроит, нажмите &#39;Apply&#39;, чтобы применить его к данным. Как показано ниже, вы также можете добавить дополнительные фильтры.

<Image img="/images/cloud/sqlconsole/add-more-filters.webp" size="lg" border alt="Диалог, показывающий, как добавить дополнительный фильтр для диапазона со значением больше 2000" />

Как и при сортировке, нажмите кнопку &#39;x&#39; рядом с фильтром, чтобы удалить его.

<div id="filtering-and-sorting-together">
  ### Одновременная фильтрация и сортировка
</div>

Консоль SQL позволяет одновременно фильтровать и сортировать таблицу. Для этого добавьте все нужные фильтры и параметры сортировки, как описано выше, и нажмите кнопку «Применить».

<Image img="/images/cloud/sqlconsole/filtering-and-sorting-together.webp" size="lg" border alt="Интерфейс с одновременно применёнными фильтрацией и сортировкой" />

<div id="creating-a-query-from-filters-and-sorts">
  ### Создание запроса из фильтров и сортировок
</div>

Консоль SQL может одним щелчком преобразовать выбранные сортировки и фильтры в запросы. Просто задайте нужные параметры сортировки и фильтрации и нажмите кнопку &#39;Create Query&#39; на панели инструментов. После нажатия &#39;Create Query&#39; откроется новая вкладка запроса, уже заполненная SQL-командой, соответствующей данным в текущем представлении таблицы.

<Image img="/images/cloud/sqlconsole/create-a-query-from-sorts-and-filters.webp" size="lg" border alt="Интерфейс с кнопкой «Create Query», которая генерирует SQL на основе фильтров и сортировок" />

<Note>
  Для использования возможности &#39;Create Query&#39; фильтры и сортировки не обязательны.
</Note>

Подробнее о выполнении запросов в Консоли SQL см. в (link) документации по запросам.

<div id="creating-and-running-a-query">
  ## Создание и выполнение запроса
</div>

<div id="creating-a-query">
  ### Создание запроса
</div>

Создать новый запрос в Консоли SQL можно двумя способами.

* Нажмите кнопку &#39;+&#39; на панели вкладок
* Выберите кнопку &#39;New Query&#39; в списке запросов на левой боковой панели

<Image img="/images/cloud/sqlconsole/creating-a-query.webp" size="lg" border alt="Интерфейс, показывающий, как создать новый запрос с помощью кнопки + или кнопки New Query" />

<div id="running-a-query">
  ### Запуск запроса
</div>

Чтобы выполнить запрос, введите SQL-команду или несколько команд в редакторе SQL и нажмите кнопку &#39;Run&#39; либо используйте сочетание клавиш `cmd / ctrl + enter`. Чтобы писать и выполнять несколько команд последовательно, обязательно ставьте точку с запятой после каждой команды.

Параметры выполнения запроса
По умолчанию при нажатии кнопки запускаются все команды, содержащиеся в редакторе SQL. Консоль SQL поддерживает еще два варианта выполнения запроса:

* Выполнить выбранные команды
* Выполнить команду в позиции курсора

Чтобы выполнить выбранные команды, выделите нужную команду или последовательность команд и нажмите кнопку &#39;Run&#39; (или используйте сочетание клавиш `cmd / ctrl + enter`). Если текст выделен, вы также можете выбрать &#39;Run selected&#39; в контекстном меню редактора SQL (оно открывается щелчком правой кнопкой мыши в любом месте редактора).

<Image img="/images/cloud/sqlconsole/run-selected-query.webp" size="lg" border alt="Интерфейс, показывающий, как выполнить выбранную часть SQL-запроса" />

Выполнить команду в текущей позиции курсора можно двумя способами:

* Выберите &#39;At Cursor&#39; в меню дополнительных параметров запуска (или используйте соответствующее сочетание клавиш `cmd / ctrl + shift + enter`

<Image img="/images/cloud/sqlconsole/run-at-cursor-2.webp" size="lg" border alt="Параметр Run at cursor в меню дополнительных параметров запуска" />

* Выберите &#39;Run at cursor&#39; в контекстном меню редактора SQL

<Image img="/images/cloud/sqlconsole/run-at-cursor.webp" size="lg" border alt="Параметр Run at cursor в контекстном меню редактора SQL" />

<Note>
  При выполнении команда в позиции курсора кратковременно подсвечивается желтым.
</Note>

<div id="canceling-a-query">
  ### Отмена запроса
</div>

Пока запрос выполняется, кнопка &#39;Run&#39; на панели инструментов редактора запросов заменяется кнопкой &#39;Cancel&#39;. Просто нажмите эту кнопку или клавишу `Esc`, чтобы отменить запрос. Примечание: все результаты, которые уже были получены, сохранятся после отмены.

<Image img="/images/cloud/sqlconsole/cancel-a-query.webp" size="lg" border alt="Кнопка Cancel, которая появляется во время выполнения запроса" />

<div id="saving-a-query">
  ### Сохранение запроса
</div>

Если имя запроса ещё не задано, он будет называться «Untitled Query». Нажмите на название запроса, чтобы изменить его. При переименовании запрос будет сохранён.

<Image img="/images/cloud/sqlconsole/give-a-query-a-name.webp" size="lg" border alt="Интерфейс, показывающий, как переименовать запрос «Untitled Query»" />

Вы также можете сохранить запрос с помощью кнопки сохранения или сочетания клавиш `cmd / ctrl + s`.

<Image img="/images/cloud/sqlconsole/save-the-query.webp" size="lg" border alt="Кнопка сохранения на панели инструментов редактора запросов" />

<div id="using-genai-to-manage-queries">
  ## Использование GenAI для работы с запросами
</div>

Эта возможность позволяет писать запросы в виде вопросов на естественном языке, а консоль запросов будет создавать SQL-запросы с учётом контекста доступных таблиц. GenAI также может помочь с отладкой запросов.

Подробнее о GenAI читайте в [блоге Announcing GenAI powered query suggestions in ClickHouse Cloud](https://clickhouse.com/blog/announcing-genai-powered-query-suggestions-clickhouse-cloud).

<div id="table-setup">
  ### Настройка таблицы
</div>

Давайте импортируем демонстрационный набор данных UK Price Paid и используем его для создания нескольких GenAI-запросов.

1. Откройте сервис ClickHouse Cloud.

2. Создайте новый запрос, нажав значок *+*.

3. Вставьте и выполните следующий код:

   ```sql
   CREATE TABLE uk_price_paid
   (
       price UInt32,
       date Date,
       postcode1 LowCardinality(String),
       postcode2 LowCardinality(String),
       type Enum8('terraced' = 1, 'semi-detached' = 2, 'detached' = 3, 'flat' = 4, 'other' = 0),
       is_new UInt8,
       duration Enum8('freehold' = 1, 'leasehold' = 2, 'unknown' = 0),
       addr1 String,
       addr2 String,
       street LowCardinality(String),
       locality LowCardinality(String),
       town LowCardinality(String),
       district LowCardinality(String),
       county LowCardinality(String)
   )
   ENGINE = MergeTree
   ORDER BY (postcode1, postcode2, addr1, addr2);
   ```

   Выполнение этого запроса займет около 1 секунды. После этого у вас будет пустая таблица с именем `uk_price_paid`.

4. Создайте новый запрос и вставьте в него следующий запрос:

   ```sql
   INSERT INTO uk_price_paid
   WITH
      splitByChar(' ', postcode) AS p
   SELECT
       toUInt32(price_string) AS price,
       parseDateTimeBestEffortUS(time) AS date,
       p[1] AS postcode1,
       p[2] AS postcode2,
       transform(a, ['T', 'S', 'D', 'F', 'O'], ['terraced', 'semi-detached', 'detached', 'flat', 'other']) AS type,
       b = 'Y' AS is_new,
       transform(c, ['F', 'L', 'U'], ['freehold', 'leasehold', 'unknown']) AS duration,
       addr1,
       addr2,
       street,
       locality,
       town,
       district,
       county
   FROM url(
       'http://prod.publicdata.landregistry.gov.uk.s3-website-eu-west-1.amazonaws.com/pp-complete.csv',
       'CSV',
       'uuid_string String,
       price_string String,
       time String,
       postcode String,
       a String,
       b String,
       c String,
       addr1 String,
       addr2 String,
       street String,
       locality String,
       town String,
       district String,
       county String,
       d String,
       e String'
   ) SETTINGS max_http_get_redirects=10;
   ```

Этот запрос получает набор данных с сайта `gov.uk`. Размер файла — около 4 ГБ, поэтому выполнение запроса займет несколько минут. После того как ClickHouse обработает запрос, весь набор данных будет загружен в таблицу `uk_price_paid`.

<div id="query-creation">
  #### Создание запроса
</div>

Давайте создадим запрос с помощью естественного языка.

1. Выберите table **uk&#95;price&#95;paid**, затем нажмите **Create Query**.

2. Нажмите **Generate SQL**. Вас могут попросить подтвердить, что ваши запросы отправляются в Chat-GPT. Чтобы продолжить, необходимо выбрать **I agree**.

3. Теперь вы можете использовать этот промпт, чтобы ввести запрос на естественном языке, а ChatGPT преобразует его в SQL-запрос. В этом примере мы введём:

   > Покажи общую стоимость и общее количество всех транзакций в uk&#95;price&#95;paid по годам.

4. Консоль сгенерирует нужный запрос и отобразит его на новой вкладке. В нашем примере GenAI создал следующий запрос:

   ```sql
   -- Покажи общую стоимость и общее количество всех транзакций в uk_price_paid по годам.
   SELECT year(date), sum(price) as total_price, Count(*) as total_transactions
   FROM uk_price_paid
   GROUP BY year(date)
   ```

5. Убедившись, что запрос корректен, нажмите **Run**, чтобы выполнить его.

<div id="debugging">
  ### Отладка
</div>

Теперь давайте проверим возможности GenAI по отладке запросов.

1. Создайте новый запрос, нажав на значок *+*, и вставьте следующий код:

   ```sql
   -- Покажи мне общую цену и общее количество всех транзакций uk_price_paid по годам.
   SELECT year(date), sum(pricee) as total_price, Count(*) as total_transactions
   FROM uk_price_paid
   GROUP BY year(date)
   ```

2. Нажмите **Run**. Запрос завершится ошибкой, поскольку мы пытаемся получить значения из `pricee`, а не из `price`.

3. Нажмите **Fix Query**.

4. GenAI попытается исправить запрос. В этом случае он заменил `pricee` на `price`. Он также определил, что в этом случае лучше использовать функцию `toYear`.

5. Выберите **Apply**, чтобы добавить предложенные изменения в запрос, и нажмите **Run**.

Имейте в виду, что GenAI — экспериментальная возможность. Соблюдайте осторожность при выполнении запросов, сгенерированных GenAI, для любого набора данных.

В этом примере мы найдём все вхождения `breakfast` в таблице `hackernews` среди комментариев, содержащих `ClickHouse` (регистронезависимо):

<div id="visualizing-query-data">
  ## Визуализация данных запроса
</div>

Некоторые данные легче воспринимать в виде диаграмм. Вы можете быстро создавать визуализации по данным результатов запроса прямо в Консоли SQL всего за несколько щелчков мышью. Для примера используем запрос, который рассчитывает еженедельную статистику по поездкам нью-йоркского такси:

```sql
SELECT
   toStartOfWeek(pickup_datetime) AS week,
   sum(total_amount) AS fare_total,
   sum(trip_distance) AS distance_total,
   count(*) AS trip_total
FROM
   nyc_taxi
GROUP BY
   1
ORDER BY
   1 ASC
```

<Image img="/images/cloud/sqlconsole/tabular-query-results.webp" size="lg" border alt="Табличные результаты запроса" />

Без визуализации эти результаты трудно понять. Давайте представим их в виде диаграммы.

<div id="creating-charts">
  ### Создание диаграмм
</div>

Чтобы приступить к созданию визуализации, выберите на панели инструментов панели результатов запроса опцию &#39;Chart&#39;. Откроется панель конфигурации диаграммы:

<Image img="/images/cloud/sqlconsole/switch-from-query-to-chart.webp" size="lg" border alt="Переключение от запроса к диаграмме" />

  ### Настройка диаграмм

Консоль SQL поддерживает десять типов диаграмм, которые можно выбрать в селекторе типа диаграммы на панели конфигурации диаграммы. Например, мы можем легко изменить предыдущий тип диаграммы с Bar на Area:

Заголовки диаграмм соответствуют имени запроса, который предоставляет данные. Если изменить имя запроса, заголовок диаграммы также обновится:

<Image img="/images/cloud/sqlconsole/update-query-name.webp" size="lg" border alt="Обновление имени запроса" />

Ряд более продвинутых параметров диаграммы также можно настроить в разделе &#39;Advanced&#39; на панели конфигурации диаграммы. Для начала изменим следующие настройки:

* Подзаголовок
* Заголовки осей
* Ориентация меток на оси x

Наша диаграмма будет обновлена соответствующим образом:

Например, приведенная выше диаграмма выглядит хорошо, но, чтобы продемонстрировать корреляцию между нашими полями `trip_total` и `fare_total`, диапазоны осей нужно немного скорректировать:

<div id="sharing-queries">
  ## Общий доступ к запросам
</div>

Консоль SQL позволяет делиться запросами с вашей командой. Когда запрос открыт для общего доступа, все участники команды могут просматривать и редактировать его. Запросы с общим доступом — удобный способ совместной работы с командой.

Чтобы поделиться запросом, нажмите кнопку &#39;Share&#39; на панели инструментов запроса.

<Image img="/images/cloud/sqlconsole/sql-console-share.webp" size="lg" border alt="Кнопка Share на панели инструментов запроса" />

Откроется диалоговое окно, в котором можно открыть доступ к запросу для всех участников команды. Если у вас несколько команд, вы можете выбрать, с какой из них поделиться запросом.

<Image img="/images/cloud/sqlconsole/sql-console-edit-access.webp" size="lg" border alt="Диалоговое окно для изменения доступа к запросу с общим доступом" />

<Image img="/images/cloud/sqlconsole/sql-console-add-team.webp" size="lg" border alt="Интерфейс для добавления команды к запросу с общим доступом" />

<Image img="/images/cloud/sqlconsole/sql-console-edit-member.webp" size="lg" border alt="Интерфейс для изменения доступа участника к запросу с общим доступом" />

В некоторых случаях может потребоваться отдельно настроить масштаб осей для каждого поля. Это также можно сделать в разделе &#39;Advanced&#39; на панели конфигурации диаграммы, указав минимальное и максимальное значения для диапазона оси. Например, приведённая выше диаграмма выглядит хорошо, но, чтобы продемонстрировать корреляцию между полями `trip_total` и `fare_total`, диапазоны осей нужно немного скорректировать:
