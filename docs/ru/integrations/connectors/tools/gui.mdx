<div id="agx">
  ### agx
</div>

[agx](https://github.com/agnosticeng/agx) — это настольное приложение, созданное на базе Tauri и SvelteKit, с современным интерфейсом для изучения данных и выполнения запросов с использованием встроенного движка баз данных ClickHouse (chdb).

* Использует ch-db при запуске нативного приложения.
* Может подключаться к экземпляру ClickHouse при запуске веб-версии.
* Редактор Monaco — так что освоиться будет легко.
* Множество визуализаций данных, число и возможности которых продолжают расти.

<div id="ch-ui">
  ### ch-ui
</div>

[ch-ui](https://github.com/caioricciuti/ch-ui) — это простой интерфейс на React.js для баз данных ClickHouse, предназначенный для выполнения запросов и визуализации данных. Построенный на React и веб-клиенте ClickHouse, он предоставляет современный и удобный интерфейс для простой работы с базой данных.

Возможности:

* Интеграция с ClickHouse: легко управляйте подключениями и выполняйте запросы.
* Удобное управление вкладками: динамически работайте с несколькими вкладками, например вкладками запросов и таблиц.
* Оптимизация производительности: использует IndexedDB для эффективного кэширования и управления состоянием.
* Локальное хранение данных: все данные хранятся локально в браузере, поэтому они никуда больше не отправляются.

<div id="chartdb">
  ### ChartDB
</div>

[ChartDB](https://chartdb.io) — бесплатный инструмент с открытым исходным кодом для визуализации и проектирования схем баз данных, включая ClickHouse, с помощью одного запроса. Он создан на React и предлагает удобный и понятный интерфейс, не требуя ни учётных данных базы данных, ни регистрации для начала работы.

Возможности:

* Визуализация схемы: мгновенно импортируйте и визуализируйте схему ClickHouse, включая ER-диаграммы с materialized view и обычными представлениями, с отображением связей с таблицами.
* Экспорт DDL с помощью ИИ: легко генерируйте DDL-скрипты для более удобного управления схемой и её документирования.
* Поддержка нескольких диалектов SQL: совместим с рядом SQL-диалектов, что делает его универсальным для различных сред баз данных.
* Без регистрации и учётных данных: все возможности доступны прямо в браузере, что обеспечивает простоту и безопасность работы.

[Исходный код ChartDB](https://github.com/chartdb/chartdb).

<div id="datastoria">
  ### DataStoria
</div>

[DataStoria](https://github.com/FrankChen021/datastoria) — это консольное веб-приложение на базе ИИ для управления несколькими кластерами ClickHouse из одного места.

Возможности:

* **Интеллект на базе ИИ**: Используйте естественный язык, чтобы изучать данные, оптимизировать и исправлять SQL-запросы, а также визуализировать данные.
* **Интеграция с официальными навыками ClickHouse Agent**: Используйте [официальные рекомендации](https://github.com/ClickHouse/agent-skills), чтобы получать от ИИ предложения по оптимизации базы данных и полезные рекомендации.
* **Умная диагностика ошибок**: Мгновенно находите синтаксические ошибки благодаря точной подсветке строк и столбцов и получайте предложения по исправлению на базе ИИ в один клик.
* **Анализ системных таблиц**: Подробно изучайте `system.query_log`, `system.query_views_log`, `system.zookeeper`, `system.ddl_distributed_queue`, `system.part_log` и `system.processes` с помощью мощной панели мониторинга с визуализацией и фильтрами, чтобы быстро понять состояние кластера.
* **Explain в один клик**: Мгновенно разбирайтесь в планах выполнения запросов с помощью визуальных представлений AST и конвейера.
* **Граф зависимостей**: Визуализируйте связи между таблицами и отслеживайте потоки данных через materialized views, distributed таблицы и внешние системы.
* **Мониторинг кластера**: Отслеживайте все узлы с помощью метрик в реальном времени, операций слияния, статуса репликации, производительности запросов и многого другого.
* **Privacy &amp; Security**: Все SQL-запросы выполняются напрямую из вашего браузера на сервер ClickHouse, обеспечивая полную конфиденциальность.

[Документация DataStoria](https://docs.datastoria.app).

<div id="clickhouse-schemaflow-visualizer">
  ### ClickHouse Schema Flow Visualizer
</div>

[ClickHouse Schema Flow Visualizer](https://github.com/FulgerX2007/clickhouse-schemaflow-visualizer) — веб-приложение с открытым исходным кодом для визуализации связей между таблицами ClickHouse.
Оно подключается к экземпляру ClickHouse, разбирает метаданные `system.tables` (типы движков, зависимости, SELECT-запросы materialized view) и отображает интерактивные диаграммы потоков данных на уровне таблиц вместе со связями на уровне столбцов, где на каждом ребре указано выражение преобразования. Для компоновки диаграмм используется Dagre, а рендеринг выполняется в виде обычного встроенного SVG — клиентский runtime для построения диаграмм не загружается.

Возможности:

* Просмотр баз данных и таблиц ClickHouse через интуитивно понятную боковую панель
* Представление Data Flow: входящие источники и исходящие materialized view на уровне таблиц
* Представление Relationships: сопоставление на уровне столбцов с разобранным выражением преобразования на каждом ребре (например, `toStartOfHour(scheduled_departure)`, `avgState(delay_minutes)`)
* Значки и цветовое кодирование с учетом движков для `MergeTree`, `Replicated*`, `Distributed`, `MaterializedView` и `Dictionary`
* Щелкните столбец в представлении Relationships, чтобы подсветить весь путь данных через конвейер
* Фильтр в боковой панели в реальном времени и палитра команд `Ctrl+K` / `⌘K` для быстрого перехода к любой таблице, столбцу или движку
* Необязательное наложение метаданных с отображением количества строк и размера на диске для каждой таблицы
* Экспорт текущей диаграммы в виде самодостаточного HTML-файла
* TLS‑соединение с ClickHouse с возможностью отключить проверку сертификата и использовать пользовательские сертификаты CA и клиента

[ClickHouse Schema Flow Visualizer — исходный код](https://github.com/FulgerX2007/clickhouse-schemaflow-visualizer)

<div id="dbeaver">
  ### DBeaver
</div>

[DBeaver](https://dbeaver.io/) — универсальный настольный клиент для работы с базами данных с поддержкой ClickHouse.

Возможности:

* Разработка запросов с подсветкой синтаксиса и автодополнением.
* Список таблиц с фильтрами и поиском по метаданным.
* Предпросмотр данных таблицы.
* Полнотекстовый поиск.

По умолчанию DBeaver не использует сеанс при подключении (в отличие, например, от CLI). Если вам нужна поддержка сеансов (например, чтобы задавать настройки для своего сеанса), измените свойства подключения драйвера и установите для `session_id` случайное значение (внутри используется HTTP-соединение). После этого вы сможете использовать любые настройки из окна запроса.

</div>

<div id="chouse-ui">
  ### CHouse UI
</div>

[CHouse UI](https://chouse-ui.com) — это самоуправляемый веб-интерфейс ClickHouse с открытым исходным кодом, созданный для **команд, эксплуатирующих ClickHouse в продакшне**. Большинство инструментов хорошо решают какую-то одну задачу — рабочее пространство для запросов, панель мониторинга, AI assistant, мониторинг кластера; CHouse UI объединяет всё это: уровень управления командным доступом, мониторинг парка из нескольких кластеров и автономный AI SRE в режиме только для чтения. В отличие от клиентов, которым нужны прямые учетные данные базы данных, он хранит их в зашифрованном виде на стороне сервера и управляет доступом через собственный слой **Ролевого управления доступом (RBAC)**, поэтому браузер никогда не получает пароль ClickHouse.

Возможности:

* **Командный доступ и безопасность** - RBAC на уровне приложения (предопределенные и пользовательские роли, детализированные правила доступа к данным на уровне базы данных и таблицы), журнал аудита с реальным контекстом сеанса и зашифрованные на стороне сервера учетные данные с AES-256-GCM.
* **Парк из нескольких кластеров** - Отслеживайте все настроенные кластеры в одной панели (состояние, память, активные запросы, исключения, мини-графики трендов); каждая карточка опрашивается независимо, а в основе лежит серверный опросчик снимков.
* **Chouse AI — Fleet Doctor** - Автономный AI SRE в режиме только для чтения: он сканирует парк с помощью защищенного инструмента `SELECT`, ограниченного `system.*` (ClickHouse `readonly=1`), точно определяет первопричины и формирует структурированный отчет с подробным разбором тяжелых запросов и предложениями по их переписыванию. Он никогда не изменяет кластер.
* **AI во вкладках мониторинга** - &quot;Optimize with Chouse AI&quot; для строки в журнале запросов (переписывание + оценка `EXPLAIN` до→после + открытие в рабочем пространстве SQL), а также &quot;Diagnose&quot; в один клик для строки `system.errors` или записи в журнале частей.
* **Пороговые оповещения** - Правила для % памяти узла, памяти на запрос и долгих запросов с доставкой в Slack и по электронной почте — с приложенным автономным анализом первопричины при срабатывании.
* **Полноценное рабочее пространство** - редактор SQL Monaco, обозреватель схем, представление выполняющихся запросов с возможностью завершения, встроенный мониторинг ClickHouse (разбивка памяти, части/слияния, отставание реплик, перцентили задержки) и импорт/экспорт данных.

С открытым исходным кодом (Apache 2.0), ориентирован прежде всего на on-prem — все возможности входят в поставку, без платного уровня.
