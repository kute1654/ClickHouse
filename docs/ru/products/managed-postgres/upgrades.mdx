Managed Postgres обновляет версии PostgreSQL, чтобы ваши экземпляры оставались защищёнными и актуальными. Поддерживаются как минорные, так и мажорные обновления с минимальными перебоями.

<div id="maintenance-updates">
  ## Обновления обслуживания
</div>

Регулярное обслуживание экземпляров PostgreSQL включает следующее:

* Обновления минорных версий (например, с 17.4 до 17.5) включают исправления ошибок и патчи безопасности движка PostgreSQL.
* Возможности Managed Service: улучшения нативной поддержки CDC, обсервабилити, pg&#95;clickhouse и других расширений.
* Патчи операционной системы и системных компонентов, включая исправления безопасности, повышение эффективности и другие улучшения.

Эти работы выполняются через failover и обычно приводят лишь к кратковременному разрыву соединения, часто всего на несколько секунд.

Для экземпляров с включенными [резервными узлами](/ru/products/managed-postgres/high-availability) обновление сначала применяется к резервному узлу, а затем выполняется failover, чтобы минимизировать простой.

<div id="maintenance-windows">
  ## Окна обслуживания
</div>

Окно обслуживания по умолчанию — с 14:00 до 16:00 UTC по воскресеньям.
Ожидаемый простой в течение этого окна составляет менее 1 минуты.

Для организаций уровня Enterprise в Managed Postgres поддерживаются окна обслуживания, чтобы обновления и другие операции можно было запланировать на время, которое меньше всего влияет на вашу рабочую нагрузку. Поддержка настройки окон обслуживания в интерфейсе и API появится в ближайшее время. Пока же обратитесь в [поддержку](https://clickhouse.com/support/program), чтобы настроить окно обслуживания для вашего экземпляра.

<div id="major-version-upgrades">
  ## Обновление основных версий
</div>

Возможность обновления основных версий (например, с 17.x до 18.x) через интерфейс и API скоро появится.
Пока же, чтобы обновить свой экземпляр Managed Postgres, обратитесь в [службу поддержки](https://clickhouse.com/support/program).
