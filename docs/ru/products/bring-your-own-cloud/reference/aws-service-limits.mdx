Для успешного развертывания BYOC в вашей учетной записи AWS должны быть доступны достаточные квоты сервисов AWS (ранее называвшиеся *лимитами сервисов*). AWS применяет к большинству сервисов квоты по умолчанию для каждого региона. Многие из этих значений по умолчанию ниже того, что требуется для развертывания BYOC в продакшн, особенно в недавно созданных или малоиспользуемых учетных записях AWS.

На этой странице приведен контрольный список квот, которые нужно проверить перед развертыванием. Отслеживайте использование и обращайтесь напрямую к облачному провайдеру, чтобы запросить увеличение квот.

<div id="pre-deployment-quota-checklist">
  ## Контрольный список квот перед развертыванием
</div>

Перед началом онбординга BYOC проверьте следующие квоты в регионе AWS, где планируется развертывание. Квоты устанавливаются для каждого региона и каждой учетной записи.

<div id="required-quotas">
  ### Обязательные квоты
</div>

| Сервис                     | Название квоты                                                   | Требование BYOC                                                                                                                                        | По умолчанию                             | Действие                                                                                                           |
| -------------------------- | ---------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------ | ---------------------------------------- | ------------------------------------------------------------------------------------------------------------------ |
| **EC2**                    | Running On-Demand Standard (A, C, D, H, I, M, R, T, Z) instances | ≥ пиковое число vCPU для уровня сервиса × 1.5 (запас для автомасштабирования и обновлений MBB) + 100 ядер vCPU для системной рабочей нагрузки и Keeper | Часто 32–256 vCPU на новых аккаунтах     | **Запросите увеличение**, чтобы выполнить требование BYOC                                                          |
| **EC2 (VPC)**              | VPCs per Region                                                  | ≥ 1 (BYOC создает 1 выделенный VPC)                                                                                                                    | 5                                        | Проверьте, что квота доступна                                                                                      |
| **EC2 (VPC)**              | Elastic IPs per Region                                           | ≥ 3 (по одному на каждую AZ для NAT Gateway)                                                                                                           | 5                                        | Проверьте, что квота доступна. Запросите увеличение, если запускаете несколько развертываний BYOC в одном регионе. |
| **EC2 (VPC)**              | NAT Gateways per AZ                                              | ≥ 1                                                                                                                                                    | 5                                        | Проверьте, что квота доступна                                                                                      |
| **EC2 (VPC)**              | Internet Gateways per Region                                     | ≥ 1                                                                                                                                                    | 5                                        | Проверьте, что квота доступна                                                                                      |
| **EC2 (VPC)**              | Subnets per VPC                                                  | ≥ 6 (3 публичные + 3 приватные)                                                                                                                        | 200                                      | Действий не требуется                                                                                              |
| **EC2 (VPC)**              | Security groups per VPC                                          | ≥ 10                                                                                                                                                   | 2,500                                    | Действий не требуется                                                                                              |
| **EKS**                    | Clusters per Region                                              | ≥ 1                                                                                                                                                    | 100                                      | Действий не требуется                                                                                              |
| **EKS**                    | Managed node groups per cluster                                  | ≥ 4                                                                                                                                                    | 30                                       | Действий не требуется                                                                                              |
| **EKS**                    | Nodes per managed node group                                     | ≥ пиковое число узлов для уровня сервиса                                                                                                               | 450                                      | Действий не требуется                                                                                              |
| **S3**                     | Buckets per account                                              | ≥ 4 (данные, резервные копии, биллинг, мониторинг)                                                                                                     | 100 (поддерживается увеличение до 1,000) | Проверьте запас для других рабочих нагрузок                                                                        |
| **EBS**                    | Storage for General Purpose SSD (gp3)                            | ≥ пиковый объем Log ClickHouse + том ОС × число узлов                                                                                                  | 50 TiB                                   | Проверьте, что квота доступна                                                                                      |
| **Elastic Load Balancing** | Network Load Balancers per Region                                | ≥ 1 на сервис ClickHouse                                                                                                                               | 50                                       | Проверьте, что квота доступна                                                                                      |
| **CloudWatch Logs**        | Log groups per Region                                            | ≥ 5                                                                                                                                                    | 1,000,000                                | Действий не требуется                                                                                              |

<div id="optional-feature-quotas">
  ### Квоты для проверки включения дополнительных возможностей
</div>

| Включённая возможность | Сервис    | Квота                                                                                                                                                     |
| ---------------------- | --------- | --------------------------------------------------------------------------------------------------------------------------------------------------------- |
| AWS PrivateLink        | EC2 (VPC) | Сервисы конечных точек VPC в регионе (по умолчанию 20) — запрашивайте увеличение для каждого одновременно используемого сервиса с включённым PrivateLink. |
| VPC Peering            | EC2 (VPC) | Активные пиринговые соединения VPC на VPC (по умолчанию 50).                                                                                              |
