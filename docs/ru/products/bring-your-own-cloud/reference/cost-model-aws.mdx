Развертывание ClickHouse по модели BYOC предполагает два отдельных счета:

1. **Начисления ClickHouse Cloud** — ClickHouse выставляет их за ваши сервисы ClickHouse на основе общего объема выделенной памяти.
2. **Расходы на инфраструктуру AWS** — AWS выставляет их напрямую вашему аккаунту AWS за каждый ресурс, который развертывание BYOC создает в вашей инфраструктуре.

На этой странице описано, как рассчитывается каждая из этих сумм и как вместе они складываются в совокупную стоимость владения (TCO).

<div id="clickhouse-cloud-charges">
  ## Начисления ClickHouse Cloud
</div>

Размер начислений ClickHouse Cloud зависит от общего объема выделенной памяти. [Свяжитесь с командой](https://clickhouse.com/cloud/bring-your-own-cloud), чтобы узнать, как это применимо к вашей конфигурации.

<div id="aws-infrastructure-charges">
  ## Расходы на инфраструктуру AWS
</div>

AWS выставляет счета вашему аккаунту напрямую за каждый ресурс, развернутый в рамках BYOC. ClickHouse не делает наценку и не перепродаёт мощности AWS. Полный список обязательных и дополнительных сервисов см. в разделе [Платные сервисы AWS](/ru/products/bring-your-own-cloud/reference/billable-aws-services).

Основные факторы затрат, обычно в порядке убывания их вклада в счёт BYOC:

1. **Amazon EC2** — рабочие инстансы, на которых работают управляемые группы узлов EKS. По умолчанию используются стандартные семейства Graviton (например, `m7g`). Семейство и количество инстансов масштабируются в зависимости от объёма памяти, выделенного вашему сервису, и автомасштабирования группы узлов.
2. **Amazon S3** — хранение данных таблиц ClickHouse и резервных копий в ваших бакетах. Плата взимается за каждый ГБ в месяц, а также за запросы и межрегиональную передачу данных.
3. **Amazon EBS** — тома gp3, подключённые к рабочим узлам для ОС, образов контейнеров и журналов ClickHouse.
4. **NAT Gateway и передача данных между зонами доступности** — исходящий трафик из приватных подсетей, а также трафик между зонами доступности (в развертываниях с несколькими AZ данные реплицируются между AZ).
5. **Amazon EKS** — фиксированная почасовая плата за control plane кластера.
6. **Elastic Load Balancing (NLB)** — плата за LCU-час для входящего клиентского трафика.
7. **CloudWatch Logs, Route 53, KMS, конечные точки VPC** — как правило, составляют небольшую долю общего счёта, но зависят от рабочей нагрузки.

Актуальные цены AWS см. на страницах с тарифами по отдельным сервисам на [aws.amazon.com](https://aws.amazon.com/pricing/).
