<div id="overview">
  ## Обзор
</div>

BYOC (Собственное облако) позволяет развернуть ClickHouse Cloud в вашей собственной облачной инфраструктуре. Это удобно, если у вас есть особые требования или ограничения, из-за которых вы не можете использовать управляемый сервис ClickHouse Cloud.

> **Если вы хотите получить доступ, [свяжитесь с нами](https://clickhouse.com/cloud/bring-your-own-cloud).** Дополнительную информацию см. в наших [Условиях предоставления услуг](https://clickhouse.com/legal/agreements/terms-of-service).

В настоящее время BYOC поддерживается только для AWS. Записаться в список ожидания для GCP и Azure можно [здесь](https://clickhouse.com/cloud/bring-your-own-cloud).

<Note>
  BYOC разработан специально для крупных развертываний и требует от клиентов заключения долгосрочного контракта.
</Note>

<div id="glossary">
  ## Глоссарий
</div>

* **ClickHouse VPC:**  VPC, принадлежащая ClickHouse Cloud.
* **Customer BYOC VPC:** VPC, принадлежащая облачному аккаунту клиента, разворачиваемая и управляемая ClickHouse Cloud и выделенная для развертывания ClickHouse Cloud BYOC.
* **Customer VPC** Другие VPC, принадлежащие облачному аккаунту клиента и используемые приложениями, которым необходимо подключаться к Customer BYOC VPC.

<div id="features">
  ## Возможности
</div>

<div id="supported-features">
  ### Поддерживаемые возможности
</div>

* **SharedMergeTree**: ClickHouse Cloud и BYOC используют один и тот же бинарный файл и конфигурацию. Поэтому в BYOC поддерживаются все возможности ClickHouse, включая SharedMergeTree.
* **Доступ к консоли для управления состоянием сервиса**:
  * Поддерживаются такие операции, как запуск, остановка и удаление.
  * Просмотр сервисов и их состояния.
* **Резервное копирование и восстановление.**
* **Вертикальное и горизонтальное масштабирование вручную.**
* **Режим простоя.**
* **Хранилища**: разделение вычислительных ресурсов
* **Сеть Zero Trust через Tailscale.**
* **Мониторинг**:
  * В консоли Cloud есть встроенные панели мониторинга для контроля состояния сервиса.
  * Скрапинг Prometheus для централизованного мониторинга с помощью Prometheus, Grafana и Datadog. Инструкции по настройке см. в [документации Prometheus](/ru/products/cloud/features/monitoring/prometheus).
* **VPC Peering.**
* **Интеграции**: полный список см. на [этой странице](/ru/integrations/home).
* **Защищённый доступ к S3.**
* **[AWS PrivateLink](https://aws.amazon.com/privatelink/).**

<div id="planned-features-currently-unsupported">
  ### Запланированные возможности (пока не поддерживаются)
</div>

* [AWS KMS](https://aws.amazon.com/kms/) (также известен как CMEK — ключи шифрования, управляемые клиентом)
* ClickPipes для приёма данных
* Автомасштабирование
* Интерфейс MySQL
