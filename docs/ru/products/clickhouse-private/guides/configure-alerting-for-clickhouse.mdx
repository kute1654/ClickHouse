Настройте цели сбора метрик Prometheus и оповещения для сервер ClickHouse, Keeper и оператора.

<div id="set-up-prometheus-scrape-targets-for-clickhouse-server">
  ## Настройте цели сбора метрик Prometheus для сервера ClickHouse
</div>

Поды сервера ClickHouse содержат аннотации `prometheus.io/*`, которые предоставляют базовые метрики. Чтобы собирать полный набор метрик (включая `ClickHouse_CustomMetrics_*`), настройте конечную точку `:8123/metrics` в качестве цели сбора метрик Prometheus для каждого пода сервера с помощью PodMonitor или аналогичного механизма.

Для доступа к этой конечной точке **требуется аутентификация**. Используйте отдельного пользователя с привилегиями только для чтения.

<div id="set-up-podmonitor-for-keeper">
  ## Настройка PodMonitor для Keeper
</div>

Создайте PodMonitor, нацеленный на конечную точку `:8001/metrics` в подах Keeper, чтобы собирать метрики Keeper.

<div id="alert-rules">
  ## Правила оповещений
</div>

Полный список рекомендуемых определений оповещений (выражений Prometheus, пороговых значений и действий) см. в [reference/metrics-and-alerts.md](/ru/products/clickhouse-private/reference/metrics-and-alerts-reference).

Ключевые оповещения включают:

| Оповещение                                 | Метрика                                                | Назначение                                                           |
| ------------------------------------------ | ------------------------------------------------------ | -------------------------------------------------------------------- |
| Оператор не выполняет реконсиляцию         | `last_cluster_reconcile`                               | Оператор не выполнял реконсиляцию в ожидаемый интервал               |
| Ошибки реконсиляции оператора              | `controller_runtime_reconcile_errors_total`            | Частота ошибок превышает 5% от числа попыток реконсиляции            |
| Повреждённые отсоединённые части           | `ClickHouse_CustomMetric_NumberOfBrokenDetachedParts`  | Более 100 повреждённых отсоединённых частей в течение более 60 минут |
| Потеря данных                              | `ClickHouse_CustomMetric_LostPartCount`                | Обнаружены потерянные части (возможна потеря данных)                 |
| Сервер завис при завершении работы         | `ClickHouseMetrics_IsServerShuttingDown`               | Сервер завершает работу уже более часа                               |
| Реплики таблицы в режиме только для чтения | `ClickHouse_CustomMetric_TableReadOnlyDurationSeconds` | Таблица находится в режиме только для чтения более часа              |

<div id="standard-kubernetes-alerts">
  ## Стандартные оповещения Kubernetes
</div>

В дополнение к оповещениям, специфичным для ClickHouse, добавьте стандартные оповещения Kubernetes для:

* Подов в состоянии CrashLoopBackOff
* Подов, которые не удаётся разместить на узлах
* Других проблем, связанных с инфраструктурой и характерных для вашей среды
