Сброс паролей пользователей ClickHouse с помощью Private API.

<div id="prerequisites">
  ## Предварительные требования
</div>

* ClickHouse Private API установлен и доступен (например, через проброс порта на `http://localhost:8080/`)
* Развернут как минимум один кластер ClickHouse (в этом руководстве используется `default-xx-01`)

<Steps>
  <Step>
    ## 1. Сгенерируйте хеш пароля

    Вычислите SHA-256-хеш нового пароля и закодируйте результат в base64:

    ```ini
    PASSWORD='My super secret p@$$w0rd'
    HASHED_PASSWORD=$(echo -n "$PASSWORD" | shasum -a 256 | awk '{printf $1}' | base64)
    ```
  </Step>

  <Step>
    ## 2. Создайте тело запроса

    Создайте JSON‑файл с хешем пароля:

    ```json
    {
      "user_hashed_password": "<base64-encoded-sha256-hash>",
      "hashing_function": "sha256",
      "username": "default"
    }
    ```

    Сохраните его как `password_reset.json`.

    | Поле                   | Обязательно | По умолчанию | Описание                                             |
    | ---------------------- | ----------- | ------------ | ---------------------------------------------------- |
    | `user_hashed_password` | Да          | --           | Base64-кодированный хеш нового пароля                |
    | `hashing_function`     | Нет         | `sha256`     | Используемый алгоритм хеширования                    |
    | `username`             | Нет         | `default`    | Имя пользователя, для которого нужно сбросить пароль |
  </Step>

  <Step>
    ## 3. Отправьте запрос

    ```bash
    curl -X POST "http://localhost:8080/api/v1/instances/default-xx-01/reset-user-password" \
        -H "Content-Type: application/json" \
        --data-binary "@password_reset.json"
    ```

    Успешный ответ:

    ```json
    {
      "message": "User password reset successfully"
    }
    ```
  </Step>

  <Step>
    ## 4. Проверьте изменение

    Сброс пароля вступит в силу после того, как оператор обработает изменение. Проверьте это, подключившись к кластеру с новым паролем.

    Также можно отслеживать операцию через конечную точку status:

    ```bash
    curl "http://localhost:8080/api/v1/instances/default-xx-01/status"
    ```
  </Step>
</Steps>

<div id="hydra-limitation">
  ## Ограничение Hydra
</div>

Сбросить пароль **нельзя** на дочерних инстансах Hydra. Чтобы сбросить пароль для дочернего инстанса Hydra, выполните сброс на родительском инстансе.
