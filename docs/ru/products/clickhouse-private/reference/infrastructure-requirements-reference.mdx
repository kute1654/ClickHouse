  ### AWS
</div>

| Компонент       | Спецификация                                     |
| --------------- | ------------------------------------------------ |

<div id="iam-service-account-requirements">
  ## Требования к IAM / сервисному аккаунту
</div>

<div id="aws">
  ### AWS
</div>

<div id="ecr-pull-role">
  #### Роль ECR Pull
</div>

* **Имя роли:** `ClickHouseAirgapECRPullRole`
* **Назначение:** Получение образов из приватного ECR ClickHouse
* **Разрешения:** `ecr:GetDownloadUrlForLayer`, `ecr:BatchGetImage`, `ecr:BatchCheckLayerAvailability`, `ecr:DescribeImages`, `ecr:ListImages`, `ecr:GetAuthorizationToken`
* **Действие:** Передайте ARN роли команде ClickHouse

<div id="clickhouse-serverkeeper-s3-role">
  #### S3-роль для ClickHouse Server/Keeper
</div>

* **Одна роль для каждого подготовленного кластера ClickHouse**
* **Соглашение об именовании:** `CH-S3-$NAME-$REGION-$ORDINAL-Role`
* `$NAME`: имя кластера (например, `default-xx-01`)
* `$REGION`: сокращённое обозначение региона (например, `uw2` для `us-west-2`)
* `$ORDINAL`: зарезервировано; укажите `00`
* Пример: `CH-S3-default-xx-01-uw2-00-Role`
* **Разрешения:** `s3:*`, `s3:ListBucket` для ресурса бакета
* **Политика доверия:** IRSA через `sts:AssumeRoleWithWebIdentity` с OIDC-провайдером кластера EKS

<div id="ebs-csi-driver-role">
  #### Роль драйвера EBS CSI
</div>

* **Название примера:** `ClickHouse_EksEbsCsiDriverRole`
* **Управляемая политика:** `arn:aws:iam::aws:policy/service-role/AmazonEBSCSIDriverPolicy`
* **Политика доверия:** IRSA с OIDC-провайдером кластера EKS

<div id="clickhouse-workload-identity-service-account">
  #### Сервисный аккаунт Workload Identity для ClickHouse
</div>

<div id="cluster-naming-conventions">
  ## Правила именования кластеров
</div>

Правила именования кластеров ClickHouse одинаковы на всех платформах:

```text
$DESCRIPTOR-$LETTERS-$ORDINAL
```

| Компонент     | Описание                                 | Пример    |
| ------------- | ---------------------------------------- | --------- |
| `$DESCRIPTOR` | Описательное имя (только буквы)          | `default` |
| `$LETTERS`    | Зарезервировано; любые две буквы         | `xx`      |
| `$ORDINAL`    | Порядковый номер, увеличивающийся с `01` | `01`      |

**Пример имени:** `default-xx-01`

Имя кластера должно быть **уникальным в пределах целевого кластера Kubernetes** и используется в: - Пространство имен Kubernetes: `ns-$CLUSTER_NAME` - Ресурсы Kubernetes: с префиксом `c-$CLUSTER_NAME` - Имена ролей IAM (AWS): `CH-S3-$CLUSTER_NAME-$REGION-$ORDINAL-Role` - Префиксы ключей S3/GCS: `ch-s3-<uuid>` (уникальны для каждого кластера)
