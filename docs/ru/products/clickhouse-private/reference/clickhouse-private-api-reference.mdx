ClickHouse Private API — это автономный дополнительный компонент для управления кластерами ClickHouse Private. Он предоставляет REST-конечные точки для резервного копирования, вертикального масштабирования, сброса паролей и мониторинга состояния кластера.

***

<div id="configuration-options">
  ## Параметры конфигурации
</div>

API устанавливается через Helm-чарт (`helm/airgap-management`).

| Значение Helm                | Тип         | По умолчанию | Описание                                               |
| ---------------------------- | ----------- | ------------ | ------------------------------------------------------ |
| `image.repository`           | строка      | --           | Репозиторий ECR/GAR для образа Private API             |
| `image.tag`                  | строка      | --           | Тег образа для развертывания                           |
| `api.port`                   | целое число | `8080`       | Порт, на котором API прослушивает подключения          |
| `api.basicAuth.enabled`      | булевый     | `false`      | Включить базовую HTTP-аутентификацию                   |
| `api.basicAuth.username`     | строка      | `"admin"`    | Имя пользователя для базовой аутентификации            |
| `api.basicAuth.password`     | строка      | `"password"` | Пароль для базовой аутентификации                      |
| `serviceAccount.enabled`     | булевый     | `true`       | Создать сервисный аккаунт для API                      |
| `serviceAccount.annotations` | map         | `{}`         | Аннотации для сервисного аккаунта (например, для IRSA) |

***

<div id="authentication">
  ## Аутентификация
</div>

По умолчанию базовая аутентификация отключена. Для продакшн-среды включите её через Helm:

```text
--set-json='api.basicAuth.enabled=true'
--set-json='api.basicAuth.username="admin"'
--set-json='api.basicAuth.password="YOUR_SECURE_PASSWORD"'
```

Если параметр включен, добавляйте учетные данные в запросы:

```bash
curl -u admin:YOUR_SECURE_PASSWORD http://localhost:8080/api/v1/...
```

<div id="resource-quantity-format">
  ## Формат указания количества ресурсов
</div>

API принимает значения в формате quantity ресурсов Kubernetes для полей CPU и памяти.

<div id="cpu-quantities">
  ### Количество CPU
</div>

| Формат           | Пример   | Значение     |
| ---------------- | -------- | ------------ |
| Целое число      | `"4"`    | 4 ядра CPU   |
| Десятичное число | `"2.5"`  | 2.5 ядра CPU |
| Миллиядра        | `"500m"` | 0.5 ядра CPU |

<div id="memory-quantities">
  ### Объёмы памяти
</div>

| Формат                   | Пример     | Значение       |
| ------------------------ | ---------- | -------------- |
| Двоичный (рекомендуется) | `"16Gi"`   | 16 гибибайт    |
| Двоичный                 | `"4096Mi"` | 4096 мебибайт  |
| Десятичный               | `"16G"`    | 16 гигабайт    |
| Десятичный               | `"16000M"` | 16000 мегабайт |

Для указания объёма памяти рекомендуется использовать двоичные единицы (Ki, Mi, Gi), чтобы следовать принятым в Kubernetes соглашениям.

<div id="backups">
  ### Резервные копии
</div>

<div id="create-backup">
  #### Создать резервную копию
</div>

`POST /api/v1/backups?instance_id={instance_id}`

Создаёт новую резервную копию для указанного кластера ClickHouse.

**Параметры запроса:**

| Параметр      | Тип    | Обязательно | Описание                                            |
| ------------- | ------ | ----------- | --------------------------------------------------- |
| `instance_id` | string | Да          | Имя кластера ClickHouse (например, `default-xx-01`) |

**Тело запроса:**

```json
{
    "incremental": false,
    "baseBackupUuid": "uuid-of-previous-backup",
    "databases": ["mydb"],
    "tables": ["mydb2.table"]
}
```

| Поле             | Тип          | Обязательно | Описание                                                                                                                         |
| ---------------- | ------------ | ----------- | -------------------------------------------------------------------------------------------------------------------------------- |
| `incremental`    | булевый      | Нет         | `false` для полной резервной копии, `true` для инкрементной. По умолчанию — `false`.                                             |
| `baseBackupUuid` | строка       | Нет         | UUID предыдущей резервной копии (обязательно, если `incremental` имеет значение `true`)                                          |
| `databases`      | массив строк | Нет         | Конкретные базы данных для резервного копирования. Не указывайте, чтобы включить все базы данных.                                |
| `tables`         | массив строк | Нет         | Конкретные таблицы для резервного копирования. Используйте полные имена (`db.table`). Не указывайте, чтобы включить все таблицы. |

**Ответ:** `201 Created` — возвращает созданный объект резервной копии, включая UUID.

**Поведение:** Создает пользовательский ресурс `Backup` в пространстве имен кластера ClickHouse. Оператор отслеживает появление новых объектов `Backup` и выполняет соответствующий SQL-оператор резервного копирования, контролируя состояние через системные таблицы.

***

<div id="get-backup">
  #### Получить резервную копию
</div>

`GET /api/v1/backups/{backup_id}?instance_id={instance_id}`

Возвращает статус и сведения о конкретной резервной копии.

**Параметры пути:**

| Параметр    | Тип    | Обязательный | Описание             |
| ----------- | ------ | ------------ | -------------------- |
| `backup_id` | string | Да           | UUID резервной копии |

**Параметры запроса:**

| Параметр      | Тип    | Обязательный | Описание                |
| ------------- | ------ | ------------ | ----------------------- |
| `instance_id` | string | Да           | Имя кластера ClickHouse |

**Ответ:** `200 OK` -- Возвращает объект резервной копии. Поле `status.state` указывает на состояние завершения (например, `Ready`, когда операция завершена).

***

<div id="list-backups">
  #### Список резервных копий
</div>

`GET /api/v1/backups?instance_id={instance_id}`

Возвращает все резервные копии для указанного кластера.

**Параметры запроса:**

| Параметр            | Тип     | Обязательный | Описание                                                |
| ------------------- | ------- | ------------ | ------------------------------------------------------- |
| `instance_id`       | string  | Да           | Имя кластера ClickHouse                                 |
| `status__state__eq` | string  | Нет          | Фильтр по состоянию резервной копии (например, `Ready`) |
| `sort`              | string  | Нет          | Поле сортировки (например, `status__finishTime`)        |
| `limit`             | integer | Нет          | Максимальное количество возвращаемых результатов        |

**Ответ:** `200 OK` -- Возвращает массив объектов резервных копий.

**Пример:** Найти UUID последней успешно созданной резервной копии:

```text
GET /api/v1/backups?instance_id=default-xx-01&status__state__eq=Ready&sort=status__finishTime&limit=1
```

<div id="restore-backup">
  #### Восстановление резервной копии
</div>

`POST /api/v1/backups/{backup_id}/restore?instance_id={instance_id}&target_instance_id={target_instance_id}`

Восстанавливает резервную копию в целевой кластер ClickHouse. Выполняет `RESTORE ALL` для резервной копии.

**Параметры пути:**

| Параметр    | Тип    | Обязательный | Описание                                         |
| ----------- | ------ | ------------ | ------------------------------------------------ |
| `backup_id` | string | Да           | UUID резервной копии, которую нужно восстановить |

**Параметры запроса:**

| Параметр             | Тип    | Обязательный | Описание                                                                   |
| -------------------- | ------ | ------------ | -------------------------------------------------------------------------- |
| `instance_id`        | string | Да           | Имя исходного кластера ClickHouse (в котором была создана резервная копия) |
| `target_instance_id` | string | Да           | Имя целевого кластера ClickHouse (в который выполняется восстановление)    |

**Ограничения:** - Восстановление в тот же кластер (`instance_id == target_instance_id`) **запрещено** из соображений безопасности. - Рекомендуется выполнять восстановление в новый кластер, чтобы не перегружать исходный.

**Ответ:** `202 Accepted` -- Возвращает идентификаторы операций восстановления.

<div id="vertical-scaling">
  ### Вертикальное масштабирование
</div>

<div id="scale-cluster">
  #### Масштабирование кластера
</div>

`POST /api/v1/instances/{instance_id}/scale`

Выполняет вертикальное масштабирование кластера ClickHouse, изменяя ресурсы CPU и памяти.

**Параметры пути:**

| Параметр      | Тип    | Обязательный | Описание                |
| ------------- | ------ | ------------ | ----------------------- |
| `instance_id` | string | Да           | имя кластера ClickHouse |

**Тело запроса:**

```json
{
  "vertical": {
    "resources": {
      "cpu": "4",
      "memory": "16Gi"
    }
  }
}
```

| Поле                        | Тип    | Обязательно | Описание                                                       |
| --------------------------- | ------ | ----------- | -------------------------------------------------------------- |
| `vertical.resources.cpu`    | string | Нет*        | Выделенный объём CPU в формате quantity ресурсов Kubernetes    |
| `vertical.resources.memory` | string | Нет*        | Выделенный объём памяти в формате quantity ресурсов Kubernetes |

*Необходимо указать как минимум один из параметров: `cpu` или `memory`.

**Соотношение CPU и памяти:** API поддерживает **соотношение 1:4** (1 ядро CPU на 4 GiB памяти) с допуском 5%. Если указан только один ресурс, второй автоматически рассчитывается, чтобы сохранить это соотношение.

**Поведение:** Обновляет `ServerPodPolicy` пользовательского ресурса `ClickhouseCluster`. Оператор выполняет **роллинг-перезапуск** StatefulSets с новым выделением ресурсов.

**Важно:** В отличие от ClickHouse Cloud, ClickHouse Private **не** использует масштабирование Make Before Break (MBB). Вертикальное масштабирование вызывает роллинг-перезапуск, который может привести к временной недоступности сервиса.

**Ответ:** `201 Created`

<div id="password-reset">
  ### Сброс пароля
</div>

<div id="reset-user-password">
  #### Сброс пароля пользователя
</div>

`POST /api/v1/instances/{instance_id}/reset-user-password`

Сбрасывает пароль пользователя кластера ClickHouse.

**Параметры пути:**

| Параметр      | Тип    | Обязательно | Описание                |
| ------------- | ------ | ----------- | ----------------------- |
| `instance_id` | string | Да          | Имя кластера ClickHouse |

**Тело запроса:**

```json
{
  "user_hashed_password": "<base64-encoded-sha256-hash>",
  "hashing_function": "sha256",
  "username": "default"
}
```

| Поле                   | Тип    | Обязательно | По умолчанию | Описание                                             |
| ---------------------- | ------ | ----------- | ------------ | ---------------------------------------------------- |
| `user_hashed_password` | string | Да          | --           | Хэш нового пароля, закодированный в Base64           |
| `hashing_function`     | string | Нет         | `"sha256"`   | Используемый алгоритм хеширования                    |
| `username`             | string | Нет         | `"default"`  | Имя пользователя, для которого нужно сбросить пароль |

**Генерация хэша пароля:**

```ini
PASSWORD='My super secret p@$$w0rd'
HASHED_PASSWORD=$(echo -n "$PASSWORD" | shasum -a 256 | awk '{printf $1}' | base64)
```

**Ограничения:** - Сбросить пароль на дочерних инстансах Hydra **нельзя**. Выполните сброс на родительском инстансе.

**Ответ:**

```json
{
  "message": "User password reset successfully"
}
```

**Поведение:** Обновляет поле `CustomerAccount` в пользовательском ресурсе `ClickhouseCluster`. Оператор применяет изменение учётных данных к кластеру. Пароль начинает действовать после того, как оператор обработает это изменение.

<div id="cluster-status">
  ### Статус кластера
</div>

<div id="get-cluster-status">
  #### Получить статус кластера
</div>

`GET /api/v1/instances/{instance_id}/status`

Возвращает текущее состояние указанного кластера ClickHouse.

**Параметры пути:**

| Параметр      | Тип    | Обязательный | Описание                |
| ------------- | ------ | ------------ | ----------------------- |
| `instance_id` | строка | Да           | Имя кластера ClickHouse |

**Ответ:**

```json
{
  "state": "Running",
  "previousState": "Provisioning",
  "message": "Cluster is healthy",
  "stateProvidedBy": "operator"
}
```

| Поле              | Тип    | Описание                                                |
| ----------------- | ------ | ------------------------------------------------------- |
| `state`           | string | Текущее состояние кластера ClickHouse                   |
| `previousState`   | string | Состояние, из которого перешёл кластер                  |
| `message`         | string | Подробности о том, почему изменилось состояние кластера |
| `stateProvidedBy` | string | Процесс, вызвавший изменение состояния                  |

**Сценарии использования:** - Проверка завершения операции вертикального масштабирования. - Подтверждение того, что сброс пароля применён во всём кластере. - Отслеживание хода развёртывания кластера.
