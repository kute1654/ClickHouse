<div id="private-regions">
  ## Частные регионы
</div>

<EnterprisePlanFeatureBadge feature="Возможность частных регионов" />

Мы предлагаем частные регионы для наших сервисов уровня Enterprise. По вопросам подключения частного региона [свяжитесь с нами](https://clickhouse.com/company/contact).

Ключевые особенности частных регионов:

* Сервисы не поддерживают автомасштабирование; однако поддерживается ручное вертикальное и горизонтальное масштабирование.
* Сервисы нельзя перевести в неактивное состояние.
* Страница статуса недоступна для частных регионов.

Для соответствия требованиям HIPAA могут предъявляться дополнительные требования (включая подписание BAA). Обратите внимание, что HIPAA в настоящее время доступна только для сервисов уровня Enterprise

<div id="hipaa-compliant-regions">
  ## Регионы, соответствующие требованиям HIPAA
</div>

<EnterprisePlanFeatureBadge feature="HIPAA" support="true" />

Чтобы развернуть сервисы в регионах, соответствующих требованиям HIPAA, клиенты должны подписать соглашение Business Associate Agreement (BAA) и запросить онбординг через отдел продаж или службу поддержки. Следующие регионы поддерживают соответствие требованиям HIPAA:

* AWS af-south-1 (Южная Африка) **Частный регион**
* AWS ca-central-1 (Канада) **Частный регион**
* AWS eu-central-1 (Франкфурт)
* AWS eu-north-1 (Стокгольм) **Частный регион**
* AWS eu-west-1 (Ирландия)
* AWS eu-west-2 (Лондон)
* AWS sa-east-1 (Южная Америка) **Частный регион**
* AWS us-east-1 (Северная Виргиния)
* AWS us-east-2 (Огайо)
* AWS us-west-2 (Орегон)
* GCP europe-west4 (Нидерланды)
* GCP us-central1 (Айова)
* GCP us-east1 (Южная Каролина)
