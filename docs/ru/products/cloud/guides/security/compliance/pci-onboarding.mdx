ClickHouse предлагает сервисы, соответствующие стандарту безопасности данных индустрии платежных карт (PCI-DSS), и проходит аудит на соответствие требованиям уровня 1 для поставщиков услуг. Клиенты могут обрабатывать первичные номера счетов (PAN) в этих сервисах, включив эту возможность и развернув сервисы в регионе, соответствующем требованиям.

Дополнительные сведения о программе соответствия ClickHouse и доступности отчета стороннего аудита см. в нашем [обзоре соответствия требованиям](/ru/products/cloud/reference/security/compliance-overview). Чтобы получить копию нашего документа о разделении ответственности PCI, посетите наш [Trust Center](https://trust.clickhouse.com). Кроме того, клиентам следует ознакомиться с нашей страницей [возможностей безопасности](/ru/products/cloud/features/security), чтобы выбрать и внедрить подходящие меры защиты для своих рабочих нагрузок.

На этой странице описано, как включить развертывание сервисов, соответствующих PCI, в ClickHouse Cloud.

<Steps>
  <Step>
    ### Подключите сервисы Enterprise

    1. Выберите имя своей организации в левом нижнем углу консоли.
    2. Нажмите **Billing**.
    3. Проверьте свой **Plan** в левом верхнем углу.
    4. Если ваш **Plan** — **Enterprise**, перейдите к следующему разделу. Если нет, нажмите **Change plan**.
    5. Выберите **Switch to Enterprise**.
  </Step>

  <Step>
    ### Включите PCI для своей организации

    1. Выберите имя своей организации в левом нижнем углу консоли.
    2. Нажмите **Organization details**.
    3. Включите переключатель **Enable PCI**.

    <br />

    <Image img="/images/cloud/security/compliance/pci_1.webp" size="md" alt="Включение PCI" background="black" />

    <br />

    4. После этого в организации можно будет развертывать PCI-сервисы.

    <br />

    <Image img="/images/cloud/security/compliance/pci_2.webp" size="md" alt="PCI включен" background="black" />

    <br />
  </Step>

  <Step>
    ### Разверните сервисы в регионах, соответствующих PCI

    1. Выберите **New service** в левом верхнем углу главного экрана консоли
    2. Измените **Region type** на **HIPAA compliant**

    <br />

    <Image img="/images/cloud/security/compliance/pci_3.webp" size="md" alt="Развертывание в регионе PCI" background="black" />

    <br />

    3. Введите имя сервиса и заполните остальные поля

    Полный список облачных провайдеров и сервисов, соответствующих PCI, см. на странице [Supported cloud regions](/ru/products/cloud/reference/supported-regions).
  </Step>
</Steps>

<div id="migrate-to-hipaa">
  ## Перенос существующих сервисов
</div>

Клиентам настоятельно рекомендуется при необходимости развертывать сервисы в средах, соответствующих требованиям. Процесс переноса сервисов из стандартного региона в регион, соответствующий требованиям PCI, включает восстановление из резервной копии и может потребовать некоторого времени простоя.

Если требуется перенос из стандартных регионов в регионы, соответствующие требованиям PCI, выполните следующие шаги для миграции в режиме самообслуживания:

1. Выберите сервис, который нужно перенести.
2. Нажмите **Backups** слева.
3. Нажмите на три точки слева от резервной копии, которую нужно восстановить.
4. Выберите **Region type**, чтобы восстановить резервную копию в регион, соответствующий требованиям PCI.
5. После завершения восстановления выполните несколько запросов, чтобы убедиться, что схемы и количество записей соответствуют ожидаемым.
6. Удалите старый сервис.

<Info>
  **Ограничения**

  Сервисы должны оставаться в рамках того же облачного провайдера и географического региона. Этот процесс переносит сервис в соответствующую требованиям среду в рамках того же облачного провайдера и региона.
</Info>
