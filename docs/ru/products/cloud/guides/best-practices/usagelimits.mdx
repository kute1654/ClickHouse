Хотя ClickHouse известен своей скоростью и надёжностью, оптимальная производительность
достигается при соблюдении определённых рабочих параметров. Например, слишком большое количество таблиц,
баз данных или частей может негативно сказаться на производительности. Чтобы этого избежать, ClickHouse
Cloud вводит ограничения по нескольким эксплуатационным параметрам.
Подробная информация об этих ограничениях приведена ниже.

<Tip>
  Если вы столкнулись с одним из этих ограничений, возможно, ваш
  сценарий использования реализован неоптимально. Свяжитесь с нашей командой поддержки, и
  мы с радостью поможем вам оптимизировать ваш сценарий использования, чтобы избежать превышения ограничений,
  или вместе рассмотрим, как можно контролируемо увеличить эти лимиты.
</Tip>

| Параметр                       | Лимит                                                                                                           |
| ------------------------------ | --------------------------------------------------------------------------------------------------------------- |
| **Базы данных**                | 1000                                                                                                            |
| **Таблицы**                    | 5000                                                                                                            |
| **Столбцы**                    | ∼1000 (предпочтителен wide format, а не compact)                                                                |
| **Партиции**                   | 50k                                                                                                             |
| **Части**                      | 10k (см. настройку [`max_parts_in_total`](/ru/products/cloud/guides/cloud-compatibility#max_parts_in_total-10000)) |
| **Размер части**               | 150gb                                                                                                           |
| **Сервисов на организацию**    | 20 (soft)                                                                                                       |
| **Сервисов на хранилище**      | 5 (soft)                                                                                                        |
| **Реплик на сервис**           | 20 (soft)                                                                                                       |
| **Низкая кардинальность**      | 10k или меньше                                                                                                  |
| **Первичных ключей в таблице** | 4-5, при условии, что они достаточно хорошо фильтруют данные                                                    |
| **Параллелизм запросов**       | 1000 (на реплику)                                                                                               |
| **Приём батчами**              | всё, что &gt; 1M, система разобьёт на блоки по 1M строк                                                         |

<Note>
  Для сервисов с одной репликой максимальное количество баз данных ограничено
  100, а максимальное количество таблиц — 500. Кроме того, объём хранилища
  для сервисов уровня Basic ограничен 1 ТБ.
</Note>
