Сервисы уровней Scale и Enterprise поддерживают автомасштабирование на основе загрузки CPU и использования памяти. Показатели использования сервиса непрерывно отслеживаются в рамках окна ретроспективного анализа, на основе которого принимаются решения о масштабировании. Если эти показатели поднимаются выше или опускаются ниже определённых пороговых значений, сервис масштабируется соответствующим образом, чтобы соответствовать нагрузке.

<div id="configuring-vertical-auto-scaling">
  ## Настройка вертикального автомасштабирования
</div>

Участники организации с ролью **Admin** могут изменять параметры масштабирования сервисов ClickHouse Cloud уровней Scale или Enterprise. Чтобы настроить вертикальное автомасштабирование, перейдите на вкладку **Настройки** вашего сервиса и задайте минимальный и максимальный объём памяти, а также параметры CPU, как показано ниже.

<Note>
  Для сервисов с одной репликой масштабирование доступно не на всех уровнях.
</Note>

<Image img="/images/cloud/manage/AutoScaling.webp" size="lg" alt="Страница настроек масштабирования" border />

Задайте для реплик значение **Максимальный объём памяти** больше, чем **Минимальный объём памяти**. После этого сервис будет масштабироваться по мере необходимости в пределах этих значений. Эти настройки также доступны при первоначальном создании сервиса. Каждой реплике в вашем сервисе будет выделен одинаковый объём памяти и одинаковые ресурсы CPU.

Вы также можете задать одинаковые значения, фактически «закрепив» сервис за определённой конфигурацией. В этом случае сервис будет немедленно принудительно масштабирован до выбранного размера.

Важно учитывать, что это отключит любое автомасштабирование кластера, и ваш сервис не будет защищён от роста использования CPU или памяти сверх заданных значений.

<Note>
  Для сервисов уровня Enterprise вертикальное автомасштабирование поддерживается стандартными профилями 1:4. Пользовательские профили не поддерживают ни вертикальное автомасштабирование, ни ручное вертикальное масштабирование. Однако такие сервисы можно масштабировать по вертикали, обратившись в службу поддержки.
</Note>
