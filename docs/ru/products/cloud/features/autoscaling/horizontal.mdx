<div id="manual-horizontal-scaling">
  ## Ручное горизонтальное масштабирование
</div>

<ScalePlanFeatureBadge feature="Ручное горизонтальное масштабирование" />

Вы можете использовать [публичные API](/api-reference/organization/get-list-of-available-organizations#/paths/~1v1~1organizations~1:organizationId~1services~1:serviceId~1scaling/patch) ClickHouse Cloud, чтобы масштабировать сервис, обновив его параметры масштабирования, или изменить количество реплик в облачной консоли.

Уровни **Scale** и **Enterprise** также поддерживают сервисы с одной репликой. После масштабирования сервисы можно уменьшить обратно до одной реплики. Обратите внимание, что сервисы с одной репликой имеют более низкую доступность и не рекомендуются для использования в продакшне.

<Note>
  Сервисы можно масштабировать по горизонтали максимум до 20 реплик. Если вам требуется больше реплик, обратитесь в службу поддержки.
</Note>

<div id="horizontal-scaling-via-api">
  ### Горизонтальное масштабирование через API
</div>

Чтобы выполнить горизонтальное масштабирование кластера, отправьте через API запрос `PATCH`, изменив число реплик. На снимках экрана ниже показан вызов API для масштабирования кластера с `3` реплик до `6`, а также соответствующий ответ.

<Image img="/images/cloud/manage/scaling-patch-request.webp" size="lg" alt="PATCH-запрос на масштабирование" border />

*Запрос `PATCH` для обновления `numReplicas`*

<Image img="/images/cloud/manage/scaling-patch-response.webp" size="md" alt="Ответ на PATCH-запрос масштабирования" border />

*Ответ на запрос `PATCH`*

Если отправить новый запрос на масштабирование или несколько запросов подряд, пока один из них уже выполняется, сервис масштабирования проигнорирует промежуточные состояния и приведёт кластер к итоговому числу реплик.

<div id="horizontal-scaling-via-ui">
  ### Горизонтальное масштабирование через интерфейс
</div>

Чтобы выполнить горизонтальное масштабирование сервиса через интерфейс, можно изменить количество реплик сервиса на странице **Настройки**.

<Image img="/images/cloud/manage/scaling-configure.webp" size="md" alt="Параметры масштабирования" border />

*Параметры масштабирования сервиса в консоли ClickHouse Cloud*

После масштабирования сервиса панель мониторинга метрик в облачной консоли должна показывать корректное выделение ресурсов для сервиса. На снимке экрана ниже показано, что кластер масштабирован до общего объёма памяти `96 GiB`, то есть до `6` реплик с выделением по `16 GiB` памяти для каждой.

<Image img="/images/cloud/manage/scaling-memory-allocation.webp" size="md" alt="Выделение памяти при масштабировании" border />
