SQL-консоль — самый быстрый и простой способ изучать ваши базы данных в ClickHouse Cloud и выполнять к ним запросы. С помощью SQL-консоли вы можете:

* Подключаться к сервисам ClickHouse Cloud
* Просматривать, фильтровать и сортировать данные в таблицах
* Выполнять запросы и визуализировать результаты всего за несколько щелчков
* Делиться запросами с участниками команды и эффективнее работать вместе.

<div id="viewing-table-list-and-schema-info">
  ### Просмотр списка таблиц и сведений о схеме
</div>

Обзор таблиц в вашем экземпляре ClickHouse доступен в левой боковой панели. Используйте селектор базы данных в верхней части левой панели, чтобы просмотреть таблицы в конкретной базе данных

<Image img="/images/cloud/sqlconsole/table-list-and-schema.webp" size="md" alt="список таблиц и схема" />

Таблицы в списке также можно развернуть, чтобы просмотреть столбцы и типы

<Image img="/images/cloud/sqlconsole/view-columns.webp" size="md" alt="просмотр столбцов" />

<div id="exploring-table-data">
  ### Просмотр данных таблицы
</div>

Нажмите на таблицу в списке, чтобы открыть её в новой вкладке. В представлении таблицы данные можно легко просматривать, выделять и копировать. Обратите внимание: при копировании и вставке в приложения для работы с электронными таблицами, такие как Microsoft Excel и Google Sheets, структура и форматирование сохраняются. Переключаться между страницами данных таблицы (по 30 строк на страницу) можно с помощью навигации в нижнем колонтитуле.

<Image img="/images/cloud/sqlconsole/abc.webp" size="md" alt="abc" />

  ### Просмотр данных в ячейке
</div>

Инструмент Cell Inspector позволяет просматривать большие объёмы данных, содержащиеся в одной ячейке. Чтобы открыть его, щёлкните правой кнопкой мыши по ячейке и выберите &#39;Inspect Cell&#39;. Содержимое Cell Inspector можно скопировать, нажав значок копирования в правом верхнем углу.

<div id="filtering-and-sorting-tables">
  ## Фильтрация и сортировка таблиц
</div>

<div id="sorting-a-table">
  ### Сортировка таблицы
</div>

Чтобы отсортировать таблицу в SQL-консоли, откройте таблицу и нажмите кнопку &#39;Sort&#39; на панели инструментов. Откроется меню, в котором можно настроить параметры сортировки. Выберите столбец, по которому нужно выполнить сортировку, и задайте порядок сортировки (по возрастанию или по убыванию). Выберите &#39;Apply&#39; или нажмите Enter, чтобы отсортировать таблицу.

<Image img="/images/cloud/sqlconsole/sort-descending-on-column.webp" size="md" alt="сортировка по убыванию по столбцу" />

SQL-консоль также позволяет добавлять для таблицы несколько сортировок. Нажмите кнопку &#39;Sort&#39; ещё раз, чтобы добавить ещё одну сортировку.

<Note>
  Сортировки применяются в том порядке, в котором они отображаются на панели сортировки (сверху вниз). Чтобы удалить сортировку, просто нажмите кнопку &#39;x&#39; рядом с ней.
</Note>

<div id="filtering-a-table">
  ### Фильтрация таблицы
</div>

Чтобы отфильтровать таблицу в SQL-консоли, откройте таблицу и нажмите кнопку &#39;Filter&#39;. Как и в случае с сортировкой, эта кнопка откроет меню, в котором можно настроить фильтр. Вы можете выбрать столбец, по которому нужно фильтровать данные, и задать необходимые критерии. SQL-консоль интеллектуально показывает параметры фильтрации, соответствующие типу данных в столбце.

<Image img="/images/cloud/sqlconsole/filter-on-radio-column-equal-gsm.webp" size="md" alt="фильтр по столбцу radio со значением GSM" />

Когда результат фильтрации вас устроит, нажмите &#39;Apply&#39;, чтобы отфильтровать данные. Вы также можете добавить дополнительные фильтры, как показано ниже.

<Image img="/images/cloud/sqlconsole/add-more-filters.webp" size="md" alt="Добавление фильтра для диапазона со значением больше 2000" />

Как и в случае с сортировкой, нажмите кнопку &#39;x&#39; рядом с фильтром, чтобы удалить его.

<div id="filtering-and-sorting-together">
  ### Одновременная фильтрация и сортировка
</div>

SQL-консоль позволяет одновременно фильтровать и сортировать таблицу. Для этого добавьте все нужные фильтры и параметры сортировки, как описано выше, и нажмите кнопку &#39;Apply&#39;.

<Image img="/images/cloud/sqlconsole/filtering-and-sorting-together.webp" size="md" alt="Добавьте фильтр для диапазона со значением больше 2000" />

<div id="creating-a-query-from-filters-and-sorts">
  ### Создание запроса из фильтров и сортировки
</div>

SQL-консоль позволяет одним щелчком преобразовать выбранные параметры сортировки и фильтрации в запрос. Просто нажмите кнопку &#39;Создать запрос&#39; на панели инструментов, предварительно задав нужные параметры сортировки и фильтрации. После нажатия &#39;Создать запрос&#39; откроется новая вкладка запроса, уже заполненная SQL-командой, соответствующей данным в текущем представлении таблицы.

<Image img="/images/cloud/sqlconsole/create-a-query-from-sorts-and-filters.webp" size="md" alt="Создать запрос из сортировки и фильтров" />

<Note>
  Для использования функции &#39;Создать запрос&#39; фильтры и сортировка не обязательны.
</Note>

Подробнее о выполнении запросов в SQL-консоли см. в документации по запросам (link).

<div id="creating-and-running-a-query">
  ## Создание и выполнение запроса
</div>

<div id="creating-a-query">
  ### Создание запроса
</div>

Создать новый запрос в SQL-консоли можно двумя способами.

* Нажмите кнопку &#39;+&#39; на панели вкладок
* Выберите кнопку &#39;New Query&#39; в списке запросов на левой боковой панели

<Image img="/images/cloud/sqlconsole/creating-a-query.webp" size="md" alt="Создание запроса" />

<div id="running-a-query">
  ### Выполнение запроса
</div>

Чтобы выполнить запрос, введите SQL-команду или несколько команд в редактор SQL и нажмите кнопку &#39;Run&#39; либо используйте сочетание клавиш `cmd / ctrl + enter`. Чтобы написать и выполнить несколько команд последовательно, обязательно ставьте точку с запятой после каждой команды.

Параметры выполнения запроса
По умолчанию при нажатии кнопки запуска выполняются все команды, содержащиеся в редакторе SQL. SQL-консоль поддерживает ещё два варианта выполнения запросов:

* Выполнить выбранные команды
* Выполнить команду в позиции курсора

Чтобы выполнить выбранные команды, выделите нужную команду или последовательность команд и нажмите кнопку &#39;Run&#39; (или используйте сочетание клавиш `cmd / ctrl + enter`). Вы также можете выбрать &#39;Run selected&#39; в контекстном меню редактора SQL (оно открывается по щелчку правой кнопкой мыши в любом месте редактора), если есть выделение.

<Image img="/images/cloud/sqlconsole/run-selected-query.webp" size="md" alt="выполнение выбранного запроса" />

Команду в текущей позиции курсора можно выполнить двумя способами:

* Выберите &#39;At Cursor&#39; в меню расширенных параметров запуска (или используйте соответствующее сочетание клавиш `cmd / ctrl + shift + enter`

<Image img="/images/cloud/sqlconsole/run-at-cursor-2.webp" size="md" alt="выполнение в позиции курсора" />

* Выберите &#39;Run at cursor&#39; в контекстном меню редактора SQL

<Image img="/images/cloud/sqlconsole/run-at-cursor.webp" size="md" alt="выполнение в позиции курсора" />

<Note>
  При выполнении команда в позиции курсора кратковременно подсветится жёлтым.
</Note>

<div id="canceling-a-query">
  ### Отмена запроса
</div>

Пока запрос выполняется, на панели инструментов редактора запросов вместо кнопки &#39;Run&#39; отображается кнопка &#39;Cancel&#39;. Просто нажмите эту кнопку или клавишу `Esc`, чтобы отменить запрос. Примечание: все результаты, которые уже были получены, сохранятся после отмены.

<Image img="/images/cloud/sqlconsole/cancel-a-query.webp" size="md" alt="Отмена запроса" />

<div id="saving-a-query">
  ### Сохранение запроса
</div>

Сохранив запрос, вы сможете легко найти его позже и поделиться им с коллегами. В SQL-консоли также можно упорядочивать запросы по папкам.

Чтобы сохранить запрос, просто нажмите кнопку &quot;Save&quot; на панели инструментов рядом с кнопкой &quot;Run&quot;. Введите нужное имя и нажмите &quot;Save Query&quot;.

<Note>
  Сочетание клавиш `cmd / ctrl` + s также сохранит все изменения в текущей вкладке запроса.
</Note>

<Image img="/images/cloud/sqlconsole/sql-console-save-query.webp" size="md" alt="Сохранение запроса" />

Кроме того, можно одновременно задать имя и сохранить запрос: нажмите &quot;Untitled Query&quot; на панели инструментов, измените имя и нажмите Enter:

<Image img="/images/cloud/sqlconsole/sql-console-rename.webp" size="md" alt="Переименование запроса" />

<div id="query-sharing">
  ### Общий доступ к запросам
</div>

SQL-консоль позволяет легко делиться запросами с участниками вашей команды. В SQL-консоли поддерживаются четыре уровня доступа, которые можно настраивать как глобально, так и отдельно для каждого пользователя:

* Владелец (может настраивать параметры общего доступа)
* Доступ на запись
* Доступ только для чтения
* Нет доступа

После сохранения запроса нажмите кнопку &quot;Share&quot; на панели инструментов. Появится модальное окно с параметрами общего доступа:

<Image img="/images/cloud/sqlconsole/sql-console-share.webp" size="md" alt="Поделиться запросом" />

Чтобы настроить доступ к запросу для всех участников организации, у которых есть доступ к сервису, просто измените уровень доступа в верхней строке:

<Image img="/images/cloud/sqlconsole/sql-console-edit-access.webp" size="md" alt="Изменить доступ" />

После этого запрос смогут просматривать (и выполнять) все участники команды, у которых есть доступ к SQL-консоли этого сервиса.

Чтобы настроить доступ к запросу для отдельных участников, выберите нужного участника команды в списке &quot;Add a team member&quot;:

<Image img="/images/cloud/sqlconsole/sql-console-add-team.webp" size="md" alt="Добавить участника команды" />

После выбора участника команды появится новая строка с выбором уровня доступа:

<Image img="/images/cloud/sqlconsole/sql-console-edit-member.webp" size="md" alt="Изменить доступ участника команды" />

  ### Доступ к общим запросам
</div>

Если с вами поделились запросом, он будет отображаться на вкладке &quot;Запросы&quot; на левой боковой панели SQL-консоли:

<Image img="/images/cloud/sqlconsole/sql-console-access-queries.webp" size="md" alt="Доступ к запросам" />

<div id="linking-to-a-query-permalinks">
  ### Ссылки на запрос (постоянные ссылки)
</div>

Для сохранённых запросов также создаются постоянные ссылки, поэтому вы можете отправлять и получать ссылки на общие запросы и открывать их напрямую.

Значения всех параметров, присутствующих в запросе, автоматически добавляются в URL сохранённого запроса как query-параметры. Например, если запрос содержит параметры `{start_date: Date}` и `{end_date: Date}`, постоянная ссылка может выглядеть так: `https://console.clickhouse.cloud/services/:serviceId/console/query/:queryId?param_start_date=2015-01-01&param_end_date=2016-01-01`.

<div id="searching-query-results">

Эта возможность помогает предварительно оценить, как будут выглядеть результаты с дополнительным условием `WHERE`, или просто проверить, что нужные данные входят в результирующий набор. <div id="visualizing-query-data">
  ## Визуализация данных запроса
</div>

Некоторые данные удобнее воспринимать в виде диаграмм. Вы можете всего за несколько щелчков создать визуализации на основе данных из результатов запроса прямо в SQL-консоли. В качестве примера мы используем запрос, который рассчитывает еженедельную статистику по поездкам на такси в Нью-Йорке:

```sql
SELECT
   toStartOfWeek(pickup_datetime) AS week,
   sum(total_amount) AS fare_total,
   sum(trip_distance) AS distance_total,
   count(*) AS trip_total
FROM
   nyc_taxi
GROUP BY
   1
ORDER BY
   1 ASC
```

<Image img="/images/cloud/sqlconsole/tabular-query-results.webp" size="md" alt="Результаты запроса в таблице" />

Без визуализации эти результаты трудно интерпретировать. Давайте представим их в виде графика.

<Image img="/images/cloud/sqlconsole/switch-from-query-to-chart.webp" size="md" alt="Переключение с запроса на диаграмму" />

Начнем с создания простой столбчатой диаграммы, показывающей `trip_total` по `week`. Для этого перетащим поле `week` на ось x, а поле `trip_total` — на ось y:

<Image img="/images/cloud/sqlconsole/trip-total-by-week.webp" size="md" alt="Общая сумма поездок по неделям" />

Большинство типов диаграмм поддерживают несколько полей на числовых осях. Для демонстрации перетащим поле `fare_total` на ось y:

<Image img="/images/cloud/sqlconsole/bar-chart.webp" size="md" alt="Столбчатая диаграмма" />

<div id="customizing-charts">
  ### Настройка диаграмм
</div>

SQL-консоль поддерживает десять типов диаграмм, которые можно выбрать в селекторе типа диаграммы на панели конфигурации диаграммы. Например, мы можем легко изменить тип предыдущей диаграммы с Bar на Area:

<Image img="/images/cloud/sqlconsole/change-from-bar-to-area.webp" size="md" alt="Изменение диаграммы с Bar на Area" />

Заголовки диаграмм соответствуют имени запроса, на основе которого строятся данные. Если изменить имя запроса, заголовок диаграммы также обновится:

<Image img="/images/cloud/sqlconsole/update-query-name.webp" size="md" alt="Обновление имени запроса" />

Некоторые дополнительные параметры диаграммы также можно настроить в разделе &#39;Advanced&#39; на панели конфигурации диаграммы. Для начала изменим следующие настройки:

* Подзаголовок
* Названия осей
* Ориентация подписей на оси x

Наша диаграмма изменится соответствующим образом:

<Image img="/images/cloud/sqlconsole/update-subtitle-etc.webp" size="md" alt="Обновление подзаголовка и т. д." />

В некоторых случаях может потребоваться настраивать масштаб осей независимо для каждого поля. Это можно сделать в разделе &#39;Advanced&#39; на панели конфигурации диаграммы, задав минимальное и максимальное значения диапазона оси. Например, диаграмма выше выглядит неплохо, но чтобы наглядно показать корреляцию между нашими полями `trip_total` и `fare_total`, диапазоны осей нужно немного скорректировать:

<Image img="/images/cloud/sqlconsole/adjust-axis-scale.webp" size="md" alt="Настройка масштаба оси" />
