Model Context Protocol (MCP) — это открытый стандарт для предоставления инструментов и источников данных модели ИИ. Подключение MCP-сервера к ClickHouse Agent дает агенту доступ ко всем возможностям, которые предоставляет этот сервер: трекерам задач, backend-системам обсервабилити, внутренним API, сторонним SaaS-сервисам и ко всему остальному, у чего есть конечная точка MCP.

<div id="attach-an-mcp-server">
  ## Подключите MCP-сервер
</div>

В Конструкторе агентов откройте раздел **MCP-серверы** и нажмите **Add server**. Введите URL сервера и параметры аутентификации, затем выберите, какие инструменты сервера должен использовать этот агент. Сохраните агента.

К одному агенту можно подключить несколько серверов. Каждый инструмент, который вызывает агент, записывается в диалоге, чтобы пользователи могли видеть, что сделал агент.

<div id="transport">
  ## Транспорт
</div>

ClickHouse Agents использует Streamable HTTP — transport MCP промышленного уровня. Подключаемый сервер должен быть доступен по HTTP(S) из ClickHouse Cloud.

<div id="authentication">
  ## Аутентификация
</div>

MCP-серверы могут требовать учетные данные. ClickHouse Agents поддерживает:

* **Bearer-токены** и другие статические заголовки — фиксированные значения, которые вы указываете при настройке сервера.
* **OAuth 2.0** — интерактивный поток. Когда вы (или любой пользователь с доступом) впервые вызывает инструмент на сервере, браузер открывает окно входа, а токены автоматически управляются и обновляются.
* **Учетные данные для каждого пользователя** — переменные в конфигурации сервера, которые подставляются из профиля вызывающего пользователя, чтобы каждый пользователь проходил аутентификацию под своей собственной учетной записью, а не через общую учетную запись сервиса.

Учетные данные, предоставленные пользователем, хранятся в зашифрованном виде и доступны только тому пользователю, который их ввел. Учетные данные одного пользователя никогда не видны при запуске агентов другим пользователем.

<div id="limits">
  ## Ограничения
</div>

Один запуск агента может ссылаться максимум на 50 различных целевых серверов MCP и не более чем на 100 расширенных конфигураций инструментов в одном запросе. Агенты, которым требуется больше, следует разбивать с помощью [субагентов](/ru/products/cloud/features/ai-ml/agents/builder/subagents).
