Субагент — это дочерний агент, которого родительский агент вызывает для выполнения отдельной подзадачи. Дочерний агент работает в собственном контекстном окне и со своими подключёнными инструментами, а затем возвращает родительскому агенту компактный результат. Родительский агент никогда не видит промежуточные шаги дочернего агента — только итоговый ответ.

Используйте субагентов, чтобы родительский агент не терял фокус. Длительные исследования, работа с большим количеством инструментов и специализированные этапы проверки эффективнее, когда их промежуточное состояние, насыщенное токенами, не попадает в контекст родительского агента.

<div id="enable-subagents">
  ## Включите субагентов
</div>

В Конструкторе агентов откройте **Дополнительные настройки** и включите **Субагентов**. Появятся ещё два элемента управления:

* **Разрешить создание собственных копий** — если параметр включён, агент может создавать свои копии для параллельной работы. Полезно для сценариев fan-out (один родительский агент и несколько одинаковых воркеров).
* **Дополнительные субагенты** — выберите конкретных агентов, которым этот родительский агент может делегировать задачи. Во время выполнения родительский агент сам решает, какого субагента вызвать (если вообще вызывать), в зависимости от задачи.

Сохраните. Когда агент будет запущен, у него появится новый инструмент для вызова субагентов.

<div id="when-a-subagent-helps">
  ## Когда субагент полезен
</div>

* **Исследовательские подзадачи** — родительский агент разбивает сложный вопрос, передаёт часть субагенту, который на ней специализируется, а затем сводит ответы воедино.
* **Сценарии с активным использованием инструментов** — сеансы Code interpreter, многошаговые циклы Веб-поиска и всё, что иначе заполнило бы контекст родительского агента промежуточными результатами работы инструментов.
* **Специализированная проверка** — один агент создаёт результат, другой его проверяет; при этом все рассуждения проверяющего остаются в его собственном контекстном окне.

Не используйте субагентов для тривиальных вызовов. Передача добавляет задержку и расходует токены.

<div id="subagents-vs-agent-chain">
  ## Субагенты vs. цепочка агентов
</div>

Субагенты — это механизм делегирования во время выполнения: родительский агент принимает решение на ходу. Цепочка агентов — это заранее определённый граф агентов, работающих в фиксированном порядке. Используйте субагентов, когда маршрутизация динамическая; используйте цепочку, когда маршрутизация фиксирована.

<div id="limits">
  ## Ограничения
</div>

* Не более **10** субагентов на одного родительского агента.
* Максимальная глубина — **5** переходов от корневого агента.
* Не более **50** уникальных целевых субагентов, загружаемых за один запуск.
* Не более **100** развёрнутых конфигураций субагентов на один запрос.

Субагенты, на которые вы ссылаетесь, должны быть видны пользователю, запускающему родительский агент. Ссылки, ведущие к недоступному или удалённому агенту, пропускаются без уведомления — проектируйте родительские агенты так, чтобы они корректно обрабатывали отсутствие дочернего агента.
