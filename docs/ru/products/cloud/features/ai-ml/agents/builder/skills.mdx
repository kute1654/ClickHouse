Навык — это набор инструкций для повторного использования, который агент может применять по мере необходимости. Используйте навыки для процедур, которые повторяются у разных агентов, — например, для руководства по фирменному стилю, чек-листа для ревью кода или runbook для конкретного рабочего процесса, — вместо того чтобы дублировать эти инструкции в системном промпте каждого агента.

<div id="anatomy-of-a-skill">
  ## Анатомия навыка
</div>

Навык — это Markdown-файл с небольшим блоком frontmatter:

```markdown
---
name: revenue-report
description: Generates the weekly revenue report using our standard segments
always-apply: false
user-invocable: true
---

When asked to generate a revenue report:
1. Filter to the requested period.
2. Apply the standard MRR formula:
     SUM(CASE
       WHEN billing_cycle = 'monthly' THEN amount
       WHEN billing_cycle = 'yearly'  THEN amount / 12
       ELSE 0
     END)
3. Break down by segment: Enterprise, Mid-Market, SMB.
4. Render the result as a Markdown table.
```

Наиболее важные параметры frontmatter:

* **`name`** — идентификатор в kebab-case.
* **`description`** — краткое описание, по которому модель определяет, в каких случаях этот навык уместен. Считайте это самым важным полем. Пишите как можно конкретнее: расплывчатые описания приводят к вызову не того навыка.
* **`always-apply`** — если значение `true`, навык добавляется в каждый ход, а не выбирается по необходимости. Используйте с осторожностью: навыки с `always-apply` расходуют контекст в каждом сообщении.
* **`user-invocable`** — если значение `true` (по умолчанию), навык отображается во всплывающем меню `$` для ручного выбора.

Вы можете включить в пакет вместе с навыком вспомогательные файлы — справочную документацию, примеры запросов, небольшие скрипты, — загрузив `.zip`-архив с `SKILL.md` и связанными с ним ресурсами.

<div id="use-a-skill">
  ## Использование навыка
</div>

Агент может задействовать навык в диалоге тремя способами:

* **Вызов пользователем** — нажмите `$` в поле ввода и выберите навык во всплывающем меню. Содержимое навыка будет подготовлено для следующего хода.
* **Автовыбор моделью** — на основе `description` навыка агент сам решает, когда его применять.
* **Всегда применять** — для навыков, настроенных таким образом, содержимое подготавливается на каждом ходу.

<div id="manage-skills">
  ## Управление навыками
</div>

Панель навыков в консоли Cloud позволяет создавать навыки прямо в интерфейсе, загружать файлы `.md` или `.zip` и управлять тем, какие навыки активны для вашей учетной записи. Ваши навыки по умолчанию активны; деактивируйте навык, чтобы убрать его из всплывающего меню и каталога модели, не удаляя его.

Навыками можно делиться с другими пользователями (см. [совместное использование и доступ](/ru/products/cloud/features/ai-ml/agents/sharing-and-access)).

<div id="skills-vs-instructions">
  ## Навыки и инструкции
</div>

* **Инструкции агента** определяют, что представляет собой агент и как он ведет себя в целом. Они всегда действуют для этого агента.
* **Навыки** зависят от ситуации: они применяются, когда это уместно, и ограничены конкретными рабочими процессами.

Используйте навык, если один и тот же набор пошаговых инструкций постоянно повторяется у нескольких агентов или если вы хотите, чтобы он срабатывал только для определённых запросов пользователя, а не на каждом шаге.
