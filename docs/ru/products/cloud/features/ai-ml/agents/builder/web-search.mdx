Веб-поиск позволяет агенту получать информацию из общедоступного интернета во время диалога. Используйте его для вопросов, где ответ должен быть актуальным: недавние релизы, документация за пределами вашего сервиса или быстрая проверка по авторитетному источнику.

<div id="enable-it">
  ## Включение
</div>

В Конструкторе агентов включите **Веб-поиск** в разделе **Возможности**. После этого агент сам решает, когда выполнять поиск, исходя из вопроса пользователя и своих инструкций. Затем выполняется поиск, из результатов извлекается содержимое, а наиболее релевантный контент передается обратно в контекст модели.

<div id="how-a-search-round-works">
  ## Как работает цикл поиска
</div>

Каждый цикл поиска в Cloud проходит в три этапа:

1. **Поиск** — запрос агента отправляется поисковому провайдеру, который возвращает URL-адреса-кандидаты.
2. **Извлечение** — загружаются релевантные страницы, и из них извлекается значимый текст.
3. **Повторное ранжирование** — reranker оценивает результаты, чтобы модель сначала увидела самые полезные.

В ответе агента приводятся URL-адреса, которые он действительно использовал.

<div id="when-to-use-it">
  ## Когда это нужно
</div>

* Найдите примечания к релизу или журналы изменений, которых нет в вашем сервисе.
* Проверьте факт по источнику, который может быть неизвестен модели.
* Добавьте в диалог для анализа общедоступный пост в блоге или документ.

Не используйте это для вопросов, на которые можно ответить по вашим данным или на основе собственных знаний модели. Каждый цикл поиска добавляет задержку.
