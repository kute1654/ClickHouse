<div id="how-it-works">
  ## Как работает память
</div>

Небольшой агент памяти работает параллельно с основным диалогом. Он читает недавние сообщения, решает, что стоит запомнить, и сохраняет эти сведения в персональное хранилище пользователя.
В следующем диалоге эти записи доступны как контекст, к которому основной агент может обращаться, и вам не придётся повторяться.

Для вас это выглядит как последовательность: достаточно один раз сказать агенту, что вы предпочитаете вывод SQL в нижнем регистре и что ваш финансовый год заканчивается в марте, — и в будущих диалогах он будет учитывать это.

<div id="manage-your-memories">
  ## Управление своими записями памяти
</div>

Откройте панель памяти с помощью значка **Memories** (мозг) в левой навигации. На панели отображаются сохранённые записи памяти, а также элементы управления для создания, редактирования, удаления и фильтрации записей.

<Image img="/images/cloud/agent-builder/memory/memories.webp" alt="Панель Memories, где в левой навигации выделен значок мозга, показаны поле фильтра, кнопка Add, флажок Use memory, запись памяти с элементами управления редактированием и удалением, а также кнопка Admin Settings" size="sm" />

Memory доступна только вашему пользователю. Агенты других пользователей никогда не увидят ваши записи, а ваши агенты — их записи.

<div id="create-memory">
  ### Создание записи в Memory
</div>

Нажмите кнопку **+** в верхней части панели, чтобы открыть диалог **Create Memory**. Введите **Key** (только строчные буквы и символы подчеркивания) и **Value**, затем нажмите **Create**.

<Image img="/images/cloud/agent-builder/memory/create.webp" alt="Панель Memory с выделенной кнопкой + для Create Memory" size="sm" />

<div id="filter-memories">
  ### Фильтрация записей
</div>

Используйте поле **Filter memories** в верхней части панели, чтобы найти запись по ключу.

<Image img="/images/cloud/agent-builder/memory/filter.webp" alt="Панель Memory с выделенным полем Filter memories и введённым значением 'demo'" size="sm" />

<div id="edit-memory">
  ### Изменение записи Memory
</div>

Нажмите значок карандаша рядом с записью Memory, чтобы открыть диалоговое окно **Редактирование Memory**. Измените ключ или значение и нажмите **Сохранить**.

<Image img="/images/cloud/agent-builder/memory/edit.webp" alt="Запись Memory с выделенным значком карандаша «Редактирование Memory»" size="sm" />

<div id="delete-memory">
  ### Удалить запись Memory
</div>

Нажмите на значок корзины рядом с записью Memory, чтобы удалить её.

<Image img="/images/cloud/agent-builder/memory/delete.webp" alt="Запись Memory с выделенным значком корзины для удаления" size="sm" />

<div id="toggle-memory">
  ## Включение и выключение памяти
</div>

Включайте или выключайте память с помощью флажка **Использовать память** в верхней части панели памяти. Отключайте её для конфиденциальных тем, которые не должны сохраняться, или для разовых диалогов, где персонализация не требуется.

Когда память выключена, агент не читает данные из хранилища памяти и не записывает их туда.

<Image img="/images/cloud/agent-builder/memory/toggle.webp" alt="Панель памяти с выделенным в верхней части флажком «Использовать память»" size="sm" />

<div id="memory-best-practices">
  ## Рекомендации по использованию Memory
</div>

Memory может быть полезен, когда нужно хранить:

* Повторяющиеся договорённости: предпочтительные форматы дат, бизнес-определения, схемы именования.
* Контекст проекта: к какому сервису или базе данных вы обычно обращаетесь, какие панели мониторинга для вас важны.
* Стиль общения: лаконичные или более разговорные ответы, упор на код или на связный текст.

Memory не предназначен для использования в качестве базы данных. Например, это не место для хранения больших объёмов справочных материалов.
Для этого лучше использовать [навык](/ru/products/cloud/features/ai-ml/agents/builder/skills) или встроить этот материал в инструкции агента.
Memory также не предназначен для поиска по прошлым чатам; эту роль выполняет сама история диалога.
