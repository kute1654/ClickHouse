<div id="overview">
  ## Обзор
</div>

Это руководство предоставляет корпоративным командам информацию о возможностях мониторинга и обсервабилити для промышленных развертываний ClickHouse Cloud. Корпоративные клиенты часто спрашивают о встроенных возможностях мониторинга, интеграции с существующими стеками обсервабилити, включая такие инструменты, как Datadog и AWS CloudWatch, а также о том, как мониторинг ClickHouse соотносится с самоуправляемыми развертываниями.

Пользователи могут использовать следующие способы для мониторинга своего развертывания ClickHouse:

| ------------------------------------------------------------------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------ | -------------------------------- |
| [Панели мониторинга Cloud Console](/ru/products/cloud/features/monitoring/cloud-console)      | Повседневный мониторинг с помощью встроенных панелей мониторинга состояния сервиса, использования ресурсов и производительности запросов | Нет                                        | Нет                              |
| [Уведомления](/ru/products/cloud/features/monitoring/notifications)                           | Оповещения о событиях масштабирования, ошибках, мутациях и выставлении счетов                                                            | Нет                                        | Нет (можно настроить)            |

<div id="quick-start">
  ## Быстрый старт
</div>

Откройте консоль ClickHouse Cloud и перейдите на вкладку **Monitoring**. В этом [блоге](https://clickhouse.com/blog/common-getting-started-issues-with-clickhouse) собраны распространённые моменты, на которые стоит обратить внимание в начале работы.

Для большинства пользователей [панели мониторинга Cloud Console](/ru/products/cloud/features/monitoring/cloud-console) предоставляют всё необходимое для отслеживания состояния сервиса, использования ресурсов и производительности запросов без дополнительной настройки. Если вам нужна интеграция с внешним стеком мониторинга, начните с [конечной точки метрик, совместимой с Prometheus](/ru/products/cloud/features/monitoring/prometheus).

<div id="system-impact">
  ## Влияние на систему
</div>

В описанных выше подходах используются конечные точки Prometheus, возможности ClickHouse Cloud или прямые [запросы к системным таблицам](/ru/products/cloud/features/monitoring/system-tables). Последний вариант предполагает выполнение запросов к рабочему сервису ClickHouse, что создает дополнительную нагрузку на наблюдаемую систему и не позволяет экземплярам ClickHouse Cloud [переходить в режим бездействия](/ru/products/cloud/features/autoscaling/overview), а это может повлиять на затраты. Кроме того, если рабочая система выйдет из строя, мониторинг тоже может пострадать, поскольку они тесно связаны.

Прямые запросы к системным таблицам хорошо подходят для глубокой интроспекции и отладки, но хуже подходят для мониторинга рабочей среды в реальном времени. И [панели мониторинга Cloud Console](/ru/products/cloud/features/monitoring/cloud-console), и [конечная точка Prometheus](/ru/products/cloud/features/monitoring/prometheus) используют заранее собранные метрики, которые не выводят бездействующие сервисы из режима простоя, поэтому они лучше подходят для постоянного мониторинга рабочей среды. Учитывайте этот компромисс между возможностями детального анализа системы и операционными накладными расходами.
