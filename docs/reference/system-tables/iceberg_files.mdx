## Description {#description}

This system table contains per-file metadata for Iceberg tables existing in ClickHouse, with one row per data or delete file referenced by the current snapshot of each table. It will be empty if you don't have any Iceberg tables in ClickHouse.

## Columns {#columns}

{/*AUTOGENERATED_START*/}
- `database` ([String](/reference/data-types/index)) — Database name.
- `table` ([String](/reference/data-types/index)) — Table name.
- `snapshot_id` ([Int64](/reference/data-types/index)) — Snapshot ID at which the file was added.
- `content` ([Enum8('DATA' = 0, 'POSITION_DELETE' = 1, 'EQUALITY_DELETE' = 2)](/reference/data-types/index)) — File content kind.
- `file_path` ([String](/reference/data-types/index)) — Resolved storage path of the file.
- `file_format` ([String](/reference/data-types/index)) — File format, e.g. 'PARQUET'.
- `record_count` ([Int64](/reference/data-types/index)) — Number of records in the file.
- `file_size_in_bytes` ([Int64](/reference/data-types/index)) — Size of the file in bytes.
- `partition` ([String](/reference/data-types/index)) — Textual representation of the partition tuple.
- `schema_id` ([Int32](/reference/data-types/index)) — Schema ID resolved for this manifest entry.
- `sequence_number` ([Int64](/reference/data-types/index)) — Resolved sequence number of the manifest entry (always 0 for format v1).
- `sort_order_id` ([Nullable(Int32)](/reference/data-types/index)) — Sort order ID of the file, if specified.
- `null_value_counts` ([Map(Int32, Int64)](/reference/data-types/index)) — Per-column null value count (column id -> count).
- `column_sizes` ([Map(Int32, Int64)](/reference/data-types/index)) — Per-column on-disk size in bytes (column id -> bytes).
- `value_counts` ([Map(Int32, Int64)](/reference/data-types/index)) — Per-column total value count (column id -> count).
- `equality_ids` ([Array(Int32)](/reference/data-types/index)) — Equality field IDs for equality delete files (empty for non-equality-delete files).
{/*AUTOGENERATED_END*/}
