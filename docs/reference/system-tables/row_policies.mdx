## Description {#description}

Contains filters for one particular table, as well as a list of roles and/or users which should use this row policy.

## Columns {#columns}

{/*AUTOGENERATED_START*/}
- `name` ([String](/reference/data-types/index)) — Name of a row policy.
- `short_name` ([String](/reference/data-types/index)) — Short name of a row policy. Names of row policies are compound, for example: myfilter ON mydb.mytable. Here 'myfilter ON mydb.mytable' is the name of the row policy, 'myfilter' is it's short name.
- `database` ([String](/reference/data-types/index)) — Database name.
- `table` ([String](/reference/data-types/index)) — Table name. Empty if policy for database.
- `id` ([UUID](/reference/data-types/index)) — Row policy ID.
- `storage` ([String](/reference/data-types/index)) — Name of the directory where the row policy is stored.
- `select_filter` ([Nullable(String)](/reference/data-types/index)) — Expression which is used for filtering in SELECT queries.
- `is_restrictive` ([UInt8](/reference/data-types/index)) — Shows whether the row policy restricts access to rows. Value: • 0 — The row policy is defined with `AS PERMISSIVE` clause, • 1 — The row policy is defined with AS RESTRICTIVE clause.
- `apply_to_all` ([UInt8](/reference/data-types/index)) — Shows that the row policies set for all roles and/or users.
- `apply_to_list` ([Array(String)](/reference/data-types/index)) — List of the roles and/or users to which the row policies is applied.
- `apply_to_except` ([Array(String)](/reference/data-types/index)) — The row policies is applied to all roles and/or users excepting of the listed ones.
{/*AUTOGENERATED_END*/}
