## Description {#description}

Contains error codes with the number of times they have been triggered.

To show all possible error codes, including ones which were not triggered, set setting [system_events_show_zero_values](/reference/settings/session-settings#system_events_show_zero_values) to 1.

## Columns {#columns}

{/*AUTOGENERATED_START*/}
- `name` ([String](/reference/data-types/index)) — Name of the error (errorCodeToName).
- `code` ([Int32](/reference/data-types/index)) — Code number of the error.
- `value` ([UInt64](/reference/data-types/index)) — The number of times this error happened.
- `last_error_time` ([DateTime](/reference/data-types/index)) — The time when the last error happened.
- `last_error_message` ([String](/reference/data-types/index)) — Message for the last error.
- `last_error_format_string` ([String](/reference/data-types/index)) — Format string for the last error.
- `last_error_trace` ([Array(UInt64)](/reference/data-types/index)) — A stack trace that represents a list of physical addresses where the called methods are stored.
- `remote` ([UInt8](/reference/data-types/index)) — Remote exception (i.e. received during one of the distributed queries).
- `query_id` ([String](/reference/data-types/index)) — Id of a query that caused an error (if available).
{/*AUTOGENERATED_END*/}

<Note>
Counters for some errors may increase during successful query execution. It's not recommended to use this table for server monitoring purposes unless you are sure that corresponding error can not be a false positive.
</Note>

## Example {#example}

```sql title="Query"
SELECT name, code, value
FROM system.errors
WHERE value > 0
ORDER BY code ASC
LIMIT 1

┌─name─────────────┬─code─┬─value─┐
│ CANNOT_OPEN_FILE │   76 │     1 │
└──────────────────┴──────┴───────┘
```

```sql title="Response"
WITH arrayMap(x -> demangle(addressToSymbol(x)), last_error_trace) AS all
SELECT name, arrayStringConcat(all, '\n') AS res
FROM system.errors
LIMIT 1
SETTINGS allow_introspection_functions=1\G
```
