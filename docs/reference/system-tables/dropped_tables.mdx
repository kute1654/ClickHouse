## Description {#description}

Contains information about tables that drop table has been executed on but for which data cleanup has not yet been performed.

## Columns {#columns}

{/*AUTOGENERATED_START*/}
- `index` ([UInt32](/reference/data-types/index)) — Index in marked_dropped_tables queue.
- `database` ([String](/reference/data-types/index)) — Database name.
- `table` ([String](/reference/data-types/index)) — Table name.
- `uuid` ([UUID](/reference/data-types/index)) — Table UUID.
- `engine` ([String](/reference/data-types/index)) — Table engine name.
- `metadata_dropped_path` ([String](/reference/data-types/index)) — Path of table's metadata file in metadata_dropped directory.
- `table_dropped_time` ([DateTime](/reference/data-types/index)) — The time when the next attempt to remove table's data is scheduled on. Usually it's the table when the table was dropped plus `database_atomic_delay_before_drop_table_sec`.
{/*AUTOGENERATED_END*/}

## Example {#example}

The following example shows how to get information about `dropped_tables`.

```sql
SELECT *
FROM system.dropped_tables\G
```

```text
Row 1:

table_dropped_time:    2023-03-16 23:43:31
```
