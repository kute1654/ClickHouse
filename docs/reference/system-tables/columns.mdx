## Description {#description}

Contains information about columns in all tables.

You can use this table to get information similar to the [DESCRIBE TABLE](/reference/statements/describe-table) query, but for multiple tables at once.

Columns from [temporary tables](/reference/statements/create/table#temporary-tables) are visible in the `system.columns` only in those session where they have been created. They are shown with the empty `database` field.

## Columns {#columns}

{/*AUTOGENERATED_START*/}
- `database` ([String](/reference/data-types/index)) — Database name.
- `table` ([String](/reference/data-types/index)) — Table name.
- `name` ([String](/reference/data-types/index)) — Column name.
- `type` ([String](/reference/data-types/index)) — Column type.
- `position` ([UInt64](/reference/data-types/index)) — Ordinal position of a column in a table starting with 1.
- `default_kind` ([String](/reference/data-types/index)) — Expression type (DEFAULT, MATERIALIZED, ALIAS) for the default value, or an empty string if it is not defined.
- `default_expression` ([String](/reference/data-types/index)) — Expression for the default value, or an empty string if it is not defined.
- `data_compressed_bytes` ([UInt64](/reference/data-types/index)) — The size of compressed data, in bytes.
- `data_uncompressed_bytes` ([UInt64](/reference/data-types/index)) — The size of decompressed data, in bytes.
- `marks_bytes` ([UInt64](/reference/data-types/index)) — The size of marks, in bytes.
- `comment` ([String](/reference/data-types/index)) — Comment on the column, or an empty string if it is not defined.
- `is_in_partition_key` ([UInt8](/reference/data-types/index)) — Flag that indicates whether the column is in the partition expression.
- `is_in_sorting_key` ([UInt8](/reference/data-types/index)) — Flag that indicates whether the column is in the sorting key expression.
- `is_in_primary_key` ([UInt8](/reference/data-types/index)) — Flag that indicates whether the column is in the primary key expression.
- `is_in_sampling_key` ([UInt8](/reference/data-types/index)) — Flag that indicates whether the column is in the sampling key expression.
- `compression_codec` ([String](/reference/data-types/index)) — Compression codec name.
- `character_octet_length` ([Nullable(UInt64)](/reference/data-types/index)) — Maximum length in bytes for binary data, character data, or text data and images. In ClickHouse makes sense only for FixedString data type. Otherwise, the NULL value is returned.
- `numeric_precision` ([Nullable(UInt64)](/reference/data-types/index)) — Accuracy of approximate numeric data, exact numeric data, integer data, or monetary data. In ClickHouse it is bit width for integer types and decimal precision for Decimal types. Otherwise, the NULL value is returned.
- `numeric_precision_radix` ([Nullable(UInt64)](/reference/data-types/index)) — The base of the number system is the accuracy of approximate numeric data, exact numeric data, integer data or monetary data. In ClickHouse it's 2 for integer types and 10 for Decimal types. Otherwise, the NULL value is returned.
- `numeric_scale` ([Nullable(UInt64)](/reference/data-types/index)) — The scale of approximate numeric data, exact numeric data, integer data, or monetary data. In ClickHouse makes sense only for Decimal types. Otherwise, the NULL value is returned.
- `datetime_precision` ([Nullable(UInt64)](/reference/data-types/index)) — Decimal precision of DateTime64 data type. For other data types, the NULL value is returned.
- `serialization_hint` ([Nullable(String)](/reference/data-types/index)) — A hint for column to choose serialization on inserts according to statistics.
- `statistics` ([String](/reference/data-types/index)) — The types of statistics created in this columns.
{/*AUTOGENERATED_END*/}
